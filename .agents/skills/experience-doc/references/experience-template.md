# 代码级经验文档模板

> 适用于 `docs/plans/归档/经验/` 目录下的文档

---

## 文档结构

```markdown
# {领域}经验

> 从{来源}中提取的{简短描述}经验

---

## 一、{问题/方案名称}

### 问题现象

{1-2 句话描述问题表现}

### 根因分析

{代码对比展示错误写法和正确写法}

### 核心原则

{一句话总结可复用的经验法则}
```

## 格式规范

### 必须包含
- ❌ 错误写法（代码示例）
- ✅ 正确写法（代码示例）
- 根因说明（为什么错）
- 核心原则（一句话总结）

### 禁止包含
- 开发过程记录
- 实施步骤清单
- 测试验证报告
- 时间估算和里程碑

### 命名规则
- 已有分类：追加到对应 `{领域}经验.md`
- 全新领域：新建 `{领域}经验.md`
- 单个文档不超过 200 行，超过时拆分

### 代码示例要求
- 必须可直接复制使用
- 使用项目实际使用的技术栈和 API
- 标注关键注释说明"为什么"而非"做什么"
