# 目录结构规范

## 完整目录树

```
/docs/项目文档/                                  <-- 项目文档知识库根目录
  ├── 项目总览.md                                <-- 全局背景、总目标、核心功能模块概览
  ├── INDEX.md                                  <-- 全局索引：目录大纲，包含所有模块入口的相对路径与简介
  │
  ├── /[功能模块名称]/                             <-- 按业务边界划分的子文件夹（如：/创作端/01-用户鉴权/）
  │   ├── INDEX.md 或 README.md                  <-- 模块索引：维护当前模块下所有需求与实现文档的链接
  │   ├── [具体业务功能]_需求文档.md                <-- 需求文档必须放在模块根目录
  │   └── /技术/                                 <-- 技术文档专属子目录
  │       ├── [对应功能]_实现设计.md              <-- 技术实现文档
  │       └── 数据库模型设计.md                   <-- （可选）模块内全局技术设计
  └── ...
```

## 模块命名约定

- 格式：`{序号}-{模块中文名}`，如 `01-用户鉴权`、`02-Demo管理`
- 序号用于排序，两位数字起步
- 模块名应直观反映业务边界，Agent 可通过名称推断内容

## 文件命名约定

| 文件类型 | 命名格式 | 示例 |
|----------|----------|------|
| 需求文档 | `{业务功能}_需求文档.md` | `用户鉴权_需求文档.md` |
| 实现文档 | `{序号}_{功能点}_实现设计.md` | `01_认证架构设计.md` |
| 模块索引 | `INDEX.md` | 固定名称 |
| 全局需求 | `全局总需求文档.md` | 固定名称 |
| 全局索引 | `INDEX.md` | 固定名称 |

## INDEX.md 格式

### 全局 INDEX.md（docs/项目文档/INDEX.md）

```markdown
# 项目文档索引

## 创作端
- [01-用户鉴权](./创作端/01-用户鉴权/) — 用户登录、注册、权限管理
- [02-Demo管理](./创作端/02-Demo管理/) — 页面模块的增删改查

## 使用端
- [01-项目浏览](./使用端/01-项目浏览/) — 项目列表与导航

## 独立服务层
- [独立Agent服务层](./独立Agent服务层/) — ACP 协议与 Agent 管理
```

### 模块 INDEX.md（docs/项目文档/[模块]/INDEX.md）

```markdown
# {模块名称}

## 需求文档
- [{业务功能}需求](./{功能}_需求文档.md)

## 技术文档
- [01 架构设计](./技术/01_架构设计.md)
- [02 具体实现](./技术/02_具体实现设计.md)
```

## 新模块创建检查清单

创建新功能模块时，确保以下结构完整：

1. `docs/项目文档/{序号}-{模块名}/` 或对应业务域下的模块目录已创建
2. 模块目录内 `INDEX.md` 或 `README.md` 已创建
3. 至少一个需求文档已创建
4. 全局 `docs/项目文档/INDEX.md` 已更新，添加新模块入口
