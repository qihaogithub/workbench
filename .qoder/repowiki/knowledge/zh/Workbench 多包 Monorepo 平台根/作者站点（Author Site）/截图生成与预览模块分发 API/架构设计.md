本模块由三个 Next.js App Router API Route 子目录组成，职责清晰分层：
- `screenshots/`：对外暴露 `/api/screenshots/*` 路由集，其中 `generate`、`generate-batch`、`status`、`cancel`、`health` 均为薄代理层，统一通过 `@/lib/screenshot-service` 中的 `fetchScreenshotService` 转发到外部截图服务，并集中处理超时/不可用错误；`ensure` 是聚合编排入口，扫描 `data/projects/{projectId}/workspace/demos` 下的页面源码与 schema，按 `high-fidelity-react` / `prototype-html-css` / `sketch-scene` 三种 runtimeType 构造批量截图任务，再异步调用截图服务的 batch 端点；`file/[projectId]/[pageId]` 优先从本地 `data/screenshots/{projectId}/{pageId}.{hash}.png` 读取，若远程服务返回 404 则回退到本地磁盘，同时根据 `.meta.json` 中的 `currentHash` / `variants` 解析当前版本，并通过 `isLikelyBlankScreenshot` 过滤空图。
- `preview-modules/[hash]/route.ts`：基于 `@/lib/preview-module-store` 以只读方式按 hash 分发已编译的预览 JS 模块，带强缓存与 `X-Content-Type-Options: nosniff`。
- `preview-runtime/shell/route.ts`：通过 `@workbench/demo-ui/iframe-template` 的 `generateIframeHtml` 动态拼装 iframe 宿主 HTML，依据 `shouldUsePreviewRuntimeCdn()` 与查询参数 `runtimeSource=cdn` 决定加载 CDN 还是本地 runtime。
依赖方向：所有 route 仅依赖 `@/lib/*` 工具包与 `@workbench/shared`、`@workbench/demo-ui` 类型/模板，不反向引用上层业务逻辑，保持纯 API 边界。
