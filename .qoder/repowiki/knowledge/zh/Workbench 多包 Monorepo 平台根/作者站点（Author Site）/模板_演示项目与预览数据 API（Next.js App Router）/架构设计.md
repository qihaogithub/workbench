模块按资源划分为三个并列的 Next.js Route Handler 目录：
- `api/templates/`：模板列表与元信息更新、删除、封面生成、模板转项目；`[id]/convert`、`[id]/cover` 作为子路由暴露动作。
- `api/demos/`：演示项目（即普通项目）的增删改查与复制、封面、关联模板等子动作，同样以 `[id]` + 子路由组织。
- `api/viewer/[projectId]/data`：聚合读取某个项目的 demo pages、配置 schema/values、画布状态与应用图，供预览端一次性拉取。

所有 handler 仅做参数校验与错误包装，业务逻辑委托给单例 `getProjectAdminService()`（来自 `@/lib/project-admin-service`），统一通过 `projectAdminResponse(result)` 将 `{ ok, data, error }` 结果映射为 HTTP 响应；文件 I/O 由 `@/lib/fs-utils` 中的 `createApiSuccess` / `createApiError` / `readProjectMeta` / `listDemoPages` 等工具函数完成。依赖方向单向：route → project-admin-service → fs-utils → 磁盘；viewer/data 还额外依赖 `@/lib/canvas-layout-file` 与 `@/lib/preview-size`。
