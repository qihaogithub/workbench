- 每个 handler 入口先调用 `getAuthCookie()` + `verifyToken()` 解析用户身份，缺失或过期时返回 `UNAUTHORIZED` 401。
- 写操作在修改前用 `crypto.createHash('sha256')` 计算 `previousContent` 的 SHA-256，作为 `WorkspaceMutationOperation.expectedHash` 传入 Authority 做乐观锁。
- live workspace 与非 live workspace 采用 `if (isLiveWorkspacePath(wsPath)) { ... } else { 直接 fs 写入 }` 的双分支模式，并在注释中明确说明差异。
- 所有响应统一通过 `createApiSuccess(data)` / `createApiError(code, message, meta?)` 包装，再由 `NextResponse.json(...)` 返回，避免手写 JSON 结构。
- Session 上下文通过 `resolveSessionWorkspace` 辅助函数集中处理：从 URL query 或 body 提取 `sessionId`，校验存在性、归属、过期及 `workspaceId` 绑定，失败即短路返回对应错误响应。
- 文件夹层级深度限制统一通过 `getFolderDepth` 检查，超过 3 层返回 `FOLDER_DEPTH_EXCEEDED` 400。
