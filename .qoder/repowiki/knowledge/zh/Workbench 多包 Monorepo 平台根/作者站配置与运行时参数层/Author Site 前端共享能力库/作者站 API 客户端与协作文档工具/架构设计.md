模块按职责拆分为四个文件：
- `api.ts`：基于 SWR 的 REST 客户端，暴露 `useDemos`、`useProjectTemplates` 等 React Hook 及 `createDemo`、`updateDemo`、`deleteDemo`、`uploadCover` 等异步函数；所有响应统一走 `ApiResponse<T>` 包装并通过 `mutate('/api/...')` 触发缓存失效。
- `mock-api.ts`：同构的内存 Mock 实现（`getMockDemos`、`createMockDemo`、`deleteMockDemo`），返回相同 `ApiResponse<T>` 签名，供开发环境或测试替换真实 fetch。
- `fs-utils.ts`：Node.js 侧的文件系统抽象层，以 `DATA_DIR`、`PROJECTS_DIR`、`TEMPLATES_DIR`、`SESSIONS_DIR`、`WORKSPACES_DIR`、`SNAPPTS_DIR` 环境变量驱动目录布局；维护 workspace-tree.json + app.graph.json 的统一清单格式，并内置从旧 `.folders.json` + `.demo.json` 格式的迁移逻辑；同时导出 Demo 页面 slug/routeKey 生成、模板保存、会话路径查找等工具。
- `hooks/useCollabDocument.ts`：基于 yjs + y-websocket 的协同一等公民 Hook，通过 `NEXT_PUBLIC_COLLAB_WS_URL` / `NEXT_PUBLIC_AGENT_SERVICE_URL` 解析 WebSocket 端点，使用 awareness 广播用户 presence，并提供 `flush` 将本地 Y.Text 落盘到 Agent 的 `/api/collab/.../flush` HTTP 接口；另导出 `useCollabPresence` 作为只读 presence 视图。
- `utils.ts`：仅封装 `clsx` + `tailwind-merge` 的 `cn` 类名合并工具。
依赖方向：`api.ts` → `@workbench/shared` 类型；`fs-utils.ts` → `@workbench/shared` + `@workbench/knowledge-service`；`useCollabDocument.ts` → `yjs` + `y-websocket` + `@workbench/shared`；`mock-api.ts` 与 `api.ts` 共享 `ApiResponse<T>` 契约以便互换。
