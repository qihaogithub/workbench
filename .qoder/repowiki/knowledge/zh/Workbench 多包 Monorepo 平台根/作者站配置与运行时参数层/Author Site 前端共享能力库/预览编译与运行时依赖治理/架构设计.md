本模块围绕「预览页面」的编译-运行链路组织，按职责拆分为若干独立文件：
- 编译入口：`compiler.ts`（Node 端）通过 `@workbench/preview-contract/compiler` 调用 `compilePreviewPageSource`，结合 `preview-dependency-policy` 把 npm import 重写为 esm.sh CDN URL，并维护 LRU 内存缓存；`compiler-client.ts` 是浏览器侧薄封装，仅负责 POST `/api/compile` 并做本地缓存。
- 依赖策略层：`preview-dependency-policy.ts` 集中实现 `getPreviewDependencyUrl`，对 react/react-dom/framer-motion/lucide-react/svgaplayerweb 等包走固定版本 ESM_SH_BASE 拼接，`@preview/sdk` 则内联 data:URL 注入；所有未登记依赖抛出 `PreviewRuntimeContractError`，形成白名单式安全边界。
- 运行时清单：`preview-runtime-manifest.ts` 读取 `public/preview-runtime/manifest.json` 生成 import map，支持 `baseUrl` 前缀与 `PREVIEW_RUNTIME_SOURCE=cdn` 回退到 CDN。
- 模块存储：`preview-module-store.ts` 将编译产物以 hash.js 形式落盘到 `${dataDir}/preview-modules`，配合内存 Map + TTL 清理，供后续加载。
- 辅助工具：`preview-page-code.ts` 在多页模式下选择最终源码；`rewrite-local-paths.ts` 把相对图片路径改写为 `/api/sessions/:sessionId/workspace/...`；`sandpack-deps.ts` 从 `// @dependency` 注释提取声明；`canvas-layout-file.ts` 解析 `.canvas-layout.json` 为 `CanvasState`；`preview-size.ts` 透传 `@workbench/demo-ui` 的尺寸能力。
- 运行时契约校验：`preview-validation.ts` 委托 `ProjectAdminService.validateWorkspacePathRuntime` 并补全 `file` 字段。
- 投影追踪：`preview-projection-tracker.ts` 维护 active-preview / canvas-preview / screenshot 三个 surface 的 appliedRevision 与 invalidated 标志，通过可插拔 `SurfaceInvalidationStrategy` 决定哪些资源变更触发失效，并提供 `ackPreview/failPreview/resetFromSnapshot` 状态机接口。
- 自愈防护：`auto-preview-repair-guard.ts` 基于指纹在 localStorage 中记录自动修复尝试次数，带 TTL 与版本迁移，防止无限重试。
依赖方向单向向外：本模块依赖 `@workbench/preview-contract/*`、`@workbench/project-core`、`@workbench/demo-ui`、`@workbench/shared/contracts` 及自身 `fs-utils`/`cdn-config`，不被其他同级模块反向依赖。
