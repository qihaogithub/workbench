- 对外暴露的函数采用纯函数风格，副作用（文件系统、网络、localStorage）集中在各自模块内部，上层只接收返回值。
- 所有跨进程/跨环境的数据结构都先定义 TypeScript interface/type，再通过显式读取器（如 `readNumber`/`readString`/`isRecord`）做类型收窄后再构造目标对象。
- 缓存统一采用「Map + 最大条目数 + 淘汰最早键」的 LRU 模式，并在写入前检查 `size >= MAX_*` 后删除 `keys().next().value`。
- 依赖/运行时版本通过独立的 `*_VERSION` 常量（如 `PREVIEW_DEPENDENCY_POLICY_VERSION`、`COMPILE_CLIENT_CACHE_VERSION`、`AUTO_PREVIEW_REPAIR_HISTORY_VERSION`）参与缓存 key 或数据迁移判断。
- 可选字段在构造对象时采用 `...(x === null ? {} : { x })` 的条件展开方式，避免向下游传递 undefined 属性。
