本模块按「来源 → 规范化 → 对外 API」分层组织，所有数据源最终收敛为强类型结构供上层消费：
- 环境层 `runtime-config.ts` / `cdn-config.ts`：集中解析 `NEXT_PUBLIC_*`、`AGENT_SERVICE_URL`、`SCREENSHOT_SERVICE_URL`、`CDN_BASE_URL` 等环境变量，并提供浏览器/服务端双分支的 URL 获取函数。
- 数据库持久层 `db-config.ts`：对 `system_configs` 表做通用 CRUD（read/write/list/init），被 `model-config.ts` 复用；`user-model-config.ts` 和 `user-authoring-preferences.ts` 直接通过 `@/lib/db` 操作 `user_model_configs` / `user_authoring_preferences` 两张用户级 JSON 列表。
- 业务配置聚合层 `model-config.ts`：以 `CONFIG_ID="model_config"` + 1 分钟内存缓存为核心，优先读 DB，失败回退到 `getModelEnvConfig()`，并通过 `normalizeConfig()` 在旧字段（`allowedPrefixes`/`blacklist`/`defaultModelIds`/`nameFilters`）与新字段（`enabledModels`/`autoEnableRules`）之间双向兼容转换。
- Schema 合并层 `config-merge.ts`：从 DemoSchema 提取 `properties.default`、`$demo.orderable`、`$demo.positionable` 生成 `__order`/`__orderH`/`__positions` 元数据；`runtime-props.ts` 在此基础上实现项目级与页面级 Schema default 的合并与冲突检测（抛出 `SchemaConflictError`），并支持基于旧/新默认值的「用户修改过」判定。
- 用户侧配置：`user-model-config.ts` 使用 AES-256-GCM 加密存储 API Key（`MODEL_CONFIG_ENCRYPTION_KEY`→SHA256 派生密钥，版本前缀 `v1:`），对外仅暴露 `SafeUserModelConfig`（`apiKey: ""` + `hasApiKey` 标记）；`user-authoring-preferences.ts` 同理只返回白名单字段。
- 特性开关 `authoring-feature-flags.ts`：纯常量导出，由 `NEXT_PUBLIC_SKETCH_SCENE_AUTHORING_ENABLED` 驱动。
依赖方向单向向下：业务层 → db-config/user-* → db；runtime-config 不依赖任何本地模块。
