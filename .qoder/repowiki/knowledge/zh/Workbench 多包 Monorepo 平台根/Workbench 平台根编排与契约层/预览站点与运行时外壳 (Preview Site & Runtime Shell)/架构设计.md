模块采用 Next.js App Router 的静态导出模式（`next.config.js` 中 `output: 'export'` + `trailingSlash: true`），构建产物为纯静态 HTML/JS/CSS，部署到任意静态托管。

- 路由层：`src/app/layout.tsx` 定义全局 `<html>` 根布局；`src/app/[[...slug]]/page.tsx` 作为唯一 catch-all 页面，通过 `generateStaticParams()` 生成根路径，实际视图由客户端组件 `ViewerApp` 根据 URL 在“项目列表”和“项目预览”之间切换。
- 业务主入口：`src/components/ViewerApp.tsx` 是单一大型 Client Component，内部包含 `ProjectListPage`、`ProjectPreviewPage`、`ScreenshotCover`、`buildTree` 等函数式子组件，集中管理项目数据加载、页面目录树、RJSF 配置合并与持久化（`localStorage`）、AI 助手抽屉等。
- API 抽象层：`src/lib/api.ts` 统一封装对后端数据源（`NEXT_PUBLIC_DATA_BASE`，默认 `http://localhost:3200`）的 JSON 请求（`getProjects`、`getProjectData`、`getDemoSchema`、`getPublishedFileUrl` 等）以及对 AI Agent 服务（`NEXT_PUBLIC_AGENT_SERVICE_URL`）的对话与模型列表调用，并暴露 `PublishedProject` / `PublishedDemoPage` / `ProjectsIndex` 等类型供上层消费。
- 运行时外壳：`src/app/api/preview-runtime/shell/route.ts` 是唯一的 Server Route，调用 `@workbench/demo-ui/iframe-template` 的 `generateIframeHtml` 动态注入 `PREVIEW_RUNTIME_SOURCE`（`local`/`cdn`）、`CDN_BASE_URL`、`PREVIEW_SHELL_MODE` 等环境变量，返回一个隔离 iframe 容器；本地开发时 shell 内依赖 `public/preview-runtime/manifest.json` 提供的 vendor 包映射，生产可通过 `PREVIEW_RUNTIME_CDN_FALLBACK=1` 回退到 CDN。
- UI 组件分层：`src/components/ui/*` 是基于 Radix UI + Tailwind 的原子级基础组件（button/dialog/popover/scroll-area 等），`src/components/demo/index.ts` 聚合 demo 专用能力（`PreviewCanvas`、`SketchPagePreview`、`IframePreviewFrame`、`extractPrototypeConfigBindingKeys` 等），被 `ViewerApp` 组合使用。
- 依赖方向：`app → components → lib/api`；`components` 同时依赖 workspace 内的 `@workbench/demo-ui`、`@workbench/sketch-core`、`@workbench/shared`，并通过 `next.config.js` 的 `transpilePackages` 让 Next.js 直接编译这些 monorepo 包。
