本模块由三个并列的命令实现文件组成，均位于 `OPS/CLI/src/commands/` 下，作为 CLI 顶层命令注册表的叶子实现：
- `system.ts`：`systemCheck` 函数聚合运行时（node/pnpm/tsc）、Agent Service 端口监听与健康检查（`/health`）、项目结构（`packages/agent-service`、`packages/author-site`、`packages/shared`）等维度，组装为 `SystemCheckResult` 类型并通过 `detectIssues` 汇总问题；输出支持 `jsonMode` 走 `outputJson` 或带 chalk 颜色的终端报告。
- `models.ts`：`listModels` 通过 `request<ModelsData>(baseUrl, "/api/llm/models")` 拉取后端模型清单，区分当前模型并格式化展示。
- `websocket-stream.ts`：`testWebSocketStream` 将 baseUrl 前缀从 http 切换为 ws，连接 `/api/agent/${sessionId}/stream`，按 `StreamEvent.type`（stream/status/thought/plan/tool_call/tool_call_update/permission_request/finish/error）分派处理，内置超时与错误诊断提示。
依赖方向单向指向 `../utils.js`（runCommand/request/createSpinner/showError 等）与 `../types.js`（SystemCheckResult、WebSocketStreamOptions、StreamEvent），不互相引用，各自独立暴露一个 async 导出函数供上层命令路由调用。
