模块按职责拆分为三个源文件并通过 `src/index.ts` 统一重导出：
- `rules.ts`：声明契约版本常量 `PREVIEW_CONTRACT_VERSION` 与受控依赖白名单 `PREVIEW_DEPENDENCY_POLICY`（react/react-dom/lucide-react/framer-motion/svgaplayerweb/@preview/sdk），并提供面向创作端的规则文本生成器。
- `runtime.ts`：核心契约层，基于 TypeScript Compiler API 解析源码 AST，实现导入提取、默认组件推断与自动包装、顶层绑定重复检测、default export 数量限制、return null 风险检测等；对外暴露 `validatePreviewPageSource` / `validateCompiledPreviewModule` 以及对应的 `assert*` 抛错入口，错误统一封装为 `PreviewRuntimeContractError`，每个问题以 `RuntimeContractIssue` 记录 stage/code/severity/message/instruction。
- `compiler.ts`：消费 runtime 的断言函数，使用 sucrase 将 TSX/JSX 转译为 ESNext，再调用 `rewriteImportsWithResolver` 把 NPM 依赖 specifier 替换为运行时 URL，最终产出 `CompileResult`（含 sha256 moduleHash）。
依赖方向单向：`compiler.ts → runtime.ts → rules.ts`，`index.ts` 仅做 barrel 聚合；`package.json` 通过 `exports` 字段同时向外部暴露 `./rules`、`./runtime`、`./compiler` 子路径。
