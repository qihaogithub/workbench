src/server.ts 是唯一的进程入口，通过工厂/管理器单例将各子模块按启动阶段串联：
- 配置/日志阶段：shared_infrastructure 的 loadConfig/getLogger 先加载 .env 与 pino。
- 后端供应商阶段：backends_providers 的 getBackendProvidersManager().initialize() 解析 PI_AGENT_PROVIDERS 注册多模型。
- 实例策略校验：workspace_session 的 assertWorkspaceAuthorityInstancePolicy 在监听前校验 Workspace Authority 策略。
- Agent 运行时：core_agent_runtime 的 getAgentFactory 注册 "pi-agent" 类型（PiAgentBackend + BackendAgent），getAgentManager 暴露全局单例。
- 数据恢复：collab/workspace-file-persistence 的 getDefaultDataDir 配合 workspace-authority-startup-recovery 在 HTTP 监听前恢复持久化权限。
- Web 层：routes_api_layer 的 registerRoutes 挂载所有 REST/WebSocket 端点，/health 聚合 agent-manager 计数与 recovery 状态。
- 优雅关停：SIGTERM 回调依次 destroyAll → destroySessionStore → fastify.close，保证会话与工作区资源有序释放。

跨子模块不直接互相 import，而是通过 server.ts 中的工厂/管理器单例（getAgentManager、getAgentFactory、getBackendProvidersManager）以及共享的 data 目录约定进行松耦合协作。
