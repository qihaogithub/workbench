模块按职责拆分为三个并列子目录，通过共享契约 `@workbench/shared/contracts` 与全局单例耦合，依赖方向单向：`collab` → `workspace`（Authority/DraftProvider）→ `session`（仅类型引用 `WorkspaceMeta`），三者不互相直接依赖。

- `workspace/`：以单例 `WorkspaceManager`（`workspace-manager.ts`）为中心，负责在系统临时目录创建/销毁临时工作空间或校验用户指定路径；`utils.ts` 暴露路径归一化、临时名生成等纯函数；`workspace-mutation-authority.ts` 实现进程内唯一写者 `WorkspaceMutationAuthority`，维护 per-workspace 串行队列、文件级 lease、prepared/receipt/journal/backups/staging 目录结构，并通过 `registerCollabDraftProvider` 暴露 DraftProvider 供 collab 层写入；`workspace-authority-instance-policy.ts` 启动时断言仅允许 `single` 实例模式，防止多进程并发写 DATA_DIR。
- `session/`：`session-store.ts` 定义 `ISessionStore` 接口并以 `MemorySessionStore` 实现全局单例，配合 `getSessionStore()` / `destroySessionStore()` 自动启动 5 分钟周期的过期清理定时器；`snapshot-service.ts` 根据 workingDir 是否含 `.git` 分支出 git 命令路径与内存快照路径，提供 diff/stage/discard/reset 等操作；`console-buffer.ts`、`run-log-store.ts`、`session-guard.ts` 分别承载控制台缓冲、运行日志与守卫逻辑。
- `collab/`：`collab-room-manager.ts` 是核心，每个资源一个 `Y.Doc` + `y-protocols/sync` + `awareness` 房间，通过 `ws` 协议收发自定义消息头（0=SYNC,1=AWARENESS,3=QUERY_AWARENESS），按 `COLLAB_SAVE_DEBOUNCE_MS` 防抖落盘，并通过 `WorkspaceFilePersistence` 调用 Authority 提交；`workspace-file-persistence.ts` 封装对 DATA_DIR 的读写与 `validateSession`/`validateWorkspaceSession` 鉴权，同时监听 `onMutationCommitted` 事件反向刷新脏 room 基线，避免外部修改导致的冲突。
