- 每个业务域导出一个 `registerXxxRoutes(fastify)` 函数，由 `index.ts` 统一调用，避免在路由文件中直接操作全局实例。
- REST 处理器一律返回 `{ success: true, data }` 或 `{ success: false, error: { code, message, details? } }` 结构，错误码使用大写蛇形字符串常量。
- 路由定义使用泛型参数声明 `Params` / `Body` / `Querystring` 类型（如 `fastify.post<{ Params: SessionParams; Body: SendMessageBody }>(...)`），结合内联接口提升类型安全。
- 对共享错误进行稳定映射：将领域异常（如 `WorkspaceMutationAuthorityError`）通过 `ERROR_STATUS` 表或 `stableErrorCode` 工具转为标准 `code` 与 HTTP 状态码。
- WebSocket 事件路由采用 `bindAgent/unbindAgent` 成对管理监听器，并在 `destroy()` 中统一清理，防止内存泄漏。
