- 每个 skill 以 `SKILL.md` 为唯一入口，YAML front-matter 中必须声明 `name`、`description`，需要执行外部命令时在 `allowed-tools` 显式列出 Bash 子命令白名单。
- 长工作流拆分为 `references/` 下的独立参考文档，主 SKILL.md 只保留步骤编排与相对路径引用，禁止把全部细节塞进单一文件。
- 对高风险操作采用 preview → execute 两阶段模式（如 ow project/page/folder delete-preview 后再 delete-execute），并在 SKILL.md 中以清单形式固化顺序。
- 跨 skill 协作通过“Use `<skill-name>` when …”文字约定而非硬编码，保持 skill 间松耦合、可单独启用或禁用。
- 涉及本地运行时的 skill（如头脑风暴）将可执行脚本集中放在 `scripts/` 目录下，并提供 start/stop shell 脚本统一生命周期管理。
