- 对外错误统一封装为 `{ ok: false, error: { code, message, recoverable } }` 的 `ProjectAdminResult` 结构，成功则 `{ ok: true, data }`，并通过 `ok()` / `fail()` 辅助函数构造。
- 所有工作区写操作必须经 `WorkspaceResourceRegistry.describe()` 匹配路径白名单并调用 `assertTextWrite` / `assertBinaryWrite`，禁止绕过注册表直接落盘。
- JSON 文件读写统一走 `readJson` / `writeJson` / `jsonBuffer` 三个内部工具，保证缩进一致且目录自动创建。
- 验证结果以 `ValidationIssue[]` 返回，每个 issue 带 `code`、`message`、`severity`（`blocking|warning`）以及可选的 `resourceId` / `pageId` 定位字段。
- CLI 行为通过 `process.argv.includes('--xxx')` 或 `process.env.XXX` 开关控制，而非命令行解析库。
