模块由两个相邻的 npm 包组成，依赖方向单向：`@workbench/project-scaffold` → `@workbench/project-core`。

- `project-core/src/types.ts` 集中定义所有对外契约（`ProjectAdminActor`、`ProjectAdminResult<T>`、`ValidationIssue`、`PageCreateInput` 等），`service.ts` 通过构造函数注入 `WorkspaceMutationPort` 实现与外部工作区的解耦；`config.ts` 暴露一组 `get*` 函数从环境变量/配置读取运行时参数；`workspace-resource-registry.ts` 以 `WorkspaceResourceRegistry` 类形式对 `demos/*`、`assets/*`、`knowledge/*` 等路径做白名单 + 大小限制 + JSON/sketch-scene 校验，是所有持久化写入的统一入口；`cli-prompt.ts` 生成 CLI 帮助文本。
- `project-core/src/index.ts` 作为唯一导出门面，仅重新导出上述子模块的公共 API 和类型。
- `project-scaffold/src/index.ts` 是脚手架导出主逻辑：调用 `ProjectAdminService.exportProjectPackage()` 获取远端产物，组装 `LocalProjectManifest`，再输出 `workbench.project.json`、`package.json`、`scripts/dev-server.mjs` 以及可选的 `.workbench/sync-state.json` / `remote.json`；同时内置纯 JS ZIP 构建器（手动拼 CRC32 + local/central/end header）用于打包；`local-preview-dev-server.ts` 把 Node HTTP 预览服务器脚本以字符串常量形式内联，被 `buildDevServerScript()` 注入到生成的脚手架中，支持 `--check`、`--preview-check`、`--screenshot` 三种模式及 Playwright 截图回退。
- 测试：core 使用 vitest（`vitest.config.ts`），scaffold 通过 `tsx` 直接执行单测文件。
