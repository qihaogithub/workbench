- 所有 spec 文件使用 `*.spec.ts` 命名并按功能域分目录组织，每个用例以英文描述性字符串作为 test title。
- 对异步 UI 变更统一使用 `expect.poll(...)` 轮询断言，而非硬编码 sleep，确保等待 DOM/JSON 最终一致。
- 通过 `page.getByRole` / `getByLabel` / `data-*` 属性定位元素，避免使用 CSS/XPath 选择器。
- 对外部 API 响应统一经 `parseApiResponse` 包装，校验 `{ success, data }` envelope 后再返回 data。
- E2E 创建的项目统一打上 `category=__e2e__` 标签，并通过 `isStaleE2EProject` 按 24h 过期策略清理。
- 登录优先注入 `auth_token` cookie，仅在 `E2E_AUTH_COOKIE_ONLY=false` 或登录接口不可用时回退到 POST `/api/auth/login`。
