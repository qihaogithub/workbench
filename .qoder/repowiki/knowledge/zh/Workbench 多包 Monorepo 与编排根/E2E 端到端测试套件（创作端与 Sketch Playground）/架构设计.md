本模块包含两个独立的 Playwright 工程：
- `test/sketch-playground/`：轻量级工程，通过 `webServer.command = "corepack pnpm --filter @workbench/sketch-playground dev"` 自启动 sketch-playground 子应用（127.0.0.1:3400），在单个 spec 中用鼠标/键盘模拟拖拽、双击编辑、右键菜单、撤销重做等交互，并通过 Dev Data 面板读取 scene JSON 断言状态。
- `test/创作端E2E回归测试/`：面向完整后端服务的 E2E 回归套件，`playwright.config.ts` 以 `workers=1`、`fullyParallel=false` 串行执行，`globalSetup`/`globalTeardown` 配合 `support/e2e-projects.ts` 管理每次 run 的 `runId`、项目注册表 `e2e-projects-<runId>.json` 以及共享项目缓存；`support/e2e-auth.ts` 通过直接读取 `users.db`（better-sqlite3）获取 userId，再用 jose 生成 HS256 JWT 写入 `auth_token` cookie 完成登录，支持回退到 `/api/auth/login` 表单登录。
- 产物输出统一落在各自目录下的 `test-outputs/`（artifacts、test-reports、e2e-run.json 等），根级 `test-results/` 由 Playwright 默认收集。依赖方向：spec → support 工具 → 后端 REST API（`/api/demos`、`/api/auth/*`），不反向依赖业务源码。
