- 包组织：pnpm-workspace.yaml 声明 `packages/*` 与 `OPS/CLI` 两个工作区目录，所有子包均以 `@workbench/*` 命名空间发布；`turbo.json` 定义 build/lint/clean/dev 任务并依赖 `^build` 实现跨包增量构建。
- 运行时拓扑：docker-compose.yml 定义 agent-service(3201)、author-site(3200)、screenshot-service(3202, profile screenshot)、viewer-site(3300) 四个服务，通过环境变量 `AGENT_SERVICE_URL` / `SCREENSHOT_SERVICE_URL` / `NEXT_PUBLIC_*_SERVICE_URL` 注入 HTTP 调用地址，形成 author-site → agent-service + screenshot-service、viewer-site → agent-service 的单向依赖链。
- 开发入口：根 `package.json` 脚本统一封装 `dev:services`(concurrently 启动四服务)、`check:*`(按包执行 typecheck+test)、`test:e2e`(Playwright)、`workspace-authority:*`(ops-cli 命令) 等，作为 monorepo 级唯一入口。
- 契约层：`shared_lib` 提供跨进程共享的类型/API 契约，`preview_contract` 约束预览页面运行期行为，二者共同构成前端站点与后端服务之间的稳定边界。
- 数据持久化：`data_artifacts` 目录通过 compose volume 挂载到 `/app/data`，被 agent-service、author-site、screenshot-service 共享（viewer-site 只读），承载会话、项目、截图、SQLite 用户库等运行时状态。
- 运维与诊断：`ops_cli` 暴露 `ow` 命令与 `diagnostics:*` 子命令，直接对接 Agent Service HTTP/WS API；`infra_docker` 为各子包产出 Dockerfile，配合 `scripts/docker-*` 脚本完成镜像预拉取、深度健康检查等 CI 流程。
