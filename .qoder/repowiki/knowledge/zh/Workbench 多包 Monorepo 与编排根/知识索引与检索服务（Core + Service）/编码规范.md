- 对外 API 以 TypeScript interface 描述输入输出（如 SearchInput、RelatedInput、ReportInput、RetrievalBackend），具体实现类在内部以 class 形式提供默认实现。
- 所有 JSON 读写统一通过模块内 `readJson`/`writeJson` 封装，失败时返回 null 而非抛错，保证索引流程可容错推进。
- 字符串 ID 采用带前缀的命名约定区分来源：`kjob_`（索引作业）、`reading-map:`（阅读地图）、`report_`（报告）、`page-`/`knowledge-`（条目），避免跨域冲突。
- 索引作业状态变更遵循写回最新 job 再返回的模式（running/ready/failed/stale），由调用方负责并发安全，服务层不做锁。
- 模板读取函数（readWorkspacePages/configs/knowledgeDocuments）均先尝试解析对应 JSON 元数据，不存在则回退到目录扫描，保持向后兼容。
