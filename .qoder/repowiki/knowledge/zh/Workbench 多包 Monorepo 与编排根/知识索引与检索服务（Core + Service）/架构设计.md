模块按依赖方向拆为两个子包：
- `@workbench/knowledge-core`：纯类型与无副作用函数库，定义 `KnowledgeItem`、`AccessContext`、`KnowledgeIndexJob`、`KnowledgeReadingMap`、`KnowledgeReport` 等核心接口，并实现权限判定 `canAccessKnowledgeItem`、可见性过滤 `filterKnowledgeItems`、权威排序 `compareKnowledgeItemsByAuthority`、报告构建 `buildKnowledgeReport` 及旧源映射 `mapLegacyKnowledgeSource`；不依赖任何文件系统或外部 IO。
- `@workbench/knowledge-service`：面向工作区磁盘的运行时服务，依赖 core 并通过 `RetrievalBackend` 抽象暴露 search/read/related/report 四个能力；内置 `BasicRetrievalBackend` 以内存数组实现，同时通过 `KnowledgeFileStore` 将索引作业与模板阅读地图持久化到 `${dataDir}/knowledge/{index-jobs,templates}` 目录。

服务层还包含模板扫描管线：`generateTemplateReadingMap` 从 workspace-tree.json / project.config.schema.json / knowledge/manifest.json 解析页面、配置与知识文档，产出 `KnowledgeReadingMap`；`createTemplateIndexJob` + `runBasicTemplateIndexJob` 驱动状态机（pending → running → ready/failed/stale），`enrichTemplateReadingMap` 通过注入的 `KnowledgeOrganizer` 回调对本地摘要做异步增强。搜索侧使用正则分词 `tokenize` 与简单打分 `scoreItem`，并按 trustLevel/sourceType/visibility 优先级排序。
