本模块按职责拆分为三个子目录，彼此之间无相互依赖，统一被上层 core / services 消费：
- `utils/`：纯函数式工具库，不持有状态。
  - `config.ts` 通过 `loadConfig()` 将 `PI_AGENT_*`、`RATE_LIMIT_*`、`SCREENSHOT_SERVICE_URL` 等环境变量映射为强类型 `ServiceConfig`；
  - `logger.ts` 基于 pino + pino-pretty 暴露 `Logger` 接口与单例 `getLogger()`，所有服务通过该接口而非直接调用 pino；
  - `error-utils.ts` 提供 `serializeErrorForLog` / `getErrorMessage`，仅拷贝白名单字段并截断超长字符串，防止敏感信息泄露；
  - `image-attachments.ts` 与 `uploaded-file-attachments.ts` 分别负责 base64/dataURL 规范化校验，以及以 `DATA_DIR/ai-attachments/<sessionId>/<attachmentId>/manifest.json+text.txt` 结构持久化上传文本附件。
- `services/`：轻量业务服务类，面向外部系统或文件系统。
  - `model-catalog-service.ts` 的 `ModelCatalogService` 通过注入的 `IAgentManager` 临时创建 session 探测后端可用模型，返回联合类型 `ModelCatalogResult` 区分成功/连接失败；
  - `image-describer.ts` 的 `ImageDescriber` 以 LRU Map 缓存视觉模型描述结果，超时由 `AbortController` 控制，对外暴露 `VisionDescribeFn` 回调以便注入不同 vision provider；
  - `viewer-ai-context.ts` 是纯文本组装器，读取 workspace 的 `workspace-tree.json`、`knowledge/manifest.json`、`project.config.schema.json`、`memory.md` 及系统知识快照，拼接成 `buildViewerAiPromptContext` 与 `buildViewerAiSystemPrompt` 两个只读提示词。
- `scripts/`：可执行迁移脚本，`workspace-authority-migrate.ts` 作为 CLI 入口解析 `--data-dir/--apply/--json` 等参数后委托给 `../workspace/workspace-authority-migration` 完成幂等迁移。

依赖方向：`services/` → `utils/` → Node 内置 / 第三方库；`scripts/` 仅依赖 `core` 与 `workspace` 层，不反向依赖 services。
