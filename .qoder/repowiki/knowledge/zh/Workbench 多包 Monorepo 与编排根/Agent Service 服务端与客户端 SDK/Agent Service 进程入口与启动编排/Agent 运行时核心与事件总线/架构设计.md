模块按职责分为三层：
- 类型层 `core/types.ts`：集中声明 `AgentConfig`、`AgentStatus`、`ErrorCode`、所有 `AgentEvent` 联合类型及 `EventHandler` 回调签名，是 core 与 events 之间的唯一契约。
- 抽象与实现层 `core/agent.ts` 中的 `BaseAgent extends EventEmitter` 提供状态机（initializing→ready→processing→error→destroyed）、`setStatus` 自动广播 status 事件、`emitEvent` 统一派发 typed event；`core/backend-agent.ts` 的 `BackendAgent` 作为适配器，把任意 `IBackendAdapter` 的 stream/消息/权限/用户选择等回调桥接到 BaseAgent 的事件模型上，并封装 busy/initialized 状态与错误归一化。
- 资源管理层 `core/agent-manager.ts` 的 `AgentManager` 以 Map<sessionId, BaseAgent> 维护多会话实例，负责 toolVersion/toolMode 变更时的重建策略、空闲超时清理（60s 轮询 + unref timer）以及全局单例 `getAgentManager()`；`core/agent-factory.ts` 的 `AgentFactory` 通过 `register(type, creator)` 注册具体 Agent 类型，当前仅硬编码 `pi-agent`，同样暴露全局单例 `getAgentFactory()`。
- 事件层 `events/event-bus.ts` 的 `EventBus` 是对 Node `EventEmitter` 的薄封装，使用 `EventType` 字符串做事件名，并通过 `getEventBus()` 提供全局单例，供上层订阅跨 Agent 的流式事件。
依赖方向：events → core/types；backend-agent → core/agent + backends/base；agent-manager → agent + backend-agent + agent-factory；factory 不反向依赖 manager。
