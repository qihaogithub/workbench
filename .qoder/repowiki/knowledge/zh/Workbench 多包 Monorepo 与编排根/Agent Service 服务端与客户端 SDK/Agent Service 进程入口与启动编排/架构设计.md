src/server.ts 是唯一的进程入口，按以下阶段完成跨子模块装配：
- 配置/日志：先通过 shared_infrastructure 的 loadConfig/getLogger 加载 .env 与 pino 实例。
- 后端供应商：调用 backends_providers 的 getBackendProvidersManager().initialize() 从 PI_AGENT_PROVIDERS 解析多模型供应商。
- 实例策略校验：workspace_session 的 assertWorkspaceAuthorityInstancePolicy 在启动时校验 Workspace Authority 实例策略。
- Agent 运行时：core_agent_runtime 的 getAgentFactory 注册 "pi-agent" 类型（PiAgentBackend + BackendAgent），getAgentManager 提供全局单例。
- 数据恢复：collab/workspace-file-persistence 的 getDefaultDataDir 配合 workspace-authority-startup-recovery 在 HTTP 监听前恢复持久化权限。
- Web 层：routes_api_layer 的 registerRoutes 挂载所有 REST/WebSocket 端点，暴露 /health 聚合 agent-manager 计数与 recovery 状态。
- 优雅关停：SIGTERM 回调依次 destroyAll → destroySessionStore → fastify.close，保证会话与工作区资源有序释放。

各子模块之间不直接互相 import，而是通过 server.ts 中的工厂/管理器单例（getAgentManager、getAgentFactory、getBackendProvidersManager）以及共享的 data 目录约定进行松耦合协作。
