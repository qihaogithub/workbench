- 每个工具以 `createXxxTool(config, ...)` 工厂函数形式定义，返回包含 `name`/`label`/`description`/`parameters`(TypeBox)/`execute` 的对象，而非直接导出类或函数。
- 工具参数一律使用 TypeBox `Type.Object({...})` 声明并在同一文件中用 `type XxxParams = Static<typeof XxxParams>` 同步推导 TS 类型。
- 需要用户确认的副作用操作（删除页面、计划审批）通过 `PermissionHandler` / `requestPlanApproval` 回调发出 `permission_request` 事件并返回 Promise，由外部 UI 层 `resolvePermission` 唤醒。
- 对 `@earendil-works/pi-agent-core` 与 `@earendil-works/pi-ai` 的引用全部走 `pi-agent-deps.ts` 的懒加载门面，禁止在工具文件顶部静态 import。
- 文件 I/O 工具在执行前统一调用 `isPathAllowed` 校验 workingDir 下的相对路径，并对 knowledge/ 目录额外施加写保护拦截。
