- 每个被测类/模块对应一个同名 `.test.ts` 文件，位于 `tests/unit/` 下，文件名与被测源文件一一对应。
- 外部依赖通过 `vi.mock('module', ...)` 配合 `vi.hoisted` 返回的工厂函数注入 Mock，并在 `beforeEach` 中 `clearAllMocks` / `mockReset` 重置状态。
- 集成测试使用 `fs.mkdtempSync(path.join(os.tmpdir(), '<prefix>-'))` 生成隔离临时目录，维护 `temporaryRoots` 数组并在 `afterEach` 中递归删除。
- HTTP 层测试统一通过 `app.inject({ method, url, payload })` 发起请求，而非启动真实监听端口；WebSocket 测试则用 `new WebSocket(url)` 连接 `127.0.0.1:<port>`。
- 异步流式事件通过封装 `waitForStreamTypes(socket, expectedTypes)` 等待期望事件集齐后 resolve，避免竞态。
