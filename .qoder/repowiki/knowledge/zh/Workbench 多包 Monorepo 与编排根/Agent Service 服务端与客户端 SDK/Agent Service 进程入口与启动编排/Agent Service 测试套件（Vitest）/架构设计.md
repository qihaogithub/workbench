测试按用例类型分为两层目录：
- `unit/`：针对单个源文件（`src/` 下各模块）的白盒测试，通过 `vi.mock` + `vi.hoisted` 构造内联 Mock（如 `MockAgentHarness`、`MockNodeExecutionEnv`），并直接 import `../../src/...` 源码路径进行断言。
- `integration/`：端到端场景，以真实 Fastify + @fastify/websocket 启动 HTTP/WebSocket 服务，使用 `app.inject` 和原生 `ws` 客户端验证 `/api/workspace-authority/*` 路由的 state/resource/mutation/stream 行为，并通过 `fs.mkdtempSync` 在 `os.tmpdir()` 下创建临时 fixture 数据目录，`afterEach` 统一清理。
依赖方向单向指向被测源码（`../../src/**`），不引入业务配置或外部网络；跨包类型仅引用 `@workbench/shared/contracts` 中的事件契约类型。
