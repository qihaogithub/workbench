- 所有 REST 请求统一经私有 `request<T>(path, options?)` 方法发起，自动合并 `Content-Type: application/json` 与可选 `X-API-Key` 头，并以 `ApiResponse<T>` 统一包裹成功/失败响应。
- REST 方法按资源域分组命名（如 `getSession`/`destroySession`、`getFiles`/`stageFiles`/`discardFiles`、`getWorkspace`/`updateWorkspace`），URL 模板以 `${sessionId}` 拼接在 `/api/agent/${sessionId}/...` 下。
- 可选项参数采用具名对象 `{ demoId?, projectId?, workingDir?, model?, options? }` 形式，避免位置参数歧义，并在方法体内显式解构/合并。
- `AgentStream` 使用 `Map<string, Set<Handler>>` 维护多监听器事件表，并通过 `emit` 分发，形成轻量级发布-订阅模型。
- WebSocket 连接错误与解析异常统一转换为 `type: 'error'` 的 `StreamEvent` 事件，而非抛出异常，保持调用侧一致的错误处理路径。
