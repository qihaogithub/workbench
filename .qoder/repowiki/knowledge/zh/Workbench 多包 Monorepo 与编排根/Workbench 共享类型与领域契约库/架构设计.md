本模块是一个纯类型 + 工具函数的无运行时依赖包，通过 `package.json` 的 `exports` 字段向工作区其他包暴露子路径入口（`@workbench/shared`、`@workbench/shared/contracts`、`@workbench/shared/demo/*`），自身仅依赖同仓库的 `@workbench/sketch-core`。

内部按职责拆分为多个平级源文件并由 `src/index.ts` 统一重导出：
- `workspace.ts`：Project / DemoPage / Version / ResourcePointer / AppGraph 等核心领域模型与版本轴（`ProjectBaseVersion`、`WorkspaceRevision`、`CanonicalSyncedRevision` 使用 branded type 防止混用）。
- `contracts.ts`：Workspace Authority 的持久化单写者契约（`WorkspaceMutationOperation`、`WorkspaceMutationReceipt`、事件流类型）以及资源路径白名单校验。
- `validator.ts`：从 author-site 迁移出的 JSON 语法与 props/schema 一致性校验函数。
- `demo/`：演示页面渲染相关能力——`sketch-scene.ts` 直接 re-export `@workbench/sketch-core`，`prototype-preview.ts` 与 `iframe-template.ts` 提供 HTML/CSS 清洗与预览文档构建。
- `knowledge.ts`、`agent-config.ts`、`external-auth.ts`、`diagnostics.ts`、`ai-error-normalizer.ts`：知识文档、Agent 配置、外部认证、诊断与 AI 错误归一化的类型定义。
- `index.ts` 同时集中定义了全局 `ErrorCode` 常量与 `ERROR_MESSAGES` 映射、`ApiSuccessResponse`/`ApiErrorResponse` 泛型、`SessionMeta`、`Collab*` 协作类型、`MultiDemoFiles` 等多页文件结构。

依赖方向单向向外：本模块被 `author-site`、`workbench-server`、`workbench-client` 等消费，自身不反向依赖任何业务实现包。
