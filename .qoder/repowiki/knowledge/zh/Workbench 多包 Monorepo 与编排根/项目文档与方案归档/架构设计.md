文档按读者角色与生命周期分层组织：
- `docs/项目文档/` 为核心知识库，以「模块（如创作端/01-用户鉴权）→ INDEX.md + 需求文档 + 技术子目录」的固定结构维护，每个模块同时包含需求侧与实现侧文档；顶层 `INDEX.md` 提供跨模块导航。
- `docs/plans/` 按状态分目录：`进行中/` 保留仍在演进的方案与跟踪清单，`已完成/` 仅归档结论性文档并附带 README 维护标准，`远期规划/` 存放长期方向，`测试脚本/` 放置 Playwright 录制产物与 shell/ps1 验收脚本。
- `docs/复盘文档/` 聚焦跨模块问题根因分析，按子系统（Session/配置系统/预览引擎/agent-service 等）切分。
- `docs/用户指南/` 面向运维与管理员的部署、模型配置、CLI 手册与故障排查。
- `docs/营销/` 存放宣讲材料。
- `docs/external/` 为外部引用占位。
依赖方向单向：各子文档通过相对路径互相链接，无构建工具或静态站点框架驱动，纯 Markdown 仓库即文档。
