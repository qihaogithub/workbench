本模块按职责分为三层：
- 顶层运维脚本（`deploy.sh`、`deploy-fast.sh`、`sync-to-cloudflare.sh`、`docker-*`）：以 Bash 为主，通过 SSH + rsync 将源码同步到远端 `/opt/workbench`，支持 `DEPLOY_BUILD_MODE=local|remote` 两种镜像构建路径；`deploy.sh` 是入口，内部再调用 `check-workspace-deploy-preflight.mjs` 做 Workspace Authority 预检，并在部署后轮询容器健康与 API 端口。
- 顶层 Node.js 校验脚本（`check-repo.mjs`、`check-contracts.mjs`、`check-viewer-contracts.mjs`、`check-workspace-authority-guards.mjs`、`build-preview-runtime.mjs`、`dev-restart.mjs`、`local-production-preview.mjs`）：纯 Node ESM（`.mjs`），通过 `spawnSync` 或 `child_process.spawn` 编排 pnpm/Docker/Next.js 子进程，统一以 errors/warnings 数组收集问题并以非零退出码报告失败。
- `development/` 子目录：面向开发期复现问题的独立诊断工具（Playwright E2E、Project CLI 集成、Agent 评估等），由 `README.md` 集中说明并映射到根 `package.json` 的 `pnpm test:*` 命令。
依赖方向单向：顶层脚本可调用 `packages/*` 源码或 `project-cli`，但 `development/` 不反向依赖顶层脚本；所有脚本均以仓库根为工作目录，通过相对路径解析 `data/`、`packages/`、`docs/` 等共享资源。
