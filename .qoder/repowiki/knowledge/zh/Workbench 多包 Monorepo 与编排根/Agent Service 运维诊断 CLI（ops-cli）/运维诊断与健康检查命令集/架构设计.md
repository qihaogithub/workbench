本模块由三个并列的 CLI 命令实现文件组成，均通过 `../utils.js` 提供的 `request`、`createSpinner`、`showSuccess/Error/Warning/Info`、`outputJson` 等工具统一交互风格：
- `diagnose.ts`：`diagnoseError` 按顺序执行三步——GET `/health` 探测服务可用性、可选 GET `/api/agent/{sessionId}` 校验会话存在性、可选 POST `/api/agent/{id}/message` 发送测试消息；失败时调用内部 `analyzeError` 基于错误码/消息匹配规则生成 `problem/possibleCauses/solutions` 分析结果。
- `diagnostics.ts`：核心是 `queryDiagnostics(kind, options)`，先解析 `since`/limit 等过滤条件，再根据 `kind`（autosave/collab/preview/trace/operation/session/recent/project/export）选择数据源：优先用 `better-sqlite3` 只读打开 `data/diagnostics/editor-events.db`，缺失或异常时回退到 `data/editor-diagnostics/*.jsonl`；支持通过 SSH + sshpass 将远端 data 目录打包为临时快照后在本地分析；输出包含事件时间线、按 workspaceId:revision 聚合的 `WorkspaceDiagnosticFlow`、以及 autosave/debounce/projection/canonicalLag 等多维百分位性能摘要。
- `logs.ts`：`collectLogs` 先在固定路径列表查找 JSON 日志文件，找不到则回退到通过 API 拉取 health + session 信息拼装成 `LogsResult`；内置 Pino 级别名↔数字互转及颜色映射。
依赖方向单向指向 `../utils.js` 与 `../types.js`，不互相引用，各自独立暴露一个 async 导出函数供上层命令路由调用。
