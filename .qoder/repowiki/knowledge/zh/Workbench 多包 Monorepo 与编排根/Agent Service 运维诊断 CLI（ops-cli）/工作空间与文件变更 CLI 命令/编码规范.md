- 每个命令函数统一接受 `(baseUrl, options|id, jsonMode)` 三参数，并在 try/catch 中先 `createSpinner` 后 `request`，成功时 `spinner.stop()`，错误分支在 `jsonMode` 下调用 `outputJson({ success: false, ... })`，否则 `showError` + `process.exit(1)`。
- JSON 模式输出采用固定结构 `{ success: true/false, projectId?, workspaceId?, sessionId?, error?, data? }`，非 JSON 模式则用 `chalk.gray`/`chalk.yellow`/`chalk.cyan` 等分级打印字段。
- 写操作（bootstrap / reconcile adopt / restore）遵循“先查 health → 若无需执行或未传 --apply 则 dry-run 返回 → 否则发起 POST”的三段式流程，并以 `action` 字段标注 `would_*`/`noop`/已应用状态。
- URL 路径通过 `healthPath`/`statePath`/`reconcile*Path` 等纯函数集中构造，使用 `encodeURIComponent` 对 projectId、workspaceId、sessionId 进行编码，避免硬编码字符串散落。
