- 每个命令文件只 export 一个 async 函数，以 `(baseUrl: string, jsonMode?: boolean)` 或 `(baseUrl: string, options: XxxOptions)` 作为统一入口签名。
- 所有用户可见输出统一经 `createSpinner` 包裹并在成功/失败路径显式 `spinner.stop()`，同时遵循 `jsonMode` 分支优先 `outputJson` 返回结构化结果的模式。
- 网络层调用一律通过内部 `request<T>` 工具封装，而非直接 fetch，以便集中处理鉴权与错误包装。
- 终端文案统一使用 `chalk.cyan/yellow/green/red/gray/dim/magenta` 分级着色，并以 `=== 标题 ===` 分隔区块。
- 错误路径在 `jsonMode` 下输出 `{ success: false, error }`，非 json 模式下则调用 `showError` 后 `process.exit(1)`。
