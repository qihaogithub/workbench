- 对外部数据源的 fetch 统一走 `lib/api.ts` 中的 `fetchJson` 包装器，失败时抛出带中文提示的 Error，并在错误消息中附带当前 `DATA_BASE` 以辅助定位。
- 所有面向用户的字符串文案使用 zh-CN，包括按钮标签、占位符、错误提示与 metadata 标题。
- URL 构造通过 `getDataUrl` / `getPublishedFileUrl` / `getCompiledJsUrl` 等工厂函数完成，避免在组件中拼接 `${DATA_BASE}/data/...` 字面量。
- 异步副作用遵循 `let cancelled = false;` + cleanup 函数的模式，在 `useEffect` 返回的清理函数中阻止状态更新，防止卸载后 setState。
- 组件间共享的复杂状态（如每个页面的 RJSF 配置）先写入 `configDataMap[pageId]` 内存态，再同步持久化到 `localStorage`，key 格式统一为 `config:${projectId}:${pageId}`。
- 环境变量通过 `process.env.NEXT_PUBLIC_*` 在 `next.config.js` 的 `env` 字段集中声明并提供默认值，使 `PREVIEW_RUNTIME_SOURCE`、`PREVIEW_SHELL_MODE`、`CDN_BASE_URL` 等可在构建期注入。
