- 对外 API 通过 `src/index.ts` 单一入口聚合导出，类型与值分别用 `export type { ... }` 与 `export { ... }` 显式区分。
- 所有面向用户的 React 组件顶部使用 `'use client'` 指令，确保在 Next.js App Router 下作为客户端组件运行。
- 画布相关工具函数（坐标转换、指针路由、图层归一化、文本摘要）全部实现为无副作用纯函数，仅依赖 `./types` 中的不可变数据结构。
- iframe 通信协议集中在 `iframe-types.ts` 中统一定义，再由 `types.ts` 重新导出，避免上游直接引用内部协议文件。
- 运行时调度采用「计算结果对象」模式：输入结构体 + 返回 `{ modes, activePageIds, sleepingPageIds }` 的纯函数，便于测试与复用（如 `computeCanvasRenderModes` 被 `computePreviewRuntimePoolPlan` 复用）。
- 样式类名通过 `cn(...)`（来自 `utils.ts`）配合 `clsx` 与 `tailwind-merge` 合并，避免硬编码冲突。
