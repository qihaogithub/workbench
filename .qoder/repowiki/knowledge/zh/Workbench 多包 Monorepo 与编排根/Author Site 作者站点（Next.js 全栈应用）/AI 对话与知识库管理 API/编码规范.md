- 所有 HTTP 响应统一包裹 `{ success, data?, error?: { code, message } }` 结构，错误码采用大写下划线命名（如 INVALID_REQUEST、NOT_FOUND、FORBIDDEN）。
- 对 live workspace 路径一律先调用 `getLiveWorkspaceRouteContext`，若 `isLiveWorkspacePath` 为真且缺少 session 则返回 `WORKSPACE_AUTHORITY_NOT_READY` 409，否则走 `commitWorkspaceMutation` 而非直接写 fs。
- 写入 manifest 前总是先 `readManifestRaw` 取旧内容并计算 `hashText(previous)`，在 `expectedHash` 中传入以实现乐观锁式并发保护。
- 每个写操作成功后通过 `createKnowledgeVersion` / `resourceDelete` 调用 `ProjectAdminService` 记录 `knowledge_document` 类型的资源版本，actor 固定为 `author-site`。
- system 来源的知识条目在 PUT/DELETE 时显式检查 `item.source === 'system'` 并返回 FORBIDDEN，禁止用户修改历史内置文档。
