模块按 Next.js Route Handler 目录组织为三个并列子路由：`commits/`（只读，通过 `ProjectAdminService.projectCommitList` 读取时间线）、`folders/`（CRUD，含 `[folderId]` PATCH/DELETE）、`config/`（GET/PUT/DELETE 项目级 `project.config.schema.json`）。所有写操作遵循同一分支策略：先通过 `isLiveWorkspacePath` 判断是否为 live workspace；live 路径经 `commitWorkspaceMutation` / `createTextWorkspaceMutation` 走 Workspace Authority 以 `expectedHash` 做乐观锁，非 live 分支则直接调用 `@/lib/fs-utils` 的同步文件写入。鉴权统一从 Cookie 取 JWT 并通过 `verifyToken` 校验，再与 Session 元数据做 `userId` 归属与 `demoId` 匹配校验，过期返回 410。错误全部经 `createApiError` / `createApiSuccess` 包装为统一 JSON 结构，Authority 客户端异常由 `WorkspaceAuthorityClientError` 捕获并透传 `authorityCode`。类型定义来自 `@workbench/shared` 与 `@workbench/shared/contracts`，业务逻辑集中在各 route.ts 内，未拆服务层。
