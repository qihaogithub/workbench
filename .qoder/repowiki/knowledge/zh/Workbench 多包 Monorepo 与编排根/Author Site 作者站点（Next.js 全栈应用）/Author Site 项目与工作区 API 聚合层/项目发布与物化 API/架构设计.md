本模块由三个并列的 Next.js App Router route handler 组成，均位于 `app/api/projects/[projectId]/` 下，通过路由参数 `projectId` 定位目标项目：
- `publish/route.ts`：POST 入口，负责鉴权（`getAuthCookie` + `verifyToken`）、可选的工作区会话校验与 flush（`flushAndSyncProjectWorkspace`），随后调用 `publishManager.publishProject` 完成快照创建；对 `PROJECT_NOT_FOUND`、`NO_CONTENT_TO_PUBLISH`、`SNAPSHOT_CREATE_ERROR`、`IMAGE_LOCALIZATION_FAILED` 等错误做显式映射。
- `publish-status/route.ts`：GET 入口，仅委托 `publishManager.getPublishStatus` 返回当前发布状态。
- `materialize/route.ts`：POST 入口，使用 `@workbench/project-core` 的 `ProjectAdminService.projectMaterialize` 执行项目物化，并以 `{ id, name, role: 'creator', source: 'author-site' }` 作为操作者上下文传入。

依赖方向单向向外：route handler → `@/lib/auth/jwt` / `@/lib/fs-utils` / `@/lib/session-manager` / `@/lib/workspace-flush` / `@/lib/publish-manager` / `@workbench/project-core`，handler 本身不持有业务逻辑，只做请求解析、鉴权、异常到统一 JSON 包装（`createApiSuccess` / `createApiError`）的适配层。
