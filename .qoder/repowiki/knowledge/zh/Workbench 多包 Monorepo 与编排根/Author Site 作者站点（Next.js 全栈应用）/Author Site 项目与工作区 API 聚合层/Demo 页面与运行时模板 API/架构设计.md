本模块由两个并列的 Route Handler 目录组成：`demos/` 提供单页级操作（GET/POST/PATCH/DELETE `[demoId]`、`[demoId]/files`、`[demoId]/runtime`），`demo-pages/reorder` 提供批量排序。所有写操作遵循同一编排模式：
- 鉴权与上下文解析：先 `getAuthCookie` + `verifyToken`，再校验 `sessionId` 与 `projectId` 归属、会话过期与工作空间路径；读操作可省略 sessionId。
- Live/Branch 双轨写入：通过 `isLiveWorkspacePath(wsPath)` 分支——live workspace 使用 `commitWorkspaceMutation` 提交 `WorkspaceMutationOperation[]`（`put_text`/`delete_path` 配合 `expectedHash`/`expectedAbsent` 做乐观锁），branch 则直接调用 `fs-utils` 的 `createWorkspaceDemoPage` / `updateWorkspaceDemoFiles` / `reorderDemoPages` 等函数落盘。
- 共享契约：操作对象类型来自 `@workbench/shared/contracts` 的 `WorkspaceMutationOperation`，元数据模型来自 `@workbench/shared` 的 `DemoPageMeta` / `WorkspaceTree`；错误统一经 `createApiError` 包装并映射 `WorkspaceAuthorityClientError` 到 HTTP 状态码。
- 资源边界：文件路径以 `demos/${demoId}/...` 为前缀，受 `isManagedWorkspaceResource` 约束；schema 变更在 `files/route.ts` 与 `runtime/route.ts` 中均通过 `validateNoSchemaConflictFromStrings` 进行跨页面冲突检测。
