- 每个 handler 入口先调用本地 `getAuthenticatedUser()` 解析 JWT，未登录一律返回 `UNAUTHORIZED` 401。
- `kind` 参数统一经 `normalizeKind` 白名单过滤，不在枚举中直接返回 `INVALID_REQUEST` 400。
- 对 `ProjectAdminService` 的每次调用都附带固定结构的 actor 对象 `{ id, name, role: 'creator', source: 'author-site' }`。
- 带 sessionId 的请求在落盘前必须经 `sessionExists` + `isSessionExpired` 校验，并对跨用户访问返回 `FORBIDDEN`。
- live workspace 写入前先调用 `flushAndSyncProjectWorkspace` 或 `flushWorkspaceBeforeCriticalAction`，并将返回的 `canonicalRevision`/`canonicalRootHash` 作为 `workspaceProof` 透传给上层服务。
- 服务层返回的 `result.ok` 为 false 时统一用 `createApiError` 包装，并根据 `result.error.code` 映射 HTTP 状态码（如 FORBIDDEN→403）。
