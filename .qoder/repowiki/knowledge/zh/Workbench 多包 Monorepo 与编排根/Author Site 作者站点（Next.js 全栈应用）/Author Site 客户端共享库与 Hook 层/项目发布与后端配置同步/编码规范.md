- HTTP 客户端统一返回 `{ success, data?, error? }` 结构，并在 `success === false` 时抛出包含错误消息的 Error。
- 对外 API 方法采用「参数校验 → fetch → 解包 response.data → 成功返回 / 失败抛错」三段式模板，避免在调用方重复处理网络异常。
- 发布流程按阶段通过 `onProgress(percent, message)` 回调上报进度，百分比按页面数量线性分配，便于前端展示实时状态。
- 异步任务（云同步、provider 同步）通过进程内 state 对象跟踪 inProgress / attemptCount / lastResult 等指标，并提供只读 snapshot 接口供监控。
- 配置文件读写采用「readManifest → 内存修改 → writeManifest」模式，先读后写并以 JSON 序列化落盘，缺失文件时回退为空集合。
