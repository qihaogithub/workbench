本模块由五个职责清晰的子文件组成，围绕「发布产物 + 运行时辅助」展开：
- `publish-manager.ts` 是核心编排器：读取 workspace 下的 demo 页（含 high-fidelity React、prototype-html-css、sketch-scene 三种 runtimeType），调用 `@/lib/compiler` 编译 TSX，通过 `processImagesForPublish` 与 `replacePathsInContent` 做资源本地化与路径替换，生成 `demos/<id>/compiled.js`、`iframe.html`、`schema.json` 等静态文件，写入 `<dataDir>/published/<projectId>`；随后创建版本快照并通过 `ProjectAdminService.projectCreatePublishCommit` 记录发布 commit，可选触发 `scripts/sync-to-cloudflare.sh` 推送至 Cloudflare Pages。
- `project-api.ts` 暴露 `ProjectApiClient` 类，封装所有 Next.js 本地路由（`/api/projects/*`、`/api/sessions/*`）的 HTTP 调用，统一使用 `localRequest` / `request` 两个内部方法，返回 `{ success, data, error }` 结构并在失败时抛错；对外导出单例 `projectApiClient`。
- `project-cli-url.ts` 纯函数工具，从请求头 `x-forwarded-host/proto` 或环境变量 `AUTHOR_SITE_URL` / `NEXT_PUBLIC_AUTHOR_SITE_URL` 推导 CLI 访问的 Author Site 基址，带协议推断与白名单校验。
- `project-images.ts` 以 JSON manifest（`<dataDir>/projects/<id>/images.json`）持久化项目级图片索引，提供增删查接口。
- `backend-providers-sync.ts` 维护进程内 `syncState`，从 `db-config` 读取 `model_config` 中的 `backendProviders`，通过 `./agent-providers` 推送到 agent-service，支持启动延迟、指数退避重试（最多 8 次）、状态快照查询。
依赖方向单向向外：本模块被上层 API handler 调用，不反向依赖 UI 层；对共享类型来自 `@workbench/shared` 与 `@workbench/project-core`。
