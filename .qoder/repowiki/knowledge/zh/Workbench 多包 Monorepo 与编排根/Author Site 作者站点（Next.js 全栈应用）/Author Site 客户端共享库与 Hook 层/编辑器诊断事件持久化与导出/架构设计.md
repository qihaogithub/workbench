模块以 `store.ts` 为唯一对外 API 入口，`types.ts` 仅做类型重导出与本地扩展。

- 双写/降级策略：写入时优先通过 `better-sqlite3` 将事件插入 `editor_events` 表（WAL 模式 + 多列索引），失败则回退到按 `editorSessionId.jsonl` 落盘的 JSONL 文件；读取/查询同样先查 SQLite，再扫描 `data/editor-diagnostics/*.jsonl` 作为 fallback。
- 存储布局：SQLite 位于 `${getDataDir()}/diagnostics/editor-events.db`，JSONL spool 位于 `${getDataDir()}/editor-diagnostics/<sessionId>.jsonl`，单文件超过 5MB 时从尾部保留最近 4MB 行并截断。
- 数据模型：事件统一归一化为 `@workbench/shared` 的 `EditorDiagnosticEvent`（驼峰字段），SQLite 行使用下划线列名并通过 `rowToEvent` 转换；同时兼容旧版 `LegacyEditorDiagnosticEvent` 以便读取历史 JSONL。
- 导出聚合：`buildEditorDiagnosticExport` 合并 SQLite 结果与 JSONL 去重后的 fallback 事件，并按事件中出现的 `sessionId` 扫描 `${getDataDir()}/agent-run-logs/<sessionId>/*.jsonl` 生成 `agentRunLogs` 索引。
- 依赖方向：本模块只向上依赖 `@workbench/shared` 的类型与构造器以及 `@/lib/fs-utils.getDataDir`，不反向被共享库引用，属于 author-site 侧的纯客户端持久化层。
