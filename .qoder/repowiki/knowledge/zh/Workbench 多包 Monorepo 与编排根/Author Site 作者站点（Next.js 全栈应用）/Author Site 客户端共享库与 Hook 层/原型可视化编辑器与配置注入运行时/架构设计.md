本模块由一组无状态纯函数组成，按职责拆分为五个独立文件：
- `prototype-visual-editor.ts`：基于 DOMParser 在内存中解析/序列化 prototype.html，通过 `data-ow-id` + `domPath` 定位节点，支持 text / attribute / style 三类变更，并可将文本、图片、颜色绑定到 JSON Schema（`data-bind-text` / `data-bind-src` / `data-bind-style-*` 标记）。
- `visual-configurator.ts`：面向真实组件源码（TSX/JSX），从 `VisualNodeInfo` 推导可配置候选，使用正则安全地插入 Props 接口声明、解构默认值、替换唯一字面量或内联 style，输出修改后的 code + schema + configPatch。
- `viewer-app-graph-runtime.ts`：将 demo-ui 发出的 `AppActionPayload` 与 `@workbench/shared` 中的 `AppGraph` 匹配，解析路由跳转、参数抽取与 `$state.$params` 表达式求值，返回联合类型区分成功/失败原因。
- `sketch-editor-engine.ts`：根据 previewMode/runtimeType/sketchEditing/viewingDocument 等上下文决定是否启用 Sketch 原生编辑器，当前仅实现 `native` 分支。
- `screenshot-service.ts`：封装截图服务的 HTTP 代理调用，统一超时 AbortController、x-request-id 透传及 503/504 错误响应构造。
- `prototype-preview-size.ts`：从 `PrototypePageMeta.previewSize` 或顶层 width/height 读取并归一化为 `PreviewSize`。
依赖方向单向向外：本模块消费 `@workbench/demo-ui` 与 `@workbench/shared` 的类型定义，不反向依赖上层 UI；各文件之间相互独立，通过统一的 `{ ok, ... } | { ok: false, error }` 结果模式传递错误。
