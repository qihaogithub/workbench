模块由两个相对独立的子域组成：
- `db/`：基于 better-sqlite3 的进程内单例数据库层。`index.ts` 通过 `getDb()` 懒加载并缓存 `users.db`（WAL + foreign_keys），首次调用时委托 `schema.ts` 的 `initializeDatabase()` 执行所有 `CREATE TABLE IF NOT EXISTS` DDL，并提供 `closeDb()` 关闭句柄；`schema.ts` 还暴露 `getUserCount()` 这类轻量查询。
- 顶层三个纯函数文件构成 JSON Schema 工具链：`schema-generator.ts` 用正则从 `interface XxxProps` / `type XxxProps = { ... }` 或函数解构参数中抽取属性，映射到 JSON Schema draft-2020-12 结构，并通过 `mergeWithExistingSchema` 保留 UI 扩展字段；`schema-defaults.ts` 从已有 schema 的 `properties[*].default` 反推默认值对象；`schema-validator.ts` 强制项目级与页面级 schema 的 `properties` 字段名互斥，提供 `validateNoSchemaConflict` 及字符串入口 `validateNoSchemaConflictFromStrings`，供 PUT config/files/generate-schema 等写入入口在落盘前调用。
依赖方向单向：`db/index.ts → db/schema.ts`；schema-* 三文件彼此独立、无运行时依赖，仅被上层 API 路由消费。
