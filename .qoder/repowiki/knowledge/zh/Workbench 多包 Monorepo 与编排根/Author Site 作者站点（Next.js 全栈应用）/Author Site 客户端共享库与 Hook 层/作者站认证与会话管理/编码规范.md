- 对外部依赖（NextRequest/cookies、db、fs）统一通过 `@/lib/*` 别名导入，避免直接耦合 Next.js 运行时细节。
- HTTP 响应统一返回 `{ success, data | error: { code, message } }` 信封结构，错误码集中映射到 `ErrorCodeType`。
- Cookie 设置遵循 httpOnly + sameSite=lax + maxAge 模式，生产环境根据 NODE_ENV 决定是否启用 secure。
- 环境变量读取集中在模块顶部，提供默认值并在缺失时抛出明确错误，禁止在业务逻辑中散落 process.env 访问。
- 数据库操作全部通过 `getDb().prepare(...).get/run/all` 预编译语句执行，不使用字符串拼接。
