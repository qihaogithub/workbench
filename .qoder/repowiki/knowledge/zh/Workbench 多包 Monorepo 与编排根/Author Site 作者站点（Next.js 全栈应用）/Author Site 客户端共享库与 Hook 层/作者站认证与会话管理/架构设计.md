模块按职责拆分为多个独立文件，彼此通过相对路径 `@/lib/*` 或 `./xxx` 互相引用，不依赖 Next.js 路由层：
- `auth/jwt.ts`：基于 jose 的 HS256 JWT 签发/校验与 auth_token Cookie 封装，供服务端 Route Handler 使用。
- `auth/password.ts`：bcrypt 加解密与用户名/密码输入校验，被 `user.ts` 复用。
- `admin-auth.ts`：面向 Edge Runtime (middleware) 的管理员 Secret 校验，兼容 URL 参数与 Cookie/Bearer Token，并暴露 `withAdminAuth` 中间件包装器。
- `dingtalk-login.ts`：企业微信式钉钉 OAuth 流程，含 app access token 进程内缓存与新旧 API 双端点回退；仅读配置，无状态。
- `external-auth.ts`：将 Figma/DingTalk 第三方凭据以 AES-256-GCM 加密后落库 `user_external_auth_configs`，并提供自动刷新 Figma refresh_token 的能力。
- `user.ts`：用户 CRUD + 钉钉身份绑定，所有 SQL 通过 `getDb()` 访问 SQLite。
- `session-manager.ts`：纯 Node 运行时模块，基于文件系统 `.session.json` 维护编辑会话生命周期（创建/续期/保存/归档/清理），并与 workspace-manager/canonical-materializer 协作。
- `workspace-authority-client.ts`：服务端 HTTP 客户端，调用独立 Workspace Authority 服务进行工作区快照、事件拉取、投影确认与 mutation 提交。
- `workspace-authority-browser-client.ts`：浏览器侧等价实现，通过同源的 `/api/workspace-authority/...` 代理访问同一套协议。
- `project-admin-service.ts`：对 `@workbench/project-core` 的薄适配层，把内部错误码映射到 NextResponse JSON 格式。

依赖方向单向向外：本模块依赖 `@/lib/db`、`@/lib/fs-utils`、`@/lib/runtime-config`、`@workbench/shared`、`@workbench/project-core`，不被其他同级模块反向依赖。
