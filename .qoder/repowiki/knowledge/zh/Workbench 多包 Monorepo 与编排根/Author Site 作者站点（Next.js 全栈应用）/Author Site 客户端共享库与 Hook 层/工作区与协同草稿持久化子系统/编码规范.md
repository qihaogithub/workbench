- 对外错误统一包装为自定义 Error 子类（如 `WorkspaceFlushError`、`ClientWorkspaceFlushError`），并通过 `code` + `status` + `message` 三元组向上返回，再由 `toApiErrorCode` / `getWorkspaceFlushErrorResponse` 映射到共享 `ErrorCodeType`。
- 对 Agent 服务返回的错误码在 `normalizeAgentFlushErrorCode` 中集中归一化为内部稳定码（如把 `WORKSPACE_RESOURCE_CONFLICT`、`WORKSPACE_EXTERNAL_DRIFT` 统一成 `WORKSPACE_STALE`），避免上游协议变动扩散。
- 工作区元数据统一写入 `.workspace.json`，并通过 `scope: 'branch'|'live'` 与 `status: 'active'|'archived'` 字段区分生命周期，判断是否 live 的工作集中在 `isLiveWorkspace` / `isLiveWorkspacePath` 两处复用。
- 关键写路径采用「先写 .tmp，再 rename」的原子替换模式（见 `syncActiveWorkspaceToCanonical` 中的 `projectWorkspacePath.tmp`），失败时确保 `fs.rmSync(tempPath)` 清理，防止脏目录残留。
- 前端异步调度器类（`WorkspaceAutosaveScheduler`、`WorkspacePerformanceSampler`）暴露 `dispose()` / `reset()` 方法并在内部维护 `disposed` 标志，所有后续操作幂等忽略，防止页面卸载后定时器泄漏。
- 状态机采用「声明式转移表 + 纯函数 `transition`」模式，新增状态只需在 `TRANSITIONS` 表中添加一行，UI 展示标签也通过 `SAVE_STATE_LABELS` 常量集中管理，避免分散 if/else。
