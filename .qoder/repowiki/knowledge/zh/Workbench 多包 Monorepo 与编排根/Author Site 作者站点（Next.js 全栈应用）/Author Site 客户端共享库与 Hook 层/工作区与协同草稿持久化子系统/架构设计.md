本模块按职责拆分为多个独立文件，围绕「工作区（Workspace）+ 会话（Session）+ 协同（Authority/Agent）」三层组织：
- 工作区生命周期与文件系统层：`workspace-manager.ts` 定义 `CreateWorkspaceResult` / `WorkspaceSyncResult` 等对外接口，负责 branch/live 两种 workspace 的创建、`syncActiveWorkspaceToCanonical` 原子替换（`.tmp` + rename）、孤儿清理、项目元数据读写；`live-workspace-route-context.ts` 从 NextRequest 解析 live workspace 路由上下文并校验 session/workspaceId/projectId 一致性。
- 文件访问与权限层：`workspace-file-utils.ts` 集中白名单 `EDITABLE_PATTERNS`、隐藏规则、运行时类型推断与文件树节点类型，作为所有文件读写的统一入口。
- 协同落盘与 canonical 物化层：`workspace-flush.ts` 通过 `flushWorkspaceBeforeCriticalAction` 调用 agent-service `/api/collab/.../flush-all`，再经 `ensureCanonicalRevision` 校验 Authority 快照后委托 `canonical-materializer.materializeCanonicalWorkspace` 将 canonical 写入项目目录，失败时回滚 `clearCanonicalSyncProofIfMatches`；`client-workspace-flush.ts` 提供浏览器端 `/api/sessions/:id/workspace-flush` 客户端封装。
- 前端实时提交层：`workspace-autosave-scheduler.ts` 实现单写者 autosave 调度器（800ms debounce + 3000ms max-wait + in-flight barrier + revision 单调 ack），由上层注入 `commitFn` 对接 Authority mutate。
- 离线草稿层：`workspace-offline-drafts.ts` 基于 IndexedDB（`workbench-offline-drafts` DB，`workspace-offline-drafts` store）缓存断网编辑，并提供 `reconcileDraft` 对比 baseHash 判定 match/conflict。
- 可观测性与 UI 状态层：`workspace-performance-sampling.ts` 用固定容量环形缓冲区采集 p50/p95/p99 指标并输出 SLO 报告；`workspace-save-state-machine.ts` 以显式转移表驱动 SaveState（editing/saving/autosaved/offline/conflict/canonical-stale），同时提供无状态的 `computeSaveStateFromContext` 便捷函数。
依赖方向：flush → manager + authority-client + canonical-materializer；autosave/离线/性能/状态机仅依赖共享类型与工具函数，不反向依赖文件系统层，保证前端可单独使用。
