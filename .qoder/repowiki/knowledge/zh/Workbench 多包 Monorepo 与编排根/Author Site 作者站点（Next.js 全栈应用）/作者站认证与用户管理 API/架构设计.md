模块按 Next.js App Router 的目录即路由约定组织为三个并列子域：
- `api/auth/`：账号体系入口，包含用户名密码登录(`login`)、钉钉 OAuth(`dingtalk/login` + `binding` + `config`)、登出(`logout`)、当前用户信息(`me`)、修改密码(`change-password`)、注册(`register`)；所有写操作在 try/catch 中调用 `@/lib/user` 与 `@/lib/dingtalk-login` 完成业务，再通过 `@/lib/auth/jwt` 签发 token 并写入 Cookie。
- `api/user/`：已登录用户可访问的个人能力，包括外部授权生命周期(`external-auth/[provider]/start|callback|route`)、模型配置(`model-config`)、作者偏好(`authoring-preferences`)；鉴权采用本地 `requireUserId()` 辅助函数从 Cookie 解析 JWT payload，变更完成后通过 `listActiveSessionsForUser` + `pushSession*ToAgent` 将配置推送到 agent-service 的活跃会话。
- `api/admin/`：管理员专用接口，全部以 `verifyAdminRequest(request)` 守卫，提供后端供应商同步(`backend-providers/sync`)、模型配置推送(`model-config`)、配置热重载(`reload-config`)、用户 CRUD(`users/[userId]`) 等。

依赖方向单向向外：Route Handler → `@/lib/*`（auth/jwt、user、agent-providers、session-manager、fs-utils）→ 底层存储/网络层；Handler 本身不直接读写磁盘或调 HTTP，仅做参数校验、鉴权与响应包装。错误统一经 `createApiError` / `createApiSuccess` 返回 `{ success, data, error }` 结构。
