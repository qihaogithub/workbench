- 每个路由文件只暴露一个或多个 HTTP 方法导出(POST/GET/PUT/DELETE)，并在顶层 try/catch 中捕获异常，成功走 `createApiSuccess`，失败走 `createApiError` 返回统一 JSON 结构。
- 需要登录的路由使用本地 `requireUserId()` 辅助函数从 Cookie 读取并验证 JWT，未登录时返回 401 + `UNAUTHORIZED` 错误码。
- 管理员路由统一在入口处调用 `verifyAdminRequest(request)` 进行权限检查，失败返回 401 + `UNAUTHORIZED`。
- 对 agent-service 的推送操作(模型配置、外部授权、后端供应商)在持久化成功后，通过 `listActiveSessionsForUser(userId)` 遍历活跃会话并 `Promise.all` 批量推送，失败仅 warn 不阻断主流程。
- 动态路由参数通过 `context.params.*` 解构并结合类型守卫(如 `isProvider`)进行白名单校验，非法值返回 400 + `INVALID_REQUEST`。
