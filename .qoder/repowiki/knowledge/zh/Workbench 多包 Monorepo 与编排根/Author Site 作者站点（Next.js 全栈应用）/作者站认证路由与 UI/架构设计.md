基于 Next.js App Router 的 `(auth)` 路由组组织认证相关页面：`layout.tsx` 提供居中布局容器；`login/page.tsx`、`register/page.tsx`、`forgot-password/page.tsx` 各自作为客户端页面（`"use client"`）处理状态与导航。

- 表单逻辑下沉到共享组件 `components/auth/login-form.tsx`，通过 `onSubmit` + `isRegister` 属性复用登录/注册两种场景。
- 第三方登录（钉钉）在 login 页面内以 `DingtalkLoginConfig` 配置驱动：先拉取 `/api/auth/dingtalk/config`，再优先走 `window.dd.runtime.permission.requestAuthCode` 免登码回调，回退到 `dingtalkConfig.authUrl` 跳转。
- 所有认证请求统一走 RESTful 风格 API：`/api/auth/login`、`/api/auth/register`、`/api/auth/logout`、`/api/auth/dingtalk/*`，响应体约定 `{ success, data, error }` 结构。
- 用户会话管理采用 httpOnly Cookie 模式：`user-menu.tsx` 仅负责触发 `/api/auth/logout` 清除服务端 Cookie，随后 `router.push('/login')` + `router.refresh()` 刷新全局状态。
- UI 层依赖 `@/components/ui/*`（shadcn/ui 的 Button/Input/Card/DropdownMenu 等），图标使用 lucide-react，提示通过 `useToast` 注入。
