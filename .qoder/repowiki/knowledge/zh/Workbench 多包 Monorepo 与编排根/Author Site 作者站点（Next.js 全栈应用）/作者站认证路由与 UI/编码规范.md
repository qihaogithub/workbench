- 页面组件统一标注 `"use client"` 并使用 `fetch` 直接调用 `/api/auth/*` REST 接口，而非封装数据获取 hook。
- API 响应按 `{ success, data, error }` 结构解析，失败时抛出 `Error(data.error?.message || '默认消息')` 并由 try/catch 捕获后通过 `toast({ variant: 'destructive' })` 反馈。
- 异步操作完成后同时调用 `router.push(redirect)` 和 `router.refresh()`，确保路由跳转与服务端状态同步刷新。
- 表单组件通过 props 暴露 `onSubmit(username, password)` 回调与 `loading` 状态，由父页面控制业务逻辑，保持表单无状态可复用。
