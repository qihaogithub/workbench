路由采用 Next.js File System Routing：/admin/layout.tsx 作为根布局统一渲染深色侧边栏导航与主内容区；各子功能以目录形式映射到 /admin/models、/admin/users、/admin/backend-providers 等页面。认证逻辑集中在 src/lib/admin-auth.ts，通过 ADMIN_SECRET 环境变量生成 SHA-256 哈希令牌，支持 URL 参数 ?secret= 与 Cookie admin_token 两种验证方式，并提供 withAdminAuth 高阶函数在中间件层拦截未授权请求并自动把 URL 参数落盘为 Cookie。所有业务页面均为 use client 客户端组件，直接通过 fetch 调用同包下的 /api/admin/* REST 接口（如 /api/admin/model-config、/api/admin/users），不依赖服务端数据获取。页面按职责拆分：models/page.tsx 承载后端供应商与模型白名单双 Tab 视图（含拖拽排序、表单校验、保存后推送至 agent-service 的异步状态面板）；users/page.tsx 负责用户列表、重置密码与删除；backend-providers/page.tsx 仅做重定向到 /admin/models?tab=providers。依赖方向单向：页面指向 @/components/ui/*、@dnd-kit/*、lucide-react、@workbench/shared，不反向引用 admin 内部文件。
