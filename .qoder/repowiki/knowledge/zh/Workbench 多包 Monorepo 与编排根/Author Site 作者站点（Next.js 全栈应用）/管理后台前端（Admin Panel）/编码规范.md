- 每个管理页面顶部先声明 'use client'，并通过 useState/useEffect 本地维护数据，再 fetch('/api/admin/...') 拉取服务端状态。
- API 错误处理统一包装为 { success, error: { code, message } } 结构，前端通过 readApiErrorMessage 或 data.error?.message 提取消息并用 toast 提示。
- 表单提交前执行本地校验（必填、默认模型归属、密码长度与一致性），失败时设置 error 状态显示红色提示条，成功则清空错误并刷新数据。
- 异步操作（保存、同步、重置密码、删除）均用 loading 状态变量控制按钮禁用与 Spinner 展示，并在 finally 中清理状态。
- 管理员凭据通过 process.env.ADMIN_SECRET 读取，哈希值经 hashSecret 计算后存入名为 admin_token 的 httpOnly Cookie，Cookie maxAge 固定为 2 小时。
