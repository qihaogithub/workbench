- 截图代理类路由统一使用 try/catch 包裹 `fetchScreenshotService` 调用，捕获时区分 `isAbortError` 返回超时响应，其余异常返回服务不可用响应，避免将上游错误直接透传到客户端。
- 对外 JSON 响应采用 `{ success, data | error }` 双分支结构，错误对象包含 `code` 与 `message` 字段，便于前端统一处理。
- 本地截图文件路径遵循 `data/screenshots/{projectId}/{pageId}.{hash}(.variant).png` 命名约定，并通过 `normalizeHash` 校验 16 位十六进制格式后再拼接路径。
- 静态资源响应一律设置 `Cache-Control: public, max-age=31536000, immutable` 配合 `X-Content-Type-Options: nosniff`，确保浏览器安全且可长期缓存。
