本模块以 Next.js App Router 为运行时边界，将前后端能力按目录域切分：
- `src/app/(auth)`、`admin`、`demo/[id]/edit`、`viewer`、`explore`、`wish`、`cli` 等路由组分别承载认证页、管理后台、可视化编辑器、预览/嵌入、探索与 CLI 文档等前端视图；
- `src/app/api/**` 下的 Route Handler 作为后端入口，按资源域拆分为项目/工作区、会话/画布、AI 聊天/知识库、截图/预览分发、模板/演示数据、认证/用户管理等子域，全部通过 JWT Cookie + Session 鉴权后委托给 `server_libs` 与文件系统持久化；
- `src/lib` 客户端库与 `components` 共享组件被各路由组复用，`client_libs` 通过 hooks（如 `useCollabDocument`）向上层页面暴露状态与副作用；
- `scripts/` 提供数据库初始化与会话迁移等一次性运维任务，`public/preview-runtime` 与 `data/ai-attachments` 由截图与 AI 附件 API 读写。
整体形成「App Router 页面 → Route Handler → server/client libs → 文件系统」的单向依赖链。
