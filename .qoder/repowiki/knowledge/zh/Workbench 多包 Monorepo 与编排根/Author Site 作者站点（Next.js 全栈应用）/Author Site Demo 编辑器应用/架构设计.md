- 路由入口 `app/demo/[id]/edit/page.tsx` 是唯一的编排层：通过 `useCanvasWorkspace` 持有 Canvas 状态，再组合 `useCommandHistory`（撤销重做）、`useVersionControl`（项目/页面级版本快照）、`useVisualEditState`（视觉编辑模式切换）和 `useWorkspaceAuthorityState`（协同权限）四个 hooks，把各自产生的命令与变更统一注入全局命令栈。
- 子模块按职责分层：`components/demo/*` 提供可复用的 UI 组件（画布预览、文件树、对话框等），`app/demo/[id]/edit/components/` 与 `hooks/` 仅服务于当前编辑页的布局与状态拼装，`single-preview-history.ts` 作为单页预览历史的轻量适配器桥接版本子系统。
- 跨模块数据契约集中在 `@workbench/shared` 类型（如 `DemoPageMeta`、`VersionInfo`、`CanvasState`）以及 `@workbench/demo-ui` 暴露的 `PreviewPanel`、`SketchEditorEngineHost` 等运行时 API；编辑页通过 `projectApiClient` 与后端交互，所有持久化路径由 `lib/workspace-autosave-scheduler` 与 `lib/client-workspace-flush` 统一调度。
- 子模块之间不直接互相 import，而是通过编辑页作为“总线”向上消费各 hook 返回值、向下透传 props，形成单向数据流：UI 组件 → 编辑页 state → 对应 hook → 版本/命令/工作区服务。
