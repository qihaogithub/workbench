本模块由三个职责互补的单元组成：
- `useVersionControl.ts`：核心 Hook，维护 `publishStatus`（never_published/published/unpublished_changes）、`versionHistory`、`pageVersionHistories`、`previewVersion` 等本地状态；通过 `projectApiClient` 拉取版本列表、调用 `createPageVersion`/`restorePageVersion`/`publishProject` 等后端接口，并通过 `/api/sessions/{sessionId}/files/{id}` 和 `/checkpoint` 直接写入 Session 文件。
- 自动检查点策略：基于 `IDLE_AUTO_CHECKPOINT_MS=5min` 与 `CONTINUOUS_AUTO_CHECKPOINT_MS=30min` 两个定时器，结合 `autoCheckpointSignature`（序列化 activeDemoId/code/schema/pageCount/canvasDirty）去重，避免重复提交；通过 `latestSnapshotRef` + `autoCheckpointInFlightRef` 保证并发安全。
- `useCommandHistory.ts`：独立于版本控制的通用撤销/重做引擎，以 `EditorCommand { label, undo, redo }` 协议驱动双栈（undoStack / redoStack），暴露 `recordCommand`/`executeCommand`/`undo`/`redo`/`reset`/`bindKeyboardShortcuts`，并通过 `shouldIgnoreGlobalUndoRedoEvent` 过滤输入框内的 Z/Y 快捷键。
- `single-preview-history.ts`：纯函数工具，将 `SinglePreviewTarget`（page 或 canvas 中的 knowledge_document）解析为带 title 的 `SinglePreviewResourceHistoryTarget`，供 UI 侧渲染历史记录条目。
依赖方向：本模块仅消费 `@workbench/shared`、`@workbench/project-core`、`@workbench/demo-ui/canvas-kernel` 的类型与导出，不反向依赖上层组件，属于可复用的业务 Hook 层。
