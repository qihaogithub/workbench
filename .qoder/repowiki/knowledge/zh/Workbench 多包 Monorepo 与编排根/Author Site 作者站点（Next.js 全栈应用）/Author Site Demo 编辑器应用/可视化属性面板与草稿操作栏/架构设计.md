模块由两个 React 客户端组件组成：`VisualPropertyPanel.tsx` 是核心 UI，通过 `useMemo` 按 `SECTION_ORDER`（位置/布局/外观/图片/文本/背景/边框/链接/阴影与模糊）将 `PropertySpec[]` 分组渲染；每个属性以 `{section, property, label, kind, input, options, unit}` 声明式描述，再由 `getCurrentValue`/`getDisplayValue` 从 `selectedNode.computedStyle`、`attrs`、`textContent` 中读取并回写至 `onPropertyChange`。面板内嵌颜色明度控制（`color-mix(in srgb, ...)`）、圆角四向独立输入、图片上传（优先 `/api/sessions/:sessionId/assets/upload`，失败回退 data URL）和本地化（调用 `image-localization.localizeSelectedImageAsset`）。`getLayerName`/`getLayerKind`/`getLayerPurpose`/`getContextualLayerName`/`shouldShowLayerItem` 共同决定左侧图层树的可见性与语义名称。`VisualDraftActionBar.tsx` 是一个无状态展示组件，根据 `VisualDraftActionState.kind`（save/send）切换图标与文案，仅负责显示 `action.count` 项修改与主/取消按钮。测试文件 `page-config-panel.test.tsx` 验证的是 `@workbench/demo-ui` 导出的 `PageConfigPanel`/`ConfigForm` 行为（分类筛选、`visibleWhen` 条件显示、项目级配置绑定），属于对依赖包的契约断言而非本模块内部逻辑。
