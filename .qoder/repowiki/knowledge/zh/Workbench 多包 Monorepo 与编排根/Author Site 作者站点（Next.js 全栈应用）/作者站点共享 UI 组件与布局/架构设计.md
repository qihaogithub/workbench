模块按职责拆分为四个子目录，依赖方向自下而上：
- `ui/`：纯展示型基础组件（button、dialog、toast、avatar、tabs 等），全部通过 `@radix-ui/*` 构建无样式骨架，再用 `class-variance-authority` + `cn` 组合 Tailwind 类名，遵循 shadcn/ui 单文件导出模式（如 `button.tsx` 同时导出 `Button` 和 `buttonVariants`）。
- `providers/`：应用级上下文提供者，`theme-provider.tsx` 通过 React Context 注入 `theme: 'dark'` 并操作 `documentElement.classList`；`index.ts` 聚合导出供根节点包裹使用。
- `layout/`：页面级容器，`main-layout.tsx` 组合 `Header` 与 `<main>` 内容区，仅负责结构布局，不持有业务状态。
- `settings/` 与 `cover-image-dialog.tsx`：面向业务的复合组件，内部组合 `ui/*` 并通过 `fetch('/api/...')` 直接调用 Next.js API Routes，是模块内唯一承载路由跳转 (`next/navigation`) 与副作用逻辑的地方。

全局状态边界清晰：主题由 `ThemeProvider` 提供，通知由 `toast-provider.tsx` 中的 `ToastProviderWrapper` + `useToast()` 提供，两者均要求调用方在树中挂载对应 Provider，并在 Hook 未找到上下文时抛出错误以快速暴露缺失。
