本模块是 Next.js App Router 下的 REST 风格路由集合，按资源路径分层：`sessions/route.ts` 处理 `POST /api/sessions`（创建或复用活跃会话并回填代码/schema/workspacePath）与 `GET /api/sessions`（通过 agentClient.listSessions 列出会话）；`[sessionId]/save/route.ts` 处理 `POST /api/sessions/:sessionId/save`，在落盘前先调用 `flushAndSyncProjectWorkspace` 把临时工作区同步到项目；`[sessionId]/discard/route.ts` 处理 `POST /api/sessions/:sessionId/discard` 以丢弃未提交的编辑。

依赖方向单向向外：路由层仅负责鉴权校验、参数解析与错误包装，所有业务逻辑下沉至 `@/lib/session-manager`（会话状态机）、`@/lib/fs-utils`（文件路径与元数据）、`@/lib/workspace-flush`（工作区同步）、`@/lib/auth/jwt`（JWT 校验）、`@/lib/agent-client`（远端 Agent 列表查询）以及 `@/lib/agent-providers`（推送模型配置与外部认证到 Agent）。`POST /api/sessions` 在创建/复用会话后还会调用 `pushUserModelConfigToAgent` 与 `pushSessionExternalAuthToAgent` 完成 Agent 侧上下文初始化。
