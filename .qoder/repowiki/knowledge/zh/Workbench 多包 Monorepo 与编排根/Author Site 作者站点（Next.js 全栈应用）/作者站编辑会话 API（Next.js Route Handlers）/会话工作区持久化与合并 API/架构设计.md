本模块基于 Next.js App Router 的 `[sessionId]` 动态路由，暴露三个 POST handler：
- `merge/route.ts`：校验 JWT 后调用 `flushAndSyncProjectWorkspace` 同步工作区，再通过 `saveEditSession(..., 'named_version', ...)` 写入带 `syncedWorkspace`（workspaceId/revision/rootHash）的命名版本。
- `persist-workspace/route.ts`：仅触发 `flushAndSyncProjectWorkspace`，返回 `canonicalRevision` 与 `canonicalRootHash`，不写 session 文件。
- `workspace-flush/route.ts`：在关键动作前调用 `flushWorkspaceBeforeCriticalAction`，并对请求体中的 `projectId`/`workspaceId` 做白名单校验，防止跨会话篡改。

依赖方向单向向外：JWT 鉴权 (`@/lib/auth/jwt`) → 会话读取 (`@/lib/session-manager.getEditSession`) → 工作区刷新 (`@/lib/workspace-flush`) → 会话写入 (`saveEditSession`)；错误统一经 `getWorkspaceFlushErrorResponse` 映射为标准 `{code, message, status}` 结构，再由 `createApiError`/`createApiSuccess` 包装为统一 JSON 响应。每个 handler 都遵循「未登录→401 / 过期→401 / 无会话→404 / 非本人→403」的守卫顺序。
