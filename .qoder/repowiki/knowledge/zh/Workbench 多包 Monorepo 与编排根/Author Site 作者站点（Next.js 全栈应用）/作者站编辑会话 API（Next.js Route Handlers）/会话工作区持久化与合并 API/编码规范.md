- 所有 handler 以相同的四步守卫开头：`getAuthCookie()` + `verifyToken()` 校验身份，随后 `getEditSession(params.sessionId)` 并比对 `session.userId !== payload.userId` 返回 FORBIDDEN。
- 业务异常通过 `getWorkspaceFlushErrorResponse(error)` 转换为标准 `{code, message, status}` 后再用 `createApiError` 包装，避免直接向上抛出原始错误对象。
- 成功响应一律使用 `createApiSuccess({...})` 包裹数据，失败响应使用 `createApiError(code, message)` 并显式指定 HTTP status。
- 对可选请求体参数采用 `typeof body.xxx === "string"` 类型守卫后再与 session 字段做等值校验，拒绝类型不匹配或越界值。
