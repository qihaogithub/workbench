基于 Next.js App Router 的动态路由，按功能拆分为三个子目录：
- `assets/upload/route.ts`：通用文件上传入口，校验 MIME/扩展名/大小后调用 `@/lib/fs-utils.saveSessionAsset` 落盘。
- `assets/[filename]/route.ts`：GET 直接返回磁盘中的二进制（带 Cache-Control），DELETE 经鉴权后调用 `deleteSessionAsset` 删除。
- `assets/localize/route.ts`：最复杂的业务端点，支持两种来源——浏览器 base64 blob 或远程 URL；对 URL 走 `assertPublicHttpUrl` + DNS 解析 + 私网 IP 白名单检查，限制 http/https、禁止 localhost 及私有地址，最多 3 次重定向，10s 超时，10MB 上限；成功后通过 `stageWorkspaceBinary` + `commitWorkspaceMutation` 以 put_binary 操作写入工作空间，并同步注册到 `@/lib/project-images` 索引。
- `ai-attachments/route.ts`：AI 场景专用上传，委托 `@/lib/ai-attachments.saveAiAttachment`。

所有端点共享统一鉴权链路：从 Cookie 取 token → `verifyToken` 验签 → 可选比对 `payload.userId` 与 session meta 的 `userId` 做跨用户拦截。错误响应一律通过 `createApiError(code, message)` / `createApiSuccess(data)` 包装，code 使用大写常量如 UNAUTHORIZED、SESSION_NOT_FOUND、INVALID_FILE_TYPE、FILE_TOO_LARGE、FORBIDDEN、SESSION_EXPIRED 等。依赖方向单向指向 `@/lib/*`（fs-utils、auth/jwt、project-images、workspace-authority-client、ai-attachments），本模块仅承担 HTTP 边界与参数校验。
