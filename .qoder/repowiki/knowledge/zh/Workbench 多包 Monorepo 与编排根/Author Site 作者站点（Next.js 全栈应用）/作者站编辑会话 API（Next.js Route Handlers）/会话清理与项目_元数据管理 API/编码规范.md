- 每个 route 入口统一先调用 `getAuthCookie()` 再 `verifyToken()`，失败返回 401 的 `UNAUTHORIZED` 错误对象。
- 成功响应一律使用 `createApiSuccess(data)` 包裹，异常路径使用 `createApiError(code, message)` 并附带 HTTP status。
- 对 JSON 文件读写采用 try/catch 吞错后 continue 或返回空数组，避免单个损坏会话导致整个请求失败。
- 对外暴露的字段更新走白名单校验（如 allowedFields 列表），禁止直接透传 request body。
