本模块由三个独立的 Next.js App Router route.ts 组成，均位于 `packages/author-site/src/app/api/sessions/` 下：
- `cleanup/route.ts`：POST `/api/sessions/cleanup`，仅清理当前登录用户的过期会话，委托 `@/lib/session-manager.cleanupExpiredSessions`。
- `project/[projectId]/route.ts`：GET `/api/sessions/project/:projectId`，遍历用户目录下对应项目的子目录，读取 `.session.json` 与 `.messages.json` 组装会话摘要列表。
- `[sessionId]/meta/route.ts`：PATCH `/api/sessions/:sessionId/meta`，白名单过滤字段（title/workbenchSessionId/status/workspaceId），状态变更为 editing 时续期 2 小时，变为 discarded/archived 时调用 `archiveSession` 归档并保留消息历史。

依赖方向单向向外：路由层仅负责鉴权（`@/lib/auth/jwt.getAuthCookie` + `verifyToken`）、参数解析与错误包装（`createApiSuccess` / `createApiError`），所有 I/O 通过 `@/lib/fs-utils`（`getSessionPath`、`sessionExists`、`getSessionsDir`）和 `@/lib/session-manager` 完成，自身不直接操作文件系统。
