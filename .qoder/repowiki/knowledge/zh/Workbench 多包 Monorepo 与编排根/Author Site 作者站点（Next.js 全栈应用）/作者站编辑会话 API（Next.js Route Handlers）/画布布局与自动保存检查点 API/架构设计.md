两个独立的 Next.js App Router route.ts 文件，均位于 `api/sessions/[sessionId]/` 动态路由下，共享同一套鉴权与错误封装约定：
- 鉴权入口统一通过 `@/lib/auth/jwt` 的 `getAuthCookie` + `verifyToken` 校验 Cookie，再结合 `@/lib/fs-utils` 中的 `sessionExists` / `getSessionMeta` / `getEditSession` 做 session 存在性与 userId 归属校验，返回 `createApiError('UNAUTHORIZED' | 'FORBIDDEN' | 'SESSION_NOT_FOUND')`。
- canvas-layout/route.ts 负责 `.canvas-layout.json` 的读写：GET 从 project workspace → session workspace → session 目录三级路径按 `updatedAt` 取最新；POST 在 live workspace 场景走 `commitWorkspaceMutation` 提交变更，否则直接原子写入临时文件后 rename，同时总是落盘到 session 副本作为 UI 恢复缓存。数据层使用大量手写 `parse*` 函数对 `unknown` 输入做严格白名单校验，最终经 `normalizeCanvasStateLayers` 归一化为 `@workbench/demo-ui` 的 `CanvasState` 类型。
- checkpoint/route.ts 仅暴露 POST：先调用 `flushAndSyncProjectWorkspace` 把内存工作区推送到 canonical，再用 `validateWorkspacePreviewRuntime` 校验预览运行时，最后通过 `createProjectVersionSnapshot` 以 type=`auto_checkpoint` 创建版本快照，返回 201。
- 依赖方向单向向外：本模块只消费 `@/lib/*` 工具包与 `@workbench/demo-ui` 的类型/归一化工具，不反向依赖上层业务逻辑。
