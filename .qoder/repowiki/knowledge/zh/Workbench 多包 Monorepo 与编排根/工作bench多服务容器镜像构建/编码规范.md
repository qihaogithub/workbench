- 每个子服务一个独立目录并在其下放置同名 Dockerfile，镜像间无共享中间层，各自维护最小依赖集。
- 全部采用两阶段构建：builder 阶段负责依赖安装与编译，runtime 阶段仅拷贝产物，避免在最终镜像中保留源码与构建工具。
- 依赖安装统一通过 `--mount=type=cache,id=workbench-pnpm-store,target=/pnpm/store` 复用 pnpm store，且先 COPY 各 package.json 再 COPY 源码以实现增量缓存。
- 对外暴露端口后紧跟 HEALTHCHECK 指令，使用 `fetch('http://localhost:<port>/health')` 进行 HTTP 健康检查。
- 面向国内环境的镜像通过 ARG 传入 `DEBIAN_MIRROR` / `NPM_REGISTRY`，并在 RUN 中动态替换 apt sources 与设置 corepack registry。
