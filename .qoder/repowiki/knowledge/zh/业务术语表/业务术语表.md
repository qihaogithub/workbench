### Workspace Mutation Authority (WMA)
- 定义：工作区变更权威系统，负责统一管理所有对 Project Workspace 的写操作，确保单写者一致性。替代原有的直接文件写入模式，通过串行队列、事务日志、快照恢复等机制保证数据一致性。包含 Authority snapshot、receipt 语义、CoalesceMaterializer 等核心组件。
- 别名：WMA、Authority、变更权威

### Project Admin CLI
- 定义：面向编码代理的本地项目管理命令行工具，提供 JSON-first 的输出格式，支持项目创建、模板管理、脚手架同步、提交发布等操作。命令包括 workspace-authority-status、workspace-authority-bootstrap、workspace-authority-reconcile-adopt 等。
- 别名：ow、workbench-project-admin、项目管理CLI

### Codex 查询 CLI
- 定义：OPS 工程诊断工具集，提供结构化的项目诊断、会话分析、事件查询等功能。支持 diagnostics:recent、diagnostics:project、diagnostics:session、diagnostics:trace 等命令，输出 JSON-first 格式便于自动化处理。
- 别名：诊断CLI、ops cli、codex

### Pi Agent 会话配置
- 定义：AI 对话的会话级配置体系，不再依赖工作空间内的 workbench 配置目录。创作端把系统提示词、模型选择、工作空间和图片附件发送到 agent-service；agent-service 通过 Pi Agent 后端、Pi Tools 权限和 SessionGuard 约束 AI 行为。
- 别名：会话配置、AI配置、system prompt

### Receipt 语义
- 定义：Workspace Mutation Authority 系统中的变更确认机制。AI 工具执行写操作后返回 receipt，证明变更已提交；Preview ack 证明预览已更新。System prompt 中约束 AI 不能声称没有 receipt 的修改已完成，也不能声称没有 preview ack 的预览已更新。
- 别名：变更回执、修改凭证、preview ack

### CoalesceMaterializer
- 定义：Canonical Materializer 的重写版本，负责将 Workspace 的变更合并应用到 canonical 工作区。从简单的顺序应用改为支持 coalesce（合并）策略，减少不必要的重复应用，提高性能和一致性。
- 别名：合并物化器、规范物化器

### 分支工作区 (branch Workspace)
- 定义：CLI、AI 大改或实验性事务使用的显式隔离工作区，不进入默认多人协作房间。与 active Workspace（默认编辑页、实时协同、预览和自动保存共同使用的工作区）和 canonical Workspace（项目基准工作区）形成三级工作区体系。
- 别名：branch workspace、隔离工作区

### 离线草稿 (Offline Draft)
- 定义：基于 IndexedDB 的前端离线存储机制，在断网或服务不可用时缓存用户的编辑内容。当服务恢复时自动同步到服务器，确保用户数据不丢失。是 Workspace Mutation Authority 的前端基础设施之一。
- 别名：IndexedDB草稿、offline draft
