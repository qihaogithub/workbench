- 每个命令导出一个 async 函数，签名形如 `(baseUrl, sessionId|options, jsonMode) => Promise<void>`，由上层路由按模式分发。
- HTTP 请求统一经 `request<T>(baseUrl, path, options?)` 封装，返回 `{ success, data, error }` 结构并据此分支成功/失败路径。
- 长时间操作前调用 `createSpinner('...')`，在响应后 `spinner.stop()`，错误分支也保证 spinner 被停止。
- `jsonMode` 为真时所有输出走 `outputJson({ success, ... })` 并直接 return，避免同时产生人类可读日志。
- 错误处理采用 try/catch + `process.exit(1)` 终止进程，并在 catch 中再次检查 `jsonMode` 以输出 JSON 错误。
- 会话状态到颜色的映射通过本地 `getStatusColor` 函数实现，对 ready/processing/error/initializing/destroyed 分别映射到 chalk 颜色。
