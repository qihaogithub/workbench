模块按职责分为三层：
- 网络接入层：`agent-client.ts` 通过 `@workbench/agent-client` 暴露单例 `AgentClient`，凭据来自 `runtime-config`；`agent-providers.ts` 使用内部 token（`X-Internal-Token`）调用 agent-service `/internal/*` REST 接口，负责推送/读取 backendProviders、session model-config、external-auth（钉钉）等配置，所有失败均返回 `{ ok, message, data }` 而非抛异常。
- 上下文采集层：`scan-workspace.ts` 基于 `listDemoPages` + `fs` 扫描 demos、schema、canvas-layout、knowledge manifest 和 memory.md，产出 `SystemPromptContext`；`active-view-context.ts` 把当前活动页/焦点页及预览诊断信息序列化为前缀文本。
- 提示词装配层：`system-prompt.ts` 将静态模板 `prompts/system-prompt.md` 与动态规则拼接——根据工具能力集分支生成「删除页面」「画布管理」段落，再追加 `generatePreviewAuthoringRules()`；同时提供 `buildDynamicContextPrefix`（L3 工作区状态）、`buildMemoryPrefix`（L4 记忆）、`buildKnowledgeIndexPrefix`（知识库索引）三个可组合的前缀构造器，配合 `WORKSPACE_STATUS_TEMPLATE` 完成最终 user/system 消息拼装。
依赖方向单向向下：providers → runtime-config；system-prompt → prompts/*.md + agent-prompts/*.template + @workbench/preview-contract；scan-workspace → fs-utils / canvas-layout-file / knowledge/builtin-documents / authoring-feature-flags。
