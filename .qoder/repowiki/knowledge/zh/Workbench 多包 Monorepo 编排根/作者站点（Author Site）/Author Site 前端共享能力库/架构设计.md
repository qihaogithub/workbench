本目录是 author-site 前端的横向能力中心，按职责拆分为多个独立子模块（agent_ai_chat、workspace_files、preview_compiler、publishing、admin_auth、user_config、db_schema、editor_diagnostics、prototype_visual_editor、api_client_utils），每个子模块自包含类型定义、HTTP/WS 客户端、本地存储与工具函数；上层页面通过 src/hooks/useCollabDocument.ts 统一消费 api_client_utils 提供的 Yjs 协同文档 Hook，其余能力由业务组件按需 import 各子模块入口。

跨子模块的协作契约：
- 认证与会话：admin_auth 负责登录态与 JWT Cookie，workspace-authority-client.ts 基于同一会话访问后端权限服务，publishing 与 agent_ai_chat 复用同一 session-manager。
- 配置注入：user_config 提供 model-config / runtime-props / db-config 等读取封装，被 preview_compiler、publishing、agent_ai_chat、prototype_visual_editor 共同消费以获取模型、CDN、沙箱依赖等运行时参数。
- 数据持久化：db_schema 提供 SQLite 连接与 JSON Schema 工具，被 editor_diagnostics、workspace_files 以及 admin_auth 的偏好/草稿落盘共用。
- 构建产物流转：preview_compiler 产出的预览代码被 publishing 打包到 published 目录，同时 prototype_visual_editor 在编辑期直接消费其运行时 manifest。
- 状态同步：workspace_files 的自动保存调度器与 workspace-save-state-machine 驱动离线草稿与 IndexedDB 写入，publishing 在发布时读取同一工作区快照。

所有子模块均遵循 TypeScript + Vite 构建约定，不直接依赖 React 渲染逻辑，仅通过纯函数与可组合 Hook 向上暴露能力。
