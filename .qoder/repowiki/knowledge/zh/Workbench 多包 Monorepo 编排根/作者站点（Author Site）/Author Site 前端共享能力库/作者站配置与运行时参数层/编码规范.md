- 对外暴露的「安全」接口一律剥离敏感字段并以 `Safe*` 命名（如 `SafeUserModelConfig.apiKey = ""` + `hasApiKey` 标记），解密仅在内部路径完成。
- JSON 列持久化采用 `config_json`/`preferences_json` 字符串 + `updated_at` 时间戳的行结构，读写时显式 `JSON.parse`/`JSON.stringify`。
- upsert 统一使用 `INSERT INTO ... (user_id, config_json, updated_at) VALUES (?, ?, ?) ON CONFLICT(user_id) DO UPDATE SET config_json = ?, updated_at = ?` 模板，避免先查后写的竞态。
- 环境变量解析集中在独立函数（`parseCsvEnv`/`parseIntegerEnv`/`trimTrailingSlashes`），并在 `runtime-config.ts` 中按 `NEXT_PUBLIC_*`（浏览器）与无前缀（服务端）两套命名空间分别暴露 getter。
- 配置兼容通过「新结构优先、旧结构反向推导」的双向归一化策略实现，新旧字段同时保留在输出对象中以维持向后兼容。
