本模块是 author-site 的 REST 入口，按资源域拆分为三个子目录：
- `projects/[projectId]/...`：项目级 CRUD（提交、文件夹、配置、Demo、发布、物化），写路径在 Live Workspace 上经 `commitWorkspaceMutation` → Workspace Authority 落盘，分支/非 Live 则直写磁盘。
- `workspaces/[workspaceId]/route.ts` 与 `workspaces/route.ts`：工作区生命周期（创建、查询）由 `@/lib/workspace-manager` 管理。
- `workspace-authority/[projectId]/[workspaceId]/[...segments]/route.ts`：白名单网关，仅放行 GET(state/snapshot/health/events/projection-acks) 与 POST(mutate/staging/projection-ack/reconcile/*)，校验 Cookie 与 sessionId 后以 `fetch` 透传到上游 `getServerAgentServiceUrl()` 暴露的 `/api/workspace-authority/projects/:projectId/workspaces/:workspaceId/...`。

跨子模块的统一契约：所有写操作均要求携带有效 JWT Cookie 并通过 `verifyToken` 解析 userId；需要会话的接口从 URL query 或 JSON body 中读取 `sessionId`，用 `getEditSession` 校验用户归属与 projectId/workspaceId 绑定关系；错误一律经 `createApiError` 返回标准 `{ code, message }` 结构。
