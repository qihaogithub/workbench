本模块是纯函数工具层，按职责拆分为四个文件：
- `parser.ts`：处理「=== DEMO CODE / === DEMO SCHEMA / === END ===」分隔符格式，提供 `parseFigmaText`、`buildFigmaText`、`isValidFigmaFormat`、`fixFigmaTextFormat`、`extractCodeFromFigma`、`extractSchemaFromFigma`。
- `markdown-parser.ts`：处理以 `# Workbench Export` + `## Component Code`（tsx）+ `## Schema Config`（json）三段 Markdown 的代码块格式，并提供 `parseFigmaImportContent` 作为统一入口，自动区分 `react`（Markdown）、`prototype`（HTML 原型，含 `.figma-export` 宽高提取）和 `unknown` 三种 kind；内部通过 `extractFencedBlock` 逐行扫描 fenced code block，避免引入外部 markdown 解析器。
- `validator.ts`：不实现校验逻辑，仅从 `@workbench/shared` 重新导出 `validateAll`、`validateJsonSyntax`、`validatePropsSchema`、`isValidJson`、`formatValidateDemoResult` 等核心校验函数及类型，并从 `@workbench/demo-ui` 导出 `getOrderable`、`getPositionable`、`getDefaultValues`、`getPreviewSize` 等运行时辅助，形成对上层包的薄封装。
- `error-mapper.ts`：将 `@workbench/shared` 的 `ValidationError[]` 聚合为 `UserFriendlyError`（summary/details/count/canAutoFix），按 `json_syntax`、`interface_not_found`、`props_*`、`required_missing` 分类统计并生成可一键修复提示。
依赖方向单向：`error-mapper.ts` → `validator.ts`（仅 import type），其余文件相互独立，均不依赖作者站点 UI 或业务组件。
