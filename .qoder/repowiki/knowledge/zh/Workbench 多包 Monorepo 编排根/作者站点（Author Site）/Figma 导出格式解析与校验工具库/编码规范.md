- 解析函数统一返回 `{ code, schema, success: boolean, error?: string }` 结构体，失败时 `success=false` 并附带中文错误信息，成功时省略 `error` 字段。
- 对外暴露成对的 `parseXxx` / `buildXxx` / `isXxxFormat` 三件套，分别负责解析、反向构建与格式判定，保持 API 对称。
- 文本规范化先做 `text.trim()`，再基于常量标记（如 `CODE_START_MARKER`、`MARKDOWN_TITLE`）进行 `indexOf` 定位与顺序校验，而非正则全量匹配。
- 可选的便捷提取函数（如 `extractCodeFromFigma`、`extractSchemaFromFigma`）在内部调用主解析器并用 try/catch 吞掉异常，返回 `undefined` 表示失败。
