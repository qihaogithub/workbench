模块以 `ai-chat.tsx` 中的 `AIChat` 作为唯一对外编排入口，内部按职责拆分为三层：
- 表现层（根目录 `.tsx`）：`conversation`、`message`、`assistant-message`、`prompt-input`、`attachments`、`reasoning`、`chain-of-thought`、`tool`、`timeline`、`agent-process-group`、`user-choice-card`、`permission-dialog`、`history-dialog` 等纯 UI 组件，通过 `index.ts` 统一导出；`split-content-renderer.tsx` + `split-by-fenced-code.ts` 负责将 Markdown 分块并渲染。
- 状态/流程层（`chat/hooks/`）：`use-chat-messages` 实现受控/非受控三态消息与流内容管理；`use-chat-models` 维护模型与深度选择；`use-chat-stream` 是核心编排器，组合上述 hooks 与 `StreamService`，处理发送、取消、重发、回滚、自动修复、用户选择、权限审批、计划解析、文件变更回调等。
- I/O 层（`chat/services/` + `chat/utils/`）：`stream-service.ts` 封装 `@workbench/agent-client` 的 `AgentStream`，负责连接建立、keepalive、上下文注入（L3/L4 记忆、知识索引）、事件分发（plan/thought/tool_call/user_choice/permission）；`message-service.ts` 负责消息持久化与会话标题更新；`chat-file-utils.ts` / `chat-stream-utils.ts` 提供文件变更合并与工具调用事件解析。
依赖方向严格自上而下：UI → hooks → services/utils → `@workbench/agent-client`，hooks 不直接访问网络，services 不持有 React state。`chat/types.ts` 仅 re-export 关键类型供外部消费。
