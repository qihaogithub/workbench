- 每个 route.ts 使用 try/catch 包裹请求处理，成功路径返回 `createApiSuccess(data)` 或 `projectAdminResponse(result)`，异常路径返回 `createApiError(code, message)` 并附带对应 HTTP 状态码。
- 对 JSON body 字段采用 `as { field?: unknown }` 后显式检查类型与非空，再调用 service 前执行 `.trim()` 清理，避免脏数据进入持久层。
- 删除类操作遵循「先 preview 再 execute」的两阶段模式：先调 `deleteXxxPreview(id)` 获取 planId/confirmToken，再调 `deleteXxxExecute(planId, confirmToken)` 确认执行。
- 动态路由参数通过 Next.js App Router 的 `{ params }: { params: { id: string } }` 解构，并在函数体中立即提取到局部变量以便后续使用。
