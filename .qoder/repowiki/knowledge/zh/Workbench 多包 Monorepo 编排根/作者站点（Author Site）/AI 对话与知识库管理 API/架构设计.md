模块由两个并列的 Next.js App Router 路由组构成：
- `api/ai/chat/route.ts`：仅暴露 POST，负责校验消息、可选扫描工作空间上下文（L3/L2/L5 前缀拼接）、组装静态 system prompt（module 级常量以利用 LLM API 缓存），再经 `getAgentClient()` 转发到外部 Agent 服务（默认 localhost:3201），统一将下游错误映射为 `{success:false,error:{code,message}}` 响应。
- `api/knowledge/`：实现知识库 CRUD + 内容读取，按资源路径拆分——`route.ts` 处理 GET（列表）和 POST（新建），`[docId]/route.ts` 处理 PUT/DELETE，`content/route.ts` 提供纯文本读取；所有写操作均通过 `commitWorkspaceMutation` 走 Workspace Authority 协议，支持 live workspace 模式（`isLiveWorkspacePath` + `getLiveWorkspaceRouteContext`），非 live 模式直接落盘 `knowledge/manifest.json` 与对应 `.md` 文件；每次变更通过 `ProjectAdminService.resourceVersionCreate/Delete` 记录版本。
依赖方向单向向外：本模块只依赖 `@/lib/*` 工具层与 `@workbench/project-core`、`@workbench/shared/contracts`，不反向引用上层页面组件。
