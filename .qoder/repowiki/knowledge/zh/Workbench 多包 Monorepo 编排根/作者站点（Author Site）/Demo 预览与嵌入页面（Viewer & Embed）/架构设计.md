基于 Next.js App Router 的两条并列路由：
- viewer/[projectId]/page.tsx 渲染项目级多页预览，支持 single/canvas 双模式、左侧页面目录、右侧可折叠配置面板；通过 /api/viewer/${projectId}/data 拉取 demoPages、projectConfigSchema、appGraph 等数据，并以 window.parent.postMessage 向父窗口广播 VIEWER_READY / VIEWER_CONFIG_CHANGE / VIEWER_PAGE_CHANGE 事件。
- viewer/[projectId]/[demoId]/page.tsx 渲染单个 Demo 页，除上述消息协议外还集成 @workbench/shared 的 AppGraph 运行时（resolveViewerAppAction），将组件触发的 AppAction 解析为状态更新与目标页跳转，并调用 /api/sessions 创建会话以支持图片上传。
- embed/[demoId]/page.tsx 是服务端渲染入口，优先读取 data/published/<demoId>/project.json 及对应 schema 文件构建已发布嵌入页，未发布时回退到扫描 workspace 下的 config.schema.json 并通过 /api/embed/${demoId}/iframe 代理 iframe 内容；EmbedConfigPanel.tsx 作为客户端子组件，在单页模式下直接 postMessage UPDATE_CONFIG 给 iframe，在多页网格模式下维护 Map 存储各页独立配置并广播项目级配置变更。
依赖方向：本模块仅消费上层共享组件 components/demo（PreviewPanel、PageConfigPanel、ConfigForm 等）、lib/runtime-props、lib/validator、lib/fs-utils 与 @workbench/shared，不反向依赖其他 app 路由。
