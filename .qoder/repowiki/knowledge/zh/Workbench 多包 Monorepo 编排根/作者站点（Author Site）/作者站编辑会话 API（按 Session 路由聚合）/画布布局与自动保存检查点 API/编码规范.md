- 所有 HTTP 响应统一通过 `createApiSuccess` / `createApiError` 包装，错误码使用大写常量字符串（如 UNAUTHORIZED、SESSION_NOT_FOUND、INVALID_REQUEST），并在 catch 块中映射为 FILE_READ_ERROR / FILE_WRITE_ERROR 等通用错误。
- 请求体一律先 `request.json().catch(() => null)` 或 `.catch(() => ({}) )` 防御性解析，再用 `isRecord` + 自定义 `readString/readNumber/readBoolean` 辅助函数逐项白名单校验，拒绝强转任意字段。
- 文件写入采用 `writeJsonFileAtomic` 模式：先写 `${filePath}.${process.pid}.${Date.now()}.tmp` 临时文件，再 `fs.renameSync` 原子替换，避免并发写入产生半截 JSON。
- 鉴权失败分支统一返回 `{ response: NextResponse.json(...) }` 并由调用方 `if (access.response) return access.response;` 短路，避免嵌套 if-else。
