本模块是 author-site 中所有「按 Session 维度」的后端入口，遵循同一组织约定：在 `packages/author-site/src/app/api/sessions/[sessionId]/` 下以目录名映射 REST 风格路由，每个功能域一个子目录并导出 `route.ts`（GET/POST/PUT/DELETE），由 Next.js App Router 自动注册。

- 生命周期入口 `/api/sessions/route.ts` 负责创建新会话，随后请求通过 `[sessionId]` 动态段路由到各子域 handler。
- 子域按职责切分：`workspace/*` 处理工作区目录与文件、`files/[demoId]` 处理多 Demo 页面代码、`canvas-layout` 与 `checkpoint` 管理画布状态与自动保存快照、`assets/*` 与 `ai-attachments` 管理上传的资源与 AI 附件、`merge` / `persist-workspace` / `workspace-flush` 提供版本化持久化、`meta` / `sync-project` / `save` / `discard` 驱动会话生命周期，`cleanup` 与 `project/[projectId]` 提供跨会话的清理与查询。
- 所有 handler 共享相同的鉴权与 Session 解析契约（从请求头或 cookie 提取 sessionId），并通过共享的 workspace 抽象层访问底层存储，从而保证不同子域对同一 Session 的数据一致性。
