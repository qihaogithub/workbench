- 每个 handler 开头统一执行 cookie→token→payload 三步鉴权，随后校验 sessionId 存在性、过期状态与 userId 归属，再 resolve workspaceId 与 workspacePath，失败时一律通过 `createApiError` 返回结构化 JSON。
- 对外部传入的路径一律使用 `path.resolve` 后再以 `startsWith(wsPath)` 或 `normalizedRelativePath.startsWith('../')` 双重校验，防止路径穿越。
- 写操作在 live workspace 下构造 `WorkspaceMutationOperation[]` 并通过 `commitWorkspaceMutation` 提交，在非 live 分支则直接调用 `updateWorkspaceDemoFiles` 落盘，形成清晰的分支分流模式。
- PUT 请求体字段先做类型断言与空值校验，缺失全部字段时返回 `INVALID_REQUEST`，再进入业务逻辑；错误统一由 `createApiError` 包装为 `{code, message}` 结构。
- Sketch patch 流程遵循“解析 payload → 校验 feature flag → 解析当前 scene → 可选 baseSceneKey 一致性检查 → 回放比对 → 记录诊断事件”的固定顺序，任一阶段失败都通过 `recordSketchPatchDiagnostic` 上报。
