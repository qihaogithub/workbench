本模块基于 Next.js App Router 的 `[sessionId]` 动态路由，按功能拆分为两条独立路径：
- `workspace/files`：面向编辑器侧的文件浏览与单文件读写。`files/route.ts` 实现 GET 目录列表（支持 `path`、`showKnowledge` 查询参数，懒加载一层子节点），`files/[...filePath]/route.ts` 通过 `resolveWorkspaceFilePath` 做路径规范化与 `../` 逃逸校验，GET 返回 `{content, editable, size}`，PUT 经 `isFileEditable` 白名单后调用 `commitWorkspaceMutation` 写入。
- `workspace/[...path]/route.ts`：轻量静态资源服务，直接以 `NextResponse(buffer)` 返回二进制/文本，带 MIME 映射与 `immutable` 缓存头，并内置多级 fallback（images/assets、去掉 demos 前缀等）。
- `files`：面向旧前端的多 Demo 页面聚合接口。根 `route.ts` 同时暴露兼容层 PUT（将 code/schema 落盘到第一个 demo 页面，并发出弃用警告），`[demoId]/route.ts` 是新版完整入口，支持 code/schema/prototypeHtml/Css/Meta、sketchScene/sketchMeta 以及 sketchPatch（含 baseSceneKey 一致性校验、`applySketchScenePatchOperations` 回放比对、诊断事件记录）。写路径根据 `isLiveWorkspacePath(wsPath)` 分流：live workspace 走 `commitWorkspaceMutation` + `expectedHash` 乐观锁；非 live 分支则直接 `updateWorkspaceDemoFiles` 落盘。

依赖方向：所有 handler 仅向上依赖 `@/lib/fs-utils`（会话元数据、workspace 路径解析）、`@/lib/auth/jwt`（cookie+token 鉴权）、`@/lib/workspace-file-utils`（隐藏文件/可编辑白名单）、`@/lib/workspace-authority-client`（Authority 提交）、`@/lib/live-workspace-route-context`（live 分支判断），不反向依赖上层 UI。
