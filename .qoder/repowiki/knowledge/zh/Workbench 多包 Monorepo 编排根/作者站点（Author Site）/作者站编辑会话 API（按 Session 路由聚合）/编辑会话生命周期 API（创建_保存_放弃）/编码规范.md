- 每个路由入口统一先通过 `getAuthCookie()` + `verifyToken()` 做登录态校验，失败时返回 `createApiError('UNAUTHORIZED', ...)` 并附带 401 状态码。
- 对外响应一律使用 `createApiSuccess(data)` / `createApiError(code, message)` 包裹，错误码采用大写英文常量（如 `SESSION_NOT_FOUND`、`FILE_WRITE_ERROR`、`INVALID_REQUEST`），并由 `NextResponse.json` 返回。
- 路由函数采用 try/catch 包裹主流程，catch 分支记录 `console.error` 后降级为统一的 `FILE_WRITE_ERROR` 或 `FILE_READ_ERROR` 500 响应，避免原始异常泄露。
- `[sessionId]` 子路由通过 `{ params }: { params: { sessionId: string } }` 解构动态段，并在操作前用 `getEditSession(sessionId)` 校验会话存在性与归属（save 中额外比对 `session.userId !== payload.userId` 返回 `FORBIDDEN`）。
