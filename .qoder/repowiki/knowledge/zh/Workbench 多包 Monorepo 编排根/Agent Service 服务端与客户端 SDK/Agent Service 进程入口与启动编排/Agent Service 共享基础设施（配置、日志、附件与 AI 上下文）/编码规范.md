- 配置统一通过 `process.env.*` 读取并提供默认值，集中封装在 `loadConfig()` 或 `readEnvConfig()` 中，禁止在业务逻辑里散落 env 访问。
- 对外错误采用带 `code` 字段的自定义 Error 子类（如 `ImageDescriptionError`），并在上层 catch 时根据 message 关键词区分 `SERVER_UNREACHABLE` / `GET_MODELS_ERROR` 等语义码。
- 对外部输入（sessionId、attachmentId、路径片段）一律先经正则白名单校验再拼入路径，并通过 `startsWith(sessionDir)` 防御路径穿越。
- 长文本输出统一用 `truncateText` / 固定长度常量（MAX_TEXT_CHARS、MAX_HISTORY_ITEMS、MAX_INLINE_PAGES）截断，避免提示词膨胀。
- 日志对象优先传结构化字段（如 `{ image, modelId }`）配合 pino 序列化，错误对象走 `serializeErrorForLog` 白名单过滤后再记录。
