模块按职责分为三层：
- 接口与实现（`backends/`）：`base.ts` 声明 `IBackendAdapter` 统一契约（initialize/sendMessage/onStream/getModelInfo/updateSystemPrompt 等），`pi-agent.ts` 提供唯一实现 `PiAgentBackend`，内部组合 `managers/` 下的 ModelManager、PermissionManager、UserInteractionManager、ToolHookManager、EventMapper 以及 `pi-tools/` 中的工作区工具集；通过 `managers/pi-agent-deps.ts` 惰性加载 `@earendil-works/pi-agent-core` 与 `@earendil-works/pi-ai` 的 AgentHarness/NodeExecutionEnv/InMemorySessionRepo/getModel 以解耦外部依赖。
- 配置中心（`config/`）：`backend-providers.ts` 暴露单例 `BackendProvidersManager`，按优先级（author-site 推送 > `.env` PI_AGENT_PROVIDERS > 空默认）维护全局 Provider 列表并支持 `setConfig` 运行时热更新；`session-model-configs.ts` 提供进程内 Map 存储每个 sessionId 的独立 `BackendProvidersConfig`；`session-external-auth.ts` 与 `system-knowledge.ts` 分别承载外部鉴权与系统知识注入。
- 依赖方向：`PiAgentBackend` → `managers/*` → `config/backend-providers` / `utils/config`，`managers/model-manager.ts` 复用 `getActiveModelId`/`splitFullModelId` 纯函数完成 provider/model 解析，避免循环引用。`index.ts` 仅导出 `IBackendAdapter` 类型与 `PiAgentBackend` 作为对外边界。
