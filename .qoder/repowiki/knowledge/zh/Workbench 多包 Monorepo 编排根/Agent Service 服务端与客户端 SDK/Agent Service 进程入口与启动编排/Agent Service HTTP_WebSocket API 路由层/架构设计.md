模块以 `index.ts` 为唯一入口，集中调用各子模块的 `registerXxxRoutes(fastify)` 完成路由注册；每个业务域（agent、projects、collab、workspace-authority、models、internal-config、viewer-ai、validate）独立一个文件，遵循“单文件一领域”的扁平结构。

- 请求/响应契约：`api-response.ts` 定义 `sendApiSuccess` / `sendApiError` 以及 `ApiErrorPayload`，所有 REST 处理器统一返回 `{ success, data }` 或 `{ success, error: { code, message, details? } }` 结构。
- 事件桥接：`ws-event-router.ts` 提供 `WebSocketEventRouter` 类，订阅 `BaseAgent` 的 `stream/thought/tool_call/plan/error/status/permission_request/user_choice_request` 事件，将其转换为 `ServerMessage` 并通过回调推送到客户端，同时维护 `activeMessage` 生命周期与 `AgentRunLog` 记录。
- 权限与协作：`workspace-authority.ts` 通过 `WorkspaceFilePersistence` 实现工作区变更授权、快照、事件流与投影确认；`collab.ts` 与之配合处理协作相关端点。
- 依赖方向：路由层仅依赖上层 core/service/workspace/session 等包，不反向被它们引用，属于纯表现层。
- WebSocket 使用原生 `ws` 库并通过 Fastify 的 `websocket: true` 配置挂载到 `/api/workspace-authority/.../stream` 等路径。
