本模块由两个子包构成：
- packages/agent-service：单一 Node.js 进程入口（server.ts）按固定阶段装配——注册后端供应商 → 恢复工作区权限 → 启动 Agent 工厂 → 挂载路由，通过 routes/ 暴露 REST + WebSocket 接口；核心 core/agent-manager.ts 统一调度 backends/ 下各 LLM 供应商实现，并横向依赖 session/、workspace/、collab/、services/ 等能力。
- packages/agent-client：纯 TypeScript SDK，封装与上述同一组 REST/WS 端点的调用，作为 agent-service 的对外消费面。
两端契约集中在 client/src/types.ts 与 server/routes/ 中定义，client SDK 仅依赖该契约而不感知服务端内部结构，形成“服务端实现 / 客户端契约”的双边边界。
