模块采用 Next.js App Router 单页应用结构：`src/app/layout.tsx` 定义站点元信息与根布局，`src/app/page.tsx` 仅作为壳路由挂载 `SketchPlaygroundApp`；核心交互逻辑集中在 `src/components/SketchPlaygroundApp.tsx`（客户端组件），通过 `useSketchEditorState(scene, updateScene, undefined, configData)` 驱动状态，并组合 `@workbench/sketch-react` 提供的 `SketchEditorCanvas`、`SketchEditorToolbar`、`SketchLayerPanel`、`SketchPropertyPanel` 构成三栏式编辑器界面（左侧图层、中间画布+底部工具栏、右侧属性面板）。`sketch-core` 以纯函数形式被直接调用（如 `validateSketchSceneDocument`、`renderSketchSceneToSvgMarkup`、`hitTestSketchScene`、`translateSketchNodes`、`getSketchSceneHashSource`），不经过 React 层，用于 JSON 校验、SVG 导出、命中测试、位移变换与哈希计算。`src/fixtures/sketch-fixtures.ts` 集中维护多组可切换的 `SketchFixture` 数据（含可选 `configData` 绑定配置），供顶部 Fixture 下拉选择加载。Dev Panel 以模态浮层 + tab 形式内嵌在画布区域，支持 Scene JSON / Config JSON 编辑回写、Metrics 性能基线（100/500/1000 节点）与 Debug 节点列表查看。依赖方向单向：playground → `@workbench/sketch-core` + `@workbench/sketch-react`，无反向依赖。
