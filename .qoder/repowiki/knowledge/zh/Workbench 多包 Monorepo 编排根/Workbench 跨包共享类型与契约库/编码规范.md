- 错误码采用 `as const` 对象 (`ErrorCode`) 配合 `Record<ErrorCodeType, string>` 的 `ERROR_MESSAGES` 映射双表模式，保证 code 与 message 一一对应且可穷举。
- 对外暴露的 API 响应统一使用 `ApiSuccessResponse<T>` 与 `ApiErrorResponse` 联合类型 `ApiResponse<T>`，并通过 `success` 字面量区分成功/失败分支。
- Workspace 版本标识使用 branded type（如 `WorkspaceRevision`、`ProjectBaseVersion`、`CanonicalSyncedRevision`）在类型层面阻止不同语义的版本互相赋值。
- 受控资源路径通过正则白名单 (`isManagedWorkspaceResource`) 校验，并在写入前调用 `assertManagedWorkspaceTextWrite` 做前置断言，形成“白名单 + 断言”的双重保护。
- 子模块通过 `export * from './xxx'` 由 `src/index.ts` 统一聚合再按需重导出，避免消费者直接 import 内部文件路径。
