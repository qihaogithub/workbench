模块由两个单向依赖的 npm 包组成：`@workbench/project-core` → `@workbench/project-scaffold`。

- `project-core/src/index.ts` 是单一导出门面，仅重新导出 `ProjectAdminService`、`WorkspaceResourceRegistry`、CLI prompt 及类型；对外契约集中在 `types.ts`（`ProjectAdminActor`、`ProjectAdminResult<T>`、`ValidationIssue`、`PageCreateInput` 等）。
- `service.ts` 实现 `ProjectAdminService` 类，通过构造函数注入 `WorkspaceMutationPort` 与外部工作区解耦，所有持久化写操作经 `workspace-resource-registry.ts` 的 `WorkspaceResourceRegistry.describe()` 匹配路径白名单并调用 `assertTextWrite` / `assertBinaryWrite`，禁止绕过注册表直接落盘；`config.ts` 暴露一组 `get*` 函数从环境变量读取运行时参数；`cli-prompt.ts` 生成 CLI 帮助文本。
- `project-scaffold/src/index.ts` 是脚手架导出主逻辑：调用 `ProjectAdminService.exportProjectPackage()` 获取远端产物，组装 `LocalProjectManifest`，再输出 `workbench.project.json`、`package.json`、`scripts/dev-server.mjs` 以及可选的 `.workbench/sync-state.json` / `remote.json`；内置纯 JS ZIP 构建器（手动拼 CRC32 + local/central/end header）用于打包；`local-preview-dev-server.ts` 把 Node HTTP 预览服务器脚本以字符串常量形式内联，被 `buildDevServerScript()` 注入到生成的脚手架中，支持 `--check`、`--preview-check`、`--screenshot` 三种模式及 Playwright 截图回退。
- 测试：core 使用 vitest（`vitest.config.ts`），scaffold 通过 `tsx` 直接执行单测文件。
