## 一、问题概述

### 现象
Demo 编辑页面的预览区（iframe）不显示内容，`#root` 元素保持为空。

### 影响范围
- AI 创作/调试场景：用户无法实时预览代码效果
- 配置面板调试：修改配置后无法在预览区看到实时反馈

## 二、当前状态

### 2.1 已修复问题 ✅

#### Session ID 不匹配问题（已修复）

**问题描述**: 浏览器中的 session ID 与文件系统中实际存储的 session ID 不一致，导致 `/api/compile` 返回 404。

**修复内容**:
1. **文件**: `packages/web/src/lib/session-manager.ts`
   - 修改 `findActiveSession` 函数，返回目录名而非 `.session.json` 中的 `sessionId` 字段
   - 自动修复 `.session.json` 中的 `sessionId` 字段，确保与目录名一致

2. **文件**: `packages/web/src/lib/fs-utils.ts`
   - 增强 `findSessionPath` 函数，同时支持目录名匹配和 `.session.json` 中的 `sessionId` 字段匹配

**验证结果**:
- `/api/sessions` 正确返回与文件系统一致的 `sessionId`
- `/api/compile` 返回 200 和编译后的代码（`codeLength: 1573`）

### 2.2 已修复问题 ✅

#### CDN CORS 问题修复尝试（已修复）

**问题描述**: `cdn.tailwindcss.com` 不返回 CORS 头，导致脚本加载失败。

**修复内容**:
- **文件**: `packages/web/src/lib/iframe-template.ts` (第39行)
- **修改**: 将 `<script src="https://cdn.tailwindcss.com"></script>` 改为 `<link rel="stylesheet" href="https://esm.sh/tailwindcss@3.4.1/dist/tailwind.min.css" crossorigin="anonymous">`

**验证结果**:
- CORS 错误已解决 ✅
- 但出现新问题：esm.sh URL 返回 404 ❌

### 2.3 新问题定位 ❌

#### Tailwind CSS CDN URL 返回 404（当前问题）

**问题描述**: 修复 CORS 问题后，使用 `https://esm.sh/tailwindcss@3.4.1/dist/tailwind.min.css` 返回 404 错误。

**诊断结果**:
```
Session API:      ✅ 200 OK
Compile API:      ✅ 200 OK (codeLength: 1573)
Blob URL:         ✅ blob:http://localhost:3200/...
Iframe Ready:     ✅ complete
#root:            ❌ empty (children: 0)
Tailwind 加载:     ❌ 404 错误
React 加载:        ❌ 依赖 Tailwind
```

**浏览器控制台错误**:
```
[ERROR] Failed to load resource: the server responded with a status of 404 () 
        @ https://esm.sh/tailwindcss@3.4.1/dist/tailwind.min.css:0
```

**根因分析**:
1. **esm.sh 不提供 Tailwind CSS 的预编译 CSS 文件**
   - esm.sh 是一个 JavaScript 模块 CDN，主要用于导入 ES 模块
   - Tailwind CSS 3.x 不提供预编译的 CSS 文件，需要通过构建工具生成
   - `https://esm.sh/tailwindcss@3.4.1/dist/tailwind.min.css` 这个路径不存在

2. **Tailwind CSS 的正确 CDN 使用方式**
   - `https://cdn.tailwindcss.com` 是一个 JavaScript 脚本，不是 CSS 文件
   - 这个脚本会在运行时动态生成 CSS 样式
   - 但这个 CDN 不返回 CORS 头（之前的问题）

3. **关键发现**
   - Tailwind CSS 3.x 不提供预编译的 CSS 文件
   - 需要使用 JavaScript 脚本方式或构建工具生成
   - 必须找到支持 CORS 的 Tailwind CDN

### 3.2 iframe 加载流程

```
PreviewPanel.tsx
      │
      │  1. 生成 Blob URL
      │     const blob = new Blob([html], { type: 'text/html' });
      │     const url = URL.createObjectURL(blob);
      │
      │  2. iframe src={url}
      │

iframe (blob:http://localhost:3200/...)
      │

      │

```

### 3.3 根因分析

#### 根本原因: CDN 未返回正确的 CORS 头

**问题定位**:
1. iframe 已使用 Blob URL，origin 为 `http://localhost:3200` ✅
2. iframe 模板中通过 `<script src="https://cdn.tailwindcss.com">` 加载外部脚本
3. **cdn.tailwindcss.com 未返回 `Access-Control-Allow-Origin` 头**
4. 浏览器阻止脚本加载，导致后续所有依赖（React、组件渲染）都无法执行
5. 组件渲染失败，`#root` 保持为空

**关键发现**:
- Blob URL 方案已正确实施（iframe src 为 `blob:http://localhost:3200/...`）
- 但 cdn.tailwindcss.com 不返回 CORS 头，这是 CDN 服务端问题，不是代码问题
- esm.sh 通常返回正确的 CORS 头，但 tailwindcss CDN 不返回

**相关代码**:
```typescript
// packages/web/src/lib/iframe-template.ts
export function generateIframeHtml(options: IframeTemplateOptions = {}): string {
  return `<!DOCTYPE html>
<html lang="zh-CN">
<head>
  ...
  <!-- 问题：此 CDN 不返回 CORS 头 -->
  <script src="https://cdn.tailwindcss.com"></script>
</head>
<body>
  <div id="root"></div>
  <script type="module">
    // 这些 import 也因依赖问题而失败
    import React from '${ESM_SH_BASE}/react@18';
    import ReactDOM from '${ESM_SH_BASE}/react-dom@18/client';
    ...
  </script>
</body>
</html>`;
}
```

## 四、修复方案

### 方案 1: 使用 jsDelivr 的 tailwindcss-cdn（推荐）

**原理**: 使用 jsDelivr 提供的 tailwindcss-cdn 包，这是一个专门为 CDN 使用优化的 Tailwind CSS 版本。

**修改文件**: `packages/web/src/lib/iframe-template.ts`

```typescript
// 修改前（当前错误）
<link rel="stylesheet" href="https://esm.sh/tailwindcss@3.4.1/dist/tailwind.min.css" crossorigin="anonymous">

// 修改后 - 使用 jsDelivr tailwindcss-cdn
<script src="https://cdn.jsdelivr.net/npm/tailwindcss-cdn@3.4.10/tailwindcss.min.js"></script>
```

**优点**:
- jsDelivr 返回正确的 CORS 头
- 在中国有 CDN 节点，加载速度快
- 支持版本锁定

**缺点**:
- 仍然是运行时生成 CSS，性能略低于预编译

### 方案 2: 使用 Tailwind CSS 官方 Play CDN（备选）

**原理**: 使用 Tailwind CSS 官方提供的 Play CDN，但需要解决 CORS 问题。

**修改文件**: `packages/web/src/lib/iframe-template.ts`

```typescript
// 修改后 - 使用官方 Play CDN
<script src="https://cdn.tailwindcss.com"></script>
```

**注意事项**:
- 官方 CDN 可能不返回 CORS 头
- 在中国访问可能较慢
- 需要测试验证 CORS 支持

### 方案 3: 内联 Tailwind CSS（长期方案）

**原理**: 将 Tailwind CSS 的核心样式直接内联到 iframe HTML 中，避免外部加载。

**优点**:
- 完全避免 CORS 问题
- 加载速度最快
- 不依赖外部 CDN

**缺点**:
- 增加 HTML 体积
- 无法使用 JIT 模式
- 需要定期更新内联样式

### 方案 4: 使用 PostCSS 预处理（生产方案）

**原理**: 在服务端编译阶段就将 Tailwind CSS 处理完成，iframe 直接加载处理后的 CSS。

**修改文件**: 
- `packages/web/src/lib/compiler.ts` - 添加 Tailwind 处理
- `packages/web/src/lib/iframe-template.ts` - 移除 CDN 引用

**优点**:
- 性能最优
- 完全避免 CORS 问题
- 支持所有 Tailwind 特性

**缺点**:
- 实现复杂度高
- 需要修改编译流程
