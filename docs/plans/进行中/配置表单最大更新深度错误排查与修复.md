# 配置表单最大更新深度错误排查与修复

## 背景

2026-06-30 用户反馈创作端页面出现 Next.js 运行时错误：`Maximum update depth exceeded`。截图中的调用栈落在 React 与 Radix compose refs 附近，但页面探针显示编辑页进入后 `ConfigForm` 的 Schema 与 initialData 同步日志会反复出现。

## 目标

- 修复配置表单在父组件重复渲染或传入等价新对象时可能产生的连续状态更新。
- 保持 Schema 默认值、已有配置值和用户输入的现有合并语义不变。
- 补充针对等价数据同步的回归测试。
- 同步更新配置与预览模块项目文档。

## 范围

- 涉及 `packages/demo-ui/src/ConfigForm.tsx`。
- 涉及 author-site 现有 ConfigForm 测试。
- 涉及 `docs/项目文档/创作端/04-配置与预览/` 下相关技术文档。
- 不处理 Next.js 版本升级、Radix 版本升级或页面整体重构。

## 方案

1. 为 ConfigForm 增加轻量结构比较，只有合并后的表单数据与当前状态不一致时才返回新对象。
2. 将 initialData 同步与 Schema 默认值同步拆清职责，避免 Schema effect 因父层传入等价新对象而重复运行。
3. 用测试覆盖等价 initialData 重新渲染和等价 schema 重新渲染，确保不会造成额外状态提交。
4. 更新表单生成器技术文档，记录配置表单状态同步约束。
