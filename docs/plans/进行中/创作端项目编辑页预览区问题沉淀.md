# 创作端项目编辑页预览区问题沉淀

## 当前状态

2026-07-22 正在优化单页面预览的可视化元素点选。已确认 HTML/CSS 原型页无法从 SVG 子节点解析可编辑目标、直接 hover 反馈未闭环，以及 React iframe 与原型页的重叠候选规则不一致；实现按“失败测试、两运行时统一交互、性能收口、文档与构建验证”推进。此前本地图片解析与预览请求终态修复已经完成，真实浏览器、Docker 截图和生产诊断时间线仍待专项验收。

## 当前结论

### 可视化元素点选

- HTML/CSS 原型页必须把 SVG/path 等非 HTMLElement 目标归一到可编辑语义节点，不能因点击图标内部节点而静默失效。
- hover 预选应在预览运行时内即时反馈，只在候选身份变化时向父页面同步完整节点信息，避免每次指针移动都触发编辑页状态更新。
- React iframe 与 HTML/CSS 原型页统一采用稳定候选顺序：普通点击选择最具体的可编辑对象，Cmd/Ctrl 连续点击在同一位置的重叠候选中循环；右键和左侧图层树继续作为精确选择入口。
- iframe 的 hover 框、选中框和批注标记需要在滚动、窗口尺寸或布局变化后按动画帧重算，避免静止指针下反馈错位。
- 本轮只增强运行时选择和临时 UI 状态，不改变页面源码定位、属性写回、配置 Schema 或发布数据协议。

### 本地图片解析

- 页面源码持久化相对路径，禁止写入带临时会话 ID 的资源 URL。
- React 页面编译时根据当前会话和页面 ID 重写静态图片路径；代码发生变化后重新计算模块 hash，避免缓存键与实际代码不一致。
- React 截图请求把页面 ID 传入编译接口，并直接使用本次返回的编译代码生成 iframe，避免截图缓存复用已被清理的 `moduleUrl`。编译缓存按会话与页面隔离。
- 原型页面截图复用共享 HTML/CSS 资源重写规则，补齐会话和页面上下文。

当前规则覆盖静态图片字符串与 CSS `url(...)` 中支持的资源扩展名；运行时拼接、远程 URL 和非图片资源不在本轮范围内。

### 预览请求终态

- 当前请求以纯 reducer 管理 `idle`、`compiling`、`waiting-shell`、`rendering`、`ready`、`failed`、`timed-out`、`cancelled`；旧 requestId 的迟到事件不会修改当前请求。
- 编译、iframe shell 和页面渲染分别设置 15 秒、10 秒和 15 秒 deadline；编译请求支持取消，组件卸载或请求替换会进入取消终态。
- iframe 仅在 React 实际提交后发送 `LOADED`；错误边界随新一轮渲染重置，避免“导入成功但渲染失败”被误报为成功，也避免旧错误粘住后续内容。
- 普通失败和超时显示错误摘要与“重试预览”，自动修复期间继续显示修复态，不再永久显示 loading 或只剩空白。
- 单页截图兜底仅在页面内容 hash 与当前输入一致、尺寸兼容时生效。画布继续负责休眠与渲染调度，但不再把过期截图当作当前页面产物。
- 预览错误、阶段 timing 和内容加载记录补齐 requestId；编辑页落账时同时关联页面 ID，便于按单次请求还原时间线。

## 已完成验证

- `PreviewPanel`：普通编译失败进入可操作终态，重试会创建新请求；无回执请求会进入超时终态。
- 生命周期 reducer：正常阶段流转、失败/超时/取消、旧请求迟到事件与成功后防回退。
- iframe 协议：`LOADED` 只在 React commit 后上报，错误边界按渲染版本重置。
- 路径重写：相对图片 URL 重写后重新计算 hash，且不污染原编译缓存对象。
- 截图服务：React 编译携带页面上下文并使用本次编译代码；原型 HTML/CSS 资源按会话和页面解析。
- author-site、demo-ui 与 screenshot-service 的相关单元测试和类型检查。

## 待验收事项

- 在真实创作端编辑 React 与原型页面，确认上传图片在单页、画布和重新打开后均可见，源码仍保持相对路径。
- 在本地 Chrome 与 Docker 环境分别生成 React/原型截图，确认图片可见且缓存命中不会复用失效模块地址。
- 用浏览器场景验证 shell 超时、render 超时、自动修复成功与修复预算耗尽，不出现永久 loading。
- 使用 `diagnostics:preview` 按 requestId 复核完整时间线，并确认 SQLite、JSONL fallback 与 event gap 状态。
- 根据真实耗时分布复核 15/10/15 秒 deadline；如需调整，应以诊断分位数为依据，不在组件中继续增加无界等待。
