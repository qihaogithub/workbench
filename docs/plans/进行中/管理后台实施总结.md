## 使用方式

### 1. 初始化数据库

```bash
cd packages/author-site
pnpm db:init
```

### 2. 启动服务

```bash
pnpm dev
```

### 3. 访问管理后台

首次访问(需带 secret 参数):

```
http://localhost:3200/admin?secret=admin-change-this-to-random-string
```

后续访问(Cookie 有效期内):

```
http://localhost:3200/admin
```

### 4. 配置模型

进入 `/admin/model-config` 页面,修改配置后点击"保存配置"即可立即生效。

## 技术亮点

1. **零停机配置更新**: 模型配置修改后立即生效,无需重启服务
2. **向后兼容**: 数据库无配置时自动 fallback 到环境变量
3. **安全性**: Admin Secret 哈希存储,httpOnly Cookie 防 XSS
4. **性能优化**: 内存缓存 + TTL,避免频繁读取数据库
5. **类型安全**: TypeScript 全链路类型检查通过
6. **客户端隔离**: 通过 `/api/models/config` API 断开 `better-sqlite3` 的客户端依赖,避免 Node.js 模块泄露
7. **ESM 兼容**: 动态导入 ESM-only 包(`@earendil-works/pi-agent-core`),加载失败时优雅降级

## 已知问题与修复记录

### 问题 1: 数据库路径错误 (已修复 ✅)

**现象**: 管理后台无法加载配置,创作端无法加载模型

**根因**: `db-config.ts` 使用 `process.cwd() + "data"` 解析路径,指向 `packages/author-site/data/users.db`(不存在),而主应用通过 `db/index.ts` 的 `findProjectRoot()` 正确解析到 monorepo 根目录的 `data/users.db`

**修复**: 改用 `getDb()` 单例连接,确保所有数据库操作指向同一文件

### 问题 2: 配置读取层未集成 (已修复 ✅)

**现象**: 管理后台配置修改后,前端模型列表不生效

**根因**: `model-config.ts` 的 `getModelConfig()` 已创建但从未被调用,`ai-models.ts` 仍只从环境变量读取

**修复**: 新增 `applyModelConfigsAsync()` 通过 `/api/models/config` API 读取完整配置(白名单+黑名单+多模态)

### 问题 3: 客户端构建 Node.js 模块泄露 (已修复 ✅)

**现象**: `Module not found: Can't resolve 'fs'` 构建错误

**根因**: `ai-models.ts` 的动态 `import("@/lib/model-config")` 让 Next.js 追踪到 `better-sqlite3` 依赖链,客户端无法使用 Node.js 的 `fs` 模块

**修复**: 创建 `/api/models/config` 服务端 API,`applyModelConfigsAsync` 改用 `fetch()` 调用,彻底断开客户端到 `better-sqlite3` 的依赖

### 问题 4: Agent-service 启动失败 (已修复 ✅)

**现象**: `Error [ERR_PACKAGE_PATH_NOT_EXPORTED]: No "exports" main defined`

**根因**: `@earendil-works/pi-agent-core` 是 ESM-only 包,但 agent-service 使用 CJS 解析(`tsx`),静态导入导致模块解析失败

**修复**: `PiAgentBackend` 改为动态 `import()` + try/catch,加载失败时优雅降级而非崩溃

## 已知限制

1. **供应商配置**: API Key、API Base 等仍需通过 `.env` 管理,修改后需重启服务
2. **单实例部署**: 多实例部署时需共享 SQLite 或改用 Redis 缓存
3. **配置版本管理**: 暂未支持配置历史与回滚功能
4. **pi-agent 后端**: 因 ESM 兼容问题暂时不可用,不影响其他后端(opencode-http, claude, gemini 等)

## 故障排查

### 问题 1: 访问 /admin 返回 401

**原因**: Admin Secret 验证失败  
**解决**:

1. 检查 URL 中的 secret 参数
2. 检查 `.env` 中的 `ADMIN_SECRET` 配置
3. 确认服务已重启

### 问题 2: 配置修改后未生效

**原因**: 缓存未清除或浏览器缓存  
**解决**:

1. 刷新页面 (Ctrl+F5 / Cmd+Shift+R)
2. 调用 `/api/admin/reload-config`
3. 检查浏览器开发者工具 Network 面板

### 问题 3: 数据库初始化失败

**原因**: 数据库文件权限问题  
**解决**:

```bash
# 检查 data 目录权限
ls -la data/

# 重新初始化
cd packages/author-site
pnpm db:init
```
