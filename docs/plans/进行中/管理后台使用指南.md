# 管理后台使用指南

## 快速开始

### 1. 初始化数据库

首次使用前需要初始化数据库(创建 system_configs 表):

```bash
cd packages/author-site
pnpm db:init
```

### 2. 配置 Admin Secret

在 `.env` 文件中设置管理后台访问密钥:

```env
ADMIN_SECRET=admin-change-this-to-random-string
```

**生产环境必须修改为强随机字符串!**

### 3. 启动服务

```bash
pnpm dev
```

### 4. 访问管理后台

在浏览器中访问:

```
http://localhost:3200/admin?secret=admin-change-this-to-random-string
```

首次访问成功后,系统会自动设置 Cookie,后续访问无需再带 secret 参数:

```
http://localhost:3200/admin
```

## 功能说明

### 模型配置页面 (`/admin/model-config`)

#### 白名单配置

- **作用**: 控制哪些模型分组可以在前端显示
- **格式**: 分组前缀,如 `xjjj/`, `jojo/`
- **示例**:
  ```
  xjjj/
  jojo/
  custom/
  ```

#### 黑名单配置

- **作用**: 禁用特定模型(即使通过了白名单过滤)
- **格式**: 完整模型 ID
- **示例**:
  ```
  xjjj/old-model
  xjjj/test-model
  ```

#### 默认模型列表

- **作用**: 设置用户首次访问时自动选中的模型
- **格式**: 完整模型 ID,按优先级从上到下排列
- **示例**:
  ```
  xjjj/deepseek-v4-flash
  xjjj/gpt-model
  ```

#### 名称过滤器

- **作用**: 分组级别的模型名称关键词过滤
- **格式**: `分组:关键词`
- **示例**:
  ```
  opencode:Free
  xjjj:Pro
  ```

#### 多模态模型

- **作用**: 标记支持图片输入的模型
- **格式**: 完整模型 ID
- **示例**:
  ```
  xjjj/gpt-4-vision
  xjjj/gpt-4o
  ```

## 配置生效机制

### 实时生效

- 前端配置(白名单/黑名单/默认模型/多模态)修改后**立即生效**,无需重启服务
- 系统会自动清除缓存,下次请求时加载新配置

### 向后兼容

- 如果数据库中没有配置,系统会自动 fallback 到环境变量
- 现有的 `.env` 配置仍然有效

### 供应商配置

- **注意**: 供应商配置(API Key, API Base)目前仍需通过 `.env` 文件管理
- 修改供应商配置后需要重启服务:
  ```bash
  docker compose down && docker compose build opencode-serve && docker compose up -d
  ```

## API 文档

### 获取配置

```http
GET /api/admin/model-config
```

**响应**:

```json
{
  "success": true,
  "data": {
    "frontend": {
      "allowedPrefixes": ["xjjj/", "jojo/"],
      "blacklist": [],
      "defaultModelIds": ["xjjj/deepseek-v4-flash"],
      "nameFilters": ["opencode:Free"]
    },
    "multimodalModels": []
  }
}
```

### 更新配置

```http
PUT /api/admin/model-config
Content-Type: application/json
```

**请求体**:

```json
{
  "frontend": {
    "allowedPrefixes": ["xjjj/", "jojo/"],
    "blacklist": ["xjjj/old-model"],
    "defaultModelIds": ["xjjj/deepseek-v4-flash"],
    "nameFilters": []
  },
  "multimodalModels": ["xjjj/gpt-4-vision"]
}
```

### 清除缓存

```http
POST /api/admin/reload-config
```

## 安全注意事项

1. **Admin Secret**: 生产环境必须使用强随机字符串(至少 32 字符)
2. **HTTPS**: 生产环境务必启用 HTTPS,防止 Cookie 被窃取
3. **访问控制**: 建议配合 IP 白名单或 VPN 使用
4. **定期更换**: 定期更换 Admin Secret

## 常见问题

### Q: 访问 /admin 返回 401?

A: 检查 URL 中的 secret 参数是否正确,或检查 `.env` 中的 `ADMIN_SECRET` 配置。

### Q: 配置修改后没有生效?

A: 配置应该立即生效。如果未生效,尝试:

1. 刷新页面
2. 清除浏览器缓存
3. 调用 `/api/admin/reload-config` 强制清除缓存

### Q: 如何恢复默认配置?

A: 如果数据库配置有问题,系统会自动 fallback 到环境变量配置。也可以直接删除数据库中的配置记录:

```sql
DELETE FROM system_configs WHERE id = 'model_config';
```

### Q: 支持多人协作管理吗?

A: 当前版本使用共享的 Admin Secret,适合单人管理。如需多人协作,建议:

1. 使用密码管理工具共享 Secret
2. 或等待后续版本支持用户角色管理

## 技术架构

- **存储**: SQLite `system_configs` 表 (位于 `data/users.db`)
- **鉴权**: Admin Secret + httpOnly Cookie (2 小时过期)
- **缓存**: 内存缓存, TTL 1 分钟
- **前端**: Next.js App Router + shadcn/ui + Tailwind CSS
