# 闯关活动原型导入创作端项目

## 背景

用户提供 `/Users/qh2/Downloads/闯关活动.pdf`，希望将营销活动原型图转换为创作端项目，并且原型图中的每个页面对应创作端项目里的一个页面。

## 目标

- 从 PDF 原型画布中识别独立页面。
- 创建一个新的创作端项目，项目名称为“闯关活动”。
- 将每个独立原型页面导入为一个创作端页面。
- 将已导入页面从整页截图展示升级为尽量代码化还原，保留可预览的视觉结构，并通过项目管理 CLI 完成校验和提交。

## 范围

- 包含 PDF 解析、页面裁剪、项目创建、页面写入、代码化还原、项目校验。
- 代码化还原以 DOM/CSS 结构表达页面布局、文字、卡片、按钮、进度和弹窗；复杂插画、人物、视频内容先用代码化占位表现，不做逐像素插画重绘。
- 不包含重新手工实现全部营销交互逻辑。
- 不包含发布到使用端或线上环境。

## 方案

1. 先低分辨率渲染 PDF 总览，确认该 PDF 是一个单页大画布，画布内包含多个手机页面稿。
2. 将画布中手机页面框裁剪为图片资产。
3. 使用项目管理服务创建新项目和编辑事务。
4. 每个裁剪出的手机页面资产生成一个 React 页面，页面主体展示对应高保真原型图。
5. 运行编辑事务校验、差异检查和提交。
6. 二阶段按用户反馈，将 27 个页面从整页图片展示替换为自包含 React/Tailwind 代码页面。
