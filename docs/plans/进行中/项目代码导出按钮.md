# 项目代码导出按钮

> 状态更新（2026-06-30）：创作端编辑页右上角“导出代码”入口和创作端专用 scaffold 下载接口已被 [创作端导出代码入口移除](./创作端导出代码入口移除.md) 覆盖。后续本地代码获取以创作端 CLI 为准；本文保留为 2026-06-28 添加导出入口的历史实施记录，下文中创作端下载按钮和 `/api/projects/[projectId]/scaffold` 方案均已过期。

## 背景

创作端项目 demo 已经可以通过项目包协议生成本地 scaffold zip，但入口主要面向 CLI 和已有接口，页面上缺少面向正式开发交付的显式导出按钮。使用端项目详情页也需要提供代码导出入口，让访问已发布项目的用户可以下载可本地开发的项目包。

## 目标

- 在创作端项目编辑页右上角添加导出代码按钮。
- 在使用端项目详情页右上角添加公开导出代码按钮。
- 复用现有项目 scaffold zip 结构，不改变项目包协议。
- 使用端导出只允许下载与当前可见发布内容一致的项目。

## 范围

- 创作端项目 scaffold 下载接口与编辑页按钮。
- 使用端公开 scaffold 下载接口与详情页按钮。
- 对应接口测试、项目文档更新和验证。

## 方案

- 历史方案：创作端曾使用 `/api/projects/[projectId]/scaffold`，支持可选 `sessionId`，传入时先校验 session 归属并同步编辑工作区；该方案已在 2026-06-30 移除。
- 使用端新增 `/api/viewer/[projectId]/scaffold` 公开下载接口，只允许已发布且无未发布改动的项目下载。
- 前端按钮使用普通浏览器下载行为，不引入额外流式下载逻辑。
