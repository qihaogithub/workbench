# 使用者视图方案

> 为系统新增"使用者"角色视图，提供仅含预览区和配置面板的轻量项目浏览界面。

---

## 一、背景与目标

### 1.1 用户角色定义

| 角色 | 权限 | 对应界面 |
|------|------|---------|
| **编辑者 (Editor)** | 创建项目、AI 对话编辑代码、管理版本 | 现有 `/projects/` 系列页面 |
| **使用者 (Viewer)** | 浏览项目列表、查看预览、调整配置参数 | **本次新增** |
| **开发者 (Developer)** | 系统管理、扩展开发 | 后续规划 |

### 1.2 核心需求

- 使用者打开首页看到项目列表（同编辑者）
- 点击进入项目后，**不进入编辑页**，而是进入**只读预览页**
- 只读预览页包含：
  - **预览区** — 实时渲染组件（复用现有 PreviewPanel）
  - **配置面板** — 可调整参数（复用现有 ConfigForm）
  - **不包含** AI 对话区和代码编辑区
- 使用者不需要登录/设置用户名，开箱即用

---

## 二、方案设计

### 2.1 路由结构

```
/viewer/
├── page.tsx              # 使用者首页 — 项目列表
└── [projectId]/
    └── page.tsx          # 使用者项目浏览页 — 预览 + 配置
```

> 独立于现有 `/projects/` 路径，避免与编辑者视图逻辑耦合。

### 2.2 首页 — 项目列表 (`/viewer/`)

- **复用** `ProjectApiClient.getProjects()` 获取项目数据
- **复用** 项目卡片样式，但操作按钮改为"查看预览"
- **不显示** 用户名选择器
- **不显示** "新建项目"按钮
- 点击项目跳转到 `/viewer/{projectId}`

### 2.3 项目浏览页 — 预览 + 配置 (`/viewer/[projectId]/`)

#### 页面结构

```
┌─────────────────────────────────────────────────┐
│  ← 返回项目列表  │  项目名称                    │
├──────────────────────┬──────────────────────────┤
│                      │                          │
│    预览区            │    配置面板               │
│    (PreviewPanel)    │    (ConfigForm)           │
│                      │                          │
│                      │                          │
├──────────────────────┴──────────────────────────┤
│  [footer: 项目信息]                              │
└─────────────────────────────────────────────────┘
```

#### 布局方案

- **左右两栏布局**（预览区占 65%，配置面板占 35%）
- 使用 `ResizablePanelGroup`（已存在于项目中）支持拖拽调整比例
- 预览区全高显示，配置面板可滚动

#### 数据获取

- 通过 `ProjectApiClient.getProject(projectId)` 获取项目详情
- 从项目详情中获取最新版本的代码和 schema
- 不需要创建 Session，不需要启动 Agent 子进程

#### 配置面板

- **完全可交互** — 使用者可以调整所有配置参数
- 配置变更实时反映在预览区（同编辑者体验）
- 配置状态保存在组件本地 state，不持久化

#### 预览区

- **复用** `PreviewPanel` 组件
- 编译代码并实时渲染
- 显示运行时错误

#### 差异点（与编辑者视图对比）

| 功能 | 编辑者 | 使用者 |
|------|--------|--------|
| AI 对话 | ✅ | ❌ |
| 代码编辑 | ✅ | ❌ |
| 保存版本 | ✅ | ❌ |
| 新建对话 | ✅ | ❌ |
| 预览 | ✅ | ✅ |
| 配置调整 | ✅ | ✅ |
| 用户名设置 | 需要 | 不需要 |
| Session 创建 | 需要 | 不需要 |

## 三、导航入口

### 3.1 入口方式

采用 **URL 直接访问**，不修改现有页面：

- **编辑者**：`/projects` — 已有的项目列表页和编辑页
- **使用者**：`/viewer` — 新增的项目列表页和预览页

### 3.2 角色说明

用户通过输入不同的 URL 进入对应角色视图。首页 `/`（当前 Demo 列表页）保持不变，不增加角色选择入口。

## 四、实现方案

### 3.1 新增文件清单

| 文件 | 说明 |
|------|------|
| `src/app/viewer/page.tsx` | 使用者首页 — 项目列表页 |
| `src/app/viewer/[projectId]/page.tsx` | 使用者项目浏览页 — 预览+配置 |
| `src/components/viewer/project-viewer-page.tsx` | 项目浏览页客户端组件 |
| `src/components/viewer/viewer-home-page.tsx` | 首页客户端组件 |

### 3.2 组件设计

#### `src/app/viewer/page.tsx`（Server Component）

```tsx
// 服务端组件，获取初始项目数据
export const dynamic = 'force-dynamic';

export default async function ViewerPage() {
  const projects = await listProjects();
  return <ViewerHomePage initialProjects={projects} />;
}
```

#### `src/components/viewer/viewer-home-page.tsx`（Client Component）

```tsx
// 项目列表展示
// 搜索过滤
// 点击跳转到 /viewer/{projectId}
// 无"新建项目"、无用户名选择器
```

#### `src/app/viewer/[projectId]/page.tsx`（Server Component）

```tsx
// 获取项目详情（包含最新版本的代码和 schema）
export default async function ViewerProjectPage({ params }) {
  const project = await getProjectDetail(params.projectId);
  return <ProjectViewerPage project={project} />;
}
```

#### `src/components/viewer/project-viewer-page.tsx`（Client Component）

核心页面，两栏布局：

```tsx
'use client';

export function ProjectViewerPage({ project }) {
  // 状态：当前配置数据
  const [configData, setConfigData] = useState(project.schema?.default || {});
  // 状态：编译后的代码
  const [compiledCode, setCompiledCode] = useState('');

  // 1. 首次加载时编译代码
  useEffect(() => {
    compileCode(project.code).then(setCompiledCode);
  }, [project.code]);

  return (
    <ResizablePanelGroup direction="horizontal">
      {/* 预览区 */}
      <ResizablePanel defaultSize={65}>
        <PreviewPanel
          code={compiledCode}
          sessionId="viewer"
          configData={configData}
        />
      </ResizablePanel>
      {/* 配置面板 */}
      <ResizableHandle />
      <ResizablePanel defaultSize={35}>
        <ConfigForm
          schema={project.schema}
          onChange={setConfigData}
          initialData={configData}
        />
      </ResizablePanel>
    </ResizablePanelGroup>
  );
}
```

### 3.3 数据流

```
Page Load
  │
  ├─→ getProject(projectId) → 获取项目详情
  │     ├── code (最新版本源码)
  │     └── schema (配置定义)
  │
  ├─→ POST /api/compile (首次加载)
  │     └── 编译 code → compiledCode
  │
  └─→ PreviewPanel 渲染
        ├── 传递 compiledCode + configData
        └── iframe 内实时渲染

配置变更
  │
  ├─→ ConfigForm.onChange → 更新 configData state
  └─→ PreviewPanel 发送 UPDATE_CONFIG → iframe 更新
```

### 3.4 API 依赖

- `GET /api/projects/:id` — 获取项目详情（已有接口，需确认返回 code 和 schema）
- `POST /api/compile` — 编译代码（已有接口）
- 不需要 Session 相关接口
- 不需要 Agent Service

### 3.5 导航入口

- 在现有首页（`/`）添加"使用者入口"按钮，链接到 `/viewer/`
- 或者在 Header 中添加导航切换

## 四、需要确认的问题

### 4.1 项目详情 API 返回值

当前 `ProjectApiClient.getProject()` 返回 `ProjectDetailResponse`，需要确认该接口是否包含最新版本的完整 `code` 和 `schema`。如果不包含，有两种方案：

**方案 A：扩展 API** — 在项目详情中增加 `latestCode` 和 `latestSchema` 字段

**方案 B：通过版本 API 获取** — 先获取最新版本 ID，再获取版本详情

**推荐方案 A**，对编辑者视图也有用。

### 4.2 编译依赖

当前编译 API 是否需要 sessionId 或其他上下文？需确认。

### 4.3 路由挂载位置

建议在 `/viewer/` 路径下，与编辑者视图完全独立。但如果需要与编辑者共享域名，也可以考虑：

- `/projects/{id}/view` — 作为项目的子路由
- `/view/{id}` — 短路径

### 4.4 项目列表数据源

首页目前通过服务端 `listProjects()`（在 `fs-utils.ts` 中）获取数据，该函数返回的数据是否包含使用者所需的全部信息？如果不包含，是否需要新增 API。
