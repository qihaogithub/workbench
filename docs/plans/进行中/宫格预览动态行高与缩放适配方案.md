## 一、问题背景

### 1.1 当前现象

项目编辑页预览区宫格模式下，当同一行存在不同尺寸设备页面时，宽屏设备（如 iPad）卡片会占据过大空间，导致视觉不协调。

**典型场景**：
- 第 1 页：手机（375×812，宽高比 0.462）
- 第 2 页：手机（375×812，宽高比 0.462）
- 第 3 页：iPad（1024×768，宽高比 1.333）

### 1.2 根本原因

当前实现使用 CSS Grid 的 `fr` 单位按宽高比分配列宽：

```typescript
// PreviewGrid.tsx L434-L449
const ratios = row.map((p) => {
  const size = p.id === activePageId ? previewSize : (p.previewSize ?? previewSize)
  return getAspectRatioValue(size)  // width / height
})
const columnTemplate = ratios.map((r) => `${r}fr`).join(" ")
```

**计算结果**：
- 三页同行时：`gridTemplateColumns: 0.462fr 0.462fr 1.333fr`
- iPad 宽度占比：1.333 / (0.462 + 0.462 + 1.333) = **59%**
- 单张手机宽度占比：0.462 / 2.257 = **20%**

iPad 卡片占据了近 60% 的行宽，视觉上明显过大。

## 二、当前方案 vs 新方案对比

### 2.1 布局策略对比

| 维度 | 当前方案（fr 比例分配） | 新方案（固定行高 + 宽度自适应） |
|:-----|:----------------------|:------------------------------|
| **布局方式** | CSS Grid `fr` 单位 | Flex 固定高度 |
| **高度控制** | 由宽度和比例间接决定 | 显式固定 |
| **宽度控制** | 按 `fr` 比例分配 | 按 `height × aspectRatio` 计算 |
| **同类卡片** | 高度可能不一致 | 高度严格一致 |
| **跨设备混排** | 宽屏设备过大 | 所有卡片高度相同，视觉统一 |

### 2.2 视觉效果对比

#### 场景：2 手机 + 1 iPad 同行（3 列模式）

**当前方案**：
```
┌─────────┐ ┌─────────┐ ┌─────────────────────────┐
│  手机   │ │  手机   │ │         iPad            │
│  20%宽  │ │  20%宽  │ │         59%宽           │
│  较小   │ │  较小   │ │         过大            │
└─────────┘ └─────────┘ └─────────────────────────┘
```

**新方案**：
```
┌──────────┐ ┌──────────┐ ┌──────────────────┐
│   手机   │ │   手机   │ │      iPad        │
│  高度一致 │ │  高度一致 │ │     高度一致      │
│ 宽度=高×比│ │ 宽度=高×比│ │   宽度=高×比     │
└──────────┘ └──────────┘ └──────────────────┘
```

### 2.3 用户控制维度对比

| 控制项 | 当前方案 | 新方案 |
|:-------|:---------|:-------|
| 每行列数 | ✅ 2/3/4 可选 | ✅ 2/3/4 可选 |
| 整体缩放 | ❌ 不支持 | ✅ 0.5x ~ 2.0x |
| 单页尺寸 | ✅ 配置面板调整 | ✅ 配置面板调整 |
| 行高自适应 | ❌ 固定逻辑 | ✅ 随列数变化 |

## 三、新方案详细设计

### 3.1 核心机制

#### 动态行高计算

根据每行列数计算基础行高：

```typescript
function getBaseRowHeight(columns: number): number {
  const heights: Record<number, number> = {
    2: 500,   // 2列：较宽松，适合细节查看
    3: 380,   // 3列：适中，平衡细节与数量
    4: 300,   // 4列：紧凑，适合概览
  }
  return heights[columns] ?? 380
}

// 实际行高 = 基础行高 × 用户缩放
const actualRowHeight = getBaseRowHeight(gridColumns) * gridScale
```

#### 列宽计算

放弃 `fr` 分配，改为固定高度下的宽度自适应：

```typescript
// 行容器样式
const rowStyle = {
  display: 'flex',
  gap: '16px',
  height: `${actualRowHeight}px`,
  alignItems: 'flex-start',
}

// 单张卡片宽度 = 行高 × 宽高比
const cardWidth = actualRowHeight * aspectRatio
```

### 3.2 状态管理扩展

在 `page.tsx` 新增缩放状态：

```typescript
// 新增：宫格缩放比例（0.5 ~ 2.0）

```

### 3.3 缩放控制 UI

在宫格工具栏添加缩放控件：

```tsx

  <span className="text-xs text-muted-foreground">缩放</span>
  <Button

  <span className="text-xs w-10 text-center">
    {Math.round(gridScale * 100)}%
  </span>
  <Button 
    variant="ghost" 
    size="sm" 
    className="h-7 w-7 p-0"
    onClick={() => setGridScale(s => Math.min(2.0, s + 0.1))}
  >
    <ZoomIn className="h-3.5 w-3.5" />
  </Button>
</div>
```

### 3.4 iframe 缩放适配

`GridIframe` 内部缩放计算调整：

```typescript
// 当前逻辑（按宽度缩放）
const scale = cardWidth > 0 ? cardWidth / iframeWidth : 0.3

// 新逻辑（按高度缩放）
const iframeHeight = parseSizeValue(effective.height) ?? 812
const scale = actualRowHeight / iframeHeight
```

### 3.5 行溢出处理

当一行内卡片总宽度超过容器时，采用**等比压缩策略**：

```typescript
const totalWidth = row.reduce(
  (sum, p) => sum + actualRowHeight * getAspectRatioValue(p.previewSize),
  0
)
const containerWidth = containerRef.current?.clientWidth ?? 0
const compressRatio = totalWidth > containerWidth 
  ? (containerWidth - gapTotal) / totalWidth 
  : 1

// 实际使用高度
const finalHeight = actualRowHeight * compressRatio
```

#### Step 3：重构 PreviewGrid.tsx 布局

```tsx
function getBaseRowHeight(columns: number): number {
  const map: Record<number, number> = { 2: 500, 3: 380, 4: 300 }
  return map[columns] ?? 380
}

export function PreviewGrid({
  // ...
  gridScale = 1.0,
  onGridScaleChange,
}: PreviewGridProps) {
  const actualRowHeight = getBaseRowHeight(gridColumns) * gridScale
  
  // 行溢出压缩计算
  const getRowHeight = (row: GridPageItem[]) => {
    const gapTotal = (row.length - 1) * 16
    const totalWidth = row.reduce(
      (sum, p) => sum + actualRowHeight * getAspectRatioValue(
        p.id === activePageId ? previewSize : (p.previewSize ?? previewSize)
      ),
      0
    )
    const containerWidth = containerRef.current?.clientWidth ?? 0
    if (totalWidth + gapTotal > containerWidth) {
      return actualRowHeight * ((containerWidth - gapTotal) / totalWidth)
    }
    return actualRowHeight
  }

  return (
    <div ref={containerRef} className="h-full overflow-y-auto">
      {/* 缩放控制栏 */}
      <div className="flex items-center justify-end gap-2 px-4 py-2">
        <span className="text-xs text-muted-foreground">缩放</span>
        <Button onClick={() => onGridScaleChange?.(Math.max(0.5, gridScale - 0.1))}>
          <ZoomOut />
        </Button>
        <span>{Math.round(gridScale * 100)}%</span>
        <Button onClick={() => onGridScaleChange?.(Math.min(2.0, gridScale + 0.1))}>
          <ZoomIn />
        </Button>
      </div>
      
      {/* 宫格内容 */}
      <div style={{ display: 'flex', flexDirection: 'column', gap: '24px', padding: '16px' }}>
        {rows.map((row, rowIndex) => {
          const rowHeight = getRowHeight(row)
          return (
            <div
              key={rowIndex}
              style={{
                display: 'flex',
                gap: '16px',
                height: `${rowHeight}px`,
                justifyContent: 'center',
              }}
            >
              {row.map((page) => {
                const aspectRatio = getAspectRatioValue(
                  page.id === activePageId ? previewSize : (page.previewSize ?? previewSize)
                )
                return (
                  <div
                    key={page.id}
                    style={{
                      height: '100%',
                      width: `${rowHeight * aspectRatio}px`,
                      flexShrink: 0,
                    }}
                  >
                    <GridIframe
                      page={page}
                      rowHeight={rowHeight}  // 传递行高用于内部缩放
                      // ...
                    />
                  </div>
                )
              })}
            </div>
          )
        })}
      </div>
    </div>
  )
}
```

#### Step 4：调整 GridIframe 缩放

function GridIframe({ page, rowHeight, previewSize }: GridIframeProps & { rowHeight: number }) {

## 五、效果预期

### 5.1 视觉改善

| 场景 | 改善前 | 改善后 |
|:-----|:-------|:-------|
| 手机+iPad 混排 | iPad 独占大半行 | 三者高度一致，宽度按实际比例 |
| 多手机并排 | 高度一致（碰巧） | 高度严格一致，可控 |
| 4 列紧凑模式 | 卡片过小难以查看 | 基础行高 300px，可缩放到合适大小 |

### 5.2 交互增强

- **列数切换**：2/3/4 列自动调整基础行高
- **缩放调节**：用户可按需放大/缩小整体预览
- **响应式**：窗口 resize 时自动重新计算压缩比例

---

## 六、风险与回退

| 风险 | 影响 | 缓解措施 |
|:-----|:-----|:---------|
| 用户习惯改变 | 中 | 保留列数切换，缩放默认 100% |
| 极端宽高比卡片 | 低 | 等比压缩确保不溢出 |
| 性能影响 | 低 | 布局计算量与之前相当 |

**回退方案**：如效果不理想，可快速回退到原 `fr` 分配逻辑，仅需恢复 `PreviewGrid.tsx` 的布局部分。
