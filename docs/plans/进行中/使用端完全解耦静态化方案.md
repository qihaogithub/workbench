## 一、背景与目标

### 1.1 问题背景

当前使用端（viewer-site）的预览功能完全依赖创作端（author-site）的实时服务。通过iframe嵌入创作端的viewer端点，如果创作端系统崩溃或不可用，使用端的预览功能将完全失效。

### 1.2 设计目标

实现使用端与创作端的**完全解耦**，确保：

1. **创作端崩溃不影响使用端**：使用端所有功能（项目列表、预览、配置）必须独立运行
2. **静态化部署**：整个viewer端点构建为静态站点，部署到CDN + 静态服务器
3. **手动触发更新**：管理员手动触发重新构建，不依赖自动同步
4. **纯前端配置面板**：配置面板完全在前端实现，配置数据存储在本地或URL参数中
5. **构建时数据生成**：所有数据在构建时从创作端获取，生成静态JSON文件

### 1.3 成功标准

- 使用端100%独立运行，不依赖任何创作端服务
- 预览功能加载时间 < 2秒（CDN缓存后）
- 配置面板响应时间 < 100ms
- 构建后自动验证完整性

#### 3.1.3 静态文件生成

```typescript
async function generateProjectStatic(project: Project, config: BuildConfig) {
  const projectDir = path.join(config.outputDir, 'projects', project.id);
  
  // 1. 创建目录结构
  await fs.mkdir(path.join(projectDir, 'data'), { recursive: true });
  await fs.mkdir(path.join(projectDir, 'demos'), { recursive: true });
  
  // 2. 生成项目数据JSON
  const projectData = {
    id: project.id,
    name: project.name,
    description: project.description,
    updatedAt: project.updatedAt,
    configSchema: project.configSchema,
    demos: project.demos.map(demo => ({
      id: demo.id,
      name: demo.name,
      code: demo.code,
      schema: demo.schema,
      previewSize: demo.previewSize,
    })),
  };
  
  await fs.writeFile(
    path.join(projectDir, 'data', 'project.json'),
    JSON.stringify(projectData, null, 2)
  );
  
  // 3. 生成配置Schema文件
  if (project.configSchema) {
    await fs.writeFile(
      path.join(projectDir, 'data', 'config-schema.json'),
      project.configSchema
    );
  }
  
  // 4. 生成Demo页面Schema文件
  for (const demo of project.demos) {
    if (demo.schema) {
      await fs.writeFile(
        path.join(projectDir, 'demos', demo.id, 'schema.json'),
        demo.schema
      );
    }
  }
}
```

```typescript
// lib/data-loader.ts

class StaticDataLoader {
  private cache = new Map<string, any>();
  
  async loadProjectList(): Promise<ProjectListResponse> {
    const cacheKey = 'project-list';
    if (this.cache.has(cacheKey)) {
      return this.cache.get(cacheKey);
    }
    
    const response = await fetch('/data/projects.json');
    const data = await response.json();
    this.cache.set(cacheKey, data);
    return data;
  }
  
  async loadProjectData(projectId: string): Promise<ProjectData> {
    const cacheKey = `project-${projectId}`;
    if (this.cache.has(cacheKey)) {
      return this.cache.get(cacheKey);
    }
    
    const response = await fetch(`/projects/${projectId}/data/project.json`);
    const data = await response.json();
    this.cache.set(cacheKey, data);
    return data;
  }
}
```

```typescript
// components/StaticConfigPanel.tsx

import { useMemo } from 'react';
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { z } from 'zod';

interface StaticConfigPanelProps {
  schema: string;  // JSON Schema字符串
  initialData?: Record<string, unknown>;
  onChange: (data: Record<string, unknown>) => void;
}

export function StaticConfigPanel({
  schema,
  initialData,
  onChange,
}: StaticConfigPanelProps) {
  // 1. 解析JSON Schema为Zod Schema
  const zodSchema = useMemo(() => {
    const parsed = JSON.parse(schema);
    return jsonSchemaToZod(parsed);
  }, [schema]);
  
  // 2. 初始化表单
  const form = useForm({
    resolver: zodResolver(zodSchema),
    defaultValues: initialData,
  });
  
  // 3. 监听变更
  useEffect(() => {
    const subscription = form.watch((data) => {
      onChange(data);
    });
    return () => subscription.unsubscribe();
  }, [form, onChange]);
  
  // 4. 渲染表单
  return (
    <form onSubmit={form.handleSubmit(onChange)}>
      {renderSchemaFields(schema, form)}
    </form>
  );
}
```

#### 3.2.3 错误边界

```typescript
// components/ErrorBoundary.tsx

interface ErrorBoundaryProps {
  fallback: React.ReactNode;
  children: React.ReactNode;
}

interface ErrorBoundaryState {
  hasError: boolean;
  error?: Error;
}

export class ErrorBoundary extends React.Component<
  ErrorBoundaryProps,
  ErrorBoundaryState
> {
  constructor(props: ErrorBoundaryProps) {
    super(props);
    this.state = { hasError: false };
  }
  
  static getDerivedStateFromError(error: Error) {
    return { hasError: true, error };
  }
  
  componentDidCatch(error: Error, errorInfo: React.ErrorInfo) {
    console.error('预览组件错误:', error, errorInfo);
  }
  
  render() {
    if (this.state.hasError) {
      return this.props.fallback;
    }
    return this.props.children;
  }
}
```

#### 3.3.2 构建后验证

```typescript
// scripts/validate-build.ts

async function validateBuild(outputDir: string) {
  const errors: string[] = [];
  
  // 1. 检查必需文件
  const requiredFiles = [
    'index.html',
    'data/projects.json',
  ];
  
  for (const file of requiredFiles) {
    const filePath = path.join(outputDir, file);
    if (!await fs.pathExists(filePath)) {
      errors.push(`缺少必需文件: ${file}`);
    }
  }
  
  // 2. 验证JSON格式
  const projectsJson = await fs.readJson(path.join(outputDir, 'data/projects.json'));
  if (!projectsJson.projects || !Array.isArray(projectsJson.projects)) {
    errors.push('projects.json格式错误: 缺少projects数组');
  }
  
  // 3. 验证项目目录
  for (const project of projectsJson.projects) {
    const projectDir = path.join(outputDir, 'projects', project.id);
    if (!await fs.pathExists(projectDir)) {
      errors.push(`项目目录不存在: ${project.id}`);
    }
    
    const projectJson = path.join(projectDir, 'data', 'project.json');
    if (!await fs.pathExists(projectJson)) {
      errors.push(`项目数据文件不存在: ${project.id}/data/project.json`);
    }
  }
  
  // 4. 输出验证结果
  if (errors.length > 0) {
    console.error('构建验证失败:');
    errors.forEach(error => console.error(`  - ${error}`));
    process.exit(1);
  } else {
    console.log('构建验证通过');
  }
}
```

## 五、风险与挑战

### 5.1 技术风险

| 风险 | 影响 | 缓解措施 |
|:-----|:-----|:---------|
| **构建时间过长** | 项目多时构建可能需要几分钟 | 增量构建、并行处理 |
| **数据一致性** | 构建时数据可能与创作端不一致 | 版本控制、构建校验 |
| **CDN缓存失效** | 更新后用户可能看到旧版本 | 版本化URL、强制刷新 |
| **浏览器兼容性** | 静态化后可能不支持旧浏览器 | 明确浏览器支持范围 |

### 5.2 业务风险

| 风险 | 影响 | 缓解措施 |
|:-----|:-----|:---------|
| **更新延迟** | 手动触发导致更新不及时 | 提供更新提醒、自动化脚本 |
| **配置丢失** | 用户配置可能丢失 | 本地存储、URL参数备份 |
| **预览差异** | 静态预览可能与动态预览有差异 | 明确差异说明、提供对比工具 |

### 5.3 运维风险

| 风险 | 影响 | 缓解措施 |
|:-----|:-----|:---------|
| **CDN故障** | 静态文件无法访问 | 多CDN源、降级策略 |
| **存储空间** | 静态文件占用存储空间 | 定期清理、压缩存储 |
| **监控缺失** | 无法及时发现问题 | 完善监控、告警机制 |

## 八、总结

本方案通过静态化部署实现使用端与创作端的完全解耦，确保创作端崩溃不影响使用端正常运行。方案采用构建时数据采集 + 纯前端渲染 + CDN部署的架构，具有以下优势：

1. **完全独立**：使用端不依赖任何创作端服务
2. **高性能**：CDN分发，加载速度快
3. **低成本**：静态文件托管成本低
4. **易维护**：架构简单，易于维护

建议按阶段实施，先完成核心功能，再逐步优化和完善。
