## 一、背景与目标

### 1.1 问题背景

当前使用端（viewer-site）的预览功能完全依赖创作端（author-site）的实时服务，存在以下问题：

1. **iframe 强耦合**：viewer-site 的项目预览页和 Demo 预览页都是纯 iframe 壳，实际渲染完全委托给 author-site 的 `/viewer/[projectId]` 路由
2. **编译服务依赖**：author-site 的 `/api/compile` 端点（Sucrase 编译器）在每次预览时实时编译 TSX 代码，viewer-site 无法脱离此服务
3. **数据源依赖**：viewer-site 通过 `AGENT_SERVICE_URL`（默认 `localhost:3201`）获取项目列表和详情，通过 `WEB_URL`（默认 `localhost:3200`）获取缩略图和 iframe 内容
4. **保存即生效**：当前保存流程（`saveEditSession`）将编辑内容直接覆盖到正式 workspace，`/api/viewer/{projectId}/data` 立即返回最新数据，没有"草稿/上线"的中间态

**结论**：author-site 崩溃时，viewer-site 的项目列表（依赖 agent-service）和预览渲染（依赖 author-site）将完全失效。

### 1.2 设计目标

1. **创作端崩溃不影响使用端**：viewer-site 所有功能（项目列表、预览、配置）100% 独立运行
2. **发布按钮驱动**：作者在 author-site 点击"发布"按钮触发，而非管理员手动运行构建脚本
3. **发布时预编译**：发布时将所有 Demo 的 TSX 代码编译为 JS，viewer-site 无需编译服务
4. **纯静态部署**：viewer-site 构建为纯静态站点，同时部署到局域网服务器和 Cloudflare Pages
5. **增量发布**：每次只发布变更的单个项目，不需要全量重建
6. **纯前端配置面板**：配置面板完全在前端实现，配置数据存储在 URL 参数或 localStorage 中
7. **双目标同步**：发布后数据自动同步到局域网服务器（共享卷即时生效）和 Cloudflare（Wrangler 增量上传）

### 1.3 成功标准

- viewer-site 100% 独立运行，不依赖 author-site 或 agent-service 的任何运行时服务
- 预览功能加载时间 < 2 秒（CDN 缓存后）
- 配置面板响应时间 < 100ms
- 单项目发布时间 < 10 秒（局域网即时生效，Cloudflare < 30 秒）
- 发布后自动验证完整性
- 局域网和 Cloudflare 两个部署目标数据一致

### 2.2 与当前架构的对比

| 维度 | 当前架构 | 新架构 |
|:-----|:---------|:-------|
| **项目列表** | SWR → agent-service `GET /api/projects` | 静态 `projects.json` |
| **项目详情** | SWR → agent-service `GET /api/projects/:id` | 静态 `{projectId}/project.json` |
| **缩略图** | `${WEB_URL}${thumbnail}` → author-site | 静态图片文件 |
| **预览渲染** | iframe → author-site `/viewer/:id` | 本地 iframe + 预编译 JS |

| **配置面板** | author-site viewer 页面内的 ConfigForm | viewer-site 前端独立实现 |
| **更新触发** | 保存即生效（无中间态） | 保存=草稿，发布=上线 |
| **部署方式** | Next.js standalone（需 Node.js 运行时） | Next.js static export（局域网 nginx + Cloudflare Pages） |

#### 3.2.2 发布流程实现

发布 API 的核心逻辑位于 `packages/author-site/src/lib/publish-manager.ts`：

```typescript
import { compileCode } from '@/lib/compiler';
import {
  readProjectMeta,
  writeProjectMeta,
  listDemoPages,
  getDemoDirPath,
  getProjectConfigSchema,
  getProjectPath,
  projectExists,
} from '@/lib/fs-utils';
import { createApiSuccess, createApiError } from '@/lib/fs-utils';
import type { Project, DemoPageMeta } from '@opencode-workbench/shared';
import type { PreviewSize } from '@/lib/preview-size';
import { extractPreviewSize } from '@/lib/preview-size';

const PUBLISHED_DIR = path.join(process.env.DATA_DIR || 'data', 'published');

interface PublishResult {
  projectId: string;
  publishedVersion: string;
  publishedAt: number;
  demoCount: number;
  duration: number;
}

export async function publishProject(projectId: string): Promise<PublishResult> {
  const startTime = Date.now();

  if (!projectExists(projectId)) {
    throw new Error('PROJECT_NOT_FOUND');
  }

  const project = readProjectMeta(projectId);
  if (!project) {
    throw new Error('PROJECT_NOT_FOUND');
  }

  const workspacePath = path.join(getProjectPath(projectId), 'workspace');
  const demoPages = listDemoPages(workspacePath);

  if (demoPages.length === 0) {
    throw new Error('NO_CONTENT_TO_PUBLISH');
  }

  const publishedProjectDir = path.join(PUBLISHED_DIR, projectId);
  fs.mkdirSync(publishedProjectDir, { recursive: true });
  fs.mkdirSync(path.join(publishedProjectDir, 'demos'), { recursive: true });

  const publishedDemoPages: PublishedDemoPage[] = [];

  for (const page of demoPages) {
    const demoDir = getDemoDirPath(workspacePath, page.id);
    const codePath = path.join(demoDir, 'index.tsx');
    const schemaPath = path.join(demoDir, 'config.schema.json');

    if (!fs.existsSync(codePath)) continue;

    const tsxSource = fs.readFileSync(codePath, 'utf-8');
    const compileResult = compileCode(tsxSource, project.lockedDependencies);

    const demoPublishDir = path.join(publishedProjectDir, 'demos', page.id);
    fs.mkdirSync(demoPublishDir, { recursive: true });

    fs.writeFileSync(
      path.join(demoPublishDir, 'compiled.js'),
      compileResult.compiledCode,
    );

    let previewSize: PreviewSize | undefined;
    if (fs.existsSync(schemaPath)) {
      const schemaContent = fs.readFileSync(schemaPath, 'utf-8');
      fs.writeFileSync(path.join(demoPublishDir, 'schema.json'), schemaContent);
      previewSize = extractPreviewSize(schemaContent);
    }

    publishedDemoPages.push({
      id: page.id,
      name: page.name,
      order: page.order,
      parentId: page.parentId,
      compiledJsPath: `demos/${page.id}/compiled.js`,
      schemaPath: fs.existsSync(schemaPath) ? `demos/${page.id}/schema.json` : undefined,
      previewSize,
    });
  }

  const projectConfigSchema = getProjectConfigSchema(workspacePath);
  if (projectConfigSchema) {
    fs.writeFileSync(
      path.join(publishedProjectDir, 'config-schema.json'),
      projectConfigSchema,
    );
  }

  if (project.thumbnail) {
    const thumbnailSrc = path.join(getProjectPath(projectId), project.thumbnail);
    if (fs.existsSync(thumbnailSrc)) {
      fs.copyFileSync(thumbnailSrc, path.join(publishedProjectDir, 'thumbnail.png'));
    }
  }

  const currentVersion = project.versions.length > 0
    ? project.versions[project.versions.length - 1].versionId
    : 'v0';

  const publishedProject: PublishedProject = {
    id: project.id,
    name: project.name,
    description: project.description,
    thumbnail: '/data/{projectId}/thumbnail.png',
    publishedVersion: currentVersion,
    publishedAt: Date.now(),
    demoPages: publishedDemoPages,
    demoFolders: project.demoFolders,
    projectConfigSchema: projectConfigSchema ?? undefined,
  };

  fs.writeFileSync(
    path.join(publishedProjectDir, 'project.json'),
    JSON.stringify(publishedProject, null, 2),
  );

  project.publishedVersion = currentVersion;
  project.publishedAt = Date.now();
  writeProjectMeta(projectId, project);

  regenerateProjectsIndex();

  return {
    projectId,
    publishedVersion: currentVersion,
    publishedAt: project.publishedAt,
    demoCount: publishedDemoPages.length,
    duration: Date.now() - startTime,
  };
}

function regenerateProjectsIndex() {
  const projects: ProjectsIndex['projects'] = [];

  if (!fs.existsSync(PUBLISHED_DIR)) return;

  for (const dirName of fs.readdirSync(PUBLISHED_DIR)) {
    const projectJsonPath = path.join(PUBLISHED_DIR, dirName, 'project.json');
    if (!fs.existsSync(projectJsonPath)) continue;

    const data = JSON.parse(fs.readFileSync(projectJsonPath, 'utf-8'));
    projects.push({
      id: data.id,
      name: data.name,
      description: data.description,
      thumbnail: data.thumbnail,
      publishedAt: data.publishedAt,
      publishedVersion: data.publishedVersion,
      demoCount: data.demoPages?.length ?? 0,
    });
  }

  projects.sort((a, b) => b.publishedAt - a.publishedAt);

  const index: ProjectsIndex = {
    projects,
    generatedAt: Date.now(),
  };

  fs.writeFileSync(
    path.join(PUBLISHED_DIR, 'projects-index.json'),
    JSON.stringify(index, null, 2),
  );
}
```

```typescript
import { createApiSuccess, createApiError, readProjectMeta } from '@/lib/fs-utils';

export async function POST(
  request: NextRequest,
  { params }: { params: { projectId: string } },
) {
  try {
    const result = await publishProject(params.projectId);
    return NextResponse.json(createApiSuccess(result));
  } catch (error) {
    if (error instanceof Error) {
      if (error.message === 'PROJECT_NOT_FOUND') {
        return NextResponse.json(createApiError('PROJECT_NOT_FOUND'), { status: 404 });
      }
      if (error.message === 'NO_CONTENT_TO_PUBLISH') {
        return NextResponse.json(createApiError('NO_CONTENT_TO_PUBLISH', '项目没有可发布的Demo页面'), { status: 400 });
      }
    }
    console.error('发布失败:', error);
    return NextResponse.json(createApiError('PUBLISH_FAILED', '发布失败'), { status: 500 });
  }
}
```

新增文件 `packages/author-site/src/app/api/projects/[projectId]/publish-status/route.ts`：

```typescript
import { NextRequest, NextResponse } from 'next/server';
import { createApiSuccess, createApiError, readProjectMeta, projectExists } from '@/lib/fs-utils';

export async function GET(
  _request: NextRequest,
  { params }: { params: { projectId: string } },
) {
  if (!projectExists(params.projectId)) {
    return NextResponse.json(createApiError('PROJECT_NOT_FOUND'), { status: 404 });
  }

  const project = readProjectMeta(params.projectId);
  if (!project) {
    return NextResponse.json(createApiError('PROJECT_NOT_FOUND'), { status: 404 });
  }

  const currentVersion = project.versions.length > 0
    ? project.versions[project.versions.length - 1].versionId
    : undefined;

  const status = !project.publishedVersion
    ? 'never_published'
    : project.publishedVersion === currentVersion
      ? 'published'
      : 'unpublished_changes';

  return NextResponse.json(createApiSuccess({
    projectId: project.id,
    publishedVersion: project.publishedVersion ?? null,
    publishedAt: project.publishedAt ?? null,
    currentVersion: currentVersion ?? null,
    hasUnpublishedChanges: status === 'unpublished_changes',
    status,
  }));
}
```

#### 3.4.3 预览引擎：iframe + 预编译 JS

viewer-site 的预览不再嵌入 author-site 的 iframe，而是自行创建 iframe 并加载预编译 JS。

核心组件 `packages/viewer-site/src/components/StaticPreviewPanel.tsx`：

```typescript
'use client';

import { useEffect, useRef, useState, useCallback } from 'react';
import { generateIframeHtml } from '@/lib/iframe-template';

interface StaticPreviewPanelProps {
  compiledJsUrl: string;
  cssImports?: string[];
  configData?: Record<string, unknown>;
  previewSize?: { width: number; height: number };
}

export function StaticPreviewPanel({
  compiledJsUrl,
  cssImports,
  configData,
  previewSize,
}: StaticPreviewPanelProps) {
  const iframeRef = useRef<HTMLIFrameElement>(null);
  const [iframeSrc, setIframeSrc] = useState<string>('');
  const [ready, setReady] = useState(false);

  useEffect(() => {
    const html = generateIframeHtml();
    const blob = new Blob([html], { type: 'text/html' });
    const url = URL.createObjectURL(blob);
    setIframeSrc(url);
    return () => URL.revokeObjectURL(url);
  }, []);

  useEffect(() => {
    if (!iframeRef.current || !ready) return;

    const handleMessage = (e: MessageEvent) => {
      if (e.data?.type === 'READY') {
        setReady(true);
      }
    };
    window.addEventListener('message', handleMessage);
    return () => window.removeEventListener('message', handleMessage);
  }, [ready]);

  useEffect(() => {
    if (!iframeRef.current || !ready) return;

    iframeRef.current.contentWindow?.postMessage({
      type: 'UPDATE_CODE',
      code: compiledJsUrl,
      configData: configData ?? {},
      cssImports: cssImports ?? [],
      isUrl: true,
    }, '*');
  }, [compiledJsUrl, configData, cssImports, ready]);

  useEffect(() => {
    if (!iframeRef.current || !ready || !configData) return;

    iframeRef.current.contentWindow?.postMessage({
      type: 'UPDATE_CONFIG',
      configData,
    }, '*');
  }, [configData, ready]);

  return (
    <iframe
      ref={iframeRef}
      src={iframeSrc}
      className="w-full h-full border-none"
      sandbox="allow-scripts allow-same-origin"
    />
  );
}
```

### 3.5 双目标部署与数据同步

发布数据存储在 author-site 的 `data/published/` 目录。发布后需要同步到两个独立的目标。

#### 3.5.1 部署目标对比

| 维度 | 目标 A：局域网服务器 | 目标 B：Cloudflare Pages |
|:-----|:---------------------|:--------------------------|
| **用途** | 内部预览、开发调试 | 公网访问、外部展示 |
| **访问方式** | `http://192.168.x.x:3300` | `https://viewer.example.com` |
| **数据同步** | Docker 共享卷，即时生效 | Wrangler CLI 上传，< 30 秒生效 |
| **HTTPS** | 可选（局域网通常不需要） | 自动（Cloudflare 强制 HTTPS） |
| **缓存控制** | nginx 配置 | Cloudflare 自动 + `_headers` 文件 |
| **压缩** | nginx gzip | Cloudflare 自动 Brotli/Gzip |
| **构建产物** | viewer-site `out/` + 共享卷 `published/` | 完整 `out/` 目录（含 `data/`） |
| **增量更新** | 共享卷文件覆盖即生效 | `wrangler pages deploy` 增量上传 |
| **依赖** | Docker + nginx | Wrangler CLI + Cloudflare 账号 |

#### 3.5.2 目标 A：局域网服务器部署

**架构**：nginx 静态文件服务 + Docker 共享卷

```
author-site 容器                viewer-site 容器
┌──────────────┐               ┌──────────────┐
│  写入         │   共享卷      │  nginx 读取   │
│  /app/data/  │───────────────│  /app/data/  │
│  published/  │  app-data     │  (只读)       │
└──────────────┘               └──────────────┘
```

viewer-site 的 nginx 配置将 `/data/` 路径映射到共享卷的 `published/` 目录：

```nginx
# docker/viewer-site/nginx.conf

server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    # viewer-site 静态文件（构建产物）
    location / {
        try_files $uri $uri/ $uri.html /index.html;
    }

    # 发布数据（共享卷映射）
    location /data/ {
        alias /app/data/published/;
        expires 1h;
        add_header Cache-Control "public, must-revalidate";
        add_header Access-Control-Allow-Origin *;

        # projects-index.json 较短缓存
        location = /data/projects-index.json {
            expires 5m;
            add_header Cache-Control "public, must-revalidate";
        }
    }

    # 预编译 JS 长缓存（内容不变）
    location ~* /data/.*/demos/.*\.js$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 3.5.4 发布流程中的双目标同步

在 `publish-manager.ts` 中，发布完成后异步触发 Cloudflare 同步：

```typescript
export async function publishProject(projectId: string): Promise<PublishResult> {
  // ... 前面的编译和快照生成逻辑不变 ...

  regenerateProjectsIndex();

  const cloudflareResult = await syncToCloudflare();

  return {
    projectId,
    publishedVersion: currentVersion,
    publishedAt: project.publishedAt,
    demoCount: publishedDemoPages.length,
    duration: Date.now() - startTime,
    deployTargets: {
      lan: { status: 'synced' },
      cloudflare: cloudflareResult,
    },
  };
}

async function syncToCloudflare(): Promise<{
  status: 'deployed' | 'skipped' | 'failed';
  url?: string;
  error?: string;
}> {
  const deployEnabled = process.env.CLOUDFLARE_DEPLOY_ENABLED === 'true';
  if (!deployEnabled) {
    return { status: 'skipped' };
  }

  try {
    const { execSync } = require('child_process');
    execSync('bash scripts/sync-to-cloudflare.sh', {
      cwd: process.cwd(),
      timeout: 60000,
      env: {
        ...process.env,
        CLOUDFLARE_API_TOKEN: process.env.CLOUDFLARE_API_TOKEN,
        CLOUDFLARE_ACCOUNT_ID: process.env.CLOUDFLARE_ACCOUNT_ID,
        CLOUDFLARE_PROJECT_NAME: process.env.CLOUDFLARE_PROJECT_NAME,
      },
    });
    return {
      status: 'deployed',
      url: `https://${process.env.CLOUDFLARE_PROJECT_NAME}.pages.dev`,
    };
  } catch (error) {
    console.error('[publishProject] Cloudflare 部署失败:', error);
    return {
      status: 'failed',
      error: error instanceof Error ? error.message : 'Unknown error',
    };
  }
}
```

**关键设计决策**：

1. **Cloudflare 同步失败不阻塞发布**：局域网部署是主要目标，Cloudflare 部署失败时发布仍然成功，只是 `deployTargets.cloudflare.status` 为 `failed`
2. **异步执行**：Cloudflare 部署在后台执行，不阻塞 API 响应（局域网已即时生效）
3. **可关闭**：通过 `CLOUDFLARE_DEPLOY_ENABLED` 环境变量控制是否同步到 Cloudflare

#### 3.5.5 两个部署目标的数据路径差异

| 数据路径 | 局域网（nginx alias） | Cloudflare（内嵌 data/） |
|:---------|:----------------------|:-------------------------|
| 项目列表 | `/data/projects.json` → 共享卷 `published/projects-index.json` | `/data/projects.json` → 部署产物内 `data/projects.json` |

## 五、风险与挑战

### 5.1 技术风险

| 风险 | 影响 | 缓解措施 |
|:-----|:-----|:---------|
| **预编译结果与实时编译不一致** | 发布后预览效果与编辑时不同 | 发布时使用与 `/api/compile` 完全相同的 `compileCode()` 函数，确保编译逻辑一致 |
| **esm.sh CDN 不可用** | 预编译 JS 中的 import 无法解析 | 发布时将依赖版本锁定到 `lockedDependencies`，esm.sh 高可用；极端情况可切换 `CDN_BASE_URL` |
| **发布时编译失败** | 单个 Demo 编译错误阻塞整个发布 | 跳过编译失败的 Demo 并在发布结果中标记，不阻塞其他 Demo 的发布 |
| **iframe 跨域限制** | viewer-site 与 esm.sh 之间的 CORS | iframe 内通过 importmap + Blob URL 加载，不涉及跨域请求 |
| **静态导出路由限制** | Next.js export 不支持动态路由 `[id]` | 使用 `generateStaticParams` 或客户端路由处理 |
| **Cloudflare 部署失败** | 公网用户看到旧版本 | Cloudflare 部署失败不阻塞发布，局域网仍即时生效；支持手动重试 `sync-to-cloudflare.sh` |
| **双目标数据不一致** | 局域网和 Cloudflare 展示内容不同 | 使用相同的 `published/` 数据源，同步脚本保证一致性；Cloudflare 部署成功后验证 `projects.json` 可访问 |
| **Wrangler 超时** | 大型项目上传时间过长 | 增量上传（仅上传变更文件）；设置 60 秒超时，失败后可手动重试 |

### 5.2 业务风险

| 风险 | 影响 | 缓解措施 |
|:-----|:-----|:---------|
| **发布延迟** | 作者忘记发布，使用端看到旧版本 | 在版本历史页明确显示"有未发布变更"状态 |
| **配置丢失** | 用户配置可能丢失 | localStorage 持久化 + URL 参数备份 |
| **预览差异** | 静态预览可能与编辑时预览有细微差异 | 使用相同编译器和 iframe 模板，差异仅来自数据时效性 |

### 5.3 运维风险

| 风险 | 影响 | 缓解措施 |
|:-----|:-----|:---------|
| **共享卷同步延迟** | Docker 场景下文件系统可能有延迟 | 使用 `inotify` 或发布后主动通知 viewer-site 刷新 |
| **存储空间增长** | 每次发布会生成新文件 | 发布时覆盖旧文件（非追加），定期清理未引用的发布数据 |
| **Cloudflare API 限流** | 频繁发布触发 Cloudflare API 限制 | 正常发布频率不会触发；必要时使用 Cloudflare API Token 提高限额 |
| **Cloudflare 缓存失效** | 发布后公网用户仍看到旧版本 | `_headers` 配置 `must-revalidate`；Cloudflare Pages 部署自动清除旧缓存 |

## 七、总结

本方案通过"发布按钮 + 发布时预编译 + viewer-site 纯静态 + 双目标部署"实现使用端与创作端的完全解耦：

1. **发布按钮**：将"保存即生效"改为"保存=草稿，发布=上线"，作者自主控制发布节奏
2. **发布时预编译**：发布时调用 `compileCode()` 将 TSX 编译为 JS 并重写 CDN URL，viewer-site 无需编译服务
3. **增量发布**：每次只发布单个项目，不需要全量重建
4. **纯静态部署**：viewer-site 改为 `output: "export"`，同时部署到局域网 nginx 和 Cloudflare Pages
5. **双目标同步**：发布后局域网即时生效（共享卷），Cloudflare 通过 Wrangler 增量上传（< 30 秒）

相比 v1.0 方案（管理员手动运行全量构建脚本），本方案在以下方面更优：

| 维度 | v1.0 方案 | v2.1 方案 |
|:-----|:---------|:----------|
| 触发者 | 管理员 | 项目作者 |
| 构建粒度 | 全量（所有项目） | 增量（单个项目） |
| 工作流 | 编辑→保存→等管理员构建 | 编辑→保存→发布 |
| 编译时机 | 构建时（独立脚本） | 发布时（复用 author-site 编译器） |
| 数据来源 | HTTP 请求采集 | 直接读取文件系统 |
| 一致性保证 | 依赖网络请求 | 进程内调用，保证一致 |
| 部署目标 | 单一 CDN | 局域网 + Cloudflare 双目标 |
| 公网访问 | 需额外配置 | Cloudflare Pages 自动 HTTPS + 全球 CDN |
