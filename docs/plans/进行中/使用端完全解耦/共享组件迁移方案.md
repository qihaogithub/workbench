## 一、背景与目标

### 1.1 问题背景

author-site 和 viewer-site 的 `components/demo/` 目录下存在 **13 个同名文件**，其中 6 个完全一致、2 个仅注释差异、3 个小幅差异、2 个架构差异较大。当前两套代码独立维护，每次修改需同步两处，已出现不一致：

- 创作端缺少宫格卡片边框闪烁效果（使用端已实现）
- 创作端点击目录时卡片尺寸闪烁 bug（使用端无此问题）
- `iframe-template.ts` 两端实现不同步（viewer-site 有 `supportUrlMode` + `URL.revokeObjectURL` 修复，author-site 缺失）

### 1.2 设计目标

1. **单一源码**：共享组件只存在一份代码，两端从同一包导入
2. **零行为差异**：两端使用共享组件后，预览面板、配置面板、宫格视图行为完全一致
3. **最小侵入**：不改变现有 API 接口，仅迁移文件位置和调整导入路径
4. **保持 shared 包纯逻辑定位**：`@opencode-workbench/shared` 继续只放纯类型+纯函数，React 组件放入新包

### 1.3 成功标准

- `pnpm typecheck` 和 `pnpm lint` 通过
- author-site 编辑页功能不受影响（编译、预览、配置面板、宫格视图）
- viewer-site 预览页功能不受影响（项目列表、预览、配置面板）
- 两端宫格视图行为一致（卡片闪烁、尺寸计算）

### 2.2 关键差异详解

#### PreviewPanel.tsx

viewer-site 新增了 `compiledJsUrl` URL 加载模式：

- 新增 `compiledJsUrl?: string` 和 `cssImports?: string[]` props
- 新增 `sendUpdateCodeUrl` 回调，通过 `isUrl: true` 标记发送 URL
- 编译 effect 中增加 `isUrlMode` 分支，跳过 `/api/compile` 请求
- `generateIframeHtml` 传入 `{ supportUrlMode: isUrlMode }`

合并方案：将 `compiledJsUrl`/`cssImports` 作为可选 props，author-site 不传即走编译模式。

#### PreviewGrid.tsx

两端架构根本不同：

- author-site：`GridIframe` 负责编译+渲染，通过 `/api/compile` 实时编译
- viewer-site：`ViewerGridIframe` 只负责渲染，通过 `postMessage` 发送 URL 指令
- `useVisiblePages` 接口不同
- 工具栏、对齐模式、滚动行为不同

合并方案：Phase 3 抽取公共 hooks，两端的 GridIframe 分别保留。

#### types.ts

- viewer-site 多了 `compiledJsUrl?: string` 和 `cssImports?: string[]`
- author-site 多了 `flashCardId?: string`
- 合并：全部作为可选属性

## 三、架构设计

### 3.1 包结构

在 `@opencode-workbench/shared` 包内新增 `demo` 子目录，存放 React UI 组件。通过 package.json `exports` 字段提供两个入口：

```
@opencode-workbench/shared          → 纯类型 + 纯函数（现有入口，agent-service 也可用）
@opencode-workbench/shared/demo     → React UI 组件（仅前端站点使用）
```

选择此方案而非创建独立 `viewer-ui` 包的原因：

1. 减少包管理复杂度，不增加 workspace 成员
2. 共享组件与共享类型天然关联，放在一起便于维护
3. `agent-service` 只引用默认入口，不会引入 React 依赖

### 3.2 目录结构

```
packages/shared/
├── src/
│   ├── index.ts              ← 现有入口（纯类型+纯函数），不变

│   ├── workspace.ts          ← 现有，不变
│   └── demo/                 ← 新增：共享 UI 组件
│       ├── index.ts          ← 统一导出
```

### 3.4 路径别名策略

共享组件中的 `@/components/ui/*` 和 `@/lib/utils` 引用需要解决。方案：

**共享组件内部使用相对路径导入 `cn()` 等工具**，不依赖宿主包的路径别名。具体做法：

1. 在 `packages/shared/src/demo/` 下创建 `utils.ts`，导出 `cn()` 函数
2. 共享组件内部 `import { cn } from "./utils"` 而非 `@/lib/utils`
3. shadcn/ui 组件（Button, Switch, Slider 等）仍由各宿主包提供，共享组件通过 `@/components/ui/*` 引用
4. 两个宿主包均配置 `@/components/ui/*` 路径别名指向各自的 shadcn/ui 组件目录

### 3.5 关键设计决策

#### D1: PreviewPanel 双模式

```typescript
interface PreviewPanelProps {
  code?: string;              // 编译模式：TSX 源码
  sessionId?: string;         // 编译模式：会话 ID
  demoId?: string;            // 编译模式：Demo ID
  compiledJsUrl?: string;     // URL 模式：预编译 JS 地址
  cssImports?: string[];      // URL 模式：CSS 导入列表
  configData?: Record<string, unknown>;
  previewSize?: PreviewSize;
  snapshotVersion?: number;
}
```

- `compiledJsUrl` 存在时走 URL 模式
- 否则走编译模式（调用 `/api/compile`）
- 两种模式共享：缩放逻辑、错误展示、ResizeObserver、滚动条隐藏

#### D2: iframe-template 合并

以 viewer-site 版本为基础（含 `supportUrlMode` + `URL.revokeObjectURL`），增加 author-site 的 `cdnBaseUrl` 可配置能力：

```typescript
interface IframeTemplateOptions {
  cssImports?: string[];
  compiledCode?: string;
  configData?: Record<string, unknown>;
  cdnBaseUrl?: string;
  supportUrlMode?: boolean;
}
```

#### D3: types.ts 合并

将两端差异属性全部作为可选属性合并：

```typescript
interface PreviewPanelProps {
  // ... 现有属性 ...
  compiledJsUrl?: string;   // 来自 viewer-site
  cssImports?: string[];    // 来自 viewer-site
}

interface PreviewGridProps {
  // ... 现有属性 ...
  flashCardId?: string;     // 来自 author-site
}
```

## 五、风险与缓解

| 风险                                   | 影响                     | 缓解措施                                                     |
| -------------------------------------- | ------------------------ | ------------------------------------------------------------ |
| shadcn/ui 路径别名解析失败             | 共享组件无法找到 UI 组件 | 两个站点均配置 `@/components/ui/*` 别名；`transpilePackages` 已包含 shared 包 |
| author-site 回归                       | 提取组件破坏现有功能     | 逐文件迁移 + 每步 typecheck 验证                             |
| peerDependencies 版本冲突              | React 版本不一致         | monorepo 内版本统一管理                                      |
| `@dnd-kit` 依赖增大 viewer-site 包体积 | 首屏加载变慢             | tree-shaking 移除未使用代码；@dnd-kit 仅 ConfigForm 排序使用 |
| agent-service 误引 React 组件          | 构建失败                 | `exports` 分入口，agent-service 只用默认入口                 |
