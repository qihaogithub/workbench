# 画布模式配置面板按需显示优化方案

> 日期：2026-06-06
> 状态：待实施

## 一、需求概述

### 创作端

画布模式下，配置面板默认隐藏，画布撑满整个右侧区域。点击画布中的页面时，配置面板从右侧滑出显示该页面的配置项，画布自动收缩让出空间；取消选中页面（点击画布空白区域 / 按 Escape）时，配置面板收起隐藏，画布重新撑满。

单页模式和宫格模式的行为不变，配置面板始终显示。

### 使用端（预览端）

画布模式下，配置面板的显隐行为由 URL 参数控制，支持两种模式：
- **常驻模式**（默认）：配置面板始终显示，与当前行为一致
- **按需显示模式**：点击页面才显示配置面板，点击空白隐藏

单页模式和宫格模式的行为不变。

## 二、现状分析

### 2.1 配置面板当前渲染位置

| 端 | 文件 | 配置面板位置 |
|---|---|---|
| 创作端 | `packages/author-site/src/app/demo/[id]/edit/page.tsx` | 三栏 `ResizablePanelGroup` 的右侧面板，始终存在 |
| 预览端 | `packages/author-site/src/app/viewer/[projectId]/page.tsx` | 右侧固定宽度区域，通过 URL 参数 `config=false` 控制显隐 |
| 嵌入端 | `packages/author-site/src/app/embed/[demoId]/EmbedConfigPanel.tsx` | 单页模式右侧 / 多页模式下方，始终显示 |

### 2.2 画布模式选中机制

创作端画布模式已有 `canvasEditingPageId` 状态：

- 点击页面 → `onPageConfigEdit(pageId)` → `setCanvasEditingPageId(pageId)` + `setActiveDemoId(pageId)`
- 点击画布空白 → `onCanvasClick()` → `setCanvasEditingPageId(null)`
- 但 `activeDemoId` 不变（保持最后选中的页面数据加载状态）

**⚠️ 关键问题**：`CanvasPageItem` 的 `onConfigEdit` 回调当前是**死代码**——该 prop 被重命名为 `_onConfigEdit`，组件内部从未调用。因此 `onPageConfigEdit` 实际上从未被触发，`canvasEditingPageId` 从未通过画布点击被设置。必须在 `CanvasPageItem` 中添加页面点击事件来触发 `onConfigEdit`，这是整个方案的前提条件。

### 2.3 配置面板在画布模式的特殊处理

当前画布模式下配置面板**始终显示**，只是去掉了 `ConfigScopeWrapper` 的标题和分隔线，直接渲染两个 `ConfigForm`（更紧凑）。这是为了节省空间，但面板仍然占据右侧 25% 的宽度。

## 三、设计方案

### 3.1 核心思路

**配置面板的显隐由「端 + 预览模式 + 页面选中状态」共同决定：**

| 端 | 预览模式 | 配置面板行为 |
|---|---|---|
| 创作端 | 单页模式 | 始终显示（不变） |
| 创作端 | 宫格模式 | 始终显示（不变） |
| 创作端 | 画布模式 | **按需显示**：选中页面时显示，无选中时隐藏，画布自动撑满 |
| 预览端 | 单页模式 | 始终显示（不变） |
| 预览端 | 宫格模式 | 始终显示（不变） |
| 预览端 | 画布模式 | **可配置**：URL 参数 `canvasConfig=always`（默认常驻）或 `canvasConfig=onclick`（按需显示） |

### 3.2 创作端实现方案

#### 3.2.1 右侧面板条件渲染

在 `DemoEditPage` 中，右侧 `ResizablePanel` 的渲染逻辑改为：

```tsx
// 判断配置面板是否可见
const isConfigPanelVisible = previewMode !== "canvas" || !!canvasEditingPageId;
```

**方案 A：动态调整三栏布局（推荐）**

当配置面板隐藏时，中间预览区自动扩展占满右侧空间。使用条件渲染替代固定三栏：

```tsx
<ResizablePanelGroup direction="horizontal" ...>
  {/* 左侧面板 - 不变 */}
  <ResizablePanel>...</ResizablePanel>

  {/* 中间预览区 - 不变 */}
  <ResizablePanel>...</ResizablePanel>

  {/* 右侧配置面板 - 条件渲染 */}
  {isConfigPanelVisible && (
    <ResizablePanel
      className="border-l bg-card flex flex-col"
      defaultSize={25}
      minSize={20}
    >
      {/* 配置面板内容 */}
    </ResizablePanel>
  )}
</ResizablePanelGroup>
```

**注意**：`ResizablePanelGroup` 在子面板数量变化时会自动重新分配空间，中间预览区会自动扩展。

**方案 B：配置面板覆盖层（备选）**

将画布模式的配置面板改为从右侧滑入的覆盖层（overlay），不影响 `ResizablePanelGroup` 布局。优点是布局稳定不跳动，缺点是会遮挡部分画布内容。

**选择方案 A**，理由：
1. 画布模式需要最大化预览空间，覆盖层会遮挡内容
2. `ResizablePanel` 的条件渲染在 shadcn/ui 中已有成熟用法
3. 面板显隐时布局变化符合用户预期（类似 Figma 的属性面板）

#### 3.2.2 配置面板内容调整

画布模式下选中页面时，配置面板需要显示当前选中页面的信息：

```tsx
{isConfigPanelVisible && (
  <ResizablePanel className="border-l bg-card flex flex-col" defaultSize={25} minSize={20}>
    <div className="px-4 py-3 border-b">
      <h2 className="text-sm font-medium">配置面板</h2>
      {previewMode === "canvas" && canvasEditingPageId && (
        <p className="text-xs text-muted-foreground mt-0.5">
          {demoPages.find(p => p.id === canvasEditingPageId)?.name}
        </p>
      )}
      {previewMode !== "canvas" && (
        <p className="text-xs text-muted-foreground mt-0.5">
          修改配置项，预览区将实时更新
        </p>
      )}
    </div>
    <ScrollArea className="flex-1">
      <div className="p-4 flex flex-col">
        {/* 配置表单内容 - 与现有逻辑一致 */}
      </div>
    </ScrollArea>
  </ResizablePanel>
)}
```

#### 3.2.3 画布模式切换时的状态处理

- 从单页/宫格切换到画布 → `setCanvasEditingPageId(null)`，配置面板隐藏
- 从画布切换到单页/宫格 → 配置面板自动显示（`isConfigPanelVisible` 为 true）
- 画布模式下点击页面树中的页面 → 同时设置 `canvasEditingPageId`，配置面板显示

在 `setPreviewMode` 调用处增加：

```tsx
const handlePreviewModeChange = useCallback((mode: PreviewMode) => {
  setPreviewMode(mode);
  if (mode === "canvas") {
    setCanvasEditingPageId(null); // 进入画布模式时清除选中
  }
}, []);
```

### 3.3 预览端实现方案

预览端（`ViewerProjectPage`）当前没有 `canvasEditingPageId` 概念，需要新增。

#### 3.3.1 新增选中状态

```tsx
const [canvasSelectedPageId, setCanvasSelectedPageId] = useState<string | null>(null);
```

#### 3.3.2 配置面板条件渲染

```tsx
const isConfigPanelVisible = showConfig && (
  previewMode !== "canvas" || !!canvasSelectedPageId
);
```

#### 3.3.3 PreviewCanvas 事件处理

```tsx
<PreviewCanvas
  editable={false}
  ...
  onPageConfigEdit={(pageId) => {
    setCanvasSelectedPageId(pageId);
    handlePageChange(pageId);
  }}
  onCanvasClick={() => {
    setCanvasSelectedPageId(null);
  }}
/>
```

#### 3.3.4 配置面板渲染

预览端画布模式选中页面时，配置面板从右侧滑出：

```tsx
{isConfigPanelVisible && (
  <div className="border-l shrink-0 flex flex-col" style={{ width: configWidth }}>
    {/* 配置面板内容 */}
  </div>
)}
```

### 3.4 嵌入端影响分析

嵌入端（`EmbedConfigPanel.tsx`）不使用画布模式，**无需修改**。

### 4.2 CanvasPageItem

**无需修改**。`onConfigEdit` 回调已存在，点击事件已正确触发。

### 4.3 CanvasViewport

**无需修改**。空白区域点击已通过 `onCanvasClick` 传递。

### 4.4 ConfigForm / ConfigScopeWrapper

**无需修改**。这些组件只负责渲染配置表单，不关心显隐逻辑。

## 五、组件维护策略

### 5.1 职责边界

| 组件 | 职责 | 是否感知配置面板 |
|---|---|---|
| `PreviewCanvas` | 画布渲染、交互、事件上报 | 否 |
| `CanvasPageItem` | 单页面渲染、拖拽、点击 | 否 |
| `CanvasViewport` | 视口平移缩放 | 否 |
| `CanvasToolbar` | 缩放工具栏 | 否 |
| `ConfigForm` | 配置表单渲染 | 否 |
| `ConfigScopeWrapper` | 配置作用域包装 | 否 |
| **页面级组件**（DemoEditPage / ViewerProjectPage） | **组合以上组件，决定配置面板显隐** | **是** |

**关键原则**：配置面板的显隐是**页面级布局决策**，不属于任何共享组件的职责。共享组件只负责上报事件（`onPageConfigEdit` / `onCanvasClick`），由页面级组件根据预览模式和选中状态决定配置面板的渲染。

### 5.2 三种模式的配置面板一致性

| 模式 | 配置面板位置 | 显隐逻辑 | ConfigScopeWrapper |
|---|---|---|---|
| 单页 | 右侧面板 | 始终显示 | 使用（带标题和分隔线） |
| 宫格 | 右侧面板 | 始终显示 | 使用（带标题和分隔线） |
| 画布 | 右侧面板 | 按需显示 | 使用（带标题和分隔线） |

**变更**：画布模式恢复使用 `ConfigScopeWrapper`，与单页/宫格模式保持一致。之前的"去掉标题和分隔线"是为了在面板始终显示时节省空间，现在面板按需显示，空间不再是问题，统一渲染逻辑反而更好维护。

## 六、交互细节

### 6.1 画布模式页面选中视觉反馈

- 选中页面：蓝色 ring 高亮（已有 `isEditing` 状态，`CanvasPageItem` 已实现）
- 取消选中：ring 消失

### 6.2 配置面板动画（可选）

配置面板出现/消失时可添加过渡动画，提升体验：

```tsx
<ResizablePanel
  className={cn(
    "border-l bg-card flex flex-col transition-all duration-200",
    !isConfigPanelVisible && "w-0 overflow-hidden opacity-0"
  )}
>
```

但 `ResizablePanel` 的条件渲染与动画可能冲突，建议**先不做动画**，确认基本功能稳定后再考虑。

### 6.3 键盘交互

- `Escape` 键：取消选中页面，配置面板隐藏（在画布模式已有 `onCanvasClick` 逻辑，需在 `CanvasViewport` 中增加 Escape 键监听）

## 八、风险与注意事项

1. **CanvasPageItem onConfigEdit 是死代码**：当前 `onConfigEdit` 被重命名为 `_onConfigEdit`，从未被调用。这是整个方案的前提条件，必须先修复（Step 0）。

2. **ResizablePanel 条件渲染**：移除面板后，剩余面板会自动扩展。需验证 `defaultSizes` 和 `minSizes` 在面板数量变化时的行为是否正常。如果出现布局抖动，可改用 CSS `display: none` 隐藏面板内容但保留面板占位（宽度设为 0）。

3. **activeDemoId 与 canvasEditingPageId 的关系**：当前 `canvasEditingPageId` 只控制蓝色 ring 高亮，`activeDemoId` 控制配置面板数据。画布模式取消选中后 `activeDemoId` 不变，但配置面板隐藏，所以不影响。再次点击同一页面时，需要确保 `canvasEditingPageId` 重新设置。

4. **预览端只读模式**：预览端配置面板是只读的（`readonly`），按需显示的交互在只读场景下同样合理——用户点击页面查看其配置，点击空白收起。

5. **postMessage 通信**：预览端通过 `VIEWER_SET_PAGE` 消息切换页面时，需要同步设置 `canvasSelectedPageId`。当前 `VIEWER_SET_PAGE` 处理不完整（仅设置 `activeDemoId`，不同步 `configData` 和 `previewSize`），需一并修复。

6. **预览端 URL 参数不支持 canvas 初始模式**：当前 `mode` URL 参数只识别 `"grid"`，其他值（包括 `"canvas"`）都默认为 `"single"`。需要扩展支持 `mode=canvas`。

7. **setPreviewMode 无包装函数**：创作端有 5 处直接调用 `setPreviewMode`，需要统一改为包装函数 `handlePreviewModeChange`，或使用 `useEffect` 监听 `previewMode` 变化来清除 `canvasEditingPageId`。

8. **预览端未传递 onCanvasClick**：当前 viewer 页面的 PreviewCanvas 调用中没有传递 `onCanvasClick` prop，按需显示模式下需要添加。
