# 工作空间术语统一治理

## 背景

项目级共享 Workspace 重构后，代码和文档中仍同时存在“正式工作空间”“临时工作空间”“草稿工作区”“active Workspace”“canonical Workspace”等称呼。不同称呼容易让后续开发误以为默认编辑仍是用户私有副本，进而破坏多人协作和实时保存模型。

## 目标

- 统一当前项目管理链路里的工作空间术语。
- 将默认编辑态明确为“项目工作区”，英文实现名对应 `active Workspace`。
- 将项目目录下的 `projects/{id}/workspace/` 明确为“项目基准工作区”，英文实现名对应 `canonical Workspace`。
- 将 `tempWorkspace` 降级为旧 API 兼容字段，不再作为新代码和文档的主语义。

## 范围

- 创作端项目管理相关需求与技术文档。
- Session API、共享类型和编辑页内部使用的命名。
- 不批量改写历史复盘文档中的旧称；历史文档只保留当时语境。

## 方案

1. 在项目管理工作区技术文档中建立统一术语表。
2. 将“草稿工作区”文档标题、文件名和索引描述调整为“项目工作区”。
3. Session API 新增 `workspacePath`，继续返回 `tempWorkspace` 作为兼容别名。
4. 编辑页内部状态从 `tempWorkspace` 迁移为 `workspacePath`，减少旧术语继续扩散。
