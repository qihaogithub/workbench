# 创作端核心流程回归用例脚本

## 背景

创作端已有 Playwright E2E 入口覆盖项目新建、编辑、保存流程，但核心流程需要更明确的回归用例脚本，便于在功能变更后快速验证登录、新建项目、编辑页会话写入、保存合并、持久化读取和清理删除是否仍可用。

## 目标

- 新增或完善创作端核心流程的 Playwright 回归用例。
- 用例需要覆盖真实 UI 入口、关键 API 响应断言和保存后的持久化校验。
- 更新测试说明文档，明确运行方式和覆盖范围。

## 范围

- `test/新建-编辑-保存项目测试/` 下的 Playwright 测试脚本与说明。
- 根目录测试命令按需补充专用回归入口。
- 与测试覆盖相关的项目文档。

不改动创作端业务代码，不处理无关的工作区已有变更。

## 方案

在现有 Playwright 配置下新增独立核心流程 spec。测试登录后从首页创建项目，进入编辑页后通过创作端公开 API 写入页面代码与 Schema，调用保存合并接口，再通过项目读取接口验证保存内容已落盘，最后删除测试项目完成清理。

## 剩余风险

- 用例依赖本地 author-site 服务、登录账号和数据目录；环境不可用时会失败。
- 若创建或保存接口在超时前异常中断，可能短暂留下 `核心流程回归-` 前缀项目，需要按测试输出中的项目 ID 删除。

## 验证方式

- 静态检查新增测试脚本类型与 Playwright 配置兼容。
- 如本地服务可用，运行专用核心流程回归命令。
- 若服务未启动或账号环境缺失，记录未执行原因和剩余风险。

## 风险与待确认事项

- E2E 依赖 author-site 服务、测试账号和本地数据目录，环境未准备时只能做静态验证。
- 测试项目删除失败会留下以 `核心流程回归-` 开头的测试数据，需要人工或后续脚本清理。
