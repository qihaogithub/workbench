# 创作端预览区加载变慢排查

## 背景

用户反馈创作端以前预览区加载很快，现在预览区预览加载耗时明显变长。当前工作区已有多处预览、编译、项目数据和文档改动，本次排查需在不回滚既有改动的前提下定位慢路径。

## 目标

- 找出创作端单页预览区加载变慢的直接原因。
- 在尽量小的改动范围内恢复预览区首屏和切页加载速度。
- 补充必要测试，避免相同回归再次出现。

## 范围

- 涉及范围：创作端 demo 编辑页、`@opencode-workbench/demo-ui` 预览组件、预览编译缓存、预览运行时加载链路。
- 暂不涉及：截图服务性能、使用端独立预览、真实 LLM/E2E、清理既有数据文件或未提交改动。

## 方案

1. 用 CodeGraph 定位 `PreviewPanel`、`PreviewCanvas`、编辑页和 `/api/compile` 的调用关系。
2. 对比当前 dirty diff 中预览链路的行为变化，重点检查是否出现重复编译、重复 iframe 初始化、缓存失效或渲染门控延迟。
3. 实施最小修复并补充对应单元测试。
4. 运行与改动范围匹配的验证命令，记录结果和剩余风险。
