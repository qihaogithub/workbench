# 创作端画布多类型节点工具方案

> 创建日期：2026-06-25
> 状态：已完成
> 范围：创作端画布节点扩展、工具栏新增节点入口、文档/图片节点浏览与编辑交互

## 背景

当前创作端画布以页面预览节点为核心，适合做多页面总览和页面配置入口。用户现在希望在同一画布中补充更多创作素材：文档说明和图片素材，使画布从“页面预览工作台”扩展为轻量素材看板。

## 目标

- 画布除当前页面节点外，支持文档和图片两类节点。
- 工具栏提供添加文档入口。
- 文档节点在画布内只浏览 Markdown 渲染结果，双击或点击悬浮编辑按钮打开编辑弹窗。
- 图片节点支持直接拖入画布，或画布中使用键盘粘贴添加后展示。
- 外部网页插入功能删除，不再提供工具栏入口、弹窗或 iframe 节点展示。
- 保持已有页面节点、截图 fallback、拖拽缩放和布局保存能力稳定。

## 范围

- 涉及 `packages/shared/src/demo/` 中画布类型、工具栏、节点渲染和画布容器。
- 涉及 `packages/author-site/src/app/demo/[id]/edit/page.tsx` 中画布新增节点入口的接入。
- 复用现有 shadcn/ui、lucide-react、MarkdownIt 与浏览器 File API，不引入新的 UI 或状态库。
- 暂不实现后端资产持久化上传；图片节点先以浏览器 data URL 存入画布布局，后续如需要发布链路再接入项目图片资产。

## 方案

- 在 `CanvasState` 中增加 `nodes` 集合，保留 `pages` 作为页面节点布局，避免破坏既有页面布局读取逻辑。
- 新增 `CanvasFreeNode` 联合类型，覆盖文档和图片节点的内容、尺寸和布局。
- `PreviewCanvas` 统一渲染页面节点与自由节点，复用外层拖拽/缩放交互。
- `CanvasToolbar` 增加文档图标按钮；图片只通过画布根容器拖拽/粘贴进入。
- 文档通过弹窗收集内容；文档编辑弹窗复用项目现有 `DocumentEditor`，不提供独立标题输入，保存时以 Markdown 第一行作为节点标题。

## 风险与待确认事项

- 图片 data URL 存入布局会增大 `.canvas-layout.json`，适合轻量素材，不适合大量大图长期保存。
- Markdown 编辑弹窗先服务画布内资料浏览，不与知识库或项目文档联动。

## 实施摘要

- `packages/shared/src/demo/types.ts`：扩展画布自由节点类型，保留文档和图片节点数据结构。
- `packages/shared/src/demo/PreviewCanvas.tsx`：接入新增节点创建、编辑、键盘删除、拖拽、缩放、吸附、图片粘贴/拖拽。
- `packages/shared/src/demo/CanvasFreeNodeItem.tsx`：新增自由节点渲染组件，隔离文档和图片展示；文档和图片节点不显示内部头部。
- `packages/shared/src/demo/DocumentEditor.tsx`：作为画布文档节点编辑的统一 Markdown 编辑器入口被复用，不新增独立编辑器实现。
- `packages/shared/src/demo/CanvasToolbar.tsx`：新增文档工具按钮。
- `packages/author-site/src/app/api/sessions/[sessionId]/canvas-layout/route.ts`：扩展画布布局 API 校验，支持保存和读取自由节点。
