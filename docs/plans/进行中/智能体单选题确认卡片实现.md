# 智能体单选题确认卡片实现

## 背景

当前 AI 对话已经支持权限确认和计划审批，但 Agent 在需求不明确时只能用普通文本追问。用户希望 Agent 可以发送可点击的单选题卡片，等待用户选择后继续执行，并在对话历史中保留问题和选择。

## 目标

- 新增 Agent 可调用的 `requestUserChoice` 工具。
- 通过独立的 `user_choice_request` / `user_choice_response` 协议传递需求确认，不复用权限审批协议。
- 前端以内联卡片展示问题、选项和“其他”输入。
- 用户选择后，卡片保留已选状态，Agent 获取结构化结果继续处理。

## 范围

- 涉及 `agent-service`、`agent-client`、`author-site` 和相关项目文档。
- 第一版只支持 WebSocket 流式会话中的交互卡片。
- HTTP 非流式 fallback 不处理中途交互，工具应返回不可用结果。

## 方案

1. 后端新增用户选择请求类型、工具、pending 管理和 WebSocket 响应入口。
2. 前端流服务监听选择请求并回传用户选择。
3. 对话消息 `parts` 新增 `user_choice` 类型，由 AssistantMessage 渲染并持久化。
4. 更新系统提示词和项目文档，明确使用边界。
