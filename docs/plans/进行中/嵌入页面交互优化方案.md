# 嵌入页面交互优化方案

> **状态**: 进行中
> **创建日期**: 2026-05-17
> **关联模块**: 创作端 - 嵌入 API (07-嵌入API)、使用端 - 预览与配置 (02-预览与配置)

## 一、问题概述

### 1.1 问题一：宫格模式下点击页面不应跳转单页模式

**问题描述**：
在嵌入页面（`/embed/{demoId}` 或 `/embed/{projectId}/{demoId}`）中，当项目包含多个 Demo 页面时，预览区以宫格模式展示所有页面。当前实现中，点击宫格中的某个页面卡片会跳转到单页面模式，导致用户体验被打断。

**期望行为**：
- 宫格模式下，点击页面卡片仅用于切换活跃页面（高亮显示）
- 不应触发视图模式的切换
- 保持宫格布局，用户可继续浏览其他页面

**相关文档**：
- [实时预览机制 - 宫格预览机制](./docs/项目文档/创作端/04-配置与预览/技术/02_实时预览机制.md)
- [嵌入架构设计](./docs/项目文档/创作端/07-嵌入API/技术/01_嵌入架构设计.md)

### 1.2 问题二：使用端缺少页面目录展示

**问题描述**：
使用端（viewer-site）的预览页中，虽然有页面选择器（Tabs）用于切换 Demo 页面，但缺少类似创作端编辑页的完整页面目录功能。用户无法：
- 查看所有页面列表
- 了解当前选中页面
- 在页面之间快速切换

**期望行为**：
- 预览页左侧或顶部展示完整的页面目录
- 清晰标识当前选中的页面
- 支持快速切换页面

**相关文档**：
- [预览与配置 - 架构设计](./docs/项目文档/使用端/02-预览与配置/技术/01_架构设计.md)
- [iframe 通信机制](./docs/项目文档/使用端/02-预览与配置/技术/02_iframe通信机制.md)

## 二、问题根因分析

### 2.1 问题一根因

```
当前实现流程：
┌─────────────────────────────────────────────────────────────┐
│  嵌入页面点击卡片                                             │
│      │                                                      │
│      ▼                                                      │
│  GridCard.onClick → setActivePageId(clickedPageId)         │
│      │                                                      │
│      ▼                                                      │
│  触发视图模式切换逻辑？                                       │
│      │                                                      │
│      ▼                                                      │
│  跳转到单页面预览模式                                         │
└─────────────────────────────────────────────────────────────┘
```

**根因**：
- 创作端编辑页的 `PreviewGrid` 组件设计为"配置面板内的宫格预览"，其点击行为与页面切换逻辑耦合
- 嵌入页面复用了编辑页的宫格预览组件，但未解耦点击行为
- 缺少视图模式切换的条件判断逻辑

**相关代码位置**（推测）：
- `packages/author-site/src/components/preview/PreviewGrid.tsx`
- `packages/author-site/src/components/preview/GridCard.tsx`

### 2.2 问题二根因

```
当前使用端预览页结构：
┌─────────────────────────────────────────────────────────────┐
│  page.tsx ([projectId]/page.tsx)                            │
│      │                                                      │
│      ├── Demo 页面选择器 (Tabs)                              │
│      │       └── [页面A] [页面B] [页面C]                     │
│      │                                                      │
│      ├── iframe 预览区 (IframeRenderer)                      │
│      │       └── src = /embed/{projectId}/{demoId}         │
│      │                                                      │
│      └── 配置面板 (ConfigPanel)                             │
└─────────────────────────────────────────────────────────────┘
```

**根因**：
- 预览页架构设计为"三栏协作"：页面选择器 + iframe 预览区 + 配置面板
- 当前页面选择器仅使用 Tabs 组件，功能单一
- 缺少独立的页面目录组件，无法展示完整的页面层级结构

**相关代码位置**（推测）：
- `packages/viewer-site/src/app/[projectId]/page.tsx`
- `packages/viewer-site/src/components/demo-tabs.tsx`

## 三、优化方案

### 3.1 方案一：解耦宫格点击行为

#### 3.1.1 目标

在嵌入页面的宫格预览中，点击页面卡片仅切换活跃页面高亮，不触发视图模式切换。

#### 3.1.2 实现思路

**方案 A：新增嵌入专用宫格组件**

```typescript
// 新增文件：packages/author-site/src/components/preview/EmbedPreviewGrid.tsx

interface EmbedPreviewGridProps {
  pages: DemoPage[];           // Demo 页面列表
  activePageId: string;        // 当前活跃页面 ID
  onPageSelect: (pageId: string) => void;  // 页面选择回调
  previewSize: PreviewSize;   // 预览尺寸
  gridColumns?: number;        // 每行显示列数，默认 3
  disabledModeSwitch?: boolean; // 禁用模式切换，默认 true
}
```

**关键差异**：

| 属性 | PreviewGrid | EmbedPreviewGrid |
|------|-------------|-----------------|
| 模式切换 | 支持切换单页/宫格 | 固定宫格模式 |
| 点击行为 | 可配置跳转 | 仅切换活跃高亮 |
| 使用场景 | 编辑页配置面板 | 嵌入页面预览 |

**方案 B：扩展现有 PreviewGrid Props**

```typescript
// 在现有 PreviewGrid 中新增属性
interface PreviewGridProps {
  // ... 现有属性

  // 新增：嵌入模式标识
  embedMode?: boolean;

  // 新增：嵌入模式下的点击行为
  onEmbedPageSelect?: (pageId: string) => void;
}
```

**推荐方案**：方案 A
- 保持现有 PreviewGrid 组件不变，避免影响编辑页功能
- 创建独立的嵌入模式组件，职责更清晰
- 便于后续维护和扩展

#### 3.1.3 嵌入页面组件调整

```tsx
// 嵌入页面文件：src/app/embed/[demoId]/page.tsx

// 替换 PreviewGrid 为 EmbedPreviewGrid
<EmbedPreviewGrid
  pages={demoPages}
  activePageId={activePageId}
  onPageSelect={handlePageSelect}
  previewSize={currentPreviewSize}
  gridColumns={3}
  disabledModeSwitch={true}  // 关键：禁用模式切换
/>
```

### 3.2 方案二：增强使用端页面目录

#### 3.2.1 目标

在使用端预览页中添加完整的页面目录组件，提供更好的页面导航体验。

#### 3.2.2 实现思路

**新增页面目录组件**

```typescript
// 新增文件：packages/viewer-site/src/components/page-directory.tsx

interface PageDirectoryProps {
  pages: DemoPage[];           // 页面列表
  activePageId: string;        // 当前页面 ID
  onPageSelect: (pageId: string) => void;
}

interface DemoPage {
  id: string;
  name: string;
  previewSize?: PreviewSize;
  hasChanges?: boolean;        // 是否有未保存变更
}
```

**组件结构设计**

```
┌─ PageDirectory ─────────────────────────────────────────────────┐
│                                                                  │
│  📄 页面目录                                                    │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ ● 默认页面                              [当前] 🔵        │   │
│  ├──────────────────────────────────────────────────────────┤   │
│  │   登录页面                                                │   │
│  ├──────────────────────────────────────────────────────────┤   │
│  │   商品详情                              ⚠️               │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                  │
│  [+ 新建页面]  ← 可选：嵌入模式下是否显示新建入口               │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

**布局调整**

```
调整前（当前）：

┌─────────────────────────────────────────────────────────────┐
│  [页面A] [页面B] [页面C]     ← Tabs 页面选择器               │
│                                                              │
│  ┌──────────────────────────┐  ┌────────────────────────┐  │
│  │                          │  │                        │  │
│  │   iframe 预览区          │  │   配置面板              │  │
│  │                          │  │                        │  │
│  └──────────────────────────┘  └────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘

调整后（优化）：

┌─────────────────────────────────────────────────────────────┐
│  📄 页面目录          │  ┌──────────────────────────────┐  │
│  ┌─────────────────┐  │  │                              │  │
│  │ ● 默认页面 [当前]│  │  │                              │  │
│  │   登录页面      │  │  │     iframe 预览区            │  │
│  │   商品详情      │  │  │                              │  │
│  └─────────────────┘  │  │                              │  │
│                      │  └──────────────────────────────┘  │
│                      │  ┌──────────────────────────────┐  │
│                      │  │        配置面板               │  │
│                      │  └──────────────────────────────┘  │
└──────────────────────┴─────────────────────────────────────┘
         │
│        │
└────────┴────────────────────────────────────────────────────┘
```

**响应式处理**

| 屏幕宽度 | 布局方式 |
|:---------|:---------|
| >= 1024px | 左侧页面目录 + 右侧预览+配置 |
| < 1024px | 收起为可展开抽屉，侧边滑出 |
| < 768px | 页面目录移至顶部 Tabs 下方 |

## 五、API 影响分析

### 5.1 嵌入页面 API

**当前 API**：`GET /api/embed/{projectId}?page={pageId}`

**无需修改**，因为：
- 页面列表已通过 iframe 内嵌获取
- 嵌入页面本身不直接调用 API
- 页面切换通过 React 状态管理

### 5.2 使用端 API

**当前 API**：`GET /api/projects/:id`

**数据补充**：确认返回数据包含完整的 `demos` 页面列表：

```typescript
interface ProjectDetail {
  id: string;
  name: string;
  demos: DemoPage[];  // 确保包含完整页面信息
}

interface DemoPage {
  id: string;
  name: string;
  previewSize?: PreviewSize;
}
```

## 六、验收标准

### 6.1 问题一验收

| 验收项 | 标准 |
|--------|------|
| 宫格点击不跳转 | 点击宫格中的任意页面卡片，仅切换高亮，不发生页面跳转 |
| 活跃页面标识 | 当前活跃页面对应的卡片有明显的视觉高亮 |
| 多页面支持 | 宫格正确展示所有 Demo 页面卡片 |
| 尺寸自适应 | 不同尺寸的页面卡片按比例正确显示 |

### 6.2 问题二验收

| 验收项 | 标准 |
|--------|------|
| 页面目录展示 | 预览页显示完整的页面列表 |
| 当前页面标识 | 清晰标识当前选中的页面 |
| 页面切换 | 点击目录项可正确切换页面 |
| 响应式布局 | 不同屏幕宽度下布局正确 |

## 七、风险与注意事项

### 7.1 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 创作端组件耦合 | 修改 PreviewGrid 可能影响编辑页 | 采用新增组件方案，避免直接修改 |
| 使用端布局冲突 | 页面目录可能与现有布局冲突 | 渐进式改造，保留 Tabs 作为降级 |
| API 数据缺失 | 页面列表可能不完整 | 确认 API 返回数据格式 |

### 7.2 注意事项

1. **保持向后兼容**：嵌入页面的 URL 格式不变
2. **性能考虑**：宫格懒加载机制保留，避免一次性加载过多 iframe
3. **使用端体验**：页面目录折叠状态下，Tabs 仍可正常工作
