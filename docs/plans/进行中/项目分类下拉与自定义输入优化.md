# 项目分类下拉与自定义输入优化

## 背景

首页项目卡片的“修改分类”弹窗需要以选择已有分类为默认路径，同时保留创建新分类的能力。用户进一步明确：最后一个选项应为“自定义分类”，点击后才进入输入模式；自定义分类支持用 `/` 添加子分类。

## 目标

- 项目分类字段默认使用下拉列表选择已有分类。
- 下拉列表最后固定提供“自定义分类”入口，点击后才允许输入。
- 自定义分类支持通过 `/` 表示子分类。
- 首页目录按 `/` 展示两级分类，并支持父分类聚合筛选。
- 空白创建、复制项目、复制模板、普通项目改分类、模板改分类共用一致体验。

## 范围

- `author-site` 首页名称/分类弹窗组件。
- 首页分类弹窗相关单元测试。
- 项目管理模块需求与技术文档。

不改动项目分类的数据结构、接口参数或持久化规则。

## 方案

- 用项目内已有 `Popover`、`Button`、`Input` 和 lucide 图标组合分类选择控件。
- 默认渲染为 Button 风格的选择器；点击后展示已有分类，下拉末尾用分隔线隔开“自定义分类”。
- 进入自定义模式后渲染输入框、返回选择按钮和辅助说明。
- 自定义输入提交前 trim 首尾空格，并把各级分类分段 trim 后用半角 `/` 合并。
- 首页目录把分类字符串按 `/` 解析为父子分类；父分类按前缀聚合，子分类完整匹配。
