# 项目分类下拉与自定义输入优化

## 背景

首页项目卡片的“修改分类”弹窗当前使用原生输入联想承载分类维护。用户希望分类字段既能通过下拉列表选择已有分类，也能直接输入新的自定义分类。

## 目标

- 项目分类字段保留自由输入能力。
- 项目分类字段提供明确的下拉入口，列出已有普通项目和模板分类。
- 空白创建、复制项目、复制模板、普通项目改分类、模板改分类共用一致体验。

## 范围

- `author-site` 首页名称/分类弹窗组件。
- 首页分类弹窗相关单元测试。
- 项目管理模块需求与技术文档。

不改动项目分类的数据结构、接口参数或持久化规则。

## 方案

- 用项目内已有 `Popover`、`Button`、`Input` 和 lucide 图标组合分类输入控件。
- 输入框负责自定义分类；右侧按钮负责展开已有分类列表。
- 选择已有分类后写回输入框并关闭下拉。
- 分类选项继续由首页从普通项目分类与模板分类去重生成。
