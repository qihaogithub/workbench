# 单页面预览尺寸读取修复

## 背景

创作端项目编辑页在单页面模式下，预览容器可能退回默认手机尺寸，导致平板或自定义尺寸页面的预览框与页面实际尺寸不一致。

## 目标

- 单页面模式预览尺寸必须来自页面的 `config.schema.json`。
- API、项目编辑页、使用端预览保持同一套尺寸解析语义。
- 修复后补充最小测试，避免尺寸解析逻辑再次分叉。

## 范围

- `packages/shared/src/demo/validator.ts` 的尺寸解析。
- `packages/author-site/src/lib/preview-size.ts` 的服务端尺寸提取。
- 创作端项目编辑页已有的 `getPreviewSize` 调用链。
- 配置与预览模块文档。

## 方案

将 `$demo.previewSize` 的读取、字段过滤和 `scale` 数值化收敛到共享解析逻辑，前端 `getPreviewSize` 与服务端 `extractPreviewSize` 复用同一语义。单页面模式渲染 `PreviewPanel` 时，从当前页 `config.schema.json` 对应的 schema 字符串派生尺寸，避免异步切页期间沿用上一页尺寸或默认手机尺寸。
