### 2.2 关键数据流

```
用户点击"对话历史"按钮
  ↓
HistoryDialog.fetchSessions()
  ↓
GET /api/sessions/project/{projectId}
  ↓
读取 data/sessions/{userId}/{projectId}/*/.session.json
  ↓
返回会话列表（SessionItem[]）
  ↓
渲染历史列表（显示 title/创建时间/消息数量）
```

### 2.3 核心数据结构

**SessionItem（HistoryDialog 内部类型）：**
```typescript
interface SessionItem {
  sessionId: string
  demoId: string
  title?: string | null
  createdAt: number
  expiresAt: number
  isExpired: boolean
  messageCount: number
  lastMessageAt: number
  hasUnsavedChanges: boolean
}
```

**会话文件存储结构：**
```
data/sessions/
  {userId}/
    {projectId（demoId）}/
      {sessionId}/
        .session.json       # 会话元数据
        .messages.json       # 消息历史
        index.tsx           # 代码文件
        config.schema.json   # Schema 文件
```

**project/[projectId]/route.ts 返回的 SessionItem：**
```typescript
{
  sessionId: meta.sessionId,
  demoId: meta.demoId,
  title: meta.title || null,
  createdAt: meta.createdAt,
  expiresAt: meta.expiresAt,
  isExpired: Date.now() > meta.expiresAt,
  messageCount,
  lastMessageAt,
  hasUnsavedChanges: meta.status === "editing",
}
```

## 3. 根因分析

### 3.1 可能的根因

经过代码审查，未发现明显的逻辑错误。列表为空的可能原因：

**可能性 1：数据目录不存在**
- 会话存储在 `data/sessions/{userId}/{projectId}/`
- 如果用户从未在该项目下创建过会话，目录不存在，返回空列表

**可能性 2：所有会话已过期被清理**
- `session-manager.ts` 的 `findActiveSession` 会主动删除过期会话目录
- `project/[projectId]/route.ts` 的列表 API 仅标记 `isExpired`，不删除
- 两者行为不一致

**可能性 3：会话创建路径不匹配**
- 新建会话调用 `POST /api/sessions` → `createEditSession(userId, projectId)`
- 存储到 `sessions/{userId}/{projectId}/{sessionId}/`
- 列表 API 从 `sessions/{userId}/{projectId}/` 读取
- **需要验证 `projectId` 参数是否一致**

### 3.2 关键代码链路

**会话创建：**
```typescript
// demo/[id]/edit/page.tsx - loadDemo
const sessionRes = await fetch("/api/sessions", {
  method: "POST",
  body: JSON.stringify({ demoId }),
});
// → POST /api/sessions { demoId }
// → createEditSession(userId, demoId)
// → sessions/{userId}/{demoId}/{sessionId}/
```

**会话列表获取：**
```typescript
// history-dialog.tsx - fetchSessions
const res = await fetch(`/api/sessions/project/${projectId}`)
// → GET /api/sessions/project/{projectId}
// → 读取 sessions/{userId}/{projectId}/*/.session.json
```

**关键问题：**
- `projectId` 参数在 `AIChat` 中来自父组件的 `demoId`
- 传递链路：`demo page` → `AIChat(projectId={demoId})` → `HistoryDialog(projectId={projectId})`

## 7. 技术债务

### 7.1 双端 Session 管理割裂
- Web 端：文件系统存储 `data/sessions/`
- agent-service 端：内存存储 `MemorySessionStore`
- 服务重启后 agent-service 的会话信息丢失，但文件系统数据保留

### 7.2 过期清理不一致
- `findActiveSession` 会主动删除过期目录
- 列表 API 仅标记 `isExpired`，不删除
- 可能导致用户看到"已过期"会话但无法点击
