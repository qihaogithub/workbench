# 配置项备注功能 - 需求文档

> 版本：v1.1（经代码库核实修正）
> 创建日期：2026-05-04
> 状态：实施中

> **📝 名称变更（待实施）**：创作端包名由 `packages/web` 更名为 `packages/studio`，npm 包名由 `@opencode-workbench/web` 更名为 `@opencode-workbench/studio`。此变更将在 `packages/studio` 文件夹创建时一并完成。**当前实施仍使用 `packages/web`。**

## 一、模块定位

为配置面板中的每个配置项提供添加富文本备注的能力，让**编辑者**（创作端用户，如设计师、配置模板维护者）能够记录配置项的设计说明、实现规范、使用指引等信息。**使用者**（使用端用户，如配置 Demo 的设计师）可以查看备注内容，了解配置项的用途和规范。备注与配置项一一绑定，随配置数据一起持久化存储。

---

## 二、功能架构

### 2.1 端侧分布

| 端 | 包 | 功能 |
|:---|:---|:-----|
| **创作端** | `packages/web`【未来更名为 `packages/studio`】 | 配置面板 + 富文本编辑器（编辑者添加/编辑/删除备注） |
| **使用端** | `packages/viewer-site` | 独立配置面板（使用者查看备注，只读模式） |

### 2.2 数据流

```
创作端（编辑者）:
  Schema 属性级 $demo.note ← 富文本编辑器 → 通过 onSchemaChange 回调 → 持久化到项目数据

使用端（使用者）:
  读取 Schema（含备注） → 独立 ConfigPanel 展示备注摘要/详情（只读）
```

---

## 角色说明

本功能涉及两类用户角色：

| 角色 | 说明 | 备注权限 | 所在端 |
|:-----|:-----|:---------|:-------|
| **编辑者** | 负责维护配置模板、编写配置项规范说明（如设计师、模板维护者） | 可**添加/编辑/删除**备注 | **创作端** (`packages/web`) |
| **使用者** | 使用配置面板调整 Demo 参数的用户（如使用 Demo 的设计师） | 仅可**查看**备注，不可编辑 | **使用端** (`packages/viewer-site`) |

> 备注的"添加/编辑/删除"操作仅对**编辑者**（创作端）开放；**使用者**（使用端）只能看到备注按钮和备注内容，无法进入编辑模式。

## 三、功能描述

> 本章节描述的功能，在**创作端**（编辑者）和**使用端**（使用者）均可访问，但交互模式不同：
> - **创作端**：完整功能，可添加/编辑/删除备注
> - **使用端**：只读模式，仅可查看备注内容

### 3.1 备注入口

#### 3.1.1 备注按钮

**用户视角**：
- 每个配置项的标签（Label）右侧显示一个"备注"图标按钮
- 按钮样式为中性（文字提示图标 `MessageSquarePlus` 或 `StickyNote`）
- 已有备注的配置项，按钮视觉上应有区别（如填充状态或小圆点标识）

**交互效果**：
- **编辑者**：按钮悬浮时显示 Tooltip"添加备注"或"编辑备注"（取决于是否已有内容），点击后弹出编辑弹窗
- **使用者**：按钮悬浮时显示 Tooltip"查看备注"，点击后弹出**只读模式**的备注查看弹窗
- 无备注的配置项，按钮仅对编辑者可见；有备注的配置项，按钮对所有人可见

#### 3.1.2 备注标识

**用户视角**：
- 已有备注的配置项，标签区域有明显的视觉标识
- 让用户一目了然地知道哪些配置项有备注说明

**交互效果**：
- 有备注的配置项，备注按钮显示为填充状态（如 `StickyNote` 图标带填充色）
- 无备注的配置项，备注按钮显示为 outline 状态（如 `MessageSquarePlus` 图标为线框）

---

### 3.2 备注弹窗

#### 3.2.1 弹窗布局

**用户视角**：
- 点击备注按钮后，弹出一个模态对话框
- 弹窗标题显示对应配置项的名称（如"背景颜色 - 备注"）
- **编辑者**：弹窗包含富文本编辑器区域和操作按钮
- **使用者**：弹窗为只读模式，仅展示备注内容的富文本渲染，无编辑器工具栏和操作按钮

**交互效果**：
- 弹窗居中显示，带有遮罩层
- 弹窗宽度建议 560px，高度自适应（最大 480px）
- 弹窗可点击遮罩层或"关闭"按钮关闭
- 弹窗内编辑内容手动保存

#### 3.2.2 富文本编辑器（仅创作端编辑者可见）

**用户视角**：
- 编辑器提供完整的格式化工具栏
- 支持以下格式功能：
  - 文字样式：加粗（`Bold`）、斜体（`Italic`）、下划线（`Underline`）
  - 列表：有序列表、无序列表
  - 链接：插入/编辑/删除超链接
  - 清除格式：一键清除选中内容的格式
- 编辑器支持 Markdown 快捷键（输入 `**粗体**` 自动转换）

**交互效果**：
- 工具栏固定在编辑器顶部，包含上述功能按钮
- 工具栏按钮使用图标表示，鼠标悬浮显示功能名称
- 编辑器内容为空时显示占位文字："输入配置项说明、设计规范或使用指引..."
- 编辑器支持键盘快捷键：`Ctrl+B` 加粗、`Ctrl+I` 斜体、`Ctrl+U` 下划线、`Ctrl+K` 插入链接

#### 3.2.3 弹窗操作按钮

**用户视角**：
- **编辑者**弹窗底部提供三个操作区域：
  - "保存"（主要按钮）：保存备注内容并关闭弹窗
  - "取消"（次要按钮）：放弃修改并关闭弹窗
  - 如果有已保存的备注内容，提供"删除备注"文字链接按钮
- **使用者**弹窗底部仅提供"关闭"按钮

**交互效果**：
- "保存"按钮在编辑器内容无变化时置灰禁用
- 点击"保存"后，备注内容持久化并关闭弹窗
- 点击"删除备注"后，弹出二次确认："确定删除此备注？"
- 删除后备注内容清空，按钮恢复为未填充状态

---

### 3.3 备注展示

#### 3.3.1 配置项标签区域

**用户视角**：
- 有备注的配置项，标签行显示备注内容摘要（1 行截断）
- 摘要文字以较小的字号（如 `text-xs text-muted-foreground`）显示在标签下方

**交互效果**：
- 备注摘要最多显示一行，超出部分省略号截断
- 摘要内容仅保留纯文本，过滤 HTML 标签

#### 3.3.2 悬浮预览（可选）

**用户视角**：
- 鼠标悬浮在备注按钮上时，显示备注内容的简要预览
- 预览 Tooltip 中支持简单的富文本渲染

**交互效果**：
- Tooltip 中渲染富文本内容（保留加粗、列表等基本格式）
- Tooltip 最大宽度 300px，高度自适应
- Tooltip 中显示截断提示："点击查看完整备注"

## 四、数据结构

### 4.1 Schema 扩展字段

在 JSON Schema 的每个配置项属性中增加 `$demo.note` 字段：

```json
{
  "type": "object",
  "properties": {
    "bgColor": {
      "type": "string",
      "title": "背景颜色",
      "format": "color",
      "default": "#ffffff",
      "$demo": {
        "note": "<p>背景色使用品牌主色的浅色变体，确保文字可读性。</p><ul><li>浅色模式：#F5F5F5</li><li>深色模式：#1A1A1A</li></ul>"
      }
    },
    "titleSize": {
      "type": "number",
      "title": "标题字号",
      "default": 24,
      "$demo": {
        "note": "<p>标题字号应遵循字体缩放比例：</p><p><strong>建议值：</strong>24px - 32px</p>"
      }
    }
  }
}
```

> **注意**：此 `$demo` 是属性级别的扩展，与根级别的 `$demo`（如 `$demo.previewSize`、`$demo.orderable`）是不同的层级。根级 `$demo` 是 Schema 元数据，属性级 `$demo` 是字段备注。

### 4.2 备注数据格式

| 字段 | 类型 | 说明 |
|:-----|:-----|:-----|
| `$demo.note` | string (HTML) | 富文本备注内容，使用 HTML 格式存储 |

### 4.3 备注与表单数据的关系

- 备注数据独立于配置值存储，不参与预览渲染
- 备注数据随 Schema 一起持久化，导出时保留
- 备注修改后通过 `onSchemaChange` 回调传递更新后的 Schema 字符串（含备注变更），与 `onChange`（传递表单数据）分离

## 五、业务规则与约束

| 规则 | 说明 |
|:-----|:-----|
| 非必填 | 备注为可选功能，无备注不影响配置项正常使用 |
| HTML 存储 | 富文本编辑器输出 HTML 格式，存储时做 XSS 过滤 |
| 长度限制 | 单条备注内容不超过 5000 字符（含 HTML 标签） |
| 空备注 | 空字符串视为无备注，不展示摘要标识 |
| Schema 无关性 | 备注不参与配置验证、预览渲染等流程 |
| 持久化范围 | 备注随 Schema 数据存储，导出时包含 |

## 六、技术方案

### 6.1 富文本编辑器选型

**选择 TipTap**（基于 ProseMirror）：

| 考量 | 说明 |
|:-----|:-----|
| 生态 | 与 shadcn/ui 风格兼容，社区活跃 |
| 扩展性 | 支持按需加载扩展，精确控制功能集 |
| 包体积 | 按需引入约 50-80KB gzip |
| 输出格式 | 原生支持 HTML 输出 |

### 6.2 引入的 TipTap 扩展

| 扩展 | 用途 |
|:-----|:-----|
| `@tiptap/starter-kit` | 核心功能（加粗、斜体、列表等） |
| `@tiptap/extension-underline` | 下划线 |
| `@tiptap/extension-link` | 超链接 |
| `@tiptap/extension-placeholder` | 占位文字 |

### 6.3 组件架构

> 以下组件位于**创作端** `packages/web/components/demo/`（注意：非 `src/components/demo/`，`src/components/demo/` 是另一组页面级组件）

```
packages/web/components/demo/
├── ConfigFormNew.tsx         # 修改：FieldConfig 增加 note 字段，FieldRenderer 增加备注按钮和摘要展示
├── types.ts                  # 修改：ConfigFormProps 增加 onSchemaChange 回调
├── NoteButton.tsx            # 新增：备注按钮组件（含标识状态，接收 readonly 属性）
├── NoteDialog.tsx            # 新增：备注弹窗（根据 readonly 属性切换编辑/只读模式）
├── RichTextEditor.tsx        # 新增：TipTap 富文本编辑器封装（编辑模式，仅创作端）
└── NotePreview.tsx           # 新增：备注内容摘要/预览展示
```

> **使用端** `packages/viewer-site/src/components/config-panel.tsx` 有独立的 `ConfigPanel` 组件，需单独增加备注只读展示功能。

### 6.4 角色控制机制

- `ConfigForm` 通过 `readonly` prop 控制备注按钮的行为：
  - `readonly=false`（创作端默认）：编辑者模式，可添加/编辑/删除备注
  - `readonly=true`（使用端）：使用者模式，仅可查看备注内容
- `readonly` 属性由上层页面根据当前所在端（创作端/使用端）传递

### 6.5 数据流

**创作端流程（编辑者）：**
```
用户点击备注按钮
  → NoteDialog 打开（编辑模式）
    → RichTextEditor 加载已有备注内容（从属性级 $demo.note 读取）
    → 用户编辑内容
    → 点击"保存"
      → 更新 Schema 中对应配置项的 $demo.note
      → 通过 onSchemaChange 回调传出更新后的 Schema 字符串
      → NoteButton 更新为"有备注"状态
      → 标签区域显示备注摘要
```

**使用端流程（使用者）：**
```
用户点击备注按钮
  → NoteDialog 打开（只读模式）
    → 渲染备注内容的富文本展示
    → 点击"关闭"关闭弹窗
```

### 6.6 XSS 安全处理

- 编辑器输出 HTML 后，使用 `DOMPurify` 过滤危险标签和属性
- 允许的标签：`p`, `strong`, `em`, `u`, `ul`, `ol`, `li`, `a`, `br`
- 允许的属性：`href`（仅 `http://`、`https://`、`mailto:` 协议）

## 七、交互细节

### 7.1 备注按钮定位

```
┌─────────────────────────────────────┐
│ [Label 文字] [*]     [💬 备注按钮] │  ← 标签行
│ 控件区域                             │
└─────────────────────────────────────┘
```

- 按钮位于标签文字右侧，与必填标记 `*` 同级
- 按钮使用 `h-4 w-4` 小尺寸图标，颜色 `text-muted-foreground`
- 按钮与标签文字之间间距 4px
- **可见性规则**：
  - 有备注时：按钮对编辑者和使用者均可见
  - 无备注时：按钮仅对编辑者可见（使用者不显示空备注按钮）

### 7.2 弹窗尺寸

| 元素 | 值 |
|:-----|:---|
| 弹窗宽度 | 560px |
| 编辑器最小高度 | 200px |
| 编辑器最大高度 | 360px |
| 弹窗最大高度 | 480px |

### 7.3 工具栏按钮列表

| 按钮 | 图标 | 快捷键 |
|:-----|:-----|:-------|
| 加粗 | Bold | Ctrl+B |
