# P0项目管理关键动作强制同步实施计划

> 状态：已完成
> 日期：2026-06-30

## 背景

[项目管理方案长期优化评估](./项目管理方案长期优化评估.md) 将 Workspace 当前态语义、跨用户协作边界和关键动作强制 flush 列为 P0。当前前端已有局部 flush 调用，但发布、导出、自动检查点、保存/合并和页面版本等后端入口仍需要统一的服务端前置协议。

## 目标

- 明确 Workspace 是项目共享草稿，Session 只承载 AI 对话上下文。
- 关键动作在生成快照、发布产物、导出包或恢复内容前，必须先确认 Workspace 当前态已同步。
- 协同服务允许不同用户通过各自有效 Session 进入同一项目 Workspace。
- 补齐最小验证，确保 flush helper 和协同持久化规则可回归。

## 范围

- `agent-service` 协同 Session/Workspace 校验与 workspace 级 flush。
- `author-site` 关键动作 flush coordinator 与现存关键 route 接入。
- 项目管理需求、实时协作需求和版本/协同技术文档。

不包含：

- 项目成员/角色模型。
- 完整事务 journal 或内容寻址快照。
- CLI submit 与 Web 协同草稿冲突策略。

## 方案

- agent-service 新增 Workspace 级 session 校验，资源级写入仍保留路径和类型校验。
- `flush-all` 由“刷新同 session 房间”调整为“调用者 session 授权后刷新同 project/workspace 的所有活跃房间”。
- author-site 新增 `workspace-flush` helper，统一调用 agent-service `/flush-all`，并把失败转为关键动作阻断错误。
- 发布、项目包导出、自动检查点、保存/合并 Session、页面命名版本和页面恢复在执行原业务前先调用 helper。
- 文档同步更新“共享 Workspace 当前态”和“关键动作服务端同步”语义。
