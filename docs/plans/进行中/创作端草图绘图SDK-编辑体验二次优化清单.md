# 创作端草图绘图 SDK - 编辑体验二次优化清单

## 当前状态

- 上一轮 Excalidraw 对比优化清单已视为完成，本清单只记录新的编辑体验问题和后续优化方向。
- P0.1 形状双击直接编辑文字已实现并验证：双击对象可见区域会通过 DOM 标记和几何 hitTest 进入画布内联编辑；锁定、运行时隐藏和预览态不会进入编辑。
- P0.2 属性栏已完成第二轮控件与信息架构改造：对象上下文头部、可折叠区块、颜色字段色块 + HEX + 最近色 + 常用色、枚举分段按钮、样式 reset 和底部 SVG/PNG 导出区均已落地；单选/多选/混合值/锁定与绑定状态继续走现有 patch/history 规则。
- P0.2 已完成第二轮收口：数值字段保持紧凑高度，长字段以短标记呈现并通过 aria label/title 保留完整含义；图片区已补“裁剪/适配”入口，但当前只复用 `style.imageFit`，不扩展 crop rect 协议。
- P1.1 悬浮快捷工具条已落地：单选对象提供填充、描边、文本、复制样式和更多入口，多选对象提供左/顶对齐、水平分布、成组、复制、删除和更多入口；工具条在内联编辑、拖拽、框选和绘制期间隐藏，并按选区位置避开编辑框与控制柄。
- P1.3 选择反馈第一项已落地：选择工具 hover 非选中对象时显示轻量高亮，选中对象显示边界、中心点、旋转点和 resize handles；这些反馈都属于编辑器临时 UI 状态。
- P1.3 重叠对象选择已落地：在选择工具下 Cmd/Ctrl 点击重叠区域会按当前 zIndex/视觉顺序循环选择候选对象，不启动拖拽，也不写入 scene。
- P1.3 分组内选择已落地：双击语义组内的可见子节点会进入组内选择，Esc 回到上一级 group 选择；focused group 仅保存在编辑器本地状态，不写入 scene。
- P1.3 空白拖拽框选反馈已落地：选择工具空白拖拽显示矩形框选虚线框和“矩形框选”提示，松开后消失；当前未新增自由套索工具或 scene 协议。
- P1.4 图层重命名已落地：图层面板优先显示节点 `name`，双击图层行可编辑 `name`；画布显示文本仍由 `text` 或绑定结果决定。
- P1.4 图层拖拽层级已落地：可编辑图层行支持拖放调整视觉层级，drop 后通过 `reorder` patch 重写 zIndex，并选中被拖动图层；锁定或运行时隐藏对象不作为拖拽源。
- P1.4 图层搜索/筛选已落地：图层面板支持按名称、类型标签和 id 搜索，并可按当前场景已有节点类型筛选；筛选条件只属于面板本地 UI 状态。
- P1.4 图层状态图标已落地：分组、锁定、隐藏/运行时隐藏和绑定对象在图层行显示常驻状态徽标，hover 操作按钮仍只负责执行锁定/显隐等命令。
- P2.1 内联文本编辑布局已增强：编辑框高度会按当前输入行数自动调整，形状文本编辑保持在对象内边距区域内并垂直居中；多行内容超过可用高度时显示“文本超出”提示并在编辑框内滚动。
- P2.1 局部选中文字样式已落地：画布内联编辑态会把 textarea 当前选区同步到编辑器本地 controller；属性栏 Text 区修改斜体、装饰、行高或字距时，只更新选区对应的 `textStyleRuns` 片段，没有选区时保持整段文字样式行为。
- P2.1 对象键盘导航已落地：画布快捷键作用域内 Tab/Shift+Tab 会在可见非 group 对象之间循环选择；focused group 内会限制在组内可见子节点间导航，不写入 scene。
- P2.2 连接候选点反馈已落地：拖动 line/arrow 端点时，画布会在可连接对象的四边中点和中心点显示候选连接点；候选点是临时 UI，不写入 scene，也不自动建立绑定。
- P2.2 吸附参考线反馈已落地：移动/缩放对象时，画布会区分显示网格、页面/对象中心线、对象边缘和间距参考线；Cmd/Ctrl 可临时隐藏参考线，Shift/Alt/Option 等拖拽 modifier 提示显示在画布上。该能力只属于编辑器 UI，不写入 scene，也不自动吸附坐标。
- P2.3 图片编辑入口已补齐：双击图片会进入画布内裁剪/适配编辑条，可切换 cover/contain/fill；拖拽图片文件到既有图片节点会替换 `src/alt`，同时保留尺寸、圆角、绑定和层级；属性栏 Image 区显示来源、估算大小和超过 2 MB 的提示。
- P2.4 右键导出入口已补齐：画布右键菜单和属性栏 Export 区都提供复制 SVG、复制 PNG、导出选区和导出整页入口；右键打开导出入口不改变 scene。
- P2.4 导出前配置已落地：属性栏 Export 区显示选区尺寸、PNG 输出尺寸、倍率和背景模式，并支持 1x/2x/3x 以及透明/白色背景导出；这些导出配置只属于属性面板本地状态。
- P2.4 剪贴板降级已落地：SVG/PNG 复制失败时会降级为文件下载，属性栏 Export 区显示复制成功或已下载的状态提示；提示和导出配置均不写入 scene。
- P3 质量护栏已补齐部分：playground E2E 新增编辑状态截图走查，P0 acceptance flow 覆盖悬浮工具条、图层重命名和导出入口；sketch-react 单测覆盖属性栏关键 aria label、键盘微调和 Enter 提交；命令入口继续复用 action registry。
- 本清单仍遵守草图 SDK 的核心边界：`SketchSceneDocument` 保持 Agent 友好、固定页面、结构化 patch；临时 UI 状态不进入场景协议。

## 当前结论

- 形状文本主入口已调整为画布双击，属性栏内容字段只保留为辅助精确编辑入口；空文本形状提交空内容不会删除对象，文本工具新建的临时纯文本节点仍保留空提交/取消即删除的特殊规则。
- 内联编辑框按形状文本区域定位，保留旋转、字号、字重、颜色、行高和文本对齐等显示线索，并根据当前输入自动调整高度；形状内编辑框保持在对象内边距区域内垂直居中，内容超过可用高度时显示“文本超出”提示，避免再简单铺满对象边界。内联编辑态的文字选区会驱动属性栏 Text 区的局部 `textStyleRuns` 更新；没有选区时仍按整段文字更新。
- 属性栏当前已经比后台表单更接近设计工具控件密度，并补齐了紧凑数值字段、短标签 tooltip、最近色、常用色、折叠区块、Layout/Arrange、Text/Appearance 拆分、连续输入合并、关键字段 reset、图片裁剪/适配入口和 SVG/PNG 导出入口；后续重点转向 P2 的真实图片双击裁剪态。
- 画布选区已补轻量悬浮快捷工具条，高频命令可直接在选区附近完成；完整命令仍由 command palette、右键菜单和属性栏承载，避免把所有能力塞进浮层。
- 选择反馈已补 hover 高亮和中心点，继续保持不写入 `SketchSceneDocument`；选中边界、旋转点和 resize handles 仍复用现有 selection overlay。
- 重叠对象命中已支持 Cmd/Ctrl 点击循环选择，候选列表按与只读/编辑命中一致的可见性和视觉层级排序，解决只能点到最上层对象的问题。
- 语义分组已补画布钻取路径：双击组内可见子节点进入子节点选择，Esc 返回 group；再次在同一 focused group 内双击子节点仍可进入原有文本内联编辑。
- 空白拖拽选择已明确为矩形框选，框选期间显示独立提示；本轮不引入自由 lasso，避免扩大命中算法和场景协议范围。
- 图层名称和画布文本已明确分离：图层面板使用 `name` 做管理标签，画布渲染继续使用 `text`/绑定结果；双击图层行只更新 `name`。
- 图层拖拽层级复用既有 `reorder` patch，不新增图层专用协议；面板顺序会换算回视觉层级顺序提交，节点内容保持不变。
- 图层搜索和类型筛选只影响面板行展示，不改变 `SketchSceneDocument`；空结果显示明确提示。
- 图层状态标识与操作入口分离：状态徽标常驻展示，锁定/显隐按钮仍在 hover 或 focus 时作为命令入口出现。
- 参考 Figma 时应借鉴右侧 Properties/Design 面板的原则：选中画布或图层后，右侧集中呈现当前对象的布局、颜色等可编辑属性，并把导出等对象能力放在面板底部或相关区块；不要照搬 Figma 的产品模型。
- 下一轮优化应优先收口 P0.2 剩余尾项，再补选择反馈、图层面板和高级编辑细节。

## P0：必须先修的体验缺陷

### P0.1 形状双击直接编辑文字

- [x] 双击矩形、圆形、菱形、卡片、便签、按钮、输入框等支持文本的对象任意可见区域，都进入画布内联文本编辑。
- [x] 空文本形状双击时也必须进入编辑态，并显示合理占位；提交空文本只清空 `text`，不能删除形状。
- [x] 新建纯文本节点时继续允许空内容取消后删除，这是文本工具的特殊规则，不能套到普通形状。
- [x] 双击命中应同时覆盖图形本体、已有文字标签、选择框内区域；不能只在 `data-sketch-node-label` 或某个 SVG 子节点上生效。
- [x] 锁定对象、运行时隐藏对象、只读模式对象不能进入编辑态，并且应有一致的禁用反馈。
- [x] 内联编辑框应贴合形状文本区域：居中、尊重旋转、缩放、字号、字重、颜色和文本对齐；不能简单铺满对象边界导致误判。
- [x] Enter、Shift+Enter、Esc、blur 的行为要稳定：Enter 提交，Shift+Enter 换行，Esc 取消，blur 提交。
- [x] 双击不应同时触发拖拽历史、框选、误选其他对象或打开右键菜单。

验收：

- `sketch-react` 测试覆盖空文本形状、本体双击、标签双击、旋转/缩放后的编辑框位置、锁定对象禁用。
- `sketch-playground` E2E 覆盖真实浏览器双击形状输入文字、撤销、重做、再次双击修改。
- 稳定项目文档已同步说明“形状文本主入口是画布双击，属性栏只作为辅助编辑入口”。

### P0.2 属性栏按 Figma 式信息架构重构

- [x] 属性栏顶部改为对象上下文头部：对象类型图标、名称、类型、锁定/可见状态、关键快捷操作；多选时显示数量和共同属性摘要。
- [x] 属性栏分为清晰区块，并支持折叠：
  - Position：X、Y、W、H、Rotation。
  - Layout/Arrange：对齐、分布、层级、成组/解组。
  - Content：只放对象内容的辅助编辑；形状文本不能依赖这里作为主入口。
  - Text：字体大小、字重、对齐、斜体、装饰、行高、字距。
  - Appearance：填充、描边、线宽、线型、透明度、圆角。
  - Line/Connector：端点、箭头、连接器绑定状态。
  - Image：图片源、alt、fit、替换、裁剪/适配入口。当前裁剪入口复用 `style.imageFit=cover|contain|fill`，不新增 crop rect 字段。
  - Bindings：绑定字段、解除绑定、绑定状态说明。
  - Export：导出相关配置，放在底部。
- [x] 控件密度参考设计工具而不是后台表单：数值输入使用双列或四列紧凑布局，图标标签优先，必要时 tooltip 解释。
- [x] 颜色字段改成色块 + HEX 输入 + 最近颜色/常用颜色；不要只给原生 color input。最近颜色历史为属性面板本地 UI 状态，不写入 `SketchSceneDocument`。
- [x] 对齐、字重、文本对齐、线型、箭头端点使用 segmented/icon controls；不要大量使用下拉框。
- [x] 混合值、多选不适用、锁定不可编辑、运行时绑定覆盖应有统一表达；禁止 silently disabled。
- [x] 支持字段级 reset：每个区块或关键字段可恢复默认值，且走正常 patch/history。本轮已完成外观、文本、多选样式和图片适配 reset，并补齐填充、描边、线宽、透明度、圆角、线型、箭头、文字颜色、字号、字重、对齐、斜体、装饰、行高和字距等关键字段独立 reset。
- [x] 属性修改应形成合理 undo 粒度：连续拖拽/连续输入合并，离散字段切换单独入栈。属性栏文本、数值和 HEX/color 连续输入首次有效变更前记录 checkpoint，后续输入以非历史预览提交；blur 或 Enter 结束会话，一次 undo 回到输入前状态。
- [x] 属性栏滚动时对象头部保持可见；长面板区块可折叠，避免每次都滚到样式区。

验收：

- `sketch-react` 测试覆盖单选、多选、锁定、绑定、混合值和不适用字段。
- 视觉检查覆盖至少：形状、文本、线条、箭头、图片、多选、锁定对象、空选区。
- 属性栏修改不改变 `SketchSceneDocument` 以外的临时 UI 状态持久化规则，最近颜色历史只在当前属性面板会话内保留。

- [x] 多选时显示对齐、分布、成组、复制、删除、更多。
- [x] 工具条只放高频命令，完整能力仍在命令面板、右键菜单和属性栏。
- [x] 工具条不得遮挡当前编辑框、resize handle、rotate handle。

验收：

- `sketch-react` 测试覆盖单选悬浮工具条的填充、文本、复制样式和更多入口，以及多选悬浮工具条的对齐、分布、成组、复制和删除入口。

### P1.3 选择和命中反馈

### P1.4 图层面板继续增强

- [x] 图层支持拖拽调整层级。
- [x] 图层支持搜索/过滤和按类型筛选。

## P3：质量与体验护栏

## 风险

- 属性栏重构容易变成纯视觉调整，必须同时改善信息架构、控件语义、混合态、锁定态和 undo 粒度。
- P0.2 的对象头部、最近色、常用色、折叠区、Layout/Arrange、Text/Appearance 拆分、连续输入合并、关键字段 reset、图片裁剪/适配入口和 SVG/PNG 导出区已完成；当前裁剪入口只复用 `style.imageFit`，后续若要支持真实 crop rect 仍需要扩展 scene 协议。
- 如果属性栏承载过多主流程，画布直接编辑会继续退化；画布操作应是主路径，属性栏是精确调整和辅助入口。
- Figma 是参考对象，不是产品目标；本项目仍要保持 Agent 可读写、固定页面和结构化 patch 的约束。
