# 元素位置配置功能方案

> 创建日期：2026-06-06
> 状态：进行中

---

## 一、背景与目标

当前配置面板支持通过 `$demo.orderable` 实现子组件的**纵向排序**，但不支持横向排序和自由坐标定位。本方案扩展配置系统的位置控制能力，新增三种位置配置类型：

| 类型 | 说明 | 当前状态 |
|:-----|:-----|:---------|
| 纵向排序 | 子组件从上到下的排列顺序 | ✅ 已实现（`$demo.orderable`） |
| 横向排序 | 子组件从左到右的排列顺序 | ❌ 未实现 |
| 自由坐标 | 子组件在画布上的 x/y 位置 | ❌ 未实现 |

**设计原则**：
- 向后兼容：现有 `$demo.orderable` 行为不变
- 统一模式：三种类型共享相似的声明和使用模式
- 配置驱动：位置信息通过 props 注入组件，组件代码自行决定如何渲染

## 二、Schema 扩展设计

### 2.1 新增扩展字段

在 `$demo` 元数据中新增两个字段：

| 字段 | 类型 | 说明 |
|:-----|:-----|:-----|
| `$demo.orderableHorizontal` | `string[]` | 声明可横向排序的属性列表 |
| `$demo.positionable` | `object` | 声明可自由定位的属性及其初始坐标 |

### 2.2 Schema 示例

```json
{
  "type": "object",
  "$demo": {
    "previewSize": { "width": 800, "height": 600 },
    "orderable": ["header", "content", "footer"],
    "orderableHorizontal": ["nav1", "nav2", "nav3"],
    "positionable": {
      "items": ["icon1", "icon2", "icon3"],
      "defaults": {
        "icon1": { "x": 50, "y": 100 },
        "icon2": { "x": 200, "y": 100 },
        "icon3": { "x": 350, "y": 100 }
      }
    }
  },
  "properties": {
    "header": { "type": "string", "title": "页头" },
    "content": { "type": "string", "title": "内容" },
    "footer": { "type": "string", "title": "页脚" },
    "nav1": { "type": "string", "title": "导航1" },
    "nav2": { "type": "string", "title": "导航2" },
    "nav3": { "type": "string", "title": "导航3" },
    "icon1": { "type": "string", "title": "图标1" },
    "icon2": { "type": "string", "title": "图标2" },
    "icon3": { "type": "string", "title": "图标3" }
  }
}
```

### 2.3 数据注入格式

位置信息通过以下隐藏字段注入组件 props：

| 字段 | 类型 | 说明 |
|:-----|:-----|:-----|
| `__order` | `string[]` | 纵向排序结果（已有） |
| `__orderH` | `string[]` | 横向排序结果（新增） |
| `__positions` | `Record<string, {x: number, y: number}>` | 自由坐标位置（新增） |

组件代码读取这些字段决定渲染位置：

```tsx
const Demo = (props) => {
  const { __orderH, __positions } = props;

  // 横向排序
  const navItems = __orderH.map(key => props[key]);

  // 自由坐标
  const positionedItems = Object.entries(__positions).map(([key, pos]) => ({
    key, ...props[key], x: pos.x, y: pos.y
  }));
};
```

## 三、配置面板 UI 设计

### 3.1 纵向排序（已有，不变）

- 排序区域标题："组件排序"
- 交互：拖拽 + 上下箭头
- 策略：`verticalListSortingStrategy`

### 3.2 横向排序（新增）

- 排序区域标题："横向排序"
- 交互：拖拽 + 左右箭头
- 策略：`horizontalListSortingStrategy`
- 视觉：排序项水平排列，每项显示为标签/卡片形式

```
[←] [导航1] [导航2] [导航3] [→]
```

### 3.3 自由坐标定位（新增）

- 区域标题："元素定位"
- 交互：迷你画布 + 拖拽定位
- 每个可定位元素显示为画布上的可拖拽标签
- 支持手动输入 x/y 坐标值
- 提供"恢复默认位置"按钮

**迷你画布设计**：
- 按预览区尺寸等比缩放
- 元素以标签形式显示，可拖拽移动
- 画布背景显示网格辅助线
- 拖拽时显示坐标提示

**坐标输入**：
- 点击元素标签后展开坐标输入框
- 支持 x/y 数字输入
- 输入与拖拽双向同步

## 四、技术实现方案

### 4.1 类型定义扩展

**文件**：`packages/shared/src/demo/types.ts`

```typescript
export interface DemoMeta {
  previewSize?: PreviewSize;
  orderable?: string[];              // 已有：纵向排序
  orderableHorizontal?: string[];    // 新增：横向排序
  positionable?: PositionableConfig; // 新增：自由定位
  [key: string]: unknown;
}

export interface PositionableConfig {
  items: string[];
  defaults?: Record<string, { x: number; y: number }>;
}

export interface PositionItem {
  x: number;
  y: number;
}
```

### 4.2 Schema 解析器扩展

**文件**：`packages/shared/src/demo/validator.ts`

新增函数：

| 函数 | 说明 |
|:-----|:-----|
| `getOrderableHorizontal(schema)` | 提取 `$demo.orderableHorizontal`，逻辑同 `getOrderable` |
| `getPositionable(schema)` | 提取 `$demo.positionable`，返回 `{ items, defaults }` |
| `getDefaultValues(schema)` | 扩展：新增 `__orderH` 和 `__positions` 默认值生成 |

### 4.3 配置面板组件扩展

**文件**：`packages/shared/src/demo/ConfigForm.tsx`

#### 4.3.1 横向排序组件 `OrderControlHorizontal`

复用现有 `OrderControl` 的拖拽逻辑，差异点：

| 维度 | 纵向排序 | 横向排序 |
|:-----|:---------|:---------|
| DnD 策略 | `verticalListSortingStrategy` | `horizontalListSortingStrategy` |
| 箭头方向 | 上/下 | 左/右 |
| 排序项布局 | 垂直列表 | 水平排列 |
| 数据字段 | `__order` | `__orderH` |
| 区域标题 | "组件排序" | "横向排序" |

实现方式：抽取 `OrderControlBase` 通用组件，通过 `direction` prop 区分纵向/横向。

#### 4.3.2 自由定位组件 `PositionControl`

新建组件，核心结构：

```
PositionControl
├── 迷你画布 (MiniCanvas)
│   ├── 网格背景
│   └── 可拖拽元素标签 × N
│       └── 拖拽手柄 + 标题
├── 坐标输入面板
│   └── 每个元素: [名称] [x输入] [y输入]
└── 操作按钮
    └── "恢复默认位置"
```

**技术选型**：
- 拖拽：复用 `@dnd-kit/core`（项目已依赖）
- 画布：CSS 定位 + `transform: translate(x, y)`
- 坐标同步：拖拽结束 → 更新输入框；输入框变更 → 更新画布位置

**坐标单位**：像素（px），与预览区尺寸一致。迷你画布按比例缩放显示。

### 4.4 配置合并与注入扩展

**文件**：`packages/author-site/src/lib/config-merge.ts`

- `mergeSchemaDefaults`：处理 `__orderH` 和 `__positions` 的默认值合并

**文件**：`packages/author-site/src/lib/runtime-props.ts`

- `mergeConfigToProps`：确保 `__orderH` 和 `__positions` 正确注入组件 props

### 4.5 ConfigForm 渲染逻辑更新

在 `ConfigForm` 主组件中，按以下顺序渲染位置控制区域：

```tsx
<>
  {/* 1. 纵向排序（已有） */}
  {orderable && <OrderControl direction="vertical" ... />}

  {/* 2. 横向排序（新增） */}
  {orderableH && <OrderControl direction="horizontal" ... />}

  {/* 3. 自由定位（新增） */}
  {positionable && <PositionControl ... />}

  <Separator />

  {/* 4. 表单字段（已有） */}
  {fieldGroups.map(...)}
</>
```

## 五、验证规则

### 5.1 横向排序

| 规则 | 说明 |
|:-----|:-----|
| 最低数量 | `orderableHorizontal` 至少包含 2 项 |
| 数据完整性 | `__orderH` 必须包含所有声明项 |
| 与纵向排序独立 | 同一属性可同时出现在 `orderable` 和 `orderableHorizontal` 中 |

### 5.2 自由定位

| 规则 | 说明 |
|:-----|:-----|
| 最低数量 | `positionable.items` 至少包含 1 项 |
| 坐标范围 | x/y 值不做硬性限制，由组件代码自行约束 |
| 默认坐标 | 未在 `defaults` 中指定的元素，默认坐标为 `{ x: 0, y: 0 }` |
| 数据完整性 | `__positions` 必须包含 `positionable.items` 中的所有项 |

## 六、向后兼容性

| 场景 | 影响 |
|:-----|:-----|
| 不含新字段的旧 Schema | 无影响，不渲染新控件 |
| 含 `orderable` 但不含新字段的 Schema | 行为完全不变 |
| 新旧字段共存 | 各自独立工作，互不干扰 |
