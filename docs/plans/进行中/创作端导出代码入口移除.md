# 创作端导出代码入口移除

## 背景

创作端项目编辑页右上角存在“导出代码”按钮，用于下载项目 scaffold zip。当前创作端 CLI 已经能通过本地项目包流程覆盖代码拉取、校验、diff 和提交，继续在编辑页保留下载入口会形成重复能力。

## 目标

- 移除创作端项目编辑页右上角的“导出代码”入口。
- 移除创作端专用 scaffold 下载接口，避免 UI 删除后仍保留一条重复下载能力。
- 保留 CLI 作为本地代码获取和开发协作入口。
- 不改变使用端项目详情页的公开导出入口。
- 不改变现有 scaffold 项目包协议。

## 范围

- 创作端编辑页 header 按钮。
- 创作端项目 scaffold 下载 API。
- CLI 与本地项目包相关长期文档。
- 原“项目代码导出按钮”任务记录的状态说明。

## 方案

- 删除编辑页 header 中指向创作端 scaffold 下载接口的按钮。
- 删除按钮专用图标导入，避免未使用代码。
- 删除创作端专用 scaffold 下载 route 和测试；使用端公开下载继续保留，并复用迁移到 `src/lib` 的 zip 响应 helper。
- 文档中明确创作端编辑页不再提供下载代码入口，本地代码获取以 CLI `project pull` 为准。
