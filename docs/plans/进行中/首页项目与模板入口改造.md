# 首页项目与模板入口改造

## 背景

当前首页通过“添加项目”弹窗集中完成空白项目和模板项目创建。用户希望模板直接出现在首页目录和项目网格中，降低新建项目时的选择成本，并让模板身份更清晰。

## 目标

- 首页第一张卡片固定为“空白项目”，点击后填写项目名称和分类。
- 首页左侧目录支持普通项目分类和模板分类浏览。
- 模板项目可从首页复制为普通项目，复制时填写项目名称和分类。
- 普通项目更多菜单新增“复制当前项目”，并保留“保存为模板”。
- 项目文档同步更新，不再描述模板只存在于新建弹窗。

## 范围

- 创作端首页项目列表与模板入口。
- 项目分类字段、创建项目和复制项目接口。
- 项目模板需求与技术文档。

## 方案

- 给项目摘要和项目元数据增加可选分类，读取历史项目时归入“未分类”。
- 创建项目和从模板创建项目时写入分类。
- 新增普通项目复制接口，复用现有项目复制服务能力。
- 将首页从弹窗选模板改为左侧目录加主网格展示。
- 以轻量名称/分类弹窗承载空白创建、模板复制、普通项目复制三类动作。
