# 子智能体并行调用与状态显示修复

## 背景

创作端 AI 对话已支持通过 `delegateTask` 委派短生命周期子 Agent。当前用户期望一次任务中可以同时委派多个子智能体，并反馈界面出现“子 Agent 已完成”但整轮 AI 仍在处理、页面结果尚未完成的误导。

## 目标

- 支持主 Agent 在同一轮工具调用中并行启动多个 `delegateTask`。
- 子 Agent 卡片状态能表达“委派任务已返回，等待主 Agent 汇总/验收”，避免误解为整轮 AI 已结束。
- 保持子 Agent 不可递归创建子 Agent 的安全边界。
- 补充最小回归测试，覆盖并行配置与状态显示。

## 范围

- 后端：`packages/agent-service/src/backends/pi-tools/subagent-tool.ts`、相关单元测试。
- 前端：`packages/author-site/src/components/ai-elements/assistant-message.tsx`、相关展示测试。
- 文档：同步 AI 对话和独立 Agent 服务层的子 Agent 行为说明。

不处理本次无关的工作区脏数据、截图缓存、历史计划文档归档。

## 方案

1. 将 `delegateTask` 的工具执行模式从串行改为并行，使 Pi Agent core 可以在同一批工具调用中并发执行多个委派任务。
2. 保留 `runSubagent` 每次委派创建独立 harness、session、env 的模型，并继续用 `activeSubagents` 统一取消和销毁。
3. 在流式中仍把单个 `delegateTask` 的 completed 解释为“委派已返回”；当整条 assistant 消息仍在 streaming 时，显示“待主 Agent 汇总”，最终消息完成后再显示“已完成”。
4. 更新测试和文档，明确“子 Agent 完成”与“整轮 AI 完成”的区别。

## 风险与待确认事项

- 多个子 Agent 并行写同一文件时仍可能产生最后写入覆盖；需要主 Agent 在任务拆分时保持文件边界清晰。
- 并行委派会增加同一时间模型 API 并发和资源占用；本次先启用能力，不新增并发数配置。
