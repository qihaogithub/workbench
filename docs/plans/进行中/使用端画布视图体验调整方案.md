# 使用端画布视图体验调整方案

> 创建日期：2026-06-25
> 状态：已完成
> 范围：使用端项目级预览页、独立 viewer-site、创作端配置面板、shared 画布与配置组件、使用端预览与配置文档

## 背景

使用端项目预览需要默认进入画布视图，并在画布视图中保持配置面板可见。现有 shared 画布组件已经具备拖动视口、缩放和工具栏能力，但这些能力与创作端编辑权限绑定，导致使用端画布在只读状态下不可拖动和缩放。

## 目标

- 使用端项目级预览默认展示画布视图，显式 `mode=single` 时进入单页视图。
- 画布视图隐藏页面目录，单页视图保留页面目录。
- 有配置项时右侧配置面板默认打开；无配置项时不展示空面板。
- 使用端画布复用 shared 画布组件，只开放视口拖动、缩放、适应屏幕和重置视图。
- 创作端画布编辑能力保持不变。
- 配置面板采用统一的二级结构：一级展示页面列表与配置数量，二级展示所选页面的共享配置和本页配置。

## 范围

- 修改 shared 画布组件，区分只读、使用端可交互和创作端编辑三类模式。
- 修改 `author-site` 项目级 viewer。
- 修改独立 `viewer-site` 项目预览页。
- 修改创作端编辑页配置面板，使创作端与使用端使用同一控件。
- 更新使用端预览与配置模块文档。
- 不修改 `/viewer/[projectId]/[demoId]` 单页直达路由为画布入口。

## 方案

- 在 `PreviewCanvas` 增加 `interactionMode`，默认兼容旧的 `editable` 行为。
- `viewer` 模式仅启用视口交互和精简工具栏，页面项和自由节点仍以非编辑态渲染。
- 使用端项目级 viewer 默认 `canvas`，并在画布模式隐藏页面目录。
- 配置面板展示条件收敛为“启用配置面板且存在配置 Schema”。
- 新增 shared `PageConfigPanel`，统一创作端和使用端配置面板控件。
- 面板一级按页面列出该页面可配置总数，数量由共享配置和本页独有配置相加得到，但不展开共享/独有摘要；二级展示该页面可用的共享配置和本页独有配置。

## 风险与待确认事项

- 使用端画布的页面布局变更不持久化，本次不开放页面移动和缩放，避免用户误认为正在编辑项目布局。
- 独立 viewer-site 与 author-site viewer 的数据来源不同，需要分别验证。
- 剩余风险：需要在真实浏览器中确认鼠标拖动、滚轮缩放和右侧栏切换的视觉体验。
