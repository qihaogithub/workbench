# 自动保存同步失败复现与修复

## 背景

项目编辑页出现“同步失败”，用户侧表现为自动保存不可信。现有开发诊断脚本 `scripts/development/detect-sync-status-flap.mjs` 只采样“连接中”和“离线待同步”抖动，不能稳定触发和记录 Workspace flush 失败。

## 目标

- 维护同步诊断脚本，使其能复现并报告编辑页 Workspace flush 失败。
- 修复 agent-service 协同持久化默认数据目录解析错误，避免从包目录启动时找不到仓库根 `data/`。
- 补充单元测试、项目文档和脚本说明，保证后续排查路径清晰。

## 范围

- 修改 `packages/agent-service/src/collab/workspace-file-persistence.ts` 及对应单测。
- 修改 `scripts/development/detect-sync-status-flap.mjs`、`scripts/development/README.md` 和自动化脚本登记。
- 更新项目管理实时协同技术文档。
- 不回滚或整理当前工作树中与本任务无关的已有改动。

## 方案

1. 为 `WorkspaceFilePersistence` 增加仓库根查找逻辑，默认 `DATA_DIR` 对齐 author-site 和 agent-service 其他模块。
2. 扩展诊断脚本，在状态采样前后获取当前项目的可用 Session/Workspace 并调用 `/api/sessions/:sessionId/workspace-flush`。
3. 报告中记录 flush 结果、失败请求、状态采样和截图；flush 失败或页面出现“同步失败”时返回失败。
4. 用单元测试覆盖“从包目录启动且未设置 DATA_DIR”场景。
