# 创作端编辑与协同

## 当前状态

本模块当前保留三个仍需跟进的问题：AI 编辑自动保存重新打开复原的真实浏览器复验、页面版本恢复后的同步失败复验、编辑页 Agent 友好诊断能力的导出包手工检查。

## AI 编辑自动保存重新打开复原

状态：已完成代码级修复，等待真实浏览器流程复验。

当前结论：问题不是单一的 checkpoint、项目基准同步或前端刷新问题，而是“非协同文件写入”和“协同房间内存态”之间缺少版本基线。AI 工具直接写入 workspace 文件后，磁盘已是修复态；但 agent-service 内存中的 Y.Doc 可能仍保留旧文本，重新连接同一 live workspace 时旧协同文本会再次成为前端权威值，并在后续 flush 中覆盖磁盘。

已落地边界：checkpoint 成功前同步 Session Workspace 到项目当前 workspace；普通 workspace 持久化接口覆盖“看到已自动保存后立刻返回首页”的路径；前端 AI 文件变更后刷新当前页协同文本、refs、预览缓存和待同步标记；agent-service 协同层记录文件哈希基线，AI 工具写入后通知活跃房间重载，flush 前发现磁盘已脱离房间基线时以磁盘文件重置房间并拒绝旧文本写回。

待办：用真实浏览器流程复验 AI 修复后退出、重新打开同一项目，首个 `snapshot.apply source=collab` 应是修复后的短代码版本，而不是旧重复声明版本。

验证状态：author-site checkpoint / persist-workspace route 定向 Jest 通过；author-site `tsc --noEmit` 通过；agent-service `collab-room-manager`、`tool-hook-manager`、`collab-persistence` 定向 Vitest 通过，24 tests；agent-service `tsc --noEmit` 通过。未运行完整 `pnpm check:author`，因为当前运行时 pnpm 11.x 与仓库 `pnpm@8.15.0` 锁文件不兼容，触发安装检查且网络受限。

风险：当前策略选择“外部写入优先”而不是复杂合并。若未来引入真正多人同时编辑同一文件并允许 AI 并发改写，需要再设计冲突提示或人工合并。

## 页面版本恢复后同步失败

状态：已完成代码级修复，等待真实浏览器复验。

当前结论：页面版本恢复会先生成新的项目级恢复版本，再把恢复后的页面文件写回当前 live workspace。此前 live workspace 只更新 `updatedAt`，没有把 `.workspace.json` 的 `baseVersion` 提升到新生成的恢复版本；后续自动保存、退出保存或同步项目当前工作区时，`syncActiveWorkspaceToCanonical` 会把该 workspace 判定为 `WORKSPACE_STALE`，编辑页顶部显示“同步失败”。

已落地边界：新增 workspace 版本基线更新 helper；页面版本恢复接口在写入 Session live workspace 后同步更新 `baseVersion`，再执行项目基准工作区同步；完整项目恢复也会在创建恢复版本后同步 active workspace 基线，避免同类 stale 误判。

待办：用真实浏览器打开“伴读广告页面”，恢复手机页历史版本后等待自动保存，并验证顶部状态回到“已自动保存”；再执行返回首页和重新打开，确认不会再次出现“当前工作区已过期，请刷新项目后重试”。

验证状态：页面版本历史定向 Jest 通过，覆盖恢复页面版本后更新 live workspace 基线并成功同步项目工作区；author-site `tsc --noEmit` 通过。

风险：现有 `detect-sync-status-flap.mjs --flush-only` 只验证 `/workspace-flush`，不能覆盖后续 `/persist-workspace` 的 stale 失败。后续应给脚本增加 persist 探针，报告 `latestVersionId`、`activeWorkspaceId` 和 workspace `baseVersion`。

## 编辑页 Agent 友好诊断能力

验证状态：editor-diagnostics 类型、store、events API、export API、`useEditorDiagnostics` 定向 Jest 通过；AI 自动修复和 console buffer 相关定向 Jest 通过。`../../node_modules/.bin/tsc --noEmit` 未通过，阻断为既有测试文件 `workspace-flush/route.test.ts` 的 `Duplicate identifier 'TestResponse'`；本任务新增同名测试已通过 `export {}` 限定为模块作用域。

风险：编辑页主文件已有较多未提交改动，后续合并时需要避免覆盖既有改动；后端 JSONL 是排查产物，应保持在 `data/editor-diagnostics/`，不纳入版本控制。
