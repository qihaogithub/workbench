# 画布预览点击反复加载排查与修复

## 背景

创作端项目编辑页的画布预览模式中，页面加载完成后，用户点击预览内容会再次进入加载状态，形成反复加载的体感；同时多页面预览下载和渲染速度变慢。

## 目标

- 避免普通点击画布页面时触发不必要的页面文件拉取和 iframe 重建。
- 保留画布页面选择、拖拽、缩放、配置面板主动选页等既有编辑能力。
- 补充回归验证，防止后续再次把普通点击和配置编辑耦合。

## 范围

- `packages/shared/src/demo/PreviewCanvas.tsx`
- `packages/author-site/src/components/demo/preview-canvas-interaction-mode.test.tsx`
- `docs/项目文档/创作端/04-配置与预览/技术/07_截图服务与预览快照机制.md`

## 方案

画布页面普通点击只更新画布内部选择态，不直接调用外层 `onPageConfigEdit`。外层配置编辑入口仍由配置面板、页面树或明确聚焦页面动作触发。这样截图态页面不会因为一次普通点击被标记为 `editingPageId`，从而避免调度器把该页强制切换到实时 iframe。
