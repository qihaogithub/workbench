# 20260702 正式环境创作端部署与 data 覆盖

## 背景

用户要求重新部署创作端，并使用本地 `data/` 覆盖正式环境生产数据。

## 目标

- 部署当前工作区的 `author-site` 到正式环境。
- 使用当前本地 `data/` 覆盖正式环境共享 data。
- 覆盖前在正式环境保留生产 data 备份。
- 覆盖后确认创作端、Agent 服务、截图服务和 viewer 服务可用。

## 范围

- 本地验证：`corepack pnpm check:author`。
- 远端部署：`scripts/deploy-fast.sh author`。
- 数据覆盖：正式环境 Docker Compose 共享 data 挂载。

不包含：

- 提交 Git。
- 清理无关本地 dirty changes。
- 覆盖 `.env` 或密钥。

## 方案

1. 运行本地创作端验证。
2. 部署 `author-site` 到正式环境。
3. 确认正式环境 data 挂载目标。
4. 将当前生产 data 备份到项目目录外。
5. 将本地 `data/` 上传到项目目录外 staging。
6. 停止共享 data 的服务并覆盖正式 data。
7. 重启服务并运行健康检查。
