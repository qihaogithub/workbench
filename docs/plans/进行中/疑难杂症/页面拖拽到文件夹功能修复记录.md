# 页面拖拽到文件夹功能修复记录

> 创建时间：2026-06-03
> 问题范围：author-site 编辑页 DemoPageTree 拖拽交互
> 当前状态：已做初步修复但问题依旧，待进一步排查

---

## 1. 问题描述

在项目编辑页（`/demo/[id]/edit`）的页面列表中：

1. **创建文件夹后，拖拽页面到文件夹上无法执行** — 页面不会被移入文件夹
2. **文件夹与文件同级时，拖拽文件夹向下排序显示"排序保存失败"**

## 4. 数据流

### 4.1 拖拽结束回调链路

```
用户释放拖拽
  → DndContext.onDragEnd
    → handleDragEnd(event)
      → 判断场景（移入文件夹 / 同级排序 / 跨级移动）
        → 调用 projectApiClient.patchDemoPageMeta() 或 patchFolder() 或 reorderDemoPages()
          → fetch → Next.js API Route → fs-utils → workspace-tree.json
            → onPagesChange / onFoldersChange (React state)
```

### 4.2 树结构

```
workspace-tree.json
{
  "folders": [{ id: "folder_xxx", name: "文件夹名", parentId: null, order: 0 }],
  "pages": [{ id: "demo_xxx", name: "页面名", parentId: null, order: 0 }]
}
```

`flattenTree(pages, folders, expandedFolders)` 将树形结构扁平化为可视列表，仅展开的文件夹递归展开。

## 5. 原代码根因（第一轮分析）

`handleDragEnd` 原来的逻辑只有：

```javascript
const activeParent = activeItem.parentId ?? null;
const overParent = overItem.parentId ?? null;

if (activeParent === overParent) {
  // 仅处理同级同类型排序（reorderSiblings）
}
```

**问题 1: 页面拖到同级文件夹**

- 页面和文件夹都在根级（`parentId` 均为 `null`）
- `activeParent === overParent` → `true`，进入排序分支
- `reorderSiblings(pages, null, pageId, folderId)` → 在 `pages` 中找 `folderId` → 找不到 → 返回原数组不变
- 拖拽无效果

**问题 2: 文件夹拖到同级页面 → "排序保存失败"**

- `activeParent === overParent` → `true`，进入排序分支
- `reorderSiblings(folders, null, folderId, pageId)` → 在 `folders` 中找 `pageId` → 找不到 → 返回原数组不变
- `pageUpdates = []`（空数组），调用 `reorderDemoPages(projectId, sessionId, [], folderUpdates)`
- 后端 `reorder/route.ts` 校验 `pages.length === 0` → 返回 400
- 前端捕获异常 → 显示 "排序保存失败"

## 6. 已做修改（未解决）

### 6.1 `handleDragEnd` 重写为三场景

```
场景A：拖到文件夹上（且不在其中、且非同级双文件夹）→ 移入该文件夹
场景B：同级同类型 → reorderSiblings 排序
场景C：父级不同 → 移到目标层级末尾
其他（同级不同类型）→ 静默跳过
```

### 6.2 后端放宽 pages 校验

`reorder/route.ts` 从 `pages.length === 0` 改为只校验 `pages` 是否为数组。

### 6.3 问题仍然存在

修改后用户反馈"问题依然存在"，说明第一轮根因分析有遗漏，或存在更深层的问题。

---

## 7. 待排查方向

### 7.1 碰撞检测问题

当前使用 `closestCenter`，在两元素中心接近时可能选中相邻元素而非目标文件夹。可尝试：

- 使用 `closestCorners` 碰撞检测
- 自定义碰撞检测算法（`collisionDetection` prop）
- 为文件夹增加更大的碰撞区域

### 7.2 dnd-kit 事件调度问题

拖拽过程中 dnd-kit 通过 `DragOverEvent` 持续更新 `over`。如果在到达文件夹时 `over` 未被正确设置，`handleDragEnd` 将不会进入场景A。

可通过以下方式验证：

```javascript
// 在 handleDragStart 或临时 onDragOver 中打印
const handleDragOver = (event: DragOverEvent) => {
  console.log('over:', event.over?.id);
};
```

### 7.3 状态闭包问题

`handleDragEnd` 使用 `useCallback`，依赖 `pages`、`folders`。如果在拖拽期间父组件更新了 `pages`/`folders`，回调中的值可能已过期。

### 7.4 SortableContext 与"移入文件夹"的冲突

`SortableContext` 中所有项（包括文件夹）都参与排序动画。拖拽页面经过文件夹时，`verticalListSortingStrategy` 会实时重新排列所有项目的位置，可能导致：
- 文件夹视觉上的位移让用户困惑
- `over` 检测在动画过程中不稳定

可能的方案：
- 为文件夹设置 `disabled` 属性，禁止其参与排序动画（但仍可作为 drop target）
- 使用 `DragOverlay` 改善视觉反馈

### 7.5 API 层面的问题

即使前端正确调用 `patchDemoPageMeta`，后端也可能因以下原因失败：
- Session 过期（JWT 2小时）
- workspace-tree.json 不存在或损坏
- 文件夹 ID 在服务端已被删除
- 权限校验失败

建议添加更详细的错误日志。
