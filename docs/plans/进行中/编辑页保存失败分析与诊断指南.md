### 根因推断

**最可能原因**：autosave 触发时，author-site 到 agent-service 之间存在**瞬时连接波动**。

autosave 三步流水线 `persist-active-page → collab-flush → persist-workspace` 中任一步骤的网络 fetch 失败（如 TCP 超时、连接重置），导致 `syncWorkspaceToProject` catch 到错误，后续状态机进入 `offline`。

agent-service 诊断日志未记录 flush 相关事件，说明**请求从未到达 agent-service**，进一步印证是网络层问题。

## 离线状态粘性（已修复 2026-07-23）

### 现象

右上角显示"离线（本地草稿已保留）"，控制台无任何报错，刷新后立即恢复为"已自动保存"。

### 根因

`useWorkspaceAuthorityState` 的 `pollEvents`（每 2s 轮询）存在**粘性离线**缺陷：

- 轮询失败时把 `isConnected` 置为 `false`；
- 轮询成功时**仅当遇到新 `workspace_mutation_committed` 事件且 revision 递增**才恢复 `isConnected: true`，事件列表为空时不恢复。

一次瞬时轮询失败后，只要工作区无新提交，`isConnected` 永久卡在 `false`，`computeSaveStateFromContext` 持续返回 `offline`。catch 块静默（无 `console.error`），故控制台无报错；刷新后 `fetchState` 重新初始拉取并无条件置 `isConnected: true`，故立即恢复。

### 修复

`useWorkspaceAuthorityState.ts` 的 `pollEvents` 成功路径在 `setState` 末尾无条件恢复 `isConnected: true`（轮询请求成功返回即代表连接正常）。已补充回归测试"轮询失败后，后续成功轮询应恢复 isConnected = true"。

### 排查优先级

遇到"离线（本地草稿已保留）"且控制台无报错时，优先确认是否为本根因：若刷新即恢复且无持续网络异常，即为粘性离线，无需进一步排查网络层。

## 下次遇到时的排查步骤

### 1. 从 UI 错误标签判断失败阶段

- **保存失败：页面暂存异常** → 检查 `/api/sessions/{sessionId}/files/{demoId}` PUT 路由
- **保存失败：协同落盘异常** → 检查 `/api/sessions/{sessionId}/workspace-flush` POST 路由
- **保存失败：项目同步异常** → 检查 `/api/sessions/{sessionId}/persist-workspace` POST 路由

### 2. 检查 agent-service 诊断日志

```powershell
Select-String -Path "data\editor-diagnostics\agent-service.jsonl" -Pattern "flush|error|failed" | Select-Object -Last 30
```

关注是否有 `workspace.flush_failed` 事件，以及其中的 `errorCode` / `httpStatus`。

### 3. 直接测试 flush 端点

```powershell
# 在 author-site 所在机器执行
Invoke-WebRequest -Uri "http://localhost:3201/api/collab/projects/{projectId}/workspaces/{workspaceId}/flush-all?sessionId={sessionId}" -Method POST -ContentType "application/json" -Body "{}"
```

### 4. 检查 workspace authority 状态

```powershell
Invoke-WebRequest -Uri "http://localhost:3201/api/workspace-authority/projects/{projectId}/workspaces/{workspaceId}/state?sessionId={sessionId}"
```

关注 `revision` 和 `rootHash` 是否与预期一致。

### 5. 检查项目元数据

检查 `data/projects/{projectId}/project.json` 中的 `activeWorkspaceId` 和 `canonicalSynced*` 字段。
