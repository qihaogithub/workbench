# 自动保存与历史版本体验新方案实施

## 背景

当前项目文档中同时存在“保存时自动创建版本”和“实时保存只进入草稿、不创建版本”的表达，容易让用户和实现方把防丢失、历史回退、发布追溯混为一类操作。本次按照 Figma-like 产品模型统一语义：用户无需手动保存当前内容，系统持续自动保存并按策略沉淀历史记录。

## 目标

- 当前工作状态持续自动保存，用户不需要依赖“保存”按钮防丢失。
- 历史记录支持自动检查点、命名版本、发布快照和恢复快照四类条目。
- 发布前自动生成发布快照，保证发布状态可追溯。
- 恢复历史版本后生成恢复快照，保留操作轨迹。
- 同步更新项目管理模块需求文档和技术文档。

## 范围

- 共享版本类型与 author-site 项目版本/页面版本工具。
- author-site 发布流程。
- project-core 发布与恢复相关版本语义。
- 项目管理模块文档。

## 方案

- 在 `VersionInfo` 上增加兼容字段 `type`，历史旧数据缺省视为命名/普通版本。
- 新增统一版本创建选项，写入不同类型的历史条目。
- 发布流程先从当前 workspace 生成发布快照，再使用该版本号写入发布产物。
- 恢复流程生成 `restore_snapshot`，页面手动创建版本生成 `named_version`。
- 文档中将面向用户术语统一为自动保存、历史记录、自动保存记录、命名版本、发布快照。
