# 预览画布页面框尺寸不一致修复

## 背景

预览区画布模式偶发页面内容与页面选中框尺寸不一致。截图显示页面内容停留在较小尺寸，但外层页面框已经使用较大布局尺寸。

## 目标

- 定位画布页面框尺寸与预览内容尺寸不同步的原因。
- 修复 `PreviewPanel` 在画布页面框尺寸变化时未及时重新量测容器的问题。
- 补充最小测试覆盖，避免 ResizeObserver 未回调时再次回归。
- 同步更新预览机制文档。

## 范围

- `packages/demo-ui/src/PreviewPanel.tsx`
- `packages/author-site/components/demo/__tests__/PreviewPanel.test.tsx`
- `docs/项目文档/创作端/04-配置与预览/技术/02_实时预览机制.md`

不处理已有数据目录中的未提交变更，不重置用户保存的画布布局。

## 方案

`PreviewPanel` 在 `fillContainer` 模式下依赖容器宽高计算 iframe 的缩放。当前只在 `previewSize` 变化和 ResizeObserver 回调时更新尺寸；当父级页面框宽高变化但 ResizeObserver 没有及时回调，iframe 会继续使用旧尺寸或回退尺寸，导致内容与页面框不一致。

修复方式：

- 抽出正尺寸量测函数，只接受宽高都大于 0 的测量值。
- 每次渲染后的 layout effect 都读取当前容器尺寸，作为 ResizeObserver 的兜底。
- ResizeObserver 继续保留，但不再用 0 尺寸覆盖已有正尺寸。
- 用 Jest 覆盖“父级尺寸变化、ResizeObserver 无回调、组件 rerender 后 iframe 缩放更新”的场景。

## 风险与待确认事项

- 画布支持手动缩放页面框，本次修复不自动改写已保存布局宽高，避免覆盖用户意图。
- 已存在异常布局的数据可通过右键“重置大小”恢复；若后续需要批量修复历史布局，应另开数据迁移任务。
