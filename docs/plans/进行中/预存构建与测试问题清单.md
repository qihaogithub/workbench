## 背景

## 问题列表（已修复）

### 1. `fs-utils.ts:restoreVersion` 死代码阻塞守卫检查 ✅

- **文件**：`packages/author-site/src/lib/fs-utils.ts`
- **现象**：`check:workspace-authority` 守卫要求 `fs-utils.ts` 不包含 `export function restoreVersion`，但该函数仍存在且无任何引用方
- **修复**：删除 `restoreVersion` 函数及其版本恢复区块注释。`markWorkspaceBasedOnVersion`、`cleanupOldVersions` 等辅助函数仍被其他模块使用，保留不动。
- **验证**：`pnpm check:workspace-authority` 通过

### 2. `project-cli` TypeScript 类型错误 ✅

- **文件**：`packages/project-cli/src/index.ts`
- **错误**：`error TS2345` — `withPageListSummary` 参数类型期望 `{ pages: Array<{ runtimeType?: string }> }`，但 `DemoPageMeta` 已不再包含 `runtimeType` 字段（改为通过文件系统推断）
- **修复**：将 `withPageListSummary` 参数类型改为 `{ pages: unknown[]; folders: unknown[] }`，移除已失效的 `runtimeTypes` 计数逻辑和输出字段
- **验证**：`pnpm --filter @workbench/project-cli typecheck` 通过

### 3. `auto-preview-repair-guard.test.ts` — hash fingerprint 测试失败 ✅

- **文件**：`packages/author-site/src/lib/auto-preview-repair-guard.ts`
- **失败测试**：`uses code hash changes as a new repair fingerprint`
- **根因**：`buildAutoPreviewRepairFingerprint` 的 `stableDiagnosticKey` 未包含 `codeHash` 和 `moduleHash` 字段，导致仅 `codeHash` 不同的两次 repair 生成相同 fingerprint
- **修复**：在 `stableDiagnosticKey` 数组中加入 `diagnostic.codeHash` 和 `diagnostic.moduleHash`
- **验证**：8 个测试全部通过

### 4. `folders/[folderId]/route.test.ts` — 删除文件夹后文件存在性断言 ✅

- **文件**：`packages/author-site/src/app/api/projects/[projectId]/folders/[folderId]/route.test.ts`
- **失败测试**：`live Workspace 删除文件夹及内容通过 Authority 提交页面文件删除和 workspace-tree`
- **根因**：路由 DELETE handler 在 Authority mutation 后执行 `fs.rmSync(pageDir, { recursive: true, force: true })` 清理本地页面目录，测试断言 `toBe(true)` 与实际行为相反
- **修复**：将断言改为 `toBe(false)`
- **验证**：测试通过

### 5. `demos/[demoId]/route.test.ts` — 删除页面后文件存在性断言 ✅

- **文件**：`packages/author-site/src/app/api/projects/[projectId]/demos/[demoId]/route.test.ts`
- **失败测试**：`live Workspace 删除页面通过 Authority 一次性提交页面文件删除和 workspace-tree`
- **根因**：与问题 4 相同 — 路由 DELETE handler 在 Authority mutation 后执行 `fs.rmSync` 清理本地页面目录
- **修复**：将断言改为 `toBe(false)`
- **验证**：测试通过

### 6. `useVisualEditState.test.tsx` — `received is not iterable` ✅

- **文件**：`packages/author-site/src/app/demo/[id]/edit/__tests__/useVisualEditState.test.tsx`
- **失败测试**：3 个原型页相关测试 + 1 个高保真页图片本地化测试
- **根因**：`handleSendVisualPropertiesToAI` 中 `setTriggerAutoSend` 的调用参数已从 `string` 改为对象 `{ kind, visibleTitle, visibleSummary, hiddenPrompt }`，但测试仍直接对 `mock.calls[0][0]` 调用 `.toContain()`
- **修复**：将测试中的 `mock.calls[0][0]` 改为访问 `.hiddenPrompt` 属性后再调用 `.toContain()`。共修复 4 处断言。
- **验证**：8 个测试全部通过
