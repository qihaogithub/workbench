# 配置面板公共与独有配置项分离显示方案

> 版本：v1.0
> 创建日期：2026-05-10
> 状态：方案设计中

---

## 一、问题背景

当前配置面板已支持项目级配置（所有页面共享）与页面级配置（当前页面独有）的分开展示，但存在以下体验问题：

1. **视觉层级弱**：项目配置与页面配置仅通过一段文字标签区分（"📋 项目配置（所有页面共享）" / "📄 当前页面配置"），缺乏强烈的视觉边界感
2. **作用域不明确**：用户无法一眼识别某个字段的修改会影响所有页面还是仅当前页面
3. **分组混杂**：项目配置和页面配置内部都使用同一套智能分组逻辑（颜色配置、尺寸设置等），导致两个层级的分组标题重复，视觉上难以区分
4. **缺乏作用域标识**：字段级别没有任何标识表明其作用域范围

**典型用户困惑场景**：

> 用户在"基础配置"分组下修改了"页面标题"，切换到另一个页面后发现标题没变，以为系统出 Bug。实际上"页面标题"是页面级字段，但用户无法从 UI 上直观感知这一点。

---

## 二、设计目标

| 目标 | 说明 |
|:-----|:-----|
| **明确作用域** | 用户一眼即可识别字段属于"全局共享"还是"页面独有" |
| **视觉分层** | 项目配置与页面配置在视觉上有清晰的边界区分 |
| **减少困惑** | 消除"为什么改了这里，那里没变"的疑问 |
| **保持简洁** | 不增加额外的操作步骤，不改变现有交互流程 |
| **向后兼容** | 对现有 Schema 结构零侵入，纯 UI 层优化 |

## 四、详细设计

### 4.1 顶层区域划分

#### 4.1.1 项目配置区域（蓝色系）

| 元素 | 设计 |
|:-----|:-----|
| 区域标题 | "🔷 项目配置" + 副标题 "修改将影响所有页面" |
| 背景色 | 极淡的蓝色背景（`bg-blue-50/5` 或 `bg-sky-950/20`，视主题而定） |
| 边框 | 左侧 3px 蓝色边框（`border-l-3 border-blue-500`） |
| 圆角 | 与现有面板一致（`rounded-lg`） |
| 间距 | 与下方页面配置区域保持适当间距（`mb-6`） |

#### 4.1.2 页面配置区域（橙/灰色系）

| 元素 | 设计 |
|:-----|:-----|
| 区域标题 | "🟠 当前页面配置" + 副标题 "仅影响当前页面：{pageName}" |
| 背景色 | 与配置面板背景一致（不额外着色，避免页面配置区域过于花哨） |
| 边框 | 左侧 3px 橙色边框（`border-l-3 border-orange-500`） |
| 圆角 | `rounded-lg` |

**设计 rationale**：
- 项目配置使用蓝色（冷色调，代表"全局、稳定"）
- 页面配置使用橙色（暖色调，代表"局部、活跃"）
- 页面配置区域不着重着色，因为配置面板本身已是深色/浅色背景，避免色彩过多

### 4.2 字段级作用域标识

在每个配置字段的标题前添加**作用域徽标**：

| 作用域 | 徽标 | 颜色 | Tooltip |
|:-------|:-----|:-----|:--------|
| 项目级 | 🌐 | 蓝色 | "此字段为项目级配置，修改将影响所有页面" |
| 页面级 | 📄 | 橙色 | "此字段为页面级配置，仅影响当前页面" |

**徽标位置**：字段标题左侧，与标题文字保持 4px 间距

**徽标样式**：
- 尺寸：14px × 14px
- 不使用背景色块，纯图标即可（避免视觉过重）
- 鼠标悬停时显示 Tooltip 说明

**实现位置**：`FieldRenderer` 组件中，Label 渲染逻辑内

### 4.3 分组标题去重策略

当前项目配置和页面配置都使用 `parseSchemaToFields` 的智能分组逻辑，导致可能出现两个"基础配置"分组。

**方案**：保持现有分组逻辑不变，但为分组标题添加父级区域前缀的隐式区分（通过视觉区域已足够区分）。

例如：
- 项目配置区域下的"基础配置" → 用户通过区域蓝色边框已知这是项目级
- 页面配置区域下的"基础配置" → 用户通过区域橙色边框已知这是页面级

**不改动**：分组标题文字本身，避免标题过长

### 4.4 空状态处理

| 场景 | 处理 |
|:-----|:-----|
| 无项目级配置 | 不渲染项目配置区域，仅显示页面配置区域 |
| 项目配置 Schema 无字段 | 渲染项目配置区域，内部显示 "暂无项目级配置项" |
| 页面配置 Schema 无字段 | 渲染页面配置区域，内部显示 "暂无页面级配置项" |

## 五、组件改造点

### 5.1 ConfigForm 组件增强

`ConfigForm` 需新增一个可选 prop 用于标识作用域：

```typescript
interface ConfigFormProps {
  schema: string;
  onChange: (data: Record<string, unknown>) => void;
  onSchemaChange?: (schema: string) => void;
  initialData?: Record<string, unknown>;
  readonly?: boolean;
  className?: string;
  sessionId?: string;
  scope?: 'project' | 'page';  // 新增：作用域标识
  pageName?: string;           // 新增：当前页面名称（scope='page' 时显示）
}
```

**新增区域标题渲染逻辑**：

- 当 `scope='project'` 时，在表单内容外部包裹蓝色边框区域，顶部显示 "🔷 项目配置" + "修改将影响所有页面"
- 当 `scope='page'` 时，在表单内容外部包裹橙色边框区域，顶部显示 "🟠 当前页面配置" + "仅影响当前页面：{pageName}"
- 当 `scope` 未传时，保持现有行为（无区域包裹，兼容旧用法）

### 5.2 FieldRenderer 组件增强

`FieldRenderer` 需接收 `scope` prop，用于渲染字段级徽标：

```typescript
interface FieldRendererProps {
  field: FieldConfig;
  value: unknown;
  onChange: (value: unknown) => void;
  sessionId?: string;
  readonly?: boolean;
  onNoteClick: (fieldKey: string) => void;
  scope?: 'project' | 'page';  // 新增
}
```

**徽标渲染逻辑**：

- `scope='project'`：标题前渲染 🌐 蓝色图标
- `scope='page'`：标题前渲染 📄 橙色图标
- `scope` 未传：不渲染徽标（兼容旧用法）

### 5.3 edit/page.tsx 改造

编辑页中两个 `ConfigForm` 的调用处需传入新增的 props：

```typescript
// 项目级配置
<ConfigForm
  key={`project-${projectConfigSchema}`}
  schema={projectConfigSchema}
  onChange={...}
  onSchemaChange={handleProjectSchemaChange}
  initialData={configData}
  sessionId={sessionId}
  scope="project"           // 新增
/>

// 页面级配置
<ConfigForm
  key={schema}
  schema={schema}
  onChange={handleConfigChange}
  onSchemaChange={handleSchemaChange}
  initialData={configData}
  sessionId={sessionId}
  scope="page"              // 新增
  pageName={currentPageName} // 新增
/>
```

### 5.4 类型定义更新

`types.ts` 中 `ConfigFormProps` 接口需同步更新：

```typescript
export interface ConfigFormProps {
  schema: string;
  onChange: (data: Record<string, unknown>) => void;
  onSchemaChange?: (schema: string) => void;
  initialData?: Record<string, unknown>;
  readonly?: boolean;
  className?: string;
  sessionId?: string;
  scope?: 'project' | 'page';
  pageName?: string;
}
```

## 六、视觉规范

### 6.1 颜色 Token

| 用途 | 项目级（蓝色） | 页面级（橙色） |
|:-----|:-------------|:-------------|
| 区域左边框 | `border-blue-500` | `border-orange-500` |
| 区域背景（暗色主题） | `bg-blue-950/20` | 无（使用默认卡片背景） |
| 区域背景（浅色主题） | `bg-blue-50/50` | 无（使用默认卡片背景） |
| 标题文字 | `text-blue-400` | `text-orange-400` |
| 副标题文字 | `text-blue-400/60` | `text-orange-400/60` |
| 字段徽标 | 🌐 蓝色 | 📄 橙色 |

### 6.2 间距规范

```
配置面板容器
├── padding: p-4
├── 项目配置区域
│   ├── margin-bottom: mb-6
│   ├── padding: p-4
│   ├── 区域标题: mb-3
│   └── 表单内容
└── 页面配置区域
    ├── padding: p-4
    ├── 区域标题: mb-3
    └── 表单内容
```

### 6.3 区域标题样式

```
区域标题行
├── 图标: h-4 w-4
├── 主标题: text-sm font-semibold
├── 副标题: text-xs text-muted-foreground
└── 整体: flex items-center gap-2 mb-3
```

## 七、交互细节

### 7.1 Tooltip 提示

字段徽标悬停时显示 Tooltip，文案：

- 项目级："此配置项属于项目级，修改后将影响所有页面"
- 页面级："此配置项属于当前页面，仅影响「{pageName}」"

使用现有 `Tooltip` 组件实现。

### 7.2 响应式适配

- 配置面板宽度较窄时（< 280px），区域副标题换行显示
- 字段徽标在极窄宽度下可隐藏（但保留区域级区分）

### 7.3 动画过渡

- 区域切换（如从无项目配置到创建项目配置）使用淡入动画
- 使用现有 `Collapsible` 组件的过渡效果

## 九、风险与回退

| 风险 | 影响 | 缓解措施 |
|:-----|:-----|:---------|
| 新增 props 导致旧调用处 TypeScript 报错 | 中 | `scope` 和 `pageName` 设为可选，不传时保持现有行为 |
| 视觉改动用户不适应 | 低 | 改动为纯 additive，不改变原有交互逻辑 |
| 暗色主题下颜色对比度不足 | 低 | 使用 Tailwind 语义化颜色，确保通过 a11y 对比度检查 |

**回退策略**：
- 若需回退，仅需移除 `scope` 和 `pageName` props 的传入，组件自动恢复现有行为
- 不涉及数据层改动，零风险
