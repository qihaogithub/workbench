# 配置面板公共与独有配置项分离显示方案

> 版本：v2.0
> 创建日期：2026-05-10
> 状态：方案设计中

---

## 一、问题背景

当前配置面板已支持项目级配置（所有页面共享）与页面级配置（当前页面独有）的分开展示，但存在以下体验问题：

1. **视觉层级弱**：项目配置与页面配置仅通过一段文字标签区分（"📋 项目配置（所有页面共享）" / "📄 当前页面配置"），缺乏强烈的视觉边界感
2. **作用域不明确**：用户无法一眼识别某个字段的修改会影响所有页面还是仅当前页面
3. **分组混杂**：项目配置和页面配置内部都使用同一套智能分组逻辑（颜色配置、尺寸设置等），导致两个层级的分组标题重复，视觉上难以区分
4. **标签不一致**：编辑页使用 Emoji 标签（📋📄），查看器页面使用纯文字标签（"项目配置"/"页面配置"），风格不统一

**典型用户困惑场景**：

> 用户在"基础配置"分组下修改了"页面标题"，切换到另一个页面后发现标题没变，以为系统出 Bug。实际上"页面标题"是页面级字段，但用户无法从 UI 上直观感知这一点。

---

## 二、设计目标

| 目标 | 说明 |
|:-----|:-----|
| **明确作用域** | 用户一眼即可识别配置属于"全局共享"还是"页面独有" |
| **视觉分层** | 项目配置与页面配置在视觉上有清晰的边界区分 |
| **减少困惑** | 消除"为什么改了这里，那里没变"的疑问 |
| **保持简洁** | 不增加额外的操作步骤，不改变现有交互流程 |
| **职责分离** | 视觉区域包裹逻辑与表单渲染逻辑解耦，各自独立可维护 |
| **全局一致** | 编辑页与查看器页面使用同一套作用域包裹组件，视觉风格统一 |

---

## 三、方案总览

采用**"左侧色条 + 区域标题 + 分组隔离"**策略，通过独立的 `ConfigScopeWrapper` 组件实现区域视觉区分：

```
┌─ 配置面板 ─────────────────────────────────────────────┐
│                                                         │
│  ┌─ [Globe] 项目配置 ─────────────────────────────────┐ │
│  │  所有页面共享                                       │ │
│  │  ▎                                                  │ │
│  │  ▎ ┌─ 基础配置 ────────────────────────────────┐  │ │
│  │  ▎ │  品牌色          ┌──────────────────┐     │  │ │
│  │  ▎ │  Logo 图片       │ [上传]           │     │  │ │
│  │  ▎ └──────────────────────────────────────────┘  │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                         │
│  ┌─ [FileText] 当前页面配置 ──────────────────────────┐ │
│  │  仅「首页」                                         │ │
│  │  ▎                                                  │ │
│  │  ▎ ┌─ 基础配置 ────────────────────────────────┐  │ │
│  │  ▎ │  页面标题        ┌──────────────────┐     │  │ │
│  │  ▎ │  副标题          │ 探索色彩的无限可能│     │  │ │
│  │  ▎ └──────────────────────────────────────────┘  │ │
│  │  ▎                                                  │ │
│  │  ▎ ┌─ 显示选项 ────────────────────────────────┐  │ │
│  │  ▎ │  启用动画        [●──────────────]         │  │ │
│  │  ▎ └──────────────────────────────────────────┘  │ │
│  └─────────────────────────────────────────────────────┘
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**设计决策**：

- **不加区域背景色**：配置面板本身已有背景色，嵌套背景色导致"卡片套卡片"感，左侧色条已足够区分
- **不加字段级徽标**：区域级区分已充分传达作用域信息，字段级徽标是冗余的视觉噪音
- **使用 lucide 图标**：替代 Emoji，跨平台渲染一致，与项目现有图标体系统一

## 四、详细设计

### 4.1 顶层区域划分

#### 4.1.1 项目配置区域（蓝色系）

| 元素 | 设计 |
|:-----|:-----|
| 区域标题 | lucide `Globe` 图标 + "项目配置" |
| 副标题 | "所有页面共享" |
| 左侧色条 | 3px 蓝色边框（`border-l-3 border-blue-500`） |
| 背景 | 无额外着色，使用配置面板默认背景 |
| 圆角 | `rounded-lg` |
| 间距 | 与下方页面配置区域保持 `mb-4` |

#### 4.1.2 页面配置区域（橙色系）

| 元素 | 设计 |
|:-----|:-----|
| 区域标题 | lucide `FileText` 图标 + "当前页面配置" |
| 副标题 | "仅「{pageName}」" |
| 左侧色条 | 3px 橙色边框（`border-l-3 border-orange-500`） |
| 背景 | 无额外着色 |
| 圆角 | `rounded-lg` |

**色彩语义**：
- 蓝色（冷色调）→ 全局、稳定、共享
- 橙色（暖色调）→ 局部、活跃、独立

### 4.2 分组标题去重策略

项目配置和页面配置内部都使用 `parseSchemaToFields` 的智能分组逻辑，可能出现两个"基础配置"分组。

**方案**：保持分组标题文字不变，通过区域左侧色条隐式区分。用户看到蓝色色条区域下的"基础配置"即知为项目级，橙色色条下即知为页面级，无需修改分组标题。

### 4.3 空状态处理

| 场景 | 处理 |
|:-----|:-----|
| 无项目级配置 | 不渲染项目配置区域，仅显示页面配置区域 |
| 项目配置 Schema 无字段 | 渲染项目配置区域，内部显示 "暂无项目级配置项" |
| 页面配置 Schema 无字段 | 渲染页面配置区域，内部显示 "暂无页面级配置项" |

## 五、组件设计

### 5.1 新增 ConfigScopeWrapper 组件

**职责**：为配置表单提供作用域视觉包裹，与 `ConfigForm` 完全解耦。

**文件位置**：`components/demo/ConfigScopeWrapper.tsx`

```typescript
import { Globe, FileText } from "lucide-react";
import { cn } from "@/lib/utils";

type ConfigScope = "project" | "page";

interface ConfigScopeWrapperProps {
  scope: ConfigScope;
  pageName?: string;
  children: React.ReactNode;
  className?: string;
}

const SCOPE_CONFIG: Record<
  ConfigScope,
  {
    icon: LucideIcon;
    title: string;
    subtitle: string;
    borderClass: string;
    titleClass: string;
  }
> = {
  project: {
    icon: Globe,
    title: "项目配置",
    subtitle: "所有页面共享",
    borderClass: "border-l-3 border-blue-500",
    titleClass: "text-blue-400",
  },
  page: {
    icon: FileText,
    title: "当前页面配置",
    subtitle: "仅「{pageName}」",
    borderClass: "border-l-3 border-orange-500",
    titleClass: "text-orange-400",
  },
};

export function ConfigScopeWrapper({
  scope,
  pageName,
  children,
  className,
}: ConfigScopeWrapperProps) {
  const config = SCOPE_CONFIG[scope];
  const Icon = config.icon;
  const subtitle = scope === "page" && pageName
    ? config.subtitle.replace("{pageName}", pageName)
    : config.subtitle;

  return (
    <div className={cn("rounded-lg p-4", config.borderClass, className)}>
      <div className="flex items-center gap-2 mb-3">
        <Icon className={cn("h-4 w-4", config.titleClass)} />
        <span className={cn("text-sm font-semibold", config.titleClass)}>
          {config.title}
        </span>
        <span className="text-xs text-muted-foreground">{subtitle}</span>
      </div>
      {children}
    </div>
  );
}
```

**设计要点**：

- `SCOPE_CONFIG` 常量集中管理两种作用域的视觉配置，新增作用域类型只需扩展此对象
- `ConfigForm` 无需任何改动，保持纯粹的表单渲染职责
- 组件轻量（~50 行），无内部状态，纯展示组件

### 5.2 edit/page.tsx 改造

编辑页中两个 `ConfigForm` 的调用处用 `ConfigScopeWrapper` 包裹：

```typescript
import { ConfigScopeWrapper } from "../../../../../components/demo";

// 当前页面名称（从 demoPages 中获取）
const activePageName = demoPages.find(p => p.id === activeDemoId)?.name ?? "";

// 项目级配置
{projectConfigSchema && (
  <ConfigScopeWrapper scope="project">
    <ConfigForm
      key={`project-${projectConfigSchema}`}
      schema={projectConfigSchema}
      onChange={(data) => { setConfigData((prev) => ({ ...prev, ...data })); }}
      onSchemaChange={handleProjectSchemaChange}
      initialData={configData}
      sessionId={sessionId}
    />
  </ConfigScopeWrapper>
)}

// 页面级配置
<ConfigScopeWrapper scope="page" pageName={activePageName}>
  <ConfigForm
    key={schema}
    schema={schema}
    onChange={handleConfigChange}
    onSchemaChange={handleSchemaChange}
    initialData={configData}
    sessionId={sessionId}
  />
</ConfigScopeWrapper>
```

**改动点**：
- 移除原有的文字标签（`📋 项目配置（所有页面共享）` / `📄 当前页面配置`）
- 移除 `<Separator />`（区域色条已提供视觉分隔）
- 用 `ConfigScopeWrapper` 包裹 `ConfigForm`

### 5.3 viewer 页面改造

两个查看器页面同样使用 `ConfigScopeWrapper` 替换原有文字标签：

**`viewer/[projectId]/page.tsx`**：

```typescript
{data.projectConfigSchema && (
  <ConfigScopeWrapper scope="project">
    <ConfigForm ... />
  </ConfigScopeWrapper>
)}

<ConfigScopeWrapper scope="page" pageName={activePageName}>
  <ConfigForm ... />
</ConfigScopeWrapper>
```

**`viewer/[projectId]/[demoId]/page.tsx`**：同上模式。

### 5.4 组件导出

在 `components/demo/index.ts` 中新增导出：

```typescript
export { ConfigScopeWrapper } from './ConfigScopeWrapper';
```

### 5.5 ConfigForm 无需改动

`ConfigForm`、`FieldRenderer`、`FieldGroupSection` 均无需任何改动。作用域视觉逻辑完全由外层 `ConfigScopeWrapper` 承担，`ConfigForm` 保持纯粹的表单渲染职责。

## 六、视觉规范

### 6.1 颜色 Token

| 用途 | 项目级（蓝色） | 页面级（橙色） |
|:-----|:-------------|:-------------|
| 左侧色条 | `border-blue-500` | `border-orange-500` |
| 标题图标 | `text-blue-400` | `text-orange-400` |
| 标题文字 | `text-blue-400` | `text-orange-400` |
| 副标题文字 | `text-muted-foreground` | `text-muted-foreground` |

### 6.2 间距规范

```
配置面板容器
├── padding: p-4
├── 项目配置区域 (ConfigScopeWrapper scope="project")
│   ├── margin-bottom: mb-4
│   ├── padding: p-4
│   ├── 区域标题: mb-3
│   └── ConfigForm 内容
└── 页面配置区域 (ConfigScopeWrapper scope="page")
    ├── padding: p-4
    ├── 区域标题: mb-3
    └── ConfigForm 内容
```

### 6.3 区域标题样式

```
区域标题行
├── 图标: h-4 w-4 (lucide Globe / FileText)
├── 主标题: text-sm font-semibold
├── 副标题: text-xs text-muted-foreground
└── 整体: flex items-center gap-2 mb-3
```
