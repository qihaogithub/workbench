# 预览区跨页面内容串用排查与修复

## 背景

用户反馈预览区偶发显示错误页面：当前选中页面 A，但预览会使用页面 A 的尺寸展示页面 B 的内容；同一操作有时又正常。

## 目标

- 定位页面尺寸与页面内容来自不同页面的原因。
- 修复画布预览、配置变更截图和截图批量任务中的跨页面代码回退问题。
- 修复单页预览复用 iframe 时旧页面运行时消息晚到导致的跨页内容露出。
- 补充最小验证，确保未加载页面不会误用当前活动页代码，旧请求消息不会解锁当前预览。

## 范围

- 创作端编辑页的页面代码选择逻辑。
- 画布预览页卡与截图批量任务的页面代码来源。
- `PreviewPanel` 与 iframe runtime 的加载完成、尺寸测量、运行错误消息版本校验。
- 不调整截图服务或页面尺寸解析规则。

## 方案

- 抽出纯函数统一判断某个页面预览应使用的代码。
- 规则限定为：优先使用该页面自身已加载代码；只有能确认当前编辑器代码归属目标页时，才允许使用当前编辑器代码作为临时来源；其他页面没有代码时保持空值，避免串用内容。
- 将画布页卡和截图任务中的 `pageCodes[p.id] || code` 替换为统一函数。
- 让 `PreviewPanel` 每次代码更新生成 requestId，iframe 回传加载完成、运行错误、内容高度、positionable 尺寸时携带同一 requestId；父窗口忽略非当前 requestId 的消息。
- 增加单元测试覆盖跨页回退边界和旧请求 `LOADED` 消息忽略逻辑。
