# 找回密码与管理后台功能方案

> 方案状态：待评审  
> 创建日期：2026-05-28  
> 更新日期：2026-05-28  
> 目标：提供用户密码找回机制 + 简化管理后台

---

## 一、现状分析

### 1.1 当前认证机制

| 项目       | 现状                                            |
| ---------- | ----------------------------------------------- |
| 密码存储   | bcrypt 加密（salt rounds = 10）                 |
| 认证方式   | JWT Token（httpOnly Cookie，7 天过期）          |
| 用户数据库 | SQLite（`data/users.db`）                       |
| 用户表结构 | `id`, `username`, `password_hash`, `created_at` |
| 注册验证   | 用户名 3-20 字符，密码 ≥6 字符                  |
| 安全机制   | 登录失败统一返回"用户名或密码错误"              |

### 1.2 存在的问题

1. **无找回密码入口**：登录页仅提供"注册"链接，无"忘记密码"选项
2. **无安全验证渠道**：系统未收集用户邮箱/手机号，无法通过传统方式验证身份
3. **无密码重置机制**：数据库缺少重置令牌、过期时间等字段
4. **管理员介入缺失**：无后台管理界面供管理员重置用户密码

### 1.3 约束条件

- 系统为内部开发工具，用户量较小
- 无第三方邮件服务/SMS 服务集成
- 需要保持简单、低维护成本的方案

---

## 二、方案设计

### 2.1 推荐方案：简化管理后台 + 用户自助修改

**核心功能**：

1. **管理员后台**：管理用户列表、重置用户密码
2. **用户设置**：登录后修改密码（免原密码验证）
3. **忘记密码入口**：引导用户联系管理员

## 三、详细设计

### 3.1 功能流程

#### 3.1.1 用户忘记密码流程

```
用户点击"忘记密码"
  ↓
显示提示信息："请联系系统管理员重置密码"
  ↓
管理员登录管理后台
  ↓
选择目标用户 → 设置新密码
  ↓
告知用户新密码
  ↓
用户登录后可自行修改密码
```

#### 3.1.2 管理员操作流程

```
管理员访问 /admin
  ↓
系统验证管理员身份（通过环境变量配置）
  ↓
显示用户列表
  ↓
选择用户 → 点击"重置密码"
  ↓
输入新密码 → 确认
  ↓
系统更新密码 + 记录操作日志
```

#### 3.1.3 用户修改密码流程

```
用户登录后点击右上角"设置"
  ↓
弹出设置弹窗
  ↓
选择"修改密码"
  ↓
输入新密码 + 确认密码
  ↓
提交（无需输入原密码）
  ↓
修改成功，提示重新登录
```

### 3.2 管理员认证机制

#### 3.2.1 方案：环境变量配置管理员

**实现方式**：

- 通过环境变量 `ADMIN_USERNAME` 配置管理员用户名
- 管理员使用自己的账号密码登录，系统验证是否为管理员
- 无需复杂的角色系统，适合小规模内部工具

**环境变量配置**：

```bash
# .env
ADMIN_USERNAME=admin  # 管理员用户名
```

**权限验证逻辑**：

```typescript
// middleware.ts 或 API 中验证
function isAdmin(user: UserPayload): boolean {
  return user.username === process.env.ADMIN_USERNAME;
}
```

#### 3.2.2 数据库改造

**新增表：密码重置日志**

```sql
CREATE TABLE password_reset_logs (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,           -- 被重置密码的用户 ID
  reset_by TEXT NOT NULL,          -- 操作的管理员用户名
  reset_method TEXT NOT NULL,      -- 'admin_reset' | 'self_change'
  created_at INTEGER NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

| 字段           | 类型    | 说明                                                         |
| -------------- | ------- | ------------------------------------------------------------ |
| `id`           | TEXT    | 日志 ID（UUID）                                              |
| `user_id`      | TEXT    | 被操作的用户 ID                                              |
| `reset_by`     | TEXT    | 操作者（管理员用户名或用户自己）                             |
| `reset_method` | TEXT    | 重置方式：admin_reset（管理员重置）/ self_change（用户自改） |
| `created_at`   | INTEGER | 操作时间戳                                                   |

### 3.3 API 设计

#### 3.3.1 管理员重置用户密码

**接口**: `POST /api/admin/users/:userId/reset-password`

**权限**: 需要管理员身份（验证 `username === process.env.ADMIN_USERNAME`）

**请求体**:

```json
{
  "newPassword": "string" // 新密码（必填）
}
```

**响应**:

```json
{
  "success": true,
  "data": {
    "userId": "string",
    "username": "string",
    "message": "密码重置成功"
  }
}
```

**错误响应**:

```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "无管理员权限"
  }
}
```

#### 3.3.2 获取用户列表（管理后台）

**接口**: `GET /api/admin/users`

**权限**: 需要管理员身份

**响应**:

```json
{
  "success": true,
  "data": {
    "users": [
      {
        "id": "string",
        "username": "string",
        "createdAt": 1234567890
      }
    ]
  }
}
```

#### 3.3.3 用户修改密码（免原密码）

**接口**: `POST /api/auth/change-password`

**权限**: 需要登录

**请求体**:

```json
{
  "newPassword": "string" // 新密码（必填）
}
```

**响应**:

```json
{
  "success": true,
  "data": {
    "message": "密码修改成功，请重新登录"
  }
}
```

**说明**：

- 此接口设计为**免原密码验证**，适用于内部工具场景
- 用户已登录状态下直接修改
- 修改成功后清除当前 session，要求重新登录

### 3.4 前端页面

**路径**: `/admin`

**权限**: 仅管理员可访问（通过 middleware 验证）

**页面结构**:

```

  ├── 标题："用户管理"
  ├── 用户列表（表格）
  │   ├── 用户名
  │   ├── 注册时间

**交互流程**:

```
点击"重置密码"按钮

### 3.5 安全策略

#### 3.5.1 密码规则

| 规则     | 要求                                   |
| -------- | -------------------------------------- |
| 最小长度 | 6 字符（保持现有规则）                 |
| 复杂度   | 建议包含大小写字母+数字（非强制）      |
| 验证逻辑 | 使用 `validatePassword()` 函数（已有） |

#### 3.5.2 管理后台访问控制

| 项目       | 策略                                           |
| ---------- | ---------------------------------------------- |
| 身份验证   | 通过 JWT Cookie 验证登录状态                   |
| 权限验证   | 检查 `username === process.env.ADMIN_USERNAME` |
| 未授权处理 | 重定向到首页 + 提示无权限                      |
| 环境变量   | `ADMIN_USERNAME` 必须在 `.env` 中配置          |

#### 3.5.3 操作日志

| 操作类型       | 记录内容                                                              |
| -------------- | --------------------------------------------------------------------- |
| 管理员重置密码 | user_id, reset_by（管理员）, reset_method='admin_reset', created_at   |
| 用户修改密码   | user_id, reset_by（用户自己）, reset_method='self_change', created_at |

## 六、风险评估

| 风险               | 影响                       | 应对措施                             |
| ------------------ | -------------------------- | ------------------------------------ |
| 管理员账号泄露     | 所有用户密码可被重置       | 强化管理员密码 + 操作日志审计        |
| 免原密码修改被滥用 | 用户设备未锁屏时被他人修改 | 内部工具场景，风险可控               |
| 环境变量未配置     | 管理后台无法使用           | 启动时检查 `ADMIN_USERNAME` 是否存在 |
| 日志表数据增长     | 长期运行后数据量大         | 定期清理或归档（非关键数据）         |

### 8.1 数据库迁移脚本示例

```sql
-- migration_001_add_password_reset_logs.sql

-- 创建密码重置日志表
CREATE TABLE IF NOT EXISTS password_reset_logs (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  reset_by TEXT NOT NULL,
  reset_method TEXT NOT NULL,
  created_at INTEGER NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_reset_logs_user ON password_reset_logs(user_id);
CREATE INDEX IF NOT EXISTS idx_reset_logs_created ON password_reset_logs(created_at);
```

### 8.2 管理员权限验证示例

```typescript
// packages/author-site/src/lib/admin.ts
import { UserPayload } from "./auth/jwt";

export function isAdmin(user: UserPayload | null): boolean {
  if (!user) return false;
  return user.username === process.env.ADMIN_USERNAME;
}

export function requireAdmin(user: UserPayload | null): void {
  if (!isAdmin(user)) {
    throw new Error("UNAUTHORIZED: 无管理员权限");
  }
}
```
