# 预览区右键图层菜单交互修正方案

> 状态：已实施  
> 创建时间：2026-06-30 14:17 CST  
> 更新时间：2026-06-30  
> 适用模块：创作端项目编辑页 / 单页预览 / 预览区顶部工具栏 / 属性面板  

## 背景

创作端项目编辑页目前有两个图层菜单入口：

- 预览区内右键元素后弹出的图层菜单。
- 预览区单页面模式顶部、页面下拉选择左侧的图层按钮。

两处入口服务的是同一件事：帮助用户从页面结构里选择要编辑的元素。因此后续不应各自维护菜单结构、命名规则和交互状态。

当前问题主要是菜单更像调试用 DOM 命中栈，用户看到多个“内容区域 / 容器”时无法判断真实对应关系。新的方案应展示正式 DOM 树状结构，同时对非技术用户做有限可读性优化。

## 目标

- 右键图层菜单和预览区顶部图层按钮菜单使用同一个菜单组件。
- 菜单展示真实 DOM 树状结构，不再展示扁平命中列表。
- 右键触发时，菜单只展示当前选中元素及其子级。
- 预览区顶部图层按钮触发时，也复用同一树形组件，只是展示范围由顶部入口场景决定。
- 节点名称面向用户可读，但不改变真实父子关系。
- 不引入完整设计工具图层面板，不扩大本次改动边界。

## 方案

### 1. 统一图层菜单组件

新增或整理为一个共享组件，例如 `LayerTreeMenu`。两个入口都只负责传入数据和打开位置：

- 预览区右键：传入当前右键选中节点作为根节点。
- 预览区顶部图层按钮：传入当前页面或当前选中上下文需要展示的树。

菜单展开、选中、高亮、节点点击、空状态、滚动限制和键盘关闭等交互统一由该组件维护。

### 2. 使用正式 DOM 树数据

菜单数据应是树，而不是 `elementsFromPoint()` 的扁平数组。

每个节点至少包含：

- 稳定节点标识。
- 父子关系。
- 标签名或语义类型。
- 可读名称。
- 是否为当前选中节点。
- 子节点列表。

树的顺序遵循 DOM 真实顺序，不因为“图片更重要”“按钮更重要”而重排。这样用户看到的是页面结构，而不是系统猜测出的候选列表。

### 3. 右键菜单只展示当前节点子树

右键触发流程建议为：

1. 根据右键位置命中并确认当前选中元素。
2. 以该元素作为菜单根节点。
3. 只展示该元素及其子级。
4. 点击子节点后，预览区选中框和属性栏同步切换到该节点。

这样可以避免右键时弹出整页复杂结构，也避免暴露与当前操作无关的父级、兄弟节点和全局页面节点。

### 4. 顶部图层按钮复用同一组件

图层按钮放在预览区单页面模式顶部，位于页面下拉选择左侧。该按钮可展示更完整的当前页面结构，但必须复用同一个 `LayerTreeMenu`。

差异只放在数据范围上，不再维护第二套菜单：

- 右键入口：当前节点子树。
- 顶部按钮入口：当前页面或当前选中上下文定义的图层范围。

两处入口的节点样式、命名、点击行为和选中反馈保持一致。

### 5. 有限优化非技术用户可读性

可读性优化只改显示文案，不改树结构。

建议规则：

- `img` 显示为“图片”，有 `alt` 时优先使用 `alt`。
- CSS 背景图所在节点显示为“背景图容器”。
- `button` 或 `role=button` 显示按钮文字；无文字时显示“按钮”。
- `a` 显示链接文字；无文字时显示“链接”。
- `input`、`textarea` 显示为“输入框”。
- `section`、`main`、`header`、`footer` 使用“区域 / 主体 / 页眉 / 页脚”等自然名称。
- 普通 `div`、`span` 只有在缺少更明确语义时显示“容器”或“内容”。

DOM 标签、选择器路径、`nth-of-type` 等技术信息不作为默认主文案。后续若需要调试，可仅在开发模式或悬浮提示中展示。

## 不做

- 不做扁平化视觉候选列表。
- 不按视觉重要性重排节点。
- 不把 CSS 背景图当成普通 `<img>` 处理。
- 不新增完整全局图层面板。
- 不在属性栏放置图层列表按钮。
- 不让右键菜单自动展开或改写顶部图层按钮状态。

## 实施摘要

- 已新增统一树形图层菜单组件，右键菜单和预览区顶部图层按钮共用同一套展示、命名、悬停和选择交互。
- 右键触发时，iframe 上报当前选中元素子树，菜单只展示该节点及其子级。
- 预览区单页面模式顶部已新增图层按钮，位置在页面下拉选择左侧；点击后采集当前页面 DOM 树并打开同一树形菜单。
- 属性栏不再放置图层列表按钮，只接收图层菜单选择后的当前节点并展示属性。
- 菜单保留 DOM 父子关系和顺序，只对节点名称做有限可读性优化。
- 点击预览页面外层空白区域时，会清空当前选中对象、关闭图层菜单并让属性面板回到未选中状态。
- 图层菜单滚动区域已改用项目内细滚动条样式。

## 验收标准

- 右键图层菜单和预览区顶部图层按钮菜单来自同一个组件。
- 菜单以树形缩进展示真实父子关系。
- 右键图片、按钮、弹窗或容器时，只展示当前选中元素及其子级。
- 节点顺序与 DOM 顺序一致。
- 节点名称比 `div:nth-of-type(...)` 更容易理解。
- 选择菜单节点后，预览选中框和属性栏展示同步到该节点。
- 两个入口对同一节点显示相同名称。
- 点击预览页面外层空白区域后，当前选中框和属性面板选中态应清空。
- 图层菜单滚动条应保持细窄、低干扰样式。
