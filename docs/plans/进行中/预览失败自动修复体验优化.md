# 预览失败自动修复体验优化

## 背景

当前预览运行失败后，系统会自动把修复指令发送给 AI，但这条指令在对话区表现为用户消息。用户会看到一条自己没有输入过的技术错误长文本，容易误解为系统伪造了用户发言，也违背“技术错误不直接暴露给用户”的既有治理目标。

## 目标

- 自动修复仍然自动执行，不增加用户确认成本。
- 对话区把自动修复展示为系统发起任务，而不是用户气泡。
- 完整错误和修复要求继续传给 AI，但技术细节默认收起。
- 自动修复不影响用户消息历史、输入框语义和首条会话标题。

## 范围

- 涉及创作端编辑页的预览错误触发逻辑。
- 涉及 AI 对话消息模型、发送入口和消息渲染。
- 涉及 AI 对话与配置预览相关项目文档更新。
- 不调整预览运行时错误检测、`@preview/sdk` 策略或 Agent 后端协议。

## 方案

- 将自动发送参数从纯字符串升级为结构化系统任务触发。
- `useChatStream` 为用户发送和系统自动修复分别记录消息：用户发送继续追加用户气泡，自动修复追加系统任务卡片。
- 系统任务卡片展示轻量文案、状态与可展开技术详情；真实发送给 AI 的 hidden prompt 不直接展示。
- 自动修复完成或失败后，更新同一张系统任务卡片的状态。
