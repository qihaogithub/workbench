# 创作端画布自动排版功能方案

> 创建日期：2026-06-25
> 状态：已完成
> 范围：创作端预览区画布工作台的页面自动整理能力

## 背景

创作端画布工作台已经支持多页面总览、拖拽、缩放、适应屏幕和布局持久化。页面较多或经过多次手动拖拽后，画布容易变得松散，用户需要一个一键整理入口，把页面重新排成规整、可扫描的布局。

相关长期说明见 [画布功能体验与维护优化方案](./画布功能体验与维护优化方案.md) 与 [预览系统需求](../../项目文档/创作端/04-配置与预览/预览系统_需求文档.md)。

## 目标

- 在创作端画布工具栏增加“自动排版”入口。
- 点击后基于当前摆放状态轻量整理页面，保留大致分组、相对顺序和每个页面的真实尺寸。
- 自动排版后视口适应全部页面，方便立即查看整理结果。
- 复用现有 `canvasState` 保存链路，不新增后端接口。

## 范围

- 修改共享画布组件和工具栏。
- 增加可测试的布局纯函数。
- 不改变单页面预览、配置面板、截图生成和页面文件数据结构。

## 方案

自动排版采用保守整理策略：

- 优先读取当前 `canvasState.pages` 中的页面位置和尺寸。
- 根据当前纵向位置识别现有行，不重新把页面分配到新的紧凑网格。
- 行内保留当前从左到右顺序，只做顶部对齐、8px 网格吸附和最小间距修正。
- 下方行会尽量对齐上一行相近的列位置，但不会跨行重排页面。
- 排版完成后调用现有“适应屏幕”逻辑，将整理后的包围盒居中展示。

## 实施摘要

- 自动排版按当前位置识别行和列，保留页面已有大致空间关系。
- 页面宽高优先来自当前画布布局，不强行重置为默认预览尺寸。
- 排版后复用现有 `canvasState` 更新与 debounce 保存链路。
- 排版后自动计算包围盒并适应全部页面视图。
