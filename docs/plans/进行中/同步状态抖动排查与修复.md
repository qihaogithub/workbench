# 同步状态抖动排查与修复

## 背景

创作端项目编辑页右上角同步状态在“连接中”和“离线待同步”之间反复切换。检测脚本 `scripts/development/detect-sync-status-flap.mjs` 会采样页面可见同步状态，并在连接中与离线待同步之间出现多次双向切换时失败。

## 目标

- 定位同步状态抖动的前端状态来源。
- 修复 WebSocket 自动重连期间的可见状态反复切换。
- 运行 `pnpm test:sync-status-flap` 直到通过。

## 范围

- `packages/author-site/src/hooks/useCollabDocument.ts`
- `docs/项目文档/创作端/03-项目管理/技术/11_实时保存与协同编辑.md`

## 方案

`useCollabDocument` 继续监听 y-websocket provider 状态，但将 `disconnected` 视为可自动恢复的短暂断连，不立即把 UI 状态置为 `offline`。只有断连持续超过阈值且仍未恢复时，才展示“离线待同步”。这样保留真实离线提示，同时避免多个协同房间在短暂重连期间把页面汇总状态拉成“连接中/离线待同步”反复横跳。

## 风险与待确认事项

- 离线提示会延迟出现；若协同服务真实不可用，页面会先显示连接中，超过阈值后才显示离线待同步。
- 当前剩余风险为离线提示存在 5 秒延迟；这是为了避免短暂 WebSocket 重连抖动污染项目级汇总状态。
