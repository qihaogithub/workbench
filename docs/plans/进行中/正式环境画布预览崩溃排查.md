# 正式环境画布预览崩溃排查

## 背景

正式环境项目编辑页进入预览区画布后，页面出现客户端异常并显示 `Application error: a client-side exception has occurred`。浏览器控制台报 React minified error `#185`，指向生产包中的 React 更新循环。

## 目标

- 定位进入画布时触发无限更新的具体组件或状态同步链路。
- 修复正式环境画布打开即崩溃的问题。
- 用本地验证和必要的线上验证确认修复有效。

## 范围

- 创作端项目编辑页画布入口。
- 画布状态、视口自适应、可见页面观测、截图预览和协同布局同步。
- 不处理与本次崩溃无关的历史容器、截图任务历史错误或本地 `data/` 脏改。

## 方案

1. 读取编辑页和画布组件中进入 canvas 模式后的状态更新路径。
2. 复现或通过测试定位 React `#185` 对应的重复更新触发点。
3. 对状态更新增加稳定性保护，避免值未变化时重复 setState。
4. 补充针对进入画布的回归测试。
5. 运行最小必要验证，修复后按部署脚本上线。
