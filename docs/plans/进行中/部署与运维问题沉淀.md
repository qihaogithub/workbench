## 正式环境迁移到 macOS 主机 + 密码登录 + 本机 compose 部署（2026-07-17）

- **现象/背景**：正式环境从 `root@10.130.33.131`（SSH 私钥）迁移到 `jojo@10.131.75.39`（密码 `123456`）。`10.131.75.39` 是一台 **Mac**，代码已在其本机 `/Users/jojo/Documents/workbench`。最终采用的部署方式是：SSH 登录该机后**在本机用 `docker compose` 构建并启动**（方式 A），不再走「开发机 rsync 推送」（方式 B，保留但非主路径）。
- **改动范围**：
  - 认证：`scripts/deploy.sh`、`deploy-author-with-data.sh`、`sync-production-data-to-local.sh` 默认服务器改为 `jojo@10.131.75.39`，新增 `SSH_PASSWORD`（默认 `123456`）；三脚本统一 `SSH_CMD`/`RSYNC_RSH` 封装 ssh/rsync，设了密码走 `sshpass`、置空回退 `-i ${SSH_KEY}`。相关变量均可环境变量覆盖。
  - 路径适配 Mac：`.env.docker` 的 `APP_DATA_DIR` → `/Users/jojo/Documents/workbench/data`（Docker Desktop/OrbStack 默认只共享 `/Users`，不共享 `/opt`）；三脚本 `REMOTE_DIR` → `/Users/jojo/Documents/workbench`，`deploy-author-with-data.sh` 的备份/暂存目录 → `/Users/jojo/workbench-data-{backups,staging}`。
  - 截图服务默认开启：`docker-compose.yml` 移除 `screenshot-service` 的 `profiles: [screenshot]`，并移除强制 `platform: linux/amd64`（改为跟随宿主机原生平台，Apple 芯片下 arm64，避免 amd64 模拟拖慢/崩溃；Dockerfile 用 `apt-get install chromium` + `FROM --platform=$TARGETPLATFORM`，原生可装）。`deploy.sh` 默认 `DEPLOY_SERVICES` 含 `screenshot-service`、`INCLUDE_SCREENSHOT_SERVICE=true`。
  - 浏览器 URL：`.env.docker` 的 `NEXT_PUBLIC_*`、`CORS_ORIGINS`、`FIGMA_OAUTH_REDIRECT_URI` → `10.131.75.39`（NEXT_PUBLIC_* 为构建期变量，必须同步）。

## 目标机 10.131.75.39（OrbStack）本机部署联调（2026-07-17）

- **现象**：在该 Mac 上执行 `docker compose --env-file .env.docker up -d --build` 报 `unknown flag: --env-file` 并打印 base `docker` 用法。
- **根因**（由 `docker version` / `docker compose version` / `docker-compose version` 三条输出确认）：
  1. `docker compose`（v2 插件）在该机 **不存在**：`docker: unknown command: docker compose`。只有 OrbStack 自带的 **`docker-compose`（连字符，v5.1.2）** 可用。所以 `docker` 把 `--env-file` 当成自身未知参数。
  2. `docker` CLI 连不上 daemon：`failed to connect ... /Users/jojo/.docker/run/docker.sock`（Docker Desktop 的 socket 路径），说明 **context 指向了未运行的 Desktop，而非 OrbStack**；需 `orb start` 起引擎并 `docker context use orbstack`。
- **结论/正确姿势**（已写入部署指南「方式 A」）：
  1. `orb start`（或 `open -a OrbStack`）起引擎，`docker context use orbstack`，`docker info` 能连通再继续。
  2. `cp .env.docker .env` 让 compose 自动读变量，规避 `--env-file`。
  3. 用 `docker-compose up -d --build`（连字符），**不要**用 `docker compose`（该机无 v2 插件）。
- **待验证**：上述步骤在该机实跑通过、四服务（含截图）起来并健康。
- **备注**：`deploy.sh`/`deploy-author-with-data.sh`/`sync-production-data-to-local.sh` 内部仍用 `docker compose --env-file`，在该机同样不可用——方式 B 若要在此机跑需把远端命令改成 `docker-compose` + `.env`；当前主路径是方式 A，暂不改脚本。
