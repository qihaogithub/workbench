## 正式环境服务器迁移 + 改用密码登录（2026-07-17）

- **现象/背景**：正式环境从 `root@10.130.33.131`（SSH 私钥 `~/.ssh/figma-mirror-deploy-key`）迁移到 `jojo@10.131.75.39`（密码 `123456`）。
- **改动范围**：
  - `scripts/deploy.sh`、`scripts/deploy-author-with-data.sh`、`scripts/sync-production-data-to-local.sh`：默认服务器改为 `jojo@10.131.75.39`；新增 `SSH_PASSWORD`（默认 `123456`）。三个脚本统一用 `SSH_CMD`（数组）/`RSYNC_RSH`（字符串）封装 ssh 与 rsync：设置了 `SSH_PASSWORD` 走 `sshpass`，置空则回退 `-i ${SSH_KEY}` 私钥登录。`SERVER_IP/SERVER_USER/SERVER_PORT/REMOTE_DIR/SSH_PASSWORD/SSH_KEY` 均可用同名环境变量覆盖。
  - `.env.docker`：`NEXT_PUBLIC_*`、`CORS_ORIGINS`、`FIGMA_OAUTH_REDIRECT_URI` 中的 `10.130.33.131` → `10.131.75.39`（NEXT_PUBLIC_* 为构建期变量，浏览器端要访问新机必须同步）。
  - `docs/用户指南/正式环境部署指南.md`：服务器、认证方式、访问地址、排障命令同步更新。
  - `deploy-fast.sh` 未改：它只透传环境变量调用 `deploy.sh`，自动继承新默认值。
- **前置依赖**：本机需安装 `sshpass`（macOS `brew install sshpass`；已确认本机 `/opt/homebrew/bin/sshpass` 存在）。脚本会在缺失时报错并提示。
- **验证状态**：
  - 已完成：三个脚本 `bash -n` 语法通过；无旧 IP/旧私钥/内联 `ssh -i` 残留。
  - 未完成：`10.131.75.39` 是内网地址，开发机不在该内网、`ssh` 端口 22 超时，无法现场联调。真实部署需在能连通该内网的机器上执行 `scripts/deploy-fast.sh ...` 验证连通、构建、健康检查全链路。
- **后续/风险**：
  - `jojo` 为非 root 账号，需确认它对 `REMOTE_DIR=/opt/workbench`、`APP_DATA_DIR=/opt/opencode-workbench/data`、`/etc/docker/daemon.json` 读写及 `docker` 命令的权限；若无权限，需服务器侧授予 docker 组/目录属主，或用 `REMOTE_DIR=` 环境变量指向 jojo 家目录下路径。
  - 密码 `123456` 以明文写入脚本默认值与 `.env`/文档，仅适用于内网低敏环境；后续如提升安全性，应改回私钥登录或将密码移出仓库（`.env` 已被 git 忽略，脚本默认值仍在版本库中）。
  - 项目文档中仍有以旧 IP `10.130.33.131` 作示例的段落（`docs/项目文档/创作端/06-基础设施/技术/03_Docker部署方案.md`、`使用端/03-部署与嵌入/技术/01_部署与CORS配置.md`、`11-诊断与日志/技术/02_Codex查询CLI与导出包.md`、`OPS/CLI/README.md`），属示例值非强制契约，按需再统一清扫。
