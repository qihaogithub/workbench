## 当前状态

本文档专门沉淀创作端项目编辑页预览区问题。当前保留两类仍有接手价值的问题：“预览画布页面框尺寸不一致”和“闯关活动项目画布批量预览白屏/空块”。

## 闯关活动项目画布批量预览白屏/空块

现象：`proj_1782839405716_tqjl1f` 单页预览可进入，但切换画布模式后页面框长时间显示白/灰色空块，且触发“检测到预览异常，正在自动修复”。

当前结论：直接原因是 `prototype-01` 当前项目工作区代码被重复拼接，导致画布批量预览调用 `/api/compile` 时返回 422。错误为 `DUPLICATE_TOP_LEVEL_DECLARATION` 和 `MULTIPLE_DEFAULT_EXPORTS`，指向 `prototype-01` 的重复 `interface DemoProps` 与 `export default function Demo`。单页模式可能显示旧成功结果或进入自动修复状态，画布批量编译会稳定暴露该坏页。

修复摘要：已清理 `data/projects/proj_1782839405716_tqjl1f/workspace/demos/prototype-01/index.tsx` 中第二段重复模块；浏览器自动修复流程也把 active live workspace 的 `prototype-01` 恢复为单份模块。

验证状态：`node packages/project-cli/bin/ow.mjs project validate-runtime proj_1782839405716_tqjl1f --json` 通过，27 个页面 `issues: []`；Playwright 复现曾捕获画布 `/api/compile` 422 并确认根因，修复后不再继续用浏览器触发自动修复写入。前序 25 秒浏览器采样显示画布冷启动期间大量 iframe 处于编译/加载状态，且当前 fit 后缩放约 7%，页面内容在画布上非常小，后续若用户仍认为“白屏”，优先区分“编译硬错误”与“冷启动/缩放/截图兜底体验问题”。

后续事项：如果需要继续优化该项目画布体验，优先检查批量预览是否应先展示已有截图缓存、限制冷启动 iframe 数量，或在 27 页项目首次进入画布时避免全量 fit 到过小缩放。

## 预览画布页面框尺寸不一致

当前结论：最终根因是 `PreviewPanel` 在画布工作台中读取了被 `CanvasViewport` CSS transform 缩放后的容器尺寸，并把该屏幕尺寸再次用于 iframe 内部缩放。当前 active workspace 中“平板”页面 schema、页面布局和页面代码已经对齐 `1133x749`；截图缓存 meta 与 PNG 尺寸也都是 `1133x749`，因此前序“历史 `.canvas-layout.json` 缺少 `previewSize` 兼容校验”是真实相邻缺陷，但不是当前仍复现的最终根因。

已落地行为：`CanvasPageLayout` 增加 `sizeMode: "preview" | "custom"` 与 `previewSizeKey`；画布布局合成前按当前页面 `previewSize` 归一化历史布局，历史坏尺寸只保留 `x/y/zIndex`；用户手动 resize 写入 `custom`，右键“重置大小”恢复 `preview`；内容高度上报不再使用过期 `layout.width` 推导高度；canvas-layout API、workspace 发布读取和 agent-service `arrangeCanvasPages` 保留新增尺寸语义字段；`PreviewPanel` 初始容器量测优先使用未受 transform 影响的 `clientWidth/clientHeight`，只在无布局尺寸时回退 `getBoundingClientRect()`。

待办：用户在真实浏览器页面确认 bundle 刷新后，画布 zoom 小于 1 时 iframe 内容是否不再二次缩小；若仍异常，优先检查 `PreviewPanel` 容器量测值是否等于布局尺寸 `1133x749`，而不是画布 zoom 后的屏幕尺寸。

验证状态：`corepack pnpm --filter @opencode-workbench/author-site test -- --testPathPatterns=preview-panel-sleep --runInBand` 通过；`corepack pnpm --filter @opencode-workbench/author-site test -- --testPathPatterns=canvas-layout --runInBand` 通过。`corepack pnpm --filter @opencode-workbench/author-site typecheck` 当前失败在既有 `src/app/api/sessions/[sessionId]/workspace-flush/route.test.ts` 的 `TestResponse` 重复标识符，与本次画布尺寸链路无直接关系。前序 `corepack pnpm check:agent` 通过。

风险：当前修复依赖 `clientWidth/clientHeight` 在画布 transform 下返回布局尺寸；若未来容器改成非 HTMLElement 或不可布局节点，需要保留 ResizeObserver/contentRect 作为主路径。
