# AGENTS.md — docs/plans/进行中

本目录只保留当前仍需要行动、复核或跟踪的问题。它不是长期项目知识库，也不是历史流水账目录。

## 写作边界

- 只记录当前仍成立的信息：未解决问题、待验证事项、关键根因、当前方案、验证结论和下一步动作。
- 不记录逐次命令流水、完整日志输出、重复背景、已失效判断或已经完成且没有复用价值的过程。
- 新增或更新本目录文档时，不需要同步 `docs/项目文档/INDEX.md`。
- 如果任务改变了产品行为、接口契约、架构边界或模块职责，再另行更新 `docs/项目文档/` 下对应模块文档。

## 文档形态

优先维护少量按功能模块划分的固定文档，文件名必须体现模块边界，而不是单个 bug、单次排查或一次任务名称。

单个问题文档只在跨多个模块、需要独立验收，或用户明确要求时临时保留；文件名仍必须先体现功能模块边界`。任务完成后应压缩归档到 `docs/plans/已完成/`，或把可复用结论合并回对应模块文档后删除临时文档。

推荐结构：

- `## 当前状态`：一句话说明状态，例如“已实现，待真实浏览器复核”。
- `## 当前结论`：保留当前根因、方案和关键边界。
- `## 待办`：只列仍未完成的下一步，不保留已完成清单。
- `## 验证状态`：列当前仍有用的验证结论和阻塞。
- `## 风险`：只保留当前仍可能影响后续工作的风险。

## 更新规则

- 每次更新前先清理同一文档：合并重复条目，删除已失效描述，压缩已完成事项。
- 已完成且没有复用价值的记录应直接删除。
- 已完成但值得追溯的问题，应移动到 `docs/plans/已完成/`，并压缩为归档摘要。
- 未实现的远期想法应移动到 `docs/plans/远期规划/`，不要长期停留在 `进行中`。
- 明确过期且无保留价值的文档，清理前先确认没有其他现存文档引用。
