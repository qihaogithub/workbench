# 嵌入页面配置面板修改后预览不更新 — 修复方案

> **状态**: 进行中
> **创建日期**: 2026-05-24
> **关联模块**: 创作端 — 嵌入 API（07-嵌入API）

---

## 一、问题描述

嵌入页面（`/embed/{demoId}`）的配置面板中修改 Schema 定义的配置项后，预览区的 iframe 不会实时更新，组件始终以初始默认值渲染。

---

## 二、根因分析

### 2.1 正常流程对比（编辑页 vs 嵌入页）

**编辑页（`/demo/[id]/edit`）** — 配置修改后预览正常更新：

```
ConfigForm (右侧面板)
    │ onChange(newData)
    ▼
DemoEditPage.handleConfigChange()
    │ setConfigDataMap(...)
    ▼
PreviewPanel (configData prop 变化)
    │ sendUpdateConfig(configData)
    ▼
iframeRef.current.contentWindow.postMessage({ type: "UPDATE_CONFIG", configData })
    │
    ▼
iframe 内部 (iframe-template.ts)    ← 收到 postMessage
    │ currentConfig = newConfigData
    ▼
renderComponent()                   ← 用新 props 重新渲染组件 ✅
```

**嵌入页（`/embed/{demoId}`）** — 配置修改后预览不更新：

```
ConfigForm (嵌入页面上某处)
    │ onChange(newData)
    ▼
handleConfigChange()
    │ ???
    ▼
iframe.contentWindow.postMessage(...)    ← **这里断了！**
    │
    ╳  没有 iframe ref，无法获取 contentWindow
    ╳  没有 sendUpdateConfig 等价逻辑
```

### 2.2 代码层面根因：iframe 缺少 ref，无 postMessage 发送端

对比两个页面的 iframe 渲染：

[embed/[demoId]/page.tsx#L58-L63](file:///e:/重要文件/Programming/1_Work/opencode-workbench/packages/author-site/src/app/embed/[demoId]/page.tsx#L58-L63) — **嵌入页的 iframe**：

```tsx
{/* Server Component，没有 ref */}
<iframe
  src={iframeUrl}
  sandbox="allow-scripts allow-same-origin"
  className="w-full"
  style={{ minHeight: '400px' }}
/>
```

[PreviewPanel.tsx#L174-L491](file:///e:/重要文件/Programming/1_Work/opencode-workbench/packages/author-site/components/demo/PreviewPanel.tsx#L174-L491) — **编辑页的 iframe**：

```tsx
{/* Client Component，有 ref=iframeRef */}
<iframe
  ref={iframeRef}                    // ← 关键：有 ref
  sandbox="allow-scripts allow-same-origin"
  src={iframeSrcUrl}
  style={iframeStyle}
/>
```

**三个关键差异导致通信断裂**：

| 维度 | 编辑页 PreviewPanel | 嵌入页 page.tsx | 影响 |
|:-----|:---------------------|:----------------|:-----|
| 组件类型 | Client Component (`"use client"`) | Server Component | 无法使用 `useRef`、`useEffect` |
| iframe ref | ✅ `ref={iframeRef}` | ❌ 无 ref | 无法通过 `iframeRef.current.contentWindow` 获取通信目标 |
| postMessage 发送 | ✅ `sendUpdateConfig()` | ❌ 不存在 | `UPDATE_CONFIG` 消息从未被发出 |

### 2.3 iframe 接收端已就绪

虽然发送端缺失，但接收端（iframe 内部）的处理逻辑是完整的。查看 [iframe-template.ts#L174-L181](file:///e:/重要文件/Programming/1_Work/opencode-workbench/packages/author-site/src/lib/iframe-template.ts#L174-L181)：

```javascript
if (type === 'UPDATE_CONFIG') {
  currentConfig = newConfigData || {};
  window.__DEMO_PROPS__ = currentConfig;
  if (currentComponent) {
    renderComponent();  // React.createElement(currentComponent, currentConfig)
  }
}
```

以及 `/embed/{demoId}/iframe/route.ts` 中生成的 iframe HTML 已包含完整的 postMessage 监听器。

**结论**：问题只在发送端——嵌入页面的 iframe 没有 ref，配置面板无法向 iframe 发送 `UPDATE_CONFIG` 消息。

## 三、修复方案

### 3.1 方案概述

核心思路：参考编辑页 `PreviewPanel` + `ConfigForm` 的协作模式，在嵌入页创建一个 Client Component，同时持有 iframe ref 和 ConfigForm，通过 postMessage 桥接二者。

```
┌─ 嵌入页 (Server Component) ──────────────────────────────────────────┐
│  服务端读取 schema 和 configData                                      │
│                                                                       │
│  ┌─ EmbedConfigPanel (Client Component) ──────────────────────────┐  │
│  │                                                                  │  │
│  │  ┌─ 预览 iframe ────────────┐  ┌─ 配置面板 ──────────────────┐  │  │
│  │  │                           │  │                              │  │  │
│  │  │  <iframe                  │  │  ConfigForm                  │  │  │
│  │  │    ref={iframeRef}   ←─────── onChange ───────────┐        │  │  │
│  │  │    src="/embed/           │  │                      │        │  │  │
│  │  │      {demoId}/iframe"     │  │  handleChange() {    │        │  │  │
│  │  │  />                       │  │    iframeRef.current  │       │  │  │
│  │  │                           │  │      .contentWindow  │       │  │  │
│  │  │                           │  │      .postMessage(    │       │  │  │
│  │  │                           │  │        UPDATE_CONFIG, │       │  │  │
│  │  │                           │  │        configData      │       │  │  │
│  │  │                           │  │      )                │       │  │  │
│  │  │                           │  │  }                    │       │  │  │
│  │  │                           │  │                      │       │  │  │
│  │  └───────────────────────────┘  └──────────────────────┘       │  │  │
│  │                                                                  │  │
│  └──────────────────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────────────────┘
```

### 3.2 具体实现

#### 3.2.1 新增 Client Component：`EmbedConfigPanel`

**文件**：`packages/author-site/src/app/embed/[demoId]/EmbedConfigPanel.tsx`（**新增**）

```tsx
"use client";

import { useRef, useState, useCallback } from "react";
import { ConfigForm, ConfigScopeWrapper } from "../../../../components/demo";

interface EmbedConfigPanelProps {
  demoId: string;
  iframeUrl: string;
  schema: string;
  projectConfigSchema?: string;
  initialConfigData: Record<string, unknown>;
}

export function EmbedConfigPanel({
  demoId,
  iframeUrl,
  schema,
  projectConfigSchema,
  initialConfigData,
}: EmbedConfigPanelProps) {
  // ★ 核心：通过 ref 获取 iframe 引用
  const iframeRef = useRef<HTMLIFrameElement>(null);

  const [configData, setConfigData] = useState<Record<string, unknown>>(initialConfigData);

  // ★ 核心：配置变更 → postMessage 发送 UPDATE_CONFIG
  const handleConfigChange = useCallback((data: Record<string, unknown>) => {
    setConfigData(data);

    const iframe = iframeRef.current;
    if (!iframe?.contentWindow) return;

    iframe.contentWindow.postMessage(
      { type: "UPDATE_CONFIG", configData: data },
      "*"
    );
  }, []);

  return (
    <div className="flex gap-6">
      {/* 预览区 */}
      <div className="flex-1">
        <iframe
          ref={iframeRef}        // ★ 关键：加上 ref
          src={iframeUrl}
          sandbox="allow-scripts allow-same-origin"
          className="w-full border rounded-lg"
          style={{ minHeight: '400px' }}
        />
      </div>

      {/* 配置面板 */}
      <div className="w-72 shrink-0">
        {/* 项目级配置 */}
        {projectConfigSchema && (
          <ConfigScopeWrapper scope="project">
            <ConfigForm
              key={`project-${projectConfigSchema}`}
              schema={projectConfigSchema}
              onChange={handleConfigChange}
              initialData={configData}
            />
          </ConfigScopeWrapper>
        )}

        {/* 页面级配置 */}
        <ConfigScopeWrapper scope="page">
          <ConfigForm
            key={schema}
            schema={schema}
            onChange={handleConfigChange}
            initialData={configData}
          />
        </ConfigScopeWrapper>
      </div>
    </div>
  );
}
```

#### 3.2.2 改造嵌入页面

**文件**：`packages/author-site/src/app/embed/[demoId]/page.tsx`（修改）

核心改动：
1. 从服务端读取 schema 文件
2. 计算初始 configData
3. 引入 `EmbedConfigPanel` 客户端组件替代静态 `<iframe>`

```tsx
import fs from "fs";
import path from "path";
import { getProjectPath } from "@/lib/fs-utils";
import { mergeConfigToProps } from "@/lib/runtime-props";
import { EmbedConfigPanel } from "./EmbedConfigPanel";
// ... 保留原有 imports

export default function EmbedPage({ params }: EmbedPageProps) {
  const { demoId } = params;
  if (!projectExists(demoId)) notFound();

  const iframeUrl = `/embed/${demoId}/iframe`;
  const embedCode = `...`; // 嵌入代码片段保持不变

  // ★ 服务端读取 schema
  const projectPath = getProjectPath(demoId);
  const workspacePath = path.join(projectPath, "workspace");
  const schemaPath = path.join(workspacePath, "config.schema.json");
  const projectSchemaPath = path.join(workspacePath, "project.config.schema.json");

  const schema = fs.existsSync(schemaPath)
    ? fs.readFileSync(schemaPath, "utf-8")
    : "{}";

  const projectConfigSchema = fs.existsSync(projectSchemaPath)
    ? fs.readFileSync(projectSchemaPath, "utf-8")
    : undefined;

  // ★ 计算初始配置数据
  const configData = getDefaultValuesFromSchema(schema);
  const mergedConfigData = projectConfigSchema
    ? mergeConfigToProps(projectConfigSchema, schema)
    : configData;

  return (
    <div className="min-h-screen bg-background">
      <div className="max-w-5xl mx-auto px-6 py-12">
        <h1>嵌入 Demo</h1>
        <p>将以下 iframe 代码复制到你的页面中即可嵌入此 Demo。</p>

        {/* 嵌入代码区域 — 保持不变 */}
        <div className="space-y-6">
          <div>
            <pre className="...">{embedCode}</pre>
            {/* 复制按钮 */}
          </div>

          {/* ★ 预览 + 配置面板 — 用 EmbedConfigPanel 替换原来的静态 iframe */}
          <EmbedConfigPanel
            demoId={demoId}
            iframeUrl={iframeUrl}
            schema={schema}
            projectConfigSchema={projectConfigSchema}
            initialConfigData={mergedConfigData}
          />
        </div>
      </div>
    </div>
  );
}
```

#### 3.2.3 辅助：提取 Schema 默认值

从 [iframe/route.ts#L39-L55](file:///e:/重要文件/Programming/1_Work/opencode-workbench/packages/author-site/src/app/embed/[demoId]/iframe/route.ts#L39-L55) 中已有现成的提取逻辑，可抽取为公共函数复用：

```typescript
// 可放到 src/lib/fs-utils.ts 或新建 utils
function getDefaultValuesFromSchema(schema: Record<string, unknown>): Record<string, unknown> {
  const configData: Record<string, unknown> = {};
  if (schema?.properties) {
    for (const [key, prop] of Object.entries(schema.properties)) {
      const p = prop as Record<string, unknown>;
      if (p.default !== undefined) {
        configData[key] = p.default;
      }
    }
  }
  return configData;
}
```

## 五、验收标准

| 验收项 | 标准 |
|:-------|:-----|
| 配置面板可见 | 嵌入页面展示完整的配置面板，字段与 Schema 一致 |
| 配置修改→预览更新 | 修改任意配置项值后，预览区 iframe 内组件**实时更新** |
| 项目级 Schema | 存在 `project.config.schema.json` 时，全局配置分区可见可修改 |
| 嵌入代码功能正常 | 嵌入代码区域和复制按钮功能不受影响 |
| 不影响编辑页 | `/demo/[id]/edit` 编辑页功能不受影响 |
| 无控制台错误 | 修改配置时无 JavaScript 错误 |
