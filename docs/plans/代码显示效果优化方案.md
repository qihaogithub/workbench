# 代码显示效果优化方案

## 1. 问题描述

### 1.1 当前实现

在需求文档中，代码编辑区采用统一的分隔符格式展示：

```
=== DEMO CODE ===
import React from 'react';

export default function Demo({ title }) {
  return <h1>{title}</h1>;
}
=== DEMO SCHEMA ===
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "Demo 配置",
  "type": "object"
}
=== END ===
```

### 1.2 存在问题

| 问题 | 影响 |
| :--- | :--- |
| 无语法高亮 | 使用 `` ``` `` 无语言标识，Markdown 渲染器无法识别代码类型 |
| 可读性差 | 代码与配置混在一起，视觉上没有清晰分隔 |
| 不够美观 | 缺乏现代文档应有的专业展示效果 |

---

## 2. 优化方案

### 2.1 方案概述

采用**完全独立展示**的方案：
1. **代码部分**：使用 ` ```tsx ` 或 ` ```jsx ` 代码块独立展示
2. **Schema 部分**：使用 ` ```json ` 代码块独立展示
3. **分开标题**：分别标注"代码"和"Schema 配置"

### 2.2 标准展示格式

**代码：**

```tsx
import React from 'react';

export default function Demo({ title }: { title: string }) {
  return <h1>{title}</h1>;
}
```

**Schema 配置：**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "Demo 配置",
  "type": "object",
  "properties": {
    "title": {
      "type": "string",
      "title": "标题"
    }
  },
  "required": ["title"]
}
```

---

## 3. 格式规范

### 3.1 代码块类型定义

| 代码块标识 | 用途 | 说明 |
| :--- | :--- | :--- |
| `tsx` | React TypeScript 组件 | 用于展示 React 组件代码 |
| `jsx` | React JavaScript 组件 | 用于展示 React 组件代码（无 TypeScript） |
| `json` | JSON Schema 配置 | 用于展示 JSON Schema |

### 3.2 标题标注

| 标题 | 说明 | 示例 |
| :--- | :--- | :--- |
| `**代码：**` | 标注下方为代码部分 | `**代码：**` |
| `**Schema 配置：**` | 标注下方为 Schema 部分 | `**Schema 配置：**` |

## 4. 文档示例

### 4.1 简单组件

**代码：**

```tsx
import React from 'react';

export default function Banner({ images }: { images: string[] }) {
  return (
    <div className="banner">
      {images.map((src, i) => (
        <img key={i} src={src} alt={`banner-${i}`} />
      ))}
    </div>
  );
}
```

**Schema 配置：**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "Banner 组件",
  "type": "object",
  "properties": {
    "images": {
      "type": "array",
      "title": "图片列表",
      "items": { "type": "string", "format": "uri" }
    }
  },
  "required": ["images"]
}
```

### 4.2 配置型组件

**代码：**

```tsx
import React from 'react';

interface Props {
  variant?: 'default' | 'compact';
  showTitle?: boolean;
}

export default function Header({ variant = 'default', showTitle = true }: Props) {
  return (
    <header className={`header header--${variant}`}>
      {showTitle && <h1>Welcome</h1>}
    </header>
  );
}
```

**Schema 配置：**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "Header 组件",
  "type": "object",
  "properties": {
    "variant": {
      "type": "string",
      "title": "样式变体",
      "enum": ["default", "compact"],
      "default": "default"
    },
    "showTitle": {
      "type": "boolean",
      "title": "显示标题",
      "default": true
    }
  }
}
```

## 5. 迁移策略

### 5.1 文档更新

按以下优先级逐步更新文档：

1. **高优先级**：需求文档、API 文档
2. **中优先级**：使用指南、教程文档
3. **低优先级**：内部说明、临时文档

### 5.2 转换示例

旧格式：

```
=== DEMO CODE ===
...code...
=== DEMO SCHEMA ===
...schema...
=== END ===
```

新格式：

```
**代码：**

```tsx
...code...
```

**Schema 配置：**

```json
...schema...
```
```

## 6. 方案优势

| 优势 | 说明 |
| :--- | :--- |
| **标准兼容** | 使用标准 Markdown 语法，兼容性最好 |
| **语法高亮** | 代码块有正确的语言标识，可获得语法高亮 |
| **视觉清晰** | 代码与 Schema 清晰分隔，易于阅读 |
| **维护性好** | 代码与配置独立，便于单独修改 |
| **无需开发** | 无需额外开发自定义组件或工具 |

---

## 7. 注意事项

1. **保持格式一致**：同一文档中保持标题标注风格统一
2. **语言标识正确**：tsx/jsx/json 等标识要准确
3. **验证渲染**：更新后检查 Markdown 渲染效果
