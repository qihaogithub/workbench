# 代码显示效果优化方案

## 1. 问题描述

### 1.1 当前实现

在需求文档中，代码编辑区采用统一的分隔符格式展示：

```
=== DEMO CODE ===
import React from 'react';

export default function Demo({ title }) {
  return <h1>{title}</h1>;
}
=== DEMO SCHEMA ===
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "Demo 配置",
  "type": "object"
}
=== END ===
```

### 1.2 存在问题

| 问题 | 影响 |
| :--- | :--- |
| 无语法高亮 | 使用 `` ``` `` 无语言标识，Markdown 渲染器无法识别代码类型 |
| 可读性差 | 代码与配置混在一起，视觉上没有清晰分隔 |
| 不够美观 | 缺乏现代文档应有的专业展示效果 |

---

## 2. 优化方案

### 2.1 方案概述

采用**多语言代码块 + 注释分隔**的组合方案：

1. **分别展示**：使用独立的代码块分别展示代码和 Schema
2. **语言标识**：为每个代码块添加正确的语言标识（`tsx`、`json`）
3. **注释嵌入**：在代码中通过注释方式标注对应关系

### 2.2 标准展示格式

**React 组件代码：**

```tsx
import React from 'react';

export default function Demo({ title }: { title: string }) {
  return <h1>{title}</h1>;
}
```

**JSON Schema 配置：**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "Demo 配置",
  "type": "object",
  "properties": {
    "title": {
      "type": "string",
      "title": "标题"
    }
  },
  "required": ["title"]
}
```

### 2.3 需要同时展示两者的场景

当需要同时展示代码与 Schema 的对应关系时，使用注释嵌入方式：

```tsx
import React from 'react';

// === Schema 配置 ===
// {
//   "title": { "type": "string", "title": "标题" }
// }

export default function Demo({ title }: { title: string }) {
  return <h1>{title}</h1>;
}
```

## 3. 格式规范

### 3.1 代码块类型定义

| 代码块标识 | 用途 | 说明 |
| :--- | :--- | :--- |
| `tsx` | React TypeScript 组件 | 用于展示 React 组件代码 |
| `jsx` | React JavaScript 组件 | 用于展示 React 组件代码（无 TypeScript） |
| `json` | JSON Schema 配置 | 用于展示 JSON Schema |
| `jsonc` | 带注释的 JSON | 用于展示带注释的 Schema（兼容 VS Code） |

### 3.2 注释标记规范

| 注释标记 | 说明 | 示例 |
| :--- | :--- | :--- |
| `/* === TITLE === */` | 区块标题 | 标记代码区块的标题 |
| `/* === Schema 配置 === */` | Schema 区域 | 标注下方为 Schema 内容 |
| `// === END ===` | 结束标记 | 可选，用于标识区块结束 |

### 3.3 多行注释格式

```tsx
/*
 * === Schema 配置 ===
 * {
 *   "title": { "type": "string" }
 * }
 */

export default function Demo() {
  return <h1>Hello</h1>;
}
```

## 4. 文档示例

### 4.1 基础展示

#### 示例一：简单组件

**React 组件：**

```tsx
import React from 'react';

export default function Banner({ images }: { images: string[] }) {
  return (
    <div className="banner">
      {images.map((src, i) => (
        <img key={i} src={src} alt={`banner-${i}`} />
      ))}
    </div>
  );
}
```

**Schema 配置：**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "title": "Banner 组件",
  "type": "object",
  "properties": {
    "images": {
      "type": "array",
      "title": "图片列表",
      "items": { "type": "string", "format": "uri" }
    }
  },
  "required": ["images"]
}
```

#### 示例二：配置型组件

**代码（带 Schema 注释）：**

```tsx
import React from 'react';

/*
 * === Schema 配置 ===
 * {
 *   "variant": { "enum": ["default", "compact"], "default": "default" },
 *   "showTitle": { "type": "boolean", "default": true }
 * }
 */

interface Props {
  variant?: 'default' | 'compact';
  showTitle?: boolean;
}

export default function Header({ variant = 'default', showTitle = true }: Props) {
  return (
    <header className={`header header--${variant}`}>
      {showTitle && <h1>Welcome</h1>}
    </header>
  );
}
```

## 5. 迁移策略

### 5.1 文档更新

按以下优先级逐步更新文档：

1. **高优先级**：需求文档、API 文档
2. **中优先级**：使用指南、教程文档
3. **低优先级**：内部说明、临时文档

## 6. 方案优势

| 优势 | 说明 |
| :--- | :--- |
| **标准兼容** | 使用标准 Markdown 语法，兼容性最好 |
| **语法高亮** | 代码块有正确的语言标识，可获得语法高亮 |
| **视觉清晰** | 代码与 Schema 清晰分隔，易于阅读 |
| **维护性好** | 代码与配置独立，便于单独修改 |
| **无需开发** | 无需额外开发自定义组件或工具 |

---

## 7. 注意事项

1. **避免混用格式**：同一文档中保持格式一致性
2. **选择合适场景**：根据展示需求选择独立展示或注释嵌入
3. **保持简洁**：注释不宜过多，只标注关键对应关系
4. **验证渲染**：更新后检查 Markdown 渲染效果
