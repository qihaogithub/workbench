## 一、安全设计

### 1.1 API Key 安全

| 措施 | 说明 |
|:---|:---|
| **加密存储** | API Key 使用 AES-256 加密，密钥由环境变量管理 |
| **不日志输出** | 禁止在日志中打印 API Key |
| **按需解密** | 仅在调用 OpenCode 前解密 |
| **配置访问** | 仅管理员可修改系统配置 |

### 1.2 权限控制

| 层级 | 控制方式 |
|:---|:---|
| **网关层** | 用户认证（已有） |
| **系统配置** | 仅管理员可访问 `/api/admin/*` |
| **会话隔离** | 只能访问自己创建的会话 |
| **模型选择** | 仅允许选择系统配置的模型列表 |

### 1.3 OpenCode 沙箱

| 机制 | 说明 |
|:---|:---|
| **工作空间隔离** | 每个 Session 绑定独立工作目录 |
| **文件访问限制** | 禁止访问工作空间外的文件 |
| **命令执行限制** | 通过 OpenCode 内置权限控制 |

### 2.2 启动流程

```bash
# 1. 配置系统级 LLM（环境变量或配置文件）
export OPENCODE_API_KEY="sk-xxx-xxx"
export OPENCODE_API_BASE="https://api.anthropic.com"
export OPENCODE_MODELS="claude-3-5-sonnet-20240620,claude-opus-4-5"

# 2. 启动 OpenCode Server
opencode serve --port 4096 --hostname 0.0.0.0

# 3. 启动 agent-service
cd packages/agent-service
pnpm start

# 4. Web 前端（已有）
pnpm dev
```

### 2.3 环境变量

```bash
# agent-service
PORT=3201
HOST=0.0.0.0
LOG_LEVEL=info

# OpenCode 系统级配置
OPENCODE_API_KEY=xxx          # 企业大模型 API Key
OPENCODE_API_BASE=xxx        # API 端点
OPENCODE_MODELS=xxx          # 可用模型列表（逗号分隔）
```

## 三、与现有系统集成

### 3.1 向后兼容与双轨支持

| 场景 | 处理方式 | 当前状态 |
|:---|:---|:---|
| 未配置系统 LLM | 提示管理员配置；ACP 后端不受影响 | 待实现 |
| 旧 Session | 自动识别后端类型，ACP Session 继续使用 ACP | ✅ 已支持（agent.ts:85-86） |
| 用户未选择后端 | 默认使用 `opencode`（当前）→ `opencode-http`（Phase 3） | 当前默认 `opencode` |
| 用户未选择模型 | 使用系统默认模型（HTTP）或 ACP 默认模型 | 当前硬编码 `sensenova/deepseek-v4-flash` |
| ACP 生态需求 | 显式指定 `backend: "opencode"` | ✅ 已支持 |

### 3.2 后端切换示例

```typescript
// 使用 HTTP 后端（默认）
await agentClient.sendMessage(sessionId, message, {
  backend: 'opencode-http',  // 或不传，默认就是
});

// 使用 ACP 后端（兼容）
await agentClient.sendMessage(sessionId, message, {
  backend: 'opencode',  // 显式指定 ACP
});
```
