# 底部弹窗创作端预览不显示排查

## 背景

项目 `proj_1779608460372`（底部弹窗）打开创作端项目编辑页后，画布中的“手机”和“平板”预览区域一直停留在加载状态，预览内容不显示。用户提供的截图显示右侧配置面板已能列出两个页面，说明项目基础元数据已加载。

## 目标

- 定位预览不显示的根因。
- 修复导致创作端编辑页预览卡住的问题。
- 运行与改动范围匹配的验证命令。
- 同步更新长期项目文档中与预览机制相关的说明。

## 范围

- 创作端编辑页和画布预览组件。
- 项目 `proj_1779608460372` 的页面源码、Schema、配置数据与预览编译输入。
- 预览编译 API、运行时 shell、iframe 或组件加载链路中与本问题直接相关的逻辑。

不主动修改项目业务数据，除非确认根因是项目数据损坏且需要通过 Project Admin CLI 修复。

## 方案

1. 使用 Project Admin CLI 读取目标项目、页面和配置数据，确认数据结构是否异常。
2. 使用 CodeGraph 定位创作端编辑页、画布预览、动态编译和 iframe 运行时的调用关系。
3. 复现或最小化触发预览 loading 的失败路径，区分是项目源码错误、编译失败、运行时未回调，还是编辑页状态未解除 loading。
4. 按现有模块边界做局部修复，并补充或调整测试。
5. 更新配置与预览模块文档。
