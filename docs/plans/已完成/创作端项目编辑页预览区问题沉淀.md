## P4：预览区编译/运行时错误后 loading 覆盖层永久不消失 ✅

### 现象

- 打开项目后，预览区长期显示 "正在加载页面预览，若页面没有生成预览产物会继续尝试渲染源码"
- 若页面没有生成预览产物（如编译失败），loading 覆盖层持续遮挡，用户看不到编译错误信息
- 同一项目中一个页面正常显示，另一个页面始终卡在加载中
- 复现项目：`proj_1783334663364_30hsrv`（`product-showcase_a3f2` 页面代码有重复声明导致编译失败，`contact-section_k8m2` 正常）

### 影响范围

- 所有编译失败的页面（代码语法错误、重复声明、import 失败等）
- 所有运行时错误导致 iframe 未发送 LOADED 消息的页面
- 单页预览模式和 canvas 多页预览模式均受影响

### 根因

#### 核心缺陷：`showPreviewLoading` 条件未排除错误状态

`packages/demo-ui/src/PreviewPanel.tsx` 第 1265-1272 行：

```typescript
const hasPreviewSource =
  isUrlMode || (typeof code === "string" && code.length > 0);
const showPreviewLoading = hasPreviewSource && (isCompiling || !contentLoaded);
const showPreviewPendingText = showPreviewLoading && !isCompiling;
const showLoadingOverlay = showPreviewLoading && !showPlaceholder;
```

当编译失败后：

- `hasPreviewSource = true`（代码存在）
- `isCompiling = false`（编译已完成）
- `contentLoaded = false`（iframe 从未发送 LOADED）
- `compileError = "..."` （已设置）

代入：`showPreviewLoading = true && (false || true) = true`

结果：loading 覆盖层（z-20）永久显示，遮挡所有错误信息。

#### 渲染层级遮挡关系

```
z-10: placeholder screenshot（有截图时）
z-20: loading overlay（spinner + "正在加载页面预览" 文字）  ← 卡在这里
z-20: empty preview（"等待页面代码加载"，仅无源码时）
z-30: auto-repairing 提示（"正在修复预览"）
iframe: opacity=0（contentLoaded=false 时不可见）
```

编译错误时，`compileError` 在 PreviewPanel 内部被设置但 **没有独立的错误 UI 覆盖层**，错误仅通过 `onError` 回调传递给父组件。loading 覆盖层在 z-20 遮挡了 iframe 区域。

### 修复方案

#### 方案 A（推荐）：`showPreviewLoading` 排除错误状态

- 文件：`packages/demo-ui/src/PreviewPanel.tsx`，第 1267-1268 行
- 改动：`showPreviewLoading` 条件增加 `!compileError && !runtimeError`
- 效果：编译失败或运行时报错时 loading 覆盖层消失

```
修改前：hasPreviewSource && (isCompiling || !contentLoaded)
修改后：hasPreviewSource && (isCompiling || (!contentLoaded && !compileError && !runtimeError))
```

#### 方案 B（可选补充）：为编译错误添加独立的 loading 替代 UI

在 loading 覆盖层消失后，可考虑在预览区显示编译错误摘要（错误消息 + 文件名 + 行号），而非空白。当前错误仅通过 `onError` 回调传递给父组件的 AI 诊断面板，预览区本身没有错误展示。
