# 项目文档代码一致性审计与更新

## 背景

`docs/项目文档/` 是当前项目的长期知识库，需要与实际代码实现保持一致。本次任务根据当前代码结构与关键实现，审计并更新项目文档，降低后续代理或开发者被过期说明误导的风险。

## 目标

- 核对项目文档索引、模块文档与当前 workspace 代码结构是否一致。
- 修正明显过期、缺失或表述不准确的模块说明。
- 补齐必要的技术文档 `covers` 映射和跨文档入口。
- 保持需求文档与技术文档的内容边界清晰。

## 范围

- 包含：`docs/项目文档/` 下的项目总览、创作端、使用端、独立 Agent 服务层、Figma 插件等长期文档。
- 包含：与上述文档直接相关的源码结构核对。
- 不包含：业务功能代码改动、历史计划文档大规模整理、生成物或运行数据清理。

## 方案

1. 读取项目文档总索引与模块索引，建立当前文档覆盖面。
2. 读取 workspace 包、关键入口、API 路由、服务入口和共享类型，建立当前实现概览。
3. 按模块比对文档与代码，优先修正影响理解和后续开发决策的差异。
4. 更新相关模块索引，避免新增孤儿文档。
5. 运行最小可行的文档链接与格式检查，记录验证结果。
