# 找回密码功能方案

> 方案状态：待评审  
> 创建日期：2026-05-28  
> 更新日期：2026-05-29  
> 目标：提供用户密码找回机制，复用现有管理后台实现管理员重置密码

---

## 一、现状分析

### 1.1 当前认证机制

| 项目       | 现状                                            |
| ---------- | ----------------------------------------------- |
| 密码存储   | bcrypt 加密（salt rounds = 10）                 |
| 认证方式   | JWT Token（httpOnly Cookie，7 天过期）          |
| 用户数据库 | SQLite（`data/users.db`）                       |
| 用户表结构 | `id`, `username`, `password_hash`, `created_at` |
| 注册验证   | 用户名 3-20 字符，密码 ≥6 字符                  |
| 安全机制   | 登录失败统一返回"用户名或密码错误"              |

### 1.2 现有管理后台

管理后台已具备完整基础设施（详见 [管理后台架构设计](../../项目文档/创作端/08-管理后台/技术/01_架构设计.md)）：

| 项目            | 现状                                                                 |
| --------------- | -------------------------------------------------------------------- |
| 路由            | `/admin`（概览首页）、`/admin/model-config`（模型配置）              |
| 鉴权机制        | `ADMIN_SECRET` 环境变量 + httpOnly Cookie（2 小时过期）              |
| 鉴权模块        | `lib/admin-auth.ts`（`verifyAdminRequest()`、`verifyAdminSecret()`） |
| Middleware 保护 | `middleware.ts` 拦截 `/admin` 和 `/api/admin/` 路径                  |
| 数据库          | SQLite `data/users.db`（`users` 表 + `system_configs` 表）           |
| 前端布局        | 侧边栏导航 + shadcn/ui 组件                                          |

### 1.3 存在的问题

1. **无找回密码入口**：登录页仅提供"注册"链接，无"忘记密码"选项
2. **无安全验证渠道**：系统未收集用户邮箱/手机号，无法通过传统方式验证身份
3. **无密码重置机制**：数据库缺少重置令牌、过期时间等字段
4. **管理后台无用户管理**：现有管理后台仅支持模型配置，无用户列表和密码重置功能

### 1.4 约束条件

- 系统为内部开发工具，用户量较小
- 无第三方邮件服务/SMS 服务集成
- 需要保持简单、低维护成本的方案
- **复用现有管理后台基础设施**，不引入新的鉴权体系

---

## 二、方案设计

### 2.1 核心思路

**在现有管理后台中扩展用户管理功能**，而非新建独立系统：

1. **管理后台新增用户管理页**（`/admin/users`）：复用已有 Admin Secret 鉴权，管理员可查看用户列表、重置密码
2. **用户自助修改密码**：登录后通过设置弹窗修改密码（免原密码验证）
3. **忘记密码入口**：登录页新增"忘记密码"链接，引导用户联系管理员

## 三、详细设计

### 3.1 功能流程

#### 3.1.1 用户忘记密码流程

```
用户点击"忘记密码"
  ↓
显示提示信息："请联系系统管理员重置密码"
  ↓
管理员登录管理后台
  ↓
选择目标用户 → 设置新密码
  ↓
告知用户新密码
  ↓
用户登录后可自行修改密码
```

#### 3.1.2 管理员操作流程

```
管理员访问 /admin?secret=xxx（首次）或直接访问 /admin（Cookie 有效时）
  ↓
Middleware 验证 Admin Secret（复用现有 verifyAdminSecret()）
  ↓
侧边栏点击"用户管理" → 进入 /admin/users
  ↓
显示用户列表
  ↓
选择用户 → 点击"重置密码"
  ↓
输入新密码 → 确认
  ↓
系统调用 POST /api/admin/users/:userId/reset-password
  ↓
更新密码 + 记录操作日志
```

#### 3.1.3 用户修改密码流程

```
用户登录后点击右上角"设置"
  ↓
弹出设置弹窗
  ↓
选择"修改密码"
  ↓
输入新密码 + 确认密码
  ↓
提交（无需输入原密码）
  ↓
修改成功，提示重新登录
```

### 3.2 管理员认证机制

#### 3.2.1 复用现有 Admin Secret 鉴权

**无需新增鉴权体系**，直接复用管理后台已有的 Admin Secret 机制：

- 环境变量：`ADMIN_SECRET`（已在 `.env` 中配置）
- 鉴权模块：`lib/admin-auth.ts` 的 `verifyAdminRequest()`
- Middleware：`middleware.ts` 已拦截 `/admin` 和 `/api/admin/` 路径
- Cookie：`admin_token`（httpOnly，2 小时过期）

**新增 API 的鉴权方式**：

```typescript
// 在 /api/admin/users/ 相关路由中复用现有验证
import { verifyAdminRequest } from "@/lib/admin-auth";

export async function POST(request: Request) {
  if (!(await verifyAdminRequest(request))) {
    return NextResponse.json(
      {
        success: false,
        error: { code: "UNAUTHORIZED", message: "未授权访问" },
      },
      { status: 401 },
    );
  }
  // ... 业务逻辑
}
```

#### 3.2.2 数据库改造

在现有 `data/users.db` 中新增密码重置日志表（在 `lib/db/schema.ts` 的 `initializeDatabase()` 中追加建表语句）：

```sql
CREATE TABLE IF NOT EXISTS password_reset_logs (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,           -- 被重置密码的用户 ID
  reset_by TEXT NOT NULL,          -- 操作者标识（'admin' 或用户自己的 username）
  reset_method TEXT NOT NULL,      -- 'admin_reset' | 'self_change'
  created_at INTEGER NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE INDEX IF NOT EXISTS idx_reset_logs_user ON password_reset_logs(user_id);
CREATE INDEX IF NOT EXISTS idx_reset_logs_created ON password_reset_logs(created_at);
```

与现有 `users`、`system_configs` 表一起初始化，无需单独迁移脚本。

### 3.3 API 设计

#### 3.3.1 管理员重置用户密码

**接口**: `POST /api/admin/users/[userId]/reset-password`

**权限**: Admin Secret 鉴权（复用 `verifyAdminRequest()`）

**请求体**:

```json
{
  "newPassword": "string" // 新密码（必填）
}
```

**响应**:

```json
{
  "success": true,
  "data": {
    "userId": "string",
    "username": "string",
    "message": "密码重置成功"
  }
}
```

**错误响应**:

```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "未授权访问"
  }
}
```

#### 3.3.2 获取用户列表（管理后台）

**接口**: `GET /api/admin/users`

**权限**: Admin Secret 鉴权（复用 `verifyAdminRequest()`）

**响应**:

```json
{
  "success": true,
  "data": {
    "users": [
      {
        "id": "string",
        "username": "string",
        "createdAt": 1234567890
      }
    ]
  }
}
```

#### 3.3.3 用户修改密码（免原密码）

**接口**: `POST /api/auth/change-password`

**权限**: 需要登录

**请求体**:

```json
{
  "newPassword": "string" // 新密码（必填）
}
```

**响应**:

```json
{
  "success": true,
  "data": {
    "message": "密码修改成功，请重新登录"
  }
}
```

**说明**：

- 此接口设计为**免原密码验证**，适用于内部工具场景
- 用户已登录状态下直接修改
- 修改成功后清除当前 session，要求重新登录

### 3.5 安全策略

#### 3.5.1 密码规则

| 规则     | 要求                                   |
| -------- | -------------------------------------- |
| 最小长度 | 6 字符（保持现有规则）                 |
| 复杂度   | 建议包含大小写字母+数字（非强制）      |
| 验证逻辑 | 使用 `validatePassword()` 函数（已有） |

#### 3.5.2 管理后台访问控制

| 项目       | 策略                                                                  |
| ---------- | --------------------------------------------------------------------- |
| 身份验证   | Admin Secret（URL 参数 `?secret=xxx` 或 `admin_token` Cookie）        |
| 鉴权模块   | `lib/admin-auth.ts` 的 `verifyAdminSecret()` / `verifyAdminRequest()` |
| Middleware | 现有 `middleware.ts` 已拦截 `/admin` 和 `/api/admin/` 路径            |
| 未授权处理 | 页面访问返回 401，API 返回 401 JSON                                   |
| 环境变量   | `ADMIN_SECRET`（已在 `.env` 中配置）                                  |

#### 3.5.3 操作日志

| 操作类型       | 记录内容                                                              |
| -------------- | --------------------------------------------------------------------- |
| 管理员重置密码 | user_id, reset_by（管理员）, reset_method='admin_reset', created_at   |
| 用户修改密码   | user_id, reset_by（用户自己）, reset_method='self_change', created_at |

## 六、风险评估

| 风险               | 影响                       | 应对措施                                             |
| ------------------ | -------------------------- | ---------------------------------------------------- |

| 日志表数据增长     | 长期运行后数据量大         | 定期清理或归档（非关键数据）                         |

```typescript
// packages/author-site/src/lib/db/schema.ts（修改后）
export function initializeDatabase(): void {
  const db = getDb();

  // ... 现有 users、system_configs 建表语句保持不变 ...

  // 新增：密码重置日志表
  db.exec(`
    CREATE TABLE IF NOT EXISTS password_reset_logs (
      id TEXT PRIMARY KEY,

      FOREIGN KEY (user_id) REFERENCES users(id)
    )

  db.exec(`
    CREATE INDEX IF NOT EXISTS idx_reset_logs_user ON password_reset_logs(user_id)
  `);
  db.exec(`
    CREATE INDEX IF NOT EXISTS idx_reset_logs_created ON password_reset_logs(created_at)
  `);

```

### 8.2 用户管理 API 鉴权示例

所有 `/api/admin/users` 相关路由复用现有 Admin Secret 鉴权，**无需新建鉴权模块**：

```typescript
// packages/author-site/src/app/api/admin/users/route.ts
import { verifyAdminRequest } from "@/lib/admin-auth";
import { listAllUsers } from "@/lib/user";

export async function GET(request: Request) {
  if (!(await verifyAdminRequest(request))) {
    return NextResponse.json(
      {
        success: false,
        error: { code: "UNAUTHORIZED", message: "未授权访问" },
      },
      { status: 401 },
    );
  }

  const users = listAllUsers();
  return NextResponse.json({ success: true, data: { users } });
}
```
