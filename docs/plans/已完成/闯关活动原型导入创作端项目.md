## 归档结论

本任务已完成。用户提供的 `/Users/qh2/Downloads/闯关活动.pdf` 是一张单页超大画布，最终被拆分为 27 个手机原型页面，并导入创作端项目 `proj_1782839405716_tqjl1f`。最终版本为 `v8`：27 个页面均改为无整页截图、无 TS 类型语法、无手写 `react/jsx-runtime` 的原始 JSX 源码，由创作端预览编译链路统一转换。

该任务的主要复用价值不是 PDF 裁剪流程本身，而是暴露了三类创作端项目导入风险：项目 workspace 与 session workspace 可能分叉、相对图片路径在 session 编译链路中不可直接依赖、预先转换 JSX runtime 形态会绕开 author-site 编译器预期输入契约。

## 保留经验

- PDF 原型如果是单页大画布，应先低分辨率渲染总览，再裁剪独立手机页面，避免把整页画布误当成单个页面。
- 创作端页面不能长期依赖 `../../assets/images/...` 这类项目内相对图片路径；session 编译链路需要明确资产路径改写，否则浏览器无法读取。
- 已打开的 session workspace 不会自动跟随项目主版本修复。项目 workspace 修好后，旧 session 仍可能引用旧源码，需要刷新、重开会话或显式同步 session workspace。
- 为了兼容 author-site 编译链路，导入页应保留原始 JSX 作为输入，不应提前写成 `react/jsx-runtime` 导入产物。
