## 📝 文档维护规范

### 新增文档

1. 新文档完成后放入对应主题目录
2. 文档命名格式：`{功能/问题描述}+{文档类型}.md`
   - 示例：`AI对话多模态图片上传方案.md`、`Docker部署登录不跳转问题-综合报告.md`
3. 文档开头应包含：
   - 版本号和创建/更新日期
   - 文档状态（已完成/进行中/已废弃）
   - 关联文档链接（如有）

### 文档评估

- **每季度评估一次**文档价值
- 评估标准：
  - ⭐⭐⭐ **高价值**：架构决策、复杂 bug 根因、技术选型、最佳实践 → 保留并置顶
  - ⭐⭐ **中价值**：功能方案、API 设计、部署流程 → 保留并分类
  - ⭐ **低价值**：简单 UI 调整、一次性任务、纯过程记录 → 删除
  - ❌ **过期**：技术栈已移除、功能被替代、问题已根治 → 删除

### 过期文档处理

1. 过期文档移至 `99-归档/` 目录
2. 满 3 个月后安全删除
3. 仍有价值的文档移回分类目录
