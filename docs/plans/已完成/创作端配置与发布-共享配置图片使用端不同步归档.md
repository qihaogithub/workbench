# 创作端配置与发布 - 共享配置图片使用端不同步归档

## 结论

共享配置图片不同步的根因不在单一 viewer 渲染点，而是运行值持久化、发布初始配置和使用端合并顺序共同缺口。

本轮已收敛为当前契约：

- 共享配置面板的实际运行值写入 `project.config.values.json`，并推进 workspace 更新时间。
- 发布前等待共享配置保存队列完成，再进入 workspace 同步和发布。
- 发布包在 `project.json` 携带 `projectConfigValues`，存在运行值时额外写出 `config-values.json`。
- 发布图片扫描包含 `project.config.values.json` 和 `/api/sessions/.../assets/...` 上传图；运行值覆盖 schema default 时，不再因旧 default 图失败阻断发布。
- 发布 iframe、author-site viewer 和独立 viewer-site 均按“项目级 schema 默认值 + 页面级 schema 默认值 + 共享配置运行值”初始化，使发布运行值覆盖同名页面默认值。
- 发布 API 会在 flush 后用 live workspace 的非空运行值兜底补写 canonical workspace，避免发布前同步遗漏 `project.config.values.json`。
- 直嵌发布 iframe 在加载和配置变化时接收 `UPDATE_CONFIG`，并按 iframe 源地址解析图片资源 URL，避免使用端右侧配置面板和 iframe 画面分裂。
- 独立 viewer-site 本地开发未配置 `NEXT_PUBLIC_DATA_BASE` 时默认从 `http://localhost:3200` 读取发布数据，避免 `localhost:3300/data/...` 被 Next 页面路由拦截成 HTML。

## 关键证据

- 本地项目 `proj_1782718596686_wmv9yu` 的旧发布产物没有 `config-values.json`，`project.json` 也没有 `projectConfigValues`；旧 iframe 初始配置仍是 schema default。
- 2026-07-08 17:41 的远端截图对应 `10.130.33.131:3300` 发布包 `v20`：`project.json` 没有 `projectConfigValues`，`config-values.json` 返回 404，iframe 中 `currentConfig.modalImage` 仍是 schema default。
- 同一截图中使用端右侧配置缩略图和 iframe 画面不一致，原因是独立 viewer-site 的直嵌 iframe 未接收右侧配置状态。
- 本地开发环境中 `localhost:3300/data/proj_1782718596686_wmv9yu/project.json` 返回 viewer HTML；修复后 Playwright 打开 `localhost:3300/proj_1782718596686_wmv9yu/`，发布数据请求均命中 `http://localhost:3200/data/...` 并返回 JSON/静态资源。
- `proj_1782718596686_wmv9yu` 的发布 iframe 只监听 `UPDATE_CONFIG`；实测给 `phone-popup_k8m2` iframe 手动发送 `UPDATE_CONFIG` 后，`window.__DEMO_PROPS__.modalImage` 与 `img[alt="popup"]` 均更新。修复直嵌 iframe 消息类型后，在使用端右侧详情上传合法 670x670 图片，配置面板缩略图和 iframe 内弹窗图片同步更新。
- 结构化诊断 `recent`、`project`、`preview`、`autosave` 均使用 SQLite，未发现 JSONL fallback、数据库不可用或 event gap。
- 回归测试新增覆盖“共享配置图片改为 session 上传图后，发布包 `config-values.json`、`project.json.projectConfigValues` 和 iframe 初始配置一致，且使用本地化后的 `/data/.../assets/images/...` URL”。
