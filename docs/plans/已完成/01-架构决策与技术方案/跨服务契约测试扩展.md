# 跨服务契约测试扩展

> 最终状态：已完成
> 完成日期：2026-06-28

## 背景

[Codex 开发模式下项目维护性剩余优化任务拆分](./Codex开发模式下项目维护性优化建议.md) 将“扩展跨服务契约测试”列为 P1-2。当前已有 viewer 发布数据的最小契约检查，但 author、agent、screenshot、project-core、project-cli 等边界仍缺少统一契约校验入口。

## 目标

- 增加不依赖真实 LLM、真实浏览器或外部网络的契约检查。
- 让 API success/error、viewer 发布数据、screenshot 请求/响应、agent session/stream 事件结构有最小稳定校验。
- 将契约检查接入根脚本，成为后续改协议时的低成本验证入口。

## 范围

- `scripts/check-viewer-contracts.mjs`
- 新增或扩展契约检查脚本
- `package.json`
- 维护治理文档与任务池

## 方案

1. 先复用现有 Node 脚本方式，不新增测试框架依赖。
2. 将通用契约校验集中在脚本内，先覆盖样例 fixture 和当前发布数据。
3. 对 agent/screenshot/project 契约使用静态样例结构校验，避免启动服务。
