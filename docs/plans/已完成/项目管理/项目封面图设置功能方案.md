# 项目封面图设置功能方案

> 创建日期：2026-05-03
> 状态：已实施

---

## 一、背景与目标

### 1.1 现状

- 首页 DemoCard 已支持缩略图展示：有 `thumbnail` 时显示图片，无则显示占位图标
- `DemoMeta` 类型已预留 `thumbnail?: string` 字段
- 封面图存储在 `public/thumbnails/` 目录
- **但用户无法手动设置封面图**

### 1.2 目标

在项目编辑页左上角的项目名称旁，添加"设置封面图"按钮，允许用户手动上传自定义封面图，用于首页卡片展示。

---

## 二、功能设计

### 2.1 入口位置

在编辑页顶部导航栏，项目名称右侧添加一个图标按钮：

```
┌─────────────────────────────────────────────────────────┐
│  📄 项目名称  [🖼️]                    [取消] [保存]     │
│              ↑                                         │
│         设置封面图按钮                                   │
└─────────────────────────────────────────────────────────┘
```

- 按钮使用 `ImageIcon`（lucide-react），尺寸与文字协调
- hover 时显示 tooltip "设置封面图"
- 若已有封面图，按钮使用实心图标样式提示

### 2.2 交互流程

点击按钮后弹出 Dialog 对话框，包含以下内容：

```
┌─────────── 设置封面图 ───────────┐
│                                   │
│  ┌─────────────────────────────┐  │
│  │                             │  │
│  │     当前封面图预览           │  │
│  │     （或占位区域）           │  │
│  │                             │  │
│  └─────────────────────────────┘  │
│                                   │
│  ┌─────────────────────────────┐  │
│  │  📁 点击或拖拽上传封面图     │  │
│  │     支持 JPG/PNG/WebP       │  │
│  │     建议 16:9 比例          │  │
│  └─────────────────────────────┘  │
│                                   │
│         [删除封面图]                │
│                                   │
│              [关闭]                │
└───────────────────────────────────┘
```

**两种操作**：

| 操作 | 说明 |
|------|------|
| 上传自定义图片 | 选择本地图片文件，上传后替换当前封面 |
| 删除封面图 | 移除封面图，首页卡片恢复占位图标显示 |

### 2.3 上传约束

| 约束项 | 值 |
|--------|-----|
| 允许格式 | JPG、PNG、WebP |
| 最大文件大小 | 5MB |
| 建议比例 | 16:9（与 DemoCard 的 `aspect-video` 一致） |
| 建议最小尺寸 | 640×360 |

## 三、技术方案

### 3.1 数据模型

`DemoMeta.thumbnail` 字段已存在，无需修改类型定义。封面图路径存储规则：

| 来源 | 路径格式 |
|------|----------|
| 手动上传 | `/thumbnails/{projectId}_cover.{ext}` |

封面图统一使用 `_cover` 后缀命名，上传时自动覆盖旧文件。

### 3.2 新增 API

**上传封面图**：

```
POST /api/demos/[id]/cover
Content-Type: multipart/form-data

请求体：file (图片文件)
响应：{ success: true, data: { thumbnail: "/thumbnails/{id}_cover.png" } }
```

**删除封面图**：

```
DELETE /api/demos/[id]/cover
响应：{ success: true, data: { thumbnail: null } }
```

### 3.3 新增文件

| 文件 | 说明 |
|------|------|
| `packages/web/src/app/api/demos/[id]/cover/route.ts` | 封面图上传/删除 API |
| `packages/web/src/components/cover-image-dialog.tsx` | 封面图设置对话框组件 |

### 3.4 修改文件

| 文件 | 修改内容 |
|------|----------|
| `packages/web/src/app/demo/[id]/edit/page.tsx` | 在项目名称右侧添加封面图按钮，引入对话框 |
| `packages/web/src/lib/api.ts` | 添加 `uploadCover`、`deleteCover` 函数 |
| `packages/shared/src/index.ts` | 新增错误码 `COVER_UPLOAD_FAILED` |

### 3.5 图片存储

封面图存储在 `public/thumbnails/` 目录下，文件名为 `{projectId}_cover.{ext}`，上传时删除旧封面再保存新封面。

### 3.6 关键实现逻辑

**上传 API 核心流程**：

1. 校验文件类型和大小
2. 删除已有的手动封面文件（`*_cover.*`）
3. 保存新文件到 `public/thumbnails/{projectId}_cover.{ext}`
4. 更新 `DemoMeta.thumbnail` 为新路径
5. 返回新路径

**编辑页按钮逻辑**：

1. 点击按钮 → 打开 CoverImageDialog
2. Dialog 内展示当前封面图（读取 `demo.thumbnail`）
3. 上传/删除后，更新本地状态 + revalidate 首页列表

**首页联动**：

DemoCard 已根据 `demo.thumbnail` 渲染封面图，无需修改。上传/删除封面后，通过 SWR 的 `mutate` 刷新项目列表即可。

## 四、UI 细节

### 4.1 封面图按钮样式

```tsx
<Button
  variant="ghost"
  size="icon"
  className="h-7 w-7"
  onClick={() => setCoverDialogOpen(true)}
  title="设置封面图"
>
  <ImageIcon className="h-4 w-4" />
</Button>
```

- 使用 `ghost` 变体，不抢视觉焦点
- 尺寸 7×7，与项目名称文字高度协调
- 已有封面图时，可考虑使用实心 `ImageIcon` 或添加小圆点指示

### 4.2 上传区域

参考已有的 `FileUploadWidget` 组件风格，使用拖拽上传区域：

- 虚线边框区域，hover 时高亮
- 支持点击选择文件和拖拽上传
- 上传中显示进度/loading 状态
- 预览区域展示当前封面图（16:9 比例）

### 4.3 Dialog 尺寸

- 最大宽度 `max-w-md`
- 预览区域固定 16:9 比例

## 五、边界情况

| 场景 | 处理方式 |
|------|----------|
| 上传超大图片 | 前端限制 5MB，后端也做校验，超限返回 `FILE_TOO_LARGE` 错误 |
| 上传非图片文件 | 前端 accept 限制 + 后端 MIME 校验，返回 `INVALID_FILE_TYPE` 错误 |
| 项目无封面图 | Dialog 预览区显示占位图，"删除封面图"按钮禁用 |
| 并发编辑同一项目 | 最后一次操作覆盖，与现有名称编辑行为一致 |
