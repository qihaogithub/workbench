# 项目级配置与页面级配置设计

> 版本：v1.0
> 创建日期：2026-05-05
> 状态：方案设计中

---

## 一、配置层级概览

多 Demo 页面系统中，配置分为两个层级：

| 层级 | 文件位置 | 作用范围 |
|:-----|:---------|:---------|
| **项目级** | `workspace/project.config.schema.json` | 所有页面共享 |
| **页面级** | `workspace/demos/{demoId}/config.schema.json` | 仅当前页面使用 |

---

## 二、项目级配置

### 2.1 定位

项目级配置用于定义**所有页面共享的配置项**，如 Logo、品牌色、联系方式等。

典型场景：

```
用户上传一张 Logo 图片 → 存入项目级配置
首页组件 Props 中包含 logo
详情页组件 Props 中也包含 logo
用户只需上传一次，所有页面自动可用
```

### 2.2 文件结构

```json
// project.config.schema.json
{
  "type": "object",
  "properties": {
    "logo": {
      "type": "string",
      "title": "Logo 图片",
      "default": ""
    },
    "brandColor": {
      "type": "string",
      "title": "品牌色",
      "default": "#3B82F6"
    }
  }
}
```

### 2.3 存在性判定

是否存在项目级配置由文件本身的存在性决定：

- 文件存在 → 有项目级配置
- 文件不存在 → 无项目级配置

**无冗余字段**：`project.json` 中不保存 `hasProjectConfig` 等标记字段。

---

## 三、页面级配置

### 3.1 定位

页面级配置用于定义**当前页面特有的配置项**，如页面标题、内容、布局等。

### 3.2 文件结构

```json
// demos/home/config.schema.json
{
  "type": "object",
  "properties": {
    "title": {
      "type": "string",
      "title": "页面标题",
      "default": "首页"
    },
    "subtitle": {
      "type": "string",
      "title": "副标题",
      "default": "欢迎来到我的网站"
    }
  }
}
```

## 四、配置合并机制

### 4.1 合并规则

页面组件在渲染时接收的 Props 来自两个配置 Schema 的 `default` 值合并：

```
┌─────────────────────────────────────────────┐
│           project.config.schema.json          │
│  { logo: { default: "" },                   │
│    brandColor: { default: "#3B82F6" } }     │
│                                              │
│                      +                       │
│                                              │
│         demos/home/config.schema.json         │
│  { title: { default: "首页" },              │
│    subtitle: { default: "副标题" } }         │
│                                              │
│                      =                       │
│                                              │
│         传入 home/index.tsx 的 Props          │
│  { logo: "", brandColor: "#3B82F6",         │
│    title: "首页", subtitle: "副标题" }      │
└─────────────────────────────────────────────┘
```

### 4.2 组件代码示例

```tsx
// demos/home/index.tsx — 同时使用项目配置和页面配置的字段
interface HomeProps {
  // 来自 project.config.schema.json
  logo: string
  brandColor: string
  // 来自 demos/home/config.schema.json
  title: string
  subtitle: string
}

export default function Home({ logo, brandColor, title, subtitle }: HomeProps) {
  return (
    <div style={{ backgroundColor: brandColor }}>
      <img src={logo} alt="Logo" />
      <h1>{title}</h1>
      <p>{subtitle}</p>
    </div>
  )
}
```

### 4.3 运行时注入机制

> **核心设计**：项目级字段不通过 Props 接口声明，由 PreviewPanel / embed 在编译时统一注入。

页面组件的 Props 接口中**只声明页面级字段**，项目级字段通过运行时注入获取：

```tsx
// 正确写法：项目级字段从 props 解构使用，不在接口中声明
interface HomeProps {
  // 页面级字段（显式声明）
  title: string
  subtitle: string
}

export default function Home(props: HomeProps) {
  const { logo = '', brandColor = '#000' } = props as Record<string, unknown>
  return (
    <div style={{ backgroundColor: brandColor as string }}>
      <img src={logo as string} alt="Logo" />
      <h1>{props.title}</h1>
    </div>
  )
}
```

## 五、冲突处理规则

### 5.1 禁止重名

**项目级配置与页面级配置禁止出现同名字段**。

原因：项目配置的初衷是"全局共享，一处修改全站生效"。如果允许同名字段且页面级覆盖项目级，会导致"用户改了项目 Logo，但某页面因字段重名卡死在旧值"。

### 5.2 校验时机

| 时机 | 校验方式 |
|:-----|:---------|
| **写入时校验** | 所有 Schema 写入入口（`PUT /api/projects/{id}/config`、`PUT /api/projects/{id}/demos/{demoId}/files`、`POST /api/generate-schema`）在落盘前调用 `validateNoSchemaConflict`，发现重名直接 400 拒绝 |
| **合并时校验（兜底）** | `mergeConfigToProps()` 检测到重名时抛出 `SchemaConflictError`，由 PreviewPanel 捕获后展示明确错误提示 |

### 5.3 配置面板展示

配置面板以**视觉分组区分来源**：

```
┌─ 配置面板 ──────────────────────────────────────────┐
│                                                      │
│  ┌─ 📋 项目配置（所有页面共享）─────────────────┐   │
│  │  Logo图片  ┌──────────────────────────┐     │   │
│  │            │ [上传] 📷 current-logo.png │     │   │
│  │            └──────────────────────────┘     │   │
│  │  品牌色    ┌──────────────────────────┐     │   │
│  │            │ 🎨 #3B82F6                 │   │   │
│  │            └──────────────────────────┘     │   │
│  └──────────────────────────────────────────────┘   │
│                                                      │
│  ┌─ 📄 当前页面：首页 ─────────────────────────┐   │
│  │  页面标题  ┌──────────────────────────┐     │   │
│  │            │ 欢迎来到我的网站           │   │   │
│  │            └──────────────────────────┘     │   │
│  └──────────────────────────────────────────────┘   │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## 六、AI 约束（简化版）

### 6.1 新增项目配置字段

1. 创建或编辑 `project.config.schema.json`，加入新字段
2. 在确实需要展示该字段的页面，编辑 `index.tsx` 渲染逻辑（从 props 解构使用）
3. **不需要**修改不使用该字段的页面
4. **不需要**改动任何页面的 Props 接口声明
5. **不需要**把项目级字段写进任何页面的 `config.schema.json`

### 6.2 删除项目配置字段

1. 编辑 `project.config.schema.json` 移除字段
2. 在使用了该字段的页面渲染逻辑里清理引用
3. 其他页面无需改动

### 6.3 新增页面

1. 创建目录和默认 `index.tsx` + `config.schema.json`
2. 默认 `index.tsx` 模板里通过解构方式取项目配置字段（如 `const { logo } = props as Record<string, unknown>`）
3. 不需要在 Props 接口或 page schema 里重复定义项目配置字段

### 6.4 禁止行为

- ❌ 在页面 `config.schema.json` 中重复定义项目配置已有的字段（写入会被后端拒绝）
- ❌ 在 Props 接口中重复声明项目级字段（违反运行时注入约定）

## 七、配置值推导机制

> **说明**：系统不持久化 `configData`，每次进入编辑页都靠 `getDefaultValues(loadedSchema)` 从 Schema 的 `default` 字段实时推导。

如果未来需要"记住用户填写的配置值"，可在页面目录下新增 `config.data.json`，改动范围局部且向后兼容（找不到 data.json 就回退到 Schema default 推导）。

## 八、相关类型定义

### 8.1 ProjectConfig

```typescript
interface ProjectConfig {
  schema: string                          // project.config.schema.json 内容
  exists: boolean                         // 是否存在项目级配置
}
```

### 8.2 MergedComponentProps

```typescript
interface MergedComponentProps {
  // 合并规则：
  // 1. 取 project.config.schema.json 定义的所有字段（从 default 提取值）
  // 2. 取 demos/{pageId}/config.schema.json 定义的所有字段（从 default 提取值）
  // 3. 字段必须互斥不重名（写入时已强校验，运行时再兜底检测）
  [key: string]: unknown
}
```

### 8.3 mergeConfigToProps 函数签名

```typescript
function mergeConfigToProps(
  projectSchema: string | undefined,
  pageSchema: string
): Record<string, unknown> {
  // 1. 解析两个 Schema，从 default 字段提取默认值
  // 2. 兜底校验：若两个 Schema 字段集合存在交集，抛出 SchemaConflictError
  // 3. 合并不重名的字段，返回 Props 对象
}
```
