## 背景

`project-core` 与 `project-cli` 当前新增了面向创作端预览运行契约的只读校验能力，但 CLI 自动维护账本还没有同步登记本轮变化，下一次定时任务会把它误判成潜在缺口。

## 目标

- 确认 runtime contract 校验能力属于低风险只读 CLI 对齐项。
- 验证 `project-core` / `project-cli` 当前实现可通过标准检查。
- 把能力清单、长期跟踪和 OPS state 更新到与代码一致。

## 范围

- `packages/project-core/src/service.ts`

- `OPS/automations/state/cli-maintenance-current.md`
- `docs/plans/进行中/CLI与创作端能力对齐长期跟踪.md`

## 方案

1. 复查现有未提交改动，确认 runtime contract 校验已由 `project-core` 复用共享能力承载，CLI 只做参数适配与 JSON 输出。
2. 运行 `corepack pnpm check:project-core` 与 `corepack pnpm check:project-cli`。
3. 若验证受阻，只做与本轮链路直接相关的最小修复，不扩展到无关语义。
4. 更新能力清单、当前状态和长期跟踪，保留剩余结构性缺口。
