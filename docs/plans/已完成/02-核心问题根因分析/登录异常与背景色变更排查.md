# 登录异常与背景色变更排查

> 状态：已完成
> 完成时间：2026-06-29

## 背景

2026-06-29 用户反馈登录页背景色变浅，并且使用现有账号无法登录。需要判断是用户凭据不匹配，还是登录链路、用户数据或前端交互存在系统缺陷。

## 目标

- 核实本地用户数据中目标账号是否存在。
- 验证用户提供的登录凭据是否能通过当前认证逻辑。
- 定位登录页背景色变化来源，判断是否属于代码或样式变更。
- 如确认是系统缺陷，实施最小范围修复并完成对应验证。

## 范围

- 创作端登录页 UI 与认证请求。
- 创作端登录 API 与用户数据读取逻辑。
- 本地数据目录中的用户记录。

不处理无关页面、历史目录或现有未提交改动。

## 方案

1. 读取登录页和认证 API 的实现，确认请求路径、错误反馈与跳转逻辑。
2. 检查本地用户数据库，仅验证账号存在性和密码校验结果，不暴露敏感凭据。
3. 对比登录页样式来源，确认背景色是否由近期代码变更导致。
4. 根据结论决定是否修复代码、更新文档并运行匹配验证。

## 实施摘要

- 登录页背景从固定浅色渐变调整为全局主题背景，避免深色卡片叠在浅色页面上。
- Docker Compose 的 `USE_SECURE_COOKIE` 默认值调整为 `false`，匹配当前 HTTP 内网访问方式；HTTPS 部署仍可通过环境变量显式开启。
- 登录 API 和登录页提交前会裁剪用户名首尾空白，保留密码原文校验。
- 鉴权和 Docker 部署长期文档已同步更新。
