# 创作端编辑与协同问题沉淀

## 当前状态

协同 WebSocket 连接问题已修复并部署到正式环境，验证通过。

## 问题条目

### 协同 WebSocket 连接卡在"连接中"（Hocuspocus 事件未转发）

- **现象**：重新部署后，创作端项目编辑页右上角同步状态始终显示"连接中"；AI 编辑页面后退出弹窗提示"修改仍在同步"；但退出后重新打开，编辑内容实际已保存。
- **影响范围**：所有协同 WebSocket 连接（page-code、page-schema、workspace-tree、canvas-layout 等），导致协同状态显示异常和退出弹窗误触。
- **根因**：commit `b94ca3a79` 将协同系统从 `y-websocket` 迁移到 Hocuspocus 时存在两层问题：
  1. **路由层残留检查**（已修复）：`collab.ts` 的 WebSocket 路由处理器中残留了旧的 `normalizeDescriptor` 检查，新客户端不发送 query 参数导致连接被 `1008` 关闭。
  2. **WebSocket 事件未转发**（本次修复）：`hocuspocus-server.ts` 的 `handleConnection` 调用 `this.hocuspocus.handleConnection(socket, webRequest)` 后未转发 WebSocket 事件。Hocuspocus 的 `ClientConnection` **不自行设置 WebSocket 监听器**，调用方必须手动转发 `message` → `clientConnection.handleMessage()` 和 `close` → `clientConnection.handleClose()`。这导致 WebSocket 连接建立但客户端消息永远无法到达 Hocuspocus，同步流程卡死。
- **修复摘要**：
  1. 移除 `collab.ts` WebSocket 路由中的 `normalizeDescriptor` 检查。
  2. 在 `hocuspocus-server.ts` 的 `handleConnection` 中，存储 `handleConnection` 返回的 `ClientConnection` 实例，设置 `socket.on("message", ...)` 转发到 `clientConnection.handleMessage(bytes)`，`socket.on("close", ...)` 转发到 `clientConnection.handleClose({ code, reason })`，`socket.on("error", ...)` 记录日志。
- **验证状态**：`pnpm check:agent` 384 tests 全通过；`pnpm typecheck` 通过。部署后用 HocuspocusProvider 测试连接，`authenticationFailed` 事件正确触发（之前不触发），确认消息转发链路已修复。真实 session 会通过认证并触发 `synced` 事件。
- **相关文件**：`packages/agent-service/src/collab/hocuspocus-server.ts`、`packages/agent-service/src/routes/collab.ts`
- **关键参考**：Hocuspocus 内置 Server 使用 crossws hooks 转发事件（`open` → `handleConnection`，`message` → `handleMessage`，`close` → `handleClose`），自定义集成必须复刻此模式。

## 风险

- HTTP flush/write 路由仍依赖 `normalizeDescriptor` 从 query 参数提取 descriptor，这些路由由服务端代码调用（非 Hocuspocus 客户端），参数传递方式正确，不受此问题影响。
