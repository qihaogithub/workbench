# 配置与预览 - AI 配置化边界优化归档

## 结论

本次优化确认：`config.schema.json` 应继续定位为“用户确认后的配置字段协议”，不是页面结构 DSL、AI 生成页面的必填内容模型，也不是布局或组件层级协议。

没有用户明确配置诉求时，新建页面应保留空 `properties`，页面标题、文案、图片、颜色、按钮和布局默认写入页面源码或原型页文件。用户明确选择“设为配置项”后，系统才进入配置化写回链路。

## 核对结果

- `02_实时预览机制.md` 已准确描述 HTML/CSS 原型页通过 `data-bind-*` / `{{fieldKey}}` 绑定配置字段，且不走 React Props 注入。
- `06_可视化批注与编辑机制.md` 已准确描述高保真页自动配置化只覆盖文本、图片、颜色三类高确定性内容，并在无法唯一定位时拒绝自动写回。
- 本轮不新增代码校验层。现有 Agent prompt、可视化配置化接口和项目文档已对齐；如果后续发现 CLI/Codex 或直接文件写入绕过配置诉求边界，再评估写入边界的轻量校验。

## 剩余风险

- `$demo.*` 仍是有限运行时扩展，后续新增能力前必须先判断归属，避免继续膨胀为页面结构语言。
- 复杂列表、组件组、布局结构、动态表达式和跨页面业务动作应继续走页面源码、原型页、属性面板结构化上下文、AI 修改链路、画布布局或 app graph。
