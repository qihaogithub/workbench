# 统一创作端页面契约实施方案

## 归档结论

本任务已完成主体实施。创作端页面源码的运行规则已经从 author-site 编译器、Agent prompt、CLI 静态校验等分散位置收敛到共享包 `@opencode-workbench/preview-contract`，由 author-site、project-core、project-cli 和创作端 Agent prompt 复用。

author-site 仍保留最终预览编译入口和 runtime URL 解析职责；共享 contract 负责页面源码契约、依赖策略、编译前包装、Sucrase 转换和 Agent 创作规则生成。CLI 增加页面/项目运行时校验能力，`preview compile` 复用同一套页面级校验输出。

## 保留经验

- 页面运行契约必须由共享包产出，不能在 prompt、CLI、author-site 编译器里复制第二套规则。
- project-core / CLI 可在提交前发现页面源码运行契约问题，但 author-site 仍是最终预览编译入口。
- 新增 workspace 包时必须坚持项目要求的 pnpm 8 / lockfile v6；使用系统 pnpm v11 会带来 lockfile 和 node_modules 差异。
