## 1. 背景

### 1.1 用户反馈

用户使用"对话历史"功能时发现：切换历史对话，预览区代码/效果也会跟着变化。用户期望切换的只是与 AI 的对话上下文，而不是整个工作空间。

### 1.2 问题本质

当前实现将"会话"与"工作空间"强绑定——每个会话目录下都存有独立的代码副本（index.tsx、config.schema.json），切换会话时必然切换代码文件，导致预览区变化。

```
当前模型：Session = 工作空间 + 对话消息 + 代码文件
期望模型：Session = 对话消息；Workspace = 独立实体，不随会话切换
```

### 1.3 已发现的附带问题

- `onSelectSession` 将旧会话状态设为 `"archived"`，但 `SessionMeta.status` 类型定义为 `'editing' | 'saved' | 'discarded'`，不包含 `"archived"`，存在类型不一致
- `findActiveSession` 会主动删除过期会话目录，而列表 API 仅标记 `isExpired` 不删除，行为不一致
- Web 端（文件系统）与 agent-service 端（内存）的会话管理割裂，服务重启后 agent-service 会话信息丢失

## 4. 方案设计

### 4.1 目标数据模型

```
Workspace（工作空间）          1:N           Session（会话）
┌──────────────────────┐      ──────►      ┌──────────────────────┐
│ workspaceId          │                   │ sessionId            │
│ demoId               │                   │ workspaceId  ← 新增  │
│ userId               │                   │ demoId               │
│ createdAt            │                   │ userId               │
│ updatedAt            │                   │ title                │
│                      │                   │ createdAt            │
│ 文件存储：            │                   │ expiresAt            │
│  index.tsx           │                   │ status               │
│  config.schema.json  │                   │ basedOnVersion       │
│  .opencode/          │                   │ opencodeSessionId    │
└──────────────────────┘                   └──────────────────────┘
                                           仅存储：
                                            .session.json
                                            .messages.json
```

### 4.2 目标存储结构

```
data/workspaces/{userId}/{projectId}/{workspaceId}/
  .workspace.json       # 工作空间元数据
  index.tsx             # 代码文件
  config.schema.json    # Schema 文件
  .opencode/            # Agent 配置

data/sessions/{userId}/{projectId}/{workspaceId}/{sessionId}/
  .session.json         # 会话元数据（新增 workspaceId 字段）
  .messages.json        # 消息历史（不再包含代码文件）
```

### 4.3 改造后的切换逻辑

```
用户点击历史对话
  ↓
HistoryDialog.onSelectSession(sessionId, workspaceId)
  ↓
1. 更新旧/新会话状态
2. 获取新会话消息历史 → setAiMessages
3. 重置对话流状态
4. setSessionId(newSessionId)
   ← 不再调用 setCode / setSchema / setTempWorkspace
   ← 预览区保持不变
```

onSelectSession={async (newSessionId, workspaceId) => {

interface CreateSessionResponse {
