# 验证错误提示优化方案

## 一、问题背景

### 1.1 当前验证错误展示现状

在 Demo 编辑页面保存时，系统会执行两类校验：

1. **JSON Schema 语法校验** - 验证 Schema 是否为合法 JSON
2. **Props 与 Schema 一致性校验** - 对比代码中的 props 与 Schema 定义是否匹配

但目前错误展示存在以下问题：

| 问题 | 现状 | 影响 |
|------|------|------|
| 错误信息过于笼统 | 只显示"当前页面存在验证错误，保存后可能无法正常预览" | 用户不知道具体哪里出错 |
| 缺少错误分类 | 所有错误类型混在一起显示 | 无法区分优先级 |
| 无具体位置信息 | 没有告诉用户在哪一行或哪个字段出错 | 修复困难 |
| 无修复建议 | 没有提供解决方案的提示 | 用户需要自行排查 |

### 1.2 错误展示代码位置

- **验证器**: `packages/author-site/lib/validator.ts`
- **展示层**: `packages/author-site/src/app/demo/[id]/edit/page.tsx` (第 425-430 行)

## 二、验证错误类型定义

### 2.1 当前错误类型

```typescript
// validator.ts
interface ValidationError {
  type:
    | "json_syntax"      // JSON 语法错误
    | "props_mismatch"   // Props 不匹配
    | "required_missing" // 必填字段缺失
    | "interface_not_found"; // 未找到接口定义

```

### 2.2 当前错误消息示例

| 类型 | 当前消息 |
|------|----------|
| `json_syntax` | `JSON 语法错误: ${errorMessage}` |
| `props_mismatch` | `代码中的 props "${prop}" 未在 Schema 的 properties 中定义` |
| `props_mismatch` | `Schema 中的 property "${prop}" 未在代码的 DemoProps 中定义` |

## 三、优化方案

### 3.1 方案一：增强 ValidationError 结构（推荐）

#### 3.1.1 扩展错误类型枚举

```typescript
// 新增更多错误类型，便于分类处理
type ValidationErrorType =
  // JSON Schema 相关
  | "json_syntax"           // JSON 语法错误
  | "schema_missing"        // Schema 为空
  | "schema_invalid"        // Schema 结构无效

  // Props 一致性相关
  | "props_code_not_in_schema"    // 代码中有但 Schema 中没有
  | "props_schema_not_in_code"   // Schema 中有但代码中没有
  | "interface_not_found"         // 未找到 DemoProps 定义
  | "interface_ambiguous"        // 接口定义不明确（多个匹配）

  // Required 相关
  | "required_missing_in_props"  // 必填字段在 properties 中缺失
  | "required_field_undefined"    // 必填字段未定义

  // 类型相关
  | "type_mismatch"         // 属性类型不匹配
  | "enum_mismatch";        // 枚举值不匹配
```

#### 3.1.2 增强错误信息结构

```typescript
interface ValidationError {
  type: ValidationErrorType;

  // 友好的中文消息（展示给用户）
  message: string;

  // 技术细节（可选展示）
  technicalDetails?: string;

  // 错误位置
  location?: {
    type: "code" | "schema" | "both";
    file?: string;           // 文件名
    line?: number;           // 行号（如果能确定）
    column?: number;         // 列号
    range?: {                // 范围（起始-结束位置）
      start: { line: number; column: number };
      end: { line: number; column: number };
    };
  };

  // 涉及的字段
  field?: {
    name: string;            // 字段名
    path?: string;           // 完整路径，如 "properties.title"
  };

  // 错误严重程度
  severity: "error" | "warning" | "info";

  // 修复建议
  fixSuggestion?: {
    action: "add_to_schema" | "remove_from_code" | "fix_json" | "add_interface" | "none";
    description: string;
    example?: string;         // 代码示例
  };
}
```

### 3.3 方案三：构建验证结果展示组件

#### 3.3.1 验证状态面板组件

```tsx
// 位置: components/demo/ValidationPanel.tsx

interface ValidationPanelProps {
  result: ValidationResult;
  onFixClick?: (error: ValidationError) => void;
  onDismiss?: () => void;
}

function ValidationPanel({ result, onFixClick, onDismiss }: ValidationPanelProps) {
  const errors = result.errors.filter(e => e.severity === "error");
  const warnings = result.errors.filter(e => e.severity === "warning");
  const infos = result.errors.filter(e => e.severity === "info");

  return (
    <div className="validation-panel border rounded-lg overflow-hidden">
      {/* 头部统计 */}
      <div className="p-4 border-b bg-muted/50">
        <div className="flex items-center gap-4 text-sm">
          <Badge variant="destructive">{errors.length} 错误</Badge>
          <Badge variant="warning">{warnings.length} 警告</Badge>
          <Badge variant="secondary">{infos.length} 提示</Badge>
        </div>
      </div>

      {/* 错误列表 */}
      <ScrollArea className="max-h-[300px]">
        {errors.map((error, i) => (
          <ValidationItem
            key={i}
            error={error}
            onFix={onFixClick}
            variant="error"
          />
        ))}
        {warnings.map((error, i) => (
          <ValidationItem
            key={`w-${i}`}
            error={error}
            onFix={onFixClick}
            variant="warning"
          />
        ))}
        {infos.map((error, i) => (
          <ValidationItem
            key={`i-${i}`}
            error={error}
            onFix={onFixClick}
            variant="info"
          />
        ))}
      </ScrollArea>
    </div>
  );
}
```

#### 3.3.2 单个错误项组件

```tsx
interface ValidationItemProps {
  error: ValidationError;
  onFix?: (error: ValidationError) => void;
  variant: "error" | "warning" | "info";
}

function ValidationItem({ error, onFix, variant }: ValidationItemProps) {
  const icons = {
    error: "❌",
    warning: "⚠️",
    info: "💡"
  };

  return (
    <div className={`p-4 border-b hover:bg-muted/30 ${variant}`}>
      <div className="flex items-start gap-3">
        <span className="text-xl">{icons[variant]}</span>
        <div className="flex-1">
          <div className="font-medium">{error.message}</div>

          {error.location && (
            <div className="text-sm text-muted-foreground mt-1">
              位置: {error.location.type === "code" ? "代码" : "Schema"}
              {error.location.line && ` @ 第 ${error.location.line} 行`}
              {error.location.column && ` 第 ${error.location.column} 列`}
            </div>
          )}

          {error.field && (
            <div className="text-sm font-mono bg-muted px-2 py-1 rounded mt-1 inline-block">
              {error.field.path || error.field.name}
            </div>
          )}

          {error.fixSuggestion && (
            <div className="mt-2 p-2 bg-muted rounded text-sm">
              <div className="font-medium mb-1">💡 修复建议</div>
              <div>{error.fixSuggestion.description}</div>
              {error.fixSuggestion.example && (
                <pre className="mt-2 p-2 bg-background rounded overflow-x-auto">
                  {error.fixSuggestion.example}
                </pre>
              )}
            </div>
          )}

          {onFix && error.fixSuggestion?.action !== "none" && (
            <Button
              variant="outline"
              size="sm"
              className="mt-2"
              onClick={() => onFix(error)}
            >
              快速修复
            </Button>
          )}
        </div>
      </div>
    </div>
  );
}
```

### 3.4 方案四：保存时的错误展示改进

#### 3.4.1 当前实现 (page.tsx 第 425-430 行)

```tsx

if (!validationResult.isValid) {
  toast({
    title: "验证警告",
    description: "当前页面存在验证错误，保存后可能无法正常预览",
    variant: "destructive",

```

#### 3.4.2 优化后的实现

```tsx
// 优化后：根据错误数量和类型展示不同信息
if (!validationResult.isValid) {
  const errors = validationResult.errors.filter(e => e.severity === "error");
  const warnings = validationResult.errors.filter(e => e.severity === "warning");

  if (errors.length > 0) {
    toast({
      title: "❌ 保存失败：存在语法错误",
      description: `发现 ${errors.length} 个错误，需要先修复后才能保存。\n${errors[0].message}`,
      variant: "destructive",
      action: errors.length > 1 ? (
        <Button variant="outline" size="sm" onClick={() => showValidationPanel()}>
          查看全部 {errors.length} 个错误
        </Button>
      ) : undefined,
    });
  } else if (warnings.length > 0) {
    toast({
      title: "⚠️ 存在配置不一致",
      description: `发现 ${warnings.length} 个警告，保存后预览可能异常。\n${warnings[0].message}`,
      variant: "warning",
    });
  }
}
```

## 五、修复建议生成规则

### 5.1 各错误类型的建议模板

| 错误类型 | 建议动作 | 描述模板 | 示例 |

// 保持向后兼容
function normalizeValidationResult(result: ValidationResult): ValidationResult {
  return {
    ...result,
    errors: result.errors.map(error => ({
      ...error,
      severity: error.severity ?? "error",  // 默认为 error
      location: error.location ?? undefined,
      field: error.field ?? undefined,
      fixSuggestion: error.fixSuggestion ?? undefined,
    }))
  };
