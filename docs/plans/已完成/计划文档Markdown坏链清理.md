# 计划文档 Markdown 坏链清理

> 状态：进行中
> 日期：2026-07-01

## 背景

`corepack pnpm check:repo` 在 2026-07-01 通过，但除了既有根目录临时/诊断产物 warning 外，又暴露出 9 条计划文档本地 Markdown 链接不存在 warning。它们主要集中在 `docs/plans/进行中/` 与 `docs/plans/已完成/` 的历史引用，说明近期文档迁移或目录重组后，部分相对路径没有同步更新。

该问题不会直接影响业务代码运行，但会降低仓库卫生检查的信噪比，也会让后续自动任务和人工排查难以快速定位有效文档入口。

## 目标

- 识别并修复当前 `check:repo` 暴露的计划文档坏链。
- 区分“既有根目录临时产物 warning”和“应修复的文档引用漂移”。
- 在不整理无关用户改动的前提下，把坏链问题收敛为可验证、可关闭的单独事项。

## 范围

本次范围包含：

- `docs/plans/进行中/` 与 `docs/plans/已完成/` 中被 `check:repo` 命中的本地相对链接。
- 与坏链直接相关的计划文档间交叉引用。
- 自动任务 state 和失败模式知识库中的结论同步。

本次不包含：

- 根目录临时/诊断产物清理。
- `docs/项目文档/` 的内容审计。
- 业务代码、测试代码或运行数据修复。

`corepack pnpm check:repo` 当前命中的坏链如下：

  - `./P0项目管理关键动作强制同步实施计划.md`

## 方案

1. 逐条校验坏链目标是否已经迁移、重命名或归档到其他目录。
2. 对仍存在的目标，修正相对路径；对已删除或已归档且不再保留的目标，改为新的权威文档入口或删除失效引用。
3. 修复后重新运行 `corepack pnpm check:repo`，确认 warning 只剩根目录临时/诊断产物或其他已知非阻塞项。
