# 画布多文档合并节点

## 背景

创作端画布已有文档节点能力，文档正文同源于工作空间知识库。当前文档节点是一节点一文档，用户在画布中整理多个相关文档时，需要把多个文档节点合并为一个带目录的聚合文档节点，并在节点内切换查看不同文档。

## 目标

- 支持在画布中多选文档节点并合并为一个聚合文档节点。
- 聚合节点只保存多个知识库文档引用，不复制或拼接正文。
- 聚合节点左侧显示目录，右侧显示当前目录项对应的 Markdown 文档内容。
- 合并后移除原画布文档节点，但保留原知识库文档。
- 保持历史单文档节点、画布布局持久化和知识库同步兼容。

## 范围

- 修改共享画布组件、画布类型、画布布局解析器、author-site 组件测试和项目文档。
- 不新增新的知识库 API，不创建新的合并 Markdown 文件。
- 不实现目录项手动排序，目录顺序按合并时画布位置确定。

## 方案

- 在 `CanvasDocumentNode` 上新增聚合文档引用字段和当前激活文档 ID。
- 在 `PreviewCanvas` 中新增文档节点多选状态，支持 Shift/Cmd 点击和框选文档节点。
- 在文档多选时展示合并按钮，合并后创建聚合节点、删除原节点并记录隐藏知识文档 ID。
- 在 `CanvasFreeNodeItem` 中为聚合文档节点渲染左侧目录和当前文档正文，并把目录切换回写到节点状态。
- 更新画布布局解析器，使新字段可被 API 和 workspace 文件读写。
- 补充组件测试和 parser 测试，更新长期项目文档。
