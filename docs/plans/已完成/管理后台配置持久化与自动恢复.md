# 管理后台配置持久化与自动恢复

## 背景

管理后台的全局模型配置保存在 `DATA_DIR/users.db`，但 agent-service 接收全局供应商配置后只保存在内存中。服务重启或重新部署后，如果数据卷不稳定会导致配置丢失；即使数据库仍在，agent-service 也可能退回环境变量配置，需要人工重新保存或推送。

## 目标

- author-site 重启后继续读取数据库中的管理后台配置。
- agent-service 重启后由 author-site 自动恢复全局供应商配置。
- 管理后台区分“已保存到数据库”和“已同步到 agent-service”。
- 部署文档明确 `/app/data` 持久化和备份边界。

## 范围

- 修改 author-site 管理后台模型配置同步链路。
- 增加启动自动同步和失败重试。
- 更新管理后台 UI 的同步状态展示。
- 更新项目文档和部署说明。
- 不引入外部数据库，不把动态配置反写 `.env`。

## 方案

将 SQLite 作为近期事实源，author-site 保存配置后先落库，再推送 agent-service。推送失败不影响保存，并由统一同步服务记录状态和安排有限重试。author-site 启动后读取数据库中的 `backendProviders`，延迟推送给 agent-service，覆盖 agent-service 的环境变量 fallback。
