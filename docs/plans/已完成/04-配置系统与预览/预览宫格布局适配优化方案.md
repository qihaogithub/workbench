**字符串解析（PreviewGrid.tsx）**：

```typescript
function parseSizeValue(value: string | number | undefined): number | null {
  if (typeof value === "number") return value
  if (typeof value === "string") {
    const num = parseFloat(value.replace(/px$/, ""))
    return isNaN(num) ? null : num
  }
  return null
}

function getPreviewAspectRatio(size?: PreviewSize): string {
  const effective = getEffectivePreviewSize(size)
  const w = parseSizeValue(effective.width)
  const h = parseSizeValue(effective.height)
  if (w && h) return `${w}/${h}`
  return "375/812"
}

function getAspectRatioValue(size?: PreviewSize): number {
  const ratio = getPreviewAspectRatio(size)
  const [w, h] = ratio.split("/").map(Number)
  return w / h
}
```

**布局渲染（PreviewGrid.tsx）**：

```tsx
const rows: GridPageItem[][] = []
for (let i = 0; i < demoPages.length; i += gridColumns) {
  rows.push(demoPages.slice(i, i + gridColumns))
}

return (
  <div ref={containerRef} className="h-full overflow-y-auto preview-grid-scroll"
    style={{ scrollbarWidth: "none", msOverflowStyle: "none" }}>
    <style>{`.preview-grid-scroll::-webkit-scrollbar { display: none; }`}</style>
    <div className="min-h-full p-4 flex flex-col"
      style={{ justifyContent: alignmentMode === "center" ? "center" : "flex-start" }}>
      <div ref={gridMeasureRef} style={{ display: "flex", flexDirection: "column", gap: "16px" }}>
        {rows.map((row) => {
          const ratios = row.map((p) => {
            const size = p.id === activePageId ? previewSize : (p.previewSize ?? previewSize)
            return getAspectRatioValue(size)
          })
          const columnTemplate = ratios.map((r) => `${r}fr`).join(" ")
          return (
            <div key={row.map((p) => p.id).join("-")} style={{
              display: "grid",
              gridTemplateColumns: columnTemplate,  // 例如: "0.462fr 1.333fr"
              gap: "16px",
              alignItems: "start",
            }}>
              {row.map((page) => {
                const effectiveSize = page.id === activePageId ? previewSize : (page.previewSize ?? previewSize)
                return <Card key={page.id} aspectRatio={getPreviewAspectRatio(effectiveSize)} ... />
              })}
            </div>
          )
        })}
      </div>
    </div>
  </div>
)
```

## 二、核心适配规则

### 2.1 宽度比例规则

**原则**：同一行中每张卡片的宽度与其 `previewSize.width / previewSize.height`（宽高比）成正比。

**数学原理**：

```
同行 2 张卡片，宽高比分别为 r1、r2
→ gridTemplateColumns: r1fr r2fr
→ 每行实际宽度 = Σ(cardHeight / cardRatio)
→ cardHeight = availableWidth / Σratio
→ cardWidth = cardHeight × ratio = availableWidth × ratio / Σratio
→ 每张卡片宽度与 ratio 成正比 ✅
```

**示例**：同一行 iPad（1024×768，比值 1.333）和手机（375×812，比值 0.462）
```
→ gridTemplateColumns: 1.333fr 0.462fr
→ iPad 卡片宽度占比: 1.333 / (1.333+0.462) = 74.3%
→ 手机卡片宽度占比: 0.462 / (1.333+0.462) = 25.7%
```

### 2.2 高度规则

**原则**：同一行所有卡片等高，不受内容/尺寸差异影响。

**实现**：CSS Grid 默认 `align-items: stretch`，同行卡片自然等高。无需额外约束。

### 2.3 垂直对齐规则

**判断逻辑**：

```
if (gridScrollHeight + padding < containerClientHeight) {
  align = 'center'   // 居中显示
} else {
  align = 'top'      // 顶部对齐 + 滚动
}
```

**实现方式**：
- 外层 `flex-col` wrapper
- `ResizeObserver` 同时观察滚动容器和 grid 元素高度
- 动态切换 `justify-content: center | flex-start`

### 2.4 活跃页面规则

**原则**：活跃页面（`activePageId`）使用 grid-level `previewSize`（用户当前选中的尺寸配置），而非页面自身的 `previewSize`。

**原因**：`previewSize` 是用户在配置面板中设置的当前选中页面的尺寸，切换页面时会更新。用户期望活跃页面使用当前配置尺寸进行渲染。

## 三、具体实现方案

### 3.1 字符串类型宽高解析

`$demo.previewSize` 中的 width/height 可能为字符串（如 `"375px"`，来自 JSON schema），需解析为数值：

```typescript
function parseSizeValue(value: string | number | undefined): number | null {
  if (typeof value === "number") return value
  if (typeof value === "string") {
    const num = parseFloat(value.replace(/px$/, ""))
    return isNaN(num) ? null : num
  }
  return null
}
```

应用场景：
- `getPreviewAspectRatio` — 计算卡片 aspectRatio
- `getAspectRatioValue` — 计算 grid 列宽比例
- `GridIframe` — 计算 iframeWidth / iframeHeight（确定缩放基准）

### 3.2 按行 Grid 渲染

将单一 `gridTemplateColumns: repeat(N, 1fr)` 改为按行独立 Grid：

```typescript
const rows: GridPageItem[][] = []
for (let i = 0; i < demoPages.length; i += gridColumns) {
  rows.push(demoPages.slice(i, i + gridColumns))
}
```

每行渲染为独立的 `display: grid` 容器，列宽按该行卡片的比例分配。

### 3.3 滚动条隐藏

**外层容器**：

```tsx
<div className="h-full overflow-y-auto preview-grid-scroll"
  style={{ scrollbarWidth: "none", msOverflowStyle: "none" }}>
  <style>{`.preview-grid-scroll::-webkit-scrollbar { display: none; }`}</style>
```

**iframe 内部**：

```typescript
function disableIframeScrollbar(iframe: HTMLIFrameElement) {
  try {
    const doc = iframe.contentDocument
    if (!doc) return
    doc.documentElement.style.overflow = "hidden"
    doc.body.style.overflow = "hidden"
  } catch {}
}
```

### 3.4 与 Intersection Observer 共存

- `useAlignmentMode` — 监听 `containerRef`（滚动容器）和 `gridMeasureRef`（grid 外层 flex）
- `useVisiblePages` — 监听 `containerRef` 的子元素 `[data-page-id]`
- 两者监听不同元素，**互不干扰**

## 五、边界情况处理

| 场景 | 行为 |
|:-----|:-----|
| 0 个页面 | Grid 为空 → scrollHeight = 0 < containerHeight → 居中 ✅ |
| 页面数量使 Grid 高度 > 容器 | alignmentMode = "top" → `justify-content: flex-start` → 可滚动 ✅ |
| 切换 gridColumns（2→3→4） | Grid 布局重排 → 高度变化 → ResizeObserver 触发 → 重新判断 ✅ |
| 页面懒加载（visiblePages 变化） | Grid 内 iframe mount/unmount → Grid 高度不变（占位 div 保持 aspectRatio）→ 不影响对齐判断 ✅ |
| 窗口 resize | 容器高度变化 → ResizeObserver 触发 → 判断对齐 ✅ |
| 内容极少（1 页 1 行） | Grid 高度小 → center 模式 → 居中 ✅ |
| 内容溢出（超过容器） | top 模式 → `flex-start` + 外层 `overflow-y-auto` → 滚动 ✅ |
| previewSize 为字符串类型 | `parseSizeValue` 解析 "375px" → 375，正确计算比例 ✅ |
| previewSize 为 number 类型 | `parseSizeValue` 直接返回数值 ✅ |
| 活跃页面 | 优先使用 grid-level previewSize ✅ |
| 非活跃页面 | 使用各自页面的 previewSize，无则回退到 grid-level ✅ |

## 八、向后兼容

- 所有现有 props 保持不变
- 对齐逻辑完全内部实现，外部调用者无感知
- 不影响 `PreviewPanel` 单页模式
- 不影响 `useVisiblePages`、`compile-cache`、`GridIframe` 等内部模块
- 所有页面 previewSize 相同时，`0.462fr 0.462fr` 等价于 `1fr 1fr`，视觉效果与原实现一致
- `previewSize` 为可选字段，未设置时回退到 grid-level previewSize
