# 单页预览右侧属性面板优化方案

> 状态：已完成（2026-06-29）

## 背景

创作端项目编辑页当前在单页面预览中提供批注、文本编辑和可视化配置化能力。现有入口偏工具模式：用户需要先理解“批注模式”和“配置化模式”，再到预览区点选元素完成操作。

新的产品方向是把这些能力收敛到类似 Figma 的右侧属性面板中。用户切换到属性面板后，在单页预览中点击页面位置，由系统识别该位置下的点击图层，再在右侧面板选择对象、编辑属性、预览效果，并在确认后发送给 AI 正式修改。

- [预览系统需求文档](../../../项目文档/创作端/04-配置与预览/预览系统_需求文档.md)
- [可视化批注与编辑机制](../../../项目文档/创作端/04-配置与预览/技术/06_可视化批注与编辑机制.md)

## 目标

- 将批注、文本编辑、样式调整、图片替换和配置化统一到右侧属性面板。
- 降低用户对“模式”的理解成本，让用户形成“点击元素，右侧编辑”的心智。
- 支持临时修改实时预览，确认视觉效果后再发送给 AI 正式落地。
- 支持任意可编辑属性标记为配置项，由 AI 在正式修改时同步处理代码和 Schema。
- 保留现有安全边界：临时预览不直接写回源码，不直接写 Schema。

## 范围

### 本方案覆盖

- 单页面预览模式下的右侧栏信息架构。
- 点击位置图层识别后的属性面板体验。
- 文本、样式、图片和资源的临时编辑与实时预览。
- 属性设为配置项的交互规则。
- 将一项或多项属性变更发送给 AI 的产品流程。

### 本方案不覆盖

- 画布工作台中的批注或属性编辑。
- 历史记录 Tab。
- AI 具体代码生成策略和后端接口细节。
- 多人协同下的属性面板冲突处理。

## 核心产品定义

单页预览右侧栏保留两个 Tab：

| Tab | 定位 |
| :--- | :--- |
| 配置 | 管理当前页面已有配置项，继续承载配置表单和实时配置预览 |
| 属性 | 管理当前点击位置识别到的图层，并编辑选中对象的属性 |

用户心智：

> 我点击页面上的一个位置，右侧列出这个位置下所有可编辑对象；我选择一个对象，调整属性，预览没问题后让 AI 写进项目。

## 右侧属性 Tab 结构

### 1. 当前点击图层

切换到属性 Tab 后，点击预览区，右侧属性 Tab 顶部展示该点击位置命中的对象层级。它不是全页面元素清单，只是本次点击点下方从外层到内层的可编辑图层。

排列建议：

- 从外层到内层展示，便于理解页面结构。
- 默认选中最具体、最可编辑的元素，例如文本、图片、按钮。
- 鼠标悬停图层项时，预览区同步高亮对应对象。
- 点击图层项后，预览区保留选中框。

元素项展示信息：

| 信息 | 说明 |
| :--- | :--- |
| 元素类型 | 如 Section、Card、Image、Title、Button、Text |
| 内容摘要 | 文本摘要、图片文件名、按钮文案或容器尺寸 |
| 视觉线索 | className 摘要、尺寸、背景色或图片缩略图 |
| 可编辑能力 | 文本、图片、样式、布局、链接、配置项 |

### 2. 元素预览

选中元素后展示轻量预览：

- 文本元素：展示完整文本内容。
- 图片元素：展示图片缩略图。
- 按钮元素：展示按钮文案和链接。
- 容器元素：展示尺寸、背景、边框和圆角摘要。

### 3. 属性编辑区

属性按能力分组，避免把所有 CSS 属性平铺给用户。

| 分组 | 可编辑项 |
| :--- | :--- |
| 文本 | 内容、颜色、字号、字重、行高、对齐 |
| 图片 | 缩略图、替换图片 |
| 外观 | 背景、边框、圆角、透明度 |
| 尺寸 | 宽度、高度、最小宽高 |
| 间距 | 内边距、外边距、元素间距 |
| 布局 | 方向、对齐、分布、换行 |
| 链接 | href、打开方式 |

每个属性行需要提供：

- 当前值。
- 可编辑控件。
- 单项恢复。
- 更多菜单。

更多菜单至少包含：

- 设为配置项。
- 复制当前值。
- 发送该属性给 AI 调整。

### 4. 底部发送区

属性 Tab 底部固定展示本次临时变更摘要和操作按钮。

内容：

- 已修改属性数量。
- 已标记配置项数量。
- 清空本次修改。
- 主按钮：发送给 AI。

规则：

- 没有修改时，发送按钮置灰。
- 有未发送修改时，切换页面、刷新预览、切换画布前需要确认。
- 发送中禁止继续编辑，避免本次变更内容漂移。

## 交互流程

### 常规属性修改

1. 用户在单页预览中点击某个位置。
2. 系统识别点击位置下的对象层级，右侧属性 Tab 展示当前点击图层。
3. 用户选择目标元素。
4. 用户修改一个或多个属性。
5. 预览区立即应用临时效果。
6. 用户确认效果后点击右侧底部“发送给 AI”。
7. AI 根据结构化变更正式修改代码。
8. 修改完成后刷新预览，并清空本次临时变更。

### 文本修改

文本修改不再隐藏在批注浮层中，而是作为属性面板的一等能力。

用户可修改：

- 文本内容。
- 文本颜色。
- 字号、字重、行高。
- 文本对齐。

产品规则：

- 输入后预览区实时展示新文本。
- 临时文本不直接写回源码。
- 发送给 AI 时携带原文本、新文本、目标元素信息和页面文件位置。
- 若后续保留“唯一文本自动写回”能力，也必须经过用户确认，不作为属性输入的默认行为。

### 图片与资源替换

图片元素需要按资源面板设计。

支持：

- 展示当前图片缩略图。
- 上传或选择替换图片。

规则：

- 替换后预览区立即展示新图片。
- 预览替换只影响当前 iframe。
- 发送给 AI 后，由 AI 正式处理资源引用、代码替换和必要的配置默认值。
- 图片地址和图片描述不在属性面板中直接编辑，避免非技术用户处理资源路径或无关字段。

### 属性设为配置项

配置化不再是单独模式，而是属性行上的操作。

流程：

1. 用户修改或选中某个属性。
2. 在属性行更多菜单中选择“设为配置项”。
3. 右侧出现配置项设置区域。
4. 用户填写配置名称、字段 key 和默认值。
5. 该属性进入本次变更摘要。
6. 用户点击“发送给 AI”后，AI 同时处理属性修改和配置 Schema 更新。

规则：

- “设为配置项”本身不立即写 Schema。
- 同一次发送可以包含多个配置项。
- 若字段 key 与已有项目级配置或页面级配置冲突，发送前提示用户调整。
- 发送成功后切回“配置”Tab，并高亮新增配置项。

### AI 修改说明

属性面板可以保留一个补充说明输入框，但它不是原来的批注入口。

用途：

- 用户对本次属性调整补充自然语言说明。
- 用户不清楚具体属性时，可以只选中元素并描述想要的效果。

规则：

- 有明确属性变更时，AI 以结构化属性变更为主。
- 自然语言说明只作为补充上下文。
- 没有属性变更但有说明时，发送按钮可用，视为“针对选中元素的 AI 修改请求”。

## 实时预览规则

临时修改通过预览 iframe 应用，不直接改变页面源码、配置 Schema 或项目数据。

规则：

- 修改任意属性后，预览区立即展示效果。
- 修改多个属性时，预览效果叠加展示。
- 切换同一点击位置下的元素时，本次临时变更保留。
- 切换页面、刷新、退出单页预览、进入画布前，如果存在未发送修改，需要确认丢弃。
- AI 正式修改完成并刷新预览后，临时变更清空。
- AI 修改失败时，保留临时变更和发送内容，允许用户重试或清空。

## 发送给 AI 的结构化内容

点击“发送给 AI”时，前端需要提交一组结构化上下文，而不是只提交自然语言。

建议包含：

| 内容 | 说明 |
| :--- | :--- |
| 页面上下文 | 当前项目、页面 ID、页面文件路径 |
| 点击上下文 | 点击坐标、命中的点击图层、用户最终选择的对象 |
| 元素信息 | tagName、domPath、nodeId、className、文本、图片地址、尺寸 |
| 属性变更 | 每项属性的原值、新值、属性类型、是否临时预览成功 |
| 配置项标记 | 字段名称、字段 key、默认值、配置作用域 |
| 补充说明 | 用户输入的自然语言说明 |

AI 修改原则：

- 优先只修改当前页面相关代码。
- 优先围绕用户选择的元素和属性变更处理。
- 需要改动其他文件时，先在响应中说明原因。
- 配置项新增必须同步处理 Schema、默认值和预览数据。

## 状态与边界

### 面板状态

| 状态 | 展示 |
| :--- | :--- |
| 未点击预览 | 提示用户点击预览区选择元素 |
| 已点击但无可编辑元素 | 展示无法识别，并建议换一个位置 |
| 已选择元素 | 展示点击图层、当前对象和属性编辑 |
| 有临时变更 | 底部展示变更摘要和发送按钮 |
| 发送中 | 锁定属性编辑，展示发送状态 |
| AI 修改完成 | 刷新预览，清空临时变更 |
| AI 修改失败 | 保留变更，允许重试 |

### 与顶部工具栏的关系

单页预览顶部只保留预览控制：

- 单页 / 画布切换。
- 页面选择。
- 尺寸。
- 刷新。

不再保留：

- 批注按钮。
- 配置化按钮。
- 发送批注按钮。

### 与配置 Tab 的关系

- 配置 Tab 管理已经存在的配置项。
- 属性 Tab 管理当前选中元素的临时属性修改。
- 属性被设为配置项并由 AI 成功落地后，配置 Tab 展示新配置项。
- 属性 Tab 不直接编辑已有配置项的值；已有配置项仍回到配置 Tab 调整。

## 风险与待确认事项

| 风险 | 说明 | 建议 |
| :--- | :--- | :--- |
| DOM 定位不等于源码定位 | 点击位置元素可描述当前渲染结果，但不保证源码唯一 | 发送给 AI 时提供完整点击图层和上下文，避免自动直接写回 |
| 属性太多导致面板复杂 | 如果暴露过多 CSS 属性，会变成低效开发工具 | 常用属性使用组合控件展示，可选视觉效果在未生效时只显示添加入口 |
| 临时预览与 AI 结果不一致 | AI 正式修改后可能无法完全复现临时效果 | 发送结构化变更，并在 AI 完成后刷新预览让用户确认 |
| 图片替换涉及资源保存 | 上传图片需要明确进入项目资源、临时缓存或外链 | 实施前补充资源处理策略 |
| 配置项命名冲突 | 字段 key 可能与页面级或项目级配置冲突 | 发送前做 key 校验和命名建议 |
| 多元素叠加选择成本 | 点击复杂区域可能命中很多元素 | 默认选中最具体可编辑元素，并提供预览高亮辅助选择 |
| 本地原生依赖可能随 Node 切换失效 | `better-sqlite3` 需要与当前 Node ABI 匹配 | 切换 Node 版本后使用带 `distutils` 的 Python 重新 rebuild，再执行 `corepack pnpm check:author` |
