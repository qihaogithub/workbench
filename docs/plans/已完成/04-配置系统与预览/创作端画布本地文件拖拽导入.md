# 创作端画布本地文件拖拽导入

## 背景

创作端画布已经支持页面卡片、文档节点和图片节点，也已有通过工具栏创建文档、通过图片拖入或粘贴创建图片的能力。当前缺口是从本地直接拖入 Markdown 文档时无法创建文档节点，且拖入图片时落点仍按视口中心计算，不够符合拖拽语义。

## 目标

- 支持用户从本地拖拽 Markdown 文档到画布窗口，创建可编辑的文档节点。
- 保持从本地拖拽图片到画布窗口创建图片节点的能力，并让新节点尽量出现在拖放位置。
- 图片节点不添加额外背景色，节点外框缩放时始终保持图片原始比例。
- 保持画布状态通过现有 `.canvas-layout.json` 自动保存链路持久化。

## 范围

- 共享画布组件 `PreviewCanvas` 的拖拽、粘贴和节点创建逻辑。
- 创作端画布相关单元测试。
- `docs/项目文档/创作端/04-配置与预览/` 下的预览需求与技术说明。

不涉及上传文件到服务器、项目工作区文件树导入、外部网页节点恢复或图片资源服务化。

## 方案

- 将画布拖放入口从“只识别图片”扩展为“识别图片与 Markdown 文件”。
- 图片文件继续读取为 Data URL 并创建图片节点；Markdown 文件读取文本内容并创建文档节点。
- 拖放事件记录屏幕坐标，转换为画布坐标后作为新增节点位置；粘贴入口保留视口中心落点。
- 对文件 MIME 和扩展名做兼容判断，覆盖常见 `.md`、`.markdown`、`.mdown` 文件。

## 风险与待确认事项

- 图片仍以 Data URL 存入画布布局，较大图片会增加 `.canvas-layout.json` 体积；本次不改变既有存储策略。
- 浏览器对本地 Markdown 文件的 MIME 识别不完全一致，需要同时使用扩展名兜底。
