## 一、功能概述

在编辑页右侧预览区实现两种展示模式的切换：

| 模式 | 说明 | 适用场景 |
|:-----|:-----|:---------|
| **单页模式**（默认） | 全尺寸渲染当前选中页面，与现有行为一致 | 精细查看、配置调试 |
| **宫格模式** | 所有页面以缩略卡片排列，每个卡片内是 iframe 实时渲染 | 全局浏览、快速定位、对比风格 |

两种模式共享 `activePageId` 状态，切换时保持选中页面不变。

## 二、现状分析

### 2.1 当前预览架构

```
PreviewPanel (packages/web/components/demo/PreviewPanel.tsx)
  │
  ├─ 接收 props: code / sessionId / demoId / configData / previewSize
  ├─ POST /api/compile → sucrase 编译 + CDN 路径替换
  ├─ generateIframeHtml() → Blob URL → <iframe src>
  ├─ postMessage("UPDATE_CODE") → iframe 加载编译产物
  └─ postMessage("UPDATE_CONFIG") → iframe 重新渲染（不重编译）
```

**关键事实：**
- 预览**不是 Sandpack**，是 `iframe + /api/compile` 模式
- 每个页面编译后产生一个 Blob URL，iframe 通过 `import(moduleUrl)` 动态加载
- 配置变更走 `UPDATE_CONFIG` 消息，不触发重编译
- iframe 通过 `window.__DEMO_PROPS__` 接收合并后的 Props

## 四、技术方案

### 4.1 状态模型

```typescript
interface PreviewState {
  mode: 'single' | 'grid'
  activePageId: string
  gridColumns: 2 | 3 | 4
}
```

状态管理原则：
- 切换到单页模式：保留 `activePageId`
- 切换到宫格模式：`activePageId` 不变，`gridColumns` 恢复上次值（默认 3）
- 点击页面列表：无论哪种模式，都更新 `activePageId` 并触发滚动或渲染

### 4.2 宫格渲染方案：CSS Grid + iframe 缩放 + Intersection Observer

每个宫格卡片内部是一个 `<iframe>`，使用 `transform: scale(factor)` 缩放到格子尺寸。

**缩放计算：**

```
scale = 卡片实际宽度 / iframe 设计宽度
iframe 实际尺寸 = 卡片实际宽度 / scale × 100%（反向放大）
```

**核心渲染结构：**

```tsx
<div
  style={{
    display: 'grid',
    gridTemplateColumns: `repeat(${columns}, 1fr)`,
    gap: '16px',
    overflowY: 'auto',
    padding: '16px',
  }}
>
  {demoPages.map(page => (
    <div
      key={page.id}
      ref={activePageId === page.id ? scrollRef : undefined}
      className={cn(
        'relative rounded-lg overflow-hidden cursor-pointer transition-all',
        activePageId === page.id
          ? 'border-2 border-blue-500 scale-[1.02]'
          : 'border border-gray-200 hover:border-gray-400'
      )}
      style={{ aspectRatio: '16/9' }}
      onClick={() => handleGridCardClick(page.id)}
    >
      <GridIframe
        projectId={projectId}
        sessionId={sessionId}
        page={page}
        visible={visiblePages.has(page.id)}
        hasChanges={changedPageIds.has(page.id)}
      />
    </div>
  ))}
</div>
```

### 4.3 Intersection Observer 驱动的 mount / unmount

**问题**：每个宫格卡片内是一个完整 iframe（持有 React 渲染上下文 + JS 堆 + 事件循环），5 个以上同时 mount 内存压力显著。

**策略**：

1. 视口可见的 iframe + 上下各 1 个 buffer 卡片视为"活跃"，正常 mount
2. 离开活跃区的卡片**直接 unmount**（卸载 iframe 节点 + `URL.revokeObjectURL` 释放 Blob URL），位置仍占据格子但不持有渲染上下文
3. 重新进入活跃区时按需 mount：重新编译 → 生成 Blob URL → 挂载 iframe
4. 同时活跃数量上限 = `gridColumns × (visibleRows + 2 buffer)`，约 9 个左右
5. **不替换为静态截图占位符**——保留实现简单性

**自定义 Hook：**

```typescript
function useVisiblePages(
  containerRef: RefObject<HTMLElement>,
  pages: DemoPageMeta[],
  bufferCount: number = 1
): Set<string> {
  const [visiblePages, setVisiblePages] = useState<Set<string>>(new Set());

  useEffect(() => {
    const container = containerRef.current;
    if (!container) return;

    const observer = new IntersectionObserver(
      (entries) => {
        setVisiblePages((prev) => {
          const next = new Set(prev);
          for (const entry of entries) {
            const pageId = entry.target.getAttribute('data-page-id');
            if (!pageId) continue;
            if (entry.isIntersecting) {
              next.add(pageId);
              // 加入 buffer 区的相邻页面
              const idx = pages.findIndex(p => p.id === pageId);
              for (let i = Math.max(0, idx - bufferCount); i <= Math.min(pages.length - 1, idx + bufferCount); i++) {
                next.add(pages[i].id);
              }
            } else {
              next.delete(pageId);
            }
          }
          return next;
        });
      },
      { root: container, rootMargin: '100% 0px' }
    );

    const cards = container.querySelectorAll('[data-page-id]');
    cards.forEach(card => observer.observe(card));

    return () => observer.disconnect();
  }, [containerRef, pages, bufferCount]);

  return visiblePages;
}
```

### 4.4 GridIframe 组件

每个宫格卡片内的 iframe 渲染组件，负责：
- 根据 `visible` 状态决定 mount / unmount
- 调用 `/api/compile` 编译页面代码
- 生成 Blob URL 加载 iframe
- 离开视口时清理资源

```typescript
interface GridIframeProps {
  projectId: string
  sessionId: string
  page: DemoPageMeta
  visible: boolean
  hasChanges: boolean
}

function GridIframe({ projectId, sessionId, page, visible, hasChanges }: GridIframeProps) {
  const iframeRef = useRef<HTMLIFrameElement>(null)
  const blobUrlRef = useRef<string | null>(null)

  useEffect(() => {
    if (!visible) {
      // unmount：清理 Blob URL，卸载 iframe
      if (blobUrlRef.current) {
        URL.revokeObjectURL(blobUrlRef.current)
        blobUrlRef.current = null
      }
      return
    }

    // mount：编译并加载
    let cancelled = false
    const load = async () => {
      const res = await fetch('/api/compile', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ sessionId, demoId: page.id }),
      })
      const data = await res.json()
      if (cancelled || !data.success) return

      const html = generateIframeHtml({
        compiledCode: data.data.compiledCode,
        cssImports: data.data.cssImports,
      })
      const blob = new Blob([html], { type: 'text/html' })
      const url = URL.createObjectURL(blob)
      if (cancelled) { URL.revokeObjectURL(url); return }
      blobUrlRef.current = url
      // 设置 iframe src
    }
    load()

    return () => { cancelled = true }
  }, [visible, sessionId, page.id])

  if (!visible) {
    // unmount 状态：显示占位（保留格子空间）
    return (
      <div className="w-full h-full bg-gray-50 flex items-center justify-center">
        <span className="text-xs text-gray-400">{page.name}</span>
      </div>
    )
  }

  // mount 状态：渲染 iframe（缩放）
  const scale = 0.3 // 根据实际卡片宽度动态计算
  return (
    <iframe
      ref={iframeRef}
      src={blobUrlRef.current || ''}
      sandbox="allow-scripts allow-same-origin"
      style={{
        width: `${100 / scale}%`,
        height: `${100 / scale}%`,
        transform: `scale(${scale})`,
        transformOrigin: 'top left',
        border: 'none',
        pointerEvents: 'none',
      }}
      title={page.name}
    />
  )
}
```

### 4.5 编译缓存策略

宫格模式中，用户来回滚动会导致同一页面反复编译。需在客户端做短期缓存：

```typescript
// 编译结果缓存（LRU，最大 20 条）
const compileCache = new Map<string, { result: CompileResult; timestamp: number }>()
const MAX_CACHE_SIZE = 20
const CACHE_TTL = 5 * 60 * 1000 // 5 分钟

function getCachedCompile(sessionId: string, demoId: string): CompileResult | null {
  const key = `${sessionId}:${demoId}`
  const cached = compileCache.get(key)
  if (!cached) return null
  if (Date.now() - cached.timestamp > CACHE_TTL) {
    compileCache.delete(key)
    return null
  }
  return cached.result
}

function setCachedCompile(sessionId: string, demoId: string, result: CompileResult) {
  const key = `${sessionId}:${demoId}`
  if (compileCache.size >= MAX_CACHE_SIZE) {
    const oldest = compileCache.keys().next().value
    if (oldest) compileCache.delete(oldest)
  }
  compileCache.set(key, { result, timestamp: Date.now() })
}
```

缓存 key 使用 `sessionId:demoId`，当 sessionId 对应的代码发生变更时（收到 AI 编辑回调），需清除对应 demoId 的缓存。

### 4.6 模式切换流程

```
单页 → 宫格：
  1. 保留当前 activePageId
  2. 初始化 Intersection Observer
  3. 渲染 Grid 布局，可见卡片开始 mount
  4. scrollIntoView 到 activePageId 对应卡片

宫格 → 单页：
  1. 保留当前 activePageId
  2. 断开 Intersection Observer
  3. 清理所有 GridIframe 的 Blob URL
  4. 渲染单页 PreviewPanel
```

## 八、与上游方案的差异说明

上游方案 [项目多Demo页面支持方案.md](../06-项目管理与页面/项目多Demo页面支持方案.md) §4.7 中宫格模式的 iframe 使用 `/embed/{projectId}?page={page.id}` 路径加载。**本开发文档改为直接调用 `/api/compile` + Blob URL**，理由：

1. `/api/embed/...` 路由是服务端生成完整 HTML 并缓存，适合使用端（viewer-site）的只读预览
2. 编辑端需要实时反映代码变更，使用 `/api/compile` + Blob URL 与现有 PreviewPanel 行为一致
3. 避免编辑端依赖 embed 路由的缓存刷新机制

使用端（viewer-site）的宫格模式不在本文档范围内，后续按需实现。

## 九、单页模式预览自适应缩放修复记录

### 9.1 问题描述

在单页模式下，当预览区域宽度较大时，预览内容（iframe）不会自动缩放以适应容器，导致预览显示不完整或被截断。

### 9.2 根因分析

`PreviewPanel` 组件的 `buildPreviewStyle` 函数仅根据 `previewSize` 配置设置固定宽高，未考虑容器实际宽度。当容器宽度小于预览设计宽度时，iframe 会超出容器边界。

### 9.3 修复方案

参考宫格模式（`PreviewGrid`）的缩放机制，为单页模式增加容器宽度感知和自动缩放：

1. **容器宽度监听**：使用 `ResizeObserver` 监听预览容器宽度变化
2. **自动缩放计算**：当容器宽度小于预览设计宽度时，计算 `scale = containerWidth / previewWidth`
3. **样式调整**：应用 `transform: scale()` 并同步调整高度，避免布局溢出
4. **保留手动缩放**：若 `previewSize.scale` 已配置，优先使用手动配置值

### 9.4 涉及代码

- **文件**：`packages/web/components/demo/PreviewPanel.tsx`
- **修改点**：
  - `buildPreviewStyle` 函数增加 `containerWidth` 参数
  - 新增 `containerRef` 和 `ResizeObserver` 监听容器宽度
  - iframe 外层包裹 `<div ref={containerRef}>` 作为测量容器

### 9.5 修复效果

- 预览区域现在会根据容器宽度自动缩放，确保内容完整显示
- 缩放是实时响应的，窗口大小变化时自动重新计算
- 与宫格模式的缩放机制保持一致

### 9.6 后续优化建议

- 考虑为缩放添加平滑过渡动画（`transition: transform 0.2s ease`）
- 在极小宽度下（如 < 200px）可考虑显示占位提示而非强制缩放
- 监控性能影响，ResizeObserver 在大量元素上可能有开销
