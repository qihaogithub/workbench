# 创作端画布缩放拖动失效排查与修复

## 背景

用户反馈创作端画布无法缩放、拖动。画布的事件入口本身仍符合既有规则，但实际使用中缩放和平移后的视口状态会被旧的协同画布布局覆盖，表现为操作没有生效。

## 目标

- 恢复画布缩放的基础可用性。
- 确认画布平移入口没有被回归破坏。
- 保留选择工具下的框选、多选和页面拖拽能力。
- 补充验证记录和项目文档。

## 范围

- `packages/author-site/src/app/demo/[id]/edit/page.tsx`
- `docs/项目文档/创作端/04-配置与预览/技术/08_协同草稿驱动预览.md`

## 方案

- 保持原有画布业务规则不变：选择工具下普通滚轮不缩放，Ctrl/Cmd + 滚轮缩放；抓手工具、空格临时抓手和中键用于平移。
- 调整编辑页中 `.canvas-layout.json` 协同草稿的远端应用逻辑，只在远端文本本身变化时应用远端画布状态。
- 避免本地 `canvasState` 变化触发旧远端草稿重放，导致本地缩放和平移刚生效就被覆盖。
