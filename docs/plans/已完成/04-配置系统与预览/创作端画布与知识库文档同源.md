# 创作端画布与知识库文档同源

## 背景

创作端画布支持添加文档节点，项目知识库也支持添加 Markdown 文档。当前两处文档各自保存，画布文档保存在画布布局状态中，知识库文档保存在工作空间 `knowledge/` 目录和 manifest 中，容易出现内容重复和不同步。

## 目标

- 添加到项目知识库的用户文档自动出现在画布中。
- 添加到画布的文档自动写入项目知识库。
- 两处展示的文档使用同一个知识库文档来源，画布只保留布局与引用。

## 范围

- 创作端项目编辑页的知识库面板、知识文档弹窗与画布联动。
- 共享画布文档节点的数据结构、渲染、编辑与拖拽导入。
- 画布布局 API 对新旧文档节点的解析兼容。
- 知识库模块长期文档同步更新。

## 方案

- 扩展画布文档节点，增加知识库文档引用字段，并保留旧 `markdown` 字段兼容历史布局。
- 画布新增或拖入 Markdown 文件时，先调用知识库创建接口，再把返回的文档条目作为画布节点引用。
- 知识库新增或上传文档后，通过回调把新文档加入画布节点。
- 画布渲染和编辑文档时，从知识库内容 API 读取正文，保存时调用知识库更新接口。
- 知识库删除文档时，画布上引用该文档的节点随下一次列表刷新清理。

## 实施摘要

- 画布文档节点支持保存知识库文档引用。
- 项目编辑页持有知识库 manifest 列表，并将其作为画布文档来源。
- 知识库新增、上传与画布新增、拖入 Markdown 均会汇入同一工作空间知识库来源。
- 画布布局 API 兼容引用型文档节点和历史 markdown 文档节点。
- 画布文档节点支持折叠状态，折叠后收缩到标题高度且仅显示标题，展开时恢复折叠前高度。
- 文档编辑弹窗支持识别 Markdown 纯文本粘贴，并按 Markdown 结构渲染后保存，包含常见表格。
- 文档编辑器和画布文档正文使用极简滚动条样式。
- 删除画布上的知识库文档节点只影响画布展示，不删除知识库原文档，并且不会被自动恢复。
