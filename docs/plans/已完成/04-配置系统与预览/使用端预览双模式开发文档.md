## 一、功能概述

在使用端（viewer-site）预览页实现以下三个目标：

| 目标 | 说明 | 当前状态 |
|:-----|:-----|:---------|
| **预览区 + 配置面板同时显示** | 预览区按 previewSize 比例渲染，右侧始终显示配置面板 | 配置面板仅在 hasConfig 时显示，预览区未按比例渲染 |
| **预览区按 previewSize 比例显示** | 从页面 Schema 中提取 previewSize，控制 iframe 容器尺寸和缩放 | 未使用 previewSize，iframe 宽 100%、高度由 RESIZE 消息决定 |
| **单页/宫格双模式** | 与创作端一致的单页模式和宫格模式切换 | 仅支持单页模式，无宫格模式 |

## 二、现状分析

### 2.1 当前预览页布局

```
┌─ header (h-14) ────────────────────────────────────────────┐
│  ← 返回  │  项目名称  │  [页面Tabs]  │  [配置按钮]          │
├────────────────────────────────────┬───────────────────────┤
│                                    │                       │
│  IframeRenderer                    │  ConfigPanel          │
│  (flex-1, overflow-auto, p-4)     │  (w-80, 可隐藏)       │
│  width: 100%                       │                       │
│  height: 由 RESIZE 消息动态调整    │                       │
│                                    │                       │
└────────────────────────────────────┴───────────────────────┘
```

### 2.2 关键问题

1. **配置面板不可见**：`hasConfig` 为 false 时配置面板不显示，但很多项目有页面级配置却无项目级配置
2. **previewSize 未使用**：IframeRenderer 不支持 previewSize，无法按比例渲染
3. **无宫格模式**：只有单页预览，无法同时浏览所有页面
4. **配置面板只读项目级 Schema**：未读取页面级 Schema，缺少页面级配置项
5. **postMessage 通信方式原始**：通过 `document.querySelector("iframe")` 直接操作，不可靠

### 2.3 创作端 vs 使用端对比

| 维度 | 创作端 (web) | 使用端 (viewer-site) |
|:-----|:------------|:---------------------|
| 预览渲染方式 | `/api/compile` + Blob URL + `UPDATE_CODE` | embed iframe URL（预编译 HTML） |
| previewSize | 从 Schema 提取，控制容器尺寸和缩放 | 未使用 |
| 配置面板 | 项目级 + 页面级 Schema | 仅项目级 Schema |
| 模式切换 | 单页 / 宫格 | 仅单页 |
| 配置通信 | PreviewPanel 内置 postMessage | 外部 querySelector + postMessage |
| 编译缓存 | 客户端 LRU 缓存 | 不需要（embed 路由自带服务端缓存） |

### 3.3 单页模式

预览区按 previewSize 比例居中渲染 iframe：

```
┌─ 预览区 ──────────────────────────────────────┐
│                                                │
│         ┌──────────────┐                       │
│         │   iframe     │  ← 按 previewSize     │
│         │   居中缩放    │     比例渲染          │
│         │              │                       │
│         └──────────────┘                       │
│                                                │
└────────────────────────────────────────────────┘
```

- 如果 previewSize 指定了 width/height，iframe 按该尺寸渲染，超出容器时等比缩放
- 如果未指定 previewSize，使用默认 375×812（移动端）或自适应（桌面端）

## 四、技术方案

### 4.1 状态模型

```typescript
interface ViewerPreviewState {
  mode: 'single' | 'grid'
  activePageId: string
  gridColumns: 2 | 3 | 4
  configVisible: boolean
}
```

### 4.2 获取 previewSize

**问题**：当前 `GET /api/projects/:id/demos` 只返回 `DemoPageMeta[]`（id/name/order），不包含 previewSize。

**方案**：扩展该 API，为每个页面返回 `previewSize`。

修改 `listDemoPages()` 函数，在返回 `DemoPageMeta` 时额外读取每个页面的 `config.schema.json`，提取 `$demo.previewSize` 或 `ui.options.preview`。

**API 返回格式变更**：

```typescript
// 扩展前
{ demoPages: DemoPageMeta[] }

// 扩展后
{ demoPages: (DemoPageMeta & { previewSize?: PreviewSize })[] }
```

**提取逻辑**（复用创作端 `getPreviewSize()` 函数）：

```typescript
function extractPreviewSize(schemaStr: string): PreviewSize | undefined {
  try {
    const schema = JSON.parse(schemaStr)
    // 优先读取 $demo.previewSize
    if (schema.$demo?.previewSize) {
      return schema.$demo.previewSize
    }
    // 向后兼容 ui.options.preview
    if (schema.ui?.options?.preview) {
      const p = schema.ui.options.preview
      return {
        width: p.width,
        height: p.height,
        minHeight: p.minHeight,
        maxHeight: p.maxHeight,
        scale: p.scale ? Number(p.scale) : undefined,
      }
    }
  } catch {}
  return undefined
}
```

### 4.3 获取页面级配置 Schema

**问题**：当前 viewer-site 只获取项目级配置 Schema（`GET /api/projects/:id/config`），不获取页面级配置 Schema。

**方案**：扩展 `GET /api/projects/:id/demos` API，同时返回每个页面的配置 Schema。

```typescript
// 扩展后
{
  demoPages: (DemoPageMeta & {
    previewSize?: PreviewSize
    schema?: string  // 页面级 config.schema.json 内容
  })[]
}
```

这样 viewer-site 可以：
1. 从 `previewSize` 获取预览尺寸
2. 从 `schema` 获取页面级配置字段
3. 合并项目级 + 页面级 Schema 生成完整配置面板

### 4.4 单页模式改造：IframeRenderer 支持 previewSize

改造 `IframeRenderer` 组件，增加 `previewSize` 属性：

```typescript
interface IframeRendererProps {
  src: string
  previewSize?: PreviewSize
  onReady?: () => void
  onLoaded?: () => void
  onError?: (error: string) => void
  onResize?: (height: number) => void
  className?: string
}
```

**渲染逻辑**：

1. 如果有 previewSize 且指定了 width/height：
   - iframe 设置为 previewSize 的 width × height
   - 外层容器居中显示
   - 如果 iframe 尺寸超过容器，使用 `transform: scale(factor)` 等比缩放
2. 如果没有 previewSize：
   - 保持现有行为（width: 100%，高度由 RESIZE 消息决定）

### 4.5 宫格模式：ViewerPreviewGrid 组件

新建 `packages/viewer-site/src/components/viewer-preview-grid.tsx`。

与创作端 `PreviewGrid` 的关键区别：

| 维度 | 创作端 PreviewGrid | 使用端 ViewerPreviewGrid |
|:-----|:------------------|:------------------------|
| iframe 加载方式 | `/api/compile` + Blob URL | embed iframe URL |
| 编译缓存 | 客户端 LRU 缓存 | 不需要（服务端缓存） |
| 配置更新 | Blob URL 重建 | postMessage UPDATE_CONFIG |
| 依赖 | sessionId | projectId |

**核心结构**：

```tsx
function ViewerPreviewGrid({
  projectId,
  demoPages,
  activePageId,
  gridColumns,
  onCardClick,
  configData,
}: ViewerPreviewGridProps) {
  const containerRef = useRef<HTMLDivElement>(null)
  const visiblePages = useVisiblePages(containerRef, demoPages)

  return (
    <div ref={containerRef} className="h-full overflow-y-auto p-4">
      <div style={{
        display: 'grid',
        gridTemplateColumns: `repeat(${gridColumns}, 1fr)`,
        gap: '16px',
      }}>
        {demoPages.map(page => (
          <div
            key={page.id}
            data-page-id={page.id}
            style={{ aspectRatio: getAspectRatio(page.previewSize) }}
            className={cn(
              'relative rounded-lg overflow-hidden cursor-pointer transition-all',
              activePageId === page.id
                ? 'border-2 border-primary ring-2 ring-primary/20 scale-[1.02]'
                : 'border border-border hover:border-primary/50'
            )}
            onClick={() => onCardClick(page.id)}
          >
            <ViewerGridCard
              projectId={projectId}
              page={page}
              visible={visiblePages.has(page.id)}
              configData={configData}
            />
          </div>
        ))}
      </div>
    </div>
  )
}
```

**ViewerGridCard 组件**：

每个卡片内是一个 embed iframe，使用 CSS transform scale 缩放：

```tsx
function ViewerGridCard({
  projectId,
  page,
  visible,
  configData,
}: ViewerGridCardProps) {
  const iframeRef = useRef<HTMLIFrameElement>(null)
  const [cardWidth, setCardWidth] = useState(0)
  const wrapperRef = useRef<HTMLDivElement>(null)

  // 监听卡片宽度变化
  useEffect(() => {
    const el = wrapperRef.current
    if (!el) return
    const ro = new ResizeObserver(entries => {
      for (const entry of entries) {
        setCardWidth(entry.contentRect.width)
      }
    })
    ro.observe(el)
    return () => ro.disconnect()
  }, [])

  // 配置变更时发送 postMessage
  useEffect(() => {
    if (!configData || !iframeRef.current?.contentWindow) return
    iframeRef.current.contentWindow.postMessage(
      { type: 'UPDATE_CONFIG', configData },
      '*'
    )
  }, [configData])

  if (!visible) {
    return (
      <div ref={wrapperRef} className="w-full h-full bg-muted/50 flex items-center justify-center">
        <span className="text-xs text-muted-foreground">{page.name}</span>
      </div>
    )
  }

  const iframeWidth = page.previewSize?.width || 375
  const iframeHeight = page.previewSize?.height || 812
  const scale = cardWidth > 0 ? cardWidth / (typeof iframeWidth === 'number' ? iframeWidth : 375) : 0.3

  return (
    <div ref={wrapperRef} className="relative w-full h-full overflow-hidden">
      <iframe
        ref={iframeRef}
        src={getEmbedIframeUrl(projectId, page.id)}
        sandbox="allow-scripts allow-same-origin"
        style={{
          width: typeof iframeWidth === 'number' ? iframeWidth : 375,
          height: typeof iframeHeight === 'number' ? iframeHeight : 812,
          transform: `scale(${scale})`,
          transformOrigin: 'top left',
          border: 'none',
          pointerEvents: 'none',
        }}
        title={page.name}
      />
    </div>
  )
}
```

### 4.6 Intersection Observer 驱动的 mount / unmount

复用创作端的 `useVisiblePages` Hook 逻辑。视口可见的 iframe + 上下各 1 个 buffer 卡片视为"活跃"，离开活跃区的卡片直接 unmount iframe 节点。

### 4.7 配置面板改造

**当前问题**：
1. 只读取项目级 Schema，不读取页面级 Schema
2. hasConfig 为 false 时不显示

**改造方案**：

1. 始终显示配置面板（无配置项时显示空状态提示）
2. 合并项目级 + 页面级 Schema 的配置字段
3. 配置变更通过 IframeRenderer 的 ref 发送 postMessage（替代 querySelector）

**配置面板布局**：

```
┌─ 配置面板 ──────────────────────┐
│  ⚙️ 配置面板                    │
├─────────────────────────────────┤
│  📋 项目配置（所有页面共享）      │  ← 项目级 Schema 字段
│  [字段1] [字段2] ...            │
│  ─────────────────────          │
│  📄 当前页面配置                 │  ← 页面级 Schema 字段
│  [字段1] [字段2] ...            │
└─────────────────────────────────┘
```

### 4.8 模式切换流程

```
单页 → 宫格：
  1. 保留当前 activePageId
  2. 初始化 Intersection Observer
  3. 渲染 Grid 布局，可见卡片开始 mount
  4. scrollIntoView 到 activePageId 对应卡片

宫格 → 单页：
  1. 保留当前 activePageId
  2. 断开 Intersection Observer
  3. 渲染单页 IframeRenderer
```

## 七、风险与对策

| 风险 | 对策 |
|:-----|:-----|
| 多 iframe 内存压力 | Intersection Observer 驱动 mount/unmount，最大并发约 9 个 |
| embed iframe 加载延迟 | 服务端 5 分钟缓存，首次加载后后续访问快 |
| 缩放后 iframe 交互失效 | 宫格模式 iframe 设置 `pointerEvents: 'none'`，交互通过卡片点击触发 |
| previewSize 未定义 | 默认使用 375×812（移动端比例） |
| 项目级 + 页面级 Schema 字段冲突 | 使用端为只读，冲突时页面级覆盖项目级 |
| demos API 返回数据量增大 | `includeSchema` 为可选参数，默认不返回 schema 内容 |
