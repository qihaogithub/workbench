# 配置面板公共与独有配置项分离显示方案

> 版本：v2.0
> 创建日期：2026-05-10
> 状态：方案设计中

---

## 一、问题背景

当前配置面板已支持项目级配置（所有页面共享）与页面级配置（当前页面独有）的分开展示，但存在以下体验问题：

1. **视觉层级弱**：项目配置与页面配置仅通过一段文字标签区分（"📋 项目配置（所有页面共享）" / "📄 当前页面配置"），缺乏强烈的视觉边界感
2. **作用域不明确**：用户无法一眼识别某个字段的修改会影响所有页面还是仅当前页面
3. **分组混杂**：项目配置和页面配置内部都使用同一套智能分组逻辑（颜色配置、尺寸设置等），导致两个层级的分组标题重复，视觉上难以区分
4. **标签不一致**：编辑页使用 Emoji 标签（📋📄），查看器页面使用纯文字标签（"项目配置"/"页面配置"），风格不统一

**典型用户困惑场景**：

> 用户在"基础配置"分组下修改了"页面标题"，切换到另一个页面后发现标题没变，以为系统出 Bug。实际上"页面标题"是页面级字段，但用户无法从 UI 上直观感知这一点。

---

## 二、设计目标

| 目标 | 说明 |
|:-----|:-----|
| **明确作用域** | 用户一眼即可识别配置属于"全局共享"还是"页面独有" |
| **视觉分层** | 项目配置与页面配置在视觉上有清晰的边界区分 |
| **减少困惑** | 消除"为什么改了这里，那里没变"的疑问 |
| **保持简洁** | 不增加额外的操作步骤，不改变现有交互流程 |
| **职责分离** | 视觉区域包裹逻辑与表单渲染逻辑解耦，各自独立可维护 |
| **全局一致** | 编辑页与查看器页面使用同一套作用域包裹组件，视觉风格统一 |

---

## 三、方案总览

采用**"卡片式区域 + 区域标题头 + 分组隔离"**策略，通过独立的 `ConfigScopeWrapper` 组件实现区域视觉区分：

```
┌─ 配置面板 ─────────────────────────────────────────────┐
│                                                         │
│  ┌─ 🌐 全局配置 ──────────────────────────────────────┐ │
│  │ ─────────────────────────────────────────────────── │ │
│  │  ┌─ 主题与样式 ────────────────────────────────┐   │ │
│  │  │  品牌色          ┌──────────────────┐       │   │ │
│  │  │  Logo 图片       │ [上传]           │       │   │ │
│  │  └──────────────────────────────────────────────┘   │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                         │
│  ┌─ 📄 手机 ─────────────────────────────────────────┐ │
│  │ ─────────────────────────────────────────────────── │ │
│  │  ┌─ 基础配置 ────────────────────────────────┐    │ │
│  │  │  页面标题        ┌──────────────────┐      │    │ │
│  │  │  副标题          │ 探索色彩的无限可能│      │    │ │
│  │  └──────────────────────────────────────────────┘   │ │
│  │                                                      │ │
│  │  ┌─ 显示选项 ────────────────────────────────┐    │ │
│  │  │  启用动画        [●──────────────]         │    │ │
│  │  └──────────────────────────────────────────────┘   │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**设计决策**：

- **卡片式区域包裹**：每个作用域使用独立卡片（带边框、背景色、标题头栏），视觉边界感强于左侧色条
- **标题头栏区分**：全局配置卡片使用蓝色边框 + 蓝色图标，页面配置卡片使用中性边框 + 中性图标，通过标题头栏颜色隐式传达作用域
- **页面配置标题直接使用页面名**：如"手机"、"首页"，比"当前页面配置"更直观、更简洁
- **不加字段级徽标**：区域级区分已充分传达作用域信息，字段级徽标是冗余的视觉噪音
- **使用 lucide 图标**：替代 Emoji，跨平台渲染一致，与项目现有图标体系统一

## 四、详细设计

### 4.1 顶层区域划分

#### 4.1.1 全局配置区域（蓝色系）

| 元素 | 设计 |
|:-----|:-----|
| 区域标题 | lucide `Globe` 图标 + "全局配置" |
| 标题头栏 | 深色背景（`bg-black/20`）+ 蓝色底边框（`border-b border-blue-500/30`） |
| 卡片边框 | 蓝色边框（`border border-blue-500/30`） |
| 卡片背景 | 蓝色微透明背景（`bg-blue-500/[0.04]`） |
| 图标颜色 | 蓝色（`text-blue-500`） |
| 圆角 | `rounded-lg` |
| 间距 | 与下方页面配置区域保持 `gap-5` |

#### 4.1.2 页面配置区域（中性色）

| 元素 | 设计 |
|:-----|:-----|
| 区域标题 | lucide `FileText` 图标 + 页面名称（如"手机"） |
| 标题头栏 | 深色背景（`bg-black/20`）+ 默认底边框（`border-b border-border`） |
| 卡片边框 | 默认边框（`border border-border`） |
| 卡片背景 | 白色微透明背景（`bg-white/[0.02]`） |
| 图标颜色 | 中性亮色（`text-neutral-300`） |
| 圆角 | `rounded-lg` |

**色彩语义**：
- 蓝色（冷色调）→ 全局、稳定、共享
- 中性色 → 局部、从属、独立

### 4.2 分组标题去重策略

项目配置和页面配置内部都使用 `parseSchemaToFields` 的智能分组逻辑，可能出现两个"基础配置"分组。

**方案**：保持分组标题文字不变，通过卡片边框颜色和标题头栏隐式区分。用户看到蓝色边框卡片下的"基础配置"即知为全局级，中性边框卡片下即知为页面级，无需修改分组标题。

### 4.3 空状态处理

| 场景 | 处理 |
|:-----|:-----|
| 无全局配置 | 不渲染全局配置区域，仅显示页面配置区域 |
| 全局配置 Schema 无字段 | 渲染全局配置区域，内部显示 "暂无全局配置项" |
| 页面配置 Schema 无字段 | 渲染页面配置区域，内部显示 "暂无页面级配置项" |

## 五、组件设计

### 5.1 新增 ConfigScopeWrapper 组件

**职责**：为配置表单提供作用域视觉包裹，与 `ConfigForm` 完全解耦。

**文件位置**：`packages/author-site/components/demo/ConfigScopeWrapper.tsx`

```typescript
import type { LucideIcon } from "lucide-react";
import { Globe, FileText } from "lucide-react";
import { cn } from "@/lib/utils";

type ConfigScope = "project" | "page";

interface ConfigScopeWrapperProps {
  scope: ConfigScope;
  pageName?: string;
  children: React.ReactNode;
  className?: string;
}

const SCOPE_CONFIG: Record<
  ConfigScope,
  {
    icon: LucideIcon;
    title: string;
    borderClass: string;
    bgClass: string;
    headerBorderClass: string;
    iconClass: string;
  }
> = {
  project: {
    icon: Globe,
    title: "全局配置",
    borderClass: "border border-blue-500/30",
    bgClass: "bg-blue-500/[0.04]",
    headerBorderClass: "border-b border-blue-500/30",
    iconClass: "text-blue-500",
  },
  page: {
    icon: FileText,
    title: "",
    borderClass: "border border-border",
    bgClass: "bg-white/[0.02]",
    headerBorderClass: "border-b border-border",
    iconClass: "text-neutral-300",
  },
};

export function ConfigScopeWrapper({
  scope,
  pageName,
  children,
  className,
}: ConfigScopeWrapperProps) {
  const config = SCOPE_CONFIG[scope];
  const Icon = config.icon;
  const title = scope === "page" && pageName ? pageName : config.title;

  return (
    <div className={cn("rounded-lg overflow-hidden", config.borderClass, config.bgClass, className)}>
      <div className={cn("flex items-center gap-2 px-3.5 py-3 bg-black/20", config.headerBorderClass)}>
        <Icon className={cn("h-[15px] w-[15px]", config.iconClass)} />
        <span className="text-sm font-semibold text-foreground">{title}</span>
      </div>
      {children}
    </div>
  );
}
```

**设计要点**：

- `SCOPE_CONFIG` 常量集中管理两种作用域的视觉配置，新增作用域类型只需扩展此对象
- 页面配置的 `title` 为空字符串，实际标题由 `pageName` prop 动态填充
- `ConfigForm` 无需任何改动，保持纯粹的表单渲染职责
- 组件轻量（~55 行），无内部状态，纯展示组件
- 卡片使用 `overflow-hidden` 确保标题头栏与卡片圆角对齐

### 5.2 edit/page.tsx 改造

编辑页中两个 `ConfigForm` 的调用处用 `ConfigScopeWrapper` 包裹：

```typescript
import { PreviewPanel, ConfigForm, PreviewGrid, invalidateCompileCache, ConfigScopeWrapper } from "../../../../../components/demo";

const activePageName = demoPages.find(p => p.id === activeDemoId)?.name ?? "";

{projectConfigSchema && (
  <ConfigScopeWrapper scope="project">
    <ConfigForm
      key={`project-${projectConfigSchema}`}
      schema={projectConfigSchema}
      onChange={(data) => { setConfigData((prev) => ({ ...prev, ...data })); }}
      onSchemaChange={handleProjectSchemaChange}
      initialData={configData}
      sessionId={sessionId}
    />
  </ConfigScopeWrapper>
)}

<ConfigScopeWrapper scope="page" pageName={activePageName}>
  <ConfigForm
    key={schema}
    schema={schema}
    onChange={handleConfigChange}
    onSchemaChange={handleSchemaChange}
    initialData={configData}
    sessionId={sessionId}
  />
</ConfigScopeWrapper>
```

**改动点**：
- 在现有 `import` 行中追加 `ConfigScopeWrapper`，而非新增一行导入
- 移除原有的文字标签（`📋 项目配置（所有页面共享）` / `📄 当前页面配置`）
- 移除 `<Separator />`（卡片边框已提供视觉分隔）
- 用 `ConfigScopeWrapper` 包裹 `ConfigForm`
- **行为变化**：页面配置区域的作用域标题始终显示（原代码中"📄 当前页面配置"仅在项目配置存在时才显示），确保用户始终能识别当前配置的作用域

### 5.3 viewer 页面改造

两个查看器页面同样使用 `ConfigScopeWrapper` 替换原有文字标签：

**`viewer/[projectId]/page.tsx`**：

```typescript
import { PreviewPanel, ConfigForm, PreviewGrid, ConfigScopeWrapper } from "../../../../components/demo";

const activePageName = activePage?.name ?? "";

{data.projectConfigSchema && (
```

```typescript
import { PreviewPanel, ConfigForm, ConfigScopeWrapper } from "../../../../../components/demo";

const currentPageName = currentPage?.name ?? "";

{data.projectConfigSchema && (

    <ConfigForm
      key={`project-${data.projectConfigSchema}`}
      schema={data.projectConfigSchema}

      initialData={configData}
      readonly
    />
```

### 5.4 组件导出

在 `packages/author-site/components/demo/index.ts` 中新增导出：

```typescript
export { ConfigScopeWrapper } from './ConfigScopeWrapper';
```

### 5.5 ConfigForm 无需改动

`ConfigForm`、`FieldRenderer`、`FieldGroupSection` 均无需任何改动。作用域视觉逻辑完全由外层 `ConfigScopeWrapper` 承担，`ConfigForm` 保持纯粹的表单渲染职责。

## 六、视觉规范

### 6.1 颜色 Token

| 用途 | 全局级（蓝色） | 页面级（中性色） |
|:-----|:-------------|:-------------|
| 卡片边框 | `border border-blue-500/30` | `border border-border` |
| 卡片背景 | `bg-blue-500/[0.04]` | `bg-white/[0.02]` |
| 标题头栏底边框 | `border-b border-blue-500/30` | `border-b border-border` |
| 标题头栏背景 | `bg-black/20` | `bg-black/20` |
| 图标颜色 | `text-blue-500` | `text-neutral-300` |
| 标题文字 | `text-foreground` | `text-foreground` |

### 6.2 间距规范

```
配置面板容器
├── padding: p-4
├── 全局配置区域 (ConfigScopeWrapper scope="project")
│   ├── 卡片: rounded-lg overflow-hidden
│   ├── 标题头栏: px-3.5 py-3
│   └── ConfigForm 内容
└── 页面配置区域 (ConfigScopeWrapper scope="page")
    ├── 卡片: rounded-lg overflow-hidden
    ├── 标题头栏: px-3.5 py-3
    └── ConfigForm 内容

两个卡片之间: gap-5 (20px)
```

### 6.3 区域标题样式

```
标题头栏
├── 图标: h-[15px] w-[15px] (lucide Globe / FileText)
├── 标题文字: text-sm font-semibold text-foreground
├── 头栏背景: bg-black/20
├── 头栏底边框: 蓝色或默认色（按作用域）
└── 整体: flex items-center gap-2 px-3.5 py-3
```
