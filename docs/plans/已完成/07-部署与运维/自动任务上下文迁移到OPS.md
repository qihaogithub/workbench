# 自动任务上下文迁移到 OPS

> 状态：已完成
> 创建日期：2026-06-30

## 背景

维护治理目录此前被重写为定时任务上下文，但用户确认这类文档的主要读者是自动任务中的 AI，不适合继续套用项目文档知识库的需求/技术拆分规范。

## 目标

- 将定时任务可执行上下文迁移到 `OPS/automations/`。
- 在 `docs/项目文档/维护治理/` 只保留项目级说明和入口链接。
- 更新根 `AGENTS.md`，明确 `OPS/automations/` 的维护规则。
- 保留 CLI 维护、测试工具治理、问题排查清理三类任务的上下文和当前状态。

## 范围

- 新增 `OPS/automations/` 目录结构。
- 精简 `docs/项目文档/维护治理/`。
- 更新 `docs/项目文档/INDEX.md` 和根 `AGENTS.md`。
- 更新本迁移任务记录。

不在本次范围：

- 创建真实 Codex 自动化配置。
- 修改测试脚本、OPS CLI 实现或根验证脚本。
- 清理既有运行数据和诊断产物。

## 方案

`OPS/automations/` 按 AI 执行效率组织：

- `README.md`：自动任务总入口。
- `AGENTS.md`：该目录专用维护规则。
- `contexts/`：长期任务上下文。
- `runbooks/`：按触发频率组织的执行手册。
- `state/`：覆盖式当前状态账本。

项目文档只保留薄说明，指出自动化上下文迁移位置和边界。
