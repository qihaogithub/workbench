# CLI 自动维护定时任务文档建设

> 状态：已完成
> 创建日期：2026-06-29

## 背景

用户希望 CLI 的迭代维护由定时任务自动完成。现有 CLI 文档已经明确了 CLI 与 Web 能力对齐的方向，但还缺少让定时任务真正执行的配套文档：它需要知道看哪些文件、如何判断缺口、怎么分级、能自动改到什么程度、如何验证，以及什么时候必须停下来等待人工审核。

## 目标

- 建立定时任务自动维护 CLI 所需的长期项目文档。
- 让自动任务可以根据文档执行“发现缺口、补齐 CLI、补测试、补文档、跑验证、分级合入或待审”。
- 明确低风险自动合入和高风险人工审核的边界。

## 范围

- CLI 模块长期文档。
- CLI 自动维护能力清单。
- CLI 自动维护定时任务执行设计。
- CLI 自动维护运行手册。
- CLI 模块索引更新。

不在本次范围内：

- 实际实现定时任务代码。
- 实际修改 CLI 命令实现。
- 配置真实 CI、GitHub Actions 或部署环境。

## 方案

在 `docs/项目文档/创作端/10-CLI/` 下补齐三类文档：

- 需求文档：说明自动维护定时任务要做什么、为什么做、哪些必须受限。
- 技术文档：说明定时任务的输入、流程、输出、分级、验证和失败处理。
- 能力清单：作为自动任务的“账本”，记录能力 ID、CLI 命令、风险等级和自动化等级。
