# 自动任务可执行维护系统第一阶段

> 状态：已完成
> 创建日期：2026-06-30

## 背景

用户希望利用 Codex 自动任务提高开发效率，让测试和诊断工具可以自动化维护，并让 AI 在日常问题排查中更快定位根因。当前已有 `OPS/automations/` 上下文骨架，但还缺少机器可读账本和可执行检查入口。

## 目标

- 建立工具、测试和开发脚本的机器可读 registry。
- 新增轻量 automation CLI，用于列出和检查 registry。
- 在根 `package.json` 暴露自动任务检查命令。
- 更新 `OPS/automations/` 文档，使定时任务优先使用机器检查入口。

## 范围

- `OPS/automations/registry/`
- `OPS/automations/bin/automation.mjs`
- `OPS/automations/README.md` 和相关 context/runbook/state
- 根 `package.json`
- 本任务记录

不在本次范围：

- 创建真实 Codex 定时任务。
- 自动修复测试或业务代码。
- 运行 E2E 或服务依赖型测试。

## 方案

第一阶段只做无副作用检查：

- `ops:automation list-tools`：列出 registry 中的工具、测试和脚本。
- `ops:automation check-tools`：检查 registry JSON、路径、根脚本引用和文档引用。
- `ops:automation report`：输出摘要，供定时任务快速判断当前工具版图。
- `check:automation`：作为根级轻量验证入口。

## 实施摘要

第一阶段已把 `OPS/automations/` 从纯文档上下文推进为可执行维护入口。定时任务现在可以先通过 registry 获取工具、测试和脚本版图，再用 `check:automation` 做无副作用一致性检查；日常排查可以通过诊断包快速进入固定路径。

## 风险与待确认事项

- registry 第一版只覆盖当前已知入口，后续定时任务应持续补齐。
- 本阶段不判断测试质量，只判断入口和账本自洽。
