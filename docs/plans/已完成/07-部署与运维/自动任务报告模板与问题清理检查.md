# 自动任务报告模板与问题清理检查

> 状态：已完成
> 创建日期：2026-06-30

## 背景

`OPS/automations/` 已有 registry 和基础检查器，但真实定时任务还需要稳定报告格式和问题清理候选扫描能力，避免每次输出形态不一致，也避免人工逐个检查进行中计划。

## 目标

- 新增每日检查、每周回归、每周清理报告模板。
- 为 automation CLI 增加 `stale-issues` 静态检查。
- 将 `stale-issues` 接入 weekly cleanup runbook、registry 和 state。
- 验证新增命令无副作用可运行。

## 范围

- `OPS/automations/templates/`
- `OPS/automations/bin/automation.mjs`
- `OPS/automations/README.md`
- `OPS/automations/runbooks/`
- `OPS/automations/state/`
- `OPS/automations/registry/`

不在本次范围：

- 移动或删除任何 `docs/plans/进行中/` 文件。
- 创建真实 Codex 定时任务。
- 修改业务代码或测试实现。

## 实施摘要

新增 3 个报告模板和 `stale-issues` 命令。`stale-issues` 会扫描 `docs/plans/进行中/`，报告以下候选项：

- 状态已完成但仍在进行中目录。
- 文件 mtime 超过阈值的长期未更新计划。
- 疑似重复标题。
- Markdown 缺失链接。

该命令只报告，不移动、不删除、不修改计划文件。

随后创建并更新了 Codex 自动任务：

- 更新既有 `opencode-workbench-cli`：改为读取 `OPS/automations/` 上下文。
- 新建 `opencode-workbench`：每日轻量维护检查。
- 新建 `opencode-workbench-2`：每周完整回归。
- 新建 `opencode-workbench-3`：每周问题清理。

## 验证结果

- `corepack pnpm ops:automation stale-issues --json`：通过，扫描 44 个进行中计划，标出 12 个 `completed-in-progress` 候选项。
- `corepack pnpm check:automation`：通过，0 error / 0 warning。
- `corepack pnpm ops:automation report --json`：通过，当前 registry 13 项。
- 已读取 `$CODEX_HOME/automations/` 下落盘配置，确认三条新任务状态为 ACTIVE，既有 CLI 任务已更新。

## 后续建议

- 每周清理任务先使用 `stale-issues` 生成候选清单，再由 AI 判断是否需要归档。
- 下一步可以创建真实 Codex 定时任务：每日轻量检查、每周回归、每周清理。
