# 维护治理定时任务上下文重写

> 状态：已完成
> 创建日期：2026-06-30

## 背景

用户希望在 `docs/项目文档/维护治理/` 中维护定时任务长期运行所需的上下文。现有维护治理文档主要描述 Codex 协作、改动验收矩阵和配置入口治理，尚不足以指导多个定时任务稳定执行、持续维护测试工具和闭环清理问题记录。

## 目标

- 重写维护治理模块，使其成为定时任务读取的长期上下文入口。
- 明确三类定时任务目标：创作端 CLI 工具维护、测试工具与脚本治理、定期检查与问题排查闭环。
- 明确自动任务可读、可改、必须停机和必须记录的位置。
- 更新维护治理模块索引和全局索引描述。

## 范围

- `docs/项目文档/维护治理/` 下的需求、技术和索引文档。
- `docs/项目文档/INDEX.md` 中维护治理模块简介。
- 本任务记录。

不在本次范围内：

- 创建或修改真实 Codex 定时任务配置。
- 修改 `OPS/CLI`、测试脚本或根验证脚本实现。
- 实际清理历史问题文档或测试脚本。

## 方案

将维护治理模块重组为：

- 需求文档：定义定时任务维护治理的目标、角色、边界和验收标准。
- 定时任务上下文总览：说明所有自动任务的读取顺序、上下文层级和输出位置。
- 维护任务执行矩阵：说明不同任务目标的触发频率、输入、动作、验证和停机条件。
- 测试工具与脚本治理：说明正式测试、诊断脚本、OPS 工具和临时脚本的迁移、补充、更新、删除规则。
- 问题排查与记录闭环：说明问题状态、记录位置、升级规则和过期清理规则。
- 配置与运行安全边界：说明自动任务运行时不能触碰的内容、外部依赖和凭据边界。
