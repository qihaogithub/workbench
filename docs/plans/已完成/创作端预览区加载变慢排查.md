# 创作端预览区加载变慢排查

## 归档结论

本任务已完成。创作端预览区变慢的主要原因不是编译器本身退化，而是预览入口新增页面/文档目录后，在查看页面时也提前读取所有可见知识文档正文，并且画布入口叠加了文档正文读取、可见页面 iframe 编译加载和全量截图批处理。

最终修复策略是把首屏任务收窄到当前可见目标：页面目标只构建文档目录元数据，只有选中文档目标时才读取对应正文；画布模式只读取可见或选中文档节点，并将进入画布后的首轮截图批处理收窄到 active、visible、nearby 页面。

## 保留经验

- 预览首屏性能应优先排查“与当前可见目标无关的后台读取”，尤其是知识文档正文、截图批处理和 iframe 编译加载是否抢占同一时间窗口。
- 文档目录和文档正文应分层加载；展示页面时不应为了构建目录而读取所有正文。
- 画布模式的批量任务要有可见性边界，不能默认对全项目页面和文档节点做首轮重任务。
