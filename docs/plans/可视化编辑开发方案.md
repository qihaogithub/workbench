## 一、方案对比与决策

### 1.1 候选方案概览

| 方案 | 核心思路 | 集成难度 | 开发工时 | 是否采用 |
|:-----|:---------|:---------|:---------|:---------|
| **A** | Puck 视觉编辑器 | 高 | 20+ h | ❌ |
| **B** | AST 可视化编辑器 | 中 | 15-20 h | ❌ |
| **C** | **可视化样式面板** | **低** | **8-12 h** | **✅** |
| **D** | AI 编辑增强 | 极低 | 2-4 h | 辅助 |
| **E** | Onlook Studio | 中 | 10-15 h | ❌ |
| **F** | react-design-helper | 极低 | 1-2 h | ❌ |
| **G** | vite-plugin-visual-edit | 低 | 4-6 h | ❌ |

### 1.2 已放弃方案及原因

| 方案 | 放弃原因 |
|:-----|:---------|
| **Puck 视觉编辑器** | 数据模型不匹配（需 JSX ↔ JSON 双向转换），维护风险高，功能过剩 |
| **AST 可视化编辑器** | 开发成本较高，动态渲染处理复杂；作为中期备选，当样式面板无法满足结构编辑需求时升级 |
| **Onlook Studio** | 独立桌面应用，嵌入现有编辑页面需大幅改造；与动态编译预览机制冲突 |
| **react-design-helper** | 仅修改运行时样式，**不写回代码**，无法保存修改 |
| **vite-plugin-visual-edit** | 项目基于 Next.js，非 Vite 构建 |

### 1.3 最终决策

**采用方案 C（可视化样式面板）**，方案 D（AI 快捷按钮）作为辅助手段。

**决策依据**：
1. **成本最低**：8-12 工时即可上线，复用现有三栏布局和动态编译架构
2. **体验最好**：点击元素直接改样式，运营设计师零学习成本
3. **完全兼容**：不破坏代码格式，与 AI 编辑能力互补
4. **可扩展**：后续可平滑升级至 AST 编辑器

## 二、背景与目标

### 2.1 背景

Demo 编辑工作台已具备：
- AI 对话修改代码
- 代码直接编辑
- 动态编译实时预览
- Schema 配置面板

运营设计师需要**不依赖 AI、即时生效**的方式调整 Demo 样式和简单结构。

### 2.2 目标

在现有三栏布局基础上，增加可视化编辑能力：
- **样式编辑**：点击预览区元素，修改 Tailwind 类名
- **结构编辑**：增删元素、调整顺序、列表项增删
- **即时预览**：修改后自动重新编译并刷新预览

### 2.3 非目标

以下能力**不在本方案范围内**，仍由 AI 编辑或代码编辑处理：
- 修改 JSX 属性（id、onClick、事件处理等）
- 修改条件渲染表达式（`{isShow && ...}`）
- 组件重命名
- 复杂逻辑修改

## 三、方案概述

### 3.1 核心思路

在现有编辑工作台右栏增加两个 Tab：
- **"配置" Tab**：已有，Schema 生成的配置表单
- **"样式" Tab**：新增，修改选中元素的 Tailwind 类名
- **"结构" Tab**：新增，简化的 JSX 树形操作（增删移）

预览区元素支持点击选中，选中后高亮边框，右栏自动切换到对应 Tab。

### 3.2 技术路线

| 能力 | 实现方式 | 复杂度 |
|:-----|:---------|:-------|
| 元素定位 | 编译时注入 `data-source-line` 和 `data-source-file` | 低 |
| 样式编辑 | 正则提取/替换 `className` | 低 |
| 结构编辑（增删移） | 行号定位 + 字符串插入/删除 | 中 |
| 列表项增删 | 识别 `.map()` 模式，操作数组元素 | 中 |
| 代码写回 | 直接替换编辑器内容，触发重新编译 | 低 |

**不引入 AST 解析库**（@babel/parser、recast 等），全部基于字符串操作。

## 四、技术设计

### 4.1 编译时注入

在 sucrase 编译阶段，给 JSX 元素注入位置信息：

```typescript
// packages/web/src/lib/compiler.ts
function injectSourceLocation(code: string, filename: string): string {
  // 简单正则匹配 JSX 开始标签
  // 在 <tag 后注入 data-source-file 和 data-source-line
  const lines = code.split('\n');
  let result = '';
  
  for (let i = 0; i < lines.length; i++) {
    const line = lines[i];
    // 匹配 JSX 开始标签（简化版，实际需处理更多边界）
    const match = line.match(/^(\s*)<([a-zA-Z][a-zA-Z0-9]*)([^>]*)>/);
    if (match) {
      const [, indent, tagName, attrs] = match;
      // 检查是否已注入
      if (!attrs.includes('data-source-file')) {
        const newAttrs = `${attrs} data-source-file="${filename}" data-source-line="${i + 1}"`;
        result += line.replace(attrs, newAttrs) + '\n';
        continue;
      }
    }
    result += line + '\n';
  }
  
  return result;
}
```

**限制**：
- 只给原生 HTML 标签和已导入的组件注入
- 不给条件表达式内的元素注入（避免运行时错误）
- 每行只处理一个 JSX 开始标签

### 4.2 代码修改引擎

#### 4.2.1 类名修改

```typescript
interface ClassNameEdit {
  file: string;
  line: number;      // 元素所在行号
  oldClassName: string;
  newClassName: string;
}

function editClassName(code: string, edit: ClassNameEdit): string {
  const lines = code.split('\n');
  const targetLine = lines[edit.line - 1];
  
  // 匹配 className="..." 或 className='...'
  const regex = /className=["']([^"']*)["']/;
  const match = targetLine.match(regex);
  
  if (!match) {
    // 元素没有 className，需要添加
    // 在标签属性末尾添加 className="..."
    const tagEndMatch = targetLine.match(/\s*\/?>/);
    if (tagEndMatch) {
      const insertPos = targetLine.indexOf(tagEndMatch[0]);
      lines[edit.line - 1] = 
        targetLine.slice(0, insertPos) + 
        ` className="${edit.newClassName}"` + 
        targetLine.slice(insertPos);
    }
  } else {
    // 替换现有 className
    lines[edit.line - 1] = targetLine.replace(
      regex,
      `className="${edit.newClassName}"`
    );
  }
  
  return lines.join('\n');
}
```

#### 4.2.2 元素插入

```typescript
interface InsertEdit {
  file: string;
  line: number;      // 目标元素所在行号
  position: 'before' | 'after';
  code: string;      // 要插入的 JSX 代码
}

function insertElement(code: string, edit: InsertEdit): string {
  const lines = code.split('\n');
  const targetIndex = edit.line - 1;
  const targetLine = lines[targetIndex];
  
  // 计算缩进（与目标元素同级）
  const indent = targetLine.match(/^(\s*)/)?.[1] || '';
  const childIndent = indent + '  ';
  
  // 给插入的代码添加正确缩进
  const indentedCode = edit.code
    .split('\n')
    .map((line, i) => i === 0 ? indent + line : childIndent + line)
    .join('\n');
  
  if (edit.position === 'before') {
    lines.splice(targetIndex, 0, indentedCode);
  } else {
    lines.splice(targetIndex + 1, 0, indentedCode);
  }
  
  return lines.join('\n');
}
```

#### 4.2.3 元素删除

```typescript
interface DeleteEdit {
  file: string;
  startLine: number;  // 元素开始行
  endLine: number;    // 元素结束行（需要计算）
}

function deleteElement(code: string, edit: DeleteEdit): string {
  const lines = code.split('\n');
  lines.splice(edit.startLine - 1, edit.endLine - edit.startLine + 1);
  return lines.join('\n');
}
```

**计算元素结束行**：
- 从 startLine 开始，向下扫描代码
- 维护一个深度计数器：遇到开始标签 +1，遇到结束标签 -1
- 当计数器回到 0 时，即为 endLine

### 4.3 列表项识别与操作

#### 4.3.1 编译时标记

```typescript
// 识别 .map((item, index) => (...)) 模式
// 给 map 内的根元素添加 data-list-index={index}
function injectListMarkers(code: string): string {
  // 匹配 .map((item, index) => ( ... ))
  // 在 map 回调的返回值根元素上添加 data-list-index={index}
  const mapRegex = /\.map\s*\(\s*(?:\([^)]*\)\s*=>|[^)]*=>)\s*\(\s*(<[^>]+>)/g;
  
  return code.replace(mapRegex, (match, openingTag) => {
    // 在 openingTag 的 > 前插入 data-list-index={index}
    return match.replace(openingTag, openingTag.replace(/>$/, ' data-list-index={index}>'));
  });
}
```

#### 4.3.2 列表项定位

用户点击列表项时，获取 `data-list-index`，然后在代码中：
1. 找到 `.map()` 调用的位置
2. 找到 map 回调函数返回的数组元素
3. 计算该元素在数组中的起始行和结束行
4. 在该位置前/后插入新数组元素，或删除当前元素

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对措施 |

| 用户误删重要元素 | 中 | 中 | 删除前确认对话框；提供撤销能力 |
| 动态 className 处理困难 | 高 | 中 | 明确不支持模板字符串和条件类名；只处理静态 className |
| 预览区元素点击与组件内点击事件冲突 | 中 | 中 | 使用事件委托和 stopPropagation；点击选中不触发组件内部事件 |
