## 一、背景与目标

### 1.1 问题背景

当前项目中，Agent 相关逻辑直接嵌入在 Next.js 的 API Routes 中，存在以下问题：

| 问题 | 影响 |
|:-----|:-----|
| **耦合度高** | Agent 逻辑与 Web 框架紧耦合，难以独立演进 |
| **扩展性差** | 只支持 opencode 后端，无法切换其他 AI 后端 |
| **复用性低** | Agent 能力与前端紧耦合，难以独立测试和维护 |
| **部署受限** | 必须与前端一起部署，无法独立扩缩容 |

### 1.2 设计目标

1. **解耦**：将 Agent 服务层从 Next.js 应用中剥离，作为独立服务运行
2. **抽象**：定义统一的 Agent 接口，支持多后端切换（opencode、Claude、Codex 等）
3. **高性能**：支持流式响应、WebSocket 实时推送
4. **易维护**：AI 编程友好，代码结构清晰

## 三、核心设计原则

### 3.1 工厂模式

支持多种 AI 后端，通过工厂方法创建对应的 Agent 实例：

```typescript
interface IAgentFactory {
  register(type: AgentType, creator: AgentCreator): void;
  create(config: AgentConfig): IAgent;
}

type AgentType = 'opencode' | 'claude' | 'codex' | 'gemini' | string;
type AgentCreator = (config: AgentConfig) => IAgent;
```

### 3.2 观察者模式

Agent 与消费层通过事件解耦：

```typescript
interface IAgent {
  on(event: 'stream' | 'error' | 'finish', handler: EventHandler): void;
  off(event: string, handler: EventHandler): void;
}

type EventHandler = (data: StreamEvent | ErrorEvent | FinishEvent) => void;
```

### 3.3 单例模式

AgentManager 作为全局单例，管理所有 Agent 实例：

```typescript
class AgentManager {
  private agents: Map<string, IAgent> = new Map();
  
  getOrCreate(sessionId: string, config: AgentConfig): IAgent;
  get(sessionId: string): IAgent | undefined;
  destroy(sessionId: string): Promise<void>;
  destroyAll(): Promise<void>;
}
```

### 3.4 策略模式

不同后端使用不同的通信策略：

```typescript
interface IBackendAdapter {
  connect(): Promise<void>;
  sendMessage(content: string): Promise<void>;
  disconnect(): Promise<void>;
}
```

## 七、安全设计

### 7.1 三层防护隔离机制

Agent 服务采用**三层防护**实现安全隔离：

| 层级 | 机制 | 强度 | 说明 |
|:-----|:-----|:-----|:-----|
| **L1** | 权限系统（opencode.json 白名单） | 强 | 配置文件级别的访问控制 |
| **L2** | 行为约束（AGENTS.md） | 软 | AI 行为规范和约束 |
| **L3** | 后端校验（文件变更检测 + 自动回滚） | 强 | 运行时校验和自动恢复 |

**L3 后端校验实现参考：**

```typescript
export async function validateFileChanges(
  sessionId: string,
  changes: FileChange[],
): Promise<{ valid: boolean; violations: string[] }> {
  const sessionConfig = await getSessionConfig(sessionId);
  const allowedFiles = sessionConfig.editWhitelist || [];

  const violations = changes
    .filter(
      (change) =>
        !allowedFiles.some((allowed) => change.path.endsWith(allowed)),
    )
    .map((change) => `非法修改：${change.path}`);

  if (violations.length > 0) {
    await rollbackChanges(sessionId, violations);
    await logViolation(sessionId, violations);
  }

  return {
    valid: violations.length === 0,
    violations,
  };
}
```

```typescript
// Agent 实例缓存，避免重复创建
class AgentManager {
  private cache: Map<string, IAgent> = new Map();
  
  getOrCreate(sessionId: string, config: AgentConfig): IAgent {
    if (this.cache.has(sessionId)) {
      return this.cache.get(sessionId)!;
    }
    const agent = this.factory.create(config);
    this.cache.set(sessionId, agent);
    return agent;
  }
}
```

#### 13.3.2 会话恢复（参考改）

```typescript
// AionUi/src/process/agent/acp/index.ts - 参考策略
private async createOrResumeSession(): Promise<void> {
  const resumeSessionId = this.extra.acpSessionId;
  
  if (resumeSessionId) {
    try {
      // 尝试恢复已有会话
      const response = await this.connection.loadSession(resumeSessionId);
      if (response.sessionId !== resumeSessionId) {
        this.onSessionIdUpdate?.(response.sessionId);
      }
      return;
    } catch {
      // 恢复失败，创建新会话
    }
  }
  
  // 创建新会话
  const response = await this.connection.newSession();
  this.onSessionIdUpdate?.(response.sessionId);
}
```
