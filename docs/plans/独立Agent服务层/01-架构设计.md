## 一、背景与目标

### 1.1 问题背景

当前项目中，Agent 相关逻辑直接嵌入在 Next.js 的 API Routes 中，存在以下问题：

| 问题 | 影响 |
|:-----|:-----|
| **耦合度高** | Agent 逻辑与 Web 框架紧耦合，难以独立演进 |
| **扩展性差** | 只支持 opencode 后端，无法切换其他 AI 后端 |
| **复用性低** | Agent 能力只能被 Web 前端使用，无法支持 CLI、IDE 插件等 |
| **部署受限** | 必须与前端一起部署，无法独立扩缩容 |

### 1.2 设计目标

1. **解耦**：将 Agent 服务层从 Next.js 应用中剥离，作为独立服务运行
2. **抽象**：定义统一的 Agent 接口，支持多后端切换（opencode、Claude、Codex 等）
3. **复用**：Agent 服务可被多个消费端使用（Web、CLI、IDE 插件）
4. **高性能**：支持流式响应、WebSocket 实时推送
5. **易维护**：AI 编程友好，代码结构清晰

## 三、核心设计原则

### 3.1 工厂模式

支持多种 AI 后端，通过工厂方法创建对应的 Agent 实例：

```typescript
interface IAgentFactory {
  register(type: AgentType, creator: AgentCreator): void;
  create(config: AgentConfig): IAgent;
}

type AgentType = 'opencode' | 'claude' | 'codex' | 'gemini' | string;
type AgentCreator = (config: AgentConfig) => IAgent;
```

### 3.2 观察者模式

Agent 与消费层通过事件解耦：

```typescript
interface IAgent {
  on(event: 'stream' | 'error' | 'finish', handler: EventHandler): void;
  off(event: string, handler: EventHandler): void;
}

type EventHandler = (data: StreamEvent | ErrorEvent | FinishEvent) => void;
```

### 3.3 单例模式

AgentManager 作为全局单例，管理所有 Agent 实例：

```typescript
class AgentManager {
  private agents: Map<string, IAgent> = new Map();
  
  getOrCreate(sessionId: string, config: AgentConfig): IAgent;
  get(sessionId: string): IAgent | undefined;
  destroy(sessionId: string): Promise<void>;
  destroyAll(): Promise<void>;
}
```

### 3.4 策略模式

不同后端使用不同的通信策略：

```typescript
interface IBackendAdapter {
  connect(): Promise<void>;
  sendMessage(content: string): Promise<void>;
  disconnect(): Promise<void>;
}
```

### 8.2 Agent 缓存

```typescript
// Agent 实例缓存，避免重复创建
class AgentManager {
  private cache: Map<string, IAgent> = new Map();
  
  getOrCreate(sessionId: string, config: AgentConfig): IAgent {
    if (this.cache.has(sessionId)) {
      return this.cache.get(sessionId)!;
    }
    const agent = this.factory.create(config);
    this.cache.set(sessionId, agent);
    return agent;
  }
}
```

### 8.3 流式响应优化

```typescript
// 使用 Transfer-Encoding: chunked
fastify.get('/stream', async (request, reply) => {
  reply.raw.writeHead(200, {
    'Content-Type': 'text/event-stream',
    'Cache-Control': 'no-cache',
    'Connection': 'keep-alive',
  });
  
  // 直接写入底层 socket
  agent.on('stream', (chunk) => {
    reply.raw.write(`data: ${JSON.stringify(chunk)}\n\n`);
  });
});
```

## 十二、总结

本架构设计遵循以下核心原则：

1. **解耦**：Agent 服务独立部署，消费层轻量化
2. **抽象**：统一接口，支持多后端
3. **高性能**：流式响应、连接池、缓存
4. **可扩展**：工厂模式、插件系统
5. **易维护**：清晰的分层结构、完善的文档
