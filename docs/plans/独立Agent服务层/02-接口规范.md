## 二、REST API

### 2.1 发送消息

**POST** `/api/agent/:sessionId/message`

向指定 Session 发送消息，返回 AI 响应。

#### 请求

```typescript
interface SendMessageRequest {
  content: string;              // 消息内容
  demoId?: string;              // Demo ID（首次创建时需要）
  backend?: AgentType;          // 后端类型，默认 'opencode'
  workingDir?: string;          // 工作目录
  options?: SendMessageOptions; // 额外选项
}

interface SendMessageOptions {
  timeout?: number;             // 超时时间（毫秒），默认 120000
  stream?: boolean;             // 是否流式响应，默认 false
}

type AgentType = 'opencode' | 'claude' | 'codex' | string;
```

#### 响应

```typescript
interface SendMessageResponse {
  sessionId: string;            // Session ID
  opencodeSessionId?: string;   // OpenCode 后端的 Session ID
  content: string;              // AI 响应内容
  files?: FileChange[];         // 变更的文件列表
  metadata?: {
    model?: string;             // 使用的模型
    tokens?: {
      prompt: number;
      completion: number;
    };
  };
}

interface FileChange {
  path: string;                 // 文件路径
  action: 'created' | 'modified' | 'deleted';
  content?: string;             // 文件内容（created/modified 时）
}
```

#### 示例

```bash
curl -X POST http://localhost:3001/api/agent/session-123/message \
  -H "Content-Type: application/json" \
  -d '{
    "content": "把按钮颜色改成蓝色",
    "demoId": "demo-banner",
    "workingDir": "/path/to/session"
  }'
```

```json
{
  "success": true,
  "data": {
    "sessionId": "session-123",
    "content": "已将按钮颜色修改为蓝色...",
    "files": [
      {
        "path": "index.tsx",
        "action": "modified",
        "content": "export function Button() { ... }"
      }
    ]
  }
}
```

## 三、WebSocket API

### 3.1 连接

```
ws://localhost:3001/api/agent/:sessionId/stream
```

### 3.2 消息格式

所有消息均为 JSON 格式。

#### 客户端 → 服务端

```typescript
// 发送消息
interface ClientMessage {
  type: 'message';
  id: string;                   // 客户端生成的消息 ID
  content: string;
  options?: SendMessageOptions;
}

// 取消当前请求
interface ClientCancel {
  type: 'cancel';
  id: string;                   // 要取消的消息 ID
}

// 心跳
interface ClientPing {
  type: 'ping';
  timestamp: number;
}
```

#### 服务端 → 客户端

```typescript
// 流式内容
interface ServerStream {
  type: 'stream';
  id: string;                   // 对应的消息 ID
  content: string;              // 增量内容
  done: boolean;                // 是否结束
}

// 错误
interface ServerError {
  type: 'error';
  id: string;
  error: {
    code: ErrorCode;
    message: string;
  };
}

// 完成
interface ServerFinish {
  type: 'finish';
  id: string;
  files?: FileChange[];
  metadata?: {
    model?: string;
    tokens?: { prompt: number; completion: number };
  };
}

// 心跳响应
interface ServerPong {
  type: 'pong';
  timestamp: number;
}

// 状态变更
interface ServerStatus {
  type: 'status';
  status: AgentStatus;
}
```

### 3.3 连接示例

```javascript
const ws = new WebSocket('ws://localhost:3001/api/agent/session-123/stream');

ws.onopen = () => {
  // 发送消息
  ws.send(JSON.stringify({
    type: 'message',
    id: 'msg-001',
    content: '把按钮颜色改成蓝色',
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  
  switch (data.type) {
    case 'stream':
      console.log('收到内容:', data.content);
      if (data.done) console.log('流式结束');
      break;
    case 'finish':
      console.log('消息完成，变更文件:', data.files);
      break;
    case 'error':
      console.error('错误:', data.error);
      break;
  }
};
```

### 3.4 连接生命周期

```
┌─────────┐     connect      ┌─────────┐
│ Client  │ ───────────────► │ Server  │
└─────────┘                  └─────────┘
     │                            │
     │  message { id, content }   │
     │ ─────────────────────────► │
     │                            │
     │  stream { id, content }    │
     │ ◄───────────────────────── │
     │  stream { id, content }    │
     │ ◄───────────────────────── │
     │  finish { id, files }      │
     │ ◄───────────────────────── │
     │                            │
     │  ping                      │
     │ ─────────────────────────► │
     │  pong                      │
     │ ◄───────────────────────── │
     │                            │
     │  close                     │
     │ ─────────────────────────► │
```

## 四、类型定义

### 4.1 核心类型

```typescript
// Agent 类型
type AgentType = 'opencode' | 'claude' | 'codex' | 'gemini' | string;

// Agent 状态
type AgentStatus = 
  | 'initializing'   // 初始化中
  | 'ready'          // 就绪
  | 'processing'     // 处理中
  | 'error'          // 错误
  | 'destroyed';     // 已销毁

// 错误码
type ErrorCode =
  | 'INVALID_PARAMS'
  | 'SESSION_NOT_FOUND'
  | 'AGENT_NOT_INITIALIZED'
  | 'BACKEND_UNAVAILABLE'
  | 'MESSAGE_SEND_ERROR'
  | 'FILE_ACCESS_DENIED'
  | 'RATE_LIMIT_EXCEEDED'
  | 'INTERNAL_ERROR';
```

### 4.2 配置类型

```typescript
// Agent 配置
interface AgentConfig {
  sessionId: string;
  backend?: AgentType;
  workingDir?: string;
  demoId?: string;
  
  // 后端特定配置
  opencode?: {
    serverUrl?: string;
    timeout?: number;
  };
  claude?: {
    apiKey?: string;
    model?: string;
  };
}

// 发送消息选项
interface SendMessageOptions {
  timeout?: number;
  stream?: boolean;
  context?: {
    files?: string[];           // 附带的文件路径
    presetRules?: string;       // 预设规则
  };
}
```

### 4.3 事件类型

```typescript
// 流式事件
interface StreamEvent {
  type: 'stream';
  sessionId: string;
  content: string;
  done: boolean;
}

// 错误事件
interface ErrorEvent {
  type: 'error';
  sessionId: string;
  error: {
    code: ErrorCode;
    message: string;
    retryable: boolean;
  };
}

// 完成事件
interface FinishEvent {
  type: 'finish';
  sessionId: string;
  files?: FileChange[];
  metadata?: Record<string, unknown>;
}
```

## 五、错误处理

### 5.1 HTTP 状态码

| 状态码 | 说明 |
|:-------|:-----|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权（需要 API Key） |
| 404 | Session 或 Agent 不存在 |
| 429 | 请求过于频繁 |
| 500 | 服务器内部错误 |
| 503 | 后端服务不可用 |

### 5.2 错误响应示例

```json
{
  "success": false,
  "error": {
    "code": "BACKEND_UNAVAILABLE",
    "message": "OpenCode 服务未运行，请先启动 opencode server",
    "details": {
      "backend": "opencode",
      "serverUrl": "http://localhost:4096",
      "lastCheck": "2026-04-05T10:00:00Z"
    }
  }
}
```

### 5.3 可重试错误

以下错误可通过重试解决：

| 错误码 | 是否可重试 | 建议重试间隔 |
|:-------|:-----------|:-------------|
| `BACKEND_UNAVAILABLE` | 是 | 5s |
| `RATE_LIMIT_EXCEEDED` | 是 | 60s |
| `MESSAGE_SEND_ERROR` | 视情况 | 2s |
| `INTERNAL_ERROR` | 是 | 3s |

## 九、SDK 客户端

### 9.1 TypeScript 客户端

```typescript
import { AgentClient } from '@opencode-workbench/agent-client';

const client = new AgentClient({
  baseUrl: 'http://localhost:3001',
  apiKey: 'your-api-key', // 可选
});

// 发送消息
const result = await client.sendMessage('session-123', '把按钮改成蓝色');

// 流式消息
const stream = client.stream('session-123');
stream.on('data', (chunk) => console.log(chunk.content));
stream.send('把按钮改成蓝色');
```
