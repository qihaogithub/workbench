# UI Demo 可配置生成系统 - 并行开发总体规划

## 1. 并行开发策略

为了让四个 AI 代理能够同时开工而不产生冲突，系统按**前后端分离及组件化**进行拆分。

- **公共依赖**：Next.js (App Router), Tailwind CSS, shadcn/ui, pnpm Monorepo。
- **接口契约**：所有代理在开发初期必须使用统一的 Mock 数据或 API 接口定义，消除互相等待的时间。
- **任务分配**：分为 1 个后端服务任务和 3 个前端模块任务。

## 2. 核心 API 契约定义（全局共享数据结构）

各个代理在开发时，以此数据结构为准进行 Mock 和联调：

### 2.1 基础数据结构

```typescript
// 1. Demo 基本信息
interface DemoMeta {
  id: string;              // Demo 目录名，如 demo-1
  name: string;            // 显示名称
  createdAt: number;       // 创建时间戳
  updatedAt: number;       // 更新时间戳
  thumbnail?: string;      // 缩略图路径（可选）
}

// 2. 文件内容结构
interface DemoFiles {
  code: string;            // index.tsx 的内容
  schema: string;          // config.schema.json 的内容
}

// 3. Session 信息
interface SessionMeta {
  sessionId: string;       // 会话唯一标识
  demoId: string;          // 关联的 Demo ID
  createdAt: number;       // 会话创建时间
  expiresAt: number;       // 会话过期时间
}
```

### 2.2 API 响应格式

```typescript
// 统一成功响应
interface ApiSuccessResponse<T> {
  success: true;
  data: T;
}

// 统一错误响应
interface ApiErrorResponse {
  success: false;
  error: {
    code: string;          // 错误码，如 'DEMO_NOT_FOUND'
    message: string;       // 人类可读的错误信息
    details?: unknown;     // 额外错误详情（可选）
  };
}

// 联合类型
type ApiResponse<T> = ApiSuccessResponse<T> | ApiErrorResponse;
```

### 2.3 错误码定义

| 错误码 | HTTP状态码 | 说明 |
|--------|-----------|------|
| `DEMO_NOT_FOUND` | 404 | Demo 不存在 |
| `SESSION_NOT_FOUND` | 404 | Session 不存在 |
| `INVALID_REQUEST` | 400 | 请求参数无效 |
| `FILE_READ_ERROR` | 500 | 文件读取失败 |
| `FILE_WRITE_ERROR` | 500 | 文件写入失败 |
| `SESSION_EXPIRED` | 410 | Session 已过期 |
| `VALIDATION_ERROR` | 422 | 数据校验失败 |

### 2.4 API 端点规范

| 方法 | 端点 | 请求体 | 响应 |
|------|------|--------|------|
| GET | `/api/demos` | - | `ApiResponse<DemoMeta[]>` |
| POST | `/api/demos` | `{ name: string }` | `ApiResponse<DemoMeta>` |
| DELETE | `/api/demos/[id]` | - | `ApiResponse<void>` |
| POST | `/api/sessions` | `{ demoId: string }` | `ApiResponse<SessionMeta>` |
| GET | `/api/sessions/[sessionId]/files` | - | `ApiResponse<DemoFiles>` |
| PUT | `/api/sessions/[sessionId]/files` | `DemoFiles` | `ApiResponse<void>` |
| POST | `/api/sessions/[sessionId]/merge` | - | `ApiResponse<void>` |
| DELETE | `/api/sessions/[sessionId]` | - | `ApiResponse<void>` |

## 4. 组件接口契约

### 4.1 任务C 产出组件（供任务D复用）

```typescript
// Sandpack 预览面板
interface PreviewPanelProps {
  code: string;                          // React 组件源码
  configData: Record<string, unknown>;   // 当前配置数据
  sdkFiles?: Record<string, string>;     // SDK 文件映射（可选）
  onError?: (error: Error) => void;      // 错误回调
}

// 动态配置表单
interface ConfigFormProps {
  schema: string;                        // JSON Schema 字符串
  onChange: (data: Record<string, unknown>) => void;  // 配置变更回调
  initialData?: Record<string, unknown>; // 初始配置数据
  readonly?: boolean;                    // 只读模式
}
```

### 4.2 任务D 产出组件（供任务B页面接入）

```typescript
// AI 编辑工作台主组件
interface EditWorkbenchProps {
  demoId: string;                        // Demo ID
  sessionId: string;                     // Session ID
}

// 代码编辑器
interface CodeEditorProps {
  initialCode: string;                   // 初始代码
  initialSchema: string;                 // 初始 Schema
  onChange: (code: string, schema: string) => void;  // 内容变更
  onValidationChange: (result: ValidationResult) => void;  // 校验结果
  debounceMs?: number;                   // 防抖延迟（默认300ms）
}

// 校验结果
interface ValidationResult {
  isValid: boolean;
  errors: ValidationError[];
}

interface ValidationError {
  type: 'json_syntax' | 'props_mismatch' | 'required_missing';
  message: string;
  line?: number;
}
```
