## 二、已知问题（待修复）

### 2.1 Session 与 Workspace 不同步（高优先级）

**现象：** 修改 workspace 下的 `config.schema.json` 后，已有 Session 仍然使用旧版本 schema。

**原因：** `createSession()` 通过 `fs.cpSync(workspacePath, sessionPath)` 复制文件，Session 创建后不再与 workspace 同步。

**影响：** 用户刷新页面或重新进入编辑时，如果复用已有 Session，看不到新的 schema 配置（如 `ui:widget: "file"`）。

**临时解决：** 手动修改 Session 目录下的 `config.schema.json`，或丢弃旧 Session 重新创建。

**根本解决：**
- 方案 A：每次加载 Session 时，从 workspace 重新复制 schema 文件（如果 workspace 更新）
- 方案 B：编辑页面在加载时对比 workspace 和 session 的 schema，自动同步

### 2.2 ImageListWidget 删除未清理服务器文件（已修复）

**现象：** 在 ImageListWidget 中点击删除按钮，仅从前端列表移除图片，服务器上的文件仍然保留。

**修复位置：** `packages/web/components/demo/ImageListWidget.tsx:handleDelete`

**修复方案：** 删除前判断 URL 是否为 `/api/sessions/...` 格式，先调用 DELETE API 删除服务器文件，再更新列表。

### 2.3 FileUploadWidget 替换上传未删除旧文件（已修复）

**现象：** FileUploadWidget 中上传新图片替换旧图片时，旧图片文件残留在服务器。

**修复位置：** `packages/web/components/demo/widgets.tsx:doUpload`

**修复方案：** 上传新文件前，如果当前 value 是已上传路径，先调用 DELETE API 删除旧文件。

### 2.4 ConfigFormNew 不支持 ui:widget 字段（已修复）

**现象：** ConfigFormNew 中所有 `type: "array"` 字段都渲染为 ImageListWidget，`type: "string"` 字段都渲染为普通输入框，无法识别 `ui:widget: "file"` 等配置。

**修复位置：** `packages/web/components/demo/ConfigFormNew.tsx`

**修复方案：**
- `FieldConfig` 接口新增 `uiWidget` 和 `uiOptions` 字段
- `parseSchemaToFields` 中读取 `prop["ui:widget"]` 和 `prop["ui:options"]`
- `FieldRenderer` 中优先根据 `field.uiWidget` 判断渲染组件：
  - `uiWidget === "file"` → FileUploadWidget（单图）
  - `uiWidget === "imageList"` → ImageListWidget（多图）
  - `type === "array"` 且未指定 uiWidget → 回退到 ImageListWidget

### 2.5 上传进度和批量上传（低优先级）

**现象：**
- 上传过程中只显示"上传中..."文字，没有进度百分比
- ImageListWidget 一次只能上传一张图片，不支持多选文件批量上传

**修复方案：**
- 使用 XMLHttpRequest 替代 fetch，通过 `onprogress` 事件获取上传进度
- ImageListWidget 的 `<input type="file">` 添加 `multiple` 属性，支持批量选择

### 2.6 拖拽上传视觉反馈（低优先级）

**现象：** 拖拽图片到上传区域时，没有明显的视觉反馈（如边框高亮、背景变化）。

**修复方案：** 添加 `onDragEnter` / `onDragLeave` 状态，切换上传区域的样式。

### 2.7 图片在 iframe 预览中的路径问题（已修复）

**现象：** 上传后的图片 URL 格式为 `/api/sessions/{sessionId}/assets/{filename}`，iframe 中的预览组件无法直接访问。

**原因：** iframe 使用 Blob URL 加载（`src="blob:..."`），其 origin 为 `null`，相对路径 `/api/sessions/...` 无法在 Blob URL 上下文中正确解析。

**修复位置：** `packages/web/components/demo/PreviewPanel.tsx`

**修复方案：** 在 `sendUpdateCode` 和 `sendUpdateConfig` 发送 configData 到 iframe 之前，通过 `resolveImageUrls()` 函数将相对路径转换为绝对路径（加上 `window.location.origin`）。该函数递归遍历 configData 对象/数组，将所有以 `/api/sessions/` 开头的字符串替换为完整 URL。

### 2.8 配置面板图片上传组件样式不协调（已修复）

**现象：** FileUploadWidget 和 ImageListWidget 中使用了 `border-gray-300` 和 `text-gray-700` 等硬编码颜色，在暗色主题下显得突兀、不协调。

**修复位置：**
- `packages/web/components/demo/widgets.tsx`（FileUploadWidget）
- `packages/web/components/demo/ImageListWidget.tsx`（ImageListWidget）

**修复方案：**
- 输入框：`border-gray-300` → `border-border`，`bg-input`
- 文字：`text-gray-700` → `text-foreground`
- 必填标记：`text-red-500` → `text-destructive`
- 上传区域：添加圆角背景图标容器，增强视觉层次
- 空状态：添加背景色和图标容器，美化整体外观
- 删除按钮：改为半透明背景 + hover 变色，更优雅

### 2.9 上传后图片占用表单空间过大（已修复）

**现象：** FileUploadWidget 中图片预览使用 `h-32` 全宽大图，在配置面板中占用过多垂直空间。

**修复位置：** `packages/web/components/demo/widgets.tsx`

**修复方案：** 改为水平紧凑布局：左侧 `64x64` 缩略图 + 右侧输入框和「重新上传」按钮，整体高度从 `h-32` 大幅缩减。

### 2.10 缺少图片尺寸校验功能（已修复）

**现象：** 上传图片时无法根据配置限制图片的宽高尺寸。

**修复位置：**
- `packages/web/components/demo/widgets.tsx`
- `packages/web/components/demo/ImageListWidget.tsx`

**修复方案：**
- `ui:options` 中支持 `minWidth`、`minHeight`、`maxWidth`、`maxHeight` 配置
- 上传前通过 `Image` 对象读取图片实际尺寸
- 若尺寸不符合要求，弹出 Dialog 提示用户：「取消上传」或「继续上传」
- 尺寸校验逻辑封装为 `validateImageDimensions()` 工具函数

### 2.11 删除上传图片未清理服务器文件（已修复）

**现象：** FileUploadWidget 和 ImageListWidget 中删除已上传的图片时，仅清除了前端状态，服务器上的文件仍然保留。

**修复位置：**
- `packages/web/components/demo/widgets.tsx:handleClear`
- `packages/web/components/demo/widgets.tsx:doUpload`（替换时删除旧文件）
- `packages/web/components/demo/ImageListWidget.tsx:handleDelete`

**修复方案：** 删除前判断 URL 是否为 `/api/sessions/...` 格式，调用 DELETE API 删除服务器文件。

## 三、Schema 配置参考

### 单图配置（ui:widget: "file"）

```json
{
  "type": "string",
  "title": "横幅图片",
  "ui:widget": "file",
  "ui:options": {
    "accept": "image/*",
    "maxSize": 5242880,
    "placeholder": "输入图片 URL 或上传",
    "minWidth": 800,
    "minHeight": 200,
    "maxWidth": 2000,
    "maxHeight": 800
  }
}
```

### 多图配置（ui:widget: "imageList"）

```json
{
  "type": "array",
  "title": "图片画廊",
  "ui:widget": "imageList",
  "ui:options": {
    "accept": "image/*",
    "maxSize": 2097152,
    "maxItems": 20,
    "minWidth": 200,
    "minHeight": 200
  }
}
```

**配置说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| `accept` | string | 接受的文件类型，默认 `image/*` |
| `maxSize` | number | 最大文件大小（字节），默认 5MB |
| `maxItems` | number | 多图时最大图片数量，默认 20 |
| `placeholder` | string | URL 输入框占位文字 |
| `minWidth` | number | 最小宽度（px），可选 |
| `minHeight` | number | 最小高度（px），可选 |
| `maxWidth` | number | 最大宽度（px），可选 |
| `maxHeight` | number | 最大高度（px），可选 |
