## 一、背景与目标

### 1.1 当前实现现状

**需求文档定义**（参见 [配置系统_需求文档.md](../../项目文档/Web前端/配置系统/配置系统_需求文档.md)）：

- **3.2 支持的控件扩展** 表中明确列出"图片选择器：选择或上传图片，配置图片资源"
- **3.1 支持的数据类型** 表中列出 `array` 类型对应"列表编辑器"

**当前实现情况**：

| 组件 | 文件 | 当前功能 | 差距 |
|:-----|:-----|:--------|:-----|
| FileUploadWidget | [widgets.tsx#L35-67](file:///e:/重要文件/Programming/1_Work/opencode工作台/packages/web/components/demo/widgets.tsx#L35-67) | URL 文本输入框 + 简单预览 | ❌ 无文件选择、无上传功能 |
| ImageListWidget | [ImageListWidget.tsx](file:///e:/重要文件/Programming/1_Work/opencode工作台/packages/web/components/demo/ImageListWidget.tsx) | 通过 Dialog 输入 URL 添加图片 | ❌ 无文件选择、无上传功能 |

> **说明**：现有 FileUploadWidget 实际上已支持 URL 输入和图片预览（通过 `<input type="url">` 和 `<img>` 实现），但不支持本地文件选择和上传。

### 1.2 目标

实现配置系统的图片上传功能，支持：

- **单图上传**：通过文件选择器选择本地图片，上传后返回 URL
- **多图上传**：通过列表编辑器批量选择上传多张图片
- **预览与删除**：上传后实时预览，支持删除已上传图片
- **Schema 集成**：在 JSON Schema 中通过 `ui:widget` 和 `ui:options` 控制控件行为

## 二、技术方案设计

### 2.1 核心设计决策

| 决策点 | 选择 | 理由 |
|:-------|:-----|:-----|
| 存储方式 | 存储在 Session 工作空间的 `assets/images/` 目录 | 与现有文件系统结构一致，便于管理 |
| 文件命名 | 使用时间戳 + 随机数生成唯一文件名 | 避免文件名冲突 |
| 访问方式 | 通过 `/api/sessions/[sessionId]/assets/[filepath]` 路由访问 | 与 Session 隔离，清理时一并删除 |
| 组件架构 | 基于原生 `<input type="file">` + 定制 UI | 避免额外依赖，符合项目轻量化原则 |
| Session 路径 | 通过 `getSessionPath()` 兼容新旧目录结构 | 支持 `{sessionId}/` 和 `{userId}/{projectId}/{sessionId}/` 两种结构 |

### 2.2 目录结构

上传的 assets 目录建立在 Session 工作区内，通过 `getSessionPath()` 函数获取实际路径：

```
# 新结构：{userId}/{projectId}/{sessionId}/assets/images/
packages/web/data/sessions/
└── {userId}/
    └── {projectId}/
        └── {sessionId}/
            ├── index.tsx
            ├── config.schema.json
            ├── .session.json
            └── assets/
                └── images/
                    ├── img_1745749200000_abc123.png
                    └── img_1745749201000_def456.jpg

# 旧结构：{sessionId}/assets/images/
packages/web/data/sessions/
└── {sessionId}/
    ├── index.tsx
    ├── config.schema.json
    ├── .session.json
    └── assets/
        └── images/
            └── img_1745749200000_abc123.png
```

> **注意**：`getSessionPath()` 会自动处理新旧两种目录结构的兼容，详见 [fs-utils.ts#L61-76](file:///e:/重要文件/Programming/1_Work/opencode工作台/packages/web/src/lib/fs-utils.ts#L61-76)。

### 2.3 Schema 配置扩展

```json
{
  "type": "object",
  "properties": {
    "avatar": {
      "type": "string",
      "title": "头像",
      "description": "上传用户头像",
      "ui:widget": "file",
      "ui:options": {
        "accept": "image/*",
        "maxSize": 5242880,
        "placeholder": "点击选择图片"
      }
    },
    "gallery": {
      "type": "array",
      "title": "图片画廊",
      "items": {
        "type": "object",
        "properties": {
          "url": { "type": "string" },
          "alt": { "type": "string" }
        }
      },
      "ui:widget": "imageListUpload",
      "ui:options": {
        "accept": "image/*",
        "maxItems": 20,
        "maxSize": 2097152
      }
    }
  }
}
```

### 2.4 UI:OPTIONS 配置项

| 选项 | 类型 | 说明 | 默认值 |
|:-----|:-----|:-----|:-------|
| accept | string | 接受的文件类型，支持 MIME 类型或扩展名 | "image/*" |
| maxSize | number | 单个文件最大大小（字节） | 5242880 (5MB) |
| maxItems | number | 最多上传数量（仅限 array 类型） | 10 |
| placeholder | string | 占位提示文字 | "点击选择图片" |
| showPreview | boolean | 是否显示缩略图预览 | true |

## 四、API 设计

### 4.1 新增 API 路由

| 方法 | 路径 | 说明 |
|:-----|:-----|:-----|
| POST | `/api/sessions/[sessionId]/upload` | 上传单个文件 |
| GET | `/api/sessions/[sessionId]/assets/[...filepath]` | 获取已上传的文件 |
| DELETE | `/api/sessions/[sessionId]/assets/[...filepath]` | 删除已上传的文件 |

> **说明**：选择 `/upload` 而非 `/assets/upload`，是为了与现有 API 风格保持一致（如 `/sessions/[sessionId]/files`、`/sessions/[sessionId]/save` 等）。

### 4.2 POST /api/sessions/[sessionId]/upload

**请求**：

```
Content-Type: multipart/form-data

file: <binary>
```

**响应（成功）**：

```json
{
  "success": true,
  "data": {
    "url": "/api/sessions/session-xxx/assets/images/img_1745749200000_abc123.png",
    "filename": "img_1745749200000_abc123.png",
    "size": 102456,
    "mimeType": "image/png"
  }
}
```

**响应（失败）**：

```json
{
  "success": false,
  "error": {
    "code": "FILE_TOO_LARGE",
    "message": "文件大小超过 5MB 限制",
    "details": { "maxSize": 5242880, "actualSize": 8192000 }
  }
}
```

### 4.3 错误码

| 错误码 | 说明 |
|:-------|:-----|
| INVALID_FILE_TYPE | 文件类型不在允许范围内 |
| FILE_TOO_LARGE | 文件大小超过限制 |
| SESSION_NOT_FOUND | Session 不存在 |
| UPLOAD_FAILED | 上传过程出错 |

## 七、风险与注意事项

### 7.1 文件清理

- Session 过期或被丢弃时，`assets/` 目录应一并删除
- 考虑添加定时清理任务处理孤立文件

### 7.2 安全考虑

- 仅允许上传图片类型（MIME check）
- 限制文件大小防止滥用
- Session 权限校验确保用户只能访问自己的资源

### 7.3 性能考虑

- 大图片上传可能耗时较长，考虑添加超时处理
- 多图上传时逐个上传避免并发问题
