
# 预览区样式隔离最佳方案 (CSS Scope 重置)

## 一、 核心策略：为什么放弃 `all: initial`？

在 Tailwind 生态中，最佳的隔离方式不是“推翻一切重来”，而是**“在局部建立一个新的正确的上下文”**。

1. **局部变量覆盖**：在预览区容器上，强制重新声明一遍所有浅色主题（Light Mode）的 CSS 变量，直接切断外层全局深色变量的污染。
2. **强制底色与排版初始化**：在容器上重置 `background`、`color`、`color-scheme`，但保留 Tailwind 的基础样式（Preflight）。
3. **完美兼容性**：不会破坏 h1, p, button 的 Tailwind 默认扁平化样式。

---

## 二、 具体实现步骤（只需 2 步）

### 步骤 1：在全局 CSS 中添加隔离域样式

打开你的 `globals.css` 或主样式表，在 `@layer base` 或者文件末尾追加以下代码。这段代码会创建一个名为 `.preview-sandbox` 的隔离黑盒。

```css
/* ============================================
   预览区沙盒样式隔离 (.preview-sandbox)
   专门针对 Tailwind + shadcn/ui 环境优化
   ============================================ */

.preview-sandbox {
  /* 1. 强制系统级属性为浅色模式 */
  color-scheme: light;
  
  /* 2. 恢复基础排版属性，防止继承外层的特殊设定 */
  text-align: left;
  line-height: 1.5;
  font-family: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
  
  /* 3. 强制重写 shadcn 的所有核心 CSS 变量为浅色（Light）模式 */
  --background: 0 0% 100%;
  --foreground: 222.2 84% 4.9%;
  --card: 0 0% 100%;
  --card-foreground: 222.2 84% 4.9%;
  --popover: 0 0% 100%;
  --popover-foreground: 222.2 84% 4.9%;
  --primary: 222.2 47.4% 11.2%;
  --primary-foreground: 210 40% 98%;
  --secondary: 210 40% 96.1%;
  --secondary-foreground: 222.2 47.4% 11.2%;
  --muted: 210 40% 96.1%;
  --muted-foreground: 215.4 16.3% 46.9%;
  --accent: 210 40% 96.1%;
  --accent-foreground: 222.2 47.4% 11.2%;
  --destructive: 0 84.2% 60.2%;
  --destructive-foreground: 210 40% 98%;
  --border: 214.3 31.8% 91.4%;
  --input: 214.3 31.8% 91.4%;
  --ring: 222.2 84% 4.9%;
  --radius: 0.5rem;

  /* 4. 强制应用基础颜色（非常关键） */
  background-color: hsl(var(--background)) !important;
  color: hsl(var(--foreground)) !important;

  /* 5. 确保沙盒占满容器且隐藏外部溢出 */
  display: block;
  width: 100%;
  height: 100%;
  overflow: hidden;
  position: relative;
}
```

### 步骤 2：在预览组件中应用包裹类

找到你们渲染动态代码的地方（例如 `PreviewPanel.tsx`），将包裹层（Wrapper）加上 `preview-sandbox` 这个类。

```tsx
// PreviewPanel.tsx
import React from 'react';

// ... 其他代码 ...

{compiledComponent && !isCompiling && (
  <PreviewErrorBoundary onError={onError}>
    {/* 👇 加上 className="preview-sandbox" */}
    <div style={previewStyle} className="preview-sandbox shadow-sm border rounded-md">
      <React.Suspense
        fallback={
          <div className="flex items-center justify-center h-full w-full">
            <div className="animate-spin rounded-full h-8 w-8 border-b-2 border-gray-900" />
          </div>
        }
      >
        {React.createElement(compiledComponent, configData)}
      </React.Suspense>
    </div>
  </PreviewErrorBoundary>
)}
```

## 三、 进阶：解决 shadcn 弹窗（Portal）逃逸问题

在你的需求中，AI 可能会生成包含下拉菜单（Select）、弹窗（Dialog）的组件。shadcn/ui 底层依赖 Radix UI，这些组件默认会**渲染到 `<body>` 标签的末尾**。

这就意味着：**弹窗会跳出 `.preview-sandbox` 的保护圈，依然被深色主题污染！**

### 解决方案：使用统一的 Provider 约束 Portal

创建一个 `PreviewProvider` 包装你的动态组件，强制所有 Radix 组件（shadcn 底层）挂载到沙盒内部。

```tsx
// PreviewPanel.tsx
import { TooltipProvider } from "@/components/ui/tooltip" // 如果用到了 shadcn tooltip
// 引入 Radix 的 Portal 管理（如果你们有用到相关弹窗）
import * as Portal from "@radix-ui/react-portal";

// ...
const previewContainerRef = React.useRef<HTMLDivElement>(null);

return (
  <div 
    ref={previewContainerRef} 
    className="preview-sandbox"
    style={previewStyle}
  >
      {/* 如果组件内部用到弹窗，让它默认挂载到这个 Ref 内部，而不是 Body */}
      {React.createElement(compiledComponent, configData)}
  </div>
)
```
*注：如果在 MVP 阶段 AI 暂时不生成带复杂层级弹窗的 Demo，这一步可以先忽略。*

## 四、 为什么这是目前的最佳实践？

| 考量点 | 此方案的表现 | Shadow DOM 的表现 | `all: initial` 的表现 |
| :--- | :--- | :--- | :--- |
| **Tailwind 兼容性** | 🟢 **完美**（复用外层解析好的样式表，保留 Preflight） | 🔴 **极差**（需要在内部重新加载完整的 CSS） | 🟡 **较差**（`revert` 会破坏部分 Tailwind 布局） |
| **性能** | 🟢 **极好**（仅仅是 CSS 渲染树的变量覆盖） | 🟡 **一般**（创建封闭树有微小开销） | 🟢 **极好** |
| **落地成本** | 🟢 **极低**（仅需 2 分钟复制粘贴） | 🔴 **极高**（需重构整个组件挂载逻辑） | 🟢 **极低** |
| **可维护性** | 🟢 **高**（随时可以改回 Dark 模式支持） | 🟡 **中** | 🟡 **中** |
