# 版本历史功能优化与颗粒度扩展方案

> 版本：v1.0
> 创建日期：2026-06-05
> 状态：需求规划（未实施）
> 类型：功能优化与扩展

---

## 一、背景与目标

### 1.1 当前问题

当前编辑页左侧栏"版本"Tab 中展示的项目历史版本列表存在以下体验问题：

**标识信息不友好**：版本卡片中显示的保存者信息为一串 UUID（如 `a5862615-26bb-4688-924d-7fd68c132e21`），用户无法直观识别是谁保存了这个版本。

**信息冗余**：每个版本卡片右侧显示"10 文件"等文件数量信息，对于用户判断版本价值帮助不大，反而造成视觉干扰。

**颗粒度单一**：当前版本管理仅支持"整个项目"一个维度。用户修改了单个页面后保存，产生的版本快照包含整个项目所有文件。当用户只想查看或恢复某个特定页面的历史时，无法从版本历史中直接定位。

**缺乏历史预览**：用户在选择恢复某个版本前，无法先预览该版本的效果，只能依赖版本号和保存时间做判断，恢复决策成本高。

### 1.2 目标

- **优化版本列表的可读性**：用"用户名"替代无意义的 UUID，去除文件数量显示，让版本列表更清爽、信息更有价值。
- **扩展版本管理的颗粒度**：在保留"项目级版本"的基础上，增加"页面级版本"，用户既可查看整个项目的演变，也可追踪单个页面的修改历史。
- **提供历史版本预览能力**：用户可在恢复前预览任意历史版本的实际效果，降低恢复决策的试错成本。

## 二、需求描述

### 2.1 版本列表展示优化

#### 2.1.1 显示保存者用户名

**用户视角**：
- 版本列表中每个版本应显示保存该版本的用户名称（如 "张三" 或 "admin"），而非 sessionId 或 UUID。
- 若系统无法获取用户名，应显示友好的默认标识（如"未知用户"），绝不暴露原始 UUID。

**交互效果**：
- 用户名显示在版本号与时间之间，作为版本的主要归属标识。
- 多用户协作场景下，用户可快速识别每个版本由谁创建。

#### 2.1.2 移除文件数量显示

**用户视角**：
- 版本卡片中不再显示"X 文件"的数量标识。
- 用户无需关注每个版本的文件数量，重点关注版本号、保存时间、保存者和备注。

**交互效果**：
- 版本卡片右侧仅保留"恢复"操作按钮。
- 列表整体视觉更简洁，减少无关信息干扰。

---

### 2.2 版本颗粒度扩展

#### 2.2.1 项目级版本（现有能力保留）

**用户视角**：
- 用户保存项目时，系统继续自动创建整个项目的版本快照。
- 项目级版本反映的是项目所有页面和配置的完整状态。
- "版本"Tab 默认仍展示项目级版本历史。

**交互效果**：
- 项目级版本列表按时间倒序排列，最新版本置顶。
- 恢复项目级版本时，整个项目回滚到该版本状态。

#### 2.2.2 页面级版本（新增）

**用户视角**：
- 用户在页面列表中选中某个具体页面后，可查看该页面独有的版本历史。
- 页面级版本只记录该页面代码和配置的变化，不存储其他页面的内容。
- 用户可选择恢复到该页面的某个历史版本，仅影响当前页面，不影响项目中其他页面。

**交互效果**：
- 在页面列表或页面编辑区域提供"查看页面历史"入口。
- 页面级版本列表同样显示版本号、保存时间、保存者、备注。
- 恢复到页面级历史版本时，仅替换当前页面的代码和配置，项目级版本记录同步新增一条（备注"从页面历史恢复"）。

#### 2.2.3 颗粒度切换

**用户视角**：
- 在"版本"Tab 或页面编辑区，用户可在"项目版本"与"页面版本"两种视角间切换。
- 切换颗粒度时，版本列表实时刷新为对应维度的历史记录。

**交互效果**：
- 提供明显的切换控件（如 Tabs 或下拉选择），当前颗粒度状态清晰可辨。
- 切换时不影响编辑器中的当前内容，仅刷新版本列表。

---

### 2.3 历史版本预览能力

#### 2.3.1 预览项目级历史版本

**用户视角**：
- 用户在项目级版本列表中 hover 或点击某个历史版本时，可触发预览。
- 系统在新标签页或弹窗中展示该版本整个项目的实际运行效果。
- 预览为只读模式，用户无法修改内容。

**交互效果**：
- 每个版本卡片提供"预览"按钮或点击版本区域触发预览。
- 预览加载时显示 loading 状态，加载完成后展示该版本的完整页面效果。
- 预览窗口支持切换查看项目内的不同页面。
- 关闭预览后，用户回到编辑页，当前编辑状态不受影响。

#### 2.3.2 预览页面级历史版本

**用户视角**：
- 用户在页面级版本列表中，可预览该页面在某一历史版本下的渲染效果。
- 预览仅展示该单页面的效果，使用当时的页面配置和全局配置。

**交互效果**：
- 预览方式与项目级一致，但范围限定为当前页面。
- 预览中同样不可编辑，仅用于效果确认。

#### 2.3.3 预览与恢复的衔接

**用户视角**：
- 预览确认效果满意后，用户可在预览界面一键恢复该版本。
- 恢复前仍需二次确认，因为恢复是破坏性操作。

**交互效果**：
- 预览窗口提供"恢复到此版本"按钮。
- 点击后弹出确认对话框，确认后执行恢复并关闭预览。
- 恢复成功后，编辑器内容刷新为对应版本，版本列表同步更新。

## 三、业务规则与约束

### 3.1 版本显示规则

| 规则 | 说明 |
|:-----|:-----|
| 用户名字段 | 版本信息中 `savedBy` 为用户名，若为空或无效则显示"未知用户" |
| 文件数量隐藏 | 版本卡片不再展示 `fileCount` 字段 |
| 保留字段 | 版本号、保存时间、保存者、备注继续保留 |

### 3.2 版本颗粒度规则

| 规则 | 说明 |
|:-----|:-----|
| 项目级触发 | 用户点击"保存"时，始终生成项目级版本快照 |
| 页面级触发 | 用户可手动为单个页面创建快照，或在自动保存时选择性生成页面级版本 |
| 存储策略 | 页面级快照仅存储该页面的 `index.tsx` 和 `config.schema.json`，不复制其他页面 |
| 恢复策略 | 恢复页面级版本时，仅替换目标页面文件，并自动生成一个新的项目级版本记录 |
| 版本号共享 | 项目级和页面级版本使用统一的版本号序列（v1, v2, ...），便于追溯时序关系 |

### 3.3 预览规则

| 规则 | 说明 |
|:-----|:-----|
| 只读预览 | 历史版本预览不允许任何编辑操作 |
| 隔离运行 | 预览环境与当前编辑环境完全隔离，预览不会修改任何工作空间文件 |
| 配置生效 | 预览时使用该历史版本当时的全局配置和页面配置 |
| 缓存策略 | 预览可复用现有编译缓存机制，避免重复编译相同版本 |

### 3.4 数据安全规则

| 规则 | 说明 |
|:-----|:-----|
| 预览不创建会话 | 预览历史版本时不创建新的编辑会话 |
| 恢复前确认 | 无论是否经过预览，恢复操作始终需要用户二次确认 |
| 恢复即新建 | 恢复到历史版本后，当前状态自动保存为一个新版本（现有机制保留） |

## 四、核心流程

### 4.1 查看项目级版本历史（优化后）

```
用户打开编辑页 → 点击"版本"Tab
      │
      ▼
加载项目级版本列表
      │
      ▼
列表展示：版本号 + 保存时间 + 保存者用户名 + 备注
               （不再显示 UUID 和文件数量）
      │
      ▼
用户 hover / 点击版本 → 触发预览 → 新窗口/弹窗展示该版本效果
      │
      ▼
用户确认恢复 → 二次确认 → 执行恢复 → 生成新版本
```

### 4.2 查看页面级版本历史（新增）

```
用户在页面列表选中某个页面
      │
      ▼
点击"查看页面历史"入口
      │
      ▼
加载该页面的独立版本列表
      │
      ▼
列表展示：版本号 + 保存时间 + 保存者用户名 + 备注
      │
      ▼
用户预览某版本 → 仅展示该单页面渲染效果
      │
      ▼
用户确认恢复 → 仅替换当前页面文件 → 同步生成项目级新版本
```
