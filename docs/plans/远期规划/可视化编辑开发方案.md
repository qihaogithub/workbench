# 可视化编辑开发方案

> 版本：v1.1
> 创建日期：2026-04-19
> 最近更新：2026-06-20
> 状态：待开发
> 前置依赖：动态编译预览系统已上线

---

## 一、方案定位

可视化编辑不是把代码编辑器藏起来，也不是做一个完整设计器，而是在现有 Demo 编辑工作台里增加一层“可看见、可点选、可批注、可确认”的编辑体验。

目标体验对齐两类产品能力：

1. **Codex 批注**：改动必须可定位、可审阅、可接受或拒绝，用户知道每一次修改会落到哪一段代码。
2. **网页内编辑**：用户可以直接在预览区点选元素，像操作网页一样修改文字、样式、布局和列表项，修改后立即看到结果。

最终形态是一个“预览即编辑”的工作流：

```text
点选元素 → 看到选区和源码位置 → 直接修改/提交批注 → 生成补丁 → 预览刷新 → 接受、撤销或继续调整
```

## 二、体验原则

### 2.1 一切修改都可看见

- 预览区必须显示 hover 边框、选中边框、父子层级提示和元素名称。
- 右侧面板必须显示选中元素的来源文件、行号、标签/组件名、当前 className、可编辑属性。
- 每次写回代码前，都应生成一张“变更卡片”，说明改了什么、影响哪个元素、是否会触发重新编译。

### 2.2 一切修改都可撤销

- 用户直接编辑和 AI 辅助编辑都进入同一套变更队列。
- 每个变更支持接受、撤销、重新生成和跳转代码。
- 变更历史至少保留 20 步，Ctrl+Z / Ctrl+Y 与按钮操作一致。

### 2.3 可视化编辑与 AI 不互相抢控制权

- 用户点选元素后，可以手动改，也可以把选区交给 AI。
- AI 不直接静默改代码，而是生成可审阅的批注式建议。
- 用户接受后才写回，拒绝后预览恢复原状。

### 2.4 复杂场景明确降级

遇到动态 className、条件渲染、复杂表达式、无法稳定定位的 JSX 时，不强行编辑。界面显示原因，并提供：

- 跳转到代码编辑器
- 交给 AI 生成建议
- 只添加批注，不写回

## 三、核心功能范围

### 3.1 本期目标

| 能力 | 说明 | 体验目标 |
|:-----|:-----|:---------|
| 元素点选 | 在预览区 hover/点击元素 | 像浏览器 DevTools 一样准确 |
| 批注式变更 | 每次修改生成变更卡片 | 像 Codex 批注一样可审阅 |
| 文本内联编辑 | 双击文本直接修改内容 | 像网页 CMS 一样顺手 |
| 样式面板 | 修改 Tailwind 类名和常用视觉属性 | 不需要记类名 |
| 结构操作 | 插入、删除、上移、下移元素 | 有确认、有撤销 |
| 列表项操作 | 对 `.map()` 列表做增删复制 | 面向运营配置场景 |
| AI 选区编辑 | 对选中元素发起局部 AI 修改 | 只改当前选区或明确邻域 |
| 代码跳转 | 从选区跳到 Monaco 对应位置 | 可解释、可追踪 |

### 3.2 非目标

以下能力不在第一期范围内：

- 完整拖拽式页面搭建器
- 任意 JSX 表达式重构
- 复杂业务逻辑、事件处理、状态管理修改
- 多文件组件重命名和跨文件迁移
- Figma 级自由画布布局

## 四、产品形态

### 4.1 工作台布局

沿用现有三栏工作台，但让中间预览区成为主编辑面：

```text
┌──────────────┬────────────────────────────┬────────────────────┐
│ AI / 会话区  │ 预览区（可点选、可内联编辑） │ 属性 / 批注 / 结构区 │
│              │                            │                    │
│ 局部指令      │ Hover 边框                  │ 选中元素信息         │
│ 变更解释      │ 选中边框                    │ 样式控件             │
│ 历史记录      │ 批注锚点                    │ 变更卡片             │
└──────────────┴────────────────────────────┴────────────────────┘
```

### 4.2 预览区交互

| 操作 | 行为 |
|:-----|:-----|
| 鼠标悬停 | 显示元素边框、标签名、简短 class 摘要 |
| 单击元素 | 选中元素，右栏展示属性和批注 |
| 双击文本 | 进入文本内联编辑 |
| Alt/Option + 单击 | 选中父级容器 |
| Esc | 退出选中或内联编辑 |
| 右键元素 | 打开快捷菜单：编辑文字、修改样式、复制、删除、交给 AI |
| 点击批注锚点 | 打开对应变更卡片 |

### 4.3 右侧面板

右侧面板包含四个 Tab：

| Tab | 用途 |
|:----|:-----|
| 配置 | 保留现有 Schema 配置能力 |
| 样式 | 修改 Tailwind 类名、颜色、间距、字号、圆角、布局 |
| 结构 | 插入、删除、上移、下移、复制元素或列表项 |
| 批注 | 展示当前选区相关变更、AI 建议和待确认补丁 |

### 4.4 批注式变更卡片

每个可视化修改都生成一张变更卡片，卡片信息包括：

| 字段 | 说明 |
|:-----|:-----|
| 标题 | 例如“修改标题字号”“替换按钮文案”“删除第 3 个卡片” |
| 选区 | 元素标签、组件名、文件、行号 |
| 修改前 | 原文案、原 className 或原 JSX 片段摘要 |
| 修改后 | 新文案、新 className 或新 JSX 片段摘要 |
| 风险提示 | 动态表达式、列表项、结构删除等风险 |
| 操作 | 接受、拒绝、撤销、重新生成、跳转代码 |

变更卡片的状态流转：

```text
草稿 → 已应用预览 → 已接受
  │         │
  │         ├── 撤销 → 已撤销
  │         └── 拒绝 → 已拒绝
  └── 取消 → 已取消
```

## 五、编辑能力设计

### 5.1 元素点选与定位

预览编译阶段为可编辑节点注入稳定定位信息：

| 属性 | 用途 |
|:-----|:-----|
| `data-visual-node-id` | 本次编译内的唯一节点 ID |
| `data-source-file` | 来源文件 |
| `data-source-start` | JSX 起始位置 |
| `data-source-end` | JSX 结束位置 |
| `data-component-name` | 标签名或组件名 |
| `data-edit-capabilities` | 当前节点支持的编辑能力 |

设计要求：

- hover 命中最内层可编辑节点。
- 用户可一键切换到父级节点，避免卡片/容器难以选中。
- 选中节点在重新编译后应尽量恢复，优先按 `data-visual-node-id`，失败时按源码位置恢复。

### 5.2 文本内联编辑

适用范围：

- 静态 JSX 文本：`<h1>标题</h1>`
- 简单字符串属性：`<Button>提交</Button>`
- 简单数组对象中的字符串字段：`{ title: '标题' }`

交互规则：

1. 双击文本后进入局部输入态。
2. Enter 保存，Esc 取消。
3. 保存后先生成变更卡片，再写回编辑器并刷新预览。
4. 如果文本来自变量或表达式，提示“该文本来自动态数据”，提供跳转代码和 AI 修改。

### 5.3 样式编辑

样式面板分为三层：

| 层级 | 用途 |
|:-----|:-----|
| 快捷控件 | 字号、字重、颜色、背景、间距、圆角、布局、对齐 |
| 类名标签 | 展示并删除当前 Tailwind class |
| 原始编辑 | 直接编辑完整 className 字符串 |

关键要求：

- 使用 `tailwind-merge` 合并冲突类名。
- 控件修改应只替换同类 class，不破坏其他 class。
- 动态 className 不直接修改，降级为批注或 AI 建议。
- 样式修改进入变更卡片，可以接受或撤销。

### 5.4 结构编辑

结构面板支持：

| 操作 | 说明 | 保护规则 |
|:-----|:-----|:---------|
| 上方插入 | 在当前节点前插入预设节点 | 根节点不可用 |
| 下方插入 | 在当前节点后插入预设节点 | 根节点不可用 |
| 包裹 | 用 div/Card/section 包裹当前节点 | 需保持 JSX 合法 |
| 删除 | 删除当前节点及子节点 | 删除前确认 |
| 复制 | 复制当前节点到下方 | 保持缩进 |
| 上移/下移 | 与同级节点交换位置 | 边界自动禁用 |

第一期预设节点：

- `div`
- `section`
- `h1`
- `h2`
- `p`
- `img`
- `Button`
- `Badge`
- `Card`
- `div.container`
- `div.flex`
- `div.grid`

### 5.5 列表项编辑

当节点来自 `.map()` 渲染时，右栏展示列表操作：

| 操作 | 行为 |
|:-----|:-----|
| 复制当前项 | 在数组中复制当前对象 |
| 在前方插入 | 根据当前对象生成新对象 |
| 在后方插入 | 根据当前对象生成新对象 |
| 删除当前项 | 删除数组元素，保留撤销 |
| 编辑字段 | 对简单字符串/数字/布尔字段显示表单 |

限制：

- 只支持本地静态数组或当前文件内常量数组。
- 不支持接口数据、异步数据和复杂计算结果的列表项直接写回。
- 不支持时显示“数据来源不可直接编辑”，并提供 AI 建议入口。

### 5.6 AI 选区编辑

选中元素后，AI 输入框自动带上选区上下文：

```text
当前选区：Button，packages/author-site/.../page.tsx:42
用户指令：把这个按钮做得更像主行动按钮
约束：优先修改当前元素 className，除非必须不要改其他区域
```

AI 输出不直接覆盖代码，而是生成批注式建议：

- 建议标题
- 修改原因
- 受影响节点
- 补丁摘要
- 接受/拒绝按钮

这让 AI 编辑和手动编辑共享同一套确认机制。

## 六、技术路线

### 6.1 总体架构

```text
源码 / Monaco
   │
   ▼
可编辑节点标记器
   │
   ▼
动态编译预览
   │
   ▼
预览交互层
   │
   ├── 选区状态
   ├── 内联编辑
   └── 批注锚点
   │
   ▼
变更引擎
   │
   ├── 文本 Patch
   ├── className Patch
   ├── JSX 结构 Patch
   └── 列表项 Patch
   │
   ▼
变更队列 / 批注卡片
   │
   ▼
写回 Monaco → 重新编译 → 恢复选区
```

### 6.2 定位策略

原方案使用 `data-source-line` 和 `data-source-file`，但行号在插入、删除后容易漂移。升级后采用“源码范围 + 节点指纹”的组合策略：

| 定位信息 | 作用 |
|:---------|:-----|
| 文件路径 | 确定写回目标 |
| 起止 offset | 精准 patch |
| 起止行列 | UI 展示与代码跳转 |
| 节点类型 | 判断可编辑能力 |
| 节点指纹 | 重新编译后恢复选区 |

节点指纹可由标签名、静态 className、文本摘要、父级路径共同生成。它不是永久 ID，只用于当前编辑会话内的恢复。

### 6.3 写回策略

第一期可以继续使用轻量字符串 patch，但必须封装为统一补丁格式：

```typescript
interface VisualEditPatch {
  id: string;
  title: string;
  file: string;
  range: {
    startOffset: number;
    endOffset: number;
    startLine: number;
    endLine: number;
  };
  before: string;
  after: string;
  kind: 'text' | 'className' | 'structure' | 'listItem' | 'aiSuggestion';
  status: 'draft' | 'previewed' | 'accepted' | 'rejected' | 'reverted';
}
```

写回前必须校验：

- `before` 与当前源码范围一致，避免覆盖用户刚刚手动修改的内容。
- Patch 后能通过动态编译。
- 失败时恢复原文并在批注卡片中显示失败原因。

### 6.4 解析策略

推荐分阶段演进：

| 阶段 | 策略 | 适用范围 |
|:-----|:-----|:---------|
| P0 | 行号 + 字符串 patch | 快速验证交互闭环 |
| P1 | TypeScript AST / Babel parser 只读定位 | 提升 JSX 边界识别准确度 |
| P2 | AST patch + prettier 格式化 | 稳定结构编辑 |

为了达到“像 Codex 批注一样可靠”的目标，建议最晚在 P1 引入只读 AST 定位。结构编辑继续字符串 patch 也可以，但节点边界、属性识别、文本节点识别不应长期依赖正则。

### 6.5 与现有系统集成点

| 模块 | 需要新增/调整 |
|:-----|:--------------|
| 动态编译服务 | 注入可编辑节点元数据 |
| 预览容器 | 增加 hover/selection overlay 和事件委托 |
| Monaco 编辑器 | 支持跳转、局部 range 高亮、patch 写回 |
| AI 会话区 | 支持选区上下文和批注式建议 |
| 右侧面板 | 新增样式、结构、批注 Tab |
| 草稿保存 | 将可视化 patch 纳入现有保存链路 |

## 七、实现阶段

### 阶段 1：选区与批注闭环（2-3 天）

| 任务 | 验收标准 |
|:-----|:---------|
| 注入 `data-visual-node-id` 和源码范围 | 预览节点可定位到源码 |
| 预览区 hover/selection overlay | hover、选中、父级切换可用 |
| 右栏展示选中元素信息 | 能看到组件名、文件、行号、className |
| 变更卡片模型 | 手动构造 patch 后可接受/拒绝/撤销 |
| Monaco 跳转 | 点击“跳转代码”定位到对应范围 |

### 阶段 2：文本和样式编辑（3-4 天）

| 任务 | 验收标准 |
|:-----|:---------|
| 双击文本内联编辑 | 静态文本可直接修改并生成变更卡片 |
| Tailwind 快捷控件 | 字号、颜色、间距、圆角、布局可修改 |
| className 标签编辑 | 可添加、删除、原始编辑 className |
| 动态 className 降级 | 不可编辑时有明确提示和 AI 入口 |
| Patch 校验 | before 不匹配或编译失败时不破坏代码 |

### 阶段 3：结构与列表编辑（4-5 天）

| 任务 | 验收标准 |
|:-----|:---------|
| JSX 边界识别 | 能稳定识别元素起止范围 |
| 插入/删除/复制/移动 | 操作后 JSX 合法、缩进清晰 |
| 包裹节点 | 能用 div/section/Card 包裹选区 |
| 列表项识别 | 静态数组 map 项可定位 |
| 列表项增删复制 | 操作后列表正常渲染且可撤销 |

### 阶段 4：AI 选区编辑（2-3 天）

| 任务 | 验收标准 |
|:-----|:---------|
| AI 输入携带选区上下文 | AI 明确知道当前选中元素 |
| AI 输出批注式建议 | 建议不会直接静默写回 |
| 接受/拒绝 AI patch | 与手动 patch 同一套状态流转 |
| 局部修改约束 | 默认只改选区，越界修改需要提示 |

### 阶段 5：体验打磨与测试（2-3 天）

| 任务 | 验收标准 |
|:-----|:---------|
| 快捷键 | Esc、Enter、Ctrl+Z、Ctrl+Y 可用 |
| 选区恢复 | 重新编译后尽量保持选中元素 |
| 错误提示 | 定位失败、编译失败、动态数据不可编辑都有可读提示 |
| E2E 覆盖 | 文本、样式、结构、AI 建议主流程有 Playwright 测试 |

## 八、关键风险与应对

| 风险 | 概率 | 影响 | 应对 |
|:-----|:-----|:-----|:-----|
| 正则定位 JSX 不稳定 | 高 | 高 | P1 引入只读 AST 定位，复杂节点降级 |
| 用户修改与 patch 冲突 | 中 | 高 | 写回前校验 before，不匹配则要求重新应用 |
| 预览点击触发业务事件 | 中 | 中 | 编辑模式下捕获事件并阻止默认交互，提供预览/编辑模式切换 |
| AI 修改越界 | 中 | 高 | AI patch 显示影响范围，越界时提示用户确认 |
| 动态数据无法直接编辑 | 高 | 中 | 明确提示数据来源，提供代码跳转和 AI 修改 |
| 删除结构造成页面空白 | 中 | 高 | 删除前确认、保留撤销、编译失败自动回滚 |
| 批注过多影响操作效率 | 中 | 中 | 支持“自动接受低风险样式修改”，但默认关闭 |

## 九、验收标准

### 9.1 体验验收

- [ ] 用户能在预览区准确 hover 和选中元素。
- [ ] 用户能像网页编辑器一样双击修改静态文本。
- [ ] 用户能通过右栏控件修改常用 Tailwind 样式。
- [ ] 每次修改都有批注式变更卡片，可接受、拒绝、撤销。
- [ ] 用户能从批注卡片跳转到 Monaco 对应源码范围。
- [ ] AI 对选区的修改以建议卡片出现，不静默覆盖代码。

### 9.2 工程验收

- [ ] Patch 写回前校验 `before` 内容，避免覆盖并发修改。
- [ ] Patch 后触发动态编译，失败时自动回滚。
- [ ] 动态 className、条件渲染、异步数据列表有降级提示。
- [ ] 重新编译后能恢复选区或给出明确失败提示。
- [ ] 操作历史至少支持 20 步撤销/重做。

### 9.3 性能验收

- [ ] hover 响应小于 50ms。
- [ ] 普通样式修改到预览刷新小于 500ms。
- [ ] 可编辑节点注入开销小于 20ms。
- [ ] 连续 10 次样式修改不会造成明显卡顿。

- 接受/拒绝/撤销 patch 的状态流转

- 对选中按钮发起 AI 局部修改并接受建议

## 十一、后续迭代方向

| 优先级 | 方向 | 说明 |
|:-------|:-----|:-----|
| P1 | AST 定位升级 | 用 AST 提升 JSX 边界和属性识别准确度 |
| P1 | 批注聚合 | 按元素、文件、风险等级聚合变更 |
| P2 | 拖拽排序 | 在结构面板或预览区直接拖拽同级节点 |
| P2 | 响应式编辑 | 针对移动端/桌面端分别调整 className |
| P2 | 设计 Token 面板 | 优先使用项目颜色、间距、字号 token |
| P3 | 属性编辑 | 支持 id、aria-*、data-*、src、alt 等安全属性 |
| P3 | 多文件组件支持 | 选中组件实例后跳转到组件定义并生成安全 patch |
| P4 | 变更回放 | 像审阅模式一样回放一组可视化修改 |

---

## 十二、推荐优先级

如果目标是尽快达到“像 Codex 批注和网页内编辑一样好用”，优先级应调整为：

1. **先做批注式变更闭环**：没有接受/拒绝/撤销，再多控件都不可靠。
2. **再做文本和样式内联编辑**：这是用户感知最强的“网页中编辑”体验。
3. **然后做 AI 选区建议**：让 AI 成为当前选区的助手，而不是全局改写器。
4. **最后做结构和列表编辑**：这部分风险更高，需要前面的 patch 机制兜底。

原 v1.0 方案中的“可视化样式面板”仍然成立，但它应当被放进更大的“选区 + 批注 + patch”框架里实现。这样体验上更接近 Codex，工程上也更安全。
