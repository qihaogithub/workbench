# 编辑页保存失败问题分析报告

## 问题描述

用户在编辑页点击保存时，显示"保存失败，合并到Demo失败"。

## 问题定位

### 1. 错误来源

错误信息 "合并到 Demo 失败" 来自 [`demo/[id]/edit/page.tsx`](file:///e:/重要文件/Programming/1_Work/opencode工作台/packages/web/src/app/demo/[id]/edit/page.tsx#L215-L216) 第 215-216 行：

```typescript

  method: "POST",

if (!mergeRes.ok) {
  throw new Error("合并到 Demo 失败");
```

### 2. 保存流程分析

用户点击保存按钮后，执行 [`handleSave`](file:///e:/重要文件/Programming/1_Work/opencode工作台/packages/web/src/app/demo/[id]/edit/page.tsx#L189-L235) 函数，流程如下：

1. **验证**：检查 `validationResult.isValid`，如果无效则提示"请修复所有错误后再保存"
2. **保存文件**：调用 `PUT /api/sessions/${sessionId}/files` 保存 code 和 schema
3. **合并到 Demo**：调用 `POST /api/sessions/${sessionId}/merge` 将临时 session 合并到 Demo

### 3. Merge API 流程

[`merge/route.ts`](file:///e:/重要文件/Programming/1_Work/opencode工作台/packages/web/src/app/api/sessions/[sessionId]/merge/route.ts) 的处理流程：

```
1. 检查 session 是否存在 (sessionExists)
   ↓ 不存在则返回 404 SESSION_NOT_FOUND
2. 获取 session 元数据 (getSessionMeta)
   ↓ 
3. 检查 session 是否过期 (isSessionExpired)
   ↓ 过期则返回 410 SESSION_EXPIRED
4. 执行合并 (mergeSession)
   ↓ 失败则返回 500 FILE_WRITE_ERROR
5. 返回成功
```

### 4. `mergeSession` 函数分析

[`fs-utils.ts`](file:///e:/重要文件/Programming/1_Work/opencode工作台/packages/web/src/lib/fs-utils.ts#L344-L391) 中的 `mergeSession` 函数：

```typescript
export function mergeSession(sessionId: string): boolean {
  const sessionMeta = getSessionMeta(sessionId);  // 可能失败点1

  if (!sessionMeta) {
    return false;
  }

  const { demoId: projectId } = sessionMeta;

  if (!projectExists(projectId)) {  // 可能失败点2
    return false;
  }

  const sessionPath = getSessionPath(sessionId);  // ⚠️ 缺少 projectId 参数
  const projectPath = getProjectPath(projectId);
  const workspacePath = path.join(projectPath, "workspace");

  // 文件操作 - 可能失败点3
  fs.rmSync(workspacePath, { recursive: true, force: true });
  fs.cpSync(sessionPath, workspacePath, { recursive: true });
  fs.rmSync(path.join(workspacePath, ".session.json"), { force: true });
  fs.rmSync(sessionPath, { recursive: true, force: true });

  return true;
}
```

### ⚠️ 发现的设计问题

**`mergeSession` 中 `getSessionPath` 缺少 `projectId` 参数**

查看 [`getSessionPath`](file:///e:/重要文件/Programming/1_Work/opencode工作台/packages/web/src/lib/fs-utils.ts#L59-L65) 函数：

```typescript
export function getSessionPath(sessionId: string, projectId?: string): string {
  if (projectId) {
    return path.join(SESSIONS_DIR, projectId, sessionId);
  }
  const foundPath = findSessionPath(sessionId);  // 需要遍历目录
  return foundPath || path.join(SESSIONS_DIR, sessionId);
}
```

当 `mergeSession` 调用 `getSessionPath(sessionId)` 时没有传入 `projectId`，导致：

1. **需要遍历目录**：调用 `findSessionPath` 进行全目录搜索
2. **查找逻辑复杂**：`findSessionPath` 需要遍历 `sessions/{userId}/{projectId}/{sessionId}` 三层结构
3. **可能查找失败**：在目录结构不匹配或存在竞态条件时可能找不到

**根本原因**：`mergeSession` 在调用 `getSessionMeta` 获取到 `projectId` 后，应该将 `projectId` 传给 `getSessionPath`，而不是再次调用 `findSessionPath`。

## 可能的失败原因

### 原因1：Session 元数据获取失败

`getSessionMeta(sessionId)` 返回 `null`，可能原因：

- `findSessionPath(sessionId)` 无法找到 session 目录
- `.session.json` 文件不存在或损坏

**Session 目录结构**：
- 新结构：`sessions/{userId}/{projectId}/{sessionId}/`
- 旧结构：`sessions/{projectId}/{sessionId}/`

### 原因2：项目不存在

`projectExists(projectId)` 返回 `false`，可能原因：

- 项目目录被删除
- session 元数据中的 `demoId` 指向不存在的项目

### 原因3：文件操作失败

文件操作过程中发生异常：

- 权限问题
- 磁盘空间不足
- 文件被占用

## 总结

问题出在 `mergeSession` 函数返回 `false`，具体原因需要通过添加的日志来确认。**最可能的根本原因是**：

### 主要原因：`mergeSession` 调用 `getSessionPath` 时缺少 `projectId` 参数

`mergeSession` 在获取 `sessionMeta` 后已经知道 `projectId`，但调用 `getSessionPath(sessionId)` 时没有传入 `projectId`，导致需要通过 `findSessionPath` 进行目录遍历搜索。

在以下情况下可能查找失败：
- 目录结构与预期不符（新旧结构混用）
- 多层目录嵌套较深时遍历逻辑出错
- 文件系统竞态条件

### 次要原因：

1. `findSessionPath` 无法正确找到 session 目录（目录结构遍历问题）
2. session 元数据中的 `demoId` 指向的项目不存在
3. 文件操作权限问题

请运行开发服务器并重现问题，查看控制台日志以确定具体失败点。
