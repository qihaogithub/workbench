# AionUi 目录结构详解

> 本文档详细讲解 AionUi 项目的目录结构和各模块职责。

## 项目概述

AionUi 是一个免费开源的 AI Agent Cowork 平台，基于 Electron 构建，支持多模型、多平台、远程访问和 24/7 自动化任务。项目采用多进程架构，包含主进程、渲染进程和工作进程。

## 核心源码目录 (`src/`)

AionUi 采用 Electron 多进程架构，`src/` 目录按进程职责划分：

```
src/
├── renderer/          # 渲染进程 — React UI，无 Node.js API
├── process/           # 主进程 — 所有 Node.js/Electron 业务逻辑
├── common/            # 公共层 — 跨进程类型、适配器、工具
├── preload.ts         # IPC 桥接 — contextBridge 连接主进程与渲染进程
└── index.ts           # 主进程入口点
```

### 进程边界规则

| 进程 | 可使用 | 不可使用 |
|------|--------|----------|
| **主进程** (`src/process/`) | Node.js, Electron 主进程 API | DOM API, React |
| **渲染进程** (`src/renderer/`) | DOM API, React, 浏览器 API | Node.js API, Electron 主进程 API |
| **工作进程** (`src/process/worker/`) | Node.js API | DOM API, Electron API |

### 关键模块说明

#### `agent/` - AI Agent 连接器

负责与各种 AI Agent 平台建立连接：

- **acp/**: ACP 协议适配器，支持 Claude Code、Codex 等 CLI Agent
- **aionrs/**: 内置 Agent 引擎，无需外部依赖
- **gemini/**: Gemini CLI 集成，支持 Google 认证
- **openclaw/**: OpenClaw 平台集成

#### `bridge/` - IPC 桥接

主进程与渲染进程之间的通信桥梁，每个 Bridge 负责一个业务域：

- 命名规范: `<domain>Bridge.ts`
- 通过 `preload.ts` 暴露给渲染进程

#### `channels/` - 多渠道消息

支持从多种平台远程控制 Agent：

- Telegram、飞书、钉钉、微信等
- 统一的消息协议和动作执行

#### `extensions/` - 扩展系统

插件化架构，支持：

- 自定义助手 (Assistant)
- 技能 (Skill)
- MCP 服务器
- 主题和国际化

#### `webserver/` - WebUI 服务器

Express + WebSocket 服务器，支持：

- 远程浏览器访问
- JWT 认证
- 实时通信

## 运行模式

AionUi 支持四种运行模式：

### 1. Desktop 模式 (`pnpm start`)

```
Electron 窗口 ←IPC→ Bridge/Services/DB
     ↑
     └─ 可选 WebUI (WebSocket)
```

### 2. WebUI 模式 (`pnpm webui`)

```
浏览器 ←WebSocket→ Bridge/Services/DB + 完整 Electron API
```

### 3. Server 模式 (`pnpm server:start`)

```
浏览器 ←WebSocket→ Bridge/Services/DB (纯 Node.js，无 Electron API)
```

### 4. Remote 模式

支持跨网络访问，通过 JWT 认证。

## 总结

AionUi 是一个架构清晰的 Electron 应用：

1. **多进程架构** - 主进程、渲染进程、工作进程各司其职
2. **模块化设计** - Agent、Bridge、Service、Channel 等模块独立
3. **扩展性强** - 支持插件、技能、MCP 工具扩展
4. **多平台支持** - Desktop、WebUI、Mobile、Remote 多种访问方式
5. **多 Agent 集成** - 内置 Agent + 外部 CLI Agent 统一管理
