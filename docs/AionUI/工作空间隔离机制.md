## AionUI 工作空间隔离机制

### 1. 核心架构：每个会话绑定独立工作空间

每个对话（`TChatConversation`）都有一个 `extra` 字段，存储了工作空间信息：

```typescript
// 会话类型定义（简化）
interface IChatConversation {
  id: string;
  type: 'gemini' | 'acp' | 'codex' | ...;
  extra: {
    workspace: string;        // 工作空间路径
    customWorkspace?: boolean; // true=用户指定，false=系统临时创建
    // ...其他配置
  };
}
```

### 2. 两种工作空间类型

**用户指定工作空间** (`customWorkspace: true`)
- 用户通过对话框选择的目录
- 显示在 Tab 标签中，支持多 Tab 切换
- 持久化保存，下次打开仍在该目录

**临时工作空间** (`customWorkspace: false`)
- 系统自动创建，命名格式：`{backend}-temp-{timestamp}`
- 例如：`gemini-temp-1712345678901`
- 不显示在 Tab 中，会话结束后可清理

### 3. 工作空间创建流程

[initAgent.ts:140-159](file:///e:\重要文件\Programming\1_Work\opencode工作台\AionUi\src\process\utils\initAgent.ts#L140-L159) 中定义了核心逻辑：

```typescript
const buildWorkspaceWidthFiles = async (
  defaultWorkspaceName: string,
  workspace?: string,
  providedCustomWorkspace?: boolean
) => {
  const customWorkspace = providedCustomWorkspace !== undefined 
    ? providedCustomWorkspace 
    : !!workspace;

  if (!workspace) {
    // 没有指定工作空间，创建临时目录
    const tempPath = getSystemDir().workDir;
    workspace = path.join(tempPath, defaultWorkspaceName);
    await fs.mkdir(workspace, { recursive: true });
  } else {
    // 规范化用户指定的路径
    workspace = path.resolve(workspace);
  }

  return { workspace, customWorkspace };
};
```

### 4. Agent 执行时的隔离机制

**方式一：子进程 cwd 设置**

Agent 启动 CLI 子进程时，通过 `cwd` 参数设置工作目录：

```typescript
// NanobotConnection.ts
this.child = spawn('nanobot', args, {
  cwd: this.workingDir,  // 关键：设置工作目录
  stdio: ['pipe', 'pipe', 'pipe'],
});
```

**方式二：CLI 配置传递**

Gemini Agent 通过配置对象传递工作空间：

```typescript
// config.ts
const config = new Config({
  targetDir: workspace,    // 目标目录
  cwd: workspace,          // 当前工作目录
  // ...
});
```

**方式三：ACP Agent 连接参数**

```typescript
// AcpAgentManager.ts
this.agent = new AcpAgent({
  id: data.conversation_id,
  workingDir: data.workspace,  // 从会话 extra.workspace 获取
  // ...
});
```

### 5. 工作空间更换机制

[useWorkspaceSelector.ts](file:///e:\重要文件\Programming\1_Work\opencode工作台\AionUi\src\renderer\hooks\file\useWorkspaceSelector.ts) 实现了工作空间切换：

```typescript
export const useWorkspaceSelector = (conversationId: string) => {
  return useCallback(async () => {
    // 1. 让用户选择新目录
    const workspacePath = await ipcBridge.dialog.showOpen.invoke({ 
      properties: ['openDirectory'] 
    });
    
    // 2. 更新会话的 extra.workspace
    const nextExtra = { ...conversation.extra, workspace: workspacePath };
    await ipcBridge.conversation.update.invoke({ 
      id: conversationId, 
      updates: { extra: nextExtra } 
    });
    
    // 3. 刷新 UI 和工作区
    emitter.emit('gemini.workspace.refresh');
  }, [conversationId]);
};
```

### 6. 文件变更追踪

[WorkspaceSnapshotService.ts](file:///e:\重要文件\Programming\1_Work\opencode工作台\AionUi\src\process\services\WorkspaceSnapshotService.ts) 提供了两种模式：

- **git-repo 模式**：如果工作空间是 Git 仓库，使用 Git 追踪变更
- **snapshot 模式**：非 Git 目录，创建快照对比变更

### 7. 隔离保证总结

| 层面 | 实现方式 |
|------|----------|
| **数据层** | 每个会话在数据库中有独立的 `extra.workspace` 字段 |
| **进程层** | Agent 子进程通过 `cwd` 参数绑定到特定目录 |
| **配置层** | CLI 配置对象携带 `targetDir`/`cwd` 参数 |
| **UI 层** | Tab 系统按会话 ID 区分，每个 Tab 显示对应工作空间 |
| **文件层** | 快照服务按 `workspacePath` 隔离追踪变更 |

这种设计确保了：
1. **路径隔离**：每个 Agent 只能操作自己工作空间内的文件
2. **状态隔离**：不同会话的工作空间状态互不干扰
3. **可追溯性**：通过快照服务追踪每个工作空间的文件变更
4. **灵活性**：支持用户随时切换工作空间，或使用临时空间
