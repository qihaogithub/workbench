01 / 项目是什么

Why now

上线不是终点，协作才是瓶颈

组件从“能写出来”到“能被别人稳定使用”，中间有三段断点。

01

代码生成和人工修改割裂

需求变化后，代码、预览、文件状态和历史回溯经常分散在不同工具里。

02

非技术同事难以参与配置

组件参数、Schema、预览效果如果依赖开发解释，协作成本会持续增加。

03

交付之后缺少统一入口

组件浏览、版本、嵌入、截图和回归验证需要可被团队复用的标准链路。

02 / 问题背景

一句话定位

一个把“创作、使用、集成”打通的组件工作台。

开发侧用 AI 和工作空间推进组件生产；业务侧通过配置和预览参与验收；外部系统通过 iframe 协议把组件直接接入业务场景。

创作端

AI 对话、代码修改、Schema、预览、版本

使用端

浏览项目、调整配置、验收效果

嵌入集成

iframe + postMessage 接入外部系统

统一项目语义

03 / 平台定位

Users

三类同事都能进入同一条链路

不是只给开发用，也不是另一个低代码壳。

开发者 / 设计师

负责把想法变成可维护组件

  • 用自然语言推进组件生成和修改
  • 维护配置 Schema 和预览状态
  • 用快照和版本保护关键变更

产品 / 运营

负责看得懂、调得动、验得准

  • 在使用端浏览组件项目
  • 通过表单调整配置参数
  • 用真实预览参与上线验收

外部系统集成方

负责把组件放进业务现场

  • iframe 方式快速嵌入
  • postMessage 动态传递配置
  • 减少重复开发和一次性页面

核心工作流

从一句需求，到一个可配置、可预览、可嵌入的组件。

05 / 端到端链路

创作端

给开发同事的是一个“能落地”的 AI 工作区。

它不只回答问题，而是围绕项目工作区、文件变更、实时预览和版本快照形成可追踪的生产过程。

$ open project workspace
$ ask agent to adjust component
$ preview schema-driven props
$ snapshot before publish

AI 辅助编码

  • 自然语言指令生成或修改组件
  • 文件读写、截图、控制台等工具协同

配置与预览

  • Schema 解析为可操作表单
  • TSX 动态编译后实时渲染

工作空间语义

  • Session 不拥有文件
  • 项目工作区承载当前态

知识库上下文

  • 工作空间参考文档可被 AI 查阅
  • 降低重复说明和上下文丢失

06 / 创作端价值

使用端

让非技术同事不用等开发，也能参与验收。

项目列表、配置面板和预览画布共同组成一个低门槛入口：能找项目、能调参数、能看真实效果。

活动组件预览

配置修改后，预览区保持同一组件真实渲染结果。

JSON Schema 驱动表单，产品和运营可以直接调参。

07 / 使用端价值

Agent Service

独立 Agent 服务层

AI 能力独立出来，前端专注 UI，服务层负责业务和工具边界。

Pi Agent 单后端

采用进程内嵌入式 Agent 后端，不依赖外部 OpenCode Server 或 CLI 子进程。

HTTP + WebSocket

REST 管理项目、Session、文件和模型配置；WebSocket 推送流式响应与文件变更。

工具权限与变更捕获

覆盖文件、截图、控制台、计划审批、页面删除和子 Agent 委派等能力。

Session Guard + Snapshot

通过边界校验、变更比较和快照，把 AI 修改变成可审计的项目行为。

08 / AI 服务架构

工程治理

能上线，也要能持续维护。

1

共享项目语义

project-core、project-scaffold、project-cli 复用同一项目读写和脚手架能力。

2

诊断与回归

创作端诊断日志、CLI 查询、Jest / Vitest / Playwright 形成问题闭环。

3

部署边界清晰

Docker Compose 覆盖 author、agent、screenshot、viewer 四个服务。

可验证

核心命令收敛到 check:* 和 E2E 流程。

可追踪

诊断事件把预览、自动保存、协同和 Session 串起来。

可回滚

快照和版本历史支撑关键节点回退。

可扩展

嵌入 API 和工具链让组件进入更多业务场景。

09 / 上线保障

Join

上线后，同事可以怎么参与

这不是“发布一个系统”，而是启动一条新的组件交付方式。

试用

用一个真实组件跑完整链路

从 AI 修改、配置、预览到快照，看每一步是否符合团队实际习惯。

沉淀

把常用组件做成模板

让后续项目不用重复搭结构、补说明、复制配置。

补充

提供知识库材料

把规范、品牌、业务说明变成 AI 可查阅的可靠上下文。

集成

选择一个外部场景嵌入

验证 iframe + postMessage 在业务系统里的真实交付效果。
