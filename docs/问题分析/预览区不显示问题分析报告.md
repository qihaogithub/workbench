# 预览区不显示问题分析报告

## 一、问题概述

**现象**: Demo 编辑页面中预览区（iframe）不显示内容，`#root` 元素为空。

**影响范围**: AI 创作/调试场景，用户无法实时预览代码效果。

**发现时间**: 2026-04-19

---

## 二、问题根因

### 2.1 直接原因

**Session ID 不匹配导致编译 API 返回 404**

浏览器中的 session ID 与文件系统中实际存储的 session ID 不一致：

| 位置 | Session ID |
|------|-----------|
| 浏览器（React State） | `session-1776603132882-dplt1h3rd` |
| 文件系统（实际存在） | `session-1776607746858-eyu5z42mp` |

### 2.2 调用链路分析

```
页面加载
  │
  ▼
POST /api/sessions (创建/获取 session)
  │
  ├── 如果存在活跃 session → 返回已有 sessionId
  │
  └── 如果不存在 → 创建新 session，生成新 sessionId
  │
  ▼
React State: setSessionId(sessionId)
  │
  ▼
PreviewPanel 组件接收 sessionId
  │
  ▼
useEffect 触发编译
  │
  ▼
POST /api/compile { sessionId: "session-1776603132882-dplt1h3rd" }
  │
  ▼
服务器查找 session 文件
  │
  ▼
❌ 文件不存在 → 返回 404
  │
  ▼
编译失败，无代码发送到 iframe
  │
  ▼
iframe 中 #root 保持为空
```

```typescript
export function findActiveSession(
  userId: string,
  projectId: string,
): string | null {
  // 遍历目录查找 session
  for (const entry of entries) {
    const metaPath = path.join(projectSessionDir, entry.name, ".session.json");
    const meta = JSON.parse(fs.readFileSync(metaPath, "utf-8"));
    
    // 返回 meta.sessionId，但 entry.name 是目录名
    if (meta.demoId === projectId) {
      return meta.sessionId;  // ← 可能返回与目录名不同的 sessionId
    }
  }
}
```

**问题**: `findActiveSession` 返回的是 `.session.json` 中存储的 `sessionId`，而非目录名。如果两者不一致，后续查找会失败。

---

## 三、问题复现条件

1. 用户之前创建过 session（如 `session-1776607746858-eyu5z42mp`）
2. `.session.json` 中的 `sessionId` 字段被手动修改或与目录名不一致
3. 页面加载时，`findActiveSession` 返回 `.session.json` 中的 `sessionId`
4. `PreviewPanel` 使用该 `sessionId` 调用 `/api/compile`
5. `compileSession` → `getSessionFiles` → `findSessionPath` 查找失败（因为查找时匹配的是目录名）

---

## 四、解决方案

### 方案 1: 统一使用目录名作为 sessionId（推荐）

**修改点**:
1. `findActiveSession` 返回目录名（`entry.name`）而非 `meta.sessionId`
2. 确保 `.session.json` 中的 `sessionId` 字段与目录名一致

**优点**: 简单直接，避免数据不一致

### 方案 2: 修复 `findSessionPath` 支持按 `.session.json` 中的 `sessionId` 查找

**修改点**:
1. `findSessionPath` 遍历目录时，同时检查 `.session.json` 中的 `sessionId` 字段

**优点**: 兼容现有数据

**缺点**: 增加查找复杂度
