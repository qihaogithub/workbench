# 索引任务与阅读地图设计

## 一、索引任务

索引任务用于记录知识整理进度。当前 v1 使用文件型状态，不引入独立队列。

状态包括：

- `pending`：任务已创建。
- `running`：正在整理。
- `ready`：阅读地图可用。
- `failed`：整理失败。
- `partial`：部分整理成功。
- `stale`：源内容更新后需要刷新。

## 二、模板保存触发

author-site 旧模板保存入口和 project-core 模板保存入口都会调用 `knowledge-service`：

```text
保存模板
  → 复制 workspace
  → 写入 template.json
  → 创建索引任务
  → 生成 reading-map.json
```

阅读地图路径为 `data/knowledge/templates/{templateId}/reading-map.json`。

## 三、阅读地图内容

阅读地图包含：

- 模板总览。
- 页面入口。
- 项目级配置入口。
- 工作空间知识文档入口。
- 局部摘要。
- 修改页面、修改配置、排查异常等任务导向入口。
- 原文入口。

阅读地图是导航层，不承担最终事实确认。主 AI 修改配置或业务规则前仍应读取原文。

## 四、异步整理扩展点

`KnowledgeOrganizer` 是后续 LLM 异步整理入口。Basic 阅读地图先保证可用，organizer 可增强局部摘要、关键词、标签和候选重排。

该扩展点不绑定具体模型供应商，也不改变外部业务 API。
