# 索引任务与阅读地图设计

## 一、索引任务

索引任务用于记录知识整理进度。当前 v1 使用文件型状态，不引入独立队列。

状态包括：

- `pending`：任务已创建。
- `running`：正在整理。
- `ready`：阅读地图可用。
- `failed`：整理失败。
- `partial`：部分整理成功。
- `stale`：源内容更新后需要刷新。

## 二、模板保存触发

author-site 旧模板保存入口和 project-core 模板保存入口都会调用 `knowledge-service`：

```text
保存模板
  → 复制 workspace
  → 写入 template.json
  → 创建索引任务
  → 生成 reading-map.json
```

阅读地图路径为 `data/knowledge/templates/{templateId}/reading-map.json`。

模板快照的知识文档来源于工作空间 `knowledge/`。CLI 本地项目包把该目录映射为 `src/knowledge/`，并在 `project pull`、`diff`、`submit`、`template init` 和 `template submit` 中作为受管理目录同步。通过本地模板项目新增的知识文档会先回写到项目工作空间，再随模板保存进入阅读地图。

## 三、阅读地图内容

阅读地图包含：

- 模板总览。
- 页面入口。
- 项目级配置入口。
- 工作空间知识文档入口。
- 局部摘要。
- 修改页面、修改配置、排查异常等任务导向入口。
- 原文入口。

阅读地图是导航层，不承担最终事实确认。主 AI 修改配置或业务规则前仍应读取原文。

## 四、异步整理扩展点

`KnowledgeOrganizer` 是后续 LLM 异步整理入口。Basic 阅读地图先保证可用，organizer 可增强局部摘要、关键词、标签和候选重排。

该扩展点不绑定具体模型供应商，也不改变外部业务 API。
## 五、manifest 路径兼容规则

`knowledge/manifest.json` 条目兼容 `path` 和旧 `fileName` 两种写法。CLI 本地项目包生成的 `src/knowledge/manifest.json` 使用 `path`，模板索引器和 agent-service `knowledgeReport` 工具需要按同一规则读取，避免模板阅读地图能看到文档而 AI 会话检索不到文档。

读取 manifest 路径时，只接受 `.md` 相对路径，并拒绝绝对路径、空路径和包含 `..` 的路径，确保知识文档读取范围不越过 workspace 的 `knowledge/` 目录。

从模板实例化出的项目会携带 `knowledge/` 文档。AI 在其他项目会话中使用 `knowledgeReport` 时，应把这些条目作为 `current-project` 知识项检索，并把来源路径呈现为 `knowledge/{path}`。

## 六、脚本化验证与指标

知识库效果验证通过 `scripts/development/knowledge-validation-suite.mjs` 固化，避免每次手工创建模板、实例化项目和复制问答结果。脚本默认使用知识库验证项目 `proj_1782547891917_g0e1l9`，也可以通过参数覆盖项目、模板、实例项目、模型和 agent-service 地址。

常用入口：

- `pnpm test:knowledge-validation`：只读取当前项目静态指标。
- `pnpm test:knowledge-validation:run`：补测试夹具、创建模板、实例化项目，并在提供 `--agent-url` 时执行 AI 问答指标。
- `node scripts/development/knowledge-validation-suite.mjs fixture`：补圆形页面和画布文档节点。
- `node scripts/development/knowledge-validation-suite.mjs ai --project-id <projectId> --agent-url <url> --model mydeepseek/deepseek-v4-flash`：对指定项目执行 AI 指标。

当前指标分为三层：

- 静态覆盖：页面数不少于 4、存在圆形页面、圆形页预览为正方形、知识文档不少于 4、画布 document 节点不少于 4。
- 模板索引：模板 reading-map 中 `knowledgeCount` 不少于 4，并保留 `knowledge/玩法规则.md`、`knowledge/FAQ与客服口径.md`、`knowledge/中奖履约SOP.md`、`knowledge/风控与异常处理.md` 等来源路径。
- AI 使用效果：显式检索题按期望事实计算事实召回率和来源引用率；客服口径题按场景期望事实计算召回率，不强制要求来源引用。验收阈值为平均事实召回率不低于 0.8、显式来源引用率不低于 0.5、无明显幻觉标记。

圆形页面用于验证非标准页面预览尺寸和画布布局共存；文档节点使用 `CanvasState.nodes` 中的 `kind: "document"`，同时保存 `knowledgeDocument` 元数据和 Markdown 正文，使文档以文档节点形式出现在画布，而不是伪装成普通页面。
