# 模板项目索引与检索设计

## 一、索引任务

索引任务由独立服务写入 SQLite，用于记录每个模板项目修订的构建结果。

状态包括：

- `pending`：任务已创建。
- `running`：正在整理。
- `ready`：阅读地图可用。
- `failed`：整理失败。
- 写入前先建立 `pending` 任务，事务开始时切换为 `running`。
- 事务成功后记录文档数、分块数并切换为 `ready`。
- 事务失败时记录错误并切换为 `failed`；旧索引不会因半次写入进入不一致状态。

## 二、模板项目发现与触发

模板不是独立快照。服务扫描 `data/projects/*/project.json`，仅处理 `projectType: "template"` 的项目：

```text
项目切换为 template
  → author-site 主动请求 reconcile
  → 周期扫描兜底
  → 对比 canonicalSyncedRevision + canonicalSyncedRootHash
  → 变化时重建该项目索引
```

项目取消模板身份或删除后，协调过程把其来源标记为 inactive 并从 FTS 表移除。服务启动时执行一次完整协调，运行中默认每 5 秒协调；同一 revision/rootHash 不重复处理。

## 三、索引内容与分块

每个模板项目索引以下原文：

- 项目名称、简介和模板描述。
- `knowledge/manifest.json` 指向的 Markdown 文档。
- 工作空间页面入口与页面源码。
- 项目级配置 Schema 和运行值。

单个文件最大读取 2 MiB；原文按 1600 字符切分并保留 160 字符重叠。文件读取、摘要准备和分块在事务外完成，事务只负责替换该项目的来源、文档、分块和 FTS 行。

## 四、中文全文检索

索引使用 SQLite FTS5 `unicode61`，同时为连续中文文本生成二元组搜索字段。查询按 BM25 排序，限制最多 50 条，并默认排除当前项目 id。

结果包含摘要片段、模板项目、文档路径、修订号、根哈希和 `knowledge://` 不透明引用。调用方不能自行拼接引用，只能使用搜索结果返回值读取原文。

## 五、manifest 与路径安全

`knowledge/manifest.json` 条目兼容 `path` 和旧 `fileName`。读取时只接受工作空间内的受管理路径，拒绝绝对路径、`..` 和超出大小限制的文件，避免索引越过项目边界。

## 六、运行与恢复

- 数据库要求 SQLite 3.51.3 或更高版本，并启用 WAL、外键、5 秒 busy timeout 和 `synchronous=FULL`。
- `/health` 和 `/api/knowledge/status` 暴露索引规模、失败任务、同步/备份状态及完整性检查。
- 服务使用 Online Backup API 默认每天备份到 `data/backups/knowledge/`，默认保留 7 天。
- 索引可从项目文件完整重建；数据库损坏时应先停服务、保留故障文件，再从最近在线备份恢复或删除索引后重建。
