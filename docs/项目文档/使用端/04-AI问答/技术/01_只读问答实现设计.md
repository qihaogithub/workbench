# AI 问答 - 只读问答实现设计

> 版本：v2.0
> 创建日期：2026-06-26
> 更新日期：2026-07-21
> 更新说明：v2.0 按「AI 架构统一 — 模式驱动」方案重写。浏览端由 HTTP 同步问答升级为与创作端同一条 WebSocket 流式链路和同一套共享聊天组件，仅通过 `mode: "viewer-readonly"` 区分行为。

## 一、整体架构

使用端 AI 与创作端 AI 全栈共用三层实现，通过 `mode` 参数驱动差异：

```
viewer-site ViewerAiPanel（抽屉壳）
  → @workbench/ai-chat-shared 的 AIChat（mode="viewer-readonly"）
  → @workbench/agent-client 的 AgentStream（WS，URL 带 ?mode=viewer-readonly）
  → agent-service 统一路由 /api/agent/:sessionId/stream
```

不再存在使用端专用后端路由（原 `POST /api/viewer-ai/chat` 已删除）。使用端因此获得与创作端一致的流式输出、思维链、只读工具调用卡片和模型切换体验。

## 二、mode 的服务端强制语义

客户端只声明 `mode`、`projectId` 和 `viewerContext`（当前页面 ID 与配置值），所有安全约束由 agent-service 强制（`services/viewer-readonly-mode.ts`）：

- 工作目录由 `projectId` 解析为项目正式工作区，忽略客户端传入的 `workingDir`。
- `toolMode: "viewer-readonly"`：只挂载 readFile、listFiles、knowledgeReport 三个只读工具。
- 路径白名单只覆盖页面、配置、记忆和知识库文件；命令全量禁止。
- 系统提示词强制使用 `buildViewerAiSystemPrompt()`，忽略客户端 `systemPrompt`。
- 每条消息由服务端拼接 `buildViewerAiPromptContext()` 只读上下文（页面清单、当前配置值、项目记忆、知识库索引），与旧 HTTP 行为一致。

## 三、会话生命周期

- 打开抽屉时生成 `viewer-{projectId}-{timestamp}-{random}` 会话 ID 并建立 WS 连接；面板首次打开前不建立连接。
- WS 会话存续期间由 Pi Agent 自身维护对话历史，客户端不再回传 history 字段。
- 「新对话」按钮生成新会话 ID 并兜底调用 `destroySession`；关闭页面或断开 WS 后，agent-service 在最后一个连接关闭时自动销毁 Agent 并清理会话元数据（websocket.ts 既有行为）。
- 对话内容仅存在于组件内存与服务端 Agent 会话中，刷新即弃，不做 localStorage 历史持久化；旧版 `viewer-ai-sessions:{projectId}` 本地历史随旧组件一并移除（项目未上线，无兼容负担）。
- 模型偏好仍按 `projectId` 保存在浏览器本地（共享 use-chat-models Hook 行为），刷新后保留。

## 四、前端组成

- `ViewerAiPanel.tsx`：360px 左侧栏壳（标题、新对话、收起按钮），内容区渲染共享包 `AIChat`；模块加载时通过 `configureAiChatShared({ getAgentClient })` 注入 viewer-readonly 模式的 AgentClient。
- 共享 `AIChat` 在 viewer-readonly 模式下的前端差异仅有三处：输入框只保留图片附件（文件上传依赖 author-site API）、隐藏历史会话入口（依赖 author-site 会话持久化）、消息持久化自动跳过（message-service 以「是否配置 authorContext」为开关）。
- 计划面板、权限弹窗、文件变更列表不做条件渲染 —— viewer 模式后端不会发出对应事件，自然不出现。
- 模型列表通过 WS `get_models` 消息获取（`AgentStream.requestModels()`），不再调用 `GET /models`。
- 图片附件在浏览器端转为 base64 随 WS 消息发送；不支持图片的模型仍走后端图片内容预描述能力，只读工具边界不因图片发送而放开。

## 五、失败处理

当 Agent 服务不可达、项目不存在或模型返回错误时，共享组件保留对话上下文并展示错误与重试入口；WS 断线自动重连（AgentStream 既有行为）。AI 连接、超时、配额和服务异常通过共享错误归一化转为面向使用者的中文提示；原始技术信息保留在 agent-service 日志中。viewer-readonly 模式缺少 `projectId` 或项目不存在时，服务端分别返回 `INVALID_PARAMS` 与 `PROJECT_NOT_FOUND`。

## 六、构建配置

viewer-site 接入共享包所需配置：`package.json` 依赖 `@workbench/agent-client` 与 `@workbench/ai-chat-shared`；`next.config.js` transpilePackages 追加两包；`tailwind.config.ts` content 追加 `./node_modules/@workbench/ai-chat-shared/src/**/*`。agent-service 地址复用 `NEXT_PUBLIC_AGENT_SERVICE_URL`，模型白名单可选配置 `NEXT_PUBLIC_ALLOWED_MODEL_PREFIXES` 等（共享包 ai-models 读取）。
