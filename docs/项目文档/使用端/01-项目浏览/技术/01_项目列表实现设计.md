## 一、整体架构

项目列表页是使用端的首页（`/`），采用客户端渲染（CSR）模式，从发布数据源读取项目索引。页面包含搜索框和排序按钮，均在前端实现。

```
页面组件 (page.tsx)
    │
    ├── useEffect ──→ getProjects() ──→ /data/projects.json
    │
    ├── 状态: searchQuery (搜索关键词)
    ├── 状态: sortBy (排序方式: newest/oldest/name)
    ├── 状态: showFilters (筛选菜单展开状态)
    │
    ├── useMemo: filteredProjects ──→ 根据 searchQuery 和 sortBy 过滤排序
    │
    ├── 加载状态 → "加载中..."
    ├── 错误状态 → 错误信息（红色）
    ├── 空状态 → "未找到匹配的项目" 或 "暂无项目"
    └── 成功状态 → 项目卡片网格（与创作端一致的横向封面卡片）
                      │
                      ├── 自定义封面 → getThumbnailUrl()
                      ├── 无自定义封面 → getProjectData() → 发布包静态截图拼图封面
                      └── 点击 → /{projectId}
```

## 二、搜索与排序实现

### 2.1 搜索功能

- 使用 `useState` 管理 `searchQuery` 状态
- 输入框实时过滤，通过 `useMemo` 计算 `filteredProjects`
- 搜索逻辑：匹配 `project.name` 和 `project.description`（不区分大小写）
- 搜索框支持一键清空（右侧 X 按钮）

### 2.2 排序功能

- 排序方式通过 `useState` 管理，支持三种：
  - `newest`：按 `updatedAt` 降序（默认）
  - `oldest`：按 `updatedAt` 升序
  - `name`：按 `name.localeCompare('zh-CN')` 中文拼音排序
- 排序按钮点击后展开下拉菜单（使用 ref 检测点击外部关闭）
- 选中项显示 Check 图标

## 三、API 客户端设计

使用端的 API 客户端需要区分发布数据、封面资源和 AI 服务来源：

| 函数                             | 目标服务             | 端点                             |
| :------------------------------- | :------------------- | :------------------------------- |
| `getProjects()`                  | 发布数据源           | `GET /data/projects.json`        |
| `getProjectData(id)`             | 发布数据源           | `GET /data/{projectId}/project.json` |
| `getThumbnailUrl(thumbnail)`     | 发布数据源           | 自定义封面路径                   |
| `getPublishedFileUrl(projectId, filePath)` | 发布数据源 | 发布包内页面资源路径 |
| `getScreenshotFileUrl(projectId, pageId)` | author-site 创作端 (3200) | 旧数据兜底截图接口 |
| `getScreenshotFileMetaUrl(projectId, pageId)` | author-site 创作端 (3200) | 同源开发态兜底截图 meta |

发布数据请求统一通过 `NEXT_PUBLIC_DATA_BASE` 拼接数据源地址；发布流程会把页面截图复制到项目发布包的 `screenshots/` 目录，并在页面条目上写入 `screenshotPath`，浏览端优先通过发布数据源读取这个静态路径。

## 四、数据获取策略

客户端数据获取分两层：

- **项目索引**：首页加载时调用 `getProjects()`，读取轻量的项目列表、名称、发布时间、自定义封面路径等摘要字段。
- **封面详情**：当项目没有自定义封面时，单张卡片再调用 `getProjectData(projectId)` 读取发布包内页面列表，用于构造创作端同款截图拼图封面。
- **发布包截图**：发布流程从创作端截图缓存中选取当前页面截图，复制为 `screenshots/{pageId}.png`，并写入 `demoPages[].screenshotPath`。这让静态 viewer 服务不需要暴露 `/api/screenshots/file` 也能展示封面。
- **截图兜底**：如果旧发布包还没有 `screenshotPath`，浏览端保留截图接口兜底；截图缺失时，该页面格子显示页面名占位，整个项目确无页面时才退回项目级占位图。

## 五、项目卡片组件

卡片采用与创作端首页一致的封面卡片风格：列表使用响应式四档网格，卡片封面固定为横向 `aspect-video` 比例，避免使用端和创作端在首页入口出现两套项目识别视觉。

卡片内容从 `ProjectsIndex.projects` 数组和按需加载的 `PublishedProject.demoPages` 中提取：

- **封面图**：如果 `project.thumbnail` 存在，通过 `getThumbnailUrl()` 拼接数据源地址，使用 Next.js `Image` 组件在横向封面中等比完整展示
- **截图拼图封面**：如果 `project.thumbnail` 不存在，但项目包含页面，使用发布包内 `demoPages[].screenshotPath` 组成与创作端 `DemoCard` 一致的页面截图拼图布局。布局会根据截图实际宽高调整页面格子的横纵比例，最多展示 10 个页面，超出时显示数量遮罩
- **占位封面**：只有项目没有可展示页面，或封面详情加载失败时，才显示与创作端一致的渐变背景、圆形底和播放形占位图
- **项目名称**：封面下方单行截断（`truncate`），使用与创作端项目卡片一致的信息区密度
- **悬停反馈**：卡片边框加深，并在封面上叠加轻量渐变遮罩

## 六、类型依赖

项目列表页使用的关键类型来自 `packages/viewer-site/src/lib/api.ts`，并复用 shared/demo-ui 的页面和画布相关类型：

- `ProjectsIndex` — 项目列表索引（含 `projects` 数组）
  - `projects[].id` — 项目 ID
  - `projects[].name` — 项目名称
  - `projects[].thumbnail` — 缩略图路径（可选）
  - `projects[].publishedAt` — 发布时间（时间戳）
- `PublishedProject` — 单个发布项目详情
  - `demoPages[]` — 用于生成截图拼图封面的页面列表
  - `demoPages[].screenshotPath` — 页面封面的发布包静态截图路径
