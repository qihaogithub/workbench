## 一、整体架构

项目列表页是使用端的首页（`/`），采用客户端渲染（CSR）模式，通过 SWR 从 agent-service 获取项目列表数据。页面包含搜索框和排序按钮，均在前端实现。

```
页面组件 (page.tsx)
    │
    ├── SWR Hook ──→ fetcher() ──→ getProjects() ──→ agent-service API
    │
    ├── 状态: searchQuery (搜索关键词)
    ├── 状态: sortBy (排序方式: newest/oldest/name)
    ├── 状态: showFilters (筛选菜单展开状态)
    │
    ├── useMemo: filteredProjects ──→ 根据 searchQuery 和 sortBy 过滤排序
    │
    ├── 加载状态 → "加载中..."
    ├── 错误状态 → 错误信息（红色）
    ├── 空状态 → "未找到匹配的项目" 或 "暂无项目"
    └── 成功状态 → 项目卡片网格（与创作端一致的横向封面卡片）
                      │
                      └── Link → /{projectId}
```

## 二、搜索与排序实现

### 2.1 搜索功能

- 使用 `useState` 管理 `searchQuery` 状态
- 输入框实时过滤，通过 `useMemo` 计算 `filteredProjects`
- 搜索逻辑：匹配 `project.name` 和 `project.description`（不区分大小写）
- 搜索框支持一键清空（右侧 X 按钮）

### 2.2 排序功能

- 排序方式通过 `useState` 管理，支持三种：
  - `newest`：按 `updatedAt` 降序（默认）
  - `oldest`：按 `updatedAt` 升序
  - `name`：按 `name.localeCompare('zh-CN')` 中文拼音排序
- 排序按钮点击后展开下拉菜单（使用 ref 检测点击外部关闭）
- 选中项显示 Check 图标

## 三、API 客户端设计

使用端的 API 客户端需要区分两个服务来源：

| 函数                             | 目标服务             | 端点                             |
| :------------------------------- | :------------------- | :------------------------------- |
| `getProjects()`                  | agent-service (3201) | `GET /api/projects`              |
| `getProject(id)`                 | agent-service (3201) | `GET /api/projects/:id`          |
| `getProjectVersions(id)`         | agent-service (3201) | `GET /api/projects/:id/versions` |
| `getDemoPages(id)`               | author-site 创作端 (3200) | `GET /api/projects/:id/demos`    |
| `getProjectConfig(id)`           | author-site 创作端 (3200) | `GET /api/projects/:id/config`   |
| `getEmbedIframeUrl(id, demoId?)` | author-site 创作端 (3200) | 生成 iframe URL                  |

所有 API 调用统一走 `fetchApi<T>()` 封装函数，自动处理 `ApiResponse<T>` 的成功/错误拆包。

## 四、数据获取策略

使用 SWR 进行客户端数据获取，配置如下：

- **SWR Key**：使用 API 路径作为 key（如 `/api/projects`），确保全局缓存一致性
- **Fetcher**：直接调用 `lib/api.ts` 中对应的函数
- **无自动刷新**：使用端数据变化不频繁，依赖 SWR 默认的 `revalidateOnFocus` 行为

## 五、项目卡片组件

卡片采用与创作端首页一致的封面卡片风格：列表使用响应式四档网格，卡片封面固定为横向 `aspect-video` 比例，避免使用端和创作端在首页入口出现两套项目识别视觉。

卡片内容从 `ProjectListResponse.projects` 数组中提取：

- **封面图**：如果 `project.thumbnail` 存在，通过 `getThumbnailUrl()` 拼接数据源地址，使用 Next.js `Image` 组件在横向封面中等比完整展示
- **无封面图**：显示与创作端一致的渐变背景、圆形底和 `FileCode` 图标占位
- **项目名称**：封面下方单行截断（`truncate`），使用与创作端项目卡片一致的信息区密度
- **悬停反馈**：卡片边框加深，并在封面上叠加轻量渐变遮罩

## 六、类型依赖

项目列表页使用的类型全部来自 `@workbench/shared`：

- `ProjectListResponse` — 项目列表响应（含 `projects` 数组和 `total`）
  - `projects[].id` — 项目 ID
  - `projects[].name` — 项目名称
  - `projects[].thumbnail` — 缩略图路径（可选）
  - `projects[].updatedAt` — 更新时间（时间戳）
- `ApiResponse<T>` — 统一 API 响应格式
