## 一、整体架构

项目列表页是使用端的首页（`/`），采用客户端渲染（CSR）模式，通过 SWR 从 agent-service 获取项目列表数据。

```
页面组件 (page.tsx)
    │
    ├── SWR Hook ──→ fetcher() ──→ getProjects() ──→ agent-service API
    │
    ├── 加载状态 → "加载中..."
    ├── 错误状态 → 错误信息 + 返回链接
    └── 成功状态 → 项目卡片网格
                      │
                      └── Link → /{projectId}
```

## 二、API 客户端设计

使用端的 API 客户端需要区分两个服务来源：

| 函数 | 目标服务 | 端点 |
|:-----|:---------|:-----|
| `getProjects()` | agent-service (3201) | `GET /api/projects` |
| `getProject(id)` | agent-service (3201) | `GET /api/projects/:id` |
| `getProjectVersions(id)` | agent-service (3201) | `GET /api/projects/:id/versions` |
| `getDemoPages(id)` | web 创作端 (3200) | `GET /api/projects/:id/demos` |
| `getProjectConfig(id)` | web 创作端 (3200) | `GET /api/projects/:id/config` |
| `getEmbedIframeUrl(id, demoId?)` | web 创作端 (3200) | 生成 iframe URL |

所有 API 调用统一走 `fetchApi<T>()` 封装函数，自动处理 `ApiResponse<T>` 的成功/错误拆包。

## 三、数据获取策略

使用 SWR 进行客户端数据获取，配置如下：

- **SWR Key**：使用 API 路径作为 key（如 `/api/projects`），确保全局缓存一致性
- **Fetcher**：直接调用 `lib/api.ts` 中对应的函数
- **无自动刷新**：使用端数据变化不频繁，依赖 SWR 默认的 `revalidateOnFocus` 行为

## 四、项目卡片组件

卡片采用响应式网格布局，通过 Tailwind CSS 的 `grid` 实现：

- 小屏（< 640px）：1 列
- 中屏（640px - 1024px）：2 列
- 大屏（> 1024px）：3 列

卡片内容从 `ProjectListResponse.projects` 数组中提取，每项包含 `id`、`name`、`description`、`demoCount`、`updatedAt` 等字段。

## 五、类型依赖

项目列表页使用的类型全部来自 `@opencode-workbench/shared`：

- `ProjectListResponse` — 项目列表响应（含 `projects` 数组和 `total`）
- `ProjectDetailResponse` — 项目详情响应（含 `project` 和 `currentVersion`）
- `DemoPageMeta` — Demo 页面元数据
- `ProjectConfig` — 项目级配置（含 `schema` 和 `exists`）
- `ApiResponse<T>` — 统一 API 响应格式
