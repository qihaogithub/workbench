> 版本：v2.0

## 一、模块定位

预览与配置模块是使用端的核心交互页面，为使用者提供组件预览功能。使用端通过 iframe 直接嵌入创作端的 viewer 端点，由创作端负责渲染预览、页面切换和配置面板。

---

## 二、功能描述

### 2.1 组件预览

**用户视角**：

- 进入项目预览页后，应看到完整的创作端 viewer 界面
- 项目级预览默认进入画布视图，提供多页面总览体验
- 预览区应能正确展示组件的各种状态
- 使用者可在画布中拖动视图、缩放、适应屏幕和重置视图

**交互效果**：

- 预览区以 iframe 形式嵌入创作端的 `/viewer/{projectId}` 端点
- iframe 继承父容器全宽高，无额外边框
- 创作者端的 viewer 已内置页面列表、配置面板、单页预览和画布预览等功能
- 项目级预览默认使用画布模式；需要单页模式时通过 `mode=single` 显式覆盖
- 使用端画布只开放视口操作，不提供移动页面、缩放页面、自动整理、添加文档或添加图片等创作端编辑能力

### 2.2 页面切换

**用户视角**：

- 单页视图下，当项目包含多个 Demo 页面时，使用者应能通过创作端的页面列表切换查看不同页面
- 画布视图下，使用者应直接通过画布中的页面进行定位和切换

**交互效果**：

- 页面切换由创作端 viewer 内部处理（通过 URL 参数 `pageList=true` 启用）
- 使用端不需要实现自己的页面切换器
- 画布模式下不展示左侧页面目录，避免目录与画布总览重复

### 2.3 配置面板

**用户视角**：

- 配置面板由创作端 viewer 提供，位于预览区右侧
- 当项目存在配置项时，配置面板默认打开；项目无配置项时不展示空面板

**交互效果**：

- 配置面板的启用由 URL 参数 `config=true` 控制
- 配置面板的 Schema 解析、表单生成、配置同步均由创作端处理
- 使用端不参与配置面板的渲染和通信
- 画布和单页视图均保持配置面板可见；画布中点击页面时，配置面板切换到对应页面配置

## 三、业务规则与约束

### 3.1 预览数据来源

- 预览区 iframe 的 URL 指向 author-site 创作端的 viewer 端点：`/viewer/{projectId}` 或 `/viewer/{projectId}/{demoId}`
- 通过 URL 参数控制 viewer 的功能启用：
  - `config=true`：启用配置面板
  - `pages=true`：启用页面列表
  - `pageList=true`：启用页面列表
  - `mode=canvas`：项目级默认画布模式
  - `mode=single`：显式切换为单页模式
  - `background=%230a0a0a`：深色背景
- 创作端 viewer 负责所有预览、配置、页面切换逻辑

### 3.2 通信约束

- 使用端与 iframe 之间无 `postMessage` 通信
- 使用端仅负责 iframe 嵌入和 URL 生成
- 所有配置同步、页面切换均在创作端 viewer 内部完成

### 3.3 路由设计

- `/{projectId}`：项目预览页，默认加载项目级 viewer
- `/{projectId}/{demoId}`：指定 Demo 页面的预览

### 3.4 前置条件

- author-site 创作端服务必须正常运行（提供 viewer 端点）
- agent-service 服务必须正常运行（提供项目详情）
- 使用端必须正确配置 `NEXT_PUBLIC_WEB_URL` 和 `NEXT_PUBLIC_AGENT_SERVICE_URL` 环境变量
