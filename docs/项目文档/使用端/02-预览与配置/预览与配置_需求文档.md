## 一、模块定位

预览与配置模块是使用端的核心交互页面，为使用者提供组件实时预览和配置参数调整功能。使用者可以在不编写代码的情况下，通过调整配置面板的参数来定制组件外观和行为，预览区会实时反映配置变更。

---

## 二、功能描述

### 2.1 组件预览

**用户视角**：
- 进入项目预览页后，应看到组件的实时渲染效果
- 预览区应占满可用空间，提供沉浸式预览体验
- 预览区应能正确展示组件的各种状态

**交互效果**：
- 预览区以 iframe 形式嵌入，与使用端样式完全隔离
- 预览区加载中显示加载状态
- 预览区加载失败显示错误提示

### 2.2 页面切换

**用户视角**：
- 当项目包含多个 Demo 页面时，使用者应能切换查看不同页面
- 切换页面后预览区应立即更新为对应页面的内容

**交互效果**：
- 页面切换器位于预览区上方，以标签页形式展示
- 当前选中的页面标签有视觉高亮
- 只有一个页面时不显示切换器

### 2.3 配置面板

**用户视角**：
- 配置面板位于预览区右侧，展示当前组件的可配置项
- 每个配置项对应一个表单控件（文本框、下拉选择、开关等）
- 修改配置项后，预览区应实时更新

**交互效果**：
- 配置面板可通过按钮收起/展开
- 配置项按 Schema 定义的顺序排列
- 每个配置项显示名称标签，鼠标悬停显示描述
- 配置项变更后立即通过 iframe 通信传递到预览区

### 2.4 配置重置

**用户视角**：
- 使用者应能一键将所有配置项恢复为默认值

**交互效果**：
- 配置面板顶部提供"重置"按钮
- 点击后所有配置项恢复为 Schema 中定义的默认值
- 预览区同步更新为默认配置的渲染效果

## 三、业务规则与约束

### 3.1 配置数据来源

- 配置 Schema 来自 web 创作端的 `GET /api/projects/:id/config` 接口
- Schema 格式遵循 JSON Schema 规范，由创作端的配置系统生成
- 配置面板完全由 Schema 驱动，使用端不硬编码任何配置项

### 3.2 预览数据来源

- 预览区 iframe 的 URL 指向 web 创作端的嵌入端点
- 嵌入端点负责组件编译和渲染，使用端只负责展示和通信
- iframe 内的编译和渲染逻辑由创作端处理，使用端不感知

### 3.3 通信约束

- 使用端与 iframe 之间仅通过 `postMessage` 通信
- 使用端只发送 `UPDATE_CONFIG` 消息类型
- 使用端不监听 iframe 的消息（单向通信即可满足需求）

### 3.4 前置条件

- web 创作端服务必须正常运行（提供嵌入端点和配置接口）
- agent-service 服务必须正常运行（提供项目详情和版本信息）
- 使用端必须正确配置 `NEXT_PUBLIC_WEB_URL` 和 `NEXT_PUBLIC_AGENT_SERVICE_URL` 环境变量
