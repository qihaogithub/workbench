```yaml
## 一、整体架构

预览页采用**iframe 直嵌**架构，使用端不再实现自己的配置面板、页面切换器或 iframe 渲染器，而是直接嵌入创作端的 viewer 端点。
```

┌─────────────────────────────────────────────────────────────────────────┐
│ 使用端预览页 ([projectId]/page.tsx) │
│ │
│ ┌───────────────────────────────────────────────────────────────────┐ │
│ │ │ │
│ │ iframe │ │
│ │ src = {WEB_URL}/viewer/{projectId}?mode=grid&config=true&... │ │
│ │ sandbox = "allow-scripts allow-same-origin" │ │
│ │ │ │
│ │ ┌─────────────────────────────────────────────────────────────┐ │ │
│ │ │ 创作端 viewer (完全独立) │ │ │
│ │ │ - 页面列表 (pageList=true) │ │ │
│ │ │ - 配置面板 (config=true) │ │ │
│ │ │ - 宫格预览 (mode=grid) │ │ │
│ │ │ - 配置同步 (内部处理) │ │ │
│ │ └─────────────────────────────────────────────────────────────┘ │ │
│ │ │ │
│ └───────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────────┘

```

## 二、路由设计

预览页使用 Next.js 嵌套动态路由：

- **`/[projectId]/page.tsx`**：项目级预览页
  - URL：`/{projectId}`
  - iframe URL：`{WEB_URL}/viewer/{projectId}?mode=grid&config=true&pages=true&background=%230a0a0a&gridSelectOnly=true&pageList=true`

- **`/[projectId]/[demoId]/page.tsx`**：指定 Demo 页面预览
  - URL：`/{projectId}/{demoId}`
  - iframe URL：`{WEB_URL}/viewer/{projectId}/{demoId}?config=true&background=%230a0a0a&pageList=true`

- **`/[projectId]/layout.tsx`**：共享布局
  - 顶部 header：返回按钮 + 项目名称
  - 从 agent-service 获取项目名称（`getProject(projectId)`）
  - 静默失败，不影响 iframe 渲染

## 三、组件协作关系

使用端预览页极简，仅负责 iframe 嵌入：

```

layout.tsx (共享布局)
│
├── useEffect: getProject(projectId) ──→ 获取项目名称
├── header: 返回按钮 + 项目名称
└── children: page.tsx (iframe)

page.tsx (项目预览)
│
├── useParams() ──→ 获取 projectId
├── 生成 viewerUrl ──→ `${WEB_URL}/viewer/${projectId}?...`
└── 渲染 iframe ──→ src={viewerUrl}

[demoId]/page.tsx (指定页面预览)
│
├── useParams() ──→ 获取 projectId + demoId
├── 生成 viewerUrl ──→ `${WEB_URL}/viewer/${projectId}/${demoId}?...`
└── 渲染 iframe ──→ src={viewerUrl}

```

## 四、URL 参数说明

iframe URL 通过查询参数控制创作端 viewer 的行为：

| 参数 | 值 | 说明 |
|:-----|:---|:-----|
| `mode` | `grid` | 默认宫格模式 |
| `config` | `true` | 启用配置面板 |
| `pages` | `true` | 启用页面列表（项目级） |
| `pageList` | `true` | 启用页面列表 |
| `gridSelectOnly` | `true` | 宫格模式仅选择 |
| `background` | `%230a0a0a` | 深色背景 (#0a0a0a) |

> 创作端 viewer 的完整参数列表，请见 [创作端嵌入 API 文档](../../创作端/07-嵌入API/)。

## 五、错误处理

- **项目详情获取失败**：layout.tsx 静默处理，项目名称留空，不影响 iframe 渲染
- **iframe 加载失败**：由浏览器原生处理（显示破碎图标）
- **viewer 端点不可达**：iframe 显示连接错误（创作端处理）

## 六、与旧架构的对比

### 旧架构（v1.0）

```

使用端 page.tsx
├── SWR: getProject() ──→ 项目详情
├── SWR: getProjectConfig() ──→ 配置 Schema
├── Tabs 组件 ──→ 页面切换
├── ConfigPanel ──→ Schema 驱动表单
└── IframeRenderer ──→ postMessage(UPDATE_CONFIG)

```

### 新架构（v2.0）

```

使用端 page.tsx
└── iframe ──→ 创作端 viewer (内置所有功能)

```

**优势**：
- 使用端代码量减少 90%+
- 配置面板、页面切换、配置同步由创作端统一维护
- 无 `postMessage` 通信复杂性
- 功能一致性保证（使用端与创作端看到相同的 viewer）
```
