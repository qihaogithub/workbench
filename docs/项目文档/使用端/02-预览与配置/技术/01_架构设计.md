# 预览与配置 - 架构设计

> 版本：v2.1
> 创建日期：2026-05-04
> 更新日期：2026-07-06

---

## 一、整体架构

预览页采用**iframe 直嵌**架构，使用端不再实现自己的配置面板、页面切换器或 iframe 渲染器，而是直接嵌入创作端的 viewer 端点。项目级预览默认进入画布模式，单页模式通过 URL 参数显式启用。

使用端项目详情页 header 只保留返回和项目标题，不再提供“导出代码”入口。代码导出统一走创作端 CLI 的本地项目包流程，避免使用端预览页和 CLI 形成重复入口。

┌─────────────────────────────────────────────────────────────────────────┐
│ 使用端预览页 ([projectId]/page.tsx) │
│ │
│ ┌───────────────────────────────────────────────────────────────────┐ │
│ │ │ │
│ │ iframe │ │
│ │ src = {WEB_URL}/viewer/{projectId}?mode=canvas&config=true&... │ │
│ │ sandbox = "allow-scripts allow-same-origin" │ │
│ │ │ │
│ │ ┌─────────────────────────────────────────────────────────────┐ │ │
│ │ │ 创作端 viewer (完全独立) │ │ │
│ │ │ - 页面列表 (pageList=true) │ │ │
│ │ │ - 配置面板 (config=true) │ │ │
│ │ │ - 画布预览 (mode=canvas) │ │ │
│ │ │ - 配置同步 (内部处理) │ │ │
│ │ └─────────────────────────────────────────────────────────────┘ │ │
│ │ │ │
│ └───────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────────┘

```

## 二、路由设计

预览页使用 Next.js 嵌套动态路由：

- **`/[projectId]/page.tsx`**：项目级预览页
  - URL：`/{projectId}`
  - iframe URL：`{WEB_URL}/viewer/{projectId}?mode=canvas&config=true&pages=true&background=%230a0a0a&pageList=true`
  - 默认展示画布；如需单页视图，使用 `mode=single`

- **`/[projectId]/[demoId]/page.tsx`**：指定 Demo 页面预览
  - URL：`/{projectId}/{demoId}`
  - iframe URL：`{WEB_URL}/viewer/{projectId}/{demoId}?config=true&background=%230a0a0a&pageList=true`

- **`/[projectId]/layout.tsx`**：共享布局
  - 顶部 header：返回按钮 + 项目名称
  - 从 agent-service 获取项目名称（`getProject(projectId)`）
  - 静默失败，不影响 iframe 渲染

## 三、组件协作关系

使用端预览页极简，仅负责 iframe 嵌入：

```

layout.tsx (共享布局)
│
├── useEffect: getProject(projectId) ──→ 获取项目名称
├── header: 返回按钮 + 项目名称
└── children: page.tsx (iframe)

page.tsx (项目预览)
│
├── useParams() ──→ 获取 projectId
├── 生成 viewerUrl ──→ `${WEB_URL}/viewer/${projectId}?...`
└── 渲染 iframe ──→ src={viewerUrl}

[demoId]/page.tsx (指定页面预览)
│
├── useParams() ──→ 获取 projectId + demoId
├── 生成 viewerUrl ──→ `${WEB_URL}/viewer/${projectId}/${demoId}?...`
└── 渲染 iframe ──→ src={viewerUrl}

```

## 四、URL 参数说明

iframe URL 通过查询参数控制创作端 viewer 的行为：

| 参数 | 值 | 说明 |
|:-----|:---|:-----|
| `mode` | `canvas` / `single` | 项目级默认画布模式；`single` 显式进入单页 |
| `config` | `true` | 启用配置面板 |
| `pages` | `true` | 启用页面列表（项目级） |
| `pageList` | `true` | 启用单页视图页面列表；画布模式隐藏页面目录 |
| `background` | `%230a0a0a` | 深色背景 (#0a0a0a) |

> 创作端 viewer 的完整参数列表，请见 [创作端嵌入 API 文档](../../../创作端/07-嵌入API/)。

## 五、画布模式边界

项目级预览默认展示画布模式。画布模式使用 shared 画布组件的 viewer 交互形态，只允许使用者移动视口、缩放、适应屏幕和重置视图。页面位置、页面尺寸、自动整理、文档节点和图片节点仍属于创作端编辑能力，不在使用端开放。

使用端从发布数据中读取页面清单、配置 Schema、画布状态和页面渲染入口。对于已发布的 React 页面，若发布数据提供 `iframeHtmlPath`，画布和单页模式优先直接嵌入该发布 iframe；这样页面代码、运行时依赖和静态资源都在同一个发布数据源下解析，避免使用端重新用编辑态 preview shell 装载发布脚本。没有 `iframeHtmlPath` 的旧发布数据才回退到 `compiledJsPath` 的 preview shell 路径。HTML/CSS 原型页继续使用发布数据中的静态 HTML、CSS 和 meta 内容渲染。

viewer 交互形态在首屏加载时会等待画布容器尺寸和页面布局都可用，然后自动计算一次适应屏幕的视口。这个计算只调整使用者看到的初始视口，不改写页面布局；使用者后续拖动画布、滚轮缩放或点击重置视图时，不会被首屏自适应逻辑反复覆盖。

使用端画布复用创作端 shared 渲染调度规则。项目页面总数少于 6 个时，不使用截图替代 iframe，所有页面直接进入实时 iframe 路径；页面总数达到 6 个及以上时，才允许继续使用截图和 iframe 预算策略来降低大项目的浏览器压力。截图规则的完整约束见 [创作端截图服务与预览快照机制](../../../创作端/04-配置与预览/技术/07_截图服务与预览快照机制.md)。

## 六、项目列表与单页舞台

项目列表读取发布索引中的 `thumbnail` 字段展示封面。发布流程会把项目封面复制到发布目录；封面源可以来自 `public/`、当前仓库 `data/`、`DATA_DIR` 或绝对路径。复制成功后，使用端通过 `/data/<projectId>/thumbnail.<ext>` 读取封面；复制失败则不写封面字段，前端展示占位图标。

单页模式以创作端预览效果为基准。高保真动态页面直接使用 shared 层的 `PreviewPanel`，HTML/CSS 原型页使用 `PrototypePagePreview`，草图页使用 `SketchPagePreview`；已发布 iframe 使用 `IframePreviewFrame` 复用同一套设计尺寸缩放、居中和边框语义。使用端不再额外包一层深色设备框，避免发布 iframe、原型页和动态预览出现不同的外层视觉。

## 七、配置面板协作

配置面板由 shared 层提供统一控件。数据先被整理成页面列表，画布模式下一级呈现每个页面可配置项总数，数量由项目级共享配置和本页独有配置相加得到，但不展开共享/独有摘要；用户点击页面后进入二级详情。单页模式下，调用方直接把当前页面作为详情页输入，并隐藏详情页头部，让表单区域直接承接右侧栏。二级详情先展示该页面可用的共享配置，再展示只影响当前页面的本页配置。创作端和使用端复用同一控件，只在是否允许编辑 Schema、是否只读等权限上不同。

## 八、错误处理

- **项目详情获取失败**：layout.tsx 静默处理，项目名称留空，不影响 iframe 渲染
- **iframe 加载失败**：由浏览器原生处理（显示破碎图标）
- **viewer 端点不可达**：iframe 显示连接错误（创作端处理）

## 九、与旧架构的对比

### 旧架构（v1.0）

```

使用端 page.tsx
├── SWR: getProject() ──→ 项目详情
├── SWR: getProjectConfig() ──→ 配置 Schema
├── Tabs 组件 ──→ 页面切换
├── ConfigPanel ──→ Schema 驱动表单
└── IframeRenderer ──→ postMessage(UPDATE_CONFIG)

```

### 新架构（v2.0）

```

使用端 page.tsx
└── iframe ──→ 创作端 viewer (内置所有功能)

```

**优势**：
- 使用端代码量减少 90%+
- 配置面板、页面切换、配置同步由创作端统一维护
- 无 `postMessage` 通信复杂性
- 功能一致性保证（使用端与创作端看到相同的 viewer）
