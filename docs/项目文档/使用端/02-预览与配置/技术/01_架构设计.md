## 一、整体架构

预览页是使用端的核心页面（`/[projectId]`），采用"三栏协作"架构：页面选择器 + iframe 预览区 + 配置面板。

```
┌─────────────────────────────────────────────────────────────────────────┐
│  预览页 ([projectId]/page.tsx)                                          │
│                                                                         │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │  页面选择器 (Tabs)                                                │  │
│  │  [页面A] [页面B] [页面C]                                          │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  ┌──────────────────────────────────┐  ┌────────────────────────────┐  │
│  │                                  │  │                            │  │
│  │  iframe 预览区                   │  │  配置面板                  │  │
│  │  (IframeRenderer)               │  │  (ConfigPanel)             │  │
│  │                                  │  │                            │  │
│  │  src = web创作端嵌入URL          │  │  Schema → 动态表单         │  │
│  │  postMessage → UPDATE_CONFIG     │  │  变更 → 通知父组件         │  │
│  │                                  │  │                            │  │
│  └──────────────────────────────────┘  └────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────────┘
```

## 二、路由设计

预览页使用 Next.js 动态路由 `[projectId]`，页面组件通过 `useParams()` 获取项目 ID。

**数据获取流程**：

1. 页面加载 → 从 URL 获取 `projectId`
2. 并行请求项目详情（agent-service）和配置（web 创作端）
3. 项目详情返回后，提取 Demo 页面列表
4. 默认选中第一个 Demo 页面，生成 iframe URL
5. 配置返回后，解析 Schema 生成配置面板

## 三、组件协作关系

```
page.tsx (页面组件，状态管理中心)
    │
    ├── SWR: getProject(projectId) ──→ 项目详情 + Demo 页面列表
    ├── SWR: getProjectConfig(projectId) ──→ 配置 Schema
    │
    ├── 状态: selectedDemoId (当前选中的 Demo 页面)
    ├── 状态: configFormData (当前配置表单数据)
    │
    ├── Tabs 组件 ──→ 切换 selectedDemoId
    │
    ├── IframeRenderer ──→ 接收 iframeUrl + configFormData
    │       └── postMessage(UPDATE_CONFIG, configFormData)
    │
    └── ConfigPanel ──→ 接收 schema + onChange 回调
            └── onChange → 更新 configFormData
```

页面组件是唯一的状态管理中心，IframeRenderer 和 ConfigPanel 均为受控组件，不持有自己的业务状态。

## 四、双服务数据流

预览页需要同时与两个后端服务通信，数据流如下：

```
agent-service (3201)                    web 创作端 (3200)
       │                                      │
       │ GET /api/projects/:id                │ GET /api/projects/:id/config
       │ → 项目详情 + 版本信息                 │ → 配置 Schema + 默认值
       │                                      │ GET /api/projects/:id/demos
       │                                      │ → Demo 页面列表
       │                                      │
       ▼                                      ▼
  ┌─────────────────────────────────────────────────┐
  │              page.tsx 状态管理                    │
  │                                                  │
  │  projectDetail  +  config  +  demoPages          │
  └─────────────────────────────────────────────────┘
       │                                      │
       │ 生成 iframe URL                      │
       │ → /embed/{projectId}/{demoId}        │
       ▼                                      │
  ┌─────────────────────────────────────────────────┐
  │           IframeRenderer                         │
  │   iframe.src = web创作端嵌入URL                   │
  │   postMessage → UPDATE_CONFIG                    │
  └─────────────────────────────────────────────────┘
```

## 五、错误处理

预览页对两个服务的数据获取分别处理错误：

- **项目详情获取失败**：显示错误页面，提供返回项目列表的链接
- **配置获取失败**：预览区正常显示，配置面板显示"暂无配置项"
- **Demo 页面列表为空**：显示"该项目暂无可用页面"提示
