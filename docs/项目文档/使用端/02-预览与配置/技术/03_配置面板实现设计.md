## 一、整体设计

配置面板是一个 Schema 驱动的动态表单组件，根据创作端返回的 JSON Schema 自动生成配置控件。使用者修改配置后，变更通过回调函数传递给父组件，再由父组件通过 iframe 通信同步到预览区。

```
ConfigPanel
    │
    ├── parseSchemaToFields(schema) ──→ 解析 Schema 为字段列表
    ├── extractDefaults(schema) ──→ 提取默认值
    │
    ├── 状态: formData (当前表单数据)
    │
    ├── 渲染: 遍历 fields，按类型映射控件
    │       ├── string + enum → Select 下拉选择
    │       ├── string → Input 文本输入
    │       ├── number + slider → Slider 滑块
    │       ├── number → Input 数字输入
    │       ├── boolean → Switch 开关
    │       └── array → Select 多选
    │
    └── onChange(formData) ──→ 通知父组件配置变更
```

## 二、Schema 解析

`parseSchemaToFields()` 函数将 JSON Schema 转换为扁平的字段列表，每个字段包含：

| 属性 | 说明 |
|:-----|:-----|
| `key` | 字段名（对应 Schema 的 property key） |
| `type` | 字段类型（string / number / boolean / array） |
| `label` | 显示名称（取自 `title`，缺省用 `key`） |
| `description` | 描述信息（取自 `description`） |
| `enum` | 枚举值列表（取自 `enum`） |
| `default` | 默认值（取自 `default`） |
| `minimum` / `maximum` | 数值范围（取自 `minimum` / `maximum`） |

解析逻辑仅处理 `schema.properties` 的第一层，不递归处理嵌套对象。

## 三、控件映射

配置面板根据字段类型和属性映射到对应的 Radix UI 控件：

| Schema 特征 | 映射控件 | Radix 组件 |
|:------------|:---------|:-----------|
| `type: "string"` + `enum` | 下拉选择 | `@radix-ui/react-select` |
| `type: "string"` | 文本输入 | 原生 `<input>` |
| `type: "number"` + `minimum/maximum` | 滑块 | `@radix-ui/react-slider` |
| `type: "number"` | 数字输入 | 原生 `<input type="number">` |
| `type: "boolean"` | 开关 | `@radix-ui/react-switch` |
| `type: "array"` + `items.enum` | 多选下拉 | `@radix-ui/react-select` |

## 四、配置变更流程

```
使用者修改控件值
    │
    ▼
setFormData(updated) ──→ 更新本地状态
    │
    ▼
onChange(updated) ──→ 调用父组件回调
    │
    ▼
page.tsx 更新 configFormData 状态
    │
    ▼
IframeRenderer 检测到 config 变化
    │
    ▼
postMessage(UPDATE_CONFIG, configFormData) ──→ iframe 接收并重新渲染
```

## 五、重置功能

配置面板顶部提供"重置"按钮，点击后：

1. 从 `extractDefaults(schema)` 获取默认值
2. 调用 `setFormData(defaults)` 重置本地状态
3. 调用 `onChange(defaults)` 通知父组件
4. 父组件通过 iframe 通信将默认配置同步到预览区

## 六、收起/展开

配置面板支持收起/展开切换：

- 收起时：面板宽度为 0，预览区占满全宽
- 展开时：面板宽度固定（如 320px），预览区自适应剩余空间
- 切换按钮始终可见，位于面板边缘

> 配置面板的 Schema 解析逻辑与创作端的 Schema 解析器不同。创作端使用 RJSF 渲染，使用端使用自研的轻量解析器。详见 [创作端 Schema 解析器](../../创作端/04-配置与预览/技术/02_Schema解析器.md)。
