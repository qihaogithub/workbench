## 一、模块定位

部署与嵌入模块关注使用端的部署方式和外部系统嵌入能力。使用端作为独立 Web 应用可单独部署，也可通过 iframe 嵌入到第三方系统中。

---

## 二、功能描述

### 2.1 独立部署

**用户视角**：
- 使用端应能作为独立的 Web 应用运行，使用者直接通过浏览器访问
- 启动后应能正常连接到 agent-service 和 web 创作端

**交互效果**：
- 访问 `http://localhost:3300` 进入项目列表页
- 所有功能（项目浏览、预览、配置）均可正常使用

### 2.2 iframe 嵌入

**用户视角**：
- 使用端应能被嵌入到第三方系统的 iframe 中
- 嵌入后所有功能应正常工作，包括 API 调用和 iframe 内的预览

**交互效果**：
- 第三方系统通过 `<iframe src="http://localhost:3300">` 嵌入使用端
- 嵌入的使用端页面功能与独立访问时完全一致

### 2.3 跨域访问

**用户视角**：
- 使用端应能正常调用 agent-service 和 web 创作端的 API
- 不应因跨域限制导致功能异常

**交互效果**：
- API 请求正常返回数据
- iframe 预览区正常加载和渲染
- 配置同步（postMessage）正常工作

---

## 三、业务规则与约束

### 3.1 端口分配

| 服务 | 默认端口 | 说明 |
|:-----|:---------|:-----|
| web 创作端 | 3200 | 提供嵌入端点和配置接口 |
| agent-service | 3201 | 提供项目列表和详情接口 |
| viewer-site | 3300 | 使用端自身 |

### 3.2 环境变量

使用端依赖以下环境变量：

| 变量名 | 必选 | 说明 |
|:-------|:-----|:-----|
| `NEXT_PUBLIC_WEB_URL` | 是 | web 创作端地址（如 `http://localhost:3200`） |
| `NEXT_PUBLIC_AGENT_SERVICE_URL` | 是 | agent-service 地址（如 `http://localhost:3201`） |

### 3.3 安全约束

- 使用端不存储任何用户凭证
- 使用端不执行写操作（不创建、修改、删除任何数据）
- iframe 预览区通过 sandbox 属性限制权限
