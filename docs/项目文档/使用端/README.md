## 文档概览

本系列文档用于指导使用端（`@opencode-workbench/viewer-site`）的开发工作。使用端是一个独立的 Next.js Web 应用，为“使用者”角色提供项目浏览、组件预览功能，与创作端完全解耦，仅通过 API 和 iframe 嵌入通信。

**核心设计理念**：

```
"极简使用端 + iframe 直嵌创作端 viewer，功能统一维护"
```

## 核心概念

### 使用者 (Viewer)

只读浏览角色，无需登录，可查看项目列表、预览组件。与创作端的“编辑者”角色完全隔离。

### 项目列表页 (`/`)

展示所有可供使用的项目，从 agent-service 获取数据。支持搜索和排序，使用者点击项目卡片进入预览页。

### 预览页 (`/[projectId]`)

项目的核心交互页面，通过 iframe 嵌入创作端的 viewer 端点，由创作端提供页面列表、配置面板、宫格预览等所有功能。

---

## 与创作端的关系

使用端与创作端是**完全解耦**的两个独立应用，仅通过 HTTP API 通信：

| 维度           | 说明                                                             |
| :------------- | :--------------------------------------------------------------- |
| **数据来源**   | 项目列表/详情来自 agent-service；预览/配置来自 web 创作端 viewer |
| **通信方式**   | HTTP API（REST）+ iframe 嵌入（无 postMessage）                  |
| **共享资源**   | `@opencode-workbench/shared` 类型定义包                          |
| **无共享状态** | 使用端无登录、无会话、无写操作、无配置面板实现                   |

> 详细的架构对比和设计决策，请见 [使用端方案](../../plans/进行中/使用端方案.md)。
