## 文档概览

本系列文档用于指导使用端（`@opencode-workbench/viewer-site`）的开发工作。使用端是一个独立的 Next.js Web 应用，为"使用者"角色提供项目浏览、组件预览和配置调整功能，与创作端完全解耦，仅通过 API 通信。

**核心设计理念**：

```
"只读浏览 + 配置调整，无 AI 对话、无代码编辑，开箱即用"
```

## 核心概念

### 使用者 (Viewer)

只读浏览角色，无需登录，可查看项目列表、预览组件、调整配置参数。与创作端的"编辑者"角色完全隔离。

### 项目列表页 (`/`)

展示所有可供使用的项目，从 agent-service 获取数据。使用者点击项目卡片进入预览页。

### 预览页 (`/[projectId]`)

项目的核心交互页面，包含 Demo 页面选择器、iframe 预览区和配置面板。支持多页面项目的页面切换。

### iframe 通信

使用端通过 iframe 嵌入创作端的嵌入端点，通过 `postMessage` 协议实现配置同步。使用端只需使用 `UPDATE_CONFIG` 消息类型。

---

## 与创作端的关系

使用端与创作端是**完全解耦**的两个独立应用，仅通过 HTTP API 通信：

| 维度 | 说明 |
|:-----|:-----|
| **数据来源** | 项目列表/详情来自 agent-service；嵌入/编译/配置来自 web 创作端 |
| **通信方式** | HTTP API（REST）+ postMessage（iframe） |
| **共享资源** | `@opencode-workbench/shared` 类型定义包 |
| **无共享状态** | 使用端无登录、无会话、无写操作 |

> 详细的架构对比和设计决策，请见 [使用端方案](../../plans/进行中/使用端方案.md)。
