## 一、项目定位

**Opencode Workbench** 是一个面向组件化开发的 **AI 辅助创作与使用平台**，提供从组件设计、AI 辅助编码、配置管理到预览嵌入的完整工作流。

### 1.1 核心价值

| 维度           | 说明                                                               |
| :------------- | :----------------------------------------------------------------- |
| **创作端**     | 开发者通过 AI 对话生成/修改组件代码，实时预览效果，管理配置 Schema |
| **使用端**     | 非技术人员浏览项目、调整配置参数、预览组件效果，无需编码           |
| **Agent 服务** | 独立的 AI 代理服务层，当前采用 Pi Agent 单后端                      |
| **嵌入集成**   | 通过 iframe + postMessage 协议，将组件嵌入外部系统使用             |

### 1.2 目标用户

| 角色              | 使用场景                           | 使用端   |
| :---------------- | :--------------------------------- | :------- |
| **开发者/设计师** | AI 辅助编码、配置定义、版本管理    | 创作端   |
| **产品/运营人员** | 浏览组件库、调整配置参数、预览效果 | 使用端   |
| **外部系统集成**  | 通过 iframe 嵌入组件，动态传递配置 | 嵌入 API |

### 2.2 技术栈

| 层级            | 技术选型                     | 说明                          |
| :-------------- | :--------------------------- | :---------------------------- |
| **前端框架**    | Next.js 14 (App Router)      | SSR/SSG、路由守卫、API Routes |
| **UI 组件库**   | shadcn/ui + Tailwind CSS     | 原子化 CSS、可定制组件        |
| **状态管理**    | SWR                          | 数据获取、缓存、自动重验证    |
| **表单引擎**    | react-jsonschema-form (RJSF) | JSON Schema 驱动表单生成      |
| **后端框架**    | Fastify                      | 高性能、原生 WebSocket 支持   |
| **认证**        | JWT (jose)                   | 无状态认证、httpOnly Cookie   |
| **数据库**      | SQLite (better-sqlite3)      | 用户数据存储                  |
| **编译**        | sucrase                      | TSX 动态编译                  |
| **截图**       | Puppeteer                     | 页面截图、缩略图生成          |
| **测试**        | Jest + Playwright + vitest   | 单元测试、E2E 测试            |

## 四、关键设计理念

### 4.1 创作端核心流程

```
创建项目 → 初始化正式工作空间
    ↓
打开编辑 → 创建临时工作空间 → 复制项目文件 → 注入 .opencode → 创建 Session
    ↓
AI 对话/手动编辑 → 修改临时空间文件
    ↓
保存变更 → 创建版本快照 → 临时空间覆盖正式空间 → 清理临时空间 → 记录版本
```

> 详细的架构设计、生命周期、版本管理，请见 [项目管理技术文档](./创作端/03-项目管理/技术/01_架构设计.md)

### 4.2 配置与预览联动

```
Schema 解析 → 表单生成 → 动态编译 → 实时预览
```

配置值作为 Props 传入组件渲染，修改配置表单后预览区实时更新（延迟 < 16ms）。

> 详细的配置系统架构、动态编译方案、实时预览机制，请见 [配置与预览技术文档](./创作端/04-配置与预览/技术/01_架构设计.md)

### 4.3 AI 对话与工作空间解耦

切换对话不影响工作空间文件：新建对话复用当前 workspaceId，仅更新 sessionId；切换对话仅切换 Session 元数据。

> 详细的分层架构、SSE 协议、行为约束机制，请见 [AI对话技术文档](./创作端/05-AI对话/技术/01_对话组件设计.md)

### 4.4 Agent 服务架构

采用 Fastify 独立服务 + Pi Agent 进程内嵌入。服务层负责会话生命周期、WebSocket 事件分发、工具权限校验和文件变更捕获，安全设计采用工具权限白名单、行为约束和后端校验。

> 详细的架构设计、接口规范、核心模块，请见 [Agent服务技术文档](./独立Agent服务层/01-架构设计.md)

### 4.5 知识库机制

AI 对话时可通过知识库附加参考文档。知识库以 `workingDir/knowledge/manifest.json` 管理元数据，文档以 `.md` 文件存储。支持系统预设和用户自定义两类来源，仅用户文档可修改/删除。

> 详细的知识库 API 和组件设计，请见 [知识库技术文档](./创作端/09-知识库/技术/01_知识库架构设计.md)

## 六、开发者指南

### 6.1 快速启动

```bash
pnpm install                    # 安装依赖
pnpm dev                        # 并行启动 author、agent、viewer、screenshot
pnpm dev:author / dev:agent / dev:viewer  # 单服务启动
pnpm build / build:viewer       # 构建
pnpm test:e2e                   # Playwright E2E 测试
```

### 6.2 代码约定

| 规范           | 说明                                                                           |
| :------------- | :----------------------------------------------------------------------------- |
| **TypeScript** | `strict: true`，禁止 `as any` / `@ts-ignore`                                   |
| **路径别名**   | `@/` = `./src/*` (author-site)，`@opencode-workbench/shared` = `../shared/src` |
| **API 响应**   | `{ success: true, data: T }` / `{ success: false, error: { code, message } }`  |
| **UI 组件**    | shadcn/ui + Tailwind CSS + lucide-react，禁止其他 UI 库                        |
| **数据获取**   | SWR (`@/lib/api.ts`)                                                           |

> 完整的开发者指南，请见根目录 [AGENTS.md](../../AGENTS.md)

## 七、项目特色

| 优势             | 说明                                              |
| :--------------- | :------------------------------------------------ |
| **AI 辅助编码**  | 基于 Pi Agent，自然语言指令生成/修改组件代码      |
| **配置驱动**     | JSON Schema 驱动表单生成，非技术人员可调整配置    |
| **实时预览**     | 动态编译方案，轻量快速，配置联动 < 16ms 响应      |
| **版本管理**     | 快照备份 + 版本历史，支持回退到任意版本           |
| **工作空间隔离** | 临时空间编辑，保存才合并，保障项目安全            |
| **知识库** | 工作空间内管理参考文档，为 AI 提供上下文依据       |
| **嵌入集成**     | iframe + postMessage 协议，外部系统可动态嵌入组件 |

**适用场景**：组件库开发、低代码平台、营销页面搭建、第三方集成、设计系统管理。
