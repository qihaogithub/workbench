# 用户鉴权模块 - 需求文档

## 1. 模块定位

用户鉴权模块是 Web 前端应用的基础设施模块，负责用户身份认证、权限控制和访问安全。本模块采用极简设计原则，实现基于纯 JWT 的无状态认证系统，为其他业务模块提供统一的身份验证能力。

## 2. 业务背景

### 2.1 业务目标

在项目管理和 Demo 编辑场景中，需要区分不同用户的操作权限，保护用户数据安全，防止未授权访问。用户鉴权模块需要实现：

1. **身份识别**：准确识别当前操作用户的身份
2. **访问控制**：限制未登录用户访问受保护资源
3. **会话管理**：维护用户登录状态，支持多设备同时登录
4. **安全保障**：防止常见的安全攻击（XSS、CSRF、暴力破解）

### 2.2 设计原则

| 原则 | 说明 |
|:-----|:-----|
| **极简优先** | 不引入复杂的权限系统，仅实现基础的用户认证 |
| **无状态设计** | 使用 JWT Token，服务端不存储会话状态 |
| **安全第一** | 采用业界标准的安全实践（bcrypt、httpOnly Cookie） |
| **用户友好** | 提供清晰的错误提示和流畅的登录体验 |

## 3. 功能需求

### 3.1 用户注册功能

#### 3.1.1 功能描述

新用户可以通过注册页面创建账号，注册成功后自动登录。

#### 3.1.2 业务规则

| 规则 | 说明 |
|:-----|:-----|
| 用户名格式 | 3-20 个字符，只能包含字母、数字和下划线 |
| 用户名唯一性 | 用户名不能重复，若已存在则提示"用户名已存在" |
| 密码格式 | 至少 6 个字符 |
| 自动登录 | 注册成功后自动登录，无需再次输入密码 |

#### 3.1.3 用户交互流程

```
用户访问注册页 → 输入用户名和密码 → 点击注册按钮
  ↓
系统验证格式 → 检查用户名是否已存在
  ↓
创建账号 → 自动登录 → 跳转到首页
```

### 3.2 用户登录功能

#### 3.2.1 功能描述

已注册用户可以通过登录页面输入用户名和密码进行身份验证。

#### 3.2.2 业务规则

| 规则 | 说明 |
|:-----|:-----|
| 登录验证 | 验证用户名和密码是否匹配 |
| 用户名空白处理 | 登录时忽略用户名首尾空白，避免复制粘贴造成不可见输入错误 |
| 错误提示 | 用户名或密码错误时，统一提示"用户名或密码错误" |
| 登录状态保持 | 登录成功后保持 7 天有效状态 |
| 重定向 | 登录成功后跳转到之前访问的页面或首页 |
| 登录页背景 | 登录页使用全局主题背景，避免认证卡片与页面背景主题不一致 |

#### 3.2.3 用户交互流程

```
用户访问登录页 → 输入用户名和密码 → 点击登录按钮
  ↓
系统验证身份 → 检查用户名和密码
  ↓
登录成功 → 跳转到目标页面
```

### 3.3 用户登出功能

#### 3.3.1 功能描述

已登录用户可以通过用户菜单执行登出操作。

#### 3.3.2 业务规则

| 规则 | 说明 |
|:-----|:-----|
| 登出操作 | 清除登录状态，跳转到登录页 |
| 多设备影响 | 登出仅影响当前设备，不影响其他设备的登录状态 |

#### 3.3.3 用户交互流程

```
用户点击右上角用户菜单 → 点击"登出"按钮
  ↓
清除登录状态 → 跳转到登录页
```

### 3.4 路由访问控制

#### 3.4.1 功能描述

系统根据用户登录状态自动控制页面访问权限。

#### 3.4.2 访问规则

| 场景 | 用户状态 | 访问目标 | 系统行为 |
|:-----|:---------|:---------|:---------|
| 访问受保护页面 | 未登录 | `/demo`、`/projects`、`/api/sessions` | 重定向到登录页，记录原始访问路径 |
| 访问认证页面 | 已登录 | `/login`、`/register` | 重定向到首页 |
| 访问公开页面 | 任意状态 | `/`、其他公开页面 | 正常访问 |

#### 3.4.3 重定向机制

```
未登录访问受保护页面 → 重定向到登录页（携带 redirect 参数）
  ↓
用户登录成功 → 跳转到 redirect 参数指定的页面（或首页）
```

### 3.5 用户信息展示

#### 3.5.1 功能描述

在页面右上角展示当前登录用户的用户名和操作菜单。

#### 3.5.2 展示内容

| 内容 | 说明 |
|:-----|:-----|
| 用户图标 | 显示用户头像图标 |
| 用户名 | 下拉菜单中显示当前登录用户名 |
| 登出按钮 | 提供登出操作入口 |

## 4. 数据规范

### 4.1 用户数据

| 字段 | 类型 | 说明 |
|:-----|:-----|:-----|
| 用户 ID | 字符串 | 全局唯一标识符（UUID） |
| 用户名 | 字符串 | 3-20 字符，字母、数字、下划线 |
| 密码哈希 | 字符串 | 加密后的密码（不对外暴露） |
| 创建时间 | 数字 | 账号创建时间戳 |

### 4.2 认证凭证

| 属性 | 说明 |
|:-----|:-----|
| 存储方式 | httpOnly Cookie |
| 有效期 | 7 天 |
| 安全属性 | sameSite: lax；HTTPS 生产环境启用 secure，HTTP 内网部署需关闭 secure |

## 5. 安全需求

### 5.1 密码安全

| 措施 | 说明 |
|:-----|:-----|
| 密码加密 | 使用 bcrypt 算法进行单向哈希 |
| 加盐处理 | bcrypt 自动生成随机盐值 |
| 强度设置 | 10 轮加密迭代 |

### 5.2 Token 安全

| 措施 | 说明 |
|:-----|:-----|
| httpOnly Cookie | 防止 JavaScript 读取，避免 XSS 攻击 |
| sameSite: lax | 防止跨站请求伪造（CSRF） |
| 签名验证 | 使用密钥签名，防止篡改 |
| 有效期限制 | 7 天自动过期 |

### 5.3 输入验证

| 措施 | 说明 |
|:-----|:-----|
| 前端验证 | 实时验证用户名和密码格式 |
| 后端验证 | API 层再次验证输入格式 |
| 错误提示 | 不暴露具体错误原因（如"用户名不存在"） |

### 5.4 防护措施

| 威胁 | 防护措施 |
|:-----|:---------|
| XSS 攻击 | httpOnly Cookie，输入内容转义 |
| CSRF 攻击 | sameSite Cookie 属性 |
| 暴力破解 | 暂未实现速率限制（后续优化） |
| 中间人攻击 | 公网生产环境使用 HTTPS；HTTP 内网部署需要显式关闭 secure Cookie，避免浏览器丢弃登录态 |

## 6. 非功能需求

### 6.1 性能需求

| 指标 | 要求 |
|:-----|:-----|
| 登录响应时间 | < 500ms |
| 注册响应时间 | < 500ms |
| Token 验证时间 | < 10ms |

### 6.2 可用性需求

| 需求 | 说明 |
|:-----|:-----|
| 错误提示清晰 | 用户名格式错误、密码过短等都有明确提示 |
| 操作流程简洁 | 注册和登录流程不超过 3 步 |
| 响应式设计 | 支持桌面和移动设备访问 |

### 6.3 兼容性需求

| 需求 | 说明 |
|:-----|:-----|
| 浏览器兼容 | 支持 Chrome、Firefox、Safari、Edge 最新版本 |
| 多设备支持 | 同一账号可在多个设备同时登录 |

## 7. 约束与限制

### 7.1 技术约束

| 约束 | 说明 |
|:-----|:-----|
| 数据库 | 使用 SQLite 存储用户数据 |
| 认证方案 | 纯 JWT，不使用 Session |
| 前端框架 | Next.js 14 App Router |

### 7.2 业务限制

| 限制 | 说明 |
|:-----|:-----|
| 无角色权限 | 不区分管理员、普通用户等角色 |
| 无密码重置 | 暂不支持密码找回和重置功能 |
| 无用户资料 | 暂不支持编辑用户昵称、头像等信息 |

## 8. 后续优化方向

| 优化项 | 说明 |
|:-----|:-----|
| 速率限制 | 防止暴力破解攻击 |
| 密码重置 | 支持通过邮箱重置密码 |
| 用户资料 | 支持编辑昵称、头像等个人信息 |
| 角色权限 | 引入管理员、普通用户等角色 |
| 第三方登录 | 支持微信、GitHub 等第三方账号登录 |
| 双因素认证 | 提升账号安全性 |
