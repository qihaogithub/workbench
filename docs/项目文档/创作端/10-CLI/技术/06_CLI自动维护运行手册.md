# CLI 自动维护运行手册

> 更新日期：2026-06-29

## 读者

这份手册面向执行 CLI 自动维护定时任务的编码代理。它告诉代理每次运行时先读什么、怎么判断风险、能改哪些文件、跑哪些验证，以及什么时候必须停下来。

## 每次运行前必须读取

按顺序读取：

1. [CLI 自动维护定时任务需求](../CLI自动维护定时任务_需求文档.md)
2. [CLI 自动对齐治理方案](./03_CLI自动对齐治理方案.md)
3. [CLI 自动维护定时任务执行设计](./04_CLI自动维护定时任务执行设计.md)
4. [CLI 能力自动化清单](./05_CLI能力自动化清单.md)
5. CLI 模块索引和相关模块文档

如果改动涉及项目、模板、页面、配置、资产、发布、AI 会话或审计，还要读取对应模块的项目文档，确认业务语义。

## 允许自动处理的改动

自动任务可以处理：

| 类型 | 条件 |
|:-----|:-----|
| CLI 查询命令 | 已有领域服务能力，且不修改远端数据 |
| CLI 命令元数据 | 只补能力编号、风险等级、自动化等级 |
| JSON 输出测试 | 不改变业务语义 |
| 文档和能力清单 | 与代码行为保持一致 |
| 命令别名 | 不影响现有命令 |

## 需要人工审核的改动

以下改动可以自动起草，但不能自动合入：

| 类型 | 原因 |
|:-----|:-----|
| 创建或更新业务数据 | 需要确认用户意图和审计影响 |
| 提交本地项目包 | 可能改变项目版本 |
| 资产替换 | 可能影响页面引用 |
| AI 会话发消息 | 会触发外部服务或在线会话 |
| 发布前检查策略变化 | 可能影响发布阻断规则 |

## 必须停止的改动

遇到以下情况，自动任务必须停止，输出报告，等待人工处理：

| 情况 | 原因 |
|:-----|:-----|
| 删除、回滚、发布能力变化 | 高风险链路不能自动合入 |
| 权限、鉴权、管理后台变化 | 涉及安全边界 |
| 领域服务缺少明确能力 | CLI 不能复制 Web 逻辑 |
| 需要新增业务规则 | 自动任务不能自行定义产品行为 |
| 改动需要密钥或真实线上环境 | 自动任务不能依赖不可控外部状态 |
| 工作区存在冲突且影响目标文件 | 不能覆盖用户改动 |

## 推荐运行节奏

| 触发方式 | 目的 |
|:-----|:-----|
| 每日定时 | 发现已经进入主分支但 CLI 未跟进的能力 |
| PR 检查 | 在 Web 新功能合入前提醒 CLI 缺口 |
| 手动触发 | 项目维护者要求立即补齐 CLI |

初期建议只启用每日定时和手动触发，自动合入范围仅限 L3 只读能力。

## 标准执行步骤

### 1. 建立运行上下文

确认当前分支、未提交改动和已有自动维护 PR。自动任务不得回滚或整理与当前目标无关的改动。

### 2. 对账

使用 [CLI 能力自动化清单](./05_CLI能力自动化清单.md) 对比：

- Web 是否有新增业务入口。
- 领域服务是否有新增业务方法。
- CLI 命令是否登记对应能力。
- CLI 测试是否覆盖所有登记命令。
- 文档是否说明新增或变化的能力。

### 3. 选择处理等级

按最高风险原则选择等级：

| 判断 | 等级 |
|:-----|:-----|
| 只需要报告 | L1 |
| 可自动起草但不合入 | L2 |
| 可自动实现并合入 | L3 |
| 自动实现后必须审核 | L4 |
| 只能输出方案或草案 | L5 |

### 4. 自动实现

实现顺序固定为：

1. 补领域服务缺口判断。如果领域服务没有能力，停止。
2. 补 CLI 命令或命令元数据。
3. 补 JSON 输出契约测试。
4. 补能力清单。
5. 补 CLI 文档和索引。
6. 生成运行报告。

自动任务不能为了让测试通过而放宽错误结构、移除确认步骤或跳过高风险保护。

### 5. 验证

按改动范围运行：

| 范围 | 命令 |
|:-----|:-----|
| CLI | `corepack pnpm check:project-cli` |
| 项目核心服务 | `corepack pnpm check:project-core` |
| 本地项目包协议 | `corepack pnpm check:project-scaffold` |
| Web API | `corepack pnpm check:author` |
| 多包共享类型 | 使用范围内所有相关 `check:*` |

自动任务必须使用 `corepack pnpm`，不要直接调用全局 `pnpm`。当前仓库声明 `pnpm@8.15.0`，直接使用 Codex 运行环境里的全局 pnpm 可能触发锁文件不兼容和依赖重装。

验证结果必须写入运行报告或 PR 描述。

### 6. 输出

每次运行必须输出：

| 字段 | 说明 |
|:-----|:-----|
| runId | 本次运行标识 |
| trigger | 定时、PR 检查或手动 |
| detectedGaps | 发现的能力缺口 |
| changedFiles | 自动改动文件 |
| automationLevel | L1 到 L5 |
| validation | 验证命令和结果 |
| decision | 自动合入、等待审核、停止或失败 |
| nextActions | 后续建议 |

## PR 规则

自动维护 PR 应满足：

- 标题包含 `CLI 自动维护` 和能力编号。
- 描述列出能力缺口、风险等级、自动化等级和验证结果。
- 只包含本次能力补齐需要的文件。
- 不包含 `.env`、生成数据、截图、数据库或缓存文件。
- L4 和 L5 必须明确写出人工审核点。

## 自动合入规则

只有同时满足以下条件，才允许自动合入：

- 所有改动都是 L3。
- 验证全绿。
- 改动不触及删除、回滚、发布、权限、鉴权、管理后台。
- 改动不新增业务规则。
- 改动范围只包含 CLI、测试、能力清单或文档。
- 没有未解释的失败、冲突或外部依赖。

任何一个条件不满足，都必须保留 PR 等待人工处理。

## 运行报告模板

运行报告应包含：

| 项目 | 内容 |
|:-----|:-----|
| 本次结论 | 自动合入、等待审核、停止或失败 |
| 发现缺口 | 能力编号、模块、缺口说明 |
| 自动改动 | 文件列表和改动摘要 |
| 风险判断 | 风险等级和依据 |
| 验证结果 | 命令、通过情况、失败摘要 |
| 待人工确认 | 需要确认的问题 |
| 后续建议 | 下一次自动任务或人工处理方向 |

## 维护原则

- 宁可多开待审核 PR，也不要错误自动合入高风险改动。
- 宁可标记待确认，也不要让 CLI 猜业务规则。
- 优先复用领域服务，不复制 Web 页面逻辑。
- 每次自动修改都要能被能力编号追溯。
- 文档、测试和命令必须一起维护，不能只补命令。
