# CLI 能力需求文档

> 更新日期：2026-06-30

## 模块定位

CLI 模块是创作端面向编码代理的本地入口。它让 Codex、Claude Code、Cursor Agent 等代理可以通过终端命令完成项目拉取、本地开发校验、差异查看、提交、发布、模板维护、审计查询和必要的管理操作。

普通用户继续通过 Web 创作端完成可视化编辑；CLI 服务的是需要直接读写本地工程、运行测试、查看终端输出和提交结构化结果的代理工作流。

当用户需要把创作端项目带到本地开发时，应通过 CLI 拉取本地项目包。创作端项目编辑页不再重复提供“下载代码”入口，避免 Web 可视化编辑和本地工程协作形成两套并列入口。

## 目标用户

- 编码代理：需要稳定的命令和 JSON 输出，不能依赖人类可读文案判断成败。
- 项目管理员：需要在必要时通过本地命令批量查看项目、模板、发布和审计状态。
- 开发者：需要把线上创作端项目拉到本地，像维护普通前端工程一样调试和验证。

## 功能需求

### 项目管理

用户可以通过 CLI 列出项目、查看项目详情、创建项目、更新项目信息、复制项目、设置封面和执行受保护的删除操作。

删除、回滚等高风险操作必须先给出影响预览，再通过确认信息执行，避免代理误删线上数据。

### 本地项目包

用户可以把线上项目拉取为本地项目包。本地项目包必须包含项目清单、页面源码、页面配置、项目配置、资产、应用逻辑图、工作空间知识文档和同步基线，让代理可以在普通目录中完成开发。

用户可以在本地运行校验、查看 diff，并把变更提交回创作端。提交完成后，本地基线需要更新，后续 diff 只显示新的业务改动。

### 模板工作流

用户可以从模板初始化本地项目，也可以把本地项目提交后保存为新的线上模板快照。模板工作流必须复用项目的版本、审计和校验规则，不绕过项目管理主流程。

### 页面、文件夹、配置和资产

用户可以通过 CLI 管理页面、虚拟文件夹、项目级配置、页面级 Schema、页面历史版本和图片资产。页面历史需要支持读取版本列表、读取单个版本内容、创建命名页面快照，以及执行受保护的恢复操作。

CLI 需要支持复杂参数输入，避免代理在命令行里处理大段代码、JSON 或图片内容时发生转义错误。

### 预览与发布

用户可以通过 CLI 执行发布前检查、发布、查询发布状态、查看发布产物摘要和回滚发布状态。

正式发布应优先走创作端 Web 发布链路；当远端发布配置缺失时，CLI 可以提供本地状态发布降级结果，但必须在输出中明确说明。

### AI 会话与审计

用户可以通过 CLI 查询项目相关 AI 会话摘要、运行日志和工作区上下文，也可以向在线 AI 会话发送指令。用户还可以查询项目审计记录，用于追溯代理或管理员操作。

## 业务规则与约束

- CLI 是代理入口，不替代 Web 创作端的可视化体验。
- 本地代码获取以 CLI 本地项目包流程为准，Web 创作端编辑页不提供重复下载入口。
- CLI 必须提供机器可读结果，关键命令都要支持 JSON 输出。
- 代理只应依赖 JSON 字段，不解析普通终端文案。
- CLI 必须提供简单稳定的本地入口。用户或代理安装依赖后，应能通过仓库根命令或包命令直接运行 CLI，而不需要理解源码运行器。
- CLI 不重新定义项目、版本、模板、发布和权限规则，这些语义以 [项目管理模块](../03-项目管理/) 和 [配置与预览模块](../04-配置与预览/) 为准。
- 本地构建或校验通过不代表可以直接成为正式发布产物，提交和发布仍需要经过创作端服务侧复验。
- 本地项目包中的同步状态属于内部基线，代理不应把它当作业务源码编辑。
- CLI 不承担通用前端工程命令，开发、测试和构建继续由本地脚手架自己的脚本完成。

## 验收标准

- 代理可以用一条命令把线上项目拉取到本地目录。
- 本地目录包含可读的项目清单、页面文件、配置文件、资产目录、应用逻辑图和知识文档目录。
- 代理修改本地项目后，可以通过 CLI 得到稳定的校验结果和 diff 摘要。
- 代理可以把本地变更提交回创作端，并在成功后获得新的版本基线。
- 发布命令返回产物摘要和访问入口。
- 所有失败结果都提供稳定错误码和下一步建议。
- CLI 能覆盖项目、模板、页面、文件夹、配置、资产、预览、发布、AI 会话、审计和管理员锁定这些管理面能力。
- 用户可以从仓库根目录用统一入口执行 `doctor`、项目列表和事务类命令，入口在受限 shell 环境中不依赖交互式源码运行器。
