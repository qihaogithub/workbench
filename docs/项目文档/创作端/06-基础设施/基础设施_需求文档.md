# 基础设施模块 - 需求文档

> 版本：v1.0
> 创建日期：2026-05-02

---

## 一、模块定位

基础设施模块为整个 Web 前端应用提供底层支撑能力，包括 API 接口规范、错误处理机制、动态编译、Schema 生成等通用服务。

---

## 二、功能描述

### 2.1 API 接口规范

#### 2.1.1 统一响应格式

所有 API 使用统一的响应格式：

**成功响应**：
```json
{
  "success": true,
  "data": { ... }
}
```

**错误响应**：
```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述",
    "details": { ... }
  }
}
```

### 2.2 动态编译功能

#### 2.2.1 代码编译

**用户视角**：
- 系统应能动态编译 TypeScript/JSX 代码
- 编译结果应可直接在预览区执行

**交互效果**：
- 编译过程应在后台执行
- 编译失败时应展示错误信息

### 2.3 Schema 生成功能

#### 2.3.1 自动提取 Props

**用户视角**：
- 系统应能从组件代码中自动提取 props 定义
- 提取结果应格式化为 JSON Schema

## 三、业务规则与约束

### 3.1 错误码规范

| 错误码 | HTTP 状态码 | 说明 |
|:-------|:------------|:-----|
| PROJECT_NOT_FOUND | 404 | 项目不存在 |
| SESSION_NOT_FOUND | 404 | 会话不存在或已过期 |
| VERSION_NOT_FOUND | 404 | 版本不存在 |
| INVALID_REQUEST | 400 | 请求参数无效 |
| FILE_READ_ERROR | 500 | 文件读取失败 |
| FILE_WRITE_ERROR | 500 | 文件写入失败 |
| AI_SERVICE_ERROR | 503 | AI 服务不可用 |

### 3.2 编译约束

| 约束 | 说明 |
|:-----|:-----|
| 依赖白名单 | 仅允许项目已有的依赖 |
| 代码来源 | 组件代码由 AI 生成，经过系统提示词约束 |
