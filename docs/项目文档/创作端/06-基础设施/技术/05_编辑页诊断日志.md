# 编辑页诊断日志

> 更新日期：2026-07-01

## 定位

编辑页诊断日志是给开发排查和 AI 代理使用的辅助能力。它把项目编辑页中的实时协作、自动保存、AI 对话和预览运行事件放进同一条时间线，帮助排查“看起来不顺畅”的跨链路问题。

普通用户不需要看到常驻诊断面板。排查时可通过编辑页 URL 增加 `?diagnostics=1` 显示隐藏导出入口，也可以使用快捷键导出诊断包。

## 事件模型

每次进入编辑页会创建一个 `editorSessionId`。关键动作会按需创建 `traceId`，用于串联一次自动保存、一次 AI 消息发送或一次预览错误处理。

诊断事件分为六类：

| 类别 | 用途 |
|:-----|:-----|
| `collab` | 记录协同房间连接、同步、离线、错误和在线人数摘要 |
| `autosave` | 记录工作区变更、延迟 flush、项目工作区同步和退出前保存 |
| `ai` | 记录 AI 消息发送、WebSocket 连接、流结束、文件变更和自动修复触发 |
| `preview` | 记录预览 runtime 阶段、iframe 加载完成和预览诊断错误 |
| `ui` | 记录隐藏导出入口、快捷键等排查操作 |
| `system` | 记录诊断能力启动和内部异常 |

事件只保存排查所需的摘要，例如页面 ID、资源路径、状态、耗时、错误码、文件数量和文本长度。完整源码、完整 AI 提示词、密钥、token、Authorization、上传文件内容和数据库快照都不写入诊断事件。

## 写入与导出

前端 `useEditorDiagnostics` 在浏览器内保留最近事件缓冲，并按短延迟批量提交到 author-site。后端把事件追加写入：

`data/editor-diagnostics/<editorSessionId>.jsonl`

单个日志文件有大小上限，超过上限时保留较新的 JSONL 行。写入失败不会影响编辑、保存、AI 对话或预览，只会在导出包中标记“部分诊断事件后端写入失败”，并附带前端本地缓冲。

导出包由后端 JSONL、前端本地缓冲、当前编辑页快照、协同状态摘要、最近预览诊断和 AI run log 索引组成。AI run log 仍按 [AI 运行进度与事件日志](../../05-AI对话/技术/07_运行进度与事件日志.md) 的路径独立保存，诊断包只提供 session 与 message 的索引。

## 接入边界

编辑页诊断不改变业务状态机：

- 协同房间仍由 [实时保存与协同编辑](../../03-项目管理/技术/11_实时保存与协同编辑.md) 定义。
- 预览错误仍复用 [实时预览机制](../../04-配置与预览/技术/02_实时预览机制.md) 中的 `previewDiagnostic` 回流。
- AI 对话仍复用既有 WebSocket 与运行日志，只额外把本地 messageId、traceId 和状态摘要回传给编辑页诊断。
- 自动保存仍先 flush 协同草稿，再推进项目当前工作区；诊断事件只记录这条链路的开始、成功、失败和耗时。

## 排查方式

排查编辑页问题时，优先让用户在复现前打开 `?diagnostics=1`，复现后点击顶部隐藏的“诊断”按钮导出 JSON。若问题发生时没有打开隐藏入口，快捷键仍可导出当前浏览器内存中的最近事件，但后端 JSONL 可能不完整。

拿到诊断包后，先按 `traceId` 查看单次操作，再按时间线确认事件是否卡在协同、自动保存、AI 发送还是预览加载阶段。
