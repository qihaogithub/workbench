## 一、定位

全局图床是跨项目共享的统一图片存储层，每张图片拥有全局唯一 ID 和稳定访问 URL。它替代了旧有的按项目隔离的 `workspace/assets/images/` 文件式存储，消除了 AI 创作中"相对路径在配置面板与页面预览之间解析不一致"的问题。

## 二、存储架构

```
data/image-store/
├── blobs/                     # 物理图片文件
│   └── {sha256[:16]}.{ext}   # SHA256 前16位命名，天然去重
└── manifest.json              # 全局注册表，记录每张图片的元信息
```

### 去重机制

每张图片入库时先计算 SHA256 哈希，在 `manifest.json` 中查找是否已有相同哈希的图片：

- **命中**：不写入物理文件，返回已有 `imageId`，在引用计数中追加当前项目
- **未命中**：写入 `blobs/{sha256[:16]}.{ext}`，新建 manifest 条目

### 图片 ID 格式

全局唯一 ID 格式为 `img_` + 随机字符串，如 `img_nRfQ2rJEw1oOpQ`。该 ID 与物理文件命名（SHA256 前缀）解耦，支持未来存储介质迁移而不影响已有引用。

### 引用追踪

每张图片的 `projectRefs` 字段记录引用了它的项目 ID 列表，用于追踪使用情况和未来的清理策略。

## 三、API 端点

### 图片上传

`POST /api/images/upload` 支持两种请求格式：

- **Multipart**（前端 FileUploadWidget）：`FormData` 含 `file` 字段和可选 `projectId`
- **JSON**（AI 工具调用）：`{ data: base64, filename: string, projectId?: string }`

上传需要登录鉴权，上传后的图片自动链接到指定项目。

### 图片获取

`GET /api/images/[imageId]` 返回原始图片数据，设置永久缓存头（`immutable`），无需鉴权。同源绝对路径 `/api/images/img_xxx` 可在页面代码、配置面板 `img` 标签中直接使用。

### 向后兼容

`GET /api/images/[imageId]` 路由同时兼容旧的 `data/images/` 目录：当 imageId 不是全局图床 ID 时，回退到 `data/images/{filename}` 文件服务。

## 四、数据流

### AI 创作流程

```
AI 调用 saveImage(url) 
  → agent-service 下载图片数据
  → 写入 data/image-store/blobs/{sha256[:16]}.png
  → 更新 data/image-store/manifest.json
  → 链接到项目 images.json（记录 imageId）
  → 返回 /api/images/img_xxx
  → AI 在页面代码和 config.schema.json 中使用该 URL
```

### 前端上传流程

```
用户在配置面板点击上传
  → FileUploadWidget 发送文件到 POST /api/images/upload
  → author-site 写入 data/image-store/blobs/
  → 更新 manifest.json
  → 链接到项目 images.json
  → 返回 /api/images/img_xxx
  → FileUploadWidget 将值设为该 URL，缩略图直接显示
```

### 页面预览流程

由于页面代码和配置数据中直接使用 `/api/images/img_xxx` 绝对路径，浏览器在同源下直接加载图片，不再需要 `resolvePreviewConfigAssetUrls` 或 `rewriteLocalAssetPaths` 的中间路径转换。

存量项目（仍使用 `../../assets/images/...` 相对路径）通过 workspace proxy 的 fallback 链继续工作：文件系统命中 → 查项目 `images.json` 找 `imageId` → 从全局图床提供。

## 五、与旧方案的对比

| 维度 | 旧方案（workspace 文件） | 新方案（全局图床） |
|------|------------------------|-------------------|
| 引用路径 | `../../assets/images/{hash}-{name}.png` | `/api/images/img_xxx` |
| 配置面板预览 | 相对路径无法显示缩略图 | 绝对路径直接显示 |
| 页面预览 | 需 `resolvePreviewConfigAssetUrls` 重写 | 浏览器直接加载 |
| 跨项目共享 | 文件拷贝到每个项目 workspace | 全局一份，按项目引用 |
| 去重 | 项目内 SHA256 文件名去重 | 全局 SHA256 内容去重 |
| AI 认知负担 | 需理解两段式路径模型 | 一个稳定 URL，位置无关 |

## 六、相关模块

- **saveImage 工具**：位于 `packages/agent-service/src/backends/pi-tools/save-image-tool.ts`，已迁移到全局图床存储
- **项目图片清单**：`images.json` 新增 `imageId` 字段，指向全局图床条目
- **配置面板**：[FileUploadWidget](../../04-配置与预览/技术/03_表单生成器.md) 通过 `POST /api/images/upload` 上传，使用返回的 `/api/images/img_xxx` URL 作为字段值
- **存量迁移**：设计文档见 `docs/plans/已完成/全局图床与AI图片使用能力设计.md`

## 七、限制与边界

- 单张图片最大 10MB
- 支持格式：png、jpg、jpeg、gif、webp、svg
- 入库时同步读取图片尺寸（宽/高），解析失败不阻塞入库
- manifest.json 损坏时启动期可从 `blobs/` 目录扫描重建
- 图片删除策略（基于引用计数）为远期规划，当前不实现
