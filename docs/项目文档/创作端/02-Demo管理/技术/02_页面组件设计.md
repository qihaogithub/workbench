## 一、组件架构

### 1.1 组件分层

```
┌─────────────────────────────────────────────────────────────────┐
│                         页面层 (Pages)                           │
│  HomePage / DemoEditPage                                        │
│  (Demo 使用页合并至编辑页)                                       │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                         业务组件层                               │
│  DemoCard / CreateDemoDialog / DeleteConfirmDialog              │
│  AIChat(ai-elements) / DemoPageTree / CodeViewDialog           │
│  NewFolderDialog / ExplorePage / WishPage                       │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                     AI 对话子组件层                               │
│  ChatMessages / ChatInput / ChatPlan / HistoryDialog            │
│  ┌─ hooks: useChatMessages / useChatStream / useChatModels     │
│  └─ services: message-service / stream-service                 │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                         基础组件层 (UI)                          │
│  Button / Card / Dialog / Input / Toast / Alert / Badge        │
│  Tabs / Select / Switch / Slider / Resizable / Tooltip         │
│  Avatar / DropdownMenu / ScrollArea / Skeleton / Toggle        │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 组件库选型

| 组件库 | 用途 | 说明 |
|:-------|:-----|:-----|
| shadcn/ui | 基础组件 | 基于 Radix UI，可定制性强 |
| Lucide React | 图标 | 轻量、Tree-shakable |
| AI Elements (ai-sdk) | AI 对话组件 | 流式响应、工具调用展示、推理面板 |
| RJSF（通过运行时注入） | 表单生成 | JSON Schema 驱动 |
| Sandpack（通过 iframe） | 代码预览 | 独立渲染环境 |

### 1.3 操作反馈

普通成功、状态刷新和轻量错误反馈使用全局 Toast，而不是在页面角落堆叠常驻通知卡片。Toast 以右下角轻量浮层展示，默认 3 秒自动消失；只有需要阻断用户决策的风险或确认场景才使用 Dialog、Alert 或页面内状态区。

## 二、核心组件设计

### 2.1 首页组件

#### 2.1.1 HomePage 组件

**文件位置**：`src/components/demo/home-page.tsx`

**职责**：首页主组件，管理 Demo 列表的搜索、创建、删除

**Props 接口**：

```typescript
interface HomePageProps {
  initialDemos: DemoMeta[]
}
```

**状态管理**：
- `searchQuery` — 搜索关键词
- `isCreateDialogOpen` — 创建对话框状态
- `deleteTarget` — 待删除的 Demo

**交互设计**：
- 搜索框实时过滤 Demo 列表（使用 useMemo）
- 空列表时显示空状态提示
- 创建成功自动跳转到编辑页

#### 2.1.2 DemoCard 组件

**文件位置**：`src/components/demo/demo-card.tsx`

**职责**：展示单个 Demo 的卡片信息

**Props 接口**：

```typescript
interface DemoCardProps {
  demo: DemoMeta
  onDelete: (id: string) => void
}
```

**交互设计**：
- 卡片整体为 Link，点击跳转到 `/demo/[id]/edit`
- 悬停显示操作菜单（DropdownMenu）
- 删除通过 DropdownMenu 触发
- 预览图缺失时显示与项目卡片一致的默认封面占位：项目级占位使用深色渐变、网格和页面线框表达项目结构；页面截图缺失时，单个截图格使用线框式页面占位，不显示灰黑空块或播放按钮
- 有自定义封面时，封面图在卡片预览区域内按原比例完整缩放展示，不裁切图片内容
- 无自定义封面但存在页面截图时，卡片封面最多展示前 10 个页面截图；截图先按默认竖屏比例占位，加载后根据实际宽高比分配行内宽度，竖图占窄位、横图占宽位；3 张以上按最多两行平衡分布，每张截图都在各自缩略卡内按原比例完整缩放展示，避免裁切页面内容。截图请求失败或尚未生成时，该截图格回退到线框式页面占位

#### 2.1.3 CreateDemoDialog 组件

**文件位置**：`src/components/demo/create-demo-dialog.tsx`

**职责**：创建新 Demo 的对话框

**Props 接口**：

```typescript
interface CreateDemoDialogProps {
  open: boolean
  onOpenChange: (open: boolean) => void
  onCreate: (name: string) => Promise<void>
}
```

**交互设计**：
- 表单提交后调用 onCreate 回调
- 名称不能为空时按钮禁用
- 支持 Enter 键快速提交

#### 2.1.4 DeleteConfirmDialog 组件

**文件位置**：`src/components/demo/delete-confirm-dialog.tsx`

**职责**：删除 Demo 的二次确认对话框

**交互设计**：
- 显示待删除 Demo 名称
- 确认后触发删除操作

### 2.3 AIChat 组件

**文件位置**：`src/components/ai-elements/ai-chat.tsx`

**职责**：AI 对话交互面板，替代旧的 ChatPanel

**Props 接口**：

```typescript
interface AIChatProps {
  sessionId: string
  agentSessionId: string
  workingDir?: string
  projectId?: string
  demoId?: string
  workspaceId?: string
  onCodeUpdate?: (code: string) => void
  onSchemaUpdate?: (schema: string) => void
  onFilesChange?: (files: FileChange[]) => void
  // 外部状态注入（父组件可接管消息管理）
  externalMessages?: ChatMessage[]
  externalIsStreaming?: boolean
  externalStreamContent?: string
  externalCurrentMessage?: ChatMessage
  onMessagesChange?: (messages: ChatMessage[]) => void
  onNewSession?: (workspaceId?: string) => void
  onSelectSession?: (sessionId: string, workspaceId?: string) => void
  currentSessionId?: string
}
```

**内部架构**（位于 `src/components/ai-elements/chat/`）：
- `hooks/use-chat-messages.ts` — 消息状态管理
- `hooks/use-chat-stream.ts` — 流式通信管理（含计划、权限请求）
- `hooks/use-chat-models.ts` — 模型切换管理
- `services/message-service.ts` — 消息服务
- `services/stream-service.ts` — 流式服务
- `utils/chat-file-utils.ts` — 文件工具
- `utils/chat-stream-utils.ts` — 流工具
- `chat-messages.tsx` — 消息列表展示
- `chat-input.tsx` — 输入框（含模型选择、历史切换）
- `chat-plan.tsx` — AI 计划展示

**技术实现**：
- WebSocket 连接 Agent Service
- 流式响应展示
- 工具调用确认交互（PermissionDialog）
- 支持历史对话切换（HistoryDialog）
- 模型运行时切换

## 三、组件通信

### 3.1 配置联动机制

配置修改通过运行时生成的 iframe 页面实现：
```
用户修改配置参数
        │
        ▼
父组件更新 configData
        │
        ▼
重新生成演示页面代码（含配置注入）
        │
        ▼
Viewer-site iframe 重新渲染
```

### 3.2 编辑联动机制

```
AI Chat 组件
        │
        ├──► onCodeUpdate(newCode)
        │          │
        │          ▼
        │    更新 Session 文件 → Viewer iframe 刷新
        │
        ├──► onSchemaUpdate(newSchema)
        │          │
        │          ▼
        │    更新配置定义 → 配置表单更新
        │
        └──► onFilesChange([{path, action}])
                   │
                   ▼
             更新文件状态 → UI 显示变更
```

## 四、性能优化

### 4.1 组件优化

| 优化措施 | 适用组件 | 说明 |
|:---------|:---------|:-----|
| React.memo | PreviewPanel | 避免不必要的重渲染 |
| useMemo | Schema 解析 | 缓存解析结果 |
| useCallback | 事件回调 | 缓存回调函数 |

### 4.2 渲染优化

| 优化措施 | 说明 |
|:---------|:-----|
| 虚拟列表 | Demo 列表数量大时使用 |
| 防抖处理 | 高频输入场景 |
| 懒加载 | 非关键组件延迟加载 |

## 五、错误处理

### 5.1 错误边界

在页面层级包裹 Error Boundary：

```typescript
<ErrorBoundary fallback={<ErrorFallback />}>
  <PageContent />
</ErrorBoundary>
```

### 5.2 错误类型

| 错误类型 | 处理方式 |
|:---------|:---------|
| API 错误 | 轻量 Toast 提示，必要时配合页面内重试按钮 |
| 编译错误 | 预览区展示错误信息 |
| 运行时错误 | Error Boundary 捕获 |
