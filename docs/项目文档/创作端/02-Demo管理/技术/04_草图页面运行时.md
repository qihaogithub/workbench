# 草图页面运行时

> 更新日期：2026-07-05
> 适用模块：页面管理、单页面预览、项目工作区、版本与恢复

## 1. 定位

草图页底层协议仍按高保真 React 页和 HTML/CSS 原型页之外的第三类页面运行时保留，但创作端当前默认暂停开放。它面向早期页面结构、线框草稿和可拖拽排版，核心数据是一份稳定的 `SketchSceneDocument`，而不是 React 代码、原型 HTML 或绘图库私有 JSON。

当前产品边界是：草图 SDK 和底层读写能力继续用于独立开发、兼容验证和未来接入准备；author-site 不展示新建草图页入口，不允许通过创作端 API 新建或切换到草图页。恢复接入前应先完成 SDK 验收，再显式打开 `NEXT_PUBLIC_SKETCH_SCENE_AUTHORING_ENABLED=true` 并补齐对应回归。

草图页不是画布工作台批注层。画布工作台保存项目级页面摆放、文档、图片和语义文本；草图页保存单个页面内部的图形、文本、图片、组件占位和配置绑定。

## 2. 文件协议

草图页页面目录使用以下文件：

| 文件 | 作用 | 文件树展示 |
| :--- | :--- | :--- |
| `sketch.scene.json` | 页面尺寸、对象、样式、资源引用、绑定和元数据 | 展示 |
| `sketch.meta.json` | 草图引擎版本、来源和转换信息 | 默认隐藏 |
| `config.schema.json` | 页面配置项定义 | 有内容时按既有规则展示 |

`@workbench/sketch-core` 是草图协议、校验、patch、绑定、几何和只读 SVG/HTML 渲染的权威实现。`packages/shared/src/demo/sketch-scene.ts` 只保留兼容转发，旧消费者可以继续导入旧路径；新代码应直接依赖 `sketch-core`。

`project-core` 是页面运行时写入的权威层。创建、切换运行时、恢复版本或保存会话文件时，都会先按 `sketch-core` 校验入口确认草图 scene 可解析，再写入页面目录。高保真页、原型页和草图页采用显式三分支判断，避免草图页误进入 React 编译或原型清洗路径。

## 3. 预览与编辑

单页面草图预览和编辑组件仍保留在代码中，但 author-site 默认不开入口。后续重新接入时，单页面预览可用 `@workbench/sketch-react/preview` 展示当前 scene；进入编辑态后，author-site 只负责会话保存、截图失效和页面状态适配，绘图、选择、图层和属性面板仍由 `@workbench/sketch-react` 受控组件承担。

SDK 还提供独立 playground。开发者可通过 `pnpm dev:sketch` 在不启动登录、项目、AI 或截图服务的情况下调试 fixtures、scene JSON、配置数据、只读渲染和性能基线。

画布工作台对草图页保持页面级交互：页面卡片仍可选择、移动、缩放和查看截图，但不挂载草图编辑器，也不展示页面内部工具栏。这样可以避免项目级画布和页面内部草图编辑互相抢占交互。

## 4. 版本与恢复

底层 `project-core` 仍保留草图 scene 和 meta 的项目工作区、保存、版本、恢复和发布前同步能力，用于兼容既有数据与 SDK 验证。创作端暂停期内，author-site 的新建页 API 会拒绝 `runtimeType=sketch-scene`，运行时切换 API 会拒绝 `targetRuntimeType=sketch-scene`，会话文件写入 API 会拒绝外部写入 `sketchScene` 或 `sketchMeta`。

后续重新开放时，草图页的转换入口仍应使用失败保护：切换为原型页或高保真页前先校验目标产物；切换为草图页前先校验目标 scene。目标校验失败时，原页面类型和内容保持不变。
