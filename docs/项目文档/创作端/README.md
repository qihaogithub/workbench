## 文档概览

本系列文档用于指导 Web 前端（`@opencode-workbench/author-site`）的开发工作，基于 Next.js 14 App Router 架构，实现 Demo 管理、项目管理、配置预览和 AI 对话功能。

**核心设计理念**：

```
"一个项目 = 一个工作空间，编辑在临时空间，保存才合并"
```

**用户鉴权设计理念**：

```
"纯 JWT 无状态认证，安全简洁，多设备友好"
```

## 核心概念

### 用户 (User)

注册用户，拥有唯一的用户名和 ID。通过 JWT Token 进行身份认证，支持多设备同时登录。

### JWT Token (JSON Web Token)

无状态认证凭证，包含用户身份信息，通过 httpOnly Cookie 传输，有效期 7 天。

### 项目 (Project)

一个项目对应一个独立的工作空间，包含项目的所有文件和版本历史。

### 工作空间 (Workspace)

项目的正式存储区域，Agent 操作这里。

### 临时空间 (Temp Workspace)

编辑时的隔离空间，不影响原项目文件。

### 会话 (Session)

编辑会话，关联用户、项目、临时空间。

### 版本快照 (Snapshot)

保存时创建的版本备份，支持回退。

### OpenCode AI 代理配置 (.opencode)

Session 临时工作区中的 AI 代理配置，用于约束 AI 生成代码的行为。包含 `opencode.json` 和 `agents/demo-generator.md`，在创建 Session 时自动注入，保存时自动清理。

## 核心流程

### 打开项目编辑

```
用户打开项目 → 创建临时工作空间 → 复制项目文件 → 注入 .opencode AI 代理配置 → 创建编辑会话
```

### 保存变更

```
用户点击保存 → 创建版本快照 → 临时空间覆盖正式空间 → 记录版本信息
```

### 恢复版本

```
用户选择恢复 → 备份当前状态 → 从快照恢复 → 创建新版本记录
```
