## 文档概览

本系列文档用于指导 Web 前端（`@opencode-workbench/author-site`）的开发工作，基于 Next.js 14 App Router 架构，实现 Demo 管理、项目管理、配置预览和 AI 对话功能。

**核心设计理念**：

```
"Session 不拥有文件，项目工作区承载当前态，历史快照承载回溯"
```

**用户鉴权设计理念**：

```
"纯 JWT 无状态认证，安全简洁，多设备友好"
```

## 核心概念

### 用户 (User)

注册用户，拥有唯一的用户名和 ID。通过 JWT Token 进行身份认证，支持多设备同时登录。

### JWT Token (JSON Web Token)

无状态认证凭证，包含用户身份信息，通过 httpOnly Cookie 传输，有效期 7 天。

### 项目 (Project)

一个项目包含项目工作区、项目基准工作区和版本历史。

### 项目工作区 (active Workspace)

默认编辑页、实时协同、预览和自动保存共同使用的项目级共享当前态。
项目工作区只在基于项目最新版本时可复用；分支事务提交新版本后，旧项目工作区必须失效，重新打开项目会从项目基准工作区创建新的当前态。

### 项目基准工作区 (canonical Workspace)

`projects/{id}/workspace/` 下的项目当前态基准文件区，供发布、导出、历史快照、模板保存、CLI 和脚手架转换等既有项目读写流程读取。

### 分支工作区 (branch Workspace)

CLI、AI 大改或实验性事务使用的显式隔离工作区，不进入默认多人协作房间。

### 会话 (Session)

AI 对话上下文，关联用户、项目和 `workspaceId`，不拥有文件副本。

### 版本快照 (Snapshot)

自动检查点、命名版本、发布和恢复时创建的不可变备份，支持回退。

### Pi Agent 会话配置

AI 会话配置不再依赖工作空间内的 OpenCode 配置目录。创作端把系统提示词、模型选择、工作空间和图片附件发送到 agent-service；agent-service 通过 Pi Agent 后端、Pi Tools 权限和 SessionGuard 约束 AI 行为。

## 核心流程

### 打开项目编辑

```
用户打开项目 → 创建或复用项目工作区 → 创建编辑会话
```

复用项目工作区前必须检查其版本基准；旧版本工作区不能继续承载新的编辑会话。

### 保存变更

```
关键动作 → 同步项目工作区到项目基准工作区 → 创建历史快照 → 记录版本信息
```

同步项目工作区前必须确认它仍是项目当前 active Workspace；过期工作区只能提示刷新，不能覆盖项目基准工作区。

### 恢复版本

```
用户选择恢复 → 备份当前状态 → 从快照恢复 → 创建新版本记录
```
