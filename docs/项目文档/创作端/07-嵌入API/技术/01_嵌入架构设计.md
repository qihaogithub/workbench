## 2. 核心组件

### 2.1 嵌入页面 (`/embed/{demoId}`)

**文件位置**：`src/app/embed/[demoId]/page.tsx`

**职责**：
- 验证 demoId 对应的项目是否存在
- 生成可复制的 iframe 嵌入代码
- 提供实时预览区域

**输出示例**：

```html
<iframe
  src="/embed/demo-123/iframe"
  sandbox="allow-scripts allow-same-origin"
  style="width: 100%; border: none;"
/>
```

### 2.2 iframe 内容服务 (`/embed/{demoId}/iframe`)

**文件位置**：`src/app/embed/[demoId]/iframe/route.ts`

**职责**：
- 读取项目的组件代码和 Schema
- 使用 sucrase 编译 TypeScript 代码
- 生成完整的 HTML 文档
- 注入通信脚本和运行时

**处理流程**：

```
1. 验证项目存在
       ↓
2. 读取 index.tsx 组件代码
       ↓
3. 读取 config.schema.json 配置
       ↓
4. 使用 sucrase 编译代码
       ↓
5. 提取 Schema 中的 default 值
       ↓
6. 生成 HTML 响应
```

### 2.3 项目级嵌入 API (`/api/embed/{projectId}`)

**文件位置**：`src/app/api/embed/[projectId]/iframe/route.ts`

**职责**：
- 支持项目级嵌入（包含多个 Demo 页面）
- 支持通过 `page` 参数指定页面
- 合并项目级 + 页面级 Schema 配置

**查询参数**：

| 参数 | 类型 | 必填 | 说明 |
|:-----|:-----|:-----|:-----|
| `page` | string | 否 | 指定页面 ID，不指定则默认第一个页面 |

## 3. iframe 运行时架构

### 3.1 HTML 生成器

**文件位置**：`src/lib/iframe-template.ts`

**核心函数**：`generateIframeHtml(options)`

```typescript
interface IframeTemplateOptions {
  cssImports?: string[];      // 外部 CSS URL 列表
  compiledCode?: string;        // 编译后的 JS 代码
  configData?: Record<string, unknown>;  // 配置数据
  cdnBaseUrl?: string;         // CDN 基础地址
}
```

**HTML 结构**：

```html
<!DOCTYPE html>
<html>
<head>
  <!-- CDN 资源预加载 -->
  <link rel="preconnect" href="${cdnBase}">

  <!-- Tailwind CSS -->
  <script src="tailwindwindcss.min.js"></script>

  <!-- 动态 CSS -->
  <link data-dynamic-css="true" href="...">

  <!-- 运行时脚本 -->
  <script type="module">
    // React 导入
    // 组件渲染逻辑
    // postMessage 通信
    // ResizeObserver
    // 错误捕获
  </script>
</head>
<body>
  <div id="root"></div>
</body>
</html>
```

### 3.2 组件渲染器

**核心逻辑**：

```javascript
let currentRoot = null;      // React 18 createRoot 实例
let currentConfig = {};      // 当前配置
let currentComponent = null; // 组件模块

function renderComponent() {
  const container = document.getElementById('root');
  currentRoot = ReactDOM.createRoot(container);
  currentRoot.render(
    React.createElement(ErrorBoundary, null,
      React.createElement(currentComponent, currentConfig)
    )
  );
}
```

### 3.3 错误边界

```javascript
class ErrorBoundary extends React.Component {
  componentDidCatch(error, errorInfo) {
    window.parent.postMessage({
      type: 'RUNTIME_ERROR',
      error: error.message,
      stack: error.stack
    }, '*');
  }
}
```

## 4. 通信协议

### 4.1 消息类型定义

```typescript
// 父页面 → iframe
type ParentMessage =
  | { type: 'UPDATE_CODE'; code: string; configData: Record<string, unknown>; cssImports: string[] }
  | { type: 'UPDATE_CONFIG'; configData: Record<string, unknown> };

// iframe → 父页面
type IframeMessage =
  | { type: 'READY' }
  | { type: 'LOADED' }
  | { type: 'RESIZE'; height: number }
  | { type: 'RUNTIME_ERROR'; error: string; stack?: string };
```

### 4.2 消息处理流程

```javascript
window.addEventListener('message', (event) => {
  // 安全校验：只处理来自父窗口的消息
  if (event.source !== window.parent) return;

  const { type, code, configData, cssImports } = event.data;

  switch (type) {
    case 'UPDATE_CODE':
      currentConfig = configData || {};
      updateCssLinks(cssImports || []);
      loadModule(code);
      break;

    case 'UPDATE_CONFIG':
      currentConfig = configData || {};
      if (currentComponent) renderComponent();
      break;
  }
});
```

### 4.3 ResizeObserver 实现

```javascript
const resizeObserver = new ResizeObserver((entries) => {
  for (const entry of entries) {
    const height = entry.contentRect.height;
    window.parent.postMessage({ type: 'RESIZE', height }, '*');
  }
});
resizeObserver.observe(document.body);
```

## 5. 配置合并机制

### 5.1 Schema 结构

**项目级 Schema**：`workspace/project.config.schema.json`

**页面级 Schema**：`workspace/demos/{pageId}/config.schema.json`

### 5.2 合并规则

**文件位置**：`src/lib/runtime-props.ts`

```typescript
function mergeConfigToProps(
  projectSchema: string | undefined,
  pageSchema: string,
): Record<string, unknown> {
  // 1. 解析两个 Schema
  // 2. 提取各自的 default 值
  // 3. 检查字段冲突
  // 4. 合并返回（项目级优先）

  return { ...projectDefaults, ...pageDefaults };
}
```

**冲突处理**：发现同名字段时抛出 `SchemaConflictError`

### 5.3 配置流向

```
┌─────────────────────┐
│ project.config.schema.json │
│   (项目级配置)         │
└─────────┬───────────┘
          │ mergeConfigToProps
          ▼
┌─────────────────────┐     ┌─────────────────────┐
│ demos/{id}/config.schema.json │
│   (页面级配置)         │ ──→ │ MergedComponentProps │
└─────────────────────┘     │   (合并后的 Props)   │
                            └─────────┬───────────┘
                                      │
                                      ▼
                            ┌─────────────────────┐
                            │ iframe 组件 Props   │
                            └─────────────────────┘
```

## 8. 安全机制

### 8.1 iframe Sandbox

```html
<iframe
  src="..."
  sandbox="allow-scripts allow-same-origin"
/>
```

**权限说明**：
- `allow-scripts`：允许执行 JavaScript
- `allow-same-origin`：允许同源脚本（CDN 资源加载需要）

### 8.2 消息来源校验

```javascript
if (event.source !== window.parent) return;
```

### 8.3 模块版本控制

```javascript
let updateVersion = 0;

const thisVersion = ++updateVersion;

import(moduleUrl)
  .then((module) => {
    if (thisVersion !== updateVersion) {
      return; // 忽略过期的模块加载
    }
    // 处理模块
  });
```
