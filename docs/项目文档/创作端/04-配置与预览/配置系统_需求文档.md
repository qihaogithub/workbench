## 一、模块定位

配置系统是 Demo 可配置化的核心支撑模块，负责将设计师定义的配置规则转换为用户可交互的表单控件。本模块实现"配置驱动 UI"的设计理念，让非技术用户也能轻松定制 Demo 外观和行为。

---

## 二、功能描述

### 2.1 Schema 解析功能

#### 2.1.1 格式识别与解析

**用户视角**：
- 系统应能自动识别并解析从设计工具导出的配置文本
- 解析结果应包含：组件代码和配置定义两部分

**交互效果**：
- 当输入文本格式不正确时，系统应提示具体的错误位置和原因
- 系统应支持自动修复常见的格式问题（如换行符不一致）

#### 2.1.2 格式验证

**用户视角**：
- 用户输入配置文本后，系统应验证其格式正确性
- 验证通过后，系统应提取出可用的代码和配置

**交互效果**：
- 验证失败时，应高亮显示错误位置
- 应提供修复建议

---

### 2.2 表单生成功能

#### 2.2.1 自动表单生成

**用户视角**：
- 系统应根据配置定义自动生成对应的表单控件
- 表单控件应与配置项一一对应

**交互效果**：
- 不同数据类型应使用不同的输入控件：
  - 文本类型：文本输入框
  - 数字类型：数字输入框或滑块
  - 布尔类型：开关或复选框
  - 枚举类型：下拉选择器
  - 颜色类型：颜色选择器
  - 数组类型：列表编辑器

#### 2.2.2 实时预览联动

**用户视角**：
- 用户修改表单中的任何配置项后，预览区应立即反映变更效果

**交互效果**：
- 配置变更应在 500ms 内反映到预览区
- 高频输入场景应支持防抖处理

#### 2.2.3 配置验证

**用户视角**：
- 用户输入的配置值应符合配置定义中的约束条件
- 不符合约束的值应被拒绝或提示

**交互效果**：
- 实时验证用户输入
- 验证失败时，应在对应字段下方显示错误提示
- 整体验证失败时，应阻止提交

---

### 2.3 配置导出功能

#### 2.3.1 格式化输出

**用户视角**：
- 用户完成配置后，系统应能将配置值格式化为标准格式输出
- 输出格式应便于其他系统解析和使用

**交互效果**：
- 输出应为有效的 JSON 格式
- 应保留配置项的原始类型信息

---

### 2.4 组件排序功能

#### 2.4.1 AI 声明可排序子组件

**用户视角**：
- AI 生成 Demo 时，可在 JSON Schema 中声明哪些子组件支持顺序调整
- 未声明的子组件不参与排序，保持固定位置

**交互效果**：
- AI 通过 Schema 中的扩展字段 `$demo.orderable` 声明可排序子组件列表
- 声明后，配置面板自动出现"组件排序"区域

#### 2.4.2 排序控件交互

**用户视角**：
- 用户可通过箭头按钮或拖拽调整子组件的排列顺序
- 操作后预览区实时反映新顺序
- 可一键恢复默认顺序

**交互效果**：
- 排序控件同时支持箭头按钮（上下移动）和拖拽排序两种方式
- 箭头按钮保证无障碍和精确操作，拖拽提供高效的大跨度移动体验
- 每个排序项左侧显示拖拽手柄，右侧显示上移/下移箭头按钮
- 排序结果持久化，切换页面后不丢失

#### 2.4.3 预览区顺序渲染

**用户视角**：
- 预览区中的子组件按照配置面板设定的顺序渲染
- 顺序变更后，预览区立即更新

**交互效果**：
- 排序数据以 `__order` 属性注入组件 props
- 组件代码读取 `__order` 决定子组件渲染顺序
- 排序变更不触发重新编译，仅通过配置更新消息刷新预览

## 三、业务规则与约束

### 3.1 支持的数据类型

| 类型 | 说明 | 对应控件 |
|:-----|:-----|:---------|
| string | 文本字符串 | 文本输入框 |
| number | 数值 | 数字输入框/滑块 |
| integer | 整数 | 数字输入框 |
| boolean | 布尔值 | 开关/复选框 |
| array | 数组 | 列表编辑器 |
| object | 对象 | 嵌套表单 |
| enum | 枚举 | 下拉选择器 |

### 3.2 支持的控件扩展

| 控件 | 说明 | 适用场景 |
|:-----|:-----|:---------|
| 颜色选择器 | 可视化选择颜色 | 配置颜色值 |
| 代码编辑器 | 编辑代码片段 | 配置代码块 |
| 富文本编辑器 | 编辑格式化文本 | 配置说明文字 |
| 图片选择器 | 选择或上传图片 | 配置图片资源 |

### 3.3 验证规则

| 规则 | 说明 |
|:-----|:-----|
| 必填验证 | 标记为 required 的字段必须填写 |
| 类型验证 | 输入值必须符合定义的数据类型 |
| 范围验证 | 数值类型可设置最小/最大值 |
| 长度验证 | 字符串类型可设置最小/最大长度 |
| 格式验证 | 可设置正则表达式验证格式 |
| 枚举验证 | 值必须在预定义的选项列表中 |

### 3.4 默认值规则

| 规则 | 说明 |
|:-----|:-----|
| 显式默认值 | 配置定义中指定的 default 值 |
| 类型默认值 | 未指定 default 时，使用类型默认值 |
| 空值处理 | 允许为空时，默认值可为 null |

### 3.5 排序规则

| 规则 | 说明 |
|:-----|:-----|
| 仅限声明项 | 只有 AI 在 `orderable` 中声明的属性才可排序 |
| 固定项不移动 | 未在 `orderable` 中的子组件保持原始位置 |
| 最低数量 | `orderable` 至少包含 2 项，否则不显示排序功能 |
| 数据完整性 | `__order` 必须包含 `orderable` 中的所有项 |

## 四、配置定义格式规范

### 4.1 基本结构

配置定义采用 JSON Schema 格式，基本结构如下：

```json
{
  "type": "object",
  "properties": {
    "配置项名称": {
      "type": "数据类型",
      "title": "显示名称",
      "description": "配置项说明",
      "default": "默认值"
    }
  },
  "required": ["必填配置项列表"]
}
```

### 4.2 扩展字段

除标准 JSON Schema 字段外，支持以下扩展字段：

| 字段 | 说明 | 用途 |
|:-----|:-----|:-----|
| `ui:widget` | 指定使用的控件类型 | 覆盖默认控件选择 |
| `ui:options` | 控件配置选项 | 自定义控件行为 |
| `ui:preview` | 预览区配置 | 设置预览区尺寸等 |
| `$demo.orderable` | 可排序属性列表 | 声明哪些子组件支持顺序调整 |
