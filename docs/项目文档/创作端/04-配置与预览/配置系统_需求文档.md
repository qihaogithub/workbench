# 配置系统 - 需求文档

> 版本：v1.5
> 创建日期：2026-04-06
> 更新日期：2026-07-06

---

## 一、模块定位

配置系统是 Demo 可配置化的核心支撑模块，负责将已经确认的配置规则转换为用户可交互的表单控件，让非技术用户可以调整被明确声明为配置项的内容、样式和行为。

配置系统不负责定义页面结构，也不要求 AI 生成页面时默认把所有标题、文案、图片、颜色、按钮和布局抽成配置字段。没有明确配置诉求时，页面内容由页面源码或原型页文件承载；只有用户明确要求“添加配置项”“这个要可配置”“设为配置项”或调整已有字段时，系统才需要维护对应配置定义。

## 二、功能描述

### 2.1 Schema 解析功能

#### 2.1.1 格式识别与解析

**用户视角**：

- 系统应能自动识别并解析从设计工具导出的配置文本
- 解析结果应包含：组件代码和配置定义两部分

**交互效果**：

- 当输入文本格式不正确时，系统应提示具体的错误位置和原因
- 系统应支持自动修复常见的格式问题（如换行符不一致）

#### 2.1.2 格式验证

**用户视角**：

- 用户输入配置文本后，系统应验证其格式正确性
- 验证通过后，系统应提取出可用的代码和配置

**交互效果**：

- 验证失败时，应高亮显示错误位置
- 应提供修复建议

---

### 2.2 表单生成功能

#### 2.2.1 自动表单生成

**用户视角**：

- 系统应根据配置定义自动生成对应的表单控件
- 表单控件应与配置项一一对应

**交互效果**：

- 不同数据类型应使用不同的输入控件：
  - 文本类型：文本输入框
  - 数字类型：数字输入框或滑块
  - 布尔类型：开关或复选框
  - 枚举类型：下拉选择器
  - 颜色类型：颜色选择器
  - 数组类型：列表编辑器

#### 2.2.2 实时预览联动

**用户视角**：

- 用户修改表单中的任何配置项后，预览区应立即反映变更效果

**交互效果**：

- 配置变更应在 500ms 内反映到预览区
- 高频输入场景应支持防抖处理

#### 2.2.3 配置验证

**用户视角**：

- 用户输入的配置值应符合配置定义中的约束条件
- 不符合约束的值应被拒绝或提示

**交互效果**：

- 实时验证用户输入
- 验证失败时，应在对应字段下方显示错误提示
- 整体验证失败时，应阻止提交

---

### 2.3 配置导出功能

#### 2.3.1 格式化输出

**用户视角**：

- 用户完成配置后，系统应能将配置值格式化为标准格式输出
- 输出格式应便于其他系统解析和使用

**交互效果**：

- 输出应为有效的 JSON 格式
- 应保留配置项的原始类型信息

---

### 2.4 组件排序功能

#### 2.4.1 声明可排序子组件

**用户视角**：

- 当页面已明确需要开放子组件顺序调整时，配置定义可声明哪些子组件支持排序
- 未声明的子组件不参与排序，保持固定位置
- 支持垂直排序和水平排序两种模式

**交互效果**：

- 系统通过 Schema 中的扩展字段 `$demo.orderable` 声明**垂直排序**子组件列表
- 系统通过 Schema 中的扩展字段 `$demo.orderableHorizontal` 声明**水平排序**子组件列表
- 声明后，配置面板自动出现"组件排序"或"横向排序"区域
- 两种排序模式可同时存在，互不干扰

#### 2.4.2 排序控件交互

**用户视角**：

- 用户可通过箭头按钮或拖拽调整子组件的排列顺序
- 操作后预览区实时反映新顺序
- 可一键恢复默认顺序

**交互效果**：

- 排序控件同时支持箭头按钮（上移/下移或左移/右移）和拖拽排序两种方式
- 箭头按钮保证无障碍和精确操作，拖拽提供高效的大跨度移动体验
- 每个排序项左侧显示拖拽手柄，右侧显示方向箭头按钮
- 排序结果持久化，切换页面后不丢失
- 两种排序模式的 UI 布局不同：
  - **垂直排序**：全宽行列表布局，显示序号徽标，箭头为 ↑↓
  - **水平排序**：行内 Chip 标签布局，项间用 `→` 分隔，箭头为 ←→（hover 时显示）

#### 2.4.3 预览区顺序渲染

**用户视角**：

- 预览区中的子组件按照配置面板设定的顺序渲染
- 顺序变更后，预览区立即更新

**交互效果**：

- 垂直排序数据以 `__order` 属性注入组件 props
- 水平排序数据以 `__orderH` 属性注入组件 props
- 组件代码读取 `__order` / `__orderH` 决定子组件渲染顺序
- 排序变更不触发重新编译，仅通过配置更新消息刷新预览

---

### 2.5 元素定位功能

#### 2.5.1 声明可定位子组件

**用户视角**：

- 当页面已明确需要开放子组件自由定位时，配置定义可声明哪些子组件支持定位
- 声明后，用户可通过可视化画布拖拽或坐标输入来调整子组件位置

**交互效果**：

- 系统通过 Schema 中的扩展字段 `$demo.positionable` 声明可定位子组件列表，格式如下：

```json
{
  "$demo": {
    "positionable": {
      "items": ["logo", "title", "button"],
      "defaults": {
        "logo": { "x": 100, "y": 50 },
        "title": { "x": 200, "y": 100 },
        "button": { "x": 300, "y": 200 }
      },
      "size": { "width": 800, "height": 600 }
    }
  }
}
```

- 声明后，配置面板自动出现"元素定位"区域
- `$demo.positionable.items` 定义可定位的子组件列表（至少 1 项）
- `$demo.positionable.defaults` 定义每个子组件的默认位置（可选，缺省为 { x: 0, y: 0 }）
- `$demo.positionable.size` 定义容器尺寸（可选，缺省使用 `$demo.previewSize` 或 800×600）

#### 2.5.2 定位控件交互

**用户视角**：

- 用户可在缩略画布上拖拽子组件标记来调整位置
- 用户也可通过坐标输入框精确设置 X/Y 值
- 可一键恢复默认位置

**交互效果**：

- 缩略画布等比例缩放容器尺寸，最大 280×200px
- 画布显示网格辅助线（100px 等间距）
- 每个子组件以可拖拽标记块显示在画布中，拖拽时高亮
- 标记块显示组件名称和移动光标
- 坐标输入框限制输入范围（0 到容器尺寸）
- "恢复默认位置"按钮在位置变更后显示

#### 2.5.3 预览区位置渲染

**用户视角**：

- 预览区中的子组件按照配置面板设定的位置渲染
- 位置变更后，预览区立即更新

**交互效果**：

- 定位数据以 `__positions` 属性注入组件 props，格式为 `Record<string, { x: number; y: number }>`
- 组件代码读取 `__positions` 决定子组件的绝对定位坐标
- 定位变更不触发重新编译，仅通过配置更新消息刷新预览

## 三、业务规则与约束

### 3.0 配置项来源边界

| 规则 | 说明 |
| :--- | :--- |
| 明确配置诉求 | 只有用户明确要求添加、删除或修改配置字段时，页面级或项目级配置定义才应变化 |
| 默认空配置 | 新建页面没有明确配置诉求时，配置定义可以保持空属性集合 |
| 页面内容归属 | 标题、文案、图片、颜色、按钮和布局默认归属于页面源码或原型页文件，不因“可编辑”自动变成配置项 |
| 配置化入口 | 用户在属性面板中选择“设为配置项”后，系统才需要追加字段并刷新配置面板 |
| 项目级配置展示 | 项目级共享配置字段只有在当前页面存在可生效绑定关系时，才应出现在该页配置面板；未绑定到当前页的共享字段不计入该页配置数量 |
| 共享配置影响范围 | 共享配置分组应显示“影响 N 个页面”，并允许展开查看受影响页面名称 |
| 分组数量展示 | 配置字段分组标题不展示字段数量徽标；页面级和共享配置的作用范围由分组标题、影响页面列表和页面列表总数表达 |
| 空项目配置 | 项目没有页面时，配置面板应展示明确的空页面状态，而不是提示“没有匹配配置项” |
| 复杂结构边界 | 复杂列表、组件层级、布局结构、动态表达式和跨页面动作不作为普通表单字段承载 |
| 字段冲突 | 页面级配置字段不得与项目级共享配置字段重名 |

### 3.1 支持的数据类型

| 类型    | 说明       | 对应控件        |
| :------ | :--------- | :-------------- |
| string  | 文本字符串 | 文本输入框      |
| number  | 数值       | 数字输入框/滑块 |
| integer | 整数       | 数字输入框      |
| boolean | 布尔值     | 开关/复选框     |
| array   | 数组       | 列表编辑器      |
| object  | 对象       | 嵌套表单        |
| enum    | 枚举       | 下拉选择器      |

### 3.2 支持的控件扩展

表单控件的选择遵循三层映射结构：

```
Layer 1: ui:widget 显式覆盖     → 最优先，用于非标准控件或强制指定
Layer 2: format 语义映射         → 根据 JSON Schema 标准 format 值推断
Layer 3: type 数据类型回退       → 根据基本数据类型推断
```

**设计原则**：`format` 是语义声明（「这个值是什么」），`ui:widget` 是控件覆盖（「我想用不同的控件」）。渲染层负责理解语义声明并给出合理的默认控件；只有需要偏离默认时，才需要 `ui:widget`。

| 控件           | 说明               | 触发方式                                                                 |
| :------------- | :----------------- | :----------------------------------------------------------------------- |
| 颜色选择器     | 可视化选择颜色     | `format: "color"`（Layer 2）                                             |
| **图片选择器** | **选择或上传图片** | **`format: "image"`（Layer 2，推荐）** 或 `ui:widget: "file"`（Layer 1） |
| 多图选择器     | 上传多张图片       | `ui:widget: "imageList"`（Layer 1）或 `type: "array"`（Layer 3）         |
| 富文本编辑器   | 编辑格式化文本     | `ui:widget: "richtext"`（Layer 1，无对应 format 标准值）                 |
| 代码编辑器     | 编辑代码片段       | `ui:widget` 扩展（Layer 1）                                              |

### 3.3 验证规则

| 规则             | 说明                                                                                     |
| :--------------- | :--------------------------------------------------------------------------------------- |
| 必填验证         | 标记为 required 的字段必须填写                                                           |
| 类型验证         | 输入值必须符合定义的数据类型                                                             |
| 范围验证         | 数值类型可设置最小/最大值                                                                |
| 长度验证         | 字符串类型可设置最小/最大长度                                                            |
| 格式验证         | 可设置正则表达式验证格式                                                                 |
| 枚举验证         | 值必须在预定义的选项列表中                                                               |
| **图片尺寸校验** | \*\*图片上传时可配置最小/最大宽高限制，详见 [3.8 图片上传验证规则](#38-图片上传验证规则) |

### 3.4 默认值规则

| 规则       | 说明                              |
| :--------- | :-------------------------------- |
| 显式默认值 | 配置定义中指定的 default 值       |
| 类型默认值 | 未指定 default 时，使用类型默认值 |
| 空值处理   | 允许为空时，默认值可为 null       |

### 3.5 排序规则

| 规则               | 说明                                                           |
| :----------------- | :------------------------------------------------------------- |
| 仅限声明项         | 只有配置定义在 `orderable` / `orderableHorizontal` 中声明的属性才可排序 |
| 固定项不移动       | 未声明的子组件保持原始位置                                     |
| 最低数量           | 至少包含 2 项，否则不显示排序功能                              |
| 数据完整性         | `__order` 必须包含 `orderable` 中的所有项                      |
| 双向独立           | 垂直排序 `__order` 与水平排序 `__orderH` 互不干扰，可同时存在 |
| 默认顺序           | 未修改时使用配置定义中的默认顺序，即 `orderable` 的数组顺序      |

### 3.6 定位规则

| 规则         | 说明                                                                 |
| :----------- | :------------------------------------------------------------------- |
| 仅限声明项   | 只有配置定义在 `positionable.items` 中声明的属性才可定位             |
| 最少一项     | `positionable.items` 至少包含 1 项，否则不显示定位功能               |
| 默认位置     | 未设置默认值时使用 `{ x: 0, y: 0 }`                                 |
| 坐标边界     | X 范围 [0, containerWidth]，Y 范围 [0, containerHeight]              |
| 容器尺寸来源 | 优先级：`positionable.size` → `$demo.previewSize` → 800×600          |
| 数据完整性   | `__positions` 必须包含 `positionable.items` 中的所有项键名           |

### 3.7 备注规则

| 规则              | 说明                                          |
| :---------------- | :-------------------------------------------- |
| 非必填            | 备注为可选功能，无备注不影响配置项正常使用    |
| HTML 存储         | 富文本编辑器输出 HTML 格式，存储时做 XSS 过滤 |
| 长度限制          | 单条备注内容不超过 5000 字符（含 HTML 标签）  |
| 空备注            | 空字符串或纯空白视为无备注，不展示摘要标识    |
| Schema 无关性     | 备注不参与配置验证、预览渲染等流程            |
| 持久化范围        | 备注随 Schema 数据存储，导出时包含            |
| 编辑者/使用者区分 | 创作端可添加/编辑/删除备注；使用端仅可查看    |

### 3.8 图片上传验证规则

#### 3.8.1 验证类型

| 验证项   | 说明                                                                                     |
| :------- | :--------------------------------------------------------------------------------------- |
| 文件大小 | 通过 `ui:options.maxSize` 配置，默认 50MB                                                |
| 图片尺寸 | 通过 `ui:options` 中的 `minWidth`、`minHeight`、`maxWidth`、`maxHeight` 配置（单位：px） |
| 文件类型 | 通过 `ui:options.accept` 配置，默认 `image/*`                                            |

#### 3.8.2 尺寸校验策略

| 规则     | 说明                                                                                  |
| :------- | :------------------------------------------------------------------------------------ |
| 可选校验 | 仅当 Schema 中声明了至少一个尺寸限制（minWidth/minHeight/maxWidth/maxHeight）时才触发 |
| 警告模式 | 尺寸不符合要求时弹出确认对话框，用户可选择「取消上传」或「继续上传」                  |
| 非阻断性 | 尺寸校验不强制阻止上传，仅提醒用户注意                                                |
| 优先级   | 先校验文件大小，再校验图片尺寸；尺寸校验失败不执行上传                                |

#### 3.8.3 交互流程

```
用户选择/拖拽图片
    │
    ▼
1. 文件大小校验
    ├─ 超限 → 显示错误，终止流程
    └─ 通过 ↓
    │
    ▼
2. 图片尺寸校验（仅当声明了尺寸限制）
    ├─ 未声明 → 直接上传
    ├─ 符合 → 直接上传
    └─ 不符合 ↓
    │
    ▼
3. 弹出尺寸警告对话框
    ├─ 显示：具体不合规项（如“宽度小于 100px”）
    ├─ 显示：实际尺寸（如“实际 80x60px”）
    ├─ 「取消上传」→ 终止流程
    └─ 「继续上传」→ 执行上传
```

#### 3.8.4 适用范围

| 控件                       | 支持情况              |
| :------------------------- | :-------------------- |
| FileUploadWidget（单图片） | ✅ 完整支持           |
| ImageListWidget（多图片）  | ✅ 完整支持，逐张校验 |

#### 3.8.5 Schema 配置示例

```json
{
  "type": "string",
  "format": "image",
  "title": "Banner图片",
  "ui:options": {
    "accept": "image/*",
    "maxSize": 5242880,
    "minWidth": 100,
    "minHeight": 100,
    "maxWidth": 2048,
    "maxHeight": 2048
  }
}
```

> **说明**：尺寸校验的完整技术实现参见 [技术/03\_表单生成器.md](./技术/03_表单生成器.md) 的 FileUploadWidget 和 ImageListWidget 章节。

## 四、配置定义格式规范

### 4.1 基本结构

配置定义采用 JSON Schema 格式，基本结构如下：

```json
{
  "type": "object",
  "properties": {
    "配置项名称": {
      "type": "数据类型",
      "title": "显示名称",
      "description": "配置项说明",
      "default": "默认值"
    }
  },
  "required": ["必填配置项列表"]
}
```

### 4.2 扩展字段

除标准 JSON Schema 字段外，支持以下扩展字段：

| 字段                          | 说明               | 用途                                                                 |
| :---------------------------- | :----------------- | :------------------------------------------------------------------- |
| `ui:widget`                   | 指定使用的控件类型 | **覆盖默认控件选择**（Layer 1），用于非标准 format 场景              |
| `ui:options`                  | 控件配置选项       | 自定义控件行为（如 accept、maxSize 等）                              |
| `format`                      | 值语义声明         | **参与默认控件选择**（Layer 2），如 `format: "image"` → 图片上传控件 |
| `$demo.previewSize`           | 预览区尺寸         | 设置预览区宽高、缩放等                                               |
| `$demo.orderable`             | 垂直可排序属性列表 | 声明哪些子组件支持垂直方向顺序调整，通过 `__order` props 注入        |
| `$demo.orderableHorizontal`   | 水平可排序属性列表 | 声明哪些子组件支持水平方向顺序调整，通过 `__orderH` props 注入       |
| `$demo.positionable`          | 可定位属性配置     | 声明哪些子组件支持自由定位，含 items/defaults/size，通过 `__positions` props 注入 |
| `$demo.note`                  | 属性级备注         | 富文本备注内容，存储于各属性下                                       |

> **关于 `format` vs `ui:widget`**：`format` 是 JSON Schema 标准字段，用于声明值的语义（如「这张图片的 URL」）。渲染层根据 `format` 自动选择合适的默认控件。`ui:widget` 是自定义扩展字段，用于覆盖默认选择——只有当需要使用与 format 语义不符的控件时，才需要使用。创建配置字段时，优先使用标准 `format` 字段即可。

> **关于 Schema 边界**：`config.schema.json` 只表达用户确认后的配置字段及其控件语义，不是页面结构 DSL，也不是 AI 生成页面的必填内容模型。`$demo.previewSize`、`$demo.orderable`、`$demo.orderableHorizontal` 和 `$demo.positionable` 属于配置面板与预览运行时的有限扩展，不应继续扩展为页面布局或组件层级协议。
