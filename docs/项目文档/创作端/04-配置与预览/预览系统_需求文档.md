# 预览系统 - 需求文档

> 版本：v2.5
> 创建日期：2026-04-06
> 更新日期：2026-06-27

---

## 一、模块定位

预览系统是 Demo 可视化呈现的核心模块，负责在浏览器端实时渲染 AI 生成的 React 组件，提供所见即所得的预览体验。本模块采用**服务端编译 + iframe 预览执行**架构，确保用户配置的变更能够即时反映在预览区，缩短反馈循环。

---

## 二、功能描述

### 2.1 实时预览功能

#### 2.1.1 组件渲染

**用户视角**：
- 系统应在预览区实时渲染 AI 生成的 React 组件
- 渲染结果应与实际使用效果一致

**交互效果**：
- 预览区应支持 React 组件的完整渲染
- 预览区应支持 TypeScript/JSX 代码的编译执行
- 组件内部的交互行为（如点击、动画）应正常工作
- 预览组件应直接使用项目的样式系统和组件库（shadcn/ui）

#### 2.1.2 配置注入

**用户视角**：
- 用户在配置面板设置的参数应自动注入到 Demo 组件
- 配置变更后，预览应即时更新

**交互效果**：
- 配置值应作为组件 Props 传递
- 配置变更后，预览应在 < 16ms 内更新（一帧内响应）

#### 2.1.3 错误展示

**用户视角**：
- 当代码存在语法错误时，系统应在预览区显示错误信息
- 当组件运行时发生错误时，系统应显示错误边界提示

**交互效果**：
- 错误信息应包含错误类型和位置
- 错误信息应使用用户友好的语言描述
- 应提供错误修复建议（如可能）

---

### 2.2 预览尺寸控制功能

#### 2.2.1 自定义尺寸

**用户视角**：
- 用户可设置预览区的宽度和高度
- 预览区应支持响应式布局

**交互效果**：
- 支持设置固定尺寸（如 800x600 像素）
- 支持设置百分比尺寸（如 100%宽度）
- 支持设置最小/最大尺寸限制

#### 2.2.2 尺寸预设

**用户视角**：
- 系统应提供常用设备尺寸预设
- 用户可快速切换不同设备预览

**交互效果**：
- 预设尺寸应包括：手机、平板、桌面等常见设备
- 切换预设后，预览区应立即调整尺寸

---
### 2.3 预览刷新功能

#### 2.3.1 手动刷新

**用户视角**：
- 用户可手动触发预览刷新
- 刷新后，预览区应重新编译并渲染组件

**交互效果**：
- 提供刷新按钮
- 编译过程中显示加载状态

#### 2.3.2 自动刷新

**用户视角**：
- 代码变更后，预览应自动重新编译并刷新
- 配置变更不触发重新编译，直接通过 props 更新
- 自动刷新应支持缓存，相同代码不重复编译

**交互效果**：
- 代码变更后自动触发编译和刷新
- 编译结果应缓存，提升重复加载速度

---

### 2.4 预览模式切换功能

#### 2.4.1 单页面预览

**用户视角**：
- 系统应提供单页面预览，用于查看、配置和编辑当前选中的 Demo 页面
- 用户在单页面预览中应能使用尺寸预设、实时配置联动、刷新和错误提示
- 用户需要进行可视化批注、局部文本编辑或添加配置项时，应在单页面预览中完成

**交互效果**：
- 单页面预览只聚焦当前页面，避免多页面内容干扰编辑判断
- 切换当前页面后，预览区应立即展示新页面内容
- 单页面预览应保留当前页面的交互能力，如点击、动画和滚动

#### 2.4.2 画布工作台

**用户视角**：
- 系统应提供画布工作台，用于查看项目中的多个 Demo 页面
- 用户可在画布中识别页面名称、选中页面、调整页面位置和尺寸
- 用户可在画布中添加文档说明、文字标注、箭头、画笔线条和图片素材
- 用户可通过直接拖入画布添加本地 Markdown 文档和图片
- 用户可通过工具栏图片按钮从本地选择图片添加到画布
- 用户可在画布中使用键盘粘贴添加图片
- 用户可一键基于当前摆放自动整理画布页面，让多页面布局在不破坏大致分组的前提下更规整
- 用户可从画布中的某个页面回到单页面预览继续细节编辑
- 系统不提供外部网页插入功能

**交互效果**：
- 画布应支持多页面总览、平移、缩放和页面聚焦
- 画布应支持自动排版，保留当前大致行列关系，仅对齐页面、修正过近间距并适应全部页面视图
- 画布应将拖入的本地 Markdown 文件创建为文档节点，将拖入的本地图片创建为图片节点，并尽量放置在用户释放鼠标的位置
- 画布应将工具栏添加的文字、箭头和画笔内容创建为可移动、可缩放的自由标注节点
- 画布文字节点应支持在画布中快速编辑文字内容
- 画布中的文档节点应在画布内只浏览 Markdown 渲染结果，编辑时打开统一 Markdown 编辑弹窗
- 文档编辑弹窗不提供独立标题输入，文档第一行即作为画布节点标题
- 画布中的图片节点应直接展示图片内容，不显示内部头部，不添加额外背景色
- 画布中的图片节点调整尺寸时应始终保持图片原始比例，外边框比例不应被自由拉伸改变
- 画布中的页面应优先使用截图展示；截图不可用时仍应能展示实时预览
- 用户选中画布中的页面后，该页面应以白色描边高亮，并尽快切换为可反映最新内容的真实预览
- 当截图尚未准备好时，画布应优先展示真实预览；当真实预览数量过多影响性能时，才对部分页面展示轻量加载态
- 页面布局调整后应自动保存，下次进入时恢复用户整理过的位置

#### 2.4.3 模式边界

**用户视角**：
- 预览区只保留单页面预览和画布工作台两种模式
- 多页面查看统一使用画布工作台
- 当前页面的深度编辑统一使用单页面预览

**交互效果**：
- 系统不再提供独立的宫格预览入口
- 模式切换不应丢失当前选中页面
- 从画布回到单页面预览时，应以画布中选中的页面作为当前页面

### 2.5 可视化批注与局部编辑

#### 2.5.1 元素点选

**用户视角**：
- 用户可在单页预览中进入批注模式，直接点选页面上的可见元素
- 选中元素后，系统应显示当前元素的标签、文本、样式类名和位置线索
- 用户可将当前选区发送给 AI，让 AI 围绕该局部范围提出修改建议

**交互效果**：
- 鼠标悬停时显示元素外框和简短标签
- 点击元素后保留选中外框
- 选区信息应包含 DOM 路径、元素标签、文本摘要和 className 摘要

#### 2.5.2 页面批注

**用户视角**：
- 用户可对选中的预览元素添加文字批注
- 用户可一次性发送多条未处理批注给 AI
- 发送后，系统应把批注范围和内容作为上下文交给 AI，而不是让 AI 任意扩大修改范围

**交互效果**：
- 批注以浮层输入框完成
- 已添加批注在预览区以标记点展示
- 点击标记点可重新打开批注内容
- 批注为空且没有样式变更时，应自动丢弃

#### 2.5.3 预览内文本编辑

**用户视角**：
- 用户可对无子元素的文本节点进行快速文本修改
- 修改不会立即覆盖项目代码，而是先生成可审阅的修改建议
- 用户接受建议后，系统才将修改写回当前页面代码

**交互效果**：
- 双击可编辑的文本元素后输入新文本
- 系统生成待审修改卡片
- 用户可接受或拒绝该修改
- 当原始文本在代码中不存在或出现多次时，系统应拒绝自动写回并提示用户手动确认

#### 2.5.4 样式调整批注

**用户视角**：
- 用户可在批注浮层中临时调整常见文本、外观、尺寸、间距和布局样式，作为给 AI 的可视化修改说明
- 样式调整应先用于预览表达意图，再随批注发送给 AI 处理

**交互效果**：
- 支持调整文本内容、文本颜色、字体、字号、字重、行高、文字对齐、背景、透明度、圆角、宽高、内外边距、布局方式、对齐和间距
- 取消样式调整时，预览应恢复到调整前状态
- 确认样式调整后，系统应记录变更项、旧值和新值
- 当前版本不要求把样式调整直接转换为 className 写回，样式落地由 AI 根据批注上下文完成

#### 2.5.5 可视化添加配置项

**用户视角**：
- 用户可在单页预览中进入配置化模式，直接点选页面上的可见元素
- 用户可把选中的文本、图片或颜色添加为配置面板中的配置项
- 用户确认后，新配置项应立即出现在配置面板，并能联动当前预览

**交互效果**：
- 鼠标悬停和点击沿用预览选区外框
- 系统应根据选中元素给出可配置内容候选，如文本内容、图片地址、文字颜色、背景颜色或边框颜色
- 用户可编辑配置项显示名称、字段 key 和默认值
- 当系统无法安全定位代码中的唯一元素时，应拒绝自动写回并提示用户改用 AI 或手动方式确认

## 三、业务规则与约束

### 3.1 执行环境约束

| 约束 | 说明 |
|:-----|:-----|
| 依赖白名单 | AI 生成的组件只能使用项目已有依赖（react、lucide-react、clsx 等） |
| 代码来源 | 组件代码由 AI 生成，经过系统提示词约束和白名单校验 |
| 存储限制 | 预览环境无法访问本地存储 |

### 3.2 支持的技术栈

| 技术 | 版本要求 | 说明 |
|:-----|:---------|:-----|
| React | 18.x | 核心渲染框架 |
| TypeScript | 5.x | 类型安全的代码编写 |
| CSS | - | 支持 Tailwind CSS、CSS Modules |
| 组件库 | shadcn/ui | 项目内置组件库，可直接使用 |

### 3.3 性能要求

| 指标 | 要求 | 说明 |
|:-----|:-----|:-----|
| 首次渲染时间 | < 1s | 从加载到首次渲染完成（含编译） |
| 配置联动时间 | < 16ms | 配置变更到预览更新（无需重新编译） |
| 代码编译时间 | < 500ms | 代码变更到编译完成 |
| 错误恢复时间 | < 1s | 错误修复到预览恢复 |

## 四、预览尺寸配置规范

### 4.1 尺寸配置格式

在配置定义中通过扩展字段设置预览尺寸：

```json
{
  "$demo": {
    "previewSize": {
      "width": "100%",
      "height": "600px",
      "minHeight": "400px",
      "maxHeight": "800px"
    }
  }
}
```

### 4.2 预设尺寸列表

| 预设名称 | 宽度 | 高度 | 适用场景 |
|:---------|:-----|:-----|:---------|
| 手机竖屏 | 375px | 667px | 移动端应用 |
| 手机横屏 | 667px | 375px | 移动端游戏 |
| 平板竖屏 | 768px | 1024px | 平板应用 |
| 平板横屏 | 1024px | 768px | 平板应用 |
| 桌面 | 100% | 100% | 桌面应用 |
| 自定义 | 用户定义 | 用户定义 | 特殊需求 |

## 2026-06-27 画布自由标注交互补充

- 画布中的图片、箭头、画笔、文字节点都应显示元素顶部标题，标题用于快速识别自由素材。
- 文字、箭头、画笔、图片工具采用先选工具再落点的编辑体验，不在点击工具按钮时立即插入节点。
- 文字工具在用户点击画布空白目标位置后创建文本节点；箭头工具按用户拖拽的起点和终点创建箭头；画笔工具按拖拽轨迹生成线条；图片工具在选择本地图片后等待用户点击画布目标位置再创建图片节点。
- 自由标注节点仍只保存到画布布局，不进入单页预览批注链路，也不直接写回 Demo 页面代码。
