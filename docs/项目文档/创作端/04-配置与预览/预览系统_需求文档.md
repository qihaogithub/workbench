# 预览系统 - 需求文档

> 版本：v2.0
> 创建日期：2026-04-06
> 更新日期：2026-04-19

---

## 一、模块定位

预览系统是 Demo 可视化呈现的核心模块，负责在浏览器端实时渲染 AI 生成的 React 组件，提供所见即所得的预览体验。本模块采用**服务端编译 + 前端直接渲染**架构，确保用户配置的变更能够即时反映在预览区，缩短反馈循环。

---

## 二、功能描述

### 2.1 实时预览功能

#### 2.1.1 组件渲染

**用户视角**：
- 系统应在预览区实时渲染 AI 生成的 React 组件
- 渲染结果应与实际使用效果一致

**交互效果**：
- 预览区应支持 React 组件的完整渲染
- 预览区应支持 TypeScript/JSX 代码的编译执行
- 组件内部的交互行为（如点击、动画）应正常工作
- 预览组件应直接使用项目的样式系统和组件库（shadcn/ui）

#### 2.1.2 配置注入

**用户视角**：
- 用户在配置面板设置的参数应自动注入到 Demo 组件
- 配置变更后，预览应即时更新

**交互效果**：
- 配置值应作为组件 Props 传递
- 配置变更后，预览应在 < 16ms 内更新（一帧内响应）

#### 2.1.3 错误展示

**用户视角**：
- 当代码存在语法错误时，系统应在预览区显示错误信息
- 当组件运行时发生错误时，系统应显示错误边界提示

**交互效果**：
- 错误信息应包含错误类型和位置
- 错误信息应使用用户友好的语言描述
- 应提供错误修复建议（如可能）

---

### 2.2 预览尺寸控制功能

#### 2.2.1 自定义尺寸

**用户视角**：
- 用户可设置预览区的宽度和高度
- 预览区应支持响应式布局

**交互效果**：
- 支持设置固定尺寸（如 800x600 像素）
- 支持设置百分比尺寸（如 100%宽度）
- 支持设置最小/最大尺寸限制

#### 2.2.2 尺寸预设

**用户视角**：
- 系统应提供常用设备尺寸预设
- 用户可快速切换不同设备预览

**交互效果**：
- 预设尺寸应包括：手机、平板、桌面等常见设备
- 切换预设后，预览区应立即调整尺寸

---
### 2.3 预览刷新功能

#### 2.3.1 手动刷新

**用户视角**：
- 用户可手动触发预览刷新
- 刷新后，预览区应重新编译并渲染组件

**交互效果**：
- 提供刷新按钮
- 编译过程中显示加载状态

#### 2.3.2 自动刷新

**用户视角**：
- 代码变更后，预览应自动重新编译并刷新
- 配置变更不触发重新编译，直接通过 props 更新
- 自动刷新应支持缓存，相同代码不重复编译

**交互效果**：
- 代码变更后自动触发编译和刷新
- 编译结果应缓存，提升重复加载速度

---

### 2.4 多页面宫格预览功能

#### 2.4.1 宫格布局渲染

**用户视角**：
- 系统应将当前页面的所有 Demo 页面以卡片形式在宫格中展示
- 每行可配置显示的卡片数量（2/3/4 列），支持实时切换
- 每张卡片内嵌 iframe 渲染对应 Demo 页面

**交互效果**：
- 宫格中每张卡片的宽度与其预览尺寸的宽高比成正比
  - 宽高比大的卡片（如 iPad 1024×768）占更多宽度
  - 宽高比小的卡片（如手机 375×812）占更少宽度
- 同一行的所有卡片自然等高，不受内容或尺寸差异影响
- 卡片内容通过 iframe 渲染，支持独立缩放

#### 2.4.2 垂直对齐

**用户视角**：
- 当宫格内容较少时，卡片区域在容器中垂直居中显示
- 当宫格内容超出容器高度时，自动切换为顶部对齐并支持滚动

**交互效果**：
- 居中/顶部对齐自动判断，无需用户手动干预
- 切换流畅，无跳动感

#### 2.4.3 卡片选中与状态

**用户视角**：
- 当前活跃页面对应的卡片应有明显的选中标识
- 当页面的配置或代码发生变更但未保存时，对应卡片应显示提示标识

**交互效果**：
- 选中卡片：边框高亮 + 缩放效果（`border-primary ring-2 scale-[1.02]`）
- 变更提示：卡片右上角显示橙色小圆点
- 点击卡片可切换活跃页面

#### 2.4.4 滚动与滚动条

**用户视角**：
- 宫格容器应支持垂直滚动以浏览全部卡片
- 滚动条应隐藏，保持界面简洁

**交互效果**：
- 外层容器滚动条完全隐藏（`scrollbar-width: none`）
- iframe 内部滚动条也隐藏（`overflow: hidden`）
- 鼠标滚轮滚动正常

#### 2.4.5 每页独立预览尺寸

**用户视角**：
- 每个 Demo 页面可拥有独立的预览尺寸（previewSize）
- 活跃页面优先使用用户在配置面板设置的当前尺寸，而非页面自身尺寸

**交互效果**：
- 每页 previewSize 从 `multi.demos[page.id].schema` 中的 `$demo.previewSize` 提取
- 活跃页面使用 grid-level `previewSize`（用户当前选中的尺寸配置）
- 非活跃页面使用各自的 `previewSize`，无设置时回退到 grid-level

#### 2.4.6 懒加载

**用户视角**：
- 只有可见区域内的卡片才加载 iframe 内容
- 滚动时，新进入视口的卡片自动加载

**交互效果**：
- 使用 Intersection Observer 检测卡片可见性
- 不可见的卡片以占位 div 保持布局空间
- 减少首屏加载数量和内存占用

## 三、业务规则与约束

### 3.1 执行环境约束

| 约束 | 说明 |
|:-----|:-----|
| 依赖白名单 | AI 生成的组件只能使用项目已有依赖（react、lucide-react、clsx 等） |
| 代码来源 | 组件代码由 AI 生成，经过系统提示词约束和白名单校验 |
| 存储限制 | 预览环境无法访问本地存储 |

### 3.2 支持的技术栈

| 技术 | 版本要求 | 说明 |
|:-----|:---------|:-----|
| React | 18.x | 核心渲染框架 |
| TypeScript | 5.x | 类型安全的代码编写 |
| CSS | - | 支持 Tailwind CSS、CSS Modules |
| 组件库 | shadcn/ui | 项目内置组件库，可直接使用 |

### 3.3 性能要求

| 指标 | 要求 | 说明 |
|:-----|:-----|:-----|
| 首次渲染时间 | < 1s | 从加载到首次渲染完成（含编译） |
| 配置联动时间 | < 16ms | 配置变更到预览更新（无需重新编译） |
| 代码编译时间 | < 500ms | 代码变更到编译完成 |
| 错误恢复时间 | < 1s | 错误修复到预览恢复 |

## 四、预览尺寸配置规范

### 4.1 尺寸配置格式

在配置定义中通过扩展字段设置预览尺寸：

```json
{
  "$demo": {
    "previewSize": {
      "width": "100%",
      "height": "600px",
      "minHeight": "400px",
      "maxHeight": "800px"
    }
  }
}
```

### 4.2 预设尺寸列表

| 预设名称 | 宽度 | 高度 | 适用场景 |
|:---------|:-----|:-----|:---------|
| 手机竖屏 | 375px | 667px | 移动端应用 |
| 手机横屏 | 667px | 375px | 移动端游戏 |
| 平板竖屏 | 768px | 1024px | 平板应用 |
| 平板横屏 | 1024px | 768px | 平板应用 |
| 桌面 | 100% | 100% | 桌面应用 |
| 自定义 | 用户定义 | 用户定义 | 特殊需求 |
