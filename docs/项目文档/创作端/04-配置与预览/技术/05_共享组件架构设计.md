## 一、架构概述

### 1.1 设计目标

共享组件架构的核心目标是消除创作端（author-site）和使用端（viewer-site）之间的代码重复，实现「单一源码、零行为差异」的组件共享机制。

**三大原则**：
- **单一源码**：共享组件只存在一份代码，两端从同一包导入
- **零行为差异**：两端使用共享组件后，预览面板、配置面板、画布工作台行为完全一致
- **最小侵入**：不改变现有 API 接口，仅迁移文件位置和调整导入路径

### 1.2 包结构设计

采用 `@opencode-workbench/shared` 包内新增 `demo` 子目录的方式存放 React UI 组件，通过 package.json 的 `exports` 字段提供双入口：

```
@opencode-workbench/shared          → 纯类型 + 纯函数（现有入口，agent-service 也可用）
@opencode-workbench/shared/demo     → React UI 组件（仅前端站点使用）
```

**选择此方案的理由**：
1. 减少包管理复杂度，不增加 workspace 成员
2. 共享组件与共享类型天然关联，放在一起便于维护
3. `agent-service` 只引用默认入口，不会意外引入 React 依赖

### 1.3 目录结构

```
packages/shared/src/
├── index.ts              ← 现有入口（纯类型+纯函数），保持不变
├── types.ts              ← 现有，不变

    ├── ConfigForm.tsx    ← 配置面板核心组件
    ├── PreviewPanel.tsx  ← 预览面板（支持双模式）
    ├── PreviewCanvas.tsx ← 画布工作台
    ├── CanvasFreeNodeItem.tsx ← 画布素材节点
    ├── DocumentEditor.tsx ← Markdown/HTML 文档编辑器
    ├── widgets.tsx       ← 单图上传、颜色选择、富文本等控件
    ├── ImageListWidget.tsx ← 多图上传控件
```

## 二、核心组件设计

### 2.1 PreviewPanel 双模式架构

PreviewPanel 是预览区的核心组件，负责渲染 Demo 预览内容。新架构支持两种工作模式：

**编译模式（默认）**：
- 接收 TSX 源码 `code` 属性
- 通过 `/api/compile` 接口实时编译
- 适用于创作端编辑时的即时预览

**URL 模式**：
- 接收预编译 JS 地址 `compiledJsUrl` 属性
- 直接加载预编译资源，跳过编译流程
- 适用于使用端加载已部署的 Demo

```typescript
interface PreviewPanelProps {
  // 编译模式属性
  code?: string;
  sessionId?: string;
  demoId?: string;

  // URL 模式属性
  compiledJsUrl?: string;
  cssImports?: string[];

  // 通用属性
  configData?: Record<string, unknown>;
  previewSize?: PreviewSize;
  snapshotVersion?: number;
}
```

**设计决策**：两种模式共享缩放逻辑、错误展示、ResizeObserver、滚动条隐藏等通用能力，差异仅在于代码加载方式。

### 2.2 ConfigForm 配置面板

ConfigForm 是 Schema 驱动的配置表单生成器，负责渲染配置面板中的各类表单控件。

**控件映射机制**：
- 内置控件：`ColorPickerWidget`、`FileUploadWidget`、`RichTextWidget`
- 复用宿主包的 shadcn/ui 基础组件：`Switch`、`Slider`、`Select` 等

**图片上传控件特殊处理**：
- 使用 `FileUploadWidget` 处理单图上传
- 使用 `ImageListWidget` 处理多图上传
- 图片预览统一采用 120x120 像素的固定尺寸容器

### 2.3 iframe 模板生成

`iframe-template.ts` 负责生成 iframe 的 HTML 内容，包含样式注入、脚本加载、配置数据传递等逻辑。

**核心能力**：
- CSS 导入支持（支持多个 CSS 文件并行加载）
- 配置数据注入（通过全局变量传递给 Demo）
- CDN 资源路径配置
- URL 模式支持（接收预编译 JS 地址）

## 三、类型系统设计

### 3.1 类型合并策略

两端类型定义差异属性全部作为可选属性合并，确保兼容性：

```typescript
// 来自使用端的可选属性
compiledJsUrl?: string;
cssImports?: string[];

// 来自创作端的可选属性
flashCardId?: string;
```

### 3.2 统一导出

通过 `index.ts` 导出所有类型和组件，两端导入方式统一：

```typescript
import { PreviewPanel, ConfigForm } from '@opencode-workbench/shared/demo';
import type { PreviewPanelProps } from '@opencode-workbench/shared/demo';
```

## 四、工具函数设计

### 4.1 验证器函数

提取到 `validator.ts` 的三个工具函数在两端完全一致，无需差异化处理：

| 函数 | 功能 | 使用场景 |
|:-----|:-----|:---------|
| `getOrderable()` | 获取垂直可排序字段列表 | 配置面板垂直排序 |
| `getOrderableHorizontal()` | 获取水平可排序字段列表 | 配置面板水平排序 |
| `getPositionable()` | 获取可定位字段配置（含 items/defaults/size） | 配置面板元素定位 |
| `getDefaultValues()` | 获取配置默认值（含 \_\_order/\_\_orderH/\_\_positions） | 表单初始化 |
| `getPreviewSize()` | 计算预览尺寸 | PreviewPanel 缩放 |

### 4.2 画布工作台组件

共享包提供画布工作台所需的预览组件与公共状态结构，让创作端和使用端在多页面总览中复用同一套页面卡片、视口和渲染兜底规则：

| 组件/能力 | 功能 |
|:----------|:-----|
| `PreviewCanvas` | 承载多页面画布、页面对象、视口和平移缩放交互 |
| `CanvasPageItem` | 负责单个页面对象的截图/iframe 兜底展示 |
| `CanvasFreeNodeItem` | 负责画布中的文档和图片素材节点展示；文档和图片节点不渲染内部头部，节点删除交给画布选中态与键盘删除 |
| `DocumentEditor` | 负责 Markdown 文档编辑；画布文档节点、知识库文档和工作区 Markdown 文件必须复用该组件 |
| 画布布局状态 | 保存页面位置、尺寸和画布视口 |

文档类编辑能力必须收敛到 `DocumentEditor`。后续新增画布文档、知识库 Markdown、计划说明或工作区 Markdown 文件编辑入口时，不应再手写 textarea、双栏预览或独立 Markdown 序列化逻辑；若现有编辑器能力不足，应扩展 `DocumentEditor`，再让各入口复用。

### 4.3 编译缓存

`compile-cache.ts` 提供编译结果缓存能力，避免重复编译相同代码。

## 五、Tailwind CSS 集成

### 5.1 问题背景

共享组件位于 `node_modules` 目录下，默认情况下 Tailwind 无法扫描到其中的类名，导致 `w-[120px]`、`h-[120px]`、`overflow-hidden` 等工具类未生成。

### 5.2 解决方案

在宿主包的 `tailwind.config.ts` 中显式添加共享包的源文件路径：

```typescript
content: [
  // ... 现有路径 ...
  './node_modules/@opencode-workbench/shared/src/**/*.{js,ts,jsx,tsx,mdx}',
],
```

此配置同时添加到 `author-site` 和 `viewer-site`，确保两端样式一致。

## 六、依赖管理

### 6.1 peerDependencies

共享包声明 React 相关依赖为 peerDependencies，要求宿主包提供：

```json
{
  "peerDependencies": {
    "react": ">=18.0.0",
    "react-dom": ">=18.0.0"
  }
}
```

### 6.2 生产依赖

共享包的生产依赖包含编辑器和拖拽排序相关库：

```json
{
  "dependencies": {
    "@dnd-kit/core": "^6.1.0",
    "@dnd-kit/sortable": "^8.0.0",
    "@dnd-kit/utilities": "^3.2.2",
    "@tiptap/extension-placeholder": "^2.1.13",
    "@tiptap/react": "^2.1.13",
    "@tiptap/starter-kit": "^2.1.13",
    "dompurify": "^3.0.8"
  }
}
```

### 6.3 路径别名策略

共享组件内部使用相对路径导入工具函数（如 `cn()`），不依赖宿主包的 `@/` 别名。shadcn/ui 组件仍由宿主包提供，共享组件通过 `@/components/ui/*` 引用。
