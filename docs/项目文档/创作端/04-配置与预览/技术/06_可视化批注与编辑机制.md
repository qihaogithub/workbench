## 一、定位

可视化批注与编辑是单页预览上的辅助编辑层。它不替代代码编辑器，也不绕过 AI 对话，而是在预览 iframe 中捕获用户点选的元素、批注文本、临时样式调整和文本修改，再交给创作端编辑页统一审阅。

当前实现以“可审阅、可回退、范围明确”为优先级：预览内直接发生的变化只作为意图表达或临时预览，真正写回代码时必须经过编辑页的 patch 卡片或 AI 修改链路。

## 二、组件协作

| 层级 | 职责 |
| :--- | :--- |
| 编辑页 | 维护批注模式、选中元素、批注列表和待审 patch；决定是否发送给 AI 或写回当前页面代码 |
| PreviewPanel | 承载 iframe 预览，并在父页面和 iframe 之间转发可视化编辑状态与事件 |
| iframe 模板 | 在预览文档内部绘制悬停框、选中框、批注标记和批注浮层，并采集用户操作 |
| 共享类型 | 定义节点、批注、样式变更、inline edit payload 和 patch 的数据形状 |

## 三、状态流向

### 3.1 父页面下发状态

编辑页把以下状态传给 `PreviewPanel`：

| 状态 | 含义 |
| :--- | :--- |
| `visualEditMode` | 是否启用预览内拦截和元素点选 |
| `visualAnnotationMode` | 是否进入批注输入流程 |
| `selectedVisualNodeId` | 当前选中节点的 DOM 路径或节点 ID |
| `visualAnnotations` | 仍需展示在预览区的批注标记 |

`PreviewPanel` 在 iframe ready 后，或这些状态变化时，向 iframe 发送 `UPDATE_VISUAL_EDIT_STATE`。iframe 收到后重绘悬停层、选中框和批注标记。

### 3.2 iframe 上报事件

iframe 通过 `postMessage` 把预览内交互发回父页面：

| 消息 | 触发场景 | 父页面处理 |
| :--- | :--- | :--- |
| `VISUAL_HOVER` | 鼠标移入或离开可编辑元素 | 更新悬停节点 |
| `VISUAL_SELECT` | 用户点击元素或批注标记 | 更新当前选中节点 |
| `VISUAL_ANNOTATION_CREATE` | 用户提交、更新或删除批注 | 新增、更新或移除批注 |
| `VISUAL_INLINE_EDIT` | 用户双击文本并确认新文本 | 生成文本修改 patch |

## 四、元素识别

iframe 不依赖源码 AST 来定位节点，而是基于实际渲染 DOM 生成运行时信息：

| 字段 | 来源与用途 |
| :--- | :--- |
| `domPath` | 由标签和 `nth-of-type` 组成，用于在本次渲染中重新找到元素 |
| `nodeId` | 优先读取 `data-visual-node-id`，没有时回退到 `domPath` |
| `tagName` | 展示选区标签和生成 AI 上下文 |
| `className` | 展示样式线索，不代表一定能安全写回源码 |
| `textContent` | 生成文本编辑 patch 或批注上下文 |
| `editCapabilities` | 根据 DOM 状态标记可批注、可文本编辑、可样式说明等能力 |

这种定位方式的好处是贴近用户看到的页面；限制是它只能稳定描述当前 iframe 内的渲染结果，不能天然保证映射到源码中的唯一位置。

## 五、批注链路

批注模式打开后，点击元素会弹出浮层。用户可以输入文字，也可以展开样式面板临时调整样式。提交后，iframe 把节点、文字、批注 ID 和样式变更列表上报给编辑页。

编辑页保存未处理批注，并在用户点击“发送批注”时组装一段限定范围的 AI 提示。提示中包含批注数量、每条批注的评论、样式变化、DOM 路径、节点 ID，以及当前页面文件位置。发送后，未处理批注会标记为已处理并从预览标记中移除。

样式面板中的调整属于意图表达：iframe 会即时应用到预览 DOM，取消时恢复，确认时随批注发送。当前实现没有把这些样式变更直接转换成 `className` 或内联样式写回源码。

## 六、文本修改链路

当用户双击一个没有子元素的文本节点时，iframe 会让用户输入新文本，并立即在预览中展示。随后它上报 `VISUAL_INLINE_EDIT`，编辑页据此创建一个 `kind: text` 的待审 patch。

用户接受 patch 时，编辑页使用唯一文本替换策略写回当前页面代码：

| 检查 | 结果 |
| :--- | :--- |
| 原始文本在当前代码中找不到 | 拒绝自动写回，提示可能来自动态数据或已被修改 |
| 原始文本出现多次 | 拒绝自动写回，提示用户跳转代码后手动确认 |
| 原始文本唯一出现 | 替换为新文本，并通过统一快照入口刷新预览 |

写回成功后，编辑页通过 `applyDemoSnapshot` 更新当前代码、重建预览，并把 patch 标记为已接受。拒绝 patch 时只改变 patch 状态，不改代码。

## 七、AI 选区建议

除了批注发送，编辑页还支持把当前选区单独发送给 AI。选区上下文包含元素标签、DOM 路径、className、文本和当前页面文件。提示语明确要求 AI 只围绕当前选区提出局部修改建议，避免在没有说明的情况下扩大影响范围。

## 八、可视化添加配置项

配置化模式复用单页预览的元素点选能力。用户点选元素后，iframe 会把节点文本、图片属性和基础样式信息打包给编辑页；编辑页再生成可配置内容候选，让用户确认字段名称、字段 key 和默认值。

确认后，编辑页把当前页面代码、当前页面 Schema、项目级 Schema、选中节点和目标字段一起发送到配置化接口。接口像一个“谨慎的整理员”：先确认字段没有和项目级配置重名，再尝试把硬编码内容替换成组件 Props，同时把字段追加到页面 Schema。成功后，编辑页通过统一快照入口刷新代码、Schema、配置数据和预览。

当前自动写回只覆盖三类高确定性内容：

| 类型 | 写回方式 |
| :--- | :--- |
| 文本 | 选中文本在当前代码中唯一出现时，替换为对应 Props |
| 图片 | 图片地址在 `src` 属性中唯一出现时，替换为对应 Props |
| 颜色 | 元素可通过唯一文本或图片地址定位时，追加或合并内联颜色样式 |

当文本、图片地址或定位线索出现多次，或目标元素已有复杂样式表达式时，接口会拒绝自动写回。拒绝时不会修改代码或 Schema，用户仍可使用批注、AI 对话或手动代码编辑继续处理。

## 九、画布文本语义节点

画布工作台支持在页面之外放置文档、图片和文本节点。文档、图片和文本都以画布节点保存，和页面卡片共享移动、缩放、层级和布局持久化能力。

文本节点不进入单页预览的 DOM 点选链路，也不自动写回 Demo 页面代码。它的职责是承载画布级短说明、修改意图和用户备注，并作为 Agent 可读写的结构化语义内容。图片继续沿用本地文件读取后的画布图片节点。

Agent 动态上下文会读取工作区 `.canvas-layout.json` 中的文本节点，并输出节点 id、标题、文本、位置、关联页面线索和更新时间。这样 Agent 在理解“画布上的说明文字”时可以直接引用结构化文本，而不需要依赖画布截图或 OCR。

画布布局接口保存前会重新校验每个节点的基础字段、布局尺寸和节点专属内容。非法节点不会写入布局文件，避免坏数据影响下次进入画布。

2026-06-28 起，画布节点数据仍以旧 `nodes` 字段兼容保存，同时同步写入 `layers.annotations.nodes`。当前正式节点只保留文档、图片和文本；箭头和画笔不再作为当前正式工具或节点类型。

画布文字工具不在按钮点击时直接插入带默认内容的节点。用户选择文字工具后，画布显示文本光标；点击空白画布落点后，系统创建空文本语义节点并立即聚焦输入。新文本节点默认透明、无白色背景、无占位文案，后续背景色只由用户显式设置。

Agent 页面整理工具 `arrangeCanvasPages` 只负责页面卡片布局和视口更新。它写回 `.canvas-layout.json` 时必须保留文本、文档、图片节点和既有分层字段，避免页面整理破坏画布语义层。

画笔、箭头、形状和高亮等视觉批注能力只作为未来批注模式评估。批注层可以给 Agent 提供截图视觉参考，但默认不可由 Agent 写入，也不能替代文本语义节点。

## 十、边界与约束

| 约束 | 说明 |
| :--- | :--- |
| 仅单页预览承载 | 当前链路接在单页 `PreviewPanel` 上，画布工作台不是主要入口 |
| DOM 定位不是源码定位 | `domPath` 能找回当前渲染元素，但不能保证源码唯一 |
| 文本写回必须唯一 | 文本 patch 只在源码中唯一匹配时自动写回 |
| 样式直接写回未实现 | 共享类型中已有 `className` 和 `structure` patch 类型，但编辑页当前只接受文本 patch |
| 批注依赖 AI 落地 | 批注和样式变化最终由 AI 或人工代码编辑转成源码修改 |
| 配置化写回范围有限 | 自动配置化只处理文本、图片和颜色；复杂结构、列表、布局和动态表达式不自动写回 |
| 画布文本不写回页面 | 文本节点只保存在画布布局中，不直接修改 Demo 源码 |
| 批注层未来评估 | 画笔、箭头、形状和高亮暂不作为当前正式工具，未来只能以独立批注模式接入 |

## 2026-06-27 画布工具态历史补充

- `CanvasToolbar` 的文字和图片按钮只负责进入对应工具态；除文档编辑弹窗外，不在按钮点击时直接创建节点。
- `CanvasViewport` 在创建工具态下只响应画布空白区域事件：文字和图片使用点击落点。
- `PreviewCanvas` 负责把画布坐标转换为 `text`、`image` 节点，并在创建完成后回到选择工具态。
- 当前正式画布不再提供箭头和画笔工具。

## 2026-06-27 可视化编辑状态归属补充

- 可视化编辑、批注、候选配置项、选中配置候选和配置提交状态统一归属 `useVisualEditState`。编辑页渲染层继续使用这些值时，必须从 Hook 返回值中解构，不能引用已经移入 Hook 内部的旧局部变量。
- 编辑页仍保留部分历史 handler 时，这些 handler 只能消费 Hook 返回的状态和 setter。后续继续收敛时，应优先删除重复 handler，而不是重新创建页面级可视化状态。
