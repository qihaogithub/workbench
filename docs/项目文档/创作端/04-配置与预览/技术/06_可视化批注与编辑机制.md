## 一、定位

可视化批注与编辑是单页预览上的辅助编辑层。它不替代代码编辑器，也不绕过 AI 对话，而是在预览 iframe 中捕获用户点选的元素、批注文本、临时样式调整和文本修改，再交给创作端编辑页统一审阅。

当前实现以“可审阅、可回退、范围明确”为优先级：预览内直接发生的变化只作为意图表达或临时预览，真正写回代码时必须经过编辑页的 patch 卡片或 AI 修改链路。

## 二、组件协作

| 层级 | 职责 |
| :--- | :--- |
| 编辑页 | 维护单页预览配置右栏、属性抽屉开关、左侧图层抽屉开关、选中元素、点击图层、临时属性变更、配置项标记、批注列表和待审 patch；决定是否发送给 AI 或写回当前页面代码 |
| VisualPropertyPanel | 作为右侧覆盖抽屉展示当前选中对象的分组属性、配置项标记和当前图层清空入口，不承载图层列表按钮、对象概览卡片、底部发送区或单独关闭入口 |
| PreviewPanel | 承载 iframe 预览，在父页面和 iframe 之间转发可视化编辑状态、临时属性变更与事件，并负责在预览区右键位置展示当前节点子树菜单 |
| LayerTreeMenu | 统一渲染右键图层菜单和单页左侧图层抽屉，保留 DOM 树父子关系并提供可读节点名称；左侧抽屉入口可按图层展示合并修改数量 |
| iframe 模板 | 在预览文档内部绘制悬停框、选中框、批注标记，并采集点击位置图层、当前节点子树、当前页面图层树和临时属性预览 |
| 共享类型 | 定义节点、批注、属性变更、样式变更、inline edit payload 和 patch 的数据形状 |

## 三、状态流向

### 3.1 父页面下发状态

编辑页把以下状态传给 `PreviewPanel`：

| 状态 | 含义 |
| :--- | :--- |
| `visualEditMode` | 是否启用预览内拦截和元素点选 |
| `visualAnnotationMode` | 是否进入批注输入流程 |
| `visualHoverNodeId` | 右侧属性面板悬停的 DOM 路径或节点 ID，用于预览区同步高亮 |
| `selectedVisualNodeId` | 当前选中节点的 DOM 路径或节点 ID |
| `visualPropertyChanges` | 右侧属性面板产生的临时属性变更，iframe 只用于预览表达 |
| `visualAnnotations` | 仍需展示在预览区的批注标记 |

`PreviewPanel` 在 iframe ready 后，或这些状态变化时，向 iframe 发送 `UPDATE_VISUAL_EDIT_STATE`。iframe 收到后重绘悬停层、选中框和批注标记，并把临时属性变更应用到当前渲染 DOM。临时属性变更不会写回源码或配置 Schema。

### 3.2 iframe 上报事件

iframe 通过 `postMessage` 把预览内交互发回父页面：

| 消息 | 触发场景 | 父页面处理 |
| :--- | :--- | :--- |
| `VISUAL_HOVER` | 鼠标移入或离开可编辑元素 | 更新悬停节点 |
| `VISUAL_SELECT` | 用户点击元素、图层项或批注标记 | 更新当前选中节点；点击预览区时同时记录命中的图层列表 |
| `VISUAL_ANNOTATION_CREATE` | 用户提交、更新或删除批注 | 新增、更新或移除批注 |
| `VISUAL_INLINE_EDIT` | 用户双击文本并确认新文本 | 生成文本修改 patch |

## 四、元素识别

iframe 不依赖源码 AST 来定位节点，而是基于实际渲染 DOM 生成运行时信息：

| 字段 | 来源与用途 |
| :--- | :--- |
| `domPath` | 由标签和 `nth-of-type` 组成，用于在本次渲染中重新找到元素 |
| `nodeId` | 优先读取 `data-visual-node-id`，没有时回退到 `domPath` |
| `tagName` | 展示选区标签和生成 AI 上下文 |
| `className` | 展示样式线索，不代表一定能安全写回源码 |
| `textContent` | 生成文本编辑 patch 或批注上下文 |
| `attrs` | 图片、链接和可访问性属性，供属性面板展示与临时替换 |
| `computedStyle` | 文本、外观、尺寸、间距和布局的当前渲染值，供属性面板作为初始值 |
| `editCapabilities` | 根据 DOM 状态标记可批注、可文本编辑、可图片编辑、可链接编辑、可样式说明等能力 |

这种定位方式的好处是贴近用户看到的页面；限制是它只能稳定描述当前 iframe 内的渲染结果，不能天然保证映射到源码中的唯一位置。

## 五、右侧属性面板链路

单页预览不再要求用户先理解批注模式或配置化模式。配置面板保持在右侧栏常驻展示；属性编辑由单页顶部“选择”按钮打开。打开后，页面进入编辑模式：右侧抽屉覆盖配置栏，左侧图层抽屉覆盖 AI 对话区并展示当前页面 DOM 树。编辑页只有在编辑模式打开时才开启 iframe 点选能力；用户点击预览区后，iframe 使用 `elementsFromPoint` 收集点击位置下从外层到内层的可编辑对象，并把默认选中的最具体对象与完整图层列表一起上报。再次点击顶部“选择”按钮、切换页面、切到文档对象或离开单页预览时，编辑页向 iframe 下发关闭状态，预览恢复原本页面交互。

用户点击 iframe 内空白处时，iframe 上报空选区和空图层列表；用户点击预览页面外层空白区域时，编辑页直接清空当前选中对象、当前点击图层、图层菜单和面板悬停态。两种空白点击都会让属性抽屉回到未选中提示态。

图层列表不常驻占用属性面板正文，属性栏也不放置图层列表按钮。用户点击顶部“选择”按钮进入编辑模式时，编辑页请求 `PreviewPanel` 采集当前页面 DOM 树，并用左侧抽屉中的 `LayerTreeMenu` 展示；图层抽屉不提供自己的关闭按钮，退出编辑模式统一由顶部“选择”按钮控制。用户也可以在预览区右键当前对象，由 iframe 上报当前选中元素子树和右键坐标，`PreviewPanel` 在预览容器内按实际 iframe 缩放换算坐标并打开同一个树形菜单。

两个入口只在数据范围上不同：右键入口展示当前节点及其子级，左侧抽屉入口展示当前页面图层树。菜单保留 DOM 父子关系和 DOM 顺序，不按视觉重要性重排；显示名称会把运行时标签名转换成更贴近日常认知的名称，例如图片、按钮、背景图容器、卡片或弹窗容器，避免把 `div` 这类 DOM 标签直接暴露给非技术用户。左侧抽屉的对象项不显示类型标签；当某个图层存在属性修改或配置项标记时，仅在对象项右侧显示合并后的数字徽标。对象项继续支持悬停高亮和点击精确选择。菜单滚动区域使用项目内细滚动条样式，避免浏览器默认滚动条破坏预览工具栏的视觉密度。

属性编辑区采用 Inspector 信息架构：位置、布局、外观、图片、文本、背景、边框、链接、阴影与模糊按固定顺序排列。前端会把 iframe 采集到的渲染样式拆成更贴近设计工具的字段，例如宽高、排列方式、对齐、元素间距、内边距、裁剪、圆角、透明度、字体、颜色、边框和阴影。数值输入只展示人类可读的数字，单位由控件固定显示；颜色输入统一展示为 Hex，用户输入带 `#` 或不带 `#` 的 Hex 都会被归一化后下发；发送给 iframe 前再归一化成 CSS 可识别的值。面板对用户展示的属性名称、按钮提示和输入占位文案均使用中文，不直接暴露英文 CSS 属性名。

布局分组不使用通用属性行平铺。它由独立的组合控件渲染：排列按钮负责在自由、横向、纵向和网格之间切换；九宫格对齐把用户选择转换成水平和垂直对齐；间距输入控制元素之间的距离；内边距输入按上下、左右成组设置；裁剪开关控制是否隐藏超出容器的内容。这样面板展示的是设计意图，iframe 接收时再转换成可预览的样式变更。

外观分组同样不再使用普通属性行平铺。它默认把不透明度和圆角放在同一行，保持接近设计工具的扫描效率；圆角默认同步四个角，用户需要更精细控制时再展开为四角独立输入。iframe 会采集整体圆角和四角圆角的当前渲染值，面板变更时把用户输入转换成临时样式变更下发预览。

边框分组按“是否存在实际可见边框”决定展开内容。iframe 会采集边框宽度、样式和颜色；当宽度为 0 或边框样式为隐藏/无时，属性面板只展示添加边框入口，不展示颜色和宽度。用户添加边框后，面板会下发实线样式、默认宽度和颜色，再展开可编辑项。

背景分组按“是否存在实际背景色”决定展开内容。iframe 会采集当前渲染背景色；当背景为空、透明或 alpha 为 0 时，属性面板只展示添加背景入口，不展示透明色输入框。用户添加背景后，面板会下发默认 Hex 背景色，再展开颜色编辑项。

阴影与模糊分组采用和边框一致的渐进展示方式。iframe 会采集当前渲染的阴影和滤镜值；当二者都没有实际效果时，属性面板只展示添加阴影入口，不显示 `none` 输入框。用户添加阴影或已有阴影、滤镜效果时，面板再展开阴影和滤镜编辑项。

文本分组只在当前节点具备明确文本编辑能力时展示。iframe 不会把容器内多个子元素的文案拼接成一个可编辑文本；例如一个按钮组容器只作为容器、布局或外观对象处理，内部按钮需要通过图层气泡或直接点击按钮后分别编辑。

属性抽屉将一次点击拆成四类状态：

| 状态 | 用途 |
| :--- | :--- |
| 当前点击图层 | 帮助用户在本次点击命中的容器、图片、文本、按钮等对象之间切换 |
| 选中元素 | 决定当前属性编辑区展示哪些文本、图片、样式、布局和链接控件 |
| 临时属性变更 | 发送给 iframe 即时预览，不改源码、不改 Schema |
| 配置项标记 | 记录用户希望由 AI 同步生成配置字段的属性 |
| 属性提交快照 | 记录已经发送给 AI 的属性变更、配置项标记、补充说明和发送状态 |

临时属性变更通过 `visualPropertyChanges` 下发到 iframe。iframe 每次应用前先恢复上一批临时变更的原始内联状态，再应用当前批次；这样用户清空、恢复单项或切换属性值时，预览可以回到当前渲染结果，而不是把旧临时值残留在 DOM 上。

用户通过左侧图层抽屉底部的“发送给AI”提交修改后，编辑页会把本次发送内容保存为属性提交快照，并继续保留已提交的临时属性变更用于预览表达。后续“是否存在未发送内容”不再直接看临时属性变更数组是否为空，而是比较当前面板状态和提交快照之间的差异：只有新增或修改过的属性、配置项标记和补充说明才算待发送草稿。

左侧图层抽屉底部展示全页面属性修改和配置项标记的合并数量，并承载统一发送入口。属性提交快照按四种状态驱动发送按钮可用性：等待 AI 接收、AI 处理中、已发送、发送失败。等待接收或处理中时，同一批内容不能重复发送；发送失败时保留快照和结构化说明，允许用户重新发送。用户在 AI 处理中继续调整属性时，新差异独立显示为待发送草稿，不会重复发送上一批已提交内容。

图片分组只提供上传替换入口，当前图片地址和图片描述不作为可直接编辑属性展示。用户选择本地图片后，属性面板优先把文件提交到会话资产目录，并把返回的资源 URL 作为临时替换值；如果上传失败或没有会话目录，则退回 data URL 临时预览，并在属性变更资源元数据中标记这是临时资源。AI 正式落地时需要把临时资源保存到项目资源目录，不能直接依赖 data URL。

左侧图层抽屉的“发送给AI”不会直接调用配置化写回接口。编辑页会把页面文件、点击图层、最终选中元素、属性变更、配置项标记和补充说明拼成结构化上下文，交给现有 AI 对话自动发送链路。AI 正式落地时仍需遵守只改相关页面、必要时同步 Schema 和默认值的边界。

当用户只选中元素并填写补充说明时，也可以通过左侧图层抽屉底部发送给 AI；这类请求没有临时属性变更，主要表达“围绕当前选中元素修改”的自然语言意图。

右侧属性抽屉顶部右侧提供“清空”按钮，只清空当前选中图层的属性修改和配置项标记。该动作不会清空其他图层的修改，也不会删除已写回的页面代码或配置 Schema。

> 旧的批注浮层、文本 patch 和自动配置化接口仍保留为兼容链路；新的单页预览主入口优先使用右侧属性面板。

## 六、批注链路

批注模式打开后，点击元素会弹出浮层。用户可以输入文字，也可以展开样式面板临时调整样式。提交后，iframe 把节点、文字、批注 ID 和样式变更列表上报给编辑页。

编辑页保存未处理批注，并在用户点击“发送批注”时组装一段限定范围的 AI 提示。提示中包含批注数量、每条批注的评论、样式变化、DOM 路径、节点 ID，以及当前页面文件位置。发送后，未处理批注会标记为已处理并从预览标记中移除。

样式面板中的调整属于意图表达：iframe 会即时应用到预览 DOM，取消时恢复，确认时随批注发送。当前实现没有把这些样式变更直接转换成 `className` 或内联样式写回源码。

## 七、文本修改链路

当用户双击一个没有子元素的文本节点时，iframe 会让用户输入新文本，并立即在预览中展示。随后它上报 `VISUAL_INLINE_EDIT`，编辑页据此创建一个 `kind: text` 的待审 patch。

用户接受 patch 时，编辑页使用唯一文本替换策略写回当前页面代码：

| 检查 | 结果 |
| :--- | :--- |
| 原始文本在当前代码中找不到 | 拒绝自动写回，提示可能来自动态数据或已被修改 |
| 原始文本出现多次 | 拒绝自动写回，提示用户跳转代码后手动确认 |
| 原始文本唯一出现 | 替换为新文本，并通过统一快照入口刷新预览 |

写回成功后，编辑页通过 `applyDemoSnapshot` 更新当前代码、重建预览，并把 patch 标记为已接受。拒绝 patch 时只改变 patch 状态，不改代码。

## 八、AI 选区建议

除了批注发送，编辑页还支持把当前选区单独发送给 AI。选区上下文包含元素标签、DOM 路径、className、文本和当前页面文件。提示语明确要求 AI 只围绕当前选区提出局部修改建议，避免在没有说明的情况下扩大影响范围。

## 九、可视化添加配置项

配置化模式复用单页预览的元素点选能力。用户点选元素后，iframe 会把节点文本、图片属性和基础样式信息打包给编辑页；编辑页再生成可配置内容候选，让用户确认字段名称、字段 key 和默认值。

确认后，编辑页把当前页面代码、当前页面 Schema、项目级 Schema、选中节点和目标字段一起发送到配置化接口。接口像一个“谨慎的整理员”：先确认字段没有和项目级配置重名，再尝试把硬编码内容替换成组件 Props，同时把字段追加到页面 Schema。成功后，编辑页通过统一快照入口刷新代码、Schema、配置数据和预览。

当前自动写回只覆盖三类高确定性内容：

| 类型 | 写回方式 |
| :--- | :--- |
| 文本 | 选中文本在当前代码中唯一出现时，替换为对应 Props |
| 图片 | 图片地址在 `src` 属性中唯一出现时，替换为对应 Props |
| 颜色 | 元素可通过唯一文本或图片地址定位时，追加或合并内联颜色样式 |

当文本、图片地址或定位线索出现多次，或目标元素已有复杂样式表达式时，接口会拒绝自动写回。拒绝时不会修改代码或 Schema，用户仍可使用批注、AI 对话或手动代码编辑继续处理。

## 十、画布文本语义节点

画布工作台支持在页面之外放置文档、图片和文本节点。文档、图片和文本都以画布节点保存，和页面卡片共享移动、缩放、层级和布局持久化能力。

文本节点不进入单页预览的 DOM 点选链路，也不自动写回 Demo 页面代码。它的职责是承载画布级短说明、修改意图和用户备注，并作为 Agent 可读写的结构化语义内容。图片继续沿用本地文件读取后的画布图片节点。

Agent 动态上下文会读取工作区 `.canvas-layout.json` 中的文本节点，并输出节点 id、标题、文本、位置、关联页面线索和更新时间。这样 Agent 在理解“画布上的说明文字”时可以直接引用结构化文本，而不需要依赖画布截图或 OCR。

画布布局接口保存前会重新校验每个节点的基础字段、布局尺寸和节点专属内容。非法节点不会写入布局文件，避免坏数据影响下次进入画布。

2026-06-28 起，画布节点数据仍以旧 `nodes` 字段兼容保存，同时同步写入 `layers.annotations.nodes`。当前正式节点只保留文档、图片和文本；箭头和画笔不再作为当前正式工具或节点类型。

画布文字工具不在按钮点击时直接插入带默认内容的节点。用户选择文字工具后，画布显示文本光标；点击空白画布落点后，系统创建空文本语义节点并立即聚焦输入。新文本节点默认透明、无白色背景、无占位文案，后续背景色只由用户显式设置。

Agent 页面整理工具 `arrangeCanvasPages` 只负责页面卡片布局和视口更新。它写回 `.canvas-layout.json` 时必须保留文本、文档、图片节点和既有分层字段，避免页面整理破坏画布语义层。

画笔、箭头、形状和高亮等视觉批注能力只作为未来批注模式评估。批注层可以给 Agent 提供截图视觉参考，但默认不可由 Agent 写入，也不能替代文本语义节点。

## 十一、边界与约束

| 约束 | 说明 |
| :--- | :--- |
| 仅单页预览承载 | 当前链路接在单页 `PreviewPanel` 上，画布工作台不是主要入口 |
| DOM 定位不是源码定位 | `domPath` 能找回当前渲染元素，但不能保证源码唯一 |
| 临时属性不写回 | 属性面板的文本、图片和样式修改只在 iframe 内预览，正式落地必须交给 AI 或其他写回链路 |
| 文本写回必须唯一 | 文本 patch 只在源码中唯一匹配时自动写回 |
| 样式直接写回未实现 | 共享类型中已有 `className` 和 `structure` patch 类型，但编辑页当前只接受文本 patch |
| 批注依赖 AI 落地 | 批注和样式变化最终由 AI 或人工代码编辑转成源码修改 |
| 配置化写回范围有限 | 自动配置化只处理文本、图片和颜色；复杂结构、列表、布局和动态表达式不自动写回 |
| 画布文本不写回页面 | 文本节点只保存在画布布局中，不直接修改 Demo 源码 |
| 批注层未来评估 | 画笔、箭头、形状和高亮暂不作为当前正式工具，未来只能以独立批注模式接入 |

## 2026-06-27 画布工具态历史补充

- `CanvasToolbar` 的文字和图片按钮只负责进入对应工具态；除文档编辑弹窗外，不在按钮点击时直接创建节点。
- `CanvasViewport` 在创建工具态下只响应画布空白区域事件：文字和图片使用点击落点。
- `PreviewCanvas` 负责把画布坐标转换为 `text`、`image` 节点，并在创建完成后回到选择工具态。
- 当前正式画布不再提供箭头和画笔工具。

## 2026-06-27 可视化编辑状态归属补充

- 可视化编辑、批注、候选配置项、选中配置候选和配置提交状态统一归属 `useVisualEditState`。编辑页渲染层继续使用这些值时，必须从 Hook 返回值中解构，不能引用已经移入 Hook 内部的旧局部变量。
- 编辑页仍保留部分历史 handler 时，这些 handler 只能消费 Hook 返回的状态和 setter。后续继续收敛时，应优先删除重复 handler，而不是重新创建页面级可视化状态。
