# 协同草稿驱动预览

> 更新日期：2026-06-29

## 定位

配置与预览模块读取的是当前 Workspace 草稿。实时协同编辑引入后，代码和 Schema 的变化可以先进入协同草稿，再同步到预览相关状态。

## 编辑入口

Workspace 文件编辑弹窗在打开页面代码、页面 Schema、项目级 Schema、页面树或画布布局文件时，会尝试进入协同模式。协同模式下，CodeMirror 不再只依赖本地输入状态，而是由协同文档驱动内容变化。

## 预览同步

当前页面代码或 Schema 发生协同变化时，编辑页通过文件保存回调刷新页面代码缓存、Schema 缓存和预览输入。这样另一个协作者保存的草稿可以反映到当前预览，而不需要等待项目版本保存。

页面树协同变化会刷新编辑页的页面和文件夹结构；当前活跃页面的代码与 Schema 仍由对应页面资源房间驱动，避免远端结构变更污染当前页面内容。

画布布局协同变化会写入 `.canvas-layout.json` 草稿并更新当前画布状态。远端画布状态应用时不会标记本地画布脏状态；本地拖拽、缩放、节点调整等画布变更会在已有画布持久化之外同步写入协同草稿，供其他协作者实时恢复。

画布远端状态只能在协同文档文本本身变化时应用。编辑页不能因为本地 `canvasState` 更新就重新应用旧的 `.canvas-layout.json` 内容，否则本地缩放、抓手平移或节点拖拽刚写入状态就会被旧远端草稿覆盖，表现为画布无法缩放或拖动。

## 连接生命周期约束

`useCollabDocument` 的 WebSocket provider 生命周期只由协同房间字段决定：`projectId`、`workspaceId`、`sessionId`、`resourcePath` 和 `kind`。调用方可以传入临时构造的 descriptor 对象，但 hook 内部必须先按这些字段稳定 descriptor，再创建或销毁 Yjs provider。

这个约束用于避免父组件普通渲染触发协同连接重建。若 provider 初始化中的状态回写又导致父组件渲染，而 descriptor 仅因对象引用变化被视为新房间，就会形成“渲染 -> 重建 provider -> 状态回写 -> 再渲染”的循环，最终表现为 React `Maximum update depth exceeded`，并伴随大量 `y-websocket` 连接被关闭或限流日志。

## 页面级 Schema 保护

页面级 Schema 在编辑页内需要同时保存在“当前活跃 Schema”和“按页面 ID 索引的 Schema 缓存”中。切换单页预览、画布选中页面或配置面板详情页前，必须先把当前活跃页面的非空 Schema 回写到页面缓存；随后从页面文件或协同草稿加载到的空 Schema 不能覆盖已知非空 Schema。这样可以避免协同连接初始化时的临时空值把页面级配置清空，导致画布选中页面后只剩项目级共享配置。

## 保存语义

实时协同只代表草稿已写入 Workspace。项目版本、发布状态和使用端可见内容仍由项目保存与发布流程控制，避免每一次实时输入都产生历史版本。

创作端编辑页将“创建版本”放在版本历史区，而不是放在右上角作为保存按钮。用户可以持续编辑并依赖草稿自动保存；需要可回退节点时再手动创建版本。发布前若存在尚未创建版本的草稿，发布按钮会先创建版本再发布。

## 后续扩展

页面配置面板的细粒度字段级协同仍可继续接入同一通道。接入时应保持“草稿实时保存，版本手动生成”的边界不变。
