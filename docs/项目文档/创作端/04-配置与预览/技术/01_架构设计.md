### 1.3 AI 编辑数据流

当 AI 通过对话编辑代码时，数据流与 Figma 导入路径不同：

```
AI 对话（SSE / HTTP）
      │
      ▼
agent-service 执行编辑，写入文件
      │
      ▼
SSE 推送 session.diff 事件（文件变更）
SSE 推送 session.idle 事件（AI 响应完成）
      │
      ▼
┌─────────────────────────────────────────────────┐
│  useChatStream — onFinish                       │
│                                                  │
│  1. 优先使用 result.files（drain 机制保证完整） │
│  2. 兜底 fetchSessionFiles() HTTP 获取          │
│  3. 统一调用 onCodeUpdate / onSchemaUpdate      │
│  4. 触发 onSnapshotReady 回调                   │
└─────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────┐
│  page.tsx — applyDemoSnapshot()                 │
│                                                  │
│  原子化更新以下状态（一次性批量设置）：         │
│  ┌────────────────────────────────────────────┐ │
│  │ • code / schema / editorContent            │ │
│  │ • previewSize                              │ │
│  │ • configDataMap（合并新 schema 默认值）    │ │
│  │ • snapshotVersion++（驱动 PreviewPanel/    │ │
│  │   ConfigForm 强制重渲染）                  │ │
│  │ • compileVersion++（向后兼容）             │ │
│  └────────────────────────────────────────────┘ │
│                                                  │
│  配置合并策略：mergeSchemaDefaults() 纯函数     │
│  - 新增字段 → 使用 schema default               │
│  - 删除字段 → 从 configData 移除                │
│  - 仍存在字段 → 保留用户值                      │
│  - 类型变化 → 使用新 default                    │
└─────────────────────────────────────────────────┘
      │
      ├──────────────────────────────────────────┐
      ▼                                          ▼
┌─────────────────┐                    ┌─────────────────┐
│  PreviewPanel   │                    │   ConfigForm    │
│  snapshotVersion│                    │  key={`${id}-$ │
│  驱动重新编译   │                    │  {snapshotVer  │
│                 │                    │  sion}`}        │
│                 │                    │  强制重新挂载   │
└─────────────────┘                    └─────────────────┘
```

**关键机制**：

- **snapshotVersion 驱动**：`snapshotVersion` 是 UI 更新的单一可信源，在所有相关状态更新后才递增
- **applyDemoSnapshot 原子更新**：AI 编辑产生的 code 和 schema 变更被一次性原子化应用，保证预览与配置强一致性
- **配置合并策略**：`mergeSchemaDefaults(existingConfig, newSchema)` 按字段级规则智能合并，保留用户修改过的值
- **key 强制挂载**：`key={\`${activeDemoId}-${snapshotVersion}\`}` 使 ConfigForm 在快照变更时完全重建
- **drain 机制保障**：后端 `session.idle` 后等待 `session.diff`（最多 2s），确保 finish 时文件快照完整

详细说明参见 [02\_实时预览机制.md](./02_实时预览机制.md) 和 [03\_表单生成器.md](./03_表单生成器.md)。

### 1.4 备注功能数据流

备注功能采用 **Schema 变更驱动** 模式，备注内容存储在属性级 `$demo.note` 字段中，通过 `onSchemaChange` 回调统一持久化：

```
用户点击备注按钮
      │
      ▼
┌─────────────────────────────────────────────────┐
│  NoteDialog 打开（编辑模式 / 只读模式）          │
│                                                  │
│  编辑模式（创作端）：RichTextEditor 富文本编辑    │
│  只读模式（使用端）：纯文本渲染                   │
└─────────────────────────────────────────────────┘
      │
      ▼
用户点击"保存"（编辑模式）
      │
      ▼
┌─────────────────────────────────────────────────┐
│  ConfigForm.updateSchemaNote                    │
│                                                  │
│  1. 解析 schema JSON                            │
│  2. 在对应属性的 $demo 下写入 note 字段         │
│  3. 通过 onSchemaChange 回调传出完整 Schema 字符串│
└─────────────────────────────────────────────────┘
      │
      ▼
page.tsx 接收新 Schema → 持久化到项目数据
```

**备注与表单数据的关系**：

| 维度           | 说明                                                            |
| :------------- | :-------------------------------------------------------------- |
| 存储位置       | 属性级 `$demo.note`，随 Schema 持久化                           |
| 与配置值的关系 | 备注独立于 `formData`，不参与预览渲染                           |
| 变更回调分离   | `onChange` 传递表单数据；`onSchemaChange` 传递 Schema（含备注） |
| 使用端展示     | 使用端读取 Schema 中的 `$demo.note`，以只读方式展示             |

**创作端 vs 使用端的角色差异**：

| 端     | 备注操作           | 实现方式                                   |
| :----- | :----------------- | :----------------------------------------- |
| 创作端 | 添加/编辑/删除备注 | `NoteDialog` + `RichTextEditor`（TipTap）  |
| 使用端 | 仅查看备注         | 独立的 `NoteDialogReadonly`（无需 TipTap） |

详细说明参见 [配置项备注功能需求文档](../../../plans/进行中/配置项备注功能.md)。

## 二、技术选型

### 2.1 表单引擎

采用 **react-jsonschema-form (RJSF)** 作为表单生成引擎：

| 特性             | 说明                     |
| :--------------- | :----------------------- |
| JSON Schema 驱动 | 根据 Schema 自动生成表单 |
| 丰富的控件       | 支持多种数据类型         |
| 自定义控件       | 可扩展 Widget            |
| 实时验证         | 表单填写时校验           |

### 2.2 解析器

自定义解析器处理 Figma 导出格式：

| 功能       | 说明                              |
| :--------- | :-------------------------------- |
| 分隔符识别 | 识别 `=== DEMO CODE ===` 等分隔符 |
| 内容提取   | 提取代码和 Schema 两部分          |
| 格式修复   | 自动修复常见格式问题              |

### 2.3 富文本编辑器（备注功能）

采用 **TipTap**（基于 ProseMirror）作为富文本编辑器内核：

| 特性     | 说明                                                 |
| :------- | :--------------------------------------------------- |
| 生态     | 与 shadcn/ui 风格兼容，社区活跃                      |
| 扩展性   | 支持按需加载扩展，精确控制功能集                     |
| SSR 安全 | `immediatelyRender: false` 避免 Next.js SSR 水合错误 |
| XSS 防护 | 输出 HTML 经 DOMPurify 过滤后再存储                  |
| 输出格式 | 原生支持 HTML 输出                                   |

**引入的 TipTap 扩展**：

| 扩展                            | 用途                           |
| :------------------------------ | :----------------------------- |
| `@tiptap/starter-kit`           | 核心功能（加粗、斜体、列表等） |
| `@tiptap/extension-underline`   | 下划线                         |
| `@tiptap/extension-link`        | 超链接                         |
| `@tiptap/extension-placeholder` | 占位文字                       |

## 三、核心模块

### 3.1 Schema 解析器

**文件位置**：`lib/parser.ts`

**核心函数**：

| 函数                           | 说明               |
| :----------------------------- | :----------------- |
| `parseFigmaText(text)`         | 解析分隔符格式文本 |
| `buildFigmaText(code, schema)` | 构建分隔符格式文本 |
| `isValidFigmaFormat(text)`     | 验证格式有效性     |
| `fixFigmaTextFormat(text)`     | 修复格式问题       |

### 3.2 表单生成器

**文件位置**：`components/demo/ConfigFormNew.tsx`

**核心功能**：

| 功能        | 说明                                            |
| :---------- | :---------------------------------------------- |
| Schema 解析 | 将 JSON 字符串解析为 FieldConfig 数组           |
| 表单生成    | 根据 FieldConfig 生成表单控件                   |
| 自定义控件  | 注册自定义 Widget                               |
| 值变更通知  | 触发 onChange 回调                              |
| 备注管理    | 通过 onSchemaChange 回调管理属性级 `$demo.note` |

### 3.3 配置验证器

**文件位置**：`lib/validator.ts`

**核心函数**：

| 函数                             | 说明               |
| :------------------------------- | :----------------- |
| `validateCode(code)`             | 验证代码有效性     |
| `validateSchema(schema)`         | 验证 Schema 有效性 |
| `validateConfig(schema, config)` | 验证配置值         |

### 3.4 备注功能组件

**文件位置**：`components/demo/` 下新增组件

| 组件                 | 说明                                                   |
| :------------------- | :----------------------------------------------------- |
| `NoteButton.tsx`     | 备注入口按钮，区分有备注（填充）/无备注（outline）状态 |
| `NoteDialog.tsx`     | 备注弹窗，根据 `readonly` 属性切换编辑/只读模式        |
| `RichTextEditor.tsx` | TipTap 富文本编辑器封装（仅创作端使用）                |
| `NotePreview.tsx`    | 备注内容摘要展示，含 `stripHtml` 纯文本提取工具        |

## 四、扩展机制

### 4.1 自定义控件

通过 Widget 机制扩展控件，遵循三层映射结构（Layer 1: ui:widget → Layer 2: format → Layer 3: type）：

| 控件         | 文件位置              | 说明                                                          |
| :----------- | :-------------------- | :------------------------------------------------------------ |
| 颜色选择器   | `widgets.tsx`         | 可视化选择颜色，`format: "color"` 触发                        |
| **图片上传** | `widgets.tsx`         | **单图片上传，`format: "image"` 或 `ui:widget: "file"` 触发** |
| 多图片上传   | `ImageListWidget.tsx` | 多图片上传，`ui:widget: "imageList"` 或 `type: "array"` 触发  |
| 富文本编辑器 | `widgets.tsx`         | 富文本编辑，`ui:widget: "richtext"` 触发                      |
| 代码编辑器   | `widgets.tsx`         | 编辑代码片段                                                  |

详细映射规则参见 [03\_表单生成器.md](./03_表单生成器.md) 第五节「控件映射体系」。

### 4.2 控件注册

```typescript
const widgets = {
  color: ColorWidget,
  code: CodeWidget,
}

<Form
  schema={parsedSchema}
  widgets={widgets}
  onChange={handleChange}
/>
```

### 4.3 富文本编辑器扩展

通过 TipTap 扩展机制为备注功能提供富文本编辑能力：

| 扩展           | 文件位置                                   | 说明                            |
| :------------- | :----------------------------------------- | :------------------------------ |
| RichTextEditor | `RichTextEditor.tsx`                       | TipTap 编辑器封装，含工具栏按钮 |
| NoteDialog     | `NoteDialog.tsx`                           | 编辑/只读模式切换弹窗           |
| XSS 过滤       | `RichTextEditor.tsx` 中 `sanitizeNoteHtml` | DOMPurify 过滤危险 HTML         |
