## 一、架构概述

### 1.1 核心思路

采用**服务端编译 + 前端直接渲染**架构，将 AI 生成的 TSX 代码字符串转换为可执行的 React 组件：

```
┌─────────────────────────────────────────────────────────┐
│  PreviewPanel (前端)                                     │
│  ┌───────────────────────────────────────────────────┐  │
│  │ 1. 调用 /api/compile 编译 TSX → CommonJS          │  │
│  │ 2. 通过 executeComponent 执行编译后代码           │  │
│  │ 3. React.createElement 直接渲染组件               │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│  /api/compile (后端 API Route)                           │
│  ┌───────────────────────────────────────────────────┐  │
│  │ 1. 扫描 import 语句，校验依赖白名单               │  │
│  │ 2. sucrase 编译 TSX → CommonJS                    │  │
│  │ 3. 返回 { compiledCode, dependencies }            │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

### 1.2 文件位置

| 文件 | 说明 |
|:-----|:-----|
| `packages/author-site/src/app/api/compile/route.ts` | 编译 API 路由 |
| `packages/author-site/src/lib/compiler-client.ts` | 编译客户端（封装 API 调用 + 缓存） |
| `packages/author-site/src/lib/compiler.ts` | 编译器核心（sucrase 编译 + 依赖白名单校验） |
| `packages/author-site/src/components/demo/PreviewPanel.tsx` | 预览面板组件 |

## 二、服务端编译

### 2.1 API 路由

**路由**: `POST /api/compile`

**请求体**:
```typescript
{
  code: string  // TSX 代码字符串
}
```

**响应体**:
- 成功：`{ success: true, data: { compiledCode, dependencies } }`
- 失败：`{ success: false, error: { code, message, line?, column? } }`

### 2.2 编译流程

```typescript
import { transform } from 'sucrase';

// 1. 扫描依赖
const dependencies = extractDependencies(code);

// 2. 校验白名单
const allowedDeps = ['react', 'lucide-react', 'clsx', 'tailwind-merge', 
                     'class-variance-authority', 'framer-motion'];
const invalidDeps = dependencies.filter(d => !allowedDeps.includes(d));
if (invalidDeps.length > 0) {
  return createApiError(ErrorCode.INVALID_DEPENDENCY, `未授权的依赖: ${invalidDeps.join(', ')}`);
}

// 3. 编译 TSX → CommonJS
const result = transform(code, {
  transforms: ['typescript', 'jsx'],
  jsxRuntime: 'automatic',
  production: true,
});

return createApiSuccess({
  compiledCode: result.code,
  dependencies,
});
```

### 2.3 编译器选型

| 维度 | sucrase | babel | swc |
|:-----|:--------|:------|:----|
| 编译速度 | ⭐⭐⭐⭐⭐ (~50-100ms) | ⭐⭐ | ⭐⭐⭐⭐⭐ |
| 包体积 | ~50KB | ~500KB | ~3MB |
| TSX 支持 | ✅ | ✅ | ✅ |
| 零原生依赖 | ✅ | ✅ | ❌ (需 wasm) |

**选择 sucrase 的理由**：
- 纯 JavaScript 实现，无需编译原生模块
- 编译速度足够满足实时预览需求
- 包体积小，不影响服务器启动速度

## 三、前端组件执行

### 3.1 执行流程

```typescript
export function executeComponent(compiledCode: string): React.ComponentType {
  // 1. 创建虚拟 module 对象
  const module = { exports: {} };

  // 2. 注入 require 函数（依赖映射）
  const requireFn = (id: string) => {
    if (requireMap[id]) return requireMap[id];
    throw new Error(`未找到依赖: "${id}"`);
  };

  // 3. 执行编译后的 CommonJS 代码
  const executor = new Function('module', 'exports', 'require', compiledCode);
  executor(module, module.exports, requireFn);

  // 4. 获取组件（兼容 export default 和 module.exports）
  const Component = module.exports.default || module.exports;
  
  if (typeof Component !== 'function') {
    throw new Error('编译后的代码未导出有效的 React 组件');
  }

  return Component;
}
```

### 3.2 依赖映射表

```typescript
const requireMap: Record<string, unknown> = {
  'react': React,
  'react/jsx-runtime': require('react/jsx-runtime'),
  'lucide-react': require('lucide-react'),
  'clsx': require('clsx'),
  'tailwind-merge': { twMerge },
  'class-variance-authority': { cva },
  '@/lib/utils': { cn },
  // shadcn/ui 组件（异步预加载）
  '@/components/ui/button': await import('@/components/ui/button'),
  '@/components/ui/card': await import('@/components/ui/card'),
  // ... 其他常用组件
};
```

**路径别名处理**：
- 编译 API 不处理 `@/` 路径
- 由 `component-executor.ts` 的 `require` 函数静态映射
- 新增 shadcn/ui 组件时，只需在 `requireMap` 中增加一行映射

### 3.3 异步预加载

为提升首次加载速度，`component-executor.ts` 在启动时异步预加载常用 shadcn/ui 组件：

```typescript
async function preloadUiComponents() {
  for (const name of uiComponentModules) {
    try {
      const mod = await import(`@/components/ui/${name}`);
      requireMap[`@/components/ui/${name}`] = mod;
    } catch {
      // 组件不存在时静默跳过
    }
  }
}
```

## 四、编译客户端

### 4.1 封装 API 调用

```typescript
export async function compileCode(code: string): Promise<CompileResult> {
  // 1. 检查缓存
  const cacheKey = getCacheKey(code);
  const cached = compileCache.get(cacheKey);
  if (cached) return cached;

  // 2. 调用 API
  const response = await fetch('/api/compile', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ code }),
  });

  const result = await response.json();
  if (!result.success) {
    throw new Error(result.error.message);
  }

  // 3. 写入缓存（FIFO，上限 50 条）
  if (compileCache.size >= MAX_CACHE_SIZE) {
    const firstKey = compileCache.keys().next().value;
    if (firstKey) compileCache.delete(firstKey);
  }
  compileCache.set(cacheKey, result.data);

  return result.data;
}
```

### 4.2 缓存策略

| 维度 | 策略 |
|:-----|:-----|
| 缓存键 | `${code.length}_${code.slice(0, 200)}` |
| 缓存上限 | 50 条 |
| 淘汰策略 | FIFO（先进先出） |
| 清空方法 | `clearCompileCache()` |

## 五、PreviewPanel 组件

### 5.1 状态管理

```typescript
const [compiledComponent, setCompiledComponent] = useState<React.ComponentType | null>(null);
const [compileError, setCompileError] = useState<string | null>(null);
const [isCompiling, setIsCompiling] = useState(false);
```

### 5.2 编译触发

```typescript
useEffect(() => {
  if (!validCode) {
    setCompiledComponent(null);
    setCompileError(null);
    return;
  }

  let cancelled = false;
  setIsCompiling(true);

  compileCode(code)
    .then((result) => {
      if (cancelled) return;
      const Component = executeComponent(result.compiledCode);
      setCompiledComponent(() => Component);
    })
    .catch((err) => {
      if (cancelled) return;
      setCompileError(err.message);
    })
    .finally(() => {
      if (!cancelled) setIsCompiling(false);
    });

  return () => { cancelled = true; };
}, [code, validCode]);
```

**关键设计点**：
- `setCompiledComponent(() => Component)` — 使用函数形式避免 React 将组件当作 updater
- 返回 cleanup 函数设置 `cancelled` 标志，防止竞态条件

### 5.3 渲染逻辑

```tsx
return (
  <div className={className || "h-full w-full"}>
    {!validCode && <InvalidCodeWarning />}
    {isCompiling && <LoadingSpinner />}
    {compileError && !isCompiling && <CompileError error={compileError} />}
    {compiledComponent && !isCompiling && (
      <PreviewErrorBoundary>
        <div style={previewStyle}>
          <React.Suspense fallback={<LoadingSpinner />}>
            {React.createElement(compiledComponent, configData)}
          </React.Suspense>
        </div>
      </PreviewErrorBoundary>
    )}
  </div>
);
```

### 5.4 错误处理

| 错误阶段 | 类型 | 处理方式 |
|:---------|:-----|:---------|
| 编译阶段 | 语法错误 | API 返回错误信息，显示在预览区 |
| 编译阶段 | 无效依赖 | API 返回依赖列表错误 |
| 执行阶段 | 运行时错误 | `PreviewErrorBoundary` 捕获 |
| 渲染阶段 | Props 不匹配 | `try/catch` + fallback |

## 六、性能优化

### 6.1 编译缓存

- 客户端缓存编译结果（`compileCache`）
- 相同代码不重复请求 API
- 缓存上限 50 条，FIFO 淘汰

### 6.2 配置联动优化

配置变更**不触发重新编译**，直接通过 props 传递给已编译组件：

```typescript
// 配置变更时，compiledComponent 不变，仅 configData 更新
React.createElement(compiledComponent, configData)
```

**性能对比**：

| 场景 | Sandpack 方案 | 动态编译方案 |
|:-----|:--------------|:-------------|
| 首次渲染 | 2-5s | < 1s |
| 配置联动 | 1-3s（需重新编译） | < 16ms（直接传 props） |
| 代码变更 | 1-3s | < 500ms（含编译） |

### 6.3 组件预加载

异步预加载常用 shadcn/ui 组件，不阻塞主流程：

```typescript
if (typeof window !== 'undefined') {
  preloadUiComponents();
}
```

## 七、安全说明

### 7.1 new Function 安全性

本系统使用 `new Function` 执行编译后代码，风险可控：

1. **代码来源可控**：AI 生成代码受系统提示词约束
2. **依赖白名单**：仅允许使用项目已有依赖
3. **内网部署**：仅限内部团队使用

### 7.2 后续可选加固

如需更高级别的隔离，可在 iframe 中执行编译后代码，通过 `postMessage` 通信。
