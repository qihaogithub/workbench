## 一、架构概述

### 1.1 核心思路

采用**服务端编译 + iframe CDN 渲染**架构，将 AI 生成的 TSX 代码字符串编译为 ESM 模块，依赖通过 esm.sh CDN 动态加载，在 iframe 沙箱中隔离执行：

```
┌───────────────────────────────────────────────────────────────┐
│  compiler.ts (服务端)                                          │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ 1. sucrase 编译 TSX → ESM (保留 ESM import/export)      │  │
│  │ 2. 提取 import 语句中的 npm 包                          │  │
│  │ 3. rewriteImportsToCdn() 替换为 esm.sh CDN URL          │  │
│  │    - 核心依赖 (react/react-dom) → 固定版本 URL            │  │
│  │    - 第三方库 → CDN URL + ?deps 参数 (共享 React 实例)   │  │
│  │ 4. 返回 { compiledCode, dependencies, cssImports }       │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌───────────────────────────────────────────────────────────────┐
│  iframe 沙箱 (浏览器)                                         │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ 1. Import Map 兜底裸 specifier 解析                      │  │
│  │ 2. 通过 Blob URL + dynamic import 加载编译后 ESM 模块   │  │
│  │ 3. esm.sh CDN 自动下载依赖 (React 实例全局唯一)          │  │
│  │ 4. ErrorBoundary 捕获运行时错误                          │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 架构演进

| 阶段 | 方案 | 说明 |
|:-----|:-----|:-----|
| v1.0 (2026-04) | CommonJS + new Function | 服务端编译为 CommonJS，前端通过 `new Function` 执行并注入 `requireMap` |
| v2.0 (2026-05) | ESM + iframe + CDN | 服务端保留 ESM 格式，依赖通过 esm.sh CDN 加载，iframe 沙箱隔离 |

v2.0 核心改进：
- **iframe 沙箱隔离**：DOM/CSS/JS 完全隔离，消除样式污染和全局变量冲突
- **动态 CDN 依赖**：任意 npm 包自动通过 esm.sh 加载，无需前端预置依赖映射表
- **依赖一致性**：通过 `?deps` 参数 + Import Map 双重保障，确保 iframe 内所有模块共享同一 React 实例

### 1.3 文件位置

| 文件 | 说明 |
|:-----|:-----|
| `packages/author-site/src/lib/compiler.ts` | 编译器核心（sucrase 编译 + CDN URL 替换 + `?deps` 参数） |
| `packages/author-site/src/lib/iframe-template.ts` | iframe HTML 模板（含 Import Map + 错误边界 + postMessage 通信） |
| `packages/author-site/src/lib/cdn-config.ts` | CDN 基础 URL 配置 |

## 二、服务端编译

### 2.1 编译入口

**函数**: `compileCode()` 位于 `packages/author-site/src/lib/compiler.ts`

**输入**:
```typescript
{
  code: string              // TSX 代码字符串
  lockedDependencies?: Record<string, string>  // 已锁定的依赖 URL
}
```

**输出**:
```typescript
{
  compiledCode: string      // 编译后代码（import 已替换为 CDN URL）
  dependencies: string[]    // 提取的依赖列表
  cssImports: string[]      // CSS 导入列表（单独处理）
}
```

### 2.2 编译流程

```typescript
import { transform } from 'sucrase';

// 1. 自动包装：裸 JSX → export default function() { return (...) }
const wrappedCode = autoWrapIfNoDefaultExport(code);

// 2. sucrase 编译 TSX → ESM（保留 ES Module import/export）
const result = transform(wrappedCode, {
  transforms: ['typescript', 'jsx'],
  jsxRuntime: 'automatic',    // 自动添加 react/jsx-runtime 导入
  production: true,
});

// 3. 从编译后的代码提取 import 语句
const dependencies = extractImports(result.code);
// sucrase 会自动注入: import { jsx as _jsx } from 'react/jsx-runtime'

// 4. 分类处理
const cssImports = dependencies.filter(isCssImport);

// 5. 将 npm 包 import 路径替换为 CDN URL
const compiledCode = rewriteImportsToCdn(result.code, dependencies, lockedDependencies);
```

### 2.3 CDN URL 替换规则

`rewriteImportsToCdn()` 函数将裸 specifier 替换为 esm.sh CDN URL，核心逻辑在 `toCdnUrl()`：

```typescript
const CORE_DEPENDENCY_VERSIONS: Record<string, string> = {
  'react': '18.3.1',
  'react-dom': '18.3.1',
};

function toCdnUrl(packageName: string, lockedUrl?: string): string {
  if (lockedUrl) return lockedUrl;                      // 已锁定版本优先

  if (CORE_DEPENDENCY_VERSIONS[packageName]) {           // react / react-dom
    return `${ESM_SH_BASE}/${packageName}@${version}`;   // → 固定版本 URL
  }

  if (packageName.startsWith('react/')) {                 // react/jsx-runtime
    return `${ESM_SH_BASE}/react@18.3.1${subpath}`;      // → 固定版本子路径
  }

  // 第三方库 → CDN URL + ?deps 参数，确保共享 React 实例
  return `${ESM_SH_BASE}/${packageName}?deps=react@18.3.1,react-dom@18.3.1`;
}
```

**?deps 参数原理**：esm.sh 在打包第三方库（如 `lucide-react`）时，默认会将 React 内联进产物。`?deps=react@18.3.1` 指示 esm.sh **不内联 React**，而是产出一个指向 `https://esm.sh/react@18.3.1` 的 import 引用。该 URL 与 iframe 模板中的 React 导入 URL 完全一致 → 同一 URL 始终返回同一 ES Module 实例 → **React 单例**。

### 2.4 编译器选型

| 维度 | sucrase | babel | swc |
|:-----|:--------|:------|:----|
| 编译速度 | ⭐⭐⭐⭐⭐ (~50ms) | ⭐⭐ | ⭐⭐⭐⭐⭐ |
| 包体积 | ~50KB | ~500KB | ~3MB |
| TSX 支持 | ✅ | ✅ | ✅ |
| 零原生依赖 | ✅ | ✅ | ❌ (需 wasm) |

**选择 sucrase 的理由**：纯 JS 实现、编译速度快、包体积小，满足实时预览需求。配置 `jsxRuntime: 'automatic'` 和 `production: true` 以获得最佳输出。

## 三、iframe 沙箱渲染

### 3.1 iframe 模板

编译后的代码通过 iframe 沙箱执行，模板由 `packages/author-site/src/lib/iframe-template.ts` 的 `generateIframeHtml()` 生成。

**模板结构**：

```html
<head>
  <!-- Import Map：兜底安全网，确保任何裸 specifier 解析到统一实例 -->
  <script type="importmap">
  {
    "imports": {
      "react": "${cdnBase}/react@18.3.1",
      "react-dom": "${cdnBase}/react-dom@18.3.1/client",
      "react/jsx-runtime": "${cdnBase}/react@18.3.1/jsx-runtime",
      "react/jsx-dev-runtime": "${cdnBase}/react@18.3.1/jsx-dev-runtime"
    }
  }
  </script>
</head>
<body>
  <div id="root"></div>

  <script type="module">
    import React from '${cdnBase}/react@18.3.1';
    import ReactDOM from '${cdnBase}/react-dom@18.3.1/client';

    // ErrorBoundary 捕获运行时错误 → postMessage 回传父窗口
    // 通过 Blob URL + dynamic import 加载编译后 ESM 模块
    // postMessage 协议：UPDATE_CODE / UPDATE_CONFIG
  </script>
</body>
```

### 3.2 渲染流程

```
父窗口发送 UPDATE_CODE（含编译后代码 + CSS 列表）
        │
        ▼
iframe 更新 <link> CSS 样式
        │
        ▼
Blob URL 创建 → dynamic import() 加载 ESM 模块
        │
        ▼
获取 module.default → currentComponent = module.default
        │
        ▼
ReactDOM.createRoot().render(
  <ErrorBoundary>
    <currentComponent {...configData} />
  </ErrorBoundary>
)
```

### 3.3 双通道 postMessage 协议

| 通道 | 触发时机 | 是否重新编译 | 是否重置组件状态 |
|:-----|:---------|:-------------|:----------------|
| `UPDATE_CODE` | AI 修改代码 | ✅ | ✅（iframe 重新 import） |
| `UPDATE_CONFIG` | 用户修改配置 | ❌ | ❌（仅更新 props） |

- 配置变更不触发重新编译，~16ms 响应
- 代码变更通过 Blob URL + `import()` 加载新模块，利用 ES Module 缓存避免重复下载

### 3.4 错误处理

| 错误阶段 | 捕获方式 | 处理方式 |
|:---------|:---------|:---------|
| 语法错误 | sucrase 编译异常 | API 返回错误行号和列号 |
| CDN 加载失败 | iframe `error` 事件 | postMessage 回传错误详情 |
| React 运行时错误 | ErrorBoundary | 显示友好错误提示，不清空上次成功结果 |
| 模块加载失败 | dynamic import `.catch()` | 回传错误信息 |

### 3.5 依赖一致性保障

iframe 沙箱中可能存在多个 React 实例的问题，采用**分层防御策略**：

| 层级 | 方案 | 作用 |
|:-----|:------|:-----|
| **Layer 1（核心）** | `toCdnUrl()` 追加 `?deps` 参数 | 根因修复：指示 esm.sh 不将 React 内联进第三方库产物 |
| **Layer 2（安全网）** | Import Map | 兜底任何遗留的裸 specifier |
| **Layer 3（预防）** | AI Prompt 补充 React 版本约束 | 减少 AI 使用不兼容库的概率 |

**Layer 1 原理**：
- 第三方库（如 `lucide-react`）通过 `?deps=react@18.3.1` 从 esm.sh 加载时，产物中不会内联 React
- 第三方库产出的 import 指向 `https://esm.sh/react@18.3.1`，与 iframe 模板中的导入 URL 一致
- ES Module 规范：同一 URL 始终返回同一模块实例 → React 单例

## 四、性能与缓存

### 4.1 服务端编译缓存

```typescript
const compileCache = new Map<string, CompileResult>();
const MAX_CACHE_SIZE = 100;
```

- 缓存键：代码内容 MD5 hash + `lockedDependencies` JSON
- 上限 100 条，FIFO 淘汰
- 命中缓存直接返回，跳过 sucrase 编译

### 4.2 CDN 资源缓存

| 资源类型 | 缓存策略 | 说明 |
|:---------|:---------|:-----|
| React / ReactDOM | 浏览器 HTTP 缓存 | esm.sh 设置强缓存，首次后复用 |
| 第三方库 (lucide-react 等) | 浏览器 HTTP 缓存 | 与 React 版本绑定，URL 不变即命中缓存 |
| 编译后代码 | Blob URL 临时引用 | iframe 内通过 `import()` 缓存，重复加载无网络开销 |

### 4.3 延迟预算

| 阶段 | 耗时 | 说明 |
|:-----|:-----|:-----|
| 服务端编译 (sucrase) | ~50ms | 纯 JS，与代码长度正相关 |
| CDN URL 替换 | ~5ms | 正则替换，开销极低 |
| CDN 下载公共依赖 | ~200-500ms | 首次冷启动，后续缓存命中 |
| CDN 下载第三方库 | ~100-300ms/个 | 首次，后续缓存命中 |
| iframe 渲染 | ~50-100ms | 组件复杂度决定 |
| **首次冷启动** | **~1-2s** | 依赖缓存后显著加速 |
| **代码变更 (依赖已缓存)** | **~200ms** | 仅重新加载编译后代码 |

## 五、依赖版本锁定

### 5.1 锁定机制

`ProjectMeta.lockedDependencies` 存储已解析的依赖版本 URL，实现发布后依赖版本固化：

```typescript
{
  "lockedDependencies": {
    "lucide-react": "https://esm.sh/lucide-react@0.344.0?deps=react@18.3.1,react-dom@18.3.1"
  }
}
```

- `compileCode()` 接收 `lockedDependencies` 参数
- `toCdnUrl()` 优先使用锁定 URL，跳过版本解析
- 项目发布时通过 `resolveDependencyVersions()` 批量锁定

### 5.2 锁定流程

```
AI 编辑阶段 → 每次编译使用 esm.sh 最新版本（不锁定）
用户点击发布 → resolveDependencyVersions() 批量 HEAD 请求获取重定向 URL
             → 写入 project.meta.json lockedDependencies
嵌入/查看阶段 → compileCode(code, lockedDependencies) 使用锁定版本
```

## 六、安全说明

### 6.1 iframe 沙箱隔离

| 隔离维度 | 实现方式 |
|:---------|:---------|
| DOM 隔离 | iframe 独立文档树 |
| CSS 隔离 | iframe 独立样式表，父窗口样式不穿透 |
| JS 隔离 | iframe 独立执行上下文，全局变量不共享 |
| 网络隔离 | `sandbox="allow-scripts allow-same-origin"`，禁止导航和弹窗 |

### 6.2 CDN 依赖安全

- 依赖版本通过 `?deps` 参数固定，防止 esm.sh 版本漂移
- 发布后可选择锁定版本 URL，确保历史页面依赖不变
- 内网部署时可通过 `cdn-config.ts` 替换 CDN 基础 URL
