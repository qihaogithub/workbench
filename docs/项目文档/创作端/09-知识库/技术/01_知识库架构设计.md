# 创作端知识库架构设计

## 一、模块边界

创作端知识库模块只负责 UI、API 和现有工作空间数据接入：

- 工作空间用户知识文档 CRUD。
- AI 对话前的项目知识轻量索引。
- 保存模板时调用项目模板流程，让独立知识库服务生成阅读地图。

知识来源优先级、权限过滤、阅读地图结构、索引任务状态和 `knowledgeReport` 报告契约见 [独立知识库服务](../../../独立知识库服务/)。

## 二、数据来源

### 2.1 工作空间用户知识

用户知识保存在项目工作空间：

```text
workingDir/
└── knowledge/
    ├── manifest.json
    ├── 设计规范.md
    └── 业务说明.md
```

`manifest.json` 只管理用户知识文档。历史遗留的系统知识 manifest 条目会在确保目录时被清理。

### 2.2 模板知识

模板知识不由创作端知识 UI 直接维护。保存模板后，`project-core` 或 author-site 旧模板入口会调用独立知识库服务生成阅读地图。

## 三、API 设计

### 3.1 工作空间知识 API

- `GET /api/knowledge?workingDir={workingDir}`：返回用户知识 manifest。
- `POST /api/knowledge?workingDir={workingDir}`：新增用户知识文档。
- `GET /api/knowledge/{docId}?workingDir={workingDir}`：读取单篇用户知识元数据。
- `PUT /api/knowledge/{docId}?workingDir={workingDir}`：更新用户知识文档。
- `DELETE /api/knowledge/{docId}?workingDir={workingDir}`：删除用户知识文档。
- `GET /api/knowledge/content?workingDir={workingDir}&fileName={fileName}`：读取用户知识文档正文。

## 四、前端组件

### 4.1 KnowledgePanel

知识面板嵌入项目编辑页，负责展示文档列表、添加入口和查看入口。列表只展示轻量字段，不加载所有正文。

### 4.2 KnowledgeDocDialog

文档对话框负责查看、编辑和删除用户知识文档。

## 五、AI 对话接入

创作端构建 workspace context 时调用 `scanKnowledgeIndex()`，将用户知识 manifest 格式化为轻量文本。正文不直接进入每轮对话上下文。

AI 需要知识正文时：

```text
readFile / readFileWithLines
  → knowledge/{文件名}
  → 读取工作空间用户知识
```

需要结构化检索报告时，由 Agent Service 的 `knowledgeReport` 工具调用独立知识库服务能力，详见 [Agent 知识报告契约](../../../独立知识库服务/技术/04_Agent知识报告契约.md)。
