# 知识库架构设计

## 一、整体架构

知识库采用"文件系统 + manifest 清单"的存储方式，不依赖额外数据库：

```

工作空间目录 (workingDir)
  └── knowledge/
      ├── manifest.json        ← 元数据清单（版本 + 文档列表）
      ├── 设计规范.md           ← 文档文件（.md 格式）
      ├── 组件说明.md
      └── ...
```

## 二、数据模型

### 2.1 Manifest 结构

```json
{
  "version": 1,
  "items": [
    {
      "id": "kb_1234567890_abc123",
      "title": "设计规范",
      "source": "system | user",
      "description": "组件库设计规范说明",
      "fileName": "设计规范.md",
      "addedAt": "2026-06-01T00:00:00.000Z",
      "updatedAt": "2026-06-01T00:00:00.000Z",
      "sizeBytes": 1024
    }
  ]
}
```

关键字段：
- `id`：由 `kb_` + 时间戳 + 随机字符串组成，保证全局唯一
- `source`：`system`（系统预设，不可修改删除）或 `user`（用户添加）
- `fileName`：依据标题自动生成的安全文件名，仅包含中文、英文、数字、下划线、连字符

### 2.2 文件命名规则

标题到文件名的转换逻辑：

1. 将非中英文、非数字、非下划线/连字符的字符替换为下划线
2. 合并连续下划线为单个
3. 去除首尾下划线
4. 如果文件已存在，自动追加 `_2`, `_3` 序号

## 三、API 设计

### 3.1 获取知识库列表

**GET** `/api/knowledge?workingDir={workingDir}`

从指定工作空间读取 manifest.json，返回文档列表。如文件不存在则返回空列表。

### 3.2 添加文档

**POST** `/api/knowledge?workingDir={workingDir}`

请求体：`{ title, description, content }`

流程：
1. 验证 title 和 content 必填
2. 生成安全文件名（同名自动加序号）
3. 写入 `.md` 文件
4. 读取/初始化 manifest
5. 追加文档条目，设置 `source: "user"`
6. 写回 manifest

### 3.3 更新文档

**PUT** `/api/knowledge/{docId}?workingDir={workingDir}`

请求体：`{ title?, description?, content? }`

流程：
1. 查找 docId 对应的文档条目
2. 校验 `source !== "system"`（系统文档禁止修改）
3. 如有 content 则更新 `.md` 文件内容
4. 更新 manifest 中的元数据和更新时间

### 3.4 删除文档

**DELETE** `/api/knowledge/{docId}?workingDir={workingDir}`

流程：
1. 查找 docId 对应的文档条目
2. 校验 `source !== "system"`（系统文档禁止删除）
3. 删除对应的 `.md` 文件
4. 从 manifest 中移除条目

## 四、前端组件架构

```
编辑页（Demo/Project）
  │
  ├── KnowledgePanel        ← 知识库面板（侧边栏嵌入）
  │     │
  │     ├── 文档列表展示（标题、来源标签、时间）
  │     ├── 添加文档按钮 → 打开 KnowledgeDocDialog（新建模式）
  │     └── 点击文档 → 打开 KnowledgeDocDialog（查看模式）
  │
  └── KnowledgeDocDialog    ← 知识文档对话框
        │
        ├── 查看模式：Markdown 渲染展示
        ├── 编辑模式：Markdown 编辑器（memory-markdown-editor）
        └── 操作：保存/取消/删除
```

### 4.1 KnowledgePanel

展示工作空间知识库的所有文档，支持：
- 以列表形式展示文档标题、来源类型、时间
- 点击文档查看详情
- 添加新文档入口
- 系统文档和用户文档通过徽章区分

### 4.2 KnowledgeDocDialog

统一的文档查看/编辑对话框，支持：
- 查看模式：渲染 Markdown 内容
- 编辑模式：使用 Markdown 编辑器修改标题、描述、内容
- 系统文档仅可查看，用户文档可编辑和删除

## 五、与 AI 对话的集成

知识库在 AI 对话中的角色：

```
用户提问前从知识库选取参考文档
       │
       ▼
文档内容作为 system prompt 的一部分
       │
       ▼
Agent 参考文档内容生成更准确的代码
```

知识库的文档内容在发送 AI 消息时，通过 `systemPrompt` 参数传递给 Agent 服务，作为 AI 的上下文参考。
