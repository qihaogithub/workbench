# 知识库需求文档

## 模块定位

知识库是 AI 可查阅的参考文档管理模块，包含平台内置知识和用户项目知识。平台内置知识由管理员统一维护，用户项目知识由工作空间用户维护。AI 对话可根据当前任务自主查阅相关知识，提升代码生成和配置修改的准确性。

## 功能需求清单

### 1. 知识文档管理

#### 1.1 文档列表查看

- 用户在工作空间内可查看当前知识库的所有文档列表
- 列表展示文档标题、描述、来源类型、添加时间、更新时间、文件大小
- 支持按来源类型（系统预设/用户自定义）区分展示
- 系统内置文档以只读方式出现在工作空间知识库中

#### 1.2 添加文档

- 用户可添加新的知识文档，指定标题、描述和 Markdown 内容
- 添加成功后自动分配文档 ID，记录添加时间和文件大小

#### 1.3 查看文档内容

- 用户可点击文档条目查看其完整 Markdown 内容
- 支持 Markdown 格式渲染展示

#### 1.4 编辑文档

- 用户可修改已有文档的标题、描述和 Markdown 内容
- 修改后自动更新修改时间和文件大小

#### 1.5 删除文档

- 用户可删除不再需要的文档
- 删除操作不可撤销，系统从文件系统和 manifest 中同时移除

### 2. 来源分类

- **系统内置文档**（`source: "system"`）：由管理员维护的全局参考文档，不可在工作空间修改和删除；管理员新增或启用后，已有工作空间应能自动获得
- **用户自定义文档**（`source: "user"`）：用户手动添加的文档，支持修改和删除

### 3. 管理员内置知识库管理

- 管理员可在管理后台新增、编辑、删除、启用或停用系统内置知识文档
- 管理员可维护标题、描述、分类、标签、正文、摘要和关键词
- 管理员保存正文后，系统应尝试更新 AI 摘要索引
- 摘要生成失败不应阻止文档保存
- 停用的系统文档不进入普通工作空间知识库和 AI 索引

### 4. AI 自主查阅

- AI 对话应能看到当前知识库的轻量索引
- 索引应优先使用持久化摘要、关键词、分类和标签，正文不应直接进入每轮对话上下文
- 当用户问题涉及配置系统、设计规范、业务术语、编码约定或用户明确要求遵循知识库时，AI 应自主选择相关文档查阅
- AI 不应一次性读取全部知识库正文，应按当前任务读取最相关的文档
- AI 不可修改或删除知识库文档

### 5. 持久化存储

- 用户项目文档以 `.md` 文件存储在工作空间的 `knowledge/` 目录下
- 用户项目文档元数据通过 `manifest.json` 管理
- 系统内置文档由管理后台全局保存，不复制到每个工作空间
- 文件名基于标题自动生成，同名文件自动追加序号

## 业务规则与约束

### 前置条件

- 必须存在有效的工作空间（`workingDir`）
- 用户项目知识库目录为 `{workingDir}/knowledge/`

### 限制条件

- 系统内置文档（`source: "system"`）不可编辑、不可删除
- 文档标题和内容为必填项
- 文档内容仅支持纯文本 Markdown 格式

## UI/UX 设计要求

- 知识面板（KnowledgePanel）嵌入在 Demo/项目编辑页中
- 知识文档对话框（KnowledgeDocDialog）用于展示文档详情和编辑
- 复用现有的 shadcn/ui 组件风格
