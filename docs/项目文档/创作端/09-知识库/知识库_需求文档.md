# 创作端知识库需求文档

## 模块定位

创作端知识库模块负责提供用户可操作的工作空间知识文档管理界面，以及 AI 对话前的轻量索引接入。

跨端复用的知识来源分层、权限过滤、模板阅读地图、索引任务和知识报告规则，统一归档在 [独立知识库服务](../../独立知识库服务/)。

## 功能需求清单

### 1. 工作空间知识文档管理

#### 1.1 文档列表查看

- 用户在项目编辑页可查看当前工作空间的知识文档列表。
- 列表展示标题、描述、来源、添加时间、更新时间和文件大小。
- 用户文档来自 `workingDir/knowledge/manifest.json`。
- 历史 system 来源条目会在目录同步时被清理，不再展示为内置知识。

#### 1.2 添加文档

- 用户可添加新的 Markdown 知识文档。
- 用户需填写标题、描述和正文。
- 添加成功后系统生成文档 ID、文件名和元数据。

#### 1.3 查看文档

- 用户可查看文档完整 Markdown 内容。
- 文档内容应使用现有 Markdown 渲染能力展示。

#### 1.4 编辑文档

- 用户可编辑自己添加的工作空间知识文档。
- 编辑后更新 `.md` 文件和 manifest 元数据。

#### 1.5 删除文档

- 用户可删除自己添加的工作空间知识文档。
- 删除时同时移除文件和 manifest 条目。

### 2. AI 对话索引接入

- 创作端在构建 workspace context 时读取用户知识 manifest，生成项目知识轻量索引。
- 每轮对话只注入轻量索引，不直接注入全部正文。
- AI 需要正文时通过只读文件工具读取 `knowledge/{文件名}`。
- 用户知识从工作空间文件读取，AI 不得写入或删除知识库文件。
- 结构化知识报告由 [独立知识库服务](../../独立知识库服务/) 和 Agent Service 的 `knowledgeReport` 工具承担。

### 3. 模板保存接入

- 创作端保存项目为模板后，应触发独立知识库服务生成模板阅读地图。
- 创作端不直接维护阅读地图规则，只负责调用模板保存流程。
- 阅读地图的字段和任务状态见 [索引任务与阅读地图设计](../../独立知识库服务/技术/03_索引任务与阅读地图.md)。

## 业务规则与约束

- 工作空间用户知识目录固定为 `{workingDir}/knowledge/`。
- 用户知识文档内容仅支持 Markdown 文本。
- 创作端管理后台不再提供内置知识库配置入口。
- AI 可以读取知识库文档，但不能写入、编辑或删除知识库文件。
- 创作端 UI 不复制独立知识库服务的权限过滤和可信等级规则。

## UI/UX 设计要求

- 知识面板嵌入在项目编辑页中。
- 文档查看和编辑使用统一对话框。
- 复用现有 shadcn/ui、Tailwind 和 lucide-react 组件风格。
