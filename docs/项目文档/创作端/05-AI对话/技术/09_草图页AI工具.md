# 草图页 AI 工具

> 更新日期：2026-07-05
> 适用模块：AI 对话、Agent 工具、草图页对象级编辑恢复准备

## 1. 定位

草图页的 AI 协作实现仍保留在代码中，但创作端当前默认暂停暴露。暂停期内，AI 助手不应知道有草图页能力：工作区扫描会过滤 `sketch-scene` 页面，Agent 默认工具集不注册草图专用工具。

后续恢复后，草图页的 AI 协作仍以结构化 scene 为中心。AI 读取和修改对象树、样式、位置、文本和配置绑定，不把草图页当作 React 文件、HTML 文件或一张截图来改。草图工具遵守 [草图页面运行时](../../02-Demo管理/技术/04_草图页面运行时.md) 的文件协议；写入前统一走 `@workbench/sketch-core` scene 校验和 patch 规则。

## 2. 工具能力

Agent 服务保留以下草图工具实现，但默认不注册到 workbench 工具列表。只有在完成 SDK 验收并明确启用 `PI_AGENT_SKETCH_TOOLS_ENABLED=true` 后，才允许把这些工具重新暴露给创作端 AI：

| 工具 | 作用 |
| :--- | :--- |
| `readSketchScene` | 读取页面 scene，并返回页面尺寸、对象、绑定和元数据摘要 |
| `patchSketchScene` | 按对象级操作更新 scene，适合移动、改样式、改文本、删除或替换对象 |
| `createSketchNodes` | 批量创建文本、图形、图片、便签和组件占位等对象 |
| `bindSketchConfig` | 将对象属性绑定到页面配置字段，并同步 scene 绑定信息 |
| `convertSketchPage` | 发起草图页转原型页或高保真页的转换请求，并返回后续动作 |

重新开放后，这些工具是 AI 修改草图页的首选路径。普通文件写入仍可作为低层能力存在，但不应成为草图页对象编辑的主要方式。

## 3. 协同刷新

暂停期内该协同刷新链路不应被创作端 AI 触发。重新开放后，草图工具写入 `sketch.scene.json`，工具 hook 会把变更映射为页面草图 scene 资源，通知协同层刷新对应房间。

## 4. 诊断与失败

重新开放后，草图工具失败时返回结构化原因，例如页面不存在、目标不是草图页、scene 解析失败、patch 操作无效或配置绑定冲突。AI 应根据诊断修正操作，而不是在失败后改用整文件盲写。草图工具、CLI 和创作端编辑器共享同一份 `sketch-core` reducer，因此对象级变更的校验口径保持一致。
