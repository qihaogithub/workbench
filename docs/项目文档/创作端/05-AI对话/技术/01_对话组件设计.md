## 一、组件关系

```
编辑页（Demo/Project）
  │
  ├── 管理状态：sessionId, workspaceId, code, schema
  │
  └── AIChat（组合层，~190 行）
        │
        ├── Hooks 层
        │     ├── useChatMessages    → 消息/流式/当前消息状态（受控/非受控）
        │     ├── useChatStream      → 流式通信、事件处理、文件操作防抖
        │     └── useChatModels      → 模型列表获取、模型切换
        │
        ├── UI 子组件
        │     ├── ChatMessages       → 消息列表展示
        │     ├── ChatPlan           → Plan 折叠展示
        │     ├── ChatInput          → 输入区域（含模型选择、附件、历史按钮）
        │     └── ModelSelectWithGuard → 模型选择器（含图片附件守卫）
        │
        ├── HistoryDialog           → 历史对话对话框
        └── PermissionDialog        → 权限确认对话框
```

## 二、workspaceId 传递链路

workspaceId 是实现"切换对话不影响工作空间"的关键，完整传递路径：

1. **Session API** 返回 `workspaceId`
2. **编辑页** 接收并存入状态
3. **AIChat** 通过 `workspaceId` 属性接收
4. **HistoryDialog** 通过 `workspaceId` 属性接收
5. **新建对话** → `onNewSession(workspaceId)` → 编辑页复用工作空间
6. **切换对话** → `onSelectSession(sessionId, workspaceId)` → 编辑页仅切换消息

## 三、HistoryDialog 组件

### 3.1 接口

| 属性 | 类型 | 说明 |
|:-----|:-----|:-----|
| open | boolean | 对话框是否打开 |
| onOpenChange | (open: boolean) => void | 打开状态变更回调 |
| projectId | string | 项目 ID |
| workspaceId | string | 当前工作空间 ID |
| currentSessionId | string | 当前对话 ID |
| onSelectSession | (sessionId: string, workspaceId?: string) => void | 选择对话回调 |
| onNewSession | (workspaceId?: string) => void | 新建对话回调 |

### 3.2 行为

- 点击"新建对话"按钮：调用 `onNewSession(workspaceId)`，将当前 workspaceId 传递给编辑页
- 点击某个历史对话：调用 `onSelectSession(sessionId, session.workspaceId)`，传递该对话关联的 workspaceId

## 四、AIChat 组件

### 4.1 关键属性

| 属性 | 类型 | 说明 |
|:-----|:-----|:-----|
| sessionId | string | 当前对话 ID |
| agentSessionId | string | Agent 会话 ID（用于模型流 WebSocket 连接） |
| workingDir | string | 工作目录（可选） |
| projectId | string | 项目 ID |
| demoId | string | Demo ID（可选） |
| workspaceId | string | 当前工作空间 ID |
| onCodeUpdate | (code: string) => void | 代码更新回调 |
| onSchemaUpdate | (schema: string) => void | Schema 更新回调 |
| onFilesChange | (files) => void | 文件变更回调 |
| externalMessages | ChatMessage[] | 外部受控消息列表 |
| externalIsStreaming | boolean | 外部受控流式状态 |
| externalStreamContent | string | 外部受控流式内容 |
| externalCurrentMessage | ChatMessage | 外部受控当前消息 |
| onMessagesChange | (messages) => void | 消息变更回调 |
| onIsStreamingChange | (isStreaming) => void | 流式状态变更回调 |
| onStreamContentChange | (content) => void | 流式内容变更回调 |
| onCurrentMessageChange | (message) => void | 当前消息变更回调 |
| onNewSession | (workspaceId?: string) => void | 新建对话回调 |
| onSelectSession | (sessionId, workspaceId?) => void | 切换对话回调 |
| currentSessionId | string | 当前会话 ID（用于 HistoryDialog 高亮） |
| triggerAutoSend | string | 触发自动发送的消息内容 |
| onTriggerAutoSendHandled | () => void | 自动发送处理完成回调 |
| errorBanner | React.ReactNode | 错误提示横幅 |

### 4.2 透传逻辑

AIChat 将 workspaceId 透传给 HistoryDialog，自身不处理工作空间逻辑：

```
AIChat.workspaceId ──► HistoryDialog.workspaceId
AIChat.onNewSession ──► HistoryDialog.onNewSession
AIChat.onSelectSession ──► HistoryDialog.onSelectSession
```

### 4.3 组合逻辑

AIChat 作为组合层，将三个核心 Hooks 的输出连接到 UI 子组件：

```
useChatMessages ──► messages, setMessages ──► ChatMessages
useChatStream   ──► plan, handleSend, handleCancel ──► ChatPlan, ChatInput
useChatModels   ──► modelState, handleModelChange, handleDepthChange ──► ChatInput
```

**状态流转**：
- `useChatMessages` 管理消息列表、流式状态、流式内容、当前消息四个核心状态
- `useChatStream` 处理流式通信、事件解析、文件操作、权限请求
- `useChatModels` 管理模型列表、模型切换、思考深度选择

**自动发送机制**：
- 当 `triggerAutoSend` 属性有值且当前不在流式输出时，自动触发 `handleSend`
- 发送完成后调用 `onTriggerAutoSendHandled` 回调，用于父组件清理状态

## 五、编辑页行为约束

### 5.1 onNewSession 回调

新建对话时，编辑页**不应重置**代码/Schema/编辑器内容：

- 调用 Session API 时传入 `workspaceId`，复用当前工作空间
- 仅更新 sessionId、workspaceId、agentSessionId
- 仅清空 AI 消息状态
- 不调用 setCode / setSchema / setEditorContent

### 5.2 onSelectSession 回调

切换对话时，编辑页**不应重置**代码/Schema/编辑器内容：

- 仅更新 sessionId、agentSessionId
- 仅加载目标对话的消息历史
- 不调用 setCode / setSchema / setEditorContent
