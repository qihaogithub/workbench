## 一、组件关系

```
编辑页（Demo/Project）
  │
  ├── 管理状态：sessionId, workspaceId, code, schema
  │
  └── AIChat（组合层，~190 行）
        │
        ├── Hooks 层
        │     ├── useChatMessages    → 消息/流式/当前消息状态（受控/非受控）
        │     ├── useChatStream      → 流式通信、事件处理、文件操作防抖
        │     └── useChatModels      → 模型列表获取、模型切换
        │
        ├── UI 子组件
        │     ├── ChatMessages       → 消息列表展示
        │     ├── ChatPlan           → Plan 折叠展示
        │     ├── ChatInput          → 输入区域（含模型选择、附件、历史按钮）
        │     └── ModelSelectWithGuard → 模型选择器（含图片附件守卫）
        │
        ├── HistoryDialog           → 历史对话对话框
        └── PermissionDialog        → 权限确认与计划审批卡片
```

## 二、workspaceId 传递链路

workspaceId 是实现"切换对话不影响工作空间"的关键，完整传递路径：

1. **Session API** 返回 `workspaceId`
2. **编辑页** 接收并存入状态
3. **AIChat** 通过 `workspaceId` 属性接收
4. **HistoryDialog** 通过 `workspaceId` 属性接收
5. **新建对话** → `onNewSession(workspaceId)` → 编辑页复用工作空间
6. **切换对话** → `onSelectSession(sessionId, workspaceId)` → 编辑页仅切换消息

## 三、HistoryDialog 组件

### 3.1 接口

| 属性 | 类型 | 说明 |
|:-----|:-----|:-----|
| open | boolean | 对话框是否打开 |
| onOpenChange | (open: boolean) => void | 打开状态变更回调 |
| projectId | string | 项目 ID |
| workspaceId | string | 当前工作空间 ID |
| currentSessionId | string | 当前对话 ID |
| onSelectSession | (sessionId: string, workspaceId?: string) => void | 选择对话回调 |
| onNewSession | (workspaceId?: string) => void | 新建对话回调 |

### 3.2 行为

- 点击"新建对话"按钮：调用 `onNewSession(workspaceId)`，将当前 workspaceId 传递给编辑页
- 点击某个历史对话：调用 `onSelectSession(sessionId, session.workspaceId)`，传递该对话关联的 workspaceId

## 四、AIChat 组件

### 4.1 关键属性

### 4.2 透传逻辑

AIChat 将 workspaceId 透传给 HistoryDialog，自身不处理工作空间逻辑：

```
AIChat.workspaceId ──► HistoryDialog.workspaceId
AIChat.onNewSession ──► HistoryDialog.onNewSession
AIChat.onSelectSession ──► HistoryDialog.onSelectSession
```

### 4.3 组合逻辑

AIChat 作为组合层，将三个核心 Hooks 的输出连接到 UI 子组件：

```
useChatMessages ──► messages, setMessages ──► ChatMessages
useChatStream   ──► plan, handleSend, handleCancel ──► ChatPlan, ChatInput
useChatModels   ──► modelState, handleModelChange, handleDepthChange ──► ChatInput
```

**状态流转**：
- `useChatMessages` 管理消息列表、流式状态、流式内容、当前消息四个核心状态
- `useChatStream` 处理流式通信、事件解析、文件操作、权限请求
- `useChatModels` 管理模型列表、模型切换、思考深度选择，并在当前编辑页内记住用户最近一次手动选择的完整模型 ID

**自动发送机制**：
- 当 `triggerAutoSend` 属性有值且当前不在流式输出时，自动触发 `handleSend`
- 字符串触发保持原有用户主动操作语义，继续作为普通发送内容进入对话
- 预览失败自动修复使用结构化触发，AIChat 只把隐藏修复提示交给 Agent，同时在消息列表插入系统任务卡片
- 系统任务卡片显示“修复中 / 已修复 / 修复失败”状态，卡片只承载标题、摘要、状态和动作入口；技术详情通过“查看详情”弹窗展示，避免长错误内容撑开对话流
- “查看详情”弹窗展示可滚动的调试详情，只读取 `debugDetail`，不暴露用于重新触发修复的隐藏提示
- 失败后可在卡片动作区重新触发修复
- 系统自动修复不追加用户气泡，也不使用隐藏提示更新会话标题
- 发送完成后调用 `onTriggerAutoSendHandled` 回调，用于父组件清理状态
- 如果触发时已有 AI 运行未结束，系统自动任务和用户消息使用同一前端队列等待发送；系统任务排队时保留任务卡标题、摘要和隐藏提示，出队后复用同一张任务卡更新“修复中 / 已修复 / 修复失败”状态，避免同一 Agent 会话并发收到多条消息。
- 系统自动任务以任务标题和隐藏提示生成去重键；同一个系统任务正在运行或已经排队时，重复触发会被忽略，避免结束后立即启动重复任务导致输入按钮长期停留在“停止”状态。

### 4.4 回复期间本地消息队列

AI 回复期间的用户输入队列由 `useChatStream` 在前端维护。后端 Agent 仍保持同一会话同时只处理一条消息，前端不会在上一轮未收尾时发送新的 WebSocket message。

数据流转：

1. `ChatInput` 在 streaming 状态下仍允许输入、粘贴图片和提交。
2. `PromptInputSubmit` 根据输入内容决定按钮语义：有文本或附件时提交消息；没有内容且正在 streaming 时停止当前回复。
3. `useChatStream.handleSend` 是消息提交入口。当前没有活动运行时直接启动 Agent；已有活动运行时把消息内容、图片、系统任务展示信息和展示消息 ID 放入本地队列。用户消息追加带“等待发送”状态的用户气泡；系统自动任务追加带“等待发送”状态的任务卡。
4. 当前运行在成功、错误、连接失败、HTTP 兜底或用户停止后，统一释放运行锁并尝试取出下一条队列消息。
5. 队列消息真正开始发送时，前端清除该消息上的等待标记，并复用同一条用户消息，不重复追加用户消息。

展示规则：

- `ChatMessages` 只渲染普通历史、系统任务卡和当前流式回复，不渲染仍在等待发送的队列消息。
- `AIChat` 从消息列表中抽取带队列状态的消息，在输入框上方渲染吸附式队列栏，右对齐展示用户消息内容或系统任务标题、“等待发送”状态和取消入口。
- 队列栏限制最大高度并可内部滚动，避免多条排队消息挤占主对话区域。
- 取消尚未发送的队列消息时，前端同时移除本地队列项和对应展示项。

边界约束：

- 系统自动任务必须复用同一队列锁，不能在已有活动运行时绕过队列直接发送 WebSocket message；后端 Agent 会话仍保持同一时间只处理一条消息。
- 系统自动任务只对相同标题和隐藏提示去重；标题或隐藏提示不同的任务仍可进入队列，确保真正的新修复请求不会被误丢弃。
- 需求确认、权限确认和计划审批继续走各自的专用响应通道，普通排队消息不会替代这些响应。
- 会话切换或组件卸载时会清空本地队列，避免旧会话等待消息发送到新会话。

## 五、PermissionDialog 计划审批态

当后端发送的权限请求标记为计划审批时，`PermissionDialog` 不展示普通权限弹窗，而是在对话流中展示执行计划待确认卡片。

交互规则：

- 卡片展示计划待确认状态，并同时提供“查看计划”和“批准”两个入口。
- 用户点击“批准”时，前端直接把当前计划内容作为批准结果返回，Agent 按该版本继续执行。
- 用户点击“查看计划”时，打开可编辑计划弹窗；弹窗内批准会返回编辑后的计划内容。
- 用户取消计划审批时，前端返回拒绝结果，Agent 应停止当前任务。

该组件只负责展示与响应用户选择；是否需要发起计划审批，由 [03_AI行为约束机制.md](./03_AI行为约束机制.md) 中的 Agent 行为规则决定。

## 六、编辑页行为约束

### 6.1 onNewSession 回调

新建对话时，编辑页**不应重置**代码/Schema/编辑器内容：

- 调用 Session API 时传入 `workspaceId`，复用当前工作空间
- 仅更新 sessionId、workspaceId、agentSessionId
- 仅清空 AI 消息状态
- 不调用 setCode / setSchema / setEditorContent

模型选择不由 `onNewSession` 写入 Session API 请求体。新建对话会得到新的 agent session，而后端新 session 会先返回自己的默认模型；`useChatModels` 在收到新 session 的模型列表后，用当前编辑页内保存的用户模型偏好重新发送一次模型切换。这样用户看到的新对话仍沿用手动选择的模型和思考深度，同时不把这个偏好持久化成项目或用户级配置。

如果新 session 的可用模型列表中找不到该偏好模型，前端不强行切换，保留后端返回的默认模型。该偏好只保存在当前编辑页运行态中，不写入项目、用户配置或历史消息；页面刷新后仍以会话和后端模型配置为准。

### 6.2 onSelectSession 回调

切换对话时，编辑页**不应重置**代码/Schema/编辑器内容：

- 仅更新 sessionId、agentSessionId
- 仅加载目标对话的消息历史
- 不调用 setCode / setSchema / setEditorContent
