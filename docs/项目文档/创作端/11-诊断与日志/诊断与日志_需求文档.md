# 诊断与日志 - 需求文档

> 版本：v1.0
> 更新日期：2026-07-02

---

## 一、模块定位

诊断与日志模块为创作端提供开发排查和 AI 代理复盘能力。它把项目打开、页面切换、协同编辑、自动保存、AI 对话、预览错误和发布等关键边界串成可查询时间线，让问题排查不再依赖人工翻多个目录和零散日志。

该模块面向开发者、运维人员和 Codex 类编码代理，不面向普通创作用户展示常驻入口。

---

## 二、功能描述

### 2.1 关键操作时间线

系统应能记录用户在创作端关键流程中的状态变化，包括：

- 打开项目、创建或复用编辑会话。
- 绑定或切换项目工作区。
- 切换页面和资源。
- 协同连接、快照应用、外部文件变更和断线。
- 自动保存、退出前保存和持久化结果。
- AI 消息发送、工具调用、文件变更和运行结束。
- 预览编译、运行时错误和自动修复触发。
- 发布前保存、检查点和发布结果。

### 2.2 统一排查入口

系统应提供稳定查询入口，让 Codex 和开发者可以按项目、编辑页会话、trace、operation、workspace 或事件类型快速查看时间线。

查询结果必须说明数据来源是否完整，不能把“日志缺失”误判为“没有发生问题”。

### 2.3 诊断包导出

系统应支持导出诊断包，用于复现和异步排查。诊断包应包含相关事件、浏览器本地缓冲、原始诊断片段、AI run log 索引、工作区元数据和资源摘要。

诊断包不替代项目数据备份，也不能包含完整源码、完整 AI prompt、完整模型回复、密钥、token、cookie 或上传文件正文。

### 2.4 隐藏式用户入口

普通用户不需要看到诊断面板。排查时可通过显式诊断模式或快捷键导出当前编辑页诊断信息。

如果后端写入失败，前端仍应保留最近诊断事件，并在导出包中提示可能存在缺口。

---

## 三、业务规则与约束

| 规则 | 说明 |
|:-----|:-----|
| 不影响主流程 | 诊断写入失败不能阻断编辑、保存、AI 对话、预览或发布 |
| 只记录关键边界 | 默认记录开始、成功、失败、版本变化、工作区绑定、文件变更摘要和用户触发操作 |
| 控制高频噪声 | 高频状态快照应采样、合并或只在显式诊断模式下保留 |
| 隐私优先 | 长期日志不得保存完整源码、完整 prompt、完整回复、密钥、token、cookie 和上传内容 |
| 可判断完整性 | 查询输出必须说明是否使用兜底数据、事件库是否可用、是否可能存在事件缺口 |
| 可被 Codex 使用 | 输出默认适合机器读取，人类阅读格式只作为辅助 |
