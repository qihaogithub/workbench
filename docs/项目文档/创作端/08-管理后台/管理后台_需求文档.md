# 管理后台需求文档

## 模块定位

管理后台是创作端的系统管理入口,允许拥有管理员密钥的用户通过 Web 界面动态调整全局配置,无需修改环境变量或重启服务。

## 功能需求清单

### 1. 管理员鉴权

- 用户通过 URL 参数携带管理员密钥访问 `/admin` 路径
- 首次验证成功后自动设置 Cookie,后续访问无需重复输入密钥
- Cookie 过期后需重新验证
- 未通过验证时返回 401 拒绝访问

### 2. 模型配置管理

#### 2.1 后端供应商配置

- 管理员可配置平台级大模型供应商
- 每个供应商可设置供应商 ID、展示名称、OpenAI 兼容 baseURL、API Key、模型列表和默认模型
- 供应商可启用或禁用
- 管理员可选择平台默认供应商和默认模型
- 保存后配置应尽量立即同步到后端服务，无需重启

#### 2.2 自动拉取可用模型

- 从后端服务获取当前所有可用模型列表
- 显示模型 ID、名称、分组信息

#### 2.3 启用/禁用控制

- 通过开关控制每个模型的启用状态
- 启用的模型在前端用户可见,禁用的模型隐藏
- 支持一键启用单个模型

#### 2.4 优先级排序

- 启用的模型列表支持拖拽调整顺序
- 列表顺序即为模型优先级
- 第一个启用的模型自动作为默认模型

#### 2.5 自动启用规则

- 支持按分组前缀设置自动启用规则(如 `xjjj/` 前缀的所有模型)
- 支持按名称关键词设置自动启用规则(如名称包含 `Free` 的模型)
- 匹配规则的模型自动加入启用列表
- 显示每条规则匹配的模型数量
- 支持一键批量启用匹配规则的所有模型

#### 2.6 多模态标记

- 可为每个启用的模型单独设置是否支持图片输入
- 标记为多模态的模型允许用户上传文件

#### 2.7 配置保存与生效

- 修改配置后点击保存按钮即可生效
- 保存后前端立即加载新配置,无需刷新页面
- 提供手动清除缓存的入口
- 个人用户仍可在账号设置中配置自己的模型 API，个人配置优先于平台默认配置

### 3. 概览首页

- 显示管理后台功能入口导航
- 展示系统状态概览信息

### 4. 用户管理

- 管理员可通过 API 获取所有注册用户的基本信息列表
- 返回的用户信息包含用户名、注册时间等，不包含密码等敏感字段
- 仅提供查询能力，不支持通过管理后台修改用户数据

## 业务规则与约束

### 前置条件

- 必须在 `.env` 中配置 `ADMIN_SECRET` 环境变量
- 数据库必须已初始化(包含 `system_configs` 表)
- 若需要把模型供应商配置实时同步到后端服务，author-site 与后端服务需共享内部调用密钥

### 限制条件

- 管理员密钥为共享密钥,不支持多用户独立账号
- 管理后台配置的是平台全局默认供应商，不会覆盖用户个人模型配置
- 后端服务不可用或内部同步失败时，配置仍可保存，但可能需要后端恢复后重新同步

### 安全要求

- 生产环境必须使用强随机字符串作为管理员密钥(至少 32 字符)
- 生产环境必须启用 HTTPS
- Cookie 必须设置为 httpOnly 防止 XSS 攻击
- API Key 等供应商凭证不得在普通用户界面明文展示

## UI/UX 设计要求

- 复用现有 shadcn/ui 组件库,保持与创作端一致的视觉风格
- 侧边栏导航布局,清晰展示各功能模块入口
- 模型列表区域分明:已启用模型区、自动启用规则区、未启用模型区
- 提供清晰的操作反馈(保存成功、错误提示等)
- 支持搜索过滤未启用模型
