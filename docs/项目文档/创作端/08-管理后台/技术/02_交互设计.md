# 模型选择交互设计

## 页面布局

AI 模型管理页面分为两个 Tab：后端供应商和模型白名单。后端供应商 Tab 负责维护平台级 provider、baseURL、API Key、模型列表和默认模型；模型白名单 Tab 负责控制前端用户可见模型、排序、自动启用规则和多模态标记。

### 已启用模型区

**展示内容**:

- 拖拽柄: 用于调整顺序
- 序号: 自动编号,表示优先级(1 为最高)
- 模型 ID: 完整标识(如 `xjjj/deepseek-v4-flash`)
- 分组标签: 从模型 ID 提取的前缀(如 `xjjj/`)
- 默认标记: 序号为 1 的模型自动显示「默认」徽章
- 多模态开关: 控制该模型是否支持图片输入
- 启用开关: 关闭后模型移至未启用区

**交互行为**:

- 拖拽调整顺序 → 实时更新 `enabledModels` 数组
- 关闭启用开关 → 从 `enabledModels` 移除,加入未启用区
- 开启多模态开关 → 模型 ID 加入 `multimodalModels` 数组
- 保存后 `multimodalModels` 会同步到 agent-service 的运行时模型配置,用于判断图片附件是直传主模型还是进入图片预描述流程

### 自动启用规则区

**规则类型**:

1. **分组前缀** (`prefix`):
   - 匹配模型 ID 的前缀(如 `xjjj/`)
   - 示例: 规则值 `xjjj/` 匹配 `xjjj/deepseek-v4-flash`、`xjjj/gpt-4o` 等

2. **名称关键词** (`nameFilter`):
   - 格式: `分组:关键词`(如 `workbench:Free`)
   - 匹配逻辑: 模型 ID 以该分组开头,且名称部分包含关键词
   - 示例: 规则值 `workbench:Free` 匹配 `workbench/free-model-1`

**交互行为**:

- 添加规则 → 输入规则类型和值 → 显示匹配计数
- 点击「批量启用」→ 所有匹配规则的模型加入已启用列表
- 删除规则 → 从 `autoEnableRules` 移除(已启用的模型不受影响)

### 未启用模型区

**展示内容**:

- 模型 ID 和分组标签
- 匹配自动启用规则的模型显示「自动启用」徽章
- 「启用」按钮: 点击后模型加入已启用列表末尾

**搜索过滤**:

- 顶部搜索框支持按模型 ID 或名称过滤
- 实时过滤,输入即搜索

**交互行为**:

- 点击「启用」→ 模型追加到 `enabledModels` 末尾 → 移至已启用区
- 如果模型匹配自动启用规则,显示徽章提示

## 技术实现

### 拖拽排序

使用 `@dnd-kit/core` + `@dnd-kit/sortable` 实现:

```typescript
import { DndContext, closestCenter } from "@dnd-kit/core";
import {
  SortableContext,
  verticalListSortingStrategy,
  useSortable,
} from "@dnd-kit/sortable";

// 拖拽结束事件处理
function handleDragEnd(event: DragEndEvent) {
  const { active, over } = event;
  if (!over || active.id === over.id) return;

  const oldIndex = enabledModels.indexOf(active.id as string);
  const newIndex = enabledModels.indexOf(over.id as string);

  // 数组元素移动
  const newOrder = arrayMove(enabledModels, oldIndex, newIndex);
  setEnabledModels(newOrder);
}
```

**关键点**:

- `verticalListSortingStrategy`: 垂直列表排序策略
- `arrayMove()`: 工具函数,安全移动数组元素
- 拖拽结束后立即更新状态,实时反映到 UI

### 自动启用规则匹配

匹配逻辑在 `ai-models.ts` 的 `applyModelConfigsWithFullData()` 中实现:

```typescript
function matchesAutoEnableRules(
  modelId: string,
  rules: AutoEnableRule[],
): boolean {
  return rules.some((rule) => {
    if (rule.type === "prefix") {
      return modelId.startsWith(rule.value);
    }
    if (rule.type === "nameFilter") {
      const [group, keyword] = rule.value.split(":");
      return modelId.startsWith(group + "/") && modelId.includes(keyword);
    }
    return false;
  });
}
```

**过滤策略**:

1. 首先保留 `enabledModels` 列表中的所有模型
2. 扫描所有后端可用模型,匹配自动启用规则的加入列表
3. 最终结果 = `enabledModels`(按原顺序) + 新匹配模型(追加末尾)

### 数据流

```
页面加载
  ↓
调用 GET /api/admin/model-config
  ↓
加载全局供应商配置与模型白名单配置
  ↓
调用 GET /api/admin/backend-providers/sync
  ↓
展示数据库保存状态、最近同步状态和 agent-service 运行时摘要
  ↓
调用 GET /api/admin/available-models
  ↓
显示后端可用模型列表
  ↓
渲染三个区域
  ↓
用户操作(开关/拖拽/添加规则)
  ↓
更新本地状态
  ↓
点击「保存配置」
  ↓
PUT /api/admin/model-config
  ↓
写入 SQLite,清除缓存
  ↓
如包含后端供应商配置,推送到 agent-service
  ↓
前端收到成功响应,重新拉取同步状态
```

保存反馈分为两层：数据库保存成功后即提示配置已保存；agent-service 同步成功时再提示运行时已生效。同步失败时页面展示失败原因，并提供“重试同步已保存配置”操作。重试不会读取当前未保存的表单状态，只会推送数据库中的最新配置。

## 后端模型查询

### Agent-service 端点

新增 `GET /models` 端点（`packages/agent-service/src/routes/models.ts`）：

```typescript
fastify.get("/models", async (request, reply) => {
  const backend = factory.getCurrentBackend();
  const models = await backend.getModelInfo();
  return { success: true, data: { models } };
});
```

**实现细节**:

- 复用 `workbenchHttpBackend.getModelInfo()` 方法
- 不依赖 WebSocket session,直接 HTTP 查询
- 返回模型列表: `[{ id: "xjjj/deepseek-v4-flash", label: "DeepSeek V4 Flash" }]`

### Author-site 代理 API

新增 `GET /api/admin/available-models`（`packages/author-site/src/app/api/admin/available-models/route.ts`）：

```typescript
export async function GET(request: Request) {
  if (!(await verifyAdminRequest(request))) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }

  // 代理请求 agent-service
  const res = await fetch(`${AGENT_SERVICE_URL}/models`, {
    signal: AbortSignal.timeout(5000), // 5s 超时
  });

  const data = await res.json();
  return NextResponse.json(data);
}
```

**设计理由**:

- 管理后台前端不直接访问 agent-service(跨域问题)
- 通过 author-site 代理,统一鉴权(admin 权限)
- 5s 超时防止后端响应慢阻塞页面
- 后端供应商配置保存后通过内部接口推送到 agent-service，使平台默认模型尽量立即生效

## 优势

1. **零配置上手**: 无需知道模型 ID,打开页面即可看到所有可用模型
2. **所见即所得**: 开关状态直观反映启用情况
3. **拖拽排序**: 优先级调整更符合直觉
4. **智能规则**: 自动启用规则减少重复操作,后端新增模型时自动匹配
5. **实时反馈**: 匹配计数、自动启用徽章等视觉提示
6. **运行时供应商配置**: 管理员可在页面维护平台级 OpenAI 兼容供应商
