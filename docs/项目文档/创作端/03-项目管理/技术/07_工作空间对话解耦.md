# 工作空间与对话解耦 - 实现设计

> 版本：v1.1
> 创建日期：2026-05-03
> 更新日期：2026-07-01
> 关联需求：[会话管理_需求文档.md](../会话管理_需求文档.md) §2.5 对话上下文切换功能

## 一、设计目标

将 Session（对话）与 Workspace（工作空间）从一对一绑定关系解耦为多会话共享关系：

- **解耦前**：每个 Session 拥有独立的文件副本，切换对话 = 切换一切
- **解耦后**：同一项目的多个 Session 默认共享项目工作区，切换对话仅切换 AI 上下文

## 二、数据模型变更

### 2.1 SessionMeta 新增 workspaceId

Session 通过 `workspaceId` 字段关联到项目工作区，不再直接持有文件：

```
Session（对话）──workspaceId──► Workspace（文件载体）
    │                                │
    ├── 对话消息                      ├── index.tsx
    ├── AI 上下文                     ├── config.schema.json
    └── 状态（editing/archived）       └── .workbench/
```

### 2.2 Session 状态扩展

新增 `archived` 状态，用于标记被切换离开的对话：

| 状态 | 说明 |
|:-----|:-----|
| editing | 当前活跃对话 |
| archived | 用户切换到其他对话，该对话被归档 |
| saved | 用户保存，对话结束 |
| discarded | 用户放弃，对话结束 |

### 2.3 目录结构变更

```
data/
├── workspaces/                         # 项目工作区与分支工作区目录
│   ├── projects/{projectId}/
│   │   └── live-{timestamp}-{random}/   # 默认项目工作区
│   └── {actorId}/{projectId}/
│       └── cli_edit-.../                # 显式 branch/transaction Workspace
│           ├── .workspace.json         # 工作空间元数据
│           ├── index.tsx
│           ├── config.schema.json
│           └── .workbench/
│
├── sessions/                           # Session 仅存储对话元数据
│   └── {userId}/{projectId}/
│       └── session-{timestamp}-{random}/
│           └── .session.json           # 含 workspaceId 字段
│
└── ...（projects、snapshots 不变）
```

## 三、核心模块

### 3.1 workspace-manager.ts（新建）

职责：项目工作区的创建、查询、同步和分支/legacy Workspace 兼容。

关键函数：

| 函数 | 说明 |
|:-----|:-----|
| `getOrCreateProjectActiveWorkspace(projectId)` | 创建或复用项目工作区 |
| `materializeCanonicalWorkspace(...)` | 业务层推进项目基准工作区的统一入口，内部使用底层 canonical 同步实现 |
| `createWorkspace(userId, projectId)` | 创建显式分支/legacy 工作空间 |
| `findActiveWorkspace(userId, projectId)` | 仅用于旧数据迁移，查找用户历史工作空间 |
| `findWorkspacePath(workspaceId)` | 根据 ID 查找工作空间路径 |
| `getWorkspaceFiles(workspaceId)` | 读取工作空间的 code 和 schema |
| `updateWorkspaceFiles(workspaceId, files)` | 更新工作空间的 code 和 schema |
| `deleteWorkspace(workspaceId)` | 删除工作空间 |
| `injectworkbenchAgentConfig(workspacePath)` | 注入 .workbench 配置（从 session-manager 迁移） |

### 3.2 session-manager.ts（改造）

变更要点：

| 变更 | 说明 |
|:-----|:-----|
| `createEditSession` 新增 `existingWorkspaceId` 参数 | 不传时绑定项目工作区；传入时用于显式分支/legacy Workspace |
| Session 元数据写入 `workspaceId` | 每个 Session 记录其关联的工作空间 |
| `archiveActiveSession` | 切换对话时将旧对话标记为 archived |
| live Workspace 清理保护 | 归档、放弃或过期 Session 不删除项目工作区 |

### 3.3 fs-utils.ts（扩展）

新增工作空间相关工具函数：

| 函数 | 说明 |
|:-----|:-----|
| `getWorkspacesDir()` | 获取工作空间根目录 |
| `getWorkspaceDir(userId, projectId)` | 获取用户项目的工作空间目录 |
| `findWorkspacePath(workspaceId)` | 全局搜索工作空间路径 |
| `getWorkspaceFiles(workspaceId)` | 读取工作空间文件 |
| `updateWorkspaceFiles(workspaceId, files)` | 写入工作空间文件 |
| `ensureWorkspaceFiles(workspacePath)` | 确保工作空间包含必要文件 |

## 四、API 变更

### 4.1 新增 Workspace API

| 方法 | 路径 | 说明 |
|:-----|:-----|:-----|
| GET | `/api/workspaces` | 列出工作空间 |
| GET | `/api/workspaces/{workspaceId}` | 获取工作空间详情 |
| DELETE | `/api/workspaces/{workspaceId}` | 删除工作空间 |

### 4.2 Session API 变更

**POST /api/sessions**：

- 请求体新增 `workspaceId`（可选）：传入时复用已有工作空间创建新对话
- 响应体新增 `workspaceId`：返回关联的工作空间 ID

**GET /api/sessions/{sessionId}**：

- 响应体包含 `workspaceId` 和 `workspacePath`

**PUT /api/sessions/{sessionId}/files**：

- 通过 `sessionMeta.workspaceId` 定位工作空间文件并更新

## 五、前端传递链路

workspaceId 从后端到前端的完整传递路径：

```
Session API 返回 workspaceId
      │
      ▼
编辑页（Demo/Project）设置 workspaceId 状态
      │
      ▼
AIChat 组件接收 workspaceId 属性
      │
      ▼
HistoryDialog 接收 workspaceId 属性
      │
      ├── 新建对话 → onNewSession(workspaceId) → 编辑页复用工作空间
      │
      └── 切换对话 → onSelectSession(sessionId, workspaceId) → 编辑页仅切换消息
```

关键约束：编辑页在 `onNewSession` 和 `onSelectSession` 回调中**不重置** code/schema/editorContent。

## 六、循环依赖解决

workspace-manager.ts 和 session-manager.ts 曾存在循环依赖：

- session-manager 导入 workspace-manager 的 `createWorkspace`
- workspace-manager 导入 session-manager 的 `injectworkbenchAgentConfig`

解决方案：将 `injectworkbenchAgentConfig` 从 session-manager 迁移到 workspace-manager，使依赖关系变为单向：

```
session-manager ──依赖──► workspace-manager（单向）
```
