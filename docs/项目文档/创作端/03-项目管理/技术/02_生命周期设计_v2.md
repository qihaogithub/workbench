# 会话与版本管理 - 生命周期设计

> 版本：v3.0
> 更新日期：2026-05-03
> 变更说明：Session 与 Workspace 解耦，新增 archived 状态

---

## 一、会话生命周期概述

### 1.1 状态定义

| 状态 | 说明 |
|:-----|:-----|
| 创建中 | 会话正在初始化 |
| 编辑中 (editing) | 会话正常使用中 |
| 已归档 (archived) | 用户切换到其他对话，该对话被归档 |
| 已保存 (saved) | 用户已保存，会话结束 |
| 已放弃 (discarded) | 用户已放弃，会话结束 |
| 过期 | 会话已超时，等待清理 |

### 1.2 状态流转

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   用户打开项目编辑                                              │
│        │                                                        │
│        ▼                                                        │
│   ┌─────────────┐                                               │
│   │   编辑中    │◄────────────────────────┐                    │
│   │  (editing)  │                          │                    │
│   └──────┬──────┘                          │                    │
│          │                                 │                    │
│          ├──── 用户活动（编辑/配置）───────┘                    │
│          │                                                      │
│          ├──── 用户保存 ─────► 创建版本 ─────► 已保存 (saved)   │
│          │                                                      │
│          ├──── 用户放弃 ─────► 清理临时空间 ─────► 已放弃       │
│          │                                                      │
│          ├──── 用户切换对话 ─────► 已归档 (archived)            │
│          │                                                      │
│          └──── 长时间无活动 ─────► 自动过期 ─────► 自动清理     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、会话元数据

### 2.1 数据结构

```typescript
interface EditSession {
  sessionId: string             // 会话唯一标识
  projectId: string             // 关联的项目 ID
  workspaceId: string           // 关联的工作空间 ID
  username: string              // 当前编辑者用户名
  basedOnVersion: string        // 基于的版本号
  status: 'editing' | 'archived' | 'saved' | 'discarded'
  opencodeSessionId: string | null  // OpenCode Agent 会话 ID
  createdAt: number             // 创建时间戳
  expiresAt: number             // 过期时间戳
}
```

### 2.2 存储位置

会话元数据存储在 Session 目录的 `.session.json` 文件中：

```
sessions/
└── {userId}/{projectId}/
    └── session-{timestamp}-{random}/
        └── .session.json    ← 会话元数据（含 workspaceId）
```

代码文件存储在关联的 Workspace 目录中，Session 不直接持有文件。

## 三、会话创建

### 3.1 创建流程

```
用户打开项目编辑
        │
        ▼
检查项目是否存在
        │
        ├── 不存在 → 返回错误
        │
        ▼ 存在
生成 Session ID
格式：session-{timestamp}-{random}
        │
        ▼
创建或复用工作空间
（通过 existingWorkspaceId 参数控制）
        │
        ▼
创建会话元数据（关联 workspaceId）
        │
        ▼
返回会话信息
```

### 3.2 Session ID 生成规则

```
session-{timestamp}-{random}

示例：session-1712345678-abc123def
```

| 部分 | 说明 |
|:-----|:-----|
| `session-` | 固定前缀 |
| `{timestamp}` | 毫秒级时间戳 |
| `{random}` | 随机字符串 |

## 四、会话保持

### 4.1 活跃度检测

| 检测方式 | 说明 |
|:---------|:-----|
| 文件更新 | 更新会话文件时刷新过期时间 |
| API 调用 | 会话相关 API 调用时刷新过期时间 |
| 心跳 | 可选的定期心跳检测 |

### 4.2 过期时间计算

```typescript
const SESSION_EXPIRY_MS = 2 * 60 * 60 * 1000  // 2 小时

function calculateExpiryTime(): number {
  return Date.now() + SESSION_EXPIRY_MS
}
```

## 五、会话过期

### 5.1 过期检测

```typescript
function isSessionExpired(session: EditSession): boolean {
  return Date.now() > session.expiresAt
}
```

### 5.2 过期策略

| 配置项 | 默认值 | 说明 |
|:-------|:-------|:-----|
| 会话过期时间 | 2 小时 | 用户无活动后会话自动过期 |
| 过期前提醒 | 10 分钟 | 会话即将过期前提醒用户 |
| 最大会话时长 | 24 小时 | 单个会话的最大生命周期 |

## 六、会话清理

### 6.1 清理触发

| 触发方式 | 说明 |
|:---------|:-----|
| 保存后 | 保存成功后立即删除临时空间 |
| 放弃后 | 用户确认放弃后立即删除 |
| 定时任务 | 定期清理过期的会话 |
| 页面加载 | 项目列表加载时触发清理 |

### 6.2 清理逻辑

```typescript
function cleanupExpiredSessions(): string[] {
  const sessionsRoot = getSessionsRoot()
  const cleaned: string[] = []

  for (const projectDir of fs.readdirSync(sessionsRoot)) {
    const projectPath = path.join(sessionsRoot, projectDir)
    
    for (const sessionDir of fs.readdirSync(projectPath)) {
      const sessionPath = path.join(projectPath, sessionDir)
      const metaPath = path.join(sessionPath, '.session.json')
      
      if (!fs.existsSync(metaPath)) continue
      
      const session = JSON.parse(fs.readFileSync(metaPath, 'utf-8'))
      
      if (Date.now() > session.expiresAt) {
        fs.rmSync(sessionPath, { recursive: true })
        cleaned.push(sessionDir)
      }
    }
  }

  return cleaned
}
```
