# 会话与版本管理 - 草稿工作区

> 版本：v3.0
> 更新日期：2026-05-03
> 变更说明：Session 与 Workspace 解耦，Workspace 成为独立实体

---

## 一、工作空间概述

### 1.1 设计理念

```
"工作空间与对话解耦：切换对话只影响 AI 上下文，不影响文件状态"
```

工作空间隔离确保用户编辑操作不影响原始项目文件，同时支持多个对话共享同一工作空间。

### 1.2 空间类型

| 空间类型 | 目录 | 说明 |
|:---------|:-----|:-----|
| 正式工作空间 | `projects/{id}/workspace/` | 项目的正式存储区域 |
| 独立工作空间 | `workspaces/{userId}/{projectId}/{workspaceId}/` | 编辑时的隔离空间，可被多个 Session 共享 |
| 版本快照 | `snapshots/{projectId}/{versionId}/` | 版本备份 |

### 1.3 工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                         文件系统                                 │
│                                                                 │
│  projects/                      workspaces/                     │
│  └── proj_xxx/                  └── {userId}/proj_xxx/         │
│      ├── project.json               ├── ws-111/                │
│      └── workspace/                 │   ├── .workspace.json    │
│          ├── index.tsx              │   ├── index.tsx (副本)    │
│          └── config.schema.json     │   └── .opencode/         │
│                                      └── ws-222/                │
│  sessions/                               ├── .workspace.json    │
│  └── {userId}/proj_xxx/                 ├── index.tsx (副本)    │
│      ├── session-aaa/                    └── .opencode/         │
│      │   └── .session.json                                    │
│      │       (workspaceId: ws-111)                             │
│      └── session-bbb/                                         │
│          └── .session.json                                    │
│              (workspaceId: ws-111)                             │
│                                                                 │
│  snapshots/                                                    │
│  └── proj_xxx/                                                │
│      ├── v1/                                                  │
│      ├── v2/                                                  │
│      └── ...                                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、目录结构

### 2.1 项目目录

```
projects/
└── proj_{id}/
    ├── project.json             # 项目元数据
    └── workspace/               # 正式工作空间
        ├── index.html
        ├── styles.css
        └── ...
```

### 2.2 独立工作空间

```
workspaces/
└── {userId}/{projectId}/
    └── ws-{timestamp}-{random}/     # 独立工作空间
        ├── .workspace.json          # 工作空间元数据
        ├── index.tsx                # 组件代码
        ├── config.schema.json       # 配置定义
        └── .opencode/               # AI 代理配置
```

### 2.3 会话目录

```
sessions/
└── {userId}/{projectId}/
    └── session-{timestamp}-{random}/
        └── .session.json            # 会话元数据（含 workspaceId）
```

### 2.4 版本快照

```
snapshots/
└── proj_{projectId}/
    ├── v1/                      # 版本 1 快照
    │   ├── index.html
    │   └── styles.css
    ├── v2/                      # 版本 2 快照
    └── ...
```

## 三、核心操作

### 3.1 创建工作空间

**文件位置**：`src/lib/workspace-manager.ts`

创建工作空间时，系统复制项目文件到独立目录，并注入 AI 代理配置。创建 Session 时可选择复用已有工作空间（传入 `existingWorkspaceId`）或创建新工作空间。

关键步骤：
1. 生成 workspaceId（格式：`ws-{timestamp}-{random}`）
2. 复制项目文件到工作空间目录
3. 写入 `.workspace.json` 元数据
4. 注入 `.opencode` AI 代理配置
5. 返回工作空间信息（workspaceId、code、schema、workspacePath）

### 3.2 读取工作空间文件

通过 `workspaceId` 查找工作空间路径，读取其中的 `index.tsx` 和 `config.schema.json`。

### 3.3 更新工作空间文件

通过 `workspaceId` 定位工作空间，写入更新后的 code 和 schema。所有通过该 workspaceId 关联的 Session 共享同一份文件。

### 3.4 合并到正式空间

合并时将工作空间内容覆盖到正式目录，自动过滤系统文件（.opencode、.workspace.json 等）。

### 3.5 删除工作空间

删除工作空间时递归删除整个目录。

## 四、文件系统工具

### 4.1 工具函数

**文件位置**：`src/lib/fs-utils.ts`

| 函数 | 说明 |
|:-----|:-----|
| `getProjectsRoot()` | 获取项目根目录 |
| `getWorkspacesDir()` | 获取工作空间根目录 |
| `getWorkspaceDir(userId, projectId)` | 获取用户项目的工作空间目录 |
| `getSessionsRoot()` | 获取会话根目录 |
| `getSnapshotsRoot()` | 获取快照根目录 |
| `getProjectDir(id)` | 获取指定项目目录 |
| `getSessionDir(sessionId)` | 获取指定会话目录 |
| `getSnapshotDir(projectId, versionId)` | 获取指定快照目录 |
| `findWorkspacePath(workspaceId)` | 全局搜索工作空间路径 |
| `getWorkspaceFiles(workspaceId)` | 读取工作空间文件 |
| `updateWorkspaceFiles(workspaceId, files)` | 写入工作空间文件 |
| `ensureDirsExist()` | 确保目录存在 |
| `countFiles(dir)` | 统计目录文件数量 |

### 4.2 路径计算

```
getProjectDir(id)        → {dataRoot}/projects/proj_{id}
getWorkspaceDir(userId, projectId) → {dataRoot}/workspaces/{userId}/{projectId}
getSessionDir(sessionId) → {dataRoot}/sessions/{userId}/{projectId}/{sessionId}
getSnapshotDir(projectId, versionId) → {dataRoot}/snapshots/{projectId}/{versionId}
```

## 五、并发控制

### 5.1 多用户隔离

多个用户同时编辑同一项目时，各自拥有独立的工作空间：

```
用户 A 编辑 proj_xxx
    │
    └──► workspaces/userA/proj_xxx/ws-111/ (proj_xxx 的副本)

用户 B 编辑 proj_xxx
    │
    └──► workspaces/userB/proj_xxx/ws-222/ (proj_xxx 的另一个副本)
```

同一用户的多个对话可共享同一工作空间：

```
用户 A 的对话 1
    │
    └──► workspaces/userA/proj_xxx/ws-111/ (共享)

用户 A 的对话 2
    │
    └──► workspaces/userA/proj_xxx/ws-111/ (共享同一工作空间)
```

### 5.2 单用户限制

同一用户同时只能有一个活跃编辑会话，但可以切换对话（共享工作空间）：

- 新建对话时，复用当前工作空间（传入 `existingWorkspaceId`）
- 切换对话时，旧对话标记为 `archived`，新对话标记为 `editing`
- 工作空间文件状态不受对话切换影响

## 六、数据安全

### 6.1 原子化操作

保存操作使用原子化复制：

```typescript
// 先创建快照，再覆盖正式空间
fs.cpSync(project.workspacePath, snapshotPath, { recursive: true })
fs.cpSync(session.tempWorkspace, project.workspacePath, { recursive: true })
```

### 6.2 异常恢复

系统异常时，工作空间可通过 workspaceId 恢复：

- 通过 `findWorkspacePath(workspaceId)` 查找工作空间路径
- 读取 `.workspace.json` 确认工作空间元数据
- 关联的 Session 可通过 workspaceId 重新连接到工作空间
