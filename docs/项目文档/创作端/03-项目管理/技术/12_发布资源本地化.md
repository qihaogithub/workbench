# 发布资源本地化

> 版本：v1.5
> 更新日期：2026-07-09

## 一、目标

项目发布时，发布产物应尽量由本项目服务器独立提供。页面、样式或配置里引用的外部图片会在发布阶段下载到发布目录，并把产物中的引用改写为 `/data/{projectId}/assets/images/...`。

这让使用端访问发布项目时不再依赖第三方图片站点的速度和可用性。发布失败时，用户应看到可理解的发布错误，而不是得到一个图片半坏的发布包。

## 二、处理范围

发布资源本地化覆盖三类图片来源：

- 工作空间内的相对图片路径。
- 项目图床路径，例如 `/api/images/...`。
- 当前编辑会话上传的资源路径，例如 `/api/sessions/.../assets/...`。
- `http:` / `https:` 外部图片地址。

扫描范围包括页面代码、原型 HTML、CSS、页面配置、项目级配置定义和项目级共享配置运行值。重复引用会按源地址和内容去重，发布目录里只保存一份可复用图片文件。

发布按钮触发的正式发布会先等待共享配置运行值保存完成，再同步当前 workspace 到项目基准工作区，随后才进入图片扫描和本地化。这样配置面板中新上传的 `/api/sessions/.../assets/...` 图片不会因为保存请求晚于发布请求而遗漏到发布包之外。

项目级配置定义里的 schema default 只在没有同名运行值时参与资源扫描；如果 `project.config.values.json` 已经覆盖了某个字段，发布本地化只处理运行值引用的实际资源。这样旧默认图床失效时，不会阻断已经改用新上传图片的发布流程。

发布 API 不再单独补写共享配置运行值到项目基准工作区；共享配置运行值必须在发布前通过当前 workspace 同步边界进入项目基准工作区。这样发布入口不会绕过 Workspace Authority 或重复维护一条局部 canonical 写入路径。

## 三、安全限制

编辑期图片资产化和发布期图片本地化是两层能力。编辑期面向当前用户正在看的页面，入口是 `POST /api/sessions/{sessionId}/assets/localize`：前端在预览区选中图片后，优先把浏览器已经可读取的图片 Blob 交给 author-site；只有浏览器不可读时，后端才按外部 URL 兜底下载。成功后图片通过 Workspace Authority staging + mutation 写入当前 workspace 的 `assets/images/`，同时登记到项目级 `images.json`，页面内通常使用从 `demos/{pageId}/` 指向该资源的相对路径。

项目级 `images.json` 是当前图片资产登记清单。`project-core` 的资产上传会计算内容哈希，默认按 `{hash}-{filename}` 写入 `assets/images/`，并记录 `contentHash`、`mimeType`、`originalUrl`、`sourceType`、`createdBy` 和创建时间；显式 `targetPath` 仍按调用方指定路径写入。`asset list` 会把清单元数据和 workspace 引用扫描结果合并返回，因此删除、替换和发布前检查可以看到同一份资产事实。

HTML/CSS 原型页选中图片本地化成功后，右侧属性面板会把 `src` 直接改写为 `../../assets/images/...` 并保存到 `prototype.html`；其他无法确定写回位置的场景仍可把已本地化资产路径交给 AI 处理。发布阶段继续扫描历史项目、手写代码和配置里的外链，定位为安全网，而不是编辑期资源管理的主路径。

通过 Project Admin CLI 导入 HTML/CSS 原型页时，`project import-prototype` 和 `page update-prototypes` 会在写入页面前扫描原型 HTML、CSS 和页面 Schema 中的远程图片 URL。能成功下载且符合图片类型和大小限制的资源会先进入项目 `assets/images/` 并登记到 `images.json`，再把页面内容改写为 `../../assets/images/...`。无法本地化的远程图片会保留原始外链，并在导入结果中返回警告；发布阶段仍会作为安全网再次尝试处理。

外部图片下载前必须验证 URL 协议和解析地址。localhost、回环地址、内网地址、链路本地地址等私有网络目标会被拒绝，避免发布流程被用来访问内部服务。

下载结果还必须满足：

- 响应 `content-type` 是受支持的图片类型。
- 响应大小不超过发布限制。
- 重定向次数受限，并且每次重定向目标仍需通过公网地址校验。

外部图片本地化采用 best-effort 策略：下载成功时替换为本项目 data URL，下载失败、content-type 不匹配或命中私网地址保护时，发布产物保留原始外链并继续发布。本地 workspace 图片、项目图床图片和当前会话上传图片属于必须可复制的项目内资源；这些资源缺失、超限或不可读时才阻断发布，并由发布 API 返回图片资源本地化失败提示。

## 四、发布产物

本地化后的图片写入发布目录：

`data/published/{projectId}/assets/images/{contentHash}.{ext}`

页面 HTML、CSS、编译结果和相关配置中的原始图片引用会被替换成本项目 data URL。使用端读取发布产物时，图片请求命中 author-site 的 `/data/...` 静态数据路由或等价部署数据源。

## 五、与发布快照的关系

发布快照仍由 [04_版本管理.md](./04_版本管理.md) 负责。资源本地化发生在发布临时目录中，成功后临时目录整体替换正式发布目录；失败时不会覆盖旧发布产物。
