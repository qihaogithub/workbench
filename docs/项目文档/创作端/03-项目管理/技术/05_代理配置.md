> 版本：v2.0
> 更新日期：2026-05-03

## 一、背景与目标

### 1.1 背景

在 OpenCode Workbench 中，AI 代理负责生成和修改 Demo 代码。为确保 AI 生成的代码符合系统标准，需要在工作空间中注入专用的 AI 代理配置。

### 1.2 目标

1. **标准化输出**：AI 只生成 `index.tsx` 和 `config.schema.json` 两个文件
2. **格式统一**：代码风格、组件结构、Schema 定义遵循统一规范
3. **质量保证**：生成的代码可直接运行，无需额外修改
4. **会话隔离**：配置仅限工作空间使用，不影响原始 Demo 文件

## 二、目录结构

### 2.1 .opencode 目录

```
workspaces/
└── {userId}/{projectId}/
    └── ws-{timestamp}-{random}/
        ├── index.tsx              # 组件代码副本
        ├── config.schema.json     # 配置定义副本
        ├── .workspace.json        # 工作空间元数据
        └── .opencode/             # OpenCode AI 代理配置
            ├── opencode.json      # OpenCode 项目配置
            └── agents/
                └── demo-generator.md  # Demo 生成代理提示词
```

### 2.2 文件说明

| 文件 | 说明 |
|------|------|
| `opencode.json` | OpenCode 项目级配置，定义代理名称、权限、指令文件 |
| `agents/demo-generator.md` | AI 代理的系统提示词，包含角色定义、规则、示例等 |

## 三、实现机制

### 3.1 注入时机

**自动注入**：在创建 Workspace 时自动完成

```typescript
// workspace-manager.ts
export function createWorkspace(userId: string, projectId: string): CreateWorkspaceResult {
  // ... 复制项目文件到工作空间 ...
  
  // 注入 .opencode 代理配置
  injectOpencodeAgentConfig(workspacePath);
  
  // ... 返回结果 ...
}
```

**说明**：`injectOpencodeAgentConfig` 函数已从 `session-manager.ts` 迁移至 `workspace-manager.ts`，因为注入时机是创建工作空间而非创建对话。

### 3.2 注入逻辑

**核心函数**：`injectOpencodeAgentConfig(sessionPath: string)`

**执行步骤**：
1. 创建 `.opencode/agents/` 目录结构
2. 写入 `opencode.json` 配置文件
3. 写入 `agents/demo-generator.md` 提示词文件

### 3.3 清理机制

**自动清理**：在以下时机自动清理

1. **保存工作空间时**：
   ```typescript
   function mergeWorkspaceToProject(workspaceId: string): void {
     // 合并时过滤 .opencode 目录
     fs.cpSync(workspaceDir, projectDir, {
       recursive: true,
       filter: (src) => !path.basename(src).startsWith('.opencode')
     })
   }
   ```

2. **删除工作空间时**：
   ```typescript
   function deleteWorkspace(workspaceId: string): boolean {
     // 递归删除整个工作空间目录（包含 .opencode）
   }
   ```

3. **会话过期清理**：
   - 定时任务扫描过期 Session
   - 递归删除 Session 目录（包含 `.opencode`）

## 四、代理配置详解

### 4.1 opencode.json

```json
{
  "$schema": "https://opencode.ai/config.json",
  "agent": {
    "demo-generator": {
      "file": ".opencode/agents/demo-generator.md",
      "description": "专门用于生成 OpenCode Demo 文件的 AI 代理",
      "tools": {
        "write": true,      // 允许写入文件
        "edit": true,       // 允许编辑文件
        "bash": false,      // 禁止执行 shell 命令
        "fetch": false      // 禁止网络请求
      }
    }
  },
  "default_agent": "demo-generator",
  "instructions": [".opencode/agents/demo-generator.md"]
}
```

**字段说明**：

| 字段 | 说明 |
|------|------|
| `agent.demo-generator.file` | 代理提示词文件路径 |
| `agent.demo-generator.tools` | 权限控制，限制 AI 行为 |
| `default_agent` | 设置默认代理，OpenCode 启动时自动激活 |
| `instructions` | 注入到 AI 上下文的指令文件 |

### 4.2 demo-generator.md

AI 代理的系统提示词，包含以下内容：

1. **角色定位**：Demo 生成专家
2. **核心规则**：
   - 只操作 `index.tsx` 和 `config.schema.json`
   - 使用 TypeScript + Tailwind CSS + shadcn/ui
   - 遵循代码质量标准
3. **禁止行为**：
   - 禁止修改系统文件（.session.json 等）
   - 禁止生成额外文件
   - 禁止使用 `as any` 等不安全代码
4. **工作流程**：理解需求 → 设计 Schema → 实现组件 → 验证输出
5. **自检清单**：生成前的最终验证项

## 五、使用场景

### 5.1 用户编辑 Demo

```
用户点击"编辑 Demo"
      │
      ▼
系统创建 Workspace，注入 .opencode
      │
      ▼
系统创建 Session，关联 Workspace
      │
      ▼
用户在 OpenCode 中打开 Workspace
      │
      ▼
demo-generator 代理自动生效
      │
      ▼
AI 按照约束生成/修改代码
      │
      ▼
用户保存
      │
      ▼
合并到项目 + 清理 .opencode
```

### 5.2 AI 生成代码

当用户在 OpenCode 中请求 AI 生成代码时：

1. AI 读取 `demo-generator.md` 提示词
2. 根据规则生成 `index.tsx` 和 `config.schema.json`
3. 自检清单验证
4. 写入文件到 Session 目录
5. 前端预览系统实时更新

## 六、安全性

### 6.1 权限控制

| 权限 | 状态 | 说明 |
|------|------|------|
| 写入文件 | ✅ 允许 | 生成/修改代码 |
| 编辑文件 | ✅ 允许 | 更新现有代码 |
| 执行命令 | ❌ 禁止 | 防止恶意操作 |
| 网络请求 | ❌ 禁止 | 防止数据泄露 |

### 6.2 目录隔离

- `.opencode` 仅在工作空间目录中
- 保存时自动过滤，不合并到项目

### 6.3 代码质量保障

**自检清单**：
- [ ] 只修改了 `index.tsx` 和 `config.schema.json`
- [ ] Props 接口与 Schema properties 一一对应
- [ ] 没有使用不安全的类型转换
- [ ] 代码完整可运行
- [ ] 遵循 TypeScript 严格模式
- [ ] 使用 Tailwind CSS（不使用内联 style）

## 八、常见问题

### Q1: .opencode 会影响原始 Demo 文件吗？

**A**: 不会。`.opencode` 只在 Session 临时工作区中，保存 Session 时会被过滤，不影响原始 Demo。

### Q2: 如何自定义 AI 行为？

**A**: 编辑 `.opencode/agents/demo-generator.md` 文件。但注意这是临时修改，保存后会被清理。如需持久化自定义规则，应修改项目级别的配置。

### Q3: AI 没有按照规则生成文件怎么办？

**A**: 检查以下几点：
1. 确认 `.opencode` 目录存在且配置正确
2. 确认 `default_agent` 设置为 `"demo-generator"`
3. 重启 OpenCode 会话使配置生效
4. 在对话中明确说明"请按照 Demo 生成规则创建文件"
