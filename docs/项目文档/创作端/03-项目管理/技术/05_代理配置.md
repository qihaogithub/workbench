# Session 代理配置 - 技术文档

> 版本：v3.1
> 更新日期：2026-06-26
> 变更说明：补充 Pi Agent `webRead` 网页读取工具边界

---

## 一、定位

当前 AI 会话不再依赖 OpenCode Server、OpenCode CLI 或工作空间内的 `.opencode/agents/demo-generator.md`。创作端通过 Agent Client 把用户消息、工作空间、页面上下文、模型选择和系统提示词发送到 agent-service；agent-service 再用 Pi Agent 后端和工作台工具集执行。

这份文档说明“Session 代理配置”在当前实现中的四个来源：

1. **创作端系统提示词**：由 author-site 根据项目、页面、知识库、记忆和行为规则组装。
2. **Session 工作空间**：Session 绑定 `workingDir`，Pi Agent 工具只能在允许范围内读写。
3. **模型配置**：管理后台和用户配置通过 backend providers 同步到 agent-service。
4. **工具权限**：Pi Tools、权限确认和 SessionGuard 共同约束 AI 能做什么。

---

## 二、配置流转

```text
用户在创作端发送消息
  -> author-site 组装 content / systemPrompt / model / images / workingDir
  -> AgentClient 或 WebSocket stream-service 发送到 agent-service
  -> agent-service 按 sessionId 获取或创建 BackendAgent
  -> BackendAgent 更新 Pi Agent system prompt
  -> PiAgentBackend 合并模型配置并创建工作台工具集
  -> Pi Agent 执行消息、工具调用和文件修改
  -> WebSocketEventRouter 把事件和文件变更推回前端
```

Session 与工作空间解耦规则见 [07_工作空间对话解耦.md](./07_工作空间对话解耦.md)。项目编辑生命周期见 [01_架构设计.md](./01_架构设计.md)。

---

## 三、系统提示词

系统提示词不再通过工作空间文件注入，而是作为消息载荷的一部分传给 agent-service。这样切换对话、切换模型或更新规则时，不需要在项目目录写入隐藏配置文件。

当前提示词通常包含：

- 当前项目和页面上下文。
- 可修改文件范围和输出要求。
- 配置 Schema、预览、画布、知识库、记忆等模块规则。
- 当前实际可用工具列表。Pi Agent 后端会根据运行时 activeTools 追加工具能力说明。

技术边界：

- 前端负责构造适合当前页面的静态系统提示词。
- agent-service 负责在 Agent 已启动后调用 `updateSystemPrompt()`。
- Pi Agent 后端负责把工具列表、权限和运行上下文交给底层 AgentHarness。

---

## 四、模型配置

模型配置分为三层：

| 层级 | 来源 | 说明 |
| :--- | :--- | :--- |
| 全局配置 | 管理后台 backend providers | author-site 通过 `/internal/backend-providers` 推送到 agent-service |
| Session 配置 | 用户或会话级模型选择 | author-site 通过 `/internal/sessions/:sessionId/model-config` 推送 |
| 环境变量默认值 | `PI_AGENT_PROVIDER`、`PI_AGENT_MODEL`、`PI_AGENT_BASE_URL` 等 | 当没有更具体配置时作为兜底 |

agent-service 内部使用 `BackendProvidersConfig` 表示供应商、模型列表、active provider 和 active model。已存在 Agent 收到 Session 级配置推送后，会更新自身配置，避免下一轮对话仍使用旧模型。

---

## 五、工具权限

Pi Agent 能力由 `packages/agent-service/src/backends/pi-tools/` 统一创建。当前工具版本为 `WORKBENCH_TOOL_VERSION = 10`，可通过 `GET /api/tools/capabilities` 查看实际工具名。

工具能力包括：

- 文件读取、带行号读取、目录列表。
- 精确编辑和写入文件。
- 受限 shell 命令。
- Schema 校验。
- 图片保存和图片清单。
- 页面控制台日志和截图。
- 公开网页正文读取；该能力只读取 HTTP/HTTPS 文本页面，并拒绝本机、内网、保留地址、凭证型 URL 和过大响应。
- 画布页面整理。
- 计划审批与计划状态更新。
- 页面删除计划、预览和执行。
- 子 Agent 委派。

安全控制分为三层：

| 层级 | 机制 | 说明 |
| :--- | :--- | :--- |
| 工具权限 | `pi-tools/permissions.ts` | 控制路径、命令和工具可用性 |
| 用户确认 | `permission_request` / `permission_response` | 删除页面等高影响操作需要确认 |
| 后端校验 | `session-guard.ts` + `snapshot-service.ts` | 校验路径边界、比较变更、支持回滚 |

## 六、工作空间边界

Session 必须绑定工作空间。工作空间可能来自项目编辑临时目录，也可能来自用户指定目录。Pi Agent 工具在执行前后都会围绕这个 `workingDir` 做边界控制：

- 读取和写入路径必须解析到工作空间内。
- `.opencode`、`.git`、`node_modules` 等特殊目录由权限和快照逻辑按场景跳过或保护。
- 文件变更不依赖工具自报，最终以快照比较结果为准。
- 保存项目时由项目工作空间管理器决定如何合并到正式工作空间。

---

## 七、与历史 OpenCode 机制的区别

| 历史机制 | 当前机制 |
| :--- | :--- |
| 工作空间注入 `.opencode/opencode.json` | 不再注入 OpenCode 配置目录 |
| 通过 OpenCode agent 文件约束行为 | 通过 systemPrompt、Pi Tools 和后端权限约束 |
| 外部 OpenCode Server/CLI 执行 | Pi Agent 进程内嵌入执行 |
| 变更依赖后端 diff 事件 | 以 SnapshotService 比较工作空间变更 |
| 多后端适配器选择 | `AgentType` 固定为 `pi-agent` |

因此，排查 AI 行为时应优先检查 WebSocket 载荷、systemPrompt、backend providers、Pi Tools 能力和 Session 工作空间，而不是查找 `.opencode` 配置是否存在。

---

## 八、排查入口

| 问题 | 优先检查 |
| :--- | :--- |
| AI 使用了错误模型 | 管理后台配置、Session 级 model config、`/models` 返回 |
| AI 工具不可用 | `/api/tools/capabilities`、`WORKBENCH_TOOL_VERSION`、Pi Tools 创建选项 |
| AI 修改了错误文件 | `workingDir`、`session-store`、`session-guard` 路径校验和快照结果 |
| 页面删除未执行 | 权限确认事件、删除计划工具、用户确认响应 |
| 截图或控制台工具无结果 | screenshot-service 健康状态、前端 `console_data` 上报、`SCREENSHOT_SERVICE_URL` |
