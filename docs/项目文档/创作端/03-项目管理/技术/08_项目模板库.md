# 项目模板库

## 设计目标

项目模板库把“可继续编辑的项目副本”从普通项目列表中分离出来。普通项目仍放在项目列表里日常编辑；模板作为首页目录中的独立入口展示，用户先复制模板，再进入生成的普通项目继续编辑。

## 数据流

保存模板时，系统读取源项目的正式工作空间，将工作空间复制到模板目录，并写入模板元信息。模板元信息包含分类、名称、简介、来源项目、页面摘要和封面等用于展示与推荐的内容。

从模板创建项目时，首页收集项目名称和分类。系统复制模板工作空间到新的项目目录，然后重新生成项目身份、工作空间路径、创建时间、更新时间和项目分类。新项目与模板之间没有共享写入关系。

复制普通项目时，系统先生成临时模板快照，再按同一套项目创建流程生成新项目，最后删除临时模板快照。这样普通项目复制和模板复制使用一致的数据路径，复制后的项目仍是独立普通项目。

AI 推荐模板时，前端只提交用户描述。服务端读取模板列表，把模板摘要交给推荐服务，并校验推荐结果必须命中模板库内的模板 id。推荐失败不会转向普通项目推荐。

## 接口协作

- 模板列表接口只返回模板库内容，供首页模板目录和模板卡片展示。
- 保存模板接口挂在项目资源下，表示从某个项目创建模板快照。
- 项目创建接口保持空白创建兼容，同时接受模板 id 和项目分类作为复制创建入口。
- 项目复制接口挂在项目资源下，接收新项目名称和分类，返回新普通项目摘要。
- 推荐接口返回模板 id、推荐理由、置信度和模板摘要，前端仍需要用户确认后才创建项目。

## 交互关系

首页普通项目卡片主体负责进入该项目编辑页，更多菜单只负责触发名称、分类、封面维护，以及保存模板和复制当前项目。首页第一个卡片固定为空白项目入口；模板卡片提供复制按钮。空白创建、模板复制和普通项目复制都使用轻量名称/分类弹窗，成功后的跳转、错误提示和项目列表缓存刷新保持一致。

## 约束

- 模板是保存时刻的快照，不跟随源项目自动更新。
- 删除源项目不会删除模板。
- 模板为空时，模板选择和 AI 推荐都显示空状态。
- AI 推荐只做选择，不做内容生成。
- 历史普通项目缺少分类时，读取层统一按“未分类”展示，不做批量迁移。
