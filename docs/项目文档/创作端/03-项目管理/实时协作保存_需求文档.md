# 实时协作保存 - 需求文档

> 版本：v1.2
> 更新日期：2026-06-30
> 变更说明：补充共享 Workspace 草稿和关键动作服务端同步约束

---

## 一、模块定位

实时协作保存用于保障多人同时编辑同一个项目工作空间时，当前工作状态能持续自动保存、实时同步，并按历史记录策略形成可回退节点。

本能力补充 [会话与版本管理](./会话管理_需求文档.md) 中的多人协作部分：协作中的即时输入先成为 Workspace 当前工作状态；系统可按策略生成自动保存记录，用户也可在需要阶段留档时命名重要版本。项目版本历史的快照类型与恢复规则以 [技术/04_版本管理.md](./技术/04_版本管理.md) 为准。

## 二、功能需求

### 2.1 实时自动保存

**用户视角**：

- 用户编辑页面代码、页面配置、项目配置、页面树、画布布局或知识文档时，修改应自动保存到当前工作空间。
- 用户不需要点击保存来避免内容丢失，编辑完成后可直接关闭页面。
- 页面刷新或重新进入编辑页后，应能恢复最近一次已同步的内容。

**交互效果**：

- 编辑界面应展示当前自动保存状态，例如连接中、同步中、已自动保存、离线待同步或同步失败。
- 自动保存失败时，应明确提示用户当前修改可能没有同步完成。
- 用户离线或连接异常时，可以继续保留本地编辑状态，并在恢复连接后尝试同步。

### 2.2 多人实时协作

**用户视角**：

- 多个协作者打开同一个项目工作空间时，可以看到彼此对同一资源的编辑结果。
- 不同用户只要拥有有效会话并进入同一项目工作空间，就应共享同一个 Workspace 草稿。
- 系统应展示当前在线协作者，帮助用户判断是否有人正在共同编辑。
- 协作者切换页面或编辑不同资源时，系统应尽量展示其活跃位置。

**交互效果**：

- 同一资源的远端修改应同步到当前编辑器或画布。
- 不同资源之间应互不干扰，例如页面 A 的代码编辑不应覆盖页面 B。
- 协作者离开后，其在线状态和光标信息应自动消失。

### 2.3 历史记录与版本

**用户视角**：

- 自动保存表示当前工作状态已同步，系统还会按策略生成自动保存记录，作为普通回退兜底。
- 用户需要阶段性留档时，可在历史记录区“命名此版本”。
- 发布项目前，系统应先确保当前工作状态已同步，并自动生成发布快照。
- 导出项目包、自动检查点和恢复历史前，也应先确认 Workspace 当前态已经同步。

**交互效果**：

- “命名此版本”应表达为标记重要阶段，而不是防止内容丢失的唯一入口。
- 命名版本、自动检查点、发布、导出和恢复前应由服务端先同步当前工作状态，避免快照或产物落后于编辑器内容。
- 历史记录应区分自动保存记录、命名版本、发布快照和恢复快照；自动保存记录可弱展示或折叠。

### 2.4 退出与异常提醒

**用户视角**：

- 用户离开编辑页时，若内容已经自动保存且同步状态正常，应直接离开，不提示命名版本。
- 若最新修改仍在同步中、离线待同步、连接中或同步失败，应收到风险提示。
- 会话过期或权限失效时，应阻止继续写入协作草稿。

**交互效果**：

- 退出提示只表达同步风险，不把“尚未命名版本”作为离开前置条件。
- 用户直接离开时，当前内容仍保留在工作空间，不应将会话标记为废弃。
- 用户重新进入时，应继续基于当前工作空间内容编辑。

## 三、业务规则与约束

| 规则 | 说明 |
|:-----|:-----|
| 自动保存优先 | 实时编辑先保存为 Workspace 当前工作状态 |
| 自动检查点 | 系统按策略生成自动保存记录，作为普通回退兜底 |
| 版本命名 | 用户可手动命名重要版本，用于阶段留档 |
| 同资源协同 | 同一项目、同一工作空间、同一资源进入同一个协作上下文 |
| 共享项目工作区 | 有效会话以项目和 Workspace 为边界加入同一个项目工作区，不以单一 workspace owner 阻断协作 |
| 资源隔离 | 页面代码、页面配置、项目配置、页面树、画布布局和知识文档分别协作 |
| 非空优先 | 页面代码、页面 Schema、项目配置、页面树和画布布局已有非空工作区内容时，空协同草稿不得覆盖该内容 |
| 会话校验 | 只有有效会话可以加入协作并写入草稿 |
| 权限收敛 | 协作保存只能写入允许的项目资源，不能写入任意工作空间文件 |
| 关键动作同步 | 命名版本、自动检查点、发布、导出和恢复前必须确保当前工作空间内容已同步 |
| 发布快照 | 发布前必须生成发布快照 |
| 版本兜底 | 历史回退能力依赖自动保存记录、命名版本、发布快照和恢复快照 |
