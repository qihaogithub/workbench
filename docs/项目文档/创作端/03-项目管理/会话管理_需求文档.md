# 会话与版本管理 - 需求文档

> 版本：v3.0
> 更新日期：2026-05-03
> 变更说明：Session 与 Workspace 解耦，支持切换对话上下文而不影响工作空间

---

## 一、模块定位

会话与版本管理模块负责管理用户编辑项目时的工作空间隔离和版本历史，确保编辑操作不影响原始项目文件，同时提供版本回退能力。本模块实现以下核心设计理念：

```
"工作空间与对话解耦：切换对话只影响 AI 上下文，不影响文件状态"
```

核心概念关系：

| 概念 | 职责 | 说明 |
|:-----|:-----|:-----|
| **Workspace（工作空间）** | 文件状态的载体 | 包含 index.tsx 和 config.schema.json，同一工作空间可被多个 Session 共享 |
| **Session（对话）** | AI 交互的上下文 | 只包含对话消息，通过 workspaceId 关联到工作空间 |
| **Project（项目）** | 业务实体 | 包含正式工作空间和版本历史 |

**解耦前**：Session = 工作空间 + 对话消息 + 代码文件（切换对话 = 切换一切）
**解耦后**：Session = 对话消息；Workspace = 独立实体（切换对话仅切换 AI 上下文）

## 二、功能描述

### 2.1 工作空间隔离功能

#### 2.1.1 创建临时工作空间

**用户视角**：
- 用户打开项目编辑时，系统应自动创建独立的临时工作空间
- 临时工作空间应包含项目文件的完整副本
- 系统应自动注入 OpenCode AI 代理配置（.opencode 目录）

**交互效果**：
- 创建过程应对用户透明，无需手动操作
- 创建失败时，应提示用户并阻止进入编辑页面

#### 2.1.2 工作空间隔离

**用户视角**：
- 用户在编辑页面的所有修改仅影响临时工作空间
- 原始项目文件应保持不变，直到用户明确保存
- AI 代理应在临时工作空间中按照约束规范生成代码

**交互效果**：
- 编辑过程中，其他用户查看该项目应看到原始版本
- 多个用户同时编辑同一项目时，应各自独立，互不干扰
- 每个 Session 都有独立的 `.opencode` AI 代理配置

#### 2.1.3 保存变更

**用户视角**：
- 用户点击"保存"后，系统应将临时工作空间的内容合并到正式工作空间
- 保存操作应自动创建版本快照
- 用户可为保存添加备注说明
- 系统应自动清理 Session 专属配置（.opencode、.session.json 等）

**交互效果**：
- 保存过程中显示进度提示
- 保存成功后，应提示用户并更新项目的更新时间
- 保存失败时，应保留临时工作空间，允许用户重试
- 合并后的 Demo 目录应保持干净，不包含系统文件
- 没有未保存修改时，保存按钮应置灰，避免用户重复保存
- 存在未保存修改时，发布入口应提示先保存再发布；修改已保存但尚未发布时，发布入口应显示为"发布"
- 当前内容已经是已发布状态且没有新修改时，发布入口应置灰，避免用户重复发布

#### 2.1.4 放弃编辑

**用户视角**：
- 用户点击"放弃"后，系统应丢弃临时工作空间的所有修改
- 放弃操作不可撤销

**交互效果**：
- 放弃前应二次确认，提示未保存的修改将丢失
- 确认后，系统应清理临时工作空间并返回项目列表

### 2.2 版本管理功能

#### 2.2.1 版本历史查看

**用户视角**：
- 用户可查看项目的版本历史列表
- 版本列表应显示：版本号、保存时间、保存者、备注、文件数量

**交互效果**：
- 版本列表按时间倒序排列，最新版本在最前面
- 显示当前最新版本标识
- 底部显示版本总数（最多 50 个）

#### 2.2.2 版本恢复

**用户视角**：
- 用户可选择恢复到任意历史版本
- 恢复操作应创建新版本，而非直接覆盖

**交互效果**：
- 恢复前应二次确认
- 恢复后，系统应创建新版本（备注："从 vX 恢复"）
- 恢复成功后，项目内容应更新为指定版本

#### 2.2.3 版本数量限制

**用户视角**：
- 系统应自动管理版本数量，保留最近 50 个版本
- 超出限制时，自动删除最旧的版本

**交互效果**：
- 版本清理应在后台静默执行
- 清理过程不影响用户正常使用

---

### 2.3 多人协作功能

#### 2.3.1 用户识别

**用户视角**：
- 系统应支持用户名识别，区分不同编辑者
- 用户可在进入系统时选择或输入用户名

**交互效果**：
- 用户名仅用于区分用户，无密码验证
- 项目列表和版本历史应显示编辑者用户名

#### 2.3.2 冲突处理

**用户视角**：
- 多人同时编辑同一项目时，各自在独立临时空间操作
- 保存时采用"后保存覆盖先保存"策略

**交互效果**：
- 后保存的用户会覆盖先保存用户的修改
- 被覆盖的修改可通过版本历史恢复
- 保存时应提示用户当前版本可能已被他人更新

---

### 2.4 会话生命周期功能

#### 2.4.1 会话创建

**用户视角**：
- 用户打开项目编辑时，系统应创建新的编辑会话
- 会话应与特定项目和用户关联

**交互效果**：
- 会话创建应快速完成（< 1s）
- 会话应包含唯一标识，便于追踪和管理

#### 2.4.2 会话保持

**用户视角**：
- 用户在编辑过程中，会话应保持活跃状态
- 用户活动（如编辑代码、修改配置）应刷新会话的活跃时间

**交互效果**：
- 会话应在用户活动时自动续期
- 页面刷新后，应能恢复之前的编辑状态

#### 2.4.3 会话过期

**用户视角**：
- 用户长时间未活动，会话应自动过期
- 过期后，临时工作空间应被清理

**交互效果**：
- 会话即将过期前，应提示用户
- 用户可选择续期或保存后离开
- 过期后重新进入，应创建新会话

---

### 2.5 对话上下文切换功能

#### 2.5.1 新建对话

**用户视角**：
- 用户可在编辑页面中新建一个对话，与 AI 开始全新的交互
- 新建对话时，页面上的代码和配置应保持当前状态不变
- 新对话与当前对话共享同一个工作空间

**交互效果**：
- 新建对话后，AI 对话区清空，但代码编辑器和预览区不变
- 新对话继承当前工作空间的文件状态
- AI 在新对话中看不到之前对话的历史消息

#### 2.5.2 切换对话

**用户视角**：
- 用户可在历史对话列表中切换到之前的对话
- 切换对话时，页面上的代码和配置应保持当前状态不变
- 切换仅影响 AI 的上下文（对话消息），不影响工作空间文件

**交互效果**：
- 切换对话后，AI 对话区显示该对话的历史消息
- 代码编辑器和预览区保持不变
- AI 恢复该对话的上下文，可以继续之前的讨论

#### 2.5.3 工作空间状态独立性

**用户视角**：
- 无论在哪个对话中修改代码，修改都写入同一个工作空间
- 切换到其他对话再切回来，代码状态是工作空间的最新状态，而非该对话创建时的状态

**交互效果**：
- 代码修改实时写入工作空间文件
- 对话切换不触发代码/Schema 的重新加载

## 三、业务规则与约束

### 3.1 版本管理规则

| 规则 | 说明 |
|:-----|:-----|
| 版本数量上限 | 最多保留 50 个版本 |
| 版本号格式 | v1, v2, v3, ... |
| 恢复创建新版本 | 恢复操作创建新版本，而非覆盖 |
| 版本清理策略 | 超出限制时删除最旧版本 |

### 3.2 会话过期策略

| 配置项 | 默认值 | 说明 |
|:-------|:-------|:-----|
| 会话过期时间 | 2 小时 | 用户无活动后会话自动过期 |
| 过期前提醒 | 10 分钟 | 会话即将过期前提醒用户 |
| 最大会话时长 | 24 小时 | 单个会话的最大生命周期 |

### 3.3 并发编辑规则

| 规则 | 说明 |
|:-----|:-----|
| 多用户独立 | 多个用户可同时编辑同一项目，各自独立 |
| 单用户单会话 | 同一用户同时只能有一个活跃编辑会话 |
| 后保存覆盖 | 后保存的用户覆盖先保存用户的修改 |
| 版本兜底 | 被覆盖的修改可通过版本历史恢复 |

### 3.4 数据安全规则

| 规则 | 说明 |
|:-----|:-----|
| 原始保护 | 编辑过程中原始项目文件不可修改 |
| 原子保存 | 保存操作应原子化，避免部分保存 |
| 快照备份 | 保存前自动备份当前版本 |
| 异常恢复 | 系统异常时，临时工作空间应可恢复 |
| 配置隔离 | .opencode 配置仅限 Session 使用，不合并到正式目录 |

### 3.5 对话切换规则

| 规则 | 说明 |
|:-----|:-----|
| 切换不改代码 | 切换对话时，代码/Schema/编辑器内容保持不变 |
| 新建不改代码 | 新建对话时，代码/Schema/编辑器内容保持不变 |
| 共享工作空间 | 同一项目的多个对话共享同一个工作空间 |
| 文件实时写入 | AI 修改的代码直接写入工作空间，不依赖对话切换 |
| 对话归档 | 切换到新对话时，旧对话状态标记为 archived |

## 四、核心流程

### 4.1 打开项目编辑

```
用户打开项目
      │
      ▼
读取项目信息，获取最新版本号
      │
      ▼
创建或复用工作空间
路径：/workspaces/{userId}/{projectId}/{workspaceId}
      │
      ▼
复制项目文件到工作空间
从：{project.workspacePath}
到：{workspacePath}
      │
      ▼
注入 .opencode AI 代理配置
自动创建 opencode.json 和 agents/demo-generator.md
      │
      ▼
创建编辑会话记录（关联 workspaceId）
      │
      ▼
返回：{ sessionId, workspaceId, tempWorkspace, basedOnVersion }
```

### 4.2 保存变更

```
用户点击保存
      │
      ▼
读取会话信息和项目信息
      │
      ▼
生成新版本号 v{N+1}
      │
      ▼
备份当前正式空间（创建快照）
从：{project.workspacePath}
到：/snapshots/{projectId}/{versionId}
      │
      ▼
临时空间覆盖正式空间
从：{session.tempWorkspace}
到：{project.workspacePath}
（自动过滤 .opencode、.session.json 等系统文件）
      │
      ▼
记录版本信息到 project.versions
      │
      ▼
清理旧版本（保留最近 50 个）
      │
      ▼
标记会话为已保存
      │
      ▼
删除临时空间（包含 .opencode）
      │
      ▼
返回：{ success: true, version: "v{N}" }
```

### 4.3 恢复版本

```
用户选择恢复版本
      │
      ▼
读取项目信息和指定版本信息
      │
      ▼
生成新版本号 v{N+1}
      │
      ▼
备份当前正式空间
（创建新版本，备注："从 {versionId} 恢复"）
      │
      ▼
从快照恢复正式空间
从：/snapshots/{projectId}/{versionId}
到：{project.workspacePath}
      │
      ▼
更新项目版本历史
      │
      ▼
返回：{ newVersionId: "v{N+1}" }
```
