# 会话与版本管理 - 需求文档

> 版本：v3.3
> 更新日期：2026-06-30
> 变更说明：统一 Workspace 当前态、共享草稿和关键动作强制同步语义

---

## 一、模块定位

会话与版本管理模块负责管理用户编辑项目时的 Workspace 当前态、AI 对话上下文和版本历史，确保编辑内容持续同步，并在关键时刻形成可追溯快照。本模块实现以下核心设计理念：

```
"工作空间与对话解耦：切换对话只影响 AI 上下文，不影响文件状态"
```

核心概念关系：

| 概念 | 职责 | 说明 |
|:-----|:-----|:-----|
| **Workspace（工作空间）** | 文件状态的载体 | 承载项目当前可编辑草稿，同一工作空间可被多个用户和多个 Session 共享 |
| **Session（对话）** | AI 交互的上下文 | 只包含对话消息，通过 workspaceId 关联到工作空间 |
| **Project（项目）** | 业务实体 | 包含正式工作空间和版本历史 |

**解耦前**：Session = 工作空间 + 对话消息 + 代码文件（切换对话 = 切换一切）
**解耦后**：Session = 对话消息；Workspace = 独立实体（切换对话仅切换 AI 上下文）

## 二、功能描述

### 2.1 工作空间当前态功能

#### 2.1.1 创建或复用项目工作空间草稿

**用户视角**：
- 用户打开项目编辑时，系统应创建或复用该项目的当前工作空间草稿
- 工作空间草稿应包含项目当前可编辑文件
- 同一项目的多个协作者应进入同一个可同步的 Workspace 当前态

**交互效果**：
- 创建过程应对用户透明，无需手动操作
- 创建失败时，应提示用户并阻止进入编辑页面

#### 2.1.2 工作空间当前态同步

**用户视角**：
- 用户在编辑页面的修改应持续进入 Workspace 当前态
- 用户不需要通过“保存”按钮把编辑器内容写入草稿
- AI 代理应在当前 Workspace 约束内修改项目文件

**交互效果**：
- 编辑页、预览和版本动作读取同一个 Workspace 当前态
- 多个用户同时编辑同一项目时，应看到同一个共享草稿的最新同步结果
- 每个 Session 只隔离 AI 对话上下文，不隔离项目文件状态

#### 2.1.3 自动保存当前工作状态

**用户视角**：
- 用户在编辑页产生的修改应持续自动保存到当前工作空间
- 用户无需通过"保存"按钮防止内容丢失
- 用户刷新或重新进入编辑页后，应看到最近一次已同步的内容
- 用户可在历史记录中命名重要版本，用于阶段留档

**交互效果**：
- 编辑页顶部展示连接中、同步中、已自动保存、离线待同步和同步失败等状态
- 同步失败时，应保留当前编辑态并提示最新修改可能未保存
- 合并后的 Demo 目录应保持干净，不包含系统文件
- 没有未同步修改时，不应要求用户执行额外保存动作
- 存在未同步修改或活跃协作房间时，命名版本、自动检查点、发布、导出和恢复入口应先由服务端确认当前工作状态已同步
- 当前内容已经是已发布状态且没有新修改时，发布入口应置灰，避免用户重复发布

#### 2.1.4 离开与放弃编辑

**用户视角**：
- 用户离开编辑页时，已同步的 Workspace 当前态应保留
- 用户明确放弃本次会话时，应结束当前 AI 对话上下文
- 若需要回退文件内容，应通过历史恢复或重新创建草稿完成

**交互效果**：
- 离开前只在存在同步风险时提醒用户
- 放弃会话不应静默删除其他协作者已经同步到共享 Workspace 的内容
- 确认后，系统应返回项目列表并保留可追溯历史

### 2.2 版本管理功能

#### 2.2.1 版本历史查看

**用户视角**：
- 用户可查看项目历史记录列表
- 历史记录应包含自动保存记录、命名版本、发布快照和恢复快照
- 版本列表应显示：版本号、类型、保存时间、保存者、备注、文件数量

**交互效果**：
- 历史记录按时间倒序排列，最新记录在最前面
- 显示当前最新版本标识
- 底部显示版本总数（最多 50 个）
- 自动保存记录可折叠或弱展示，命名版本和发布快照优先展示

#### 2.2.2 版本恢复

**用户视角**：
- 用户可选择恢复到任意历史版本
- 恢复操作应创建新版本，而非直接覆盖

**交互效果**：
- 恢复前应二次确认
- 恢复后，系统应创建恢复快照，而非直接覆盖历史
- 恢复成功后，项目内容应更新为指定版本

#### 2.2.3 版本数量限制

**用户视角**：
- 系统应自动管理版本数量，保留最近 50 个版本
- 超出限制时，自动删除最旧的版本

**交互效果**：
- 版本清理应在后台静默执行
- 清理过程不影响用户正常使用

### 2.3 多人协作功能

多人协作的完整业务语义见 [实时协作保存_需求文档.md](./实时协作保存_需求文档.md)。本节仅保留与会话和版本管理直接相关的协作边界。

#### 2.3.1 用户识别

**用户视角**：
- 系统应支持用户名识别，区分不同编辑者
- 用户可在进入系统时选择或输入用户名

**交互效果**：
- 用户名仅用于区分用户，无密码验证
- 项目列表和版本历史应显示编辑者用户名

#### 2.3.2 冲突处理

**用户视角**：
- 多人同时编辑同一项目时，共享同一个 Workspace 草稿
- 实时编辑会先进入草稿，用户需要回退节点时再手动创建版本
- 不同用户编辑同一资源时，应看到彼此最新同步的草稿内容

**交互效果**：
- 同步状态应提示草稿是否已经保存到工作空间
- 创建版本前应先确保协作草稿已同步
- 已创建的历史版本仍可作为恢复和回退兜底

---

### 2.4 会话生命周期功能

#### 2.4.1 会话创建

**用户视角**：
- 用户打开项目编辑时，系统应创建新的编辑会话
- 会话应与特定项目和用户关联

**交互效果**：
- 会话创建应快速完成（< 1s）
- 会话应包含唯一标识，便于追踪和管理

#### 2.4.2 会话保持

**用户视角**：
- 用户在编辑过程中，会话应保持活跃状态
- 用户活动（如编辑代码、修改配置）应刷新会话的活跃时间

**交互效果**：
- 会话应在用户活动时自动续期
- 页面刷新后，应能恢复之前的编辑状态

#### 2.4.3 会话过期

**用户视角**：
- 用户长时间未活动，会话应自动过期
- 过期后，临时工作空间应被清理

**交互效果**：
- 会话即将过期前，应提示用户
- 用户可选择续期或保存后离开
- 过期后重新进入，应创建新会话

---

### 2.5 对话上下文切换功能

#### 2.5.1 新建对话

**用户视角**：
- 用户可在编辑页面中新建一个对话，与 AI 开始全新的交互
- 新建对话时，页面上的代码和配置应保持当前状态不变
- 新对话与当前对话共享同一个工作空间

**交互效果**：
- 新建对话后，AI 对话区清空，但代码编辑器和预览区不变
- 新对话继承当前工作空间的文件状态
- AI 在新对话中看不到之前对话的历史消息

#### 2.5.2 切换对话

**用户视角**：
- 用户可在历史对话列表中切换到之前的对话
- 切换对话时，页面上的代码和配置应保持当前状态不变
- 切换仅影响 AI 的上下文（对话消息），不影响工作空间文件

**交互效果**：
- 切换对话后，AI 对话区显示该对话的历史消息
- 代码编辑器和预览区保持不变
- AI 恢复该对话的上下文，可以继续之前的讨论

#### 2.5.3 工作空间状态独立性

**用户视角**：
- 无论在哪个对话中修改代码，修改都写入同一个工作空间
- 切换到其他对话再切回来，代码状态是工作空间的最新状态，而非该对话创建时的状态

**交互效果**：
- 代码修改实时写入工作空间文件
- 对话切换不触发代码/Schema 的重新加载

## 三、业务规则与约束

### 3.1 版本管理规则

| 规则 | 说明 |
|:-----|:-----|
| 版本数量上限 | 最多保留 50 个版本 |
| 版本号格式 | v1, v2, v3, ... |
| 恢复创建新版本 | 恢复操作创建新版本，而非覆盖 |
| 版本清理策略 | 超出限制时优先清理最旧的自动保存记录，再清理其他旧记录 |

### 3.2 会话过期策略

| 配置项 | 默认值 | 说明 |
|:-------|:-------|:-----|
| 会话过期时间 | 2 小时 | 用户无活动后会话自动过期 |
| 过期前提醒 | 10 分钟 | 会话即将过期前提醒用户 |
| 最大会话时长 | 24 小时 | 单个会话的最大生命周期 |

### 3.3 并发编辑规则

| 规则 | 说明 |
|:-----|:-----|
| 多用户共享草稿 | 多个用户可同时编辑同一项目工作空间草稿 |
| 单用户单会话 | 同一用户同时只能有一个活跃编辑会话 |
| 自动保存当前状态 | 实时编辑先保存到 Workspace 当前工作状态 |
| 自动历史检查点 | 系统按策略生成自动保存记录 |
| 命名版本 | 用户可手动命名重要版本 |
| 发布快照 | 发布前自动生成发布快照 |
| 关键动作同步 | 命名版本、自动检查点、发布、导出和恢复前必须先确认 Workspace 当前态已同步 |
| 版本兜底 | 自动保存记录、命名版本、发布快照和恢复快照均可用于回退 |

### 3.4 数据安全规则

| 规则 | 说明 |
|:-----|:-----|
| 当前态保护 | Workspace 当前态是编辑主状态源，历史快照不可直接修改 |
| 原子保存 | 保存操作应原子化，避免部分保存 |
| 快照备份 | 保存前自动备份当前版本 |
| 异常恢复 | 系统异常时，Workspace 当前态或最近历史快照应可恢复 |
| 配置隔离 | .opencode 配置仅限 Session 使用，不合并到正式目录 |

### 3.5 对话切换规则

| 规则 | 说明 |
|:-----|:-----|
| 切换不改代码 | 切换对话时，代码/Schema/编辑器内容保持不变 |
| 新建不改代码 | 新建对话时，代码/Schema/编辑器内容保持不变 |
| 共享工作空间 | 同一项目的多个对话共享同一个工作空间 |
| 文件实时写入 | AI 修改的代码直接写入工作空间，不依赖对话切换 |
| 对话归档 | 切换到新对话时，旧对话状态标记为 archived |

## 四、核心流程

### 4.1 打开项目编辑

```
用户打开项目
      │
      ▼
读取项目信息，获取最新版本号
      │
      ▼
创建或复用工作空间
路径：项目当前 Workspace
      │
      ▼
读取项目当前可编辑文件
      │
      ▼
注入 .opencode AI 代理配置
自动创建 opencode.json 和 agents/demo-generator.md
      │
      ▼
创建编辑会话记录（关联 workspaceId）
      │
      ▼
返回：{ sessionId, workspaceId, tempWorkspace, basedOnVersion }
```

### 4.2 自动保存与命名版本

```
用户编辑内容
      │
      ▼
系统自动同步到当前工作空间
      │
      ▼
用户需要阶段留档时点击“命名此版本”
      │
      ▼
系统确保当前内容已同步
      │
      ▼
生成命名版本快照
      │
      ▼
历史记录展示新的命名版本
```

### 4.3 发布

```
用户点击发布
      │
      ▼
服务端确认 Workspace 当前态已同步
      │
      ▼
生成发布快照
      │
      ▼
发布内容并记录 publishedVersion
      │
      ▼
返回发布结果
```

### 4.4 恢复版本

```
用户选择恢复版本
      │
      ▼
读取项目信息和指定版本信息
      │
      ▼
生成新版本号 v{N+1}
      │
      ▼
创建恢复前自动保存记录
      │
      ▼
从快照恢复正式空间
从：/snapshots/{projectId}/{versionId}
到：{project.workspacePath}
      │
      ▼
生成恢复快照并更新历史记录
      │
      ▼
返回：{ newVersionId: "v{N+1}" }
```
