# 标记系统

> 提供 DSLP 标记的交互式管理界面，支持资源标记和交互标记

---

## 功能概述

标记系统已从原来的侧边栏面板迁移至**预览页底部工具栏**（`PreviewToolbar`），同时在**更多面板**（`TaggingPanel`）中提供优化建议和高级设置。

```
┌─────────────────────────────────────────┐
│              预览区域                    │
├─────────────────────────────────────────┤
│ [切图] [配置项▼] [动态布局▼] [提示词▼] [导出] │  ← PreviewToolbar
└─────────────────────────────────────────┘
```

更多面板内容：
```
┌─────────────────────────────────────────┐
│  ✓ 优化建议                              │
│    - 建议将 Group 改为 Frame...           │
├─────────────────────────────────────────┤
│  ⚙ 高级设置 ▼                            │
│    - 代码格式: [React] [HTML]            │
│    - 样式细项: [Tailwind4] [图层名称]...  │
└─────────────────────────────────────────┘
```

## 标记类型

### 资源标记 (Resource Tags)

用于标记设计稿中的内容区域，代码生成时会替换为对应的 SDK 组件。

| 标记 | 格式 | 说明 | 输出组件 |
|------|------|------|----------|
| **Slot** | `#slot:type:id` | 内容插槽 | `<SdkImage />` / `<SdkText />` / `<SdkVideo />` 等 |
| **Static** | `#static` | 静态图片 | `<img>` 标签 |
| **AI Prompt** | `#prompt` | AI 指令注释 | JSX 注释 `{/* ... */}` |

#### Slot 类型详解

Slot 标记支持多种资源类型：

```
#slot:img:avatar      → <SdkImage id="avatar" />
#slot:text:title      → <SdkText id="title">...</SdkText>
#slot:video:trailer   → <SdkVideo id="trailer" />
#slot:lottie:loading  → <SdkLottie id="loading" />
#slot:svga:gift       → <SdkSvga id="gift" />
#slot:unity:game      → <SdkUnity id="game" />
#slot:color:theme     → 颜色变量引用
```

> **文本图层限制**：当节点类型为 `TEXT` 时，配置项仅支持 `text` 类型，其他类型会被过滤掉。

### 交互标记 (Interaction Tags)

用于标记具有交互行为的容器。

| 标记 | 格式 | 说明 |
|------|------|------|
| **List** | `#list:id` | 列表容器，子元素会作为列表项处理 |
| **Canvas** | `#canvas:id` | 画布容器，支持自由拖拽布局 |

### 标记互斥规则

配置项标记和动态布局标记**不能同时存在**：

- 已配置 `#slot` 时，点击动态布局按钮会弹出确认对话框
- 已配置 `#list`/`#canvas` 时，点击配置项按钮会弹出确认对话框
- 切换时自动清除已有标记

## 交互流程

### 1. 检测当前选中节点

当用户在 Figma 中选中图层时，插件会自动检测该节点已有的标记：

```
选中图层
    │
    ▼
发送 "update-selection-tags" 消息
    │
    ▼
解析节点名称，提取标记信息
    │
    ▼
更新 UI 状态（显示当前标记类型、ID 等）
    │
    ▼
触发图层健康度检查（"check-layers"）
```

### 2. 应用标记

用户在底部工具栏选择标记类型并填写 ID 后：

```
用户点击 [配置项] → 选择类型 → 输入 ID → 确定
    │
    ▼
发送 "apply-tag" 消息到插件主线程
    │
    ▼
修改选中节点的名称为 "#slot:img:avatar"
    │
    ▼
触发代码重新生成
```

### 3. 自动布局模式切换

动态布局标记支持切换节点的 Auto Layout 模式：

| 模式 | 说明 |
|------|------|
| `NONE` | 无自动布局，使用绝对定位（Canvas） |
| `HORIZONTAL` | 水平排列（List） |
| `VERTICAL` | 垂直排列（List） |

## 优化建议系统

优化建议系统基于代码生成过程中产生的警告信息，为设计师提供可操作的改进建议。

### 建议来源

优化建议由 `designerSuggestions.ts` 将技术警告转换为设计师友好的消息：

| 原始警告 | 优化建议 |
|----------|----------|
| `Layer "X" is a Group. Recommend changing to Frame...` | 建议将 Group 改为 Frame，以提升布局还原度 |
| `Non-uniform borders are only supported...` | 不支持单独描边 |
| `Some images exported as Base64 PNG` | （过滤，不显示） |
| `渲染 [X] 的 SVG 失败` | 不支持蒙版或复杂矢量 |
| `图像填充被占位符替换` | 图片资源暂未就绪，已临时使用占位图 |
| `不支持 [类型] 类型的节点` | 图层类型暂不支持直接导出 |
| `BlendMode is not supported in Text colors` | 不支持文本颜色混合模式 |
| `Unknown tag in layer "X"` | 图层标签格式无法识别 |

### 交互功能

- **定位图层**：点击建议旁的图层名称，可自动在 Figma 画布中定位到对应图层
- **Healthy 状态**：无优化建议时显示绿色 Healthy 标识

## 与代码生成的联动

标记系统的核心是**命名约定**。当用户应用标记时，实际上是在修改 Figma 节点的名称。代码生成引擎通过解析节点名称来识别标记：

```
Figma 节点名称
    │
    ├── "#slot:img:avatar" → 识别为图片插槽
    ├── "#list:products"   → 识别为列表容器
    ├── "#static"          → 识别为静态图片
    └── "#prompt"          → 识别为 AI 指令
```

这种设计的好处是：
1. **持久化**：标记信息保存在 Figma 文件中
2. **可视化**：设计师可以直接在图层列表中看到标记
3. **可迁移**：复制粘贴图层时标记信息一同保留

## Props 自动生成联动（2026-02-23）

标记系统不仅影响组件替换，也直接影响是否生成配置面板可用的 `interface Props`：

- 仅 `#slot` / `#list` 节点会被 Props 收集器处理。
- 普通命名节点（如“标题”“副标题”）不会生成 Props 字段。
- 支持手动分组覆盖语法：
  - `#slot:img:banner[Banner区域]`
  - `#list:products[商品列表]`

当联动生效时，导出日志会包含：`[PropsCollector] collected props:`。
