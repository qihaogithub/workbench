## 1. Base64 图片导出警告

### 警告原文
> "Some images exported as Base64 PNG"
> 
> "部分图片以 Base64 PNG 格式导出"

### 这是什么意思？

你的设计稿中的某些图片被直接嵌入了代码中（变成了一长串字符），而不是以外部链接的形式引用。

### 对项目有什么影响？

| 影响项 | 说明 |
|--------|------|
| 代码体积 | Base64 编码会使图片体积增加约 33% |
| 加载性能 | 首屏加载时需要下载更多数据 |
| 缓存效率 | 无法利用浏览器缓存复用图片 |
| 维护成本 | 替换图片需要重新生成代码 |

### 为什么会出现？

1. **预览模式**：在插件的"预览"标签页查看时，为了快速展示效果
2. **上传功能关闭**：未开启自动上传功能
3. **上传失败**：图片上传到服务器失败，回退为 Base64

### 设计师应该怎么做？

#### ✅ 推荐做法

1. **使用预览功能检查效果**
   - 预览模式下的 Base64 是正常的，仅用于快速查看
   - 确认效果无误后，切换到"代码"标签页获取最终代码

2. **确保开启资源上传**
   - 在插件设置中开启"自动上传资源到 CDN"
   - 这样图片会生成独立链接，代码更干净

3. **优化图片资源**
   - 检查是否有重复使用的图片
   - 压缩大体积图片（建议使用 TinyPNG 等工具）

#### ❌ 避免做法

- 不要直接在 Figma 中粘贴超大尺寸的图片
- 避免将照片类图片直接放入设计稿（应使用占位图或压缩后使用）

## 2. SVG 渲染失败警告

### 警告原文
> "渲染 [图层名] 的 SVG 失败"
> 
> "Failed to render SVG for [layer name]"

### 这是什么意思？

插件无法将你设计中的某个矢量图形转换为代码可用的 SVG 格式。

### 对项目有什么影响？

| 影响项 | 说明 |
|--------|------|
| 图标显示 | 该图形可能无法显示或显示为占位符 |
| 交互效果 | 无法添加悬停、点击等交互效果 |
| 清晰度 | 如转为位图，放大后会模糊 |

### 为什么会出现？

1. **复杂矢量路径**：包含过多锚点或复杂路径运算
2. **特殊效果**：使用了 Figma 特有的高级矢量效果
3. **蒙版/遮罩**：复杂的蒙版组合导致导出失败
4. **混合模式**：图层使用了特殊的混合模式

### 设计师应该怎么做？

#### ✅ 推荐做法

1. **简化矢量图形**
   - 将复杂图标拆分为多个简单形状
   - 使用"轮廓化"功能简化路径（Outline Stroke）

2. **使用标准图标库**
   - 优先使用 Iconfont、Heroicons 等标准图标
   - 这些图标经过优化，导出成功率高

3. **处理蒙版**
   - 将蒙版效果"扁平化"（Flatten）
   - 或使用"解组"后重新组合

4. **替代方案**
   - 如 SVG 导出困难，可将图标标记为 `#static`
   - 插件会将其作为图片导出

#### 🛠️ 具体操作步骤

```
1. 选中失败的矢量图层
2. 右键 → Flatten Selection（扁平化选择）
3. 或右键 → Outline Stroke（轮廓化描边）
4. 重新导出代码查看是否解决
```

## 3. 非统一边框警告

### 警告原文
> "Non-uniform borders are only supported with strokeAlign set to \"inside\". Will paint inside."
> 
> "非统一边框仅在描边对齐方式为'内部'时支持，将使用内部描边渲染"

### 这是什么意思？

你的设计中有元素的四个边框宽度不一致（比如左边框 2px，右边框 4px），或者边框位置不在"内部"。

### 对项目有什么影响？

| 影响项 | 说明 |
|--------|------|
| 视觉效果 | 边框可能会比设计稿中看起来略细 |
| 元素尺寸 | 整体尺寸可能会有 1-2px 的偏差 |
| 精确还原 | 复杂边框效果无法 100% 还原 |

### 为什么会出现？

1. **边框宽度不一致**：四个边的边框宽度不同
2. **边框位置**：设置为"居中"(Center) 或"外部"(Outside)
3. **设计习惯**：从 Sketch 等其他工具导入的设计稿

### 设计师应该怎么做？

#### ✅ 推荐做法

1. **统一边框宽度**
   - 尽量使用四边相同的边框宽度
   - 如需不同效果，使用多个图层叠加实现

2. **设置边框位置为"内部"**
   - 选中图层
   - 右侧面板 → Stroke → 选择 "Inside"

3. **使用背景色替代**
   - 如需单边边框效果，可用矩形图层模拟

#### 🛠️ 具体操作步骤

```
1. 选中带有边框的元素
2. 右侧面板找到 "Stroke" 区域
3. 点击边框位置图标（三个横线图标）
4. 选择最右侧的 "Inside"（内部）
5. 确保四边边框宽度一致
```

#### 📐 设计建议

| 设计效果 | 推荐实现方式 |
|----------|--------------|
| 四边相同边框 | 直接使用 Stroke，设置 Inside |
| 单边边框 | 用 1px 高度的矩形图层模拟 |
| 双边边框 | 两个嵌套 Frame，分别设置边框 |
| 虚线边框 | 目前不支持，建议用图片替代 |

## 4. 图像填充被占位符替换警告

### 警告原文
> "图像填充被占位符替换"
> 
> "Image fill replaced with placeholder"

### 这是什么意思？

设计稿中的图片无法正常导出或上传，代码中使用了灰色占位图替代。

### 对项目有什么影响？

| 影响项 | 说明 |
|--------|------|
| 视觉效果 | 图片区域显示为灰色占位图 |
| 产品演示 | 演示时缺少真实内容 |
| 开发对接 | 需要手动替换为真实图片地址 |

### 为什么会出现？

1. **网络问题**：上传服务暂时不可用
2. **图片权限**：图片受版权保护或无法访问
3. **格式不支持**：使用了特殊格式的图片
4. **尺寸过大**：超出上传限制

### 设计师应该怎么做？

#### ✅ 推荐做法

1. **检查网络连接**
   - 确保网络正常，重新尝试导出

2. **替换图片格式**
   - 将特殊格式转为 PNG 或 JPG
   - 避免使用 PSD、AI 等源文件格式

3. **压缩图片**
   - 大图片先压缩再导入 Figma
   - 推荐尺寸：单张不超过 2MB

4. **使用占位图策略**
   - 在图片名称中添加 `#slot:img:xxx` 标记
   - 代码生成后会预留图片插槽
   - 开发人员可自行替换为真实地址

#### 🛠️ 占位图标记用法

```
1. 选中图片图层
2. 修改图层名称为：#slot:img:banner
3. 导出代码后，该位置会生成：<SdkImage id="banner" />
4. 开发人员可通过配置传入真实图片地址
```

## 5. 矢量图不支持/上传失败警告

### 警告原文
> "不支持矢量图"
> 
> "矢量图自动上传失败"
> 
> "Vectors are not supported"

### 这是什么意思？

插件无法处理设计稿中的矢量图形（如复杂形状、路径等）。

### 对项目有什么影响？

| 影响项 | 说明 |
|--------|------|
| 图形显示 | 矢量图形可能缺失或显示异常 |
| 动画效果 | 无法对矢量图形添加代码动画 |
| 缩放适配 | 转为位图后缩放会失真 |

### 为什么会出现？

1. **设置限制**：插件设置中关闭了矢量图支持
2. **复杂路径**：矢量路径过于复杂
3. **上传失败**：矢量图上传服务异常

### 设计师应该怎么做？

#### ✅ 推荐做法

1. **简化矢量图形**
   - 使用 Flatten 功能扁平化复杂形状
   - 减少锚点数量

2. **转为图片**
   - 将矢量图形标记为 `#static`
   - 插件会将其作为 PNG 导出

3. **使用图标库**
   - 优先使用 Iconfont 等标准图标组件
   - 这些图标在代码中有对应组件

4. **检查设置**
   - 确认插件设置中开启了"Embed Vectors"选项

#### 🛠️ 将矢量转为图片

```
1. 选中矢量图层
2. 修改图层名称为：#static
3. 该图形会被导出为 PNG 图片
4. 代码中显示为 <img> 标签
```

## 6. 节点类型不支持警告

### 警告原文
> "不支持 [节点类型] 类型的节点"
> 
> "Node type [type] is not supported"

### 这是什么意思？

设计稿中使用了插件无法识别的特殊元素类型。

### 对项目有什么影响？

| 影响项 | 说明 |
|--------|------|
| 元素缺失 | 该元素在代码中不会显示 |
| 布局错乱 | 可能影响周围元素布局 |

### 为什么会出现？

1. **切片工具**：使用了 Slice 工具
2. **组件实例**：特殊的组件实例
3. **Figma 新功能**：使用了插件尚未支持的新功能

### 设计师应该怎么做？

#### ✅ 推荐做法

1. **替换为 Frame**
   - 将 Slice 替换为普通 Frame
   - 使用 Rectangle 替代特殊形状

2. **解组组件**
   - 将特殊组件实例解组（Detach Instance）
   - 转为普通图层后再导出

3. **标记为静态**
   - 添加 `#static` 标记作为图片导出

## 最佳实践建议

### 设计阶段

1. **优先使用 Frame 而非 Group**
   - Frame 支持自动布局和更好的代码还原

2. **规范命名**
   - 使用有意义的图层名称
   - 便于在警告中识别问题图层

3. **简化效果**
   - 避免过多叠加效果
   - 需要时扁平化处理

### 导出前检查

1. **预览模式先行**
   - 先在预览模式查看效果
   - 确认无误再导出代码

2. **处理所有警告**
   - 导出前尽量解决所有警告
   - 无法解决的添加 `#static` 标记

3. **图层结构优化**
   - 删除隐藏图层
   - 合并重复图层
