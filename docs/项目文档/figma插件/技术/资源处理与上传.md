# 资源处理与上传

> 负责将 Figma 中的图片资源导出并上传至 CDN，确保生成的代码包含可访问的远程图片地址

---

## 架构概述

资源处理系统采用**插件端 + Worker 端**分离架构：

```
┌─────────────────────────────────────────────────────────┐
│                    Figma 插件端                          │
│  ┌──────────────┐      ┌─────────────────────────────┐  │
│  │ 图片导出模块  │ ───→ │  上传队列管理器              │  │
│  │ (images.ts)  │      │  - 并发控制                  │  │
│  └──────────────┘      │  - 缓存管理                  │  │
│         │              │  - 失败重试                  │  │
│         │              └──────────────┬──────────────┘  │
│         │                             │                 │
│         │  Uint8Array / Base64         │ HTTP POST       │
│         │                             ▼                 │
│         │              ┌─────────────────────────────┐  │
│         │              │    R2 Asset Worker          │  │
│         │              │  (Cloudflare Worker)        │  │
│         └────────────→│  - 接收上传                  │  │
│                        │  - Hash 校验                 │  │
│                        │  - 写入 R2 存储              │  │
│                        │  - 返回 CDN URL              │  │
│                        └─────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

## 插件端实现

### 核心功能 (images.ts)

#### 1. 图片导出

支持多种导出方式：

| 导出方式                | 适用场景     | 输出格式      |
| ----------------------- | ------------ | ------------- |
| `exportNodeAsBytes`     | 常规图片导出 | Uint8Array    |
| `exportNodeAsBase64PNG` | 内嵌 Base64  | Base64 字符串 |
| `exportNodeAsSVG`       | 矢量图形     | SVG 字符串    |

#### 2. 缓存机制

使用内存缓存避免重复上传：

```typescript
// Cache for uploaded assets: hash -> url
const uploadedAssetsCache = new Map<string, string>();
```

缓存命中流程：

```
节点需要导出图片
    │
    ▼
计算内容 Hash
    │
    ├── 缓存命中? → 直接返回缓存的 URL
    │
    └── 缓存未命中 → 执行上传 → 存入缓存 → 返回 URL
```

#### 3. 并发控制

防止大量图片同时上传导致内存溢出：

```typescript
let activeUploads = 0;
const uploadQueue: (() => Promise<void>)[] = [];
const MAX_CONCURRENT = 5; // 默认最大并发数
```

使用队列 + 信号量模式实现并发限制：

- 新任务进入队列
- 当前活跃上传数 < 最大并发数时，取出队列任务执行
- 任务完成后，检查队列是否还有等待任务

#### 4. Hash 计算

支持两种 Hash 算法：

| 算法    | 优先级 | 说明                              |
| ------- | ------ | --------------------------------- |
| SHA-256 | 首选   | 使用 Web Crypto API，碰撞概率极低 |
| FNV-1a  | 降级   | 纯 JavaScript 实现，兼容性更好    |

### 上传配置

通过 `AssetUploadSettings` 接口配置上传行为：

```typescript
interface AssetUploadSettings {
  enableAssetUpload: boolean; // 是否启用上传
  uploadEndpoint: string; // Worker 端点
  cacheTTLHours: number; // 缓存有效期
  maxConcurrentUploads: number; // 最大并发数
  bitmapStrategy: 'upload' | 'placeholder'; // 位图策略
  vectorStrategy: 'upload' | 'fallback_rect'; // 矢量图策略
}
```

## Worker 端实现

### R2 Asset Worker

Cloudflare Worker 负责接收上传并写入 R2 对象存储。

#### 上传流程

```
POST /upload
    │
    ├── 校验 X-Upload-Secret
    │
    ├── 读取请求体 (Uint8Array)
    │
    ├── 检查 R2 是否已有相同 Hash 的文件
    │   ├── 存在 → 直接返回已有 URL
    │   └── 不存在 → 继续上传
    │
    ├── 写入 R2: `${hash}.${ext}`
    │
    └── 返回 JSON: { url, hash, size }
```

#### 请求头

| Header                 | 说明                    |
| ---------------------- | ----------------------- |
| `X-Asset-Hash`         | 文件内容 Hash，用于去重 |
| `X-Asset-Ext`          | 文件扩展名 (png/svg)    |
| `X-Asset-Content-Type` | MIME 类型               |
| `X-Upload-Secret`      | 上传密钥，防止滥用      |

#### 响应格式

```json
{
  "success": true,
  "url": "https://cdn.example.com/assets/abc123.png",
  "hash": "abc123...",
  "size": 12345
}
```

## 使用场景

### 场景 1: #static 标记

设计师将复杂装饰图层标记为 `#static`，代码生成时：

1. 识别 `#static` 标记
2. 导出该节点为 **PNG**（不支持 SVG 导出）
3. 上传至 R2 获取 CDN URL
4. 生成 `<img src="cdn-url" />` 代码

> ⚠️ **注意**：当前 `#static` 标记只导出 PNG 格式，不处理 SVG。

### 场景 2: VECTOR 节点

Figma 中的矢量图形（图标、插画等）：

**处理路径 A：embedVectors 内嵌（默认）**

1. 识别为 VECTOR 节点
2. 若 `embedVectors = true`，导出 SVG 代码直接内嵌到 JSX 中
3. 生成 `<svg>...</svg>` 代码

**处理路径 B：上传 CDN**

1. 识别为 VECTOR 节点
2. 若 `enableAssetUpload = true` 且 `embedVectors = false`，尝试上传 SVG
3. 上传成功：生成 `<img src="cdn-url" />` 代码
4. 上传失败：回退到 `embedVectors`（若开启）或矩形占位

### 场景 3: 图片填充

节点使用了图片填充 (Image Fill)：

1. 提取填充中的图片数据
2. 导出为 PNG
3. 上传获取 URL
4. 生成 `background-image: url(cdn-url)` 或 `<img>` 标签

### 场景 4: Base64 内嵌（备用方案）

当上传失败或用户选择内嵌时：

1. 导出为 Base64
2. 直接嵌入代码: `src="data:image/png;base64,..."`
3. 优点：零依赖，离线可用
4. 缺点：代码体积大，缓存不友好

## 配置说明

### embedVectors

控制矢量图形（SVG）的处理方式：

| 值      | 行为                            |
| ------- | ------------------------------- |
| `true`  | 直接内嵌 SVG 代码到 JSX（默认） |
| `false` | 尝试上传 CDN，失败则矩形占位    |

### vectorStrategy（设计文档中的配置，实际代码未实现）

设计文档中定义了 `vectorStrategy` 配置：

```typescript
interface AssetUploadSettings {
  vectorStrategy: 'upload' | 'embed' | 'fallback_rect';
}
```

**实际状态**：

- ❌ 此配置**尚未在代码中实现**
- 当前实际行为由 `embedVectors` 和 `enableAssetUpload` 组合决定

**当前实际逻辑**：

| embedVectors | enableAssetUpload | 结果                |
| ------------ | ----------------- | ------------------- |
| true         | true/false        | ❌ 内嵌 SVG，不上传 |
| false        | true              | ✅ 上传 CDN         |
| false        | false             | ⚠️ 矩形占位         |

### bitmapStrategy

控制位图（PNG/JPG）的处理方式：

| 值            | 行为             |
| ------------- | ---------------- |
| `upload`      | 上传 CDN（默认） |
| `placeholder` | 使用占位图       |

## 已知问题

### 问题 1: SVG 未上传而是内嵌

当 `embedVectors = true` 时，SVG 会被直接内嵌到代码中，完全跳过上传逻辑。

**相关 Issue**：[figma插件优化.md 问题 6](/docs/ToDo/1-Backlog/figma插件优化.md#6-svg-未上传而是内嵌到代码中)

**修复方案**：修改 `embedVectors` 逻辑，优先尝试上传，失败才降级到内嵌。

### 问题 2: #static 不支持 SVG

当前 `#static` 标记只导出 PNG 格式，不处理 SVG 矢量图。

**修复方案**：扩展 `#static` 处理逻辑，支持导出并上传 SVG。

## 错误处理

| 错误场景      | 处理策略               |
| ------------- | ---------------------- |
| 上传超时      | 降级为 Base64 内嵌     |
| Worker 不可用 | 使用占位图 + 警告提示  |
| Hash 冲突     | 信任缓存，使用已有 URL |
| 内存不足      | 限制并发数，分批处理   |

---

## 安全考虑

1. **密钥管理**：R2 密钥仅存于 Worker 环境变量，插件端不持有
2. **上传校验**：Worker 校验 `X-Upload-Secret`，防止未授权上传
3. **文件类型限制**：仅接受 png/svg 格式
4. **大小限制**：Worker 可配置单文件大小上限
