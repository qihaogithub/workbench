# API 层 - 需求文档

> 版本：v2.0
> 更新日期：2026-04-06
> 变更说明：新增版本管理接口，支持项目管理

---

## 一、模块定位

API 层是 Web 前端与后端服务、文件系统交互的核心模块，负责提供统一的接口规范，确保前后端数据交互的一致性和可靠性。本模块是系统数据流转的关键枢纽。

---

## 二、功能描述

### 2.1 项目管理接口

#### 2.1.1 获取项目列表

**用户视角**：
- 前端请求项目列表时，系统应返回所有项目的元数据
- 返回数据应包含：ID、名称、创建时间、更新时间、版本数量

**交互效果**：
- 响应时间应 < 500ms
- 列表应按更新时间倒序排列

#### 2.1.2 创建项目

**用户视角**：
- 前端提交创建请求时，系统应创建新的项目
- 创建成功后，应返回新项目的完整信息

**交互效果**：
- 名称必填，为空时应返回错误
- 创建失败时，应返回具体错误原因

#### 2.1.3 获取项目详情

**用户视角**：
- 前端请求项目详情时，系统应返回项目的完整信息
- 包括项目元数据和最新版本信息

**交互效果**：
- 项目不存在时，应返回 404 错误

#### 2.1.4 删除项目

**用户视角**：
- 前端提交删除请求时，系统应删除指定项目
- 删除操作应同时清理关联的会话和版本数据

**交互效果**：
- 项目不存在时，应返回 404 错误
- 删除失败时，应返回具体错误原因

---

### 2.2 编辑会话接口

#### 2.2.1 打开项目编辑

**用户视角**：
- 前端请求打开编辑时，系统应创建临时工作空间
- 创建成功后，应返回会话 ID 和初始文件内容

**交互效果**：
- 项目不存在时，应返回错误
- 创建过程应复制项目的完整文件结构

#### 2.2.2 获取会话文件

**用户视角**：
- 前端请求文件内容时，系统应返回临时工作空间中的文件
- 支持获取所有项目文件

**交互效果**：
- 会话不存在或已过期时，应返回错误
- 应返回文件的完整内容

#### 2.2.3 更新会话文件

**用户视角**：
- 前端提交文件更新时，系统应保存到临时工作空间
- 更新应即时生效

**交互效果**：
- 会话不存在或已过期时，应返回错误
- 更新失败时，应保留原有内容

#### 2.2.4 保存会话

**用户视角**：
- 前端请求保存时，系统应将临时空间合并到正式空间
- 保存应自动创建版本快照
- 保存成功后，应清理会话

**交互效果**：
- 保存应原子化执行
- 保存失败时，临时空间应保留

#### 2.2.5 放弃编辑

**用户视角**：
- 前端请求放弃时，系统应清理临时工作空间
- 放弃后，所有未保存的修改将丢失

**交互效果**：
- 放弃操作不可撤销

---

### 2.3 版本管理接口

#### 2.3.1 获取版本历史

**用户视角**：
- 前端请求版本历史时，系统应返回项目的版本列表
- 版本列表应包含：版本号、保存时间、保存者、备注、文件数量

**交互效果**：
- 版本列表按时间倒序排列
- 项目不存在时，应返回错误

#### 2.3.2 恢复版本

**用户视角**：
- 前端请求恢复版本时，系统应将项目恢复到指定版本
- 恢复操作应创建新版本

**交互效果**：
- 版本不存在时，应返回错误
- 恢复成功后，应返回新版本信息

---

### 2.4 AI 对话接口

#### 2.4.1 发送对话消息

**用户视角**：
- 前端发送用户消息时，系统应调用 AI 服务处理
- AI 响应应通过流式传输返回

**交互效果**：
- 支持流式响应，实时显示 AI 输出
- AI 可请求执行工具操作（如修改代码）
- 敏感工具调用需用户确认

## 三、业务规则与约束

### 3.1 响应格式规范

所有 API 应使用统一的响应格式：

**成功响应**：
```json
{
  "success": true,
  "data": { ... }
}
```

**错误响应**：
```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述",
    "details": { ... }
  }
}
```

### 3.2 错误码规范

| 错误码 | HTTP 状态码 | 说明 |
|:-------|:------------|:-----|
| PROJECT_NOT_FOUND | 404 | 项目不存在 |
| SESSION_NOT_FOUND | 404 | 会话不存在或已过期 |
| VERSION_NOT_FOUND | 404 | 版本不存在 |
| INVALID_REQUEST | 400 | 请求参数无效 |
| FILE_READ_ERROR | 500 | 文件读取失败 |
| FILE_WRITE_ERROR | 500 | 文件写入失败 |
| AI_SERVICE_ERROR | 503 | AI 服务不可用 |

### 3.3 接口安全规则

| 规则 | 说明 |
|:-----|:-----|
| 输入验证 | 所有输入参数必须验证 |
| 错误信息 | 错误信息不应暴露系统内部细节 |
| 超时处理 | 长时间操作应设置超时 |

## 四、接口列表

### 4.1 项目接口

| 方法 | 路径 | 说明 |
|:-----|:-----|:-----|
| GET | /api/projects | 获取项目列表 |
| POST | /api/projects | 创建项目 |
| GET | /api/projects/:id | 获取项目详情 |
| DELETE | /api/projects/:id | 删除项目 |

### 4.2 编辑会话接口

| 方法 | 路径 | 说明 |
|:-----|:-----|:-----|
| POST | /api/projects/:id/edit | 打开项目编辑 |
| GET | /api/sessions/:sessionId | 获取会话信息 |
| GET | /api/sessions/:sessionId/files | 获取会话文件 |
| PUT | /api/sessions/:sessionId/files | 更新会话文件 |
| POST | /api/sessions/:sessionId/save | 保存会话 |
| POST | /api/sessions/:sessionId/discard | 放弃编辑 |

### 4.3 版本管理接口

| 方法 | 路径 | 说明 |
|:-----|:-----|:-----|
| GET | /api/projects/:id/versions | 获取版本历史 |
| POST | /api/projects/:id/restore | 恢复指定版本 |

### 4.4 AI 接口

| 方法 | 路径 | 说明 |
|:-----|:-----|:-----|
| POST | /api/ai/chat | 发送对话消息 |
