## 一、技术选型

### 1.1 Route Handlers

使用 Next.js 14 App Router 的 Route Handlers 实现 REST API：

| 特性 | 说明 |
|:-----|:-----|
| 基于 Web API | 使用 Request/Response 对象 |
| TypeScript 支持 | 完整类型支持 |
| 服务端执行 | API 路由在服务端执行 |

### 1.2 文件结构

```
src/app/api/
├── projects/
│   ├── route.ts              # GET/POST /api/projects
│   └── [id]/
│       ├── route.ts          # GET/DELETE /api/projects/:id
│       ├── edit/
│       │   └── route.ts      # POST /api/projects/:id/edit
│       ├── versions/
│       │   └── route.ts      # GET /api/projects/:id/versions
│       └── restore/
│           └── route.ts      # POST /api/projects/:id/restore
├── sessions/
│   └── [sessionId]/
│       ├── route.ts          # GET /api/sessions/:id
│       ├── files/
│       │   └── route.ts      # GET/PUT /api/sessions/:id/files
│       ├── save/
│       │   └── route.ts      # POST /api/sessions/:id/save
│       └── discard/
│           └── route.ts      # POST /api/sessions/:id/discard
└── ai/
    └── chat/
        └── route.ts          # POST /api/ai/chat
```

## 二、响应格式

### 2.1 统一格式

所有 API 使用统一的响应格式：

**成功响应**：

```typescript
{
  success: true,
  data: T
}
```

**错误响应**：

```typescript
{
  success: false,
  error: {
    code: ErrorCode,
    message: string,
    details?: unknown
  }
}
```

### 2.2 辅助函数

**文件位置**：`src/lib/api-utils.ts`

```typescript
export function createApiSuccess<T>(data: T) {
  return {
    success: true as const,
    data,
  }
}

export function createApiError(
  code: ErrorCodeType,
  message?: string,
  details?: unknown
) {
  return {
    success: false as const,
    error: {
      code,
      message: message || ERROR_MESSAGES[code],
      details,
    },
  }
}
```

## 三、项目 API

### 3.1 获取项目列表

**GET** `/api/projects`

**文件位置**：`src/app/api/projects/route.ts`

**响应**：

```typescript
{
  success: true,
  data: Project[]
}
```

**实现要点**：

| 要点 | 说明 |
|:-----|:-----|
| 目录遍历 | 遍历 projects 目录 |
| 元数据提取 | 从每个项目目录提取信息 |
| 排序 | 按更新时间倒序 |

### 3.2 创建项目

**POST** `/api/projects`

**请求体**：

```typescript
{
  name: string,
  description?: string
}
```

**响应**：

```typescript
{
  success: true,
  data: Project
}
```

**实现要点**：

| 要点 | 说明 |
|:-----|:-----|
| 参数验证 | 验证 name 必填 |
| ID 生成 | 生成唯一项目 ID |
| 目录创建 | 创建项目目录和初始文件 |

### 3.3 获取项目详情

**GET** `/api/projects/:id`

**文件位置**：`src/app/api/projects/[id]/route.ts`

**响应**：

```typescript
{
  success: true,
  data: Project
}
```

### 3.4 删除项目

**DELETE** `/api/projects/:id`

**响应**：

```typescript
{
  success: true,
  data: null
}
```

**实现要点**：

| 要点 | 说明 |
|:-----|:-----|
| 存在检查 | 检查项目是否存在 |
| 目录删除 | 递归删除项目目录 |
| 关联清理 | 清理会话和快照目录 |

## 四、编辑会话 API

### 4.1 打开项目编辑

**POST** `/api/projects/:id/edit`

**请求体**：

```typescript
{
  username: string
}
```

**响应**：

```typescript
{
  success: true,
  data: {
    sessionId: string,
    files: Record<string, string>,
    basedOnVersion: string
  }
}
```

### 4.2 获取会话信息

**GET** `/api/sessions/:sessionId`

**响应**：

```typescript
{
  success: true,
  data: EditSession
}
```

### 4.3 获取会话文件

**GET** `/api/sessions/:sessionId/files`

**响应**：

```typescript
{
  success: true,
  data: {
    files: Record<string, string>
  }
}
```

### 4.4 更新会话文件

**PUT** `/api/sessions/:sessionId/files`

**请求体**：

```typescript
{
  files: Record<string, string>
}
```

### 4.5 保存会话

**POST** `/api/sessions/:sessionId/save`

**请求体**：

```typescript
{
  note?: string
}
```

**响应**：

```typescript
{
  success: true,
  data: {
    versionId: string,
    savedAt: number
  }
}
```

### 4.6 放弃编辑

**POST** `/api/sessions/:sessionId/discard`

**响应**：

```typescript
{
  success: true,
  data: null
}
```
