# 预览系统 - 实时预览机制

> 版本：v1.0
> 创建日期：2026-04-06

---

## 一、实时预览概述

### 1.1 核心机制

实时预览通过以下机制实现：

| 机制 | 说明 |
|:-----|:-----|
| 文件监听 | Sandpack 监听文件变更 |
| 增量编译 | 仅重新编译变更的文件 |
| 热更新 | 保持组件状态，仅更新变更部分 |

### 1.2 更新流程

```
代码/配置变更
      │
      ▼
触发文件更新
      │
      ▼
Sandpack 检测变更
      │
      ▼
增量编译
      │
      ▼
热更新渲染
      │
      ▼
预览区更新
```

## 二、配置联动

### 2.1 联动机制

配置变更与预览的联动流程：

```
用户修改配置表单
        │
        ▼
ConfigForm.onChange(formData)
        │
        ▼
父组件更新 configData 状态
        │
        ▼
PreviewPanel 接收新 configData
        │
        ▼
重新生成入口文件 /App.tsx
        │
        ▼
Sandpack 检测文件变更
        │
        ▼
重新编译
        │
        ▼
预览区更新渲染
```

### 2.2 性能优化

| 优化措施 | 说明 |
|:---------|:-----|
| React.memo | PreviewPanel 使用 memo 避免不必要的重渲染 |
| useCallback | 配置变更回调使用 useCallback 缓存 |
| 防抖处理 | 高频输入时添加防抖（可选） |

## 四、预览尺寸控制

### 4.1 类型定义

```typescript
interface PreviewSize {
  width?: string | number
  height?: string | number
  minHeight?: string | number
  maxHeight?: string | number
}
```

### 4.2 配置方式

在 JSON Schema 中通过 `ui:preview` 字段配置：

```json
{
  "ui:preview": {
    "width": "100%",
    "height": "600px",
    "minHeight": "400px",
    "maxHeight": "800px"
  }
}
```

### 4.3 样式生成

```typescript
function buildPreviewStyle(size?: PreviewSize): React.CSSProperties {
  if (!size) {
    return { height: '100%', minHeight: '400px', width: '100%' }
  }

  const style: React.CSSProperties = {
    width: size.width ?? '100%',
    height: size.height ?? '100%',
    minHeight: size.minHeight ?? '400px',
  }

  if (size.maxHeight !== undefined) {
    style.maxHeight = size.maxHeight
  }

  return style
}
```

## 五、刷新机制

### 5.1 手动刷新

用户可通过刷新按钮手动触发预览刷新：

```typescript
<SandpackPreview
  showRefreshButton={true}
/>
```

### 5.2 自动刷新

代码或配置变更后自动触发刷新：

| 场景 | 刷新时机 |
|:-----|:---------|
| 代码变更 | 变更后立即刷新 |
| 配置变更 | 变更后立即刷新 |
| 高频变更 | 防抖后刷新 |

## 六、错误恢复

### 6.1 错误捕获

| 错误类型 | 捕获方式 |
|:---------|:---------|
| 编译错误 | Sandpack 自动捕获 |
| 运行时错误 | Error Boundary |
| 网络错误 | try/catch |

### 6.2 恢复机制

```
错误发生
    │
    ▼
显示错误信息
    │
    ▼
用户修复代码
    │
    ▼
重新编译
    │
    ▼
恢复正常预览
```
