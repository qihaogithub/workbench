## 一、版本管理概述

### 1.1 核心功能

| 功能 | 说明 |
|:-----|:-----|
| 版本快照 | 保存时自动创建版本快照 |
| 版本历史 | 查看项目的版本历史列表 |
| 版本恢复 | 恢复到任意历史版本 |
| 版本清理 | 自动清理超出限制的旧版本 |

### 1.2 设计原则

| 原则 | 说明 |
|:-----|:-----|
| 自动创建 | 保存时自动创建版本快照，无需用户干预 |
| 不可变 | 版本快照创建后不可修改 |
| 可恢复 | 任意版本都可恢复，恢复创建新版本 |
| 数量限制 | 最多保留 50 个版本，自动清理旧版本 |

## 二、版本数据结构

### 2.1 VersionInfo

```typescript
interface VersionInfo {
  versionId: string             // 版本号（"v1", "v2", ...）
  savedAt: number               // 保存时间戳
  savedBy: string               // 保存者用户名
  sessionId: string             // 关联的编辑会话 ID
  snapshotPath: string          // 快照备份绝对路径
  fileCount: number             // 文件数量
  note?: string                 // 用户备注（可选）
}
```

### 2.2 版本号格式

```
v1, v2, v3, ..., v50, v51, ...

格式：v{N}
```

## 三、核心操作

### 3.1 创建版本快照

**触发时机**：用户保存变更时

```typescript
function createVersionSnapshot(
  projectId: string,
  sessionId: string,
  note?: string
): VersionInfo {
  const project = getProject(projectId)
  const session = getSession(sessionId)
  
  // 生成新版本号
  const versionId = generateVersionId(project)
  
  // 创建快照目录
  const snapshotPath = path.join(
    getSnapshotsRoot(),
    projectId,
    versionId
  )
  
  // 复制当前正式空间到快照
  fs.cpSync(project.workspacePath, snapshotPath, { recursive: true })
  
  // 创建版本信息
  const versionInfo: VersionInfo = {
    versionId,
    savedAt: Date.now(),
    savedBy: session.username,
    sessionId,
    snapshotPath,
    fileCount: countFiles(snapshotPath),
    note,
  }
  
  return versionInfo
}
```

### 3.2 生成版本号

```typescript
function generateVersionId(project: Project): string {
  const lastVersion = project.versions[project.versions.length - 1]
  if (!lastVersion) return 'v1'
  
  const lastNum = parseInt(lastVersion.versionId.substring(1))
  return `v${lastNum + 1}`
}
```

### 3.3 获取版本历史

```typescript
function getVersionHistory(projectId: string): VersionInfo[] {
  const project = getProject(projectId)
  return project.versions.slice().reverse()  // 按时间倒序
}
```

### 3.4 恢复版本

```typescript
function restoreVersion(
  projectId: string,
  versionId: string,
  username: string
): VersionInfo {
  const project = getProject(projectId)
  const targetVersion = project.versions.find(v => v.versionId === versionId)
  
  if (!targetVersion) {
    throw new Error(`版本 ${versionId} 不存在`)
  }
  
  // 生成新版本号
  const newVersionId = generateVersionId(project)
  
  // 备份当前正式空间
  const newSnapshotPath = path.join(
    getSnapshotsRoot(),
    projectId,
    newVersionId
  )
  fs.cpSync(project.workspacePath, newSnapshotPath, { recursive: true })
  
  // 从目标版本恢复
  fs.cpSync(
    targetVersion.snapshotPath,
    project.workspacePath,
    { recursive: true }
  )
  
  // 创建新版本信息
  const newVersion: VersionInfo = {
    versionId: newVersionId,
    savedAt: Date.now(),
    savedBy: username,
    sessionId: '',  // 恢复操作无关联会话
    snapshotPath: newSnapshotPath,
    fileCount: countFiles(project.workspacePath),
    note: `从 ${versionId} 恢复`,
  }
  
  project.versions.push(newVersion)
  updateProject(project)
  
  // 清理旧版本
  cleanupOldVersions(projectId)
  
  return newVersion
}
```

## 四、版本清理

### 4.1 清理策略

| 规则 | 说明 |
|:-----|:-----|
| 版本上限 | 最多保留 50 个版本 |
| 清理时机 | 每次保存后检查 |
| 清理对象 | 删除最旧的版本 |

### 4.2 清理实现

```typescript
const MAX_VERSIONS = 50

function cleanupOldVersions(projectId: string): void {
  const project = getProject(projectId)
  
  if (project.versions.length <= MAX_VERSIONS) return
  
  // 计算需要删除的版本数量
  const deleteCount = project.versions.length - MAX_VERSIONS
  
  // 删除最旧的版本
  const versionsToDelete = project.versions.splice(0, deleteCount)
  
  for (const version of versionsToDelete) {
    // 删除快照目录
    fs.rmSync(version.snapshotPath, { recursive: true })
  }
  
  updateProject(project)
}
```

## 五、版本快照存储

### 5.1 存储结构

```
snapshots/
└── proj_abc123/
    ├── v1/
    │   ├── index.html
    │   └── styles.css
    ├── v2/
    │   ├── index.html
    │   └── styles.css
    └── ...
```

### 5.2 存储估算

| 项目规模 | 单版本大小 | 50 版本总大小 |
|:---------|:-----------|:--------------|
| 小型（10 文件） | ~100KB | ~5MB |
| 中型（50 文件） | ~500KB | ~25MB |
| 大型（100 文件） | ~1MB | ~50MB |

## 六、版本操作流程

### 6.1 保存创建版本

```
用户点击保存
      │
      ▼
读取会话信息和项目信息
      │
      ▼
生成新版本号 v{N+1}
      │
      ▼
备份当前正式空间（创建快照）
      │
      ▼
临时空间覆盖正式空间
      │
      ▼
记录版本信息
      │
      ▼
清理旧版本（超出 50 个时）
      │
      ▼
返回版本信息
```

### 6.2 恢复版本

```
用户选择恢复版本
      │
      ▼
验证版本是否存在
      │
      ▼
生成新版本号
      │
      ▼
备份当前正式空间
（备注："从 vX 恢复"）
      │
      ▼
从快照恢复正式空间
      │
      ▼
更新项目版本历史
      │
      ▼
清理旧版本
      │
      ▼
返回新版本信息
```
