# UI Demo 可配置生成系统 - 需求文档

## 1. 项目背景

### 1.1 业务目标

打造一套\*\*"所见即所得"\*\*的可配置 Demo 生成系统，实现设计到交付的闭环：

1. **设计侧**：通过 AI 将 Figma 稿自动转换为可交互 Demo
2. **运营侧**：提供无需代码的 Web 配置台，支持实时替换素材、调整文案、预览动态效果并导出配置包
3. **系统侧**：设计规范 CMS 和活动管理系统可通过接口调用 Demo，实现设计规范的直观展示和活动页面的快速配置

### 1.2 项目必要性

传统设计交付中，UI 设计规范仅以文档和静态图形式存在。运营设计师在制作活动资源时，无法预知资源在真实代码环境下的渲染效果，导致设计意图传达不完整、资源返工率高的问题。

## 2. 痛点分析

### 2.1 设计侧痛点

| 痛点       | 描述                             |
| :------- | :----------------------------- |
| **理解偏差** | 运营设计师无法直观感知交互逻辑和动态布局，设计意图传达不完整 |
| **返工率高** | 资源上线后发现尺寸或遮挡问题，需重新设计           |

### 2.2 运营侧痛点

| 痛点        | 描述                 |
| :-------- | :----------------- |
| **配置繁琐**  | 每次活动都需要研发人员介入修改代码  |
| **反馈周期长** | 设计修改需要等待研发排期，响应速度慢 |

## 3. 用户角色

| 角色         | 职责描述     | 核心诉求                                 |
| :--------- | :------- | :----------------------------------- |
| **UI 设计师** | Demo 生产者 | 仅需简单标记 Figma 图层，即可生成高还原度 Demo，无需编写代码 |
| **运营设计师**  | Demo 使用者 | 在 Web 端配置图片/文案/视频，实时预览效果，导出配置包交付上线   |
| **研发/管理员** | 系统维护者    | 维护通用 SDK 组件库，监控 AI 生成质量，管理活动 Demo 库  |

## 4. 项目边界

### 4.1 模块划分

| 模块           | 责任方    | 说明                                       |
| :----------- | :----- | :--------------------------------------- |
| **Web 工作台**  | 本项目    | AI 工作台前端界面开发                             |
| **Figma 插件** | 其他项目组  | 导出 React 代码 + Schema，需对接本项目定义的 Schema 格式 |
| **SDK**      | 后续按需补充 | 提供给 AI 使用的组件库                            |

### 4.2 核心原则

**"插件产出骨架，AI 注入灵魂，SDK 托底能力"**

## 5. 功能需求

### 5.1 Demo 存储与管理

#### 5.1.1 Demo 目录结构

每个 Demo 以独立目录形式存储，包含以下核心文件：

| 文件            | 说明                           |
| :------------ | :--------------------------- |
| **React 组件**  | Demo 的可视化组件代码，通过 props 接收配置值 |
| **配置 Schema** | JSON Schema 格式的配置元数据定义       |
| **规则文件**      | Demo 级行为规则（可选）               |

#### 5.1.2 Demo 管理功能

| 功能      | 说明                          |
| :------ | :-------------------------- |
| Demo 列表 | 卡片形式展示所有 Demo，显示预览图、名称、更新时间 |
| 搜索 Demo | 按 Demo 名称搜索                 |
| 新建 Demo | 创建空白 Demo 目录                |
| 删除 Demo | 删除指定 Demo 目录（需二次确认）         |

### 5.2 Demo 编辑功能

#### 5.2.1 编辑工作台布局

三栏布局设计：

| 区域        | 位置     | 功能                 |
| :-------- | :----- | :----------------- |
| **用户交互区** | 左侧 1/4 | AI 对话或直接代码编辑       |
| **预览区**   | 中间 1/2 | Sandpack 实时预览 Demo |
| **配置面板**  | 右侧 1/4 | 动态配置表单             |

#### 5.2.2 AI 对话编辑

用户通过自然语言指令与 AI 交互，AI 修改当前 Demo 代码：

| 功能     | 说明                     |
| :----- | :--------------------- |
| 自然语言指令 | 用户输入指令如"把 banner 改成轮播" |
| 代码自动修改 | AI 根据指令修改 Demo 代码      |
| 实时预览   | 修改后自动刷新预览区             |

#### 5.2.3 直接代码编辑

用户可直接编辑 Demo 代码，采用统一的分隔符格式：

| 功能    | 说明                              |
| :---- | :------------------------------ |
| 分隔符格式 | 统一展示 React 组件和 JSON Schema      |
| 自动解析  | 编辑内容自动拆分到对应文件                   |
| 语法高亮  | 支持 TypeScript/React 和 JSON 语法高亮 |

### 5.3 Demo 配置功能

#### 5.3.1 配置面板

根据 JSON Schema 自动生成表单控件：

| 功能     | 说明                          |
| :----- | :-------------------------- |
| 表单自动生成 | 根据 Schema properties 自动生成表单 |
| 配置联动   | 修改配置项实时更新预览区                |
| 实时预览   | 预览区响应配置变更                   |

#### 5.3.2 支持的控件类型

| 配置类型   | 表单控件         |
| :----- | :----------- |
| 文本     | 文本输入框        |
| URL/文件 | URL 输入框、文件上传 |
| 枚举     | 下拉选择框        |
| 富文本    | 富文本编辑器       |
| 颜色     | 颜色选择器        |
| 数字     | 数字输入框        |
| 布尔值    | 开关           |
| 数组     | 数组编辑器（动态增删）  |
| 对象     | 嵌套对象表单       |

#### 5.3.3 预览尺寸控制

可在 Schema 中配置预览面板的尺寸：

| 配置项  | 说明       |
| :--- | :------- |
| 宽度   | 预览区域宽度   |
| 高度   | 预览区域高度   |
| 最小高度 | 预览区域最小高度 |
| 最大高度 | 预览区域最大高度 |

### 5.4 Demo 使用功能

#### 5.4.1 Demo 使用页面布局

左侧 2/3 为预览区，右侧 1/3 为配置面板。

#### 5.4.2 功能清单

| 功能   | 说明                     |
| :--- | :--------------------- |
| 预览区  | Sandpack 实时渲染 Demo     |
| 配置面板 | 根据 JSON Schema 自动生成的表单 |
| 配置联动 | 修改配置项实时更新预览区           |
| 编辑入口 | 点击"编辑"跳转至 AI 编辑工作台     |

### 5.5 草稿工作区功能

#### 5.5.1 工作原理

用户和 AI 不直接修改原始 Demo 文件，而是在独立的草稿工作区中编辑。

#### 5.5.2 编辑流程

| 阶段       | 操作             | 说明            |
| :------- | :------------- | :------------ |
| **进入编辑** | 复制原始 Demo 到工作区 | 为当前编辑会话创建独立副本 |
| **编辑完成** | 保存到原位置         | 将修改合并回原文件     |
| **放弃编辑** | 删除工作副本         | 零成本回滚         |

#### 5.5.3 Session 管理策略

| 策略   | 说明                     |
| :--- | :--------------------- |
| 唯一标识 | 使用 UUID 或用户 ID + 时间戳生成 |
| 过期清理 | 用户关闭页面超时自动销毁           |
| 并发编辑 | 多用户同时编辑同一 Demo 时互不干扰   |

### 5.6 历史记录功能

#### 5.6.1 功能清单

| 功能    | 说明           |
| :---- | :----------- |
| 自动保存  | 每次代码变更自动创建快照 |
| 撤销/重做 | 快捷键操作        |
| 历史浏览  | 查看完整历史记录列表   |
| 版本回退  | 回退到指定历史版本    |
| 历史清理  | 自动清理过期记录     |

#### 5.6.2 触发时机

| 触发场景  | 说明               |
| :---- | :--------------- |
| AI 修改 | AI 完成一次代码修改后自动保存 |
| 手动编辑  | 用户完成编辑后自动保存      |
| 手动保存  | 用户点击保存按钮时创建      |
| 定时保存  | 每隔固定时间间隔自动保存     |

## 6. 数据规范

### 6.1 Demo 文件格式

每个demo包含两个文件index.tsx和config.schema.json

#### 6.1.1 React 组件格式

组件通过 props 接收配置值：

| 要求       | 说明              |
| :------- | :-------------- |
| 组件结构     | 函数组件形式导出        |
| Props 接口 | 定义配置项类型         |
| 样式方案     | 使用 Tailwind CSS |

#### 6.1.2 配置 Schema 格式

使用行业标准 JSON Schema 定义配置项：

| 字段         | 说明                          |
| :--------- | :-------------------------- |
| properties | 配置项定义，与组件 props 一一对应        |
| required   | 必填配置项                       |
| default    | 默认值                         |
| 扩展字段       | 支持 ui:widget、ui:options 等扩展 |

### 6.2 Figma 导出格式

Figma 插件导出时，将 React 组件代码和配置 Schema 使用分隔符封装成单一文本，方便用户一键复制粘贴到工作台。

| 区域          | 格式                   |
| :---------- | :------------------- |
| React 组件    | TypeScript/React 代码块 |
| JSON Schema | JSON 代码块             |

## 8. 非功能需求

### 8.1 性能需求

- 预览区加载时间应在用户可接受范围内
- 配置变更到预览更新的延迟应尽可能小

### 8.2 安全需求

- AI 编辑范围应限于当前 Demo，不能影响其他 Demo
- 敏感操作（如删除）应有多重确认机制

### 8.3 可用性需求

- 操作流程简洁，减少用户学习成本
- 错误提示清晰，帮助用户理解问题
