## 一、组件架构

### 1.1 组件分层

```
┌─────────────────────────────────────────────────────────────────┐
│                         页面层 (Pages)                           │
│  HomePage / DemoUsePage / DemoEditPage                          │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                         业务组件层                               │
│  DemoCard / ConfigForm / PreviewPanel / ChatPanel               │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│                         基础组件层 (UI)                          │
│  Button / Card / Dialog / Input / Toast                         │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 组件库选型

| 组件库 | 用途 | 说明 |
|:-------|:-----|:-----|
| shadcn/ui | 基础组件 | 基于 Radix UI，可定制性强 |
| Lucide React | 图标 | 轻量、Tree-shakable |
| RJSF | 表单生成 | JSON Schema 驱动 |
| Sandpack | 代码预览 | 浏览器端执行 |

## 二、核心组件设计

### 2.1 DemoCard 组件

**文件位置**：`src/components/demo/demo-card.tsx`

**职责**：展示单个 Demo 的卡片信息

**Props 接口**：

```typescript
interface DemoCardProps {
  demo: DemoMeta
  onDelete: (demo: DemoMeta) => void
}
```

**交互设计**：
- 卡片主体可点击，跳转到使用页
- 删除按钮独立，点击触发删除确认

### 2.2 ConfigForm 组件

**文件位置**：`components/demo/ConfigForm.tsx`

**职责**：根据 Schema 生成配置表单

**Props 接口**：

```typescript
interface ConfigFormProps {
  schema: string
  onChange: (data: Record<string, unknown>) => void
  initialData?: Record<string, unknown>
  readonly?: boolean
  className?: string
}
```

**技术实现**：
- 使用 RJSF 作为表单引擎
- 自定义 Widget 扩展控件类型
- 实时触发 onChange 回调

### 2.3 PreviewPanel 组件

**文件位置**：`components/demo/PreviewPanel.tsx`

**职责**：实时预览 Demo 组件

**Props 接口**：

```typescript
interface PreviewPanelProps {
  code: string
  configData: Record<string, unknown>
  previewSize?: PreviewSize
  className?: string
}
```

**技术实现**：
- 使用 Sandpack 作为执行环境
- 动态生成入口文件注入配置
- 错误边界处理

### 2.4 ChatPanel 组件

**职责**：AI 对话交互面板

**Props 接口**：

```typescript
interface ChatPanelProps {
  sessionId: string
  onCodeChange: (code: string) => void
  onSchemaChange: (schema: string) => void
}
```

**技术实现**：
- WebSocket 连接 Agent Service
- 流式响应展示
- 工具调用确认交互

## 三、组件通信

### 3.1 配置联动机制

```
ConfigForm.onChange(formData)
        │
        ▼
父组件 setConfigData(formData)
        │
        ▼
PreviewPanel 接收新 configData
        │
        ▼
重新生成入口文件
        │
        ▼
Sandpack 重新编译渲染
```

### 3.2 编辑联动机制

```
ChatPanel 接收 AI 修改
        │
        ├──► onCodeChange(newCode)
        │          │
        │          ▼
        │    父组件 setCode(newCode)
        │          │
        │          ▼
        │    PreviewPanel 更新预览
        │
        └──► onSchemaChange(newSchema)
                   │
                   ▼
             父组件 setSchema(newSchema)
                   │
                   ▼
             ConfigForm 更新表单
```

## 四、性能优化

### 4.1 组件优化

| 优化措施 | 适用组件 | 说明 |
|:---------|:---------|:-----|
| React.memo | PreviewPanel | 避免不必要的重渲染 |
| useMemo | Schema 解析 | 缓存解析结果 |
| useCallback | 事件回调 | 缓存回调函数 |

### 4.2 渲染优化

| 优化措施 | 说明 |
|:---------|:-----|
| 虚拟列表 | Demo 列表数量大时使用 |
| 防抖处理 | 高频输入场景 |
| 懒加载 | 非关键组件延迟加载 |

## 五、错误处理

### 5.1 错误边界

在页面层级包裹 Error Boundary：

```typescript
<ErrorBoundary fallback={<ErrorFallback />}>
  <PageContent />
</ErrorBoundary>
```

### 5.2 错误类型

| 错误类型 | 处理方式 |
|:---------|:---------|
| API 错误 | Toast 提示 + 重试按钮 |
| 编译错误 | 预览区展示错误信息 |
| 运行时错误 | Error Boundary 捕获 |
