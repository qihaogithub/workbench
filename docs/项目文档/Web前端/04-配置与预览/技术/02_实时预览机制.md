## 一、实时预览概述

### 1.1 核心机制

实时预览通过以下机制实现：

| 机制 | 说明 |
|:-----|:-----|
| 服务端编译 | 通过 `/api/compile` 将 TSX 编译为 CommonJS |
| 前端执行 | 通过 `new Function` 执行编译后代码，注入项目依赖 |
| 直接渲染 | `React.createElement` 直接渲染，无需 iframe |
| 编译缓存 | 相同代码不重复编译，提升加载速度 |

### 1.2 更新流程

```
代码变更
    │
    ▼
触发 compileCode()
    │
    ▼
检查缓存 / 调用 API 编译
    │
    ▼
executeComponent() 执行
    │
    ▼
React.createElement 渲染
    │
    ▼
预览区更新
```

## 二、配置联动

### 2.1 联动机制

配置变更与预览的联动流程：

```
用户修改配置表单
        │
        ▼
ConfigForm.onChange(formData)
        │
        ▼
父组件更新 configData 状态
        │
        ▼
PreviewPanel 接收新 configData
        │
        ▼
React.createElement(compiledComponent, configData)
        │
        ▼
预览区更新渲染（无需重新编译）
```

**关键设计点**：
- 配置变更**不触发重新编译**
- 已编译的组件函数保持不变，仅更新 props
- 联动延迟 < 16ms（一帧内响应）

### 2.2 性能优化

| 优化措施 | 说明 |
|:---------|:-----|
| React.memo | PreviewPanel 使用 memo 避免不必要的重渲染 |
| useCallback | 配置变更回调使用 useCallback 缓存 |
| 编译缓存 | 相同代码不重复请求 API |

## 三、代码编辑联动

### 3.1 联动机制

代码变更与预览的联动流程：

```
用户编辑代码
        │
        ▼
Editor.onChange(newCode)
        │
        ▼
父组件更新 code 状态
        │
        ▼
PreviewPanel useEffect 触发
        │
        ▼
compileCode(code) 编译
        │
        ▼
executeComponent(compiledCode) 执行
        │
        ▼
setCompiledComponent(Component)
        │
        ▼
预览区更新渲染
```

### 3.2 竞态处理

```typescript
useEffect(() => {
  let cancelled = false;
  
  compileCode(code)
    .then((result) => {
      if (cancelled) return;  // 防止竞态
      const Component = executeComponent(result.compiledCode);
      setCompiledComponent(() => Component);
    })
    .catch((err) => {
      if (cancelled) return;
      setCompileError(err.message);
    })
    .finally(() => {
      if (!cancelled) setIsCompiling(false);
    });

  return () => { cancelled = true; };  // cleanup
}, [code]);
```

### 3.3 编辑器集成

| 编辑器 | 说明 |
|:-------|:-----|
| Monaco Editor | 可选的高级代码编辑器 |
| CodeMirror | 轻量级代码编辑器 |
| 原生 textarea | 简单场景 |

## 四、预览尺寸控制

### 4.1 类型定义

```typescript
interface PreviewSize {
  width?: string | number
  height?: string | number
  minHeight?: string | number
  maxHeight?: string | number
  scale?: number
}
```

### 4.2 配置方式

在 JSON Schema 中通过 `ui:preview` 字段配置：

```json
{
  "ui:preview": {
    "width": "100%",
    "height": "600px",
    "minHeight": "400px",
    "maxHeight": "800px"
  }
}
```

### 4.3 样式生成

```typescript
function buildPreviewStyle(size?: PreviewSize): React.CSSProperties {
  const effectiveSize = size ?? DEFAULT_PREVIEW_SIZE;

  const style: React.CSSProperties = {
    width: effectiveSize.width,
    height: effectiveSize.height,
    minHeight: effectiveSize.minHeight ?? "400px",
    margin: "0 auto",
    border: "1px solid #e5e7eb",
    borderRadius: "8px",
    overflow: "hidden",
  };

  if (effectiveSize.maxHeight !== undefined) {
    style.maxHeight = effectiveSize.maxHeight;
  }

  if (effectiveSize.scale !== undefined) {
    style.transform = `scale(${effectiveSize.scale})`;
    style.transformOrigin = "top center";
  }

  return style;
}
```

## 五、刷新机制

### 5.1 手动刷新

用户可通过刷新按钮手动触发预览刷新：

```typescript
// 清空编译缓存，强制重新编译
clearCompileCache();
```

### 5.2 自动刷新

代码或配置变更后自动触发刷新：

| 场景 | 刷新时机 |
|:-----|:---------|
| 代码变更 | 变更后立即编译并刷新 |
| 配置变更 | 变更后立即刷新（无需编译） |
| 高频变更 | 使用 React 状态更新机制自动合并 |

## 六、错误恢复

### 6.1 错误捕获

| 错误类型 | 捕获方式 |
|:---------|:---------|
| 编译错误 | `/api/compile` 返回错误信息 |
| 执行错误 | `executeComponent` try/catch |
| 运行时错误 | `PreviewErrorBoundary` |
| 网络错误 | `compileCode` fetch 异常处理 |

### 6.2 恢复机制

```
错误发生
    │
    ▼
显示错误信息（编译错误 / 运行时错误）
    │
    ▼
用户修复代码 / 修改配置
    │
    ▼
重新编译 / 重新渲染
    │
    ▼
恢复正常预览
```

### 6.3 错误展示

```tsx
// 编译错误
<div className="p-4 bg-red-50 border border-red-200 rounded-lg">
  <p className="text-red-800 font-medium">编译错误</p>
  <pre className="text-red-600 text-sm">{compileError}</pre>
</div>

// 运行时错误（ErrorBoundary 捕获）
<div className="p-4 bg-red-50 border border-red-200 rounded-lg">
  <p className="text-red-800 font-medium">渲染出错</p>
  <p className="text-red-600 text-sm">{error.message}</p>
</div>
```
