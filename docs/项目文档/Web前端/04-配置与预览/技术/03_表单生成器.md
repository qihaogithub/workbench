> 版本：v1.2
> 创建日期：2026-04-06
> 更新日期：2026-04-28

## 一、表单生成器职责

表单生成器负责根据 JSON Schema 自动生成可交互的表单控件，实现配置驱动的 UI 生成。

---

## 二、技术选型

| 文件 | 说明 |
|:-----|:-----|
| `components/demo/ConfigForm.tsx` | 表单组件 |
| `components/demo/types.ts` | 类型定义 |
| `components/demo/widgets.tsx` | 自定义控件（含 FileUploadWidget、ColorPickerWidget、RichTextWidget） |
| `components/demo/ImageListWidget.tsx` | 多图片上传控件 |

**技术选型**：使用 **react-jsonschema-form (RJSF)** 作为表单引擎，支持 JSON Schema 驱动、丰富控件扩展、实时验证。

---

## 三、组件接口

### 3.1 Props 定义

```typescript
interface ConfigFormProps {
  schema: string                           // JSON Schema 字符串
  onChange: (data: Record<string, unknown>) => void  // 配置变更回调
  initialData?: Record<string, unknown>    // 初始配置值
  readonly?: boolean                       // 只读模式
  className?: string                       // 自定义类名
}
```

### 3.2 实现要点

| 要点 | 说明 |
|:-----|:-----|
| Schema 解析 | 使用 `useMemo` 缓存解析结果 |
| 默认值提取 | 从 Schema 中提取 default 值 |
| 控件注册 | 注册自定义 Widget |
| 变更通知 | 实时触发 onChange 回调 |

## 四、控件体系总览

### 4.1 默认控件

| 数据类型 | 默认控件 | 说明 |
|:---------|:---------|:-----|
| string | text | 文本输入框 |
| number | number | 数字输入框 |
| integer | number | 数字输入框 |
| boolean | checkbox | 复选框 |
| array | array | 列表编辑器 |
| object | object | 嵌套表单 |

### 4.2 枚举控件

当 Schema 中定义 `enum` 时，自动使用下拉选择器：

```json
{
  "type": "string",
  "enum": ["option1", "option2", "option3"],
  "enumNames": ["选项一", "选项二", "选项三"]
}
```

### 4.3 内置自定义控件

通过 `ui:widget` 字段指定自定义控件：

| 控件名称 | 适用类型 | 说明 |
|:---------|:---------|:-----|
| color | string | 颜色选择器 |
| file | string | 单图片上传 |
| richtext | string | 富文本编辑器 |

## 五、内置自定义控件详解

### 5.1 FileUploadWidget — 单图片上传

```yaml
---
covers:
  - packages/web/components/demo/widgets.tsx
---
```

**文件位置**：`components/demo/widgets.tsx`

支持点击上传和拖拽上传，提供图片尺寸实时预览、文件大小限制、尺寸校验等功能。

**可配置选项**：

| 选项 | 类型 | 默认值 | 说明 |
|:-----|:-----|:-------|:-----|
| accept | string | `image/*` | 接受的文件类型 |
| maxSize | number | 50MB | 文件大小限制 |
| minWidth | number | - | 最小宽度（px） |
| minHeight | number | - | 最小高度（px） |
| maxWidth | number | - | 最大宽度（px） |
| maxHeight | number | - | 最大高度（px） |

**Schema 示例**：

```json
{
  "type": "string",
  "title": "头像图片",
  "ui:widget": "file",
  "ui:options": {
    "accept": "image/*",
    "maxSize": 5242880,
    "minWidth": 100,
    "minHeight": 100,
    "maxWidth": 2048,
    "maxHeight": 2048
  }
}
```

**交互流程**：
1. 用户点击上传区域或拖拽图片文件
2. 前端校验文件大小和图片尺寸
3. 若尺寸不符合要求，弹出确认对话框，用户可选择"取消"或"继续上传"
4. 上传成功后显示图片预览和删除按钮
5. 若替换新图片，自动删除旧的上传文件

### 5.2 ImageListWidget — 多图片上传

```yaml
---
covers:
  - packages/web/components/demo/ImageListWidget.tsx
---
```

**文件位置**：`components/demo/ImageListWidget.tsx`

支持批量上传多张图片，提供网格布局展示（4列）、图片数量限制、尺寸校验等功能。

**接口定义**：

```typescript
interface ImageItem {
  url: string;
  alt?: string;
}

interface ImageListWidgetProps {
  value: ImageItem[];
  onChange: (value: ImageItem[]) => void;
  maxItems?: number;
  title?: string;
  sessionId?: string;
  options?: {
    accept?: string;
    maxSize?: number;
    maxItems?: number;
    minWidth?: number;
    minHeight?: number;
    maxWidth?: number;
    maxHeight?: number;
  };
}
```

**Schema 示例**：

```json
{
  "type": "array",
  "title": "产品图片",
  "items": {
    "type": "object",
    "properties": {
      "url": { "type": "string" },
      "alt": { "type": "string" }
    }
  },
  "ui:options": {
    "maxItems": 20,
    "maxSize": 5242880,
    "minWidth": 200,
    "minHeight": 200
  }
}
```

## 六、组件排序控件

### 6.1 功能概述

当 Schema 中声明 `$demo.orderable` 时，配置面板顶部显示"组件排序"分组，允许用户调整子组件的排列顺序。

**触发条件**：

| 条件 | 说明 |
|:-----|:-----|
| Schema 包含 `$demo.orderable` | 必须声明可排序属性列表 |
| `orderable` 至少 2 项 | 少于 2 项无排序意义 |

### 6.2 控件交互

排序控件以箭头按钮（上下移动）实现，而非拖拽排序：

| 对比维度 | 箭头按钮 | 拖拽排序 |
|:---------|:---------|:---------|
| 发现性 | ✅ 一看就懂 | ❌ 需要发现拖拽手柄 |
| 操作精确度 | ✅ 点击即移动 | ❌ 窄空间易拖错 |
| 实现复杂度 | ✅ 无需额外依赖 | ❌ 需引入 dnd-kit |
| 无障碍性 | ✅ 键盘可操作 | ❌ 键盘难以操作 |

### 6.3 控件位置与样式

| 元素 | 样式说明 |
|:-----|:---------|
| 分组标题 | "组件排序"，带排序图标 |
| 排序项行 | 单行布局：左侧序号 + 组件标题，右侧上移/下移图标按钮 |
| 箭头按钮 | ChevronUp / ChevronDown 图标，禁用态置灰 |
| 恢复按钮 | 分组标题右侧，点击恢复默认顺序 |

### 6.4 排序状态

排序结果通过 `__order` 字段持久化：

```typescript
// 配置数据中的排序字段
{
  "header": "标题文本",
  "content": "内容文本",
  "__order": ["content", "header"]  // 排序结果
}
```

### 6.5 与表单的集成

组件排序控件与 RJSF 表单分离，单独渲染在表单顶部：

```typescript
interface ConfigFormProps {
  schema: string
  onChange: (data: Record<string, unknown>) => void
  initialData?: Record<string, unknown>
  orderable?: string[]      // 可排序属性列表
  defaultOrder?: string[]   // 默认排序顺序
}

// 渲染顺序
<>
  <OrderableSection />      {/* 排序控件，条件渲染 */}
  <Form schema={...} />     {/* RJSF 表单 */}
</>
```

## 七、验证机制

### 7.1 内置验证

RJSF 内置以下验证：

| 验证类型 | Schema 字段 |
|:---------|:------------|
| 必填验证 | `required` |
| 类型验证 | `type` |
| 最小值 | `minimum` |
| 最大值 | `maximum` |
| 最小长度 | `minLength` |
| 最大长度 | `maxLength` |
| 格式验证 | `pattern` |
| 枚举验证 | `enum` |

### 7.2 自定义验证

可通过 `customValidate` 扩展验证逻辑：

```typescript
function customValidate(formData, errors) {
  if (formData.title && formData.title.length < 3) {
    errors.title.addError('标题至少需要 3 个字符')
  }
  return errors
}
```
