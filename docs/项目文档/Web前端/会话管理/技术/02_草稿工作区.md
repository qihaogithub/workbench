## 一、草稿工作区概述

### 1.1 设计理念

```
"编辑在草稿，保存才合并，放弃零成本"
```

草稿工作区确保用户编辑操作不影响原始 Demo 文件，提供安全的编辑环境。

### 1.2 工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                         文件系统                                 │
│                                                                 │
│  demos/                         sessions/                       │
│  ├── demo-xxx/                  ├── session-xxx/                │
│  │   ├── index.tsx              │   ├── index.tsx    (副本)     │
│  │   ├── config.schema.json     │   ├── config.schema.json      │
│  │   └── ...                    │   ├── .session.json (元数据)  │
│  │                              │   └── .opencode/    (AI配置)  │
│  └── demo-yyy/                  └── session-yyy/                │
│      └── ...                        └── ...                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**说明**：
- `.opencode/` 目录在创建 Session 时自动注入
- 包含 OpenCode AI 代理配置，用于约束 AI 生成代码的行为
- 保存 Session 时自动清理，不影响原始 Demo 文件

## 二、目录结构

### 2.1 Demo 目录

```
demos/
└── demo-{id}/
    ├── index.tsx              # React 组件代码
    ├── config.schema.json     # 配置定义
    └── preview.png            # 预览图（可选）
```

### 2.2 Session 目录

```
sessions/
└── session-{timestamp}-{random}/
    ├── index.tsx              # 组件代码副本
    ├── config.schema.json     # 配置定义副本
    ├── .session.json          # 会话元数据
    ├── preview.png            # 预览图副本（可选）
    └── .opencode/             # OpenCode AI 代理配置（自动注入）
        ├── opencode.json      # OpenCode 项目配置
        └── agents/
            └── demo-generator.md  # Demo 生成代理提示词
```

**说明**：
- `.opencode/` 目录由系统自动创建，用户无需手动配置
- 包含 OpenCode AI 代理的提示词和权限配置
- 确保 AI 在 Session 中按照统一规范生成/修改代码
- 保存 Session 时自动清理，不会合并到 Demo 中

## 三、核心操作

### 3.1 创建工作区

**文件位置**：`src/lib/session-manager.ts`

**核心函数**：

```typescript
function createSession(demoId: string): SessionMeta {
  ensureDirsExist()

  const sessionId = generateSessionId()
  const sessionDir = getSessionDir(sessionId)
  const demoDir = getDemoDir(demoId)

  // 复制 Demo 文件到 Session 目录
  fs.cpSync(demoDir, sessionDir, { recursive: true })

  // 创建会话元数据
  const meta: SessionMeta = {
    sessionId,
    demoId,
    createdAt: Date.now(),
    expiresAt: Date.now() + SESSION_EXPIRY_MS,
  }

  fs.writeFileSync(
    path.join(sessionDir, '.session.json'),
    JSON.stringify(meta, null, 2)
  )

  // 注入 .opencode AI 代理配置
  injectOpencodeAgentConfig(sessionDir)

  return meta
}
```

**注入逻辑说明**：
- `injectOpencodeAgentConfig()` 函数自动创建 `.opencode` 目录及配置文件
- 包含 `opencode.json`（代理配置）和 `agents/demo-generator.md`（提示词）
- 确保 AI 在 Session 中按照统一规范生成/修改代码

### 3.2 读取工作区

```typescript
function getSessionFiles(sessionId: string): SessionFiles {
  const sessionDir = getSessionDir(sessionId)

  return {
    code: fs.readFileSync(path.join(sessionDir, 'index.tsx'), 'utf-8'),
    schema: fs.readFileSync(path.join(sessionDir, 'config.schema.json'), 'utf-8'),
  }
}
```

### 3.3 更新工作区

```typescript
function updateSessionFile(
  sessionId: string,
  filename: string,
  content: string
): void {
  const sessionDir = getSessionDir(sessionId)
  fs.writeFileSync(path.join(sessionDir, filename), content)

  // 刷新过期时间
  refreshSessionExpiry(sessionId)
}
```

### 3.4 合并到 Demo

```typescript
function mergeSessionToDemo(sessionId: string): void {
  const meta = getSessionMeta(sessionId)
  const sessionDir = getSessionDir(sessionId)
  const demoDir = getDemoDir(meta.demoId)

  // 原子化复制（.opencode 目录会被自动过滤）
  fs.cpSync(sessionDir, demoDir, {
    recursive: true,
    filter: (src) => {
      // 排除 .opencode 和 .session.json 等系统文件
      const basename = path.basename(src)
      return !basename.startsWith('.') || basename === 'preview.png'
    }
  })

  // 删除 .session.json（Demo 不需要）
  const sessionJson = path.join(demoDir, '.session.json')
  if (fs.existsSync(sessionJson)) {
    fs.unlinkSync(sessionJson)
  }

  // 删除 .opencode（Demo 不需要）
  const opencodeDir = path.join(demoDir, '.opencode')
  if (fs.existsSync(opencodeDir)) {
    fs.rmSync(opencodeDir, { recursive: true, force: true })
  }

  // 更新 Demo 元数据
  updateDemoTimestamp(meta.demoId)

  // 清理 Session
  deleteSession(sessionId)
}
```

**说明**：
- 合并时自动过滤 `.opencode` 和 `.session.json` 等系统文件
- 确保 Demo 目录保持干净，不包含 Session 专属配置

### 3.5 删除工作区

```typescript
function deleteSession(sessionId: string): void {
  const sessionDir = getSessionDir(sessionId)
  fs.rmSync(sessionDir, { recursive: true })
}
```

## 五、并发控制

### 5.1 多用户隔离

多个用户同时编辑同一 Demo 时，各自拥有独立的 Session：

```
用户 A 编辑 demo-xxx
    │
    └──► session-aaa (demo-xxx 的副本)

用户 B 编辑 demo-xxx
    │
    └──► session-bbb (demo-xxx 的另一个副本)
```

## 六、数据安全

### 6.1 原子化操作

保存操作使用原子化复制：

```typescript
function mergeSessionToDemo(sessionId: string): void {
  // 使用 fs.cpSync 的 recursive 选项实现原子化复制
  fs.cpSync(sessionDir, demoDir, { recursive: true })
}
```

### 6.2 异常恢复

系统异常时，草稿工作区可通过 Session ID 恢复：

```typescript
function recoverSession(sessionId: string): SessionMeta | null {
  const sessionDir = getSessionDir(sessionId)
  const metaPath = path.join(sessionDir, '.session.json')

  if (!fs.existsSync(metaPath)) {
    return null
  }

  return JSON.parse(fs.readFileSync(metaPath, 'utf-8'))
}
```
