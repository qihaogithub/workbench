## 一、会话生命周期概述

### 1.1 状态定义

| 状态 | 说明 |
|:-----|:-----|
| 创建中 | 会话正在初始化 |
| 活跃 | 会话正常使用中 |
| 过期 | 会话已超时，等待清理 |
| 已清理 | 会话数据已删除 |

### 1.2 状态流转

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   用户进入编辑页面                                              │
│        │                                                        │
│        ▼                                                        │
│   ┌─────────┐                                                   │
│   │  活跃   │◄────────────────────────┐                        │
│   │ (Active)│                          │                        │
│   └────┬────┘                          │                        │
│        │                               │                        │
│        ├──── 用户活动（编辑/配置）─────┘                        │
│        │                                                        │
│        ├──── 用户保存 ─────► 合并到 Demo ─────► 会话结束        │
│        │                                                        │
│        ├──── 用户放弃 ─────► 清理草稿 ─────► 会话结束           │
│        │                                                        │
│        └──── 长时间无活动 ─────► 自动过期 ─────► 自动清理       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、会话元数据

### 2.1 数据结构

```typescript
interface SessionMeta {
  sessionId: string        // Session 唯一标识
  demoId: string           // 关联的 Demo ID
  createdAt: number        // 创建时间戳
  expiresAt: number        // 过期时间戳
  opencodeSessionId?: string  // Agent 服务 Session ID（可选）
}
```

### 2.2 存储位置

会话元数据存储在草稿工作区的 `.session.json` 文件中：

```
sessions/
└── session-xxx/
    ├── index.tsx
    ├── config.schema.json
    └── .session.json    ← 会话元数据
```

## 三、会话创建

### 3.1 创建流程

```
用户进入编辑页面
        │
        ▼
POST /api/sessions { demoId }
        │
        ├── 1. 检查 Demo 是否存在
        │
        ├── 2. 生成 Session ID
        │      session-{timestamp}-{random}
        │
        ├── 3. 复制 Demo 文件到 Session 目录
        │      cp -r demos/demo-xxx sessions/session-xxx
        │
        ├── 4. 创建 Session 元数据
        │      .session.json
        │
        └── 5. 返回 Session 信息
               { sessionId, code, schema }
```

### 3.2 Session ID 生成规则

```
session-{timestamp}-{random}

示例：session-1712304000000-abc123def
```

| 部分 | 说明 |
|:-----|:-----|
| `session-` | 固定前缀 |
| `{timestamp}` | 毫秒级时间戳 |
| `{random}` | 9 位随机字符串 |

## 四、会话保持

### 4.1 活跃度检测

| 检测方式 | 说明 |
|:---------|:-----|
| 文件更新 | 更新会话文件时刷新过期时间 |
| API 调用 | 会话相关 API 调用时刷新过期时间 |
| 心跳 | 可选的定期心跳检测 |

### 4.2 过期时间计算

```typescript
const SESSION_EXPIRY_MS = 2 * 60 * 60 * 1000  // 2 小时

function calculateExpiryTime(): number {
  return Date.now() + SESSION_EXPIRY_MS
}
```

## 五、会话过期

### 5.1 过期检测

```typescript
function isSessionExpired(meta: SessionMeta): boolean {
  return Date.now() > meta.expiresAt
}
```

### 5.2 过期策略

| 配置项 | 默认值 | 说明 |
|:-------|:-------|:-----|
| 会话过期时间 | 2 小时 | 用户无活动后会话自动过期 |
| 过期前提醒 | 10 分钟 | 会话即将过期前提醒用户 |
| 最大会话时长 | 24 小时 | 单个会话的最大生命周期 |

## 六、会话清理

### 6.1 清理触发

| 触发方式 | 说明 |
|:---------|:-----|
| 定时任务 | 定期调用清理 API |
| 页面加载 | 首页加载时触发清理 |
| 手动触发 | 管理员手动调用 |

### 6.2 清理逻辑

```typescript
function cleanupExpiredSessions(): string[] {
  const sessionsDir = getSessionsDir()
  const cleaned: string[] = []

  for (const entry of fs.readdirSync(sessionsDir, { withFileTypes: true })) {
    if (!entry.isDirectory()) continue

    const metaPath = path.join(sessionsDir, entry.name, '.session.json')
    if (!fs.existsSync(metaPath)) continue

    const meta = JSON.parse(fs.readFileSync(metaPath, 'utf-8'))

    if (Date.now() > meta.expiresAt) {
      fs.rmSync(path.join(sessionsDir, entry.name), { recursive: true })
      cleaned.push(entry.name)
    }
  }

  return cleaned
}
```
