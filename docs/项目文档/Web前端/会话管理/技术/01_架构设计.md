## 一、整体架构

### 1.1 核心概念

| 概念 | 说明 |
|:-----|:-----|
| **项目 (Project)** | 一个项目 = 一个工作空间，持久化存储 |
| **工作空间 (Workspace)** | 项目的正式存储区域，Agent 操作这里 |
| **临时空间 (Temp Workspace)** | 编辑时的隔离空间，不影响原项目 |
| **会话 (Session)** | 编辑会话，关联用户、项目、临时空间 |
| **版本快照 (Snapshot)** | 保存时创建的版本备份，支持回退 |
| **代理配置 (.opencode)** | Session 中的 OpenCode AI 代理配置，自动注入 |

## 二、目录结构

### 2.1 整体结构

由于本项目是 monorepo 架构，用户数据目录放置于 `packages/web/data/` 下，与源代码同属 web 包，职责清晰。

```
packages/web/
├── data/                        # 用户数据目录
│   ├── projects/                # 项目数据
│   │   └── proj_abc123/
│   │       ├── project.json     # 项目元数据（包含版本历史）
│   │       └── workspace/       # 正式工作空间
│   │           ├── index.html
│   │           ├── styles.css
│   │           └── ...
│   │
│   ├── sessions/                # 临时编辑会话
│   │   └── proj_abc123/
│   │       └── session_1712345678/  # 临时工作空间
│   │           ├── index.html
│   │           └── ...
│   │
│   └── snapshots/               # 版本快照
│       └── proj_abc123/
│           ├── v1/
│           ├── v2/
│           ├── ...
│           └── v50/             # 最多保留 50 个版本
│
├── src/                         # 源代码
├── package.json
└── ...
```

### 2.2 项目目录

```
packages/web/data/projects/
└── proj_{id}/
    ├── project.json             # 项目元数据
    └── workspace/               # 正式工作空间
        ├── index.html
        ├── styles.css
        └── ...
```

### 2.3 会话目录

```
packages/web/data/sessions/
└── proj_{projectId}/
    └── session_{timestamp}/     # 临时工作空间
        ├── .session.json        # 会话元数据
        ├── index.html
        ├── ...
        └── .opencode/           # OpenCode AI 代理配置（自动注入）
            ├── opencode.json    # OpenCode 项目配置
            └── agents/
                └── demo-generator.md  # Demo 生成代理提示词
```

**说明**：
- `.opencode` 目录在创建 Session 时自动注入
- 用于配置 OpenCode AI 代理行为，确保生成的代码符合规范
- 保存 Session 时会被清理，不影响原始项目文件

### 2.4 快照目录

```
packages/web/data/snapshots/
└── proj_{projectId}/
    ├── v1/                      # 版本 1 快照
    ├── v2/                      # 版本 2 快照
    └── ...
```

## 三、数据结构

### 3.1 Project（项目）

```typescript
interface Project {
  id: string                    // 项目唯一标识
  name: string                  // 项目名称
  description?: string          // 项目描述
  workspacePath: string         // 正式工作空间绝对路径
  versions: VersionInfo[]       // 版本历史（最多 50 个）
  createdAt: number             // 创建时间戳
  updatedAt: number             // 最后更新时间戳
}
```

### 3.2 VersionInfo（版本信息）

```typescript
interface VersionInfo {
  versionId: string             // 版本号（"v1", "v2", ...）
  savedAt: number               // 保存时间戳
  savedBy: string               // 保存者用户名
  sessionId: string             // 关联的编辑会话 ID
  snapshotPath: string          // 快照备份绝对路径
  fileCount: number             // 文件数量
  note?: string                 // 用户备注（可选）
}
```

### 3.3 EditSession（编辑会话）

```typescript
interface EditSession {
  sessionId: string             // 会话唯一标识
  projectId: string             // 关联的项目 ID
  username: string              // 当前编辑者用户名
  tempWorkspace: string         // 临时工作空间绝对路径
  basedOnVersion: string        // 基于的版本号
  status: 'editing' | 'saved' | 'discarded'
  createdAt: number             // 创建时间戳
  expiresAt: number             // 过期时间戳
}
```

## 四、核心流程

### 4.1 打开项目编辑

```
用户打开项目
      │
      ▼
读取项目信息，获取最新版本号
      │
      ▼
创建临时工作空间目录
路径：/sessions/{projectId}/session_{timestamp}
      │
      ▼
复制项目文件到临时空间
从：{project.workspacePath}
到：{tempWorkspace}
      │
      ▼
创建编辑会话记录
      │
      ▼
返回：{ sessionId, tempWorkspace, basedOnVersion }
```

### 4.2 保存变更

```
用户点击保存
      │
      ▼
读取会话信息和项目信息
      │
      ▼
生成新版本号 v{N+1}
      │
      ▼
备份当前正式空间（创建快照）
从：{project.workspacePath}
到：/snapshots/{projectId}/{versionId}
      │
      ▼
临时空间覆盖正式空间
从：{session.tempWorkspace}
到：{project.workspacePath}
      │
      ▼
记录版本信息到 project.versions
      │
      ▼
清理旧版本（保留最近 50 个）
      │
      ▼
标记会话为已保存
      │
      ▼
删除临时空间
      │
      ▼
返回：{ success: true, version: "v{N}" }
```

### 4.3 恢复版本

```
用户选择恢复版本
      │
      ▼
读取项目信息和指定版本信息
      │
      ▼
生成新版本号 v{N+1}
      │
      ▼
备份当前正式空间
（创建新版本，备注："从 {versionId} 恢复"）
      │
      ▼
从快照恢复正式空间
从：/snapshots/{projectId}/{versionId}
到：{project.workspacePath}
      │
      ▼
更新项目版本历史
      │
      ▼
返回：{ newVersionId: "v{N+1}" }
```

## 五、冲突处理策略

### 5.1 后保存覆盖

采用最简单的冲突处理方式：后保存覆盖先保存

```
用户 A 打开项目 → 创建 session_A
用户 B 打开项目 → 创建 session_B
    │
    ▼
用户 A 保存 → 创建 v1，session_A 结束
    │
    ▼
用户 B 保存 → 创建 v2（覆盖 A 的修改），session_B 结束
    │
    ▼
A 的修改可通过 v1 恢复
```

### 5.2 安全保障

| 保障措施 | 说明 |
|:---------|:-----|
| 版本历史 | 所有保存都创建版本快照 |
| 可回退 | 被覆盖的修改可通过版本历史恢复 |
| 简单直观 | 适合非专业人员使用 |
