## 一、工作空间概述

### 1.1 设计理念

```
"一个项目 = 一个工作空间，编辑在临时空间，保存才合并"
```

工作空间隔离确保用户编辑操作不影响原始项目文件，提供安全的编辑环境。

### 1.2 空间类型

| 空间类型 | 目录 | 说明 |
|:---------|:-----|:-----|
| 正式工作空间 | `projects/{id}/workspace/` | 项目的正式存储区域 |
| 临时工作空间 | `sessions/{projectId}/{sessionId}/` | 编辑时的隔离空间 |
| 版本快照 | `snapshots/{projectId}/{versionId}/` | 版本备份 |

### 1.3 工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                         文件系统                                 │
│                                                                 │
│  projects/                      sessions/                       │
│  └── proj_xxx/                  └── proj_xxx/                   │
│      ├── project.json               ├── session_111/            │
│      └── workspace/                 │   ├── .session.json       │
│          ├── index.html             │   └── index.html (副本)   │
│          └── styles.css             │                           │
│                                      └── session_222/            │
│  snapshots/                              ├── .session.json       │
│  └── proj_xxx/                           └── index.html (副本)   │
│      ├── v1/                                                │
│      ├── v2/                                                │
│      └── ...                                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```

    └── workspace/               # 正式工作空间
```

## 三、核心操作

### 3.1 创建临时工作空间

**文件位置**：`src/lib/workspace-manager.ts`

```typescript
function createTempWorkspace(
  projectId: string,
  username: string
): EditSession {
  const sessionId = generateSessionId()
  const project = getProject(projectId)
  const tempWorkspace = path.join(
    getSessionsRoot(),
    projectId,
    sessionId
  )

  // 复制项目文件到临时空间
  fs.cpSync(project.workspacePath, tempWorkspace, { recursive: true })

  // 创建会话元数据
  const session: EditSession = {
    sessionId,
    projectId,
    username,
    tempWorkspace,
    basedOnVersion: getLatestVersion(project).versionId,
    status: 'editing',
    createdAt: Date.now(),
    expiresAt: Date.now() + SESSION_EXPIRY_MS,
  }

  fs.writeFileSync(
    path.join(tempWorkspace, '.session.json'),
    JSON.stringify(session, null, 2)
  )

  return session
}
```

### 3.4 合并到正式空间

```typescript
function mergeToWorkspace(sessionId: string, note?: string): VersionInfo {
  const session = getSession(sessionId)
  const project = getProject(session.projectId)
  
  // 生成新版本号
  const versionId = generateVersionId(project)
  
  // 创建快照
  const snapshotPath = path.join(
    getSnapshotsRoot(),
    session.projectId,
    versionId
  )
  fs.cpSync(project.workspacePath, snapshotPath, { recursive: true })
  
  // 临时空间覆盖正式空间
  fs.cpSync(session.tempWorkspace, project.workspacePath, { recursive: true })
  
  // 删除 .session.json（正式空间不需要）
  fs.unlinkSync(path.join(project.workspacePath, '.session.json'))
  
  // 记录版本信息
  const versionInfo: VersionInfo = {
    versionId,
    savedAt: Date.now(),
    savedBy: session.username,
    sessionId,
    snapshotPath,
    fileCount: countFiles(project.workspacePath),
    note,
  }
  
  project.versions.push(versionInfo)
  updateProject(project)
  
  // 清理旧版本
  cleanupOldVersions(session.projectId)
  
  // 删除临时空间
  fs.rmSync(session.tempWorkspace, { recursive: true })
  
  // 标记会话为已保存
  session.status = 'saved'
  
  return versionInfo
}
```

### 3.5 删除临时工作空间

```typescript
function deleteTempWorkspace(sessionId: string): void {
  const session = getSession(sessionId)
  fs.rmSync(session.tempWorkspace, { recursive: true })
  session.status = 'discarded'
}
```

## 五、并发控制

### 5.1 多用户隔离

多个用户同时编辑同一项目时，各自拥有独立的临时空间：

```
用户 A 编辑 proj_xxx
    │
    └──► sessions/proj_xxx/session_111/ (proj_xxx 的副本)

用户 B 编辑 proj_xxx
    │
    └──► sessions/proj_xxx/session_222/ (proj_xxx 的另一个副本)
```

### 5.2 单用户限制

同一用户同时只能有一个活跃会话：


## 六、数据安全

### 6.1 原子化操作

保存操作使用原子化复制：

```typescript
// 先创建快照，再覆盖正式空间
fs.cpSync(project.workspacePath, snapshotPath, { recursive: true })
fs.cpSync(session.tempWorkspace, project.workspacePath, { recursive: true })
```

### 6.2 异常恢复

系统异常时，临时工作空间可通过 Session ID 恢复：

```typescript
function recoverSession(sessionId: string): EditSession | null {
  const sessionDir = findSessionDir(sessionId)
  if (!sessionDir) return null

  const metaPath = path.join(sessionDir, '.session.json')
  if (!fs.existsSync(metaPath)) return null

  return JSON.parse(fs.readFileSync(metaPath, 'utf-8'))
}
```
