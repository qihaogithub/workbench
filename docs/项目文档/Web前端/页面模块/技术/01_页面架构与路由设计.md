# 页面模块 - 页面架构与路由设计

> 版本：v1.0
> 创建日期：2026-04-06

---

## 一、技术选型

### 1.1 路由框架

采用 **Next.js 14 App Router** 作为路由管理方案：

| 特性 | 说明 |
|:-----|:-----|
| 文件系统路由 | 基于目录结构自动生成路由 |
| 服务端组件 | 默认使用 Server Components |
| 动态路由 | 支持 `[id]` 动态参数 |
| 路由组 | 支持 `(group)` 分组不影响 URL |

### 1.2 渲染策略

| 页面 | 渲染方式 | 理由 |
|:-----|:---------|:-----|
| 首页 | 客户端渲染 | 需要实时搜索、交互 |
| Demo 使用页 | 客户端渲染 | 需要实时预览、配置联动 |
| Demo 编辑页 | 客户端渲染 | 需要实时编辑、AI 对话 |

## 二、路由结构

### 2.1 路由映射表

| 路由 | 文件路径 | 说明 |
|:-----|:---------|:-----|
| `/` | `src/app/page.tsx` | 首页 |
| `/demo/[id]` | `src/app/demo/[id]/page.tsx` | Demo 使用页 |
| `/demo/[id]/edit` | `src/app/demo/[id]/edit/page.tsx` | Demo 编辑页 |

### 2.2 布局结构

根布局定义在 `src/app/layout.tsx`：

```
RootLayout
├── ThemeProvider（主题管理）
├── Header（顶部导航）
└── {children}（页面内容）
```

## 三、数据获取策略

### 3.1 客户端数据获取

使用 **SWR** 进行数据获取和缓存：

| 场景 | Hook | 说明 |
|:-----|:-----|:-----|
| Demo 列表 | `useDemos()` | 获取所有 Demo |
| Demo 详情 | `useDemo(id)` | 获取单个 Demo |
| Session 数据 | `useSession(sessionId)` | 获取会话数据 |

### 3.2 数据流设计

```
用户操作
    │
    ▼
SWR Hook 触发
    │
    ▼
API Client 发送请求
    │
    ▼
Route Handler 处理
    │
    ▼
文件系统 / Agent Service
    │
    ▼
响应返回
    │
    ▼
SWR 更新缓存
    │
    ▼
UI 重新渲染
```

## 四、状态管理

### 4.1 状态分类

| 类型 | 管理方式 | 示例 |
|:-----|:---------|:-----|
| 服务端状态 | SWR | Demo 列表、Session 数据 |
| UI 状态 | React State | 对话框开关、搜索关键词 |
| 表单状态 | RJSF 内置 | 配置表单数据 |

### 4.2 状态流转

```
┌─────────────────────────────────────────────────────────────────┐
│                         首页状态                                 │
│                                                                 │
│  demos (SWR) ─────► Demo 列表数据                               │
│  searchQuery ─────► 搜索关键词                                  │
│  isCreateDialogOpen ─────► 创建对话框状态                       │
│  deleteTarget ─────► 待删除 Demo                                │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                      Demo 使用页状态                             │
│                                                                 │
│  demo (SWR) ─────► Demo 元数据                                  │
│  session (SWR) ─────► Session 数据                              │
│  code ─────► 组件代码                                           │
│  schema ─────► 配置定义                                         │
│  configData ─────► 配置值                                       │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                      Demo 编辑页状态                             │
│                                                                 │
│  session (SWR) ─────► Session 数据                              │
│  code ─────► 组件代码（可编辑）                                  │
│  schema ─────► 配置定义（可编辑）                                │
│  configData ─────► 配置值                                       │
│  messages ─────► AI 对话历史                                    │
│  isStreaming ─────► AI 响应状态                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 五、页面加载流程

### 5.1 首页加载

```
1. 页面组件挂载
      │
      ▼
2. useDemos() 发起请求
      │
      ▼
3. GET /api/demos
      │
      ▼
4. 返回 Demo 列表
      │
      ▼
5. SWR 更新缓存
      │
      ▼
6. 页面渲染 DemoCard 列表
```

### 5.2 Demo 使用页加载

```
1. 页面组件挂载，获取路由参数 id
      │
      ▼
2. useDemos() 获取 Demo 名称
      │
      ▼
3. POST /api/sessions 创建临时 Session
      │
      ▼
4. GET /api/sessions/:sessionId/files 获取文件
      │
      ▼
5. 解析 code 和 schema
      │
      ▼
6. getDefaultValues(schema) 提取默认配置
      │
      ▼
7. 渲染预览区和配置面板
      │
      ▼
8. 用户离开时 DELETE /api/sessions/:sessionId
```

### 5.3 Demo 编辑页加载

```
1. 页面组件挂载，获取路由参数 id
      │
      ▼
2. POST /api/sessions 创建编辑 Session
      │
      ▼
3. GET /api/sessions/:sessionId/files 获取文件
      │
      ▼
4. 初始化编辑器内容
      │
      ▼
5. 建立 WebSocket 连接（AI 对话）
      │
      ▼
6. 渲染编辑工作台
```
