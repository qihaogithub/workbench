# 配置系统 - 表单生成器

> 版本：v1.0
> 创建日期：2026-04-06

---

## 一、表单生成器职责

表单生成器负责根据 JSON Schema 自动生成可交互的表单控件，实现配置驱动的 UI 生成。

---

## 二、技术实现

### 2.1 技术选型

使用 **react-jsonschema-form (RJSF)** 作为表单引擎：

| 特性 | 说明 |
|:-----|:-----|
| JSON Schema 驱动 | 根据 Schema 自动生成表单 |
| 丰富的控件 | 支持多种数据类型 |
| 自定义控件 | 可扩展 Widget |
| 实时验证 | 表单填写时校验 |

### 2.2 文件位置

| 文件 | 说明 |
|:-----|:-----|
| `components/demo/ConfigForm.tsx` | 表单组件 |
| `components/demo/types.ts` | 类型定义 |
| `components/demo/widgets.tsx` | 自定义控件 |

---

## 三、组件接口

### 3.1 Props 定义

```typescript
interface ConfigFormProps {
  schema: string                           // JSON Schema 字符串
  onChange: (data: Record<string, unknown>) => void  // 配置变更回调
  initialData?: Record<string, unknown>    // 初始配置值
  readonly?: boolean                       // 只读模式
  className?: string                       // 自定义类名
}
```

### 3.2 组件实现要点

| 要点 | 说明 |
|:-----|:-----|
| Schema 解析 | 使用 `useMemo` 缓存解析结果 |
| 默认值提取 | 从 Schema 中提取 default 值 |
| 控件注册 | 注册自定义 Widget |
| 变更通知 | 实时触发 onChange 回调 |

## 四、控件类型映射

### 4.1 默认控件

| 数据类型 | 默认控件 | 说明 |
|:---------|:---------|:-----|
| string | text | 文本输入框 |
| number | number | 数字输入框 |
| integer | number | 数字输入框 |
| boolean | checkbox | 复选框 |
| array | array | 列表编辑器 |
| object | object | 嵌套表单 |

### 4.2 枚举控件

当 Schema 中定义 `enum` 时，自动使用下拉选择器：

```json
{
  "type": "string",
  "enum": ["option1", "option2", "option3"],
  "enumNames": ["选项一", "选项二", "选项三"]
}
```

### 4.3 自定义控件

通过 `ui:widget` 字段指定自定义控件：

```json
{
  "type": "string",
  "title": "背景颜色",
  "ui:widget": "color"
}
```

| 控件名称 | 适用类型 | 说明 |
|:---------|:---------|:-----|
| color | string | 颜色选择器 |
| code | string | 代码编辑器 |
| textarea | string | 多行文本框 |

## 六、验证机制

### 6.1 内置验证

RJSF 内置以下验证：

| 验证类型 | Schema 字段 |
|:---------|:------------|
| 必填验证 | `required` |
| 类型验证 | `type` |
| 最小值 | `minimum` |
| 最大值 | `maximum` |
| 最小长度 | `minLength` |
| 最大长度 | `maxLength` |
| 格式验证 | `pattern` |
| 枚举验证 | `enum` |

### 6.2 自定义验证

可通过 `customValidate` 扩展验证逻辑：

```typescript
function customValidate(formData, errors) {
  if (formData.title && formData.title.length < 3) {
    errors.title.addError('标题至少需要 3 个字符')
  }
  return errors
}
```
