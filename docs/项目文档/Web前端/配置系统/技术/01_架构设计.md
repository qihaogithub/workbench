## 二、技术选型

### 2.1 表单引擎

采用 **react-jsonschema-form (RJSF)** 作为表单生成引擎：

| 特性 | 说明 |
|:-----|:-----|
| JSON Schema 驱动 | 根据 Schema 自动生成表单 |
| 丰富的控件 | 支持多种数据类型 |
| 自定义控件 | 可扩展 Widget |
| 实时验证 | 表单填写时校验 |

### 2.2 解析器

自定义解析器处理 Figma 导出格式：

| 功能 | 说明 |
|:-----|:-----|
| 分隔符识别 | 识别 `=== DEMO CODE ===` 等分隔符 |
| 内容提取 | 提取代码和 Schema 两部分 |
| 格式修复 | 自动修复常见格式问题 |

## 三、核心模块

### 3.1 Schema 解析器

**文件位置**：`lib/parser.ts`

**核心函数**：

| 函数 | 说明 |
|:-----|:-----|
| `parseFigmaText(text)` | 解析分隔符格式文本 |
| `buildFigmaText(code, schema)` | 构建分隔符格式文本 |
| `isValidFigmaFormat(text)` | 验证格式有效性 |
| `fixFigmaTextFormat(text)` | 修复格式问题 |

### 3.2 表单生成器

**文件位置**：`components/demo/ConfigForm.tsx`

**核心功能**：

| 功能 | 说明 |
|:-----|:-----|
| Schema 解析 | 将 JSON 字符串解析为对象 |
| 表单生成 | 根据 Schema 生成表单控件 |
| 自定义控件 | 注册自定义 Widget |
| 值变更通知 | 触发 onChange 回调 |

### 3.3 配置验证器

**文件位置**：`lib/validator.ts`

**核心函数**：

| 函数 | 说明 |
|:-----|:-----|
| `validateCode(code)` | 验证代码有效性 |
| `validateSchema(schema)` | 验证 Schema 有效性 |
| `validateConfig(schema, config)` | 验证配置值 |

## 四、扩展机制

### 4.1 自定义控件

通过 RJSF 的 Widget 机制扩展控件：

| 控件 | 文件位置 | 说明 |
|:-----|:---------|:-----|
| 颜色选择器 | `widgets.tsx` | 可视化选择颜色 |
| 代码编辑器 | `widgets.tsx` | 编辑代码片段 |

### 4.2 控件注册

```typescript
const widgets = {
  color: ColorWidget,
  code: CodeWidget,
}

<Form
  schema={parsedSchema}
  widgets={widgets}
  onChange={handleChange}
/>
```
