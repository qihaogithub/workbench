## 一、概述

本文档定义 Agent 服务层的所有对外接口，包括 REST API、WebSocket 协议和类型定义。

### 1.1 基础信息

| 项目     | 值                              |
| :------- | :------------------------------ |
| 服务地址 | `http://localhost:3201`（默认） |
| API 前缀 | `/api`                          |
| 协议版本 | `v1`                            |
| 内容类型 | `application/json`              |

### 1.2 通用响应格式

所有 API 响应采用统一的格式：

**成功响应**：包含 `success: true` 和 `data` 字段

**错误响应**：包含 `success: false` 和 `error` 对象（含 `code`、`message`、`details` 字段）

**类型定义文件**：[core/types.ts](../packages/agent-service/src/core/types.ts)

## 二、REST API

### 2.1 发送消息

**POST** `/api/agent/:sessionId/message`

向指定 Session 发送消息，返回 AI 响应。

**请求参数**：

| 字段              | 类型      | 必填 | 说明                                                 |
| :---------------- | :-------- | :--- | :--------------------------------------------------- |
| `content`         | string    | 是   | 消息内容                                             |
| `demoId`          | string    | 否   | Demo ID（首次创建时需要）                            |
| `backend`         | AgentType | 否   | 后端类型，固定为 'pi-agent'                          |
| `workingDir`      | string    | 否   | 工作目录（不指定则创建临时工作空间）                 |
| `customWorkspace` | boolean   | 否   | 是否为用户指定工作空间，默认根据 workingDir 自动推断 |
| `options.timeout` | number    | 否   | 超时时间（毫秒），默认 120000                        |
| `options.stream`  | boolean   | 否   | 是否流式响应，默认 false                             |

**响应字段**：

| 字段                | 类型         | 说明                       |
| :------------------ | :----------- | :------------------------- |
| `sessionId`         | string       | Session ID                 |
| `opencodeSessionId` | string       | OpenCode 后端的 Session ID |
| `content`           | string       | AI 响应内容                |
| `files`             | FileChange[] | 变更的文件列表             |
| `metadata.model`    | string       | 使用的模型                 |
| `metadata.tokens`   | object       | Token 使用统计             |

**路由实现**：[routes/agent.ts](../packages/agent-service/src/routes/agent.ts)

**GET** `/api/agent/:sessionId`

**响应字段**：

| 字段             | 类型        | 说明                 |
| :--------------- | :---------- | :------------------- |
| `sessionId`      | string      | Session ID           |
| `status`         | AgentStatus | 当前状态             |
| `backend`        | AgentType   | 后端类型             |
| `createdAt`      | string      | 创建时间（ISO 8601） |
| `lastActivityAt` | string      | 最后活动时间         |
| `messageCount`   | number      | 消息计数             |
| `workingDir`     | string      | 工作目录             |

**状态枚举**：`initializing` | `ready` | `processing` | `error` | `destroyed`

### 2.4 获取变更文件

**GET** `/api/agent/:sessionId/files`

获取当前 Session 中变更的文件列表，支持 Git 仓库和普通目录两种模式。

**查询参数**：

| 参数             | 类型    | 必填 | 说明                         |
| :--------------- | :------ | :--- | :--------------------------- |
| `includeContent` | boolean | 否   | 是否包含文件内容，默认 false |

**响应字段**：

| 字段        | 类型             | 说明                     |
| :---------- | :--------------- | :----------------------- |
| `sessionId` | string           | Session ID               |
| `files`     | FileChangeInfo[] | 所有变更文件列表         |
| `staged`    | FileChangeInfo[] | 已暂存的变更（Git 模式） |
| `unstaged`  | FileChangeInfo[] | 未暂存的变更             |

**FileChangeInfo 结构**：

| 字段        | 类型   | 说明                           |
| :---------- | :----- | :----------------------------- |
| `path`      | string | 文件路径（相对于工作空间）     |
| `operation` | string | 操作类型：create/modify/delete |
| `status`    | string | 变更状态：staged/unstaged      |

---

## 三、WebSocket API

### 3.1 连接地址

```
ws://localhost:3201/api/agent/:sessionId/stream
```

**实现文件**：[routes/websocket.ts](../packages/agent-service/src/routes/websocket.ts)

**事件路由**：[routes/ws-event-router.ts](../packages/agent-service/src/routes/ws-event-router.ts)

### 3.2 消息格式

所有消息均为 JSON 格式。

#### 客户端 → 服务端

| 消息类型  | 字段                       | 说明         |
| :-------- | :------------------------- | :----------- |
| `message` | `id`, `content`, `options` | 发送消息     |
| `cancel`  | `id`                       | 取消当前请求 |
| `ping`    | `timestamp`                | 心跳         |

#### 服务端 → 客户端

| 消息类型           | 字段                                          | 说明         |
| :----------------- | :-------------------------------------------- | :----------- |
| `stream`           | `id`, `content`, `done`                       | 流式内容     |
| `thought`          | `id`, `content`, `done`                       | 思考过程     |
| `tool_call`        | `id`, `toolCallId`, `status`, `title`, `kind` | 工具调用     |
| `tool_call_update` | `id`, `toolCallId`, `status`                  | 工具调用更新 |
| `plan`             | `id`, `content`                               | 计划内容     |
| `error`            | `id`, `error.code`, `error.message`           | 错误         |
| `finish`           | `id`, `files`, `metadata`                     | 完成         |
| `pong`             | `timestamp`                                   | 心跳响应     |
| `status`           | `status`                                      | 状态变更     |

### 3.3 连接生命周期

```
┌─────────┐     connect      ┌─────────┐
│ Client  │ ───────────────► │ Server  │
└─────────┘                  └─────────┘
     │                            │
     │  message { id, content }   │
     │ ─────────────────────────► │
     │                            │
     │  stream { id, content }    │
     │ ◄───────────────────────── │
     │  stream { id, content }    │
     │ ◄───────────────────────── │
     │  finish { id, files }      │
     │ ◄───────────────────────── │
     │                            │
     │  ping                      │
     │ ─────────────────────────► │
     │  pong                      │
     │ ◄───────────────────────── │
     │                            │
     │  close                     │
     │ ─────────────────────────► │
```

## 五、错误处理

### 5.1 HTTP 状态码

| 状态码 | 说明                    |
| :----- | :---------------------- |
| 200    | 成功                    |
| 400    | 请求参数错误            |
| 401    | 未授权（需要 API Key）  |
| 404    | Session 或 Agent 不存在 |
| 429    | 请求过于频繁            |
| 500    | 服务器内部错误          |
| 503    | 后端服务不可用          |

### 5.2 错误码

| 错误码                  | 说明           | 可重试       |
| :---------------------- | :------------- | :----------- |
| `INVALID_PARAMS`        | 请求参数错误   | 否           |
| `SESSION_NOT_FOUND`     | Session 不存在 | 否           |
| `AGENT_NOT_INITIALIZED` | Agent 未初始化 | 是           |
| `BACKEND_UNAVAILABLE`   | 后端服务不可用 | 是（5s）     |
| `MESSAGE_SEND_ERROR`    | 消息发送错误   | 视情况（2s） |
| `FILE_ACCESS_DENIED`    | 文件访问被拒绝 | 否           |
| `RATE_LIMIT_EXCEEDED`   | 请求过于频繁   | 是（60s）    |
| `INTERNAL_ERROR`        | 内部错误       | 是（3s）     |

## 九、总结

本接口规范定义了 Agent 服务层的所有对外接口，包括：

1. **REST API**：同步消息发送、状态查询、文件管理
2. **WebSocket API**：流式响应、实时推送
3. **类型定义**：统一的请求/响应格式
4. **错误处理**：标准化的错误码和重试策略
5. **安全机制**：认证、授权、速率限制
6. **工作空间隔离**：每个 Session 绑定独立工作空间，支持临时空间和用户指定空间

### 工作空间隔离特性

- **临时工作空间**：不指定 workingDir 时自动创建临时目录
- **用户指定工作空间**：指定 workingDir 时使用用户目录
- **工作空间切换**：支持运行时切换工作空间
- **变更检测**：支持 Git 仓库和普通目录两种模式
- **变更回滚**：支持丢弃变更恢复到基准状态
- **路径安全**：防止路径遍历攻击和符号链接攻击
