## 一、概述

本文档定义 `@opencode-workbench/agent-service` 当前对外暴露的 HTTP、WebSocket 和内部同步接口。服务默认运行在 `http://localhost:3201`，由 `packages/agent-service/src/server.ts` 注册 Fastify、CORS、WebSocket、限流和路由。

通用响应格式：

- 成功：`{ success: true, data: T }`
- 失败：`{ success: false, error: { code, message, details? } }`

类型定义入口：

- `packages/agent-service/src/core/types.ts`
- `packages/shared/src/workspace.ts`

## 二、REST API

### 2.1 Agent 消息与会话

| 端点 | 方法 | 说明 | 实现 |
| :--- | :--- | :--- | :--- |
| `/api/tools/capabilities` | GET | 返回当前 Pi Tools 版本和工具名列表 | `routes/agent.ts` |
| `/api/agent/:sessionId/message` | POST | 向指定 Session 发送非 WebSocket 消息 | `routes/agent.ts` |
| `/api/agent/:sessionId` | GET | 获取 Agent 当前状态 | `routes/agent.ts` |
| `/api/agent/:sessionId` | DELETE | 销毁 Agent、清理临时工作空间和 Session 元数据 | `routes/agent.ts` |
| `/api/agent/:sessionId/files` | GET | 返回当前工作空间 staged/unstaged 变更 | `routes/agent.ts` |
| `/api/agent/:sessionId/rollback` | POST | 回滚指定文件或全部文件变更 | `routes/agent.ts` |
| `/api/agent/:sessionId/workspace` | GET | 获取 Session 绑定的工作空间信息 | `routes/agent.ts` |
| `/api/agent/:sessionId/workspace` | PUT | 切换 Session 工作空间并初始化快照 | `routes/agent.ts` |
| `/api/agent/:sessionId/files/stage` | POST | 暂存指定文件变更 | `routes/agent.ts` |
| `/api/agent/:sessionId/files/discard` | POST | 丢弃指定文件变更 | `routes/agent.ts` |
| `/api/sessions` | GET | 列出当前 AgentManager 中的 Session | `routes/agent.ts` |

`POST /api/agent/:sessionId/message` 请求体：

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `content` | string | 是 | 用户消息内容 |
| `demoId` | string | 否 | 关联 Demo/Page ID |
| `workingDir` | string | 否 | 工作空间路径；未指定且无现有 Session 时创建临时工作空间 |
| `customWorkspace` | boolean | 否 | 是否为用户指定工作空间 |
| `model` | string | 否 | 本轮模型 ID |
| `systemPrompt` | string | 否 | author-site 构建的静态系统提示词 |
| `options.timeout` | number | 否 | 请求超时时间 |
| `options.stream` | boolean | 否 | 是否流式响应 |

响应 `data` 包含 `sessionId`、`content`、`files`、`metadata`。当前实现不再返回 OpenCode Session ID。

### 2.3 模型与校验

| 端点 | 方法 | 说明 | 实现 |
| :--- | :--- | :--- | :--- |
| `/models` | GET | 创建临时 Pi Agent 探测可用模型，返回 `models/currentModelId/canSwitch` | `routes/models.ts` |
| `/api/agent/:sessionId/validate` | POST | 读取 Session 工作空间中的 demo 代码和 schema 执行校验 | `routes/validate.ts` |
| `/api/validate` | POST | 直接传入 `code` 和 `schema` 执行校验 | `routes/validate.ts` |
| `/health` | GET | 返回服务健康状态、时间戳、运行时长和 Agent 数量 | `server.ts` |

`/models` 返回的模型项由 Pi Agent `getModelInfo()` 提供，并补充 `group`、`supportsImages`、`supportsThinkingDepth` 字段供前端展示。

### 2.4 内部配置同步

内部接口用于 author-site 管理后台向 agent-service 推送全局或 Session 级模型配置。鉴权头固定为 `x-internal-token`，生产环境必须配置 `INTERNAL_API_TOKEN`。全局配置的事实源仍在 author-site，agent-service 只保存运行时副本；服务重启后由 author-site 的启动恢复任务重新推送。

| 端点 | 方法 | 说明 | 实现 |
| :--- | :--- | :--- | :--- |
| `/internal/backend-providers` | POST | 设置完整全局 backend providers 配置 | `routes/internal-config.ts` |
| `/internal/backend-providers` | GET | 获取当前全局配置，返回时脱敏 API Key | `routes/internal-config.ts` |
| `/internal/sessions/:sessionId/model-config` | POST | 设置指定 Session 的模型配置，并更新已存在 Agent 配置 | `routes/internal-config.ts` |

配置结构使用 `packages/shared/src/agent-config.ts` 中的 `BackendProvidersConfig`。

### 2.5 使用端只读 AI 问答

| 端点 | 方法 | 说明 | 实现 |
| :--- | :--- | :--- | :--- |
| `/api/viewer-ai/chat` | POST | 面向使用端的只读 AI 问答，基于项目正式工作区上下文回答使用者问题 | `routes/viewer-ai.ts` |

`POST /api/viewer-ai/chat` 请求体：

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `projectId` | string | 是 | 当前使用端项目 ID |
| `message` | string | 是 | 使用者问题 |
| `sessionId` | string | 否 | 浏览器本地保存的只读 AI 会话 ID；未传时服务端生成 |
| `activePageId` | string | 否 | 当前预览页面 ID |
| `activeConfig` | object | 否 | 当前页面合并后的配置值 |
| `history` | array | 否 | 浏览器本地最近对话历史，用于连续追问 |

响应 `data` 包含 `sessionId` 和 `answer`。该接口使用 `viewer-readonly` 工具模式，只挂载读取类工具；不会返回文件变更，也不会开放写入、命令执行、删除页面、画布整理、图片保存或子 Agent 委派能力。

## 三、WebSocket API

连接地址：

```text
ws://localhost:3201/api/agent/:sessionId/stream
```

实现文件：

- `packages/agent-service/src/routes/websocket.ts`
- `packages/agent-service/src/routes/ws-event-router.ts`

### 3.1 客户端消息

| 类型 | 关键字段 | 说明 |
| :--- | :--- | :--- |
| `message` | `id`, `content`, `workingDir`, `demoId`, `modelId`, `images`, `systemPrompt`, `options` | 发送用户消息并触发 Pi Agent |
| `cancel` | `sessionId` | 取消当前请求 |
| `ping` | `timestamp` | 心跳 |
| `resume` | `sessionId` 或 `options.resumeSessionId` | 恢复指定会话 |
| `set_model` | `modelId` | 切换当前 Agent 模型 |
| `get_models` | `workingDir`, `demoId` | 获取当前可用模型 |
| `permission_response` | `permissionId`, `optionId`, `responseContent` | 响应工具权限确认 |
| `user_choice_response` | `requestId`, `choice` | 响应需求确认单选卡片 |
| `console_data` | `entries` | 前端上报页面控制台日志，供 `getConsoleLogs` 工具读取 |

### 3.2 服务端消息

| 类型 | 说明 |
| :--- | :--- |
| `status` | Session 状态变化，如 `initializing`、`processing`、`ready` |
| `stream` | AI 回复正文增量 |
| `thought` | 思考内容增量 |
| `tool_call` | 工具调用开始或进行中 |
| `tool_call_update` | 工具调用完成或失败 |
| `plan` | Agent 计划状态 |
| `permission_request` | 工具权限确认请求 |
| `user_choice_request` | 需求确认单选卡片请求 |
| `models` | 可用模型或当前模型信息 |
| `finish` | 本轮消息完成，包含最终内容、文件变更和 metadata |
| `error` | 错误对象 |
| `pong` | 心跳响应 |

WebSocket 路由会维护连接心跳，连续 60 秒未收到活动会关闭连接；单轮消息若连续 300 秒无 Agent 活动，会自动取消并返回 `MESSAGE_TIMEOUT`。

## 五、错误处理

常见 HTTP 状态码：

| 状态码 | 场景 |
| :--- | :--- |
| 200 | 请求成功 |
| 400 | 参数缺失或校验失败 |
| 401 | 内部接口鉴权失败 |
| 404 | Session、项目、版本或文件不存在 |
| 429 | Fastify 限流 |
| 500 | 服务内部错误 |
| 503 | 内部 token 未配置或模型探测后端不可达 |

常见错误码：

| 错误码 | 说明 |
| :--- | :--- |
| `INVALID_PARAMS` / `INVALID_REQUEST` | 请求参数不满足当前接口要求 |
| `SESSION_NOT_FOUND` | Session 不存在 |
| `PROJECT_NOT_FOUND` | 项目不存在 |
| `VERSION_NOT_FOUND` | 版本不存在 |
| `FILE_READ_ERROR` / `FILE_WRITE_ERROR` | 文件读写失败 |
| `FILE_ACCESS_DENIED` | 工作空间路径校验未通过 |
| `MESSAGE_SEND_ERROR` | Pi Agent 消息发送失败 |
| `MESSAGE_TIMEOUT` | WebSocket 消息长时间无活动后被取消 |
| `VALIDATION_ERROR` | backend providers 字段校验失败 |
| `SERVER_UNREACHABLE` | `/models` 探测 Pi Agent 失败 |
| `VIEWER_AI_ERROR` | 使用端只读 AI 问答失败 |
| `INTERNAL_ERROR` | 其他未分类服务错误 |

## 六、总结

当前 Agent 服务层接口围绕四条主线组织：

1. **Agent 会话**：HTTP 与 WebSocket 都以 `sessionId` 为主键。
2. **项目工作空间**：项目 CRUD、编辑会话、保存、放弃、版本恢复由 `project-workspace-manager.ts` 处理。
3. **Pi Agent 能力**：模型、工具、权限确认、文件变更和事件流都通过 Pi Agent 后端适配器收敛。
4. **内部配置同步**：author-site 管理后台通过内部 token 将模型供应商配置推送到 agent-service。

下一步：参考 [03-核心模块设计.md](./03-核心模块设计.md) 了解模块内部协作。
