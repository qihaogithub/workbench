# Codex 协作维护需求文档

## 模块定位

维护治理模块用于约束 Codex 在本项目中的协作方式，目标是减少误读上下文、漏跑验证和误改运行数据的风险。

## 功能需求

- Codex 应能区分源码、长期文档、计划记录、运行数据和临时产物。
- Codex 应能根据改动范围选择匹配的验证命令。
- Codex 排查问题时可以按需读取 `data/`，但不应把 `data/` 当成默认源码入口。
- Codex 应优先保留用户数据和现场证据，不应在未明确要求时清理运行数据。
- 维护性治理规则应能被后续 Agent 快速找到。

## 业务规则与约束

- `data/` 是诊断现场，不是稳定测试样例库。
- 运行数据、日志和截图可以辅助判断问题来源，但不能替代源码、测试和项目文档。
- 验证矩阵用于确定最低验证集合；高风险改动仍需追加更完整的测试。
- 文档更新应避免重复长期项目事实，详细执行矩阵见 [01 Codex 改动验收矩阵](./技术/01_Codex改动验收矩阵.md)。
