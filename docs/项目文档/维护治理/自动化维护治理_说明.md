# 自动化维护治理说明

> 更新日期：2026-06-30

## 定位

维护治理模块只保留项目级说明：本项目允许使用 Codex 定时任务维护 CLI 工具、测试工具、诊断脚本和问题记录，但具体执行上下文不放在项目文档知识库中维护。

自动任务的详细上下文已迁移到 [OPS/automations](../../../OPS/automations/README.md)。

## 边界

| 内容 | 维护位置 |
|:-----|:---------|
| 业务需求、接口契约、架构边界 | `docs/项目文档/` 对应业务模块 |
| 自动任务读取顺序、runbook、当前状态 | `OPS/automations/` |
| 具体问题、缺口和实施计划 | `docs/plans/进行中/` |
| 已关闭问题和历史方案 | `docs/plans/已完成/` |
| Codex 自动化平台运行记忆 | `$CODEX_HOME/automations/*/memory.md` |

## 维护规则

- `OPS/automations/` 面向自动任务中的 AI，按 runbook、context 和 state 组织，不套用项目文档的需求/技术拆分。
- 如果自动任务发现可复用业务规则，应更新对应项目文档。
- 如果只是自动任务执行状态或检查清单，应留在 `OPS/automations/`。
- 如果发现具体缺陷或测试缺口，应建立或更新 `docs/plans/进行中/` 中的跟踪文档。

## 当前任务族

- 创作端 CLI 工具维护。
- 测试工具与脚本治理。
- 定期问题排查与过期记录清理。
