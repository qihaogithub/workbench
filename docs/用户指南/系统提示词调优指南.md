## 一、什么是系统提示词？

系统提示词是 AI 的**行为手册**。每次你和 AI 对话时，这份手册都会被悄悄塞给 AI，告诉它：

- **能做什么**：创建页面、编辑代码、修改配置
- **怎么做**：代码规范、文件结构、命名规则
- **不能做什么**：禁止访问的文件、禁止执行的命令

> 简单理解：系统提示词 = AI 的游戏规则。改它，AI 就会换一种方式工作。

## 二、一张图看懂

系统提示词在一个 `.md` 文件里，用 Markdown 编辑器就能改：

```
system-prompt.md
│
├── # Demo Generator Agent      ← AI 的行为规则
├── ---                          ← 分隔线
├── ## 项目记忆维护               ← memory.md 维护规则
├── ---                          ← 分隔线
└── ## 权限确认                  ← 需要用户确认的操作
         ↓ 每次对话发送给 AI
```

## 三、你要改哪个文件？

| 操作 | 文件 | 路径 |
|------|------|------|
| 修改 AI 系统提示词 | `system-prompt.md` | `packages/author-site/src/lib/agent/prompts/` |

文件用 Markdown 格式编写，任何文本编辑器都能打开。**包含以下章节**：

| 章节               | 控制什么                        |
| ------------------ | ------------------------------- |
| `页面内容编辑`     | AI 如何匹配你提到的页面名称     |
| `创建页面`         | AI 创建新页面时的文件结构和格式 |
| `重命名 / 改顺序`  | AI 修改页面元数据的规则         |
| `文件夹管理`       | AI 操作文件夹的规则             |
| `项目级配置管理`   | AI 如何处理项目级共享配置       |
| `代码质量标准`     | TypeScript、Tailwind、组件规范  |
| `React 版本约束`   | 依赖库的白名单和版本限制        |
| `禁止行为`         | AI 绝对不允许做的事             |
| `文件修改决策规则` | AI 判断"改哪个文件"的规则       |

## 四、常见调优场景

### 场景 1：AI 太啰嗦了，想让它少说话

在 `system-prompt.md` 末尾添加：

```
## 交流风格

- 回答要简洁直接，不要解释"为什么这么做"
- 代码改完直接说"改好了"就行，不要说"我已经完成了……"
- 不要问我"是否要执行"，直接动手干
```

### 场景 2：AI 总用某个你不喜欢的 UI 库

检查 `代码质量标准` 章节，看是否有你不想用的库被列在可用清单里。

如果要禁止某个库，在 `禁止行为` 章节添加一条：

```
- ❌ 使用 Ant Design (antd) 组件库
```

### 场景 3：AI 输出的代码格式不符合习惯

修改 `代码质量标准` 章节中的 `index.tsx` 要求。例如：

```
- 使用单引号而非双引号
- 缩进用 2 空格
- 组件默认导出放在文件顶部
```

### 场景 4：想让 AI 自动记忆你的偏好

直接编辑 `system-prompt.md`，找到「项目记忆维护」章节进行修改。

### 场景 5：AI 对页面操作的流程不对

修改 `system-prompt.md` 中对应的章节：

- 创建页面 → 改 `创建页面` 部分
- 删除页面 → 改 L2 模板中对应的说明
- 修改配置 → 改 `项目级配置管理` 部分

## 五、改完如何生效？

1. **保存文件**（Ctrl+S / Cmd+S）
2. **重启开发服务器**：

```bash
# 在终端中按 Ctrl+C 停止当前运行的 dev 服务
# 然后重新启动
pnpm dev
```

3. **验证**：打开浏览器，在 AI 聊天中发一条消息，看 AI 的行为是否按预期改变。

> 提示词在 author-site 中，`pnpm dev` 或 `pnpm dev:author` 重启均可生效。

---

## 六、注意事项

- **改之前先备份**：复制一份 `system-prompt.md` 另存为 `.bak`，改坏了能恢复
- **一次只改一处**：改完验证生效后再改下一处，便于定位问题
- **不要删除结构标记**：`##` 标题、`-` 列表项、代码块（` ``` `）不要破坏，否则 AI 可能理解错误
- **谨慎修改禁止行为**：安全相关的限制（如禁止访问 packages/、禁止执行 rm 命令）不要取消
- **如需帮助**：描述你希望 AI 如何改变行为，我可以帮你写出对应的 prompt
