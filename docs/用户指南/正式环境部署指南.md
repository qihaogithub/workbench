## 快速开始

日常部署只需一条命令。项目根目录下执行：

```bash
# 日常前端部署（最常用）
scripts/deploy-fast.sh author viewer

# 只重新部署 agent-service
scripts/deploy-fast.sh agent

# 部署核心服务（Agent + 创作端 + 浏览端）
scripts/deploy-fast.sh core

# 部署全部服务（含截图服务）
INCLUDE_SCREENSHOT_SERVICE=true scripts/deploy.sh
```

部署流程：代码同步 → 本地构建 Docker 镜像 → 上传镜像 → 重启服务 → 自动健康检查。

## 部署命令速查

### deploy-fast.sh 快捷名

| 短名 | 等价服务 | 适用场景 |
| --- | --- | --- |
| `author` | author-site | 只改了创作端页面/接口/依赖 |
| `viewer` | viewer-site | 只改了浏览端页面/依赖 |
| `agent` | agent-service | 只改了 Agent 后端 |
| `core` | agent-service + author-site + viewer-site | 前后端同时改动 |
| `shot` | screenshot-service | 只改了截图服务（构建耗时较长） |

可组合使用：`scripts/deploy-fast.sh author viewer`

### 常用选项

| 选项 | 说明 |
| --- | --- |
| `--dry-run` | 只打印部署范围，不真正执行 |
| `--full-sync` | 完整同步所有包（默认只同步目标服务需要的包） |
| `--remote-build` | 在服务器上构建镜像（兜底模式，需服务器内存 ≥ 3GB） |

### 部署前预检

```bash
scripts/deploy-fast.sh author viewer --dry-run
```

## 截图服务

默认部署**不包含**截图服务（依赖 Chromium，构建慢、镜像大）。需要时单独部署：

```bash
scripts/deploy-fast.sh shot
# 或在完整部署中包含
INCLUDE_SCREENSHOT_SERVICE=true scripts/deploy.sh
```

## data 目录

- 日常部署**不会**覆盖正式环境 `data/`，服务器已有数据会保留
- 如果远端 `APP_DATA_DIR` 目录不存在，部署会中断（防止切到空数据）
- 首次部署到新服务器，需显式允许创建空目录：

```bash
ALLOW_CREATE_APP_DATA_DIR=true scripts/deploy.sh
```

### 本地 data → 覆盖正式环境

> 高风险操作，会替换正式数据。

```bash
# 预检
scripts/deploy-author-with-data.sh --dry-run --overwrite-data --confirm-overwrite-production-data
# 只备份正式 data
scripts/deploy-author-with-data.sh --backup-only
# 执行覆盖并验证
scripts/deploy-author-with-data.sh --deploy-author --overwrite-data --confirm-overwrite-production-data --verify
```

远端备份位置：`/opt/workbench-data-backups`

### 正式环境 data → 覆盖本地

> 会替换本地 data，执行前会自动备份。

```bash
# 预检
scripts/sync-production-data-to-local.sh --dry-run --overwrite-local-data --confirm-overwrite-local-data
# 只备份本地 data
scripts/sync-production-data-to-local.sh --backup-only
# 执行覆盖
scripts/sync-production-data-to-local.sh --overwrite-local-data --confirm-overwrite-local-data
```

本地备份位置：`../workbench-data-backups`

## 部署后验证

脚本会自动做健康检查，也可手动验证：

```bash
curl -I http://10.130.33.131:3200          # 创作端
curl -I http://10.130.33.131:3300          # 浏览端
curl http://10.130.33.131:3201/health      # Agent 服务
curl http://10.130.33.131:3202/health      # 截图服务
```

截图服务深度健康检查：

```bash
SCREENSHOT_HEALTH_URL=http://10.130.33.131:3202/health?deep=1 scripts/docker-screenshot-deep-health.sh
```

## 部署前检查清单

1. `.env.docker` 存在且包含 `INTERNAL_API_TOKEN`
2. SSH 私钥 `~/.ssh/figma-mirror-deploy-key` 存在且可连接服务器
3. 本地 Docker 运行正常（默认本地构建模式）

## 常见问题

**代码改了但线上没变** — 确认执行了构建部署命令（不是只重启容器）。如果改动不属于默认目标包，使用 `--full-sync`。

**加速部署提示缺少 packages** — 目标服务 Dockerfile 需要某个包但加速列表未包含。临时用 `--full-sync`，后续更新 `deploy.sh` 中的 targeted package 列表。

**部署后项目列表变空** — 正式 `data/` 可能被切到空目录。在服务器检查：`du -sh /opt/opencode-workbench/data`。确认后执行 `deploy-author-with-data.sh --overwrite-data --confirm-overwrite-production-data --verify` 恢复。

**SSH 连接失败** — 检查私钥 `ls ~/.ssh/figma-mirror-deploy-key` 和连接 `ssh -i ~/.ssh/figma-mirror-deploy-key root@10.130.33.131`。

**`.env.docker` 缺少配置** — 脚本要求 `INTERNAL_API_TOKEN`，缺失会中断部署。补齐后重试。
