## 一、这是什么？

OpenCode 工作台本身不包含大模型，它需要调用**你自有的模型 API** 来工作。你需要提供三样东西：

| 你需要准备 | 说明 | 例子 |
|-----------|------|------|
| **API Key** | 你的 API 密钥（相当于密码） | `sk-xxxxxxxxxxxx` |
| **API 地址** | API 服务的网址 | `https://api.deepseek.com/v1` |
| **模型名称** | 你想用的模型名字 | `deepseek-chat`、`qwen-plus` |

## 二、快速开始（5 分钟搞定）

### 前提条件

- 已经部署好了 OpenCode 工作台（Docker 方式）
- 已经拥有某个大模型提供商的 API Key

### 操作步骤

#### 第 1 步：找到配置文件

在项目根目录下，找到 `.env` 文件。如果不存在，可复制 `.env.docker` 并重命名为 `.env`。

```
opencode-workbench/
├── .env                ← 在这里修改配置（这个文件不会被提交到 Git）
├── .env.docker         ← 模板文件，仅供参考
├── docker-compose.yml
└── ...
```

#### 第 2 步：填写你的 API 信息

用文本编辑器打开 `.env` 文件，找到下面这三行：

```env
OPENCODE_API_KEY=sk-your-api-key-here
OPENCODE_API_BASE=https://api.example.com/v1
OPENCODE_MODELS=your-model-name
```

把它们改成你的真实信息。例如，如果你想用 DeepSeek：

```env
OPENCODE_API_KEY=sk-你的DeepSeek密钥
OPENCODE_API_BASE=https://api.deepseek.com/v1
OPENCODE_MODELS=deepseek-chat
```

#### 第 3 步：重新启动服务

在终端中运行：

```bash
docker compose down && docker compose build opencode-serve && docker compose up -d
```
等待 10-20 秒，所有服务启动完成后，新的模型就生效了。

## 三、支持的 API 提供商及配置示例

只要是兼容 **OpenAI API 格式** 的服务都可以使用。以下是一些常见的提供商：

### DeepSeek（推荐）

```env
OPENCODE_API_KEY=sk-你的DeepSeek密钥
OPENCODE_API_BASE=https://api.deepseek.com/v1
OPENCODE_MODELS=deepseek-chat
```

### 通义千问（阿里云）

```env
OPENCODE_API_KEY=sk-你的通义千问密钥
OPENCODE_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
OPENCODE_MODELS=qwen-plus
```

其他可用模型：`qwen-max`、`qwen-turbo`

### SiliconFlow（硅基流动）

```env
OPENCODE_API_KEY=sk-你的SiliconFlow密钥
OPENCODE_API_BASE=https://api.siliconflow.cn/v1
OPENCODE_MODELS=deepseek-ai/DeepSeek-V2.5
```

### Moonshot（月之暗面）

```env
OPENCODE_API_KEY=sk-你的Moonshot密钥
OPENCODE_API_BASE=https://api.moonshot.cn/v1
OPENCODE_MODELS=moonshot-v1-8k
```

### OpenAI

```env
OPENCODE_API_KEY=sk-你的OpenAI密钥
OPENCODE_API_BASE=https://api.openai.com/v1
OPENCODE_MODELS=gpt-4o
```

其他可用模型：`gpt-4o-mini`、`gpt-4-turbo`

### Anthropic Claude

如果你使用 Anthropic 的 API（非 OpenAI 兼容格式），请将 `OPENCODE_API_BASE` 设置为 Anthropic 的 API 地址：

```env
OPENCODE_API_KEY=sk-ant-你的Claude密钥
OPENCODE_API_BASE=https://api.anthropic.com
OPENCODE_MODELS=claude-sonnet-4-20250514
```

## 四、更换模型的步骤

如果你想换一个模型，操作非常简单：

1. **修改 `.env` 文件**中的 `OPENCODE_MODELS` 和（如果需要）`OPENCODE_API_BASE`、`OPENCODE_API_KEY`
2. **重新构建 opencode-serve 容器**并启动：
   ```bash
   docker compose down && docker compose build opencode-serve && docker compose up -d
   ```
3. 完成，新的模型已生效

## 五、常见问题

### Q：我修改了 `.env` 后需要重启哪些服务？

只需要重启 `opencode-serve` 容器（上面第 3 步的命令会自动处理全部重启）。

### Q：如何检查配置是否生效？

运行以下命令，查看容器内生成的配置文件：

```bash
docker exec opencode-workbench-opencode-serve-1 bash -c "cat ~/.opencode/opencode.json"
```

如果显示了你的 API 信息，说明配置已成功写入。

### Q：为什么对话时提示"模型不可用"？

可能的原因：

1. **API Key 不正确**：检查 `.env` 中的 `OPENCODE_API_KEY` 是否正确
2. **API 余额不足**：登录你的 API 提供商网站检查余额
3. **模型名称不正确**：确认 `OPENCODE_MODELS` 填写的是正确的模型 ID
4. **网络不通**：确认 Docker 容器能访问到你的 API 地址。可以在容器内测试：
   ```bash
   docker exec opencode-workbench-opencode-serve-1 curl -s $OPENCODE_API_BASE/models -H "Authorization: Bearer $OPENCODE_API_KEY"
   ```

### Q：`docker compose down` 会丢失数据吗？

不会。数据存储在 Docker volume 中，`down` 只会停止容器，数据不会丢失。

### Q：如何查看当前使用的模型？

访问以下 API 地址（在浏览器中打开即可）：

```
http://你的服务器IP:3201/api/llm/models
```

会返回当前可用的模型列表。

## 附录：环境变量说明

| 变量名 | 必填 | 说明 | 默认值 |
|-------|------|------|--------|
| `OPENCODE_API_KEY` | 是 | 你的大模型 API 密钥 | `sk-your-api-key-here` |
| `OPENCODE_API_BASE` | 是 | API 服务的基础地址 | `https://api.example.com/v1` |
| `OPENCODE_MODELS` | 是 | 要使用的模型名称 | `your-model-name` |
| `NEXT_PUBLIC_AGENT_SERVICE_URL` | 否 | 前端访问 Agent 服务的地址 | `http://localhost:3201` |
| `NEXT_PUBLIC_WEB_URL` | 否 | 前端访问地址 | `http://localhost:3200` |
| `CORS_ORIGINS` | 否 | 允许跨域访问的来源 | `http://localhost:3200,...` |
| `JWT_SECRET` | 否 | JWT 加密密钥（生产环境请修改） | `change-this-to-a-random-string` |
