## 一、这是什么？

Workbench本身不包含大模型，它通过内置的 **Pi Agent** 调用**你自有的模型 API** 来工作。你需要提供以下信息：

| 你需要准备           | 说明            | 环境变量            | 例子                              |
| -------------------- | --------------- | ------------------- | --------------------------------- |
| **API Provider**     | 模型提供商名称  | `PI_AGENT_PROVIDER` | `anthropic`、`openai`、`deepseek` |
| **API Key**          | 你的 API 密钥   | `PI_AGENT_API_KEY`  | `sk-xxxxxxxxxxxx`                 |
| **模型名称**         | 你想用的模型    | `PI_AGENT_MODEL`    | `claude-sonnet-4-20250514`        |
| **API 地址**（可选） | 自定义 API 地址 | `PI_AGENT_BASE_URL` | `https://api.deepseek.com/v1`     |

## 二、快速开始（5 分钟搞定）

### 前提条件

- 已经部署好了 Workbench（Docker 方式）
- 已经拥有某个大模型提供商的 API Key

### 操作步骤

#### 第 1 步：找到配置文件

在项目根目录下，找到 `.env` 文件。如果不存在，可复制 `.env.docker` 并重命名为 `.env`。

```
workbench/
├── .env                ← 在这里修改配置（这个文件不会被提交到 Git）
├── .env.docker         ← 模板文件，仅供参考
├── docker-compose.yml
└── ...
```

#### 第 2 步：填写你的 API 信息

用文本编辑器打开 `.env.docker` 文件，找到 `PI_AGENT_*` 配置项：

```env
PI_AGENT_PROVIDER=anthropic
PI_AGENT_API_KEY=your-api-key-here
PI_AGENT_MODEL=claude-sonnet-4-20250514
PI_AGENT_BASE_URL=
```

把它们改成你的真实信息。例如，如果你想用 DeepSeek：

```env
PI_AGENT_PROVIDER=openai
PI_AGENT_API_KEY=sk-你的DeepSeek密钥
PI_AGENT_MODEL=deepseek-chat
PI_AGENT_BASE_URL=https://api.deepseek.com/v1
```

#### 第 3 步：重新启动服务

在终端中运行：

```bash
docker compose down && docker compose build agent-service && docker compose up -d
```

等待 10-20 秒，所有服务启动完成后，新的模型就生效了。

## 三、支持的 API 提供商及配置示例

只要是兼容 **OpenAI API 格式** 的服务都可以使用。以下是一些常见的提供商：

### DeepSeek（推荐）

```env
PI_AGENT_PROVIDER=openai
PI_AGENT_API_KEY=sk-你的DeepSeek密钥
PI_AGENT_MODEL=deepseek-chat
PI_AGENT_BASE_URL=https://api.deepseek.com/v1
```

### 通义千问（阿里云）

```env
PI_AGENT_PROVIDER=openai
PI_AGENT_API_KEY=sk-你的通义千问密钥
PI_AGENT_MODEL=qwen-plus
PI_AGENT_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

其他可用模型：`qwen-max`、`qwen-turbo`

### SiliconFlow（硅基流动）

```env
PI_AGENT_PROVIDER=openai
PI_AGENT_API_KEY=sk-你的SiliconFlow密钥
PI_AGENT_MODEL=deepseek-ai/DeepSeek-V2.5
PI_AGENT_BASE_URL=https://api.siliconflow.cn/v1
```

### Moonshot（月之暗面）

```env
PI_AGENT_PROVIDER=openai
PI_AGENT_API_KEY=sk-你的Moonshot密钥
PI_AGENT_MODEL=moonshot-v1-8k
PI_AGENT_BASE_URL=https://api.moonshot.cn/v1
```

### OpenAI

```env
PI_AGENT_PROVIDER=openai
PI_AGENT_API_KEY=sk-你的OpenAI密钥
PI_AGENT_MODEL=gpt-4o
```

其他可用模型：`gpt-4o-mini`、`gpt-4-turbo`

### Anthropic Claude

```env
PI_AGENT_PROVIDER=anthropic
PI_AGENT_API_KEY=sk-ant-你的Claude密钥
PI_AGENT_MODEL=claude-sonnet-4-20250514
```

## 四、更换模型的步骤

如果你想换一个模型，操作非常简单：

1. **修改 `.env.docker` 文件**中的 `PI_AGENT_MODEL` 和（如果需要）`PI_AGENT_BASE_URL`、`PI_AGENT_API_KEY`
2. **重新构建 agent-service 容器**并启动：
   ```bash
   docker compose down && docker compose build agent-service && docker compose up -d
   ```
3. 完成，新的模型已生效

## 五、高级配置：白名单、黑名单与默认模型

系统提供了三个维度的模型过滤与选择机制，均通过 `.env` 文件配置。

### 5.1 白名单 — 控制哪些分组可见

白名单决定了**哪些模型分组**可以在前端下拉框中显示。

```env
NEXT_PUBLIC_ALLOWED_MODEL_PREFIXES=xjjj/
```

- 填写分组前缀，逗号分隔多个，如 `deepseek/,qwen/,custom/`
- 必须与管理后台「AI 后端供应商」中的供应商 id 对应
- 不在白名单中的分组会被完全隐藏

### 5.2 黑名单 — 排除特定模型

黑名单用于**禁用特定模型**，即使它通过了白名单过滤也会被排除。

```env
# 示例：禁用以下模型
NEXT_PUBLIC_MODEL_BLACKLIST=xjjj/old-model,xjjj/test-model
```

- 填写完整的模型 ID，逗号分隔
- 适用于：临时禁用某个模型、有问题的模型、不想暴露的模型
- 不需要时留空或注释掉即可

### 5.3 默认模型 — 自动选中的模型

系统会自动选择一个默认模型展示给用户。可以配置**多个备选**，按优先级依次尝试。

```env
# 按优先级排列，逗号分隔
NEXT_PUBLIC_DEFAULT_MODEL_IDS=xjjj/deepseek-v4-flash,xjjj/gpt-model
```

**选择逻辑：**

1. 先按顺序检查列表中的每个模型，找到第一个**实际可用**的
2. 如果配置的模型全部不可用，则**自动回退**到白名单中第一个可用的模型

**示例：**

- 假设配置了 `NEXT_PUBLIC_DEFAULT_MODEL_IDS=xjjj/model-a,xjjj/model-b`
- 实际可用模型有：`xjjj/model-b`、`xjjj/model-c`
- 结果：选中 `xjjj/model-b`（因为 model-a 不可用，model-b 可用）

### 5.4 完整配置示例

以下是一个完整的 `.env` 前端模型配置示例：

```env
# 白名单：允许显示的分组前缀
NEXT_PUBLIC_ALLOWED_MODEL_PREFIXES=xjjj/,custom/

# 黑名单：禁止显示的模型（完整 ID）
NEXT_PUBLIC_MODEL_BLACKLIST=xjjj/old-model

# 默认模型：按优先级排列（完整 ID）
NEXT_PUBLIC_DEFAULT_MODEL_IDS=xjjj/deepseek-v4-flash,xjjj/gpt-model
```

修改以上任意配置后，重新构建并启动即可生效：

```bash
docker compose down && docker compose build author-site agent-service && docker compose up -d
```

> **提示：** 如果只需要修改前端模型配置（白名单/黑名单/默认模型），**只需重新构建 `author-site`**。如果同时修改了 API 配置（API Key、模型、Provider），则需要同时重建 `agent-service`。

## 六、常见问题

### Q：我修改了 `.env.docker` 后需要重启哪些服务？

只需要重新构建并启动对应的服务（上面的命令会自动处理重启）。

### Q：如何检查配置是否生效？

运行以下命令，查看 agent-service 健康状态：

```bash
curl http://你的服务器IP:3201/health
```

如果返回 `status: ok`，说明服务正常运行。

### Q：为什么对话时提示“模型不可用”？

可能的原因：

1. **API Key 不正确**：检查 `.env.docker` 中的 `PI_AGENT_API_KEY` 是否正确
2. **API 余额不足**：登录你的 API 提供商网站检查余额
3. **模型名称不正确**：确认 `PI_AGENT_MODEL` 填写的是正确的模型 ID
4. **Provider 不匹配**：确认 `PI_AGENT_PROVIDER` 与你的 API 提供商对应（`openai` 兼容格式或 `anthropic`）

### Q：`docker compose down` 会丢失数据吗？

不会。数据存储在 Docker volume 中，`down` 只会停止容器，数据不会丢失。

### Q：如何查看当前使用的模型？

通过管理后台（`/admin`）可以查看和切换当前模型。也可以通过 API：

```
http://你的服务器IP:3201/api/models
```

会返回当前可用的模型列表。

## 附录：环境变量说明

| 变量名                               | 必填 | 说明                                                | 默认值                           |
| ------------------------------------ | ---- | --------------------------------------------------- | -------------------------------- |
| `PI_AGENT_PROVIDER`                  | 是   | 模型提供商（`anthropic`、`openai`、`deepseek` 等）  | `anthropic`                      |
| `PI_AGENT_API_KEY`                   | 是   | 你的大模型 API 密钥                                 | -                                |
| `PI_AGENT_MODEL`                     | 是   | 要使用的模型名称                                    | `claude-sonnet-4-20250514`       |
| `PI_AGENT_BASE_URL`                  | 否   | 自定义 API 基础地址（OpenAI 兼容格式时需要）        | （空）                           |
| `NEXT_PUBLIC_AGENT_SERVICE_URL`      | 否   | 前端访问 Agent 服务的地址                           | `http://localhost:3201`          |
| `NEXT_PUBLIC_WEB_URL`                | 否   | 前端访问地址                                        | `http://localhost:3200`          |
| `CORS_ORIGINS`                       | 否   | 允许跨域访问的来源                                  | `http://localhost:3200,...`      |
| `JWT_SECRET`                         | 否   | JWT 加密密钥（生产环境请修改）                      | `change-this-to-a-random-string` |
| `NEXT_PUBLIC_ALLOWED_MODEL_PREFIXES` | 否   | 前端模型白名单（分组前缀，逗号分隔）                | `xjjj/`                          |
| `NEXT_PUBLIC_MODEL_BLACKLIST`        | 否   | 前端模型黑名单（完整 ID，逗号分隔）                 | （空）                           |
| `NEXT_PUBLIC_DEFAULT_MODEL_IDS`      | 否   | 前端默认模型列表（完整 ID，逗号分隔，按优先级排列） | （空，自动用白名单第一个）       |
