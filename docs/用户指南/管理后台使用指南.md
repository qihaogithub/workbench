# 管理后台使用指南

> 适用版本: Workbench 最新版本  
> 更新日期: 2026-05-29

## 概述

管理后台是创作端的系统管理入口,允许管理员通过 Web 界面动态调整全局配置,无需修改环境变量或重启服务。当前主要提供**模型配置管理**功能。

### 核心能力

- ✅ 动态启用/禁用 AI 模型
- ✅ 拖拽调整模型优先级
- ✅ 设置自动启用规则(批量管理模型)
- ✅ 标记多模态模型(支持图片输入)
- ✅ 配置实时生效,无需重启服务

---

## 1. 访问管理后台

### 1.1 前置条件

确保已配置管理员密钥:

```bash
# 在 .env 文件中配置
ADMIN_SECRET=your-strong-secret-key-at-least-32-chars
```

> ⚠️ **安全提醒**: 生产环境必须使用强随机字符串(至少 32 字符),并启用 HTTPS。

### 1.2 首次访问

在浏览器中访问管理后台 URL,并携带密钥参数:

```
http://localhost:3200/admin?secret=your-admin-secret
```

首次验证成功后,系统会自动设置 Cookie(2 小时有效),后续访问无需再次输入密钥:

```
http://localhost:3200/admin
```

### 1.3 访问权限

- ✅ **已验证管理员**: 可访问 `/admin` 及所有子页面
- ❌ **未验证用户**: 返回 401 拒绝访问
- ⏰ **Cookie 过期后**: 需重新通过 URL 参数验证

---

## 2. 管理后台概览

访问 `/admin` 后,您将看到管理后台首页,包含:

- 📋 **功能导航**: 各管理模块入口(当前为模型配置)
- 📊 **系统状态**: 关键配置信息概览

---

## 3. 模型配置管理

点击左侧导航栏的「模型配置」或访问 `/admin/model-config` 进入模型配置页面。

### 3.1 页面布局

模型配置页面分为三个核心区域:

```
┌─────────────────────────────────────────────┐
│ 顶部操作栏                                    │
│ [拉取模型] 按钮  |  [保存配置] 按钮          │
├─────────────────────────────────────────────┤
│ 🟢 已启用模型区 (Enabled Models)             │
│ - 拖拽调整顺序                                │
│ - 控制启用/禁用                               │
│ - 设置多模态支持                              │
├─────────────────────────────────────────────┤
│ 📐 自动启用规则区 (Auto Enable Rules)        │
│ - 分组前缀规则                                │
│ - 名称关键词规则                              │
│ - 批量启用匹配模型                            │
├─────────────────────────────────────────────┤
│ ⚪ 未启用模型区 (Disabled Models)            │
│ - 搜索过滤                                    │
│ - 单个启用                                    │
└─────────────────────────────────────────────┘
```

### 3.2 拉取可用模型

首次使用或后端新增模型时,需要拉取最新模型列表:

1. 点击顶部 **「拉取模型」** 按钮
2. 系统会从 agent-service 获取当前所有可用模型
3. 模型将显示在「未启用模型区」

> 💡 **提示**: 拉取操作需要 agent-service 正常运行,超时时间为 5 秒。

### 3.3 启用/禁用模型

#### 方法一:单个启用

在「未启用模型区」找到目标模型,点击 **「启用」** 按钮:

- 模型将移至「已启用模型区」末尾
- 用户端立即可见该模型

#### 方法二:批量启用(推荐)

使用自动启用规则批量管理(详见 3.5 节)。

#### 禁用模型

在「已启用模型区」关闭模型的 **「启用开关」**:

- 模型将移至「未启用模型区」
- 用户端立即隐藏该模型

### 3.4 调整模型优先级

模型优先级决定用户在对话界面看到的默认顺序,**排在第一位的模型自动作为默认模型**。

**操作步骤**:

1. 在「已启用模型区」找到目标模型
2. 按住左侧 **拖拽柄** (⋮⋮ 图标)
3. 上下拖动调整位置
4. 松开鼠标完成排序

> 🎯 **最佳实践**: 将最常用、性能最好的模型排在前面。

### 3.5 设置多模态支持

如果某个模型支持图片输入(如 GPT-4o、Claude 3),可以标记为多模态:

1. 在「已启用模型区」找到目标模型
2. 打开 **「多模态开关」** (🖼️ 图标)
3. 保存配置后,用户端该模型将显示文件上传入口

> ⚠️ **注意**: 仅标记支持的模型,不支持图片的模型开启后无法正常使用图片功能。

### 3.6 自动启用规则

自动启用规则可以批量管理模型,特别适合模型数量多或频繁新增模型的场景。

#### 规则类型

| 规则类型       | 格式         | 示例            | 匹配逻辑                                      |
| -------------- | ------------ | --------------- | --------------------------------------------- |
| **分组前缀**   | `prefix`     | `xjjj/`         | 匹配模型 ID 以 `xjjj/` 开头的所有模型         |
| **名称关键词** | `nameFilter` | `workbench:Free` | 匹配 `workbench/` 分组且名称包含 `Free` 的模型 |

#### 添加规则

1. 在「自动启用规则区」点击 **「+ 添加规则」**
2. 选择规则类型(分组前缀 / 名称关键词)
3. 输入规则值
4. 系统自动显示匹配数量(如「匹配: 5 个模型」)

#### 批量启用

规则添加后,点击 **「批量启用」** 按钮:

- 所有匹配规则的模型将加入已启用列表
- 已启用的模型不会重复添加

#### 删除规则

点击规则右侧的 **「删除」** 按钮:

- 仅删除规则本身,不影响已启用的模型
- 后续新增的匹配模型将不再自动启用

> 💡 **使用场景**:
>
> - 某供应商提供多个模型,希望全部启用 → 使用分组前缀规则
> - 定期新增免费模型,希望自动上线 → 使用名称关键词规则

### 3.7 搜索未启用模型

当未启用模型较多时,可以使用搜索框快速定位:

1. 在「未启用模型区」顶部找到搜索框
2. 输入模型 ID 或名称关键词
3. 列表实时过滤显示匹配结果

### 3.8 保存配置

**所有修改都需要点击「保存配置」才会生效**:

1. 完成模型启用/排序/规则设置后,点击顶部 **「保存配置」** 按钮
2. 系统提示保存成功
3. 前端自动加载新配置,无需刷新页面

> ⚠️ **重要**: 未保存的配置修改仅存在于本地,刷新页面后将丢失。

## 4. 清除缓存

配置保存后通常会自动生效,但如果遇到配置未更新的情况,可以手动清除缓存:

### 方法一:重新拉取配置

1. 刷新管理后台页面
2. 系统会重新从数据库读取配置

### 方法二:调用缓存清除 API

```bash
# 需要携带 admin cookie 或通过 URL 参数验证
curl -X POST "http://localhost:3200/api/admin/reload-config?secret=your-admin-secret"
```

## 5. 常见问题

### Q1: 访问管理后台返回 401?

**原因**: 未通过管理员验证。

**解决方案**:

1. 检查 `.env` 中是否配置了 `ADMIN_SECRET`
2. 通过 URL 参数携带密钥访问: `?secret=your-admin-secret`
3. 确认 Cookie 是否过期(2 小时有效期)

### Q2: 拉取模型失败?

**原因**: agent-service 未正常运行或网络不通。

**解决方案**:

1. 检查 agent-service 是否在运行(默认端口 3201)
2. 确认 `AGENT_SERVICE_URL` 环境变量配置正确
3. 查看浏览器控制台和终端日志

### Q3: 保存配置后用户端未生效?

**原因**: 缓存未清除或浏览器缓存。

**解决方案**:

1. 在管理后台刷新页面,确认配置已保存
2. 用户端刷新页面或重新登录
3. 手动调用缓存清除 API(见第 4 节)

### Q4: 如何修改 API Key 或 API Base?

**当前限制**: 供应商配置(API Key、API Base)暂不支持通过管理后台修改。

**解决方案**:

1. 修改 `.env` 文件中的相关配置
2. 重启 agent-service 服务使配置生效

```bash
# 示例:修改 API Key
OPENAI_API_KEY=your-new-api-key

# 重启服务
pnpm dev:agent
```

### Q5: 多实例部署时配置不同步?

**原因**: 当前使用内存缓存,多实例间缓存独立。

**解决方案**:

1. 确保所有实例共享同一个 SQLite 数据库文件
2. 修改配置后,手动在每个实例调用缓存清除 API
3. 或等待缓存自动过期(TTL 1 分钟)

## 6. 安全建议

### 生产环境配置清单

- [ ] `ADMIN_SECRET` 使用强随机字符串(至少 32 字符)
- [ ] 启用 HTTPS(必须)
- [ ] 定期更换管理员密钥
- [ ] 限制管理后台访问 IP(通过反向代理)
- [ ] 监控异常访问日志

### 生成强密钥

```bash
# macOS/Linux
openssl rand -base64 48

# 或使用 Node.js
node -e "console.log(require('crypto').randomBytes(48).toString('base64'))"
```

## 7. 配置数据结构(高级)

管理后台配置存储在 SQLite `system_configs` 表中,配置 ID 为 `model_config`。

### JSON 结构

```typescript
{
  frontend: {
    enabledModels: [                     // 启用模型列表(有序)
      "xjjj/deepseek-v4-flash",
      "xjjj/gpt-4o",
      "anthropic/claude-3-opus"
    ],
    autoEnableRules: [                   // 自动启用规则
      { type: "prefix", value: "xjjj/" },
      { type: "nameFilter", value: "workbench:Free" }
    ]
  },
  multimodalModels: [                    // 多模态模型
    "xjjj/gpt-4o",
    "anthropic/claude-3-opus"
  ]
}
```

> 💡 **提示**: 一般用户无需直接操作数据库,通过管理后台界面即可完成所有配置。
