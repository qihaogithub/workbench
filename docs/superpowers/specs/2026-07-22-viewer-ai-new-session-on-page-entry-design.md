# 浏览端 AI 页面进入会话生命周期设计

## 背景

浏览端项目详情页当前会在 `ViewerAiPanel` 挂载时从项目级 localStorage 自动恢复最近一次对话。目标行为是：每次重新进入或刷新项目详情页后，首次打开 AI 都从新对话开始；同一次页面停留期间，收起并重新打开 AI 时继续当前对话。

## 目标行为

- 项目详情页每次挂载时创建新的空白 AI 会话，不自动选择最近历史会话。
- AI 抽屉在同一次页面挂载期间只切换显示状态，不重置会话 ID 或消息。
- 已完成对话继续按项目保存到 localStorage，并可从历史记录中手动选择、恢复或删除。
- 用户点击「新对话」时继续沿用现有逻辑：创建新会话、清空消息并销毁旧 Agent 会话。
- 模型偏好、附件能力、只读权限和 Agent 服务协议保持不变。

## 实现设计

`ViewerAiPanel` 的 `projectId` effect 仍读取项目历史列表，用于填充历史弹窗；但不再把最近一条历史写入当前 `sessionId` 和 `messages`。面板由父组件使用 `projectId` 作为 React key，进入或切换项目时通过重新挂载生成新的 viewer 会话 ID、空消息和创建时间。

`ViewerApp` 继续常驻挂载 `ViewerAiPanel`，`open` 仅控制侧栏显示。因此同页收起再打开不会触发初始化 effect，也不会丢失当前 React state。

本地历史服务不修改。当前新会话只有产生消息后才会写入历史，避免空白会话污染历史列表；用户从历史弹窗选择旧会话时，现有受控消息与会话 ID 切换逻辑继续生效。

## 失败与边界

- 无历史记录时与有历史记录时都以新空白会话开始。
- 刷新页面视为新的详情页挂载，因此也新建会话。
- 路由切换到另一项目时，按新项目生成新会话并加载该项目的历史列表。
- 历史记录仍保存在当前浏览器，不跨设备同步。

## 验证

- 组件级验证：挂载时即使 localStorage 有历史，当前消息仍为空且当前会话 ID 为新值。
- 组件级验证：抽屉从打开切到关闭再切回打开，当前会话和消息保持不变。
- 组件级验证：历史列表仍展示旧会话，选择后可恢复对应消息。
- 工程验证：运行 viewer-site 类型检查与构建检查；若引入适配现有测试环境的测试，则运行对应测试。
