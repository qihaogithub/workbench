# 问题排查与清理上下文

## 目标

定期运行检查、归类失败、维护问题状态，并清理已处理、过期或重复的记录。目标是减少重复报告，让下一次定时任务能从当前结论继续，而不是重新读一遍历史。

## 问题状态

| 状态 | 含义 | 处理 |
|:-----|:-----|:-----|
| 待确认 | 有失败现象，未判断类别 | 保留摘要，下一轮优先复查 |
| 已确认 | 是真实缺陷、测试缺口或脚本问题 | 建立或更新进行中文档 |
| 修复中 | 有明确方案并正在处理 | 更新任务清单和进度 |
| 待验证 | 已修复但验证不足 | 补验证命令和结果 |
| 已关闭 | 已修复、已验证或不再成立 | 归档或从当前状态移除 |
| 已过期 | 依赖的功能、脚本或现场不存在 | 标记原因后清理待办 |

## 失败分类

| 类别 | 判断 | 下一步 |
|:-----|:-----|:-------|
| 环境缺失 | 服务未启动、浏览器缺失、依赖缺失、端口不可用 | 记录前置条件，不改业务代码 |
| 测试不稳定 | 重跑结果不一致、等待策略脆弱、清理不完整 | 进入测试工具治理 |
| 脚本过期 | 路径、参数、默认数据或文档漂移 | 更新或停用脚本 |
| 代码回归 | 稳定复现且影响业务行为 | 创建修复计划 |
| 文档漂移 | 文档与代码或脚本行为不一致 | 更新对应项目文档 |
| 外部依赖异常 | LLM、网络、OSS、生产服务异常 | 记录外部状态，不自动修复 |

如果现象命中 `diagnostics/` 中的诊断包，优先按诊断包执行，不要自由探索。

## 记录位置

| 问题 | 位置 |
|:-----|:-----|
| 当前仍需处理的问题 | `docs/plans/进行中/` |
| 已处理的问题 | `docs/plans/已完成/` 合理分类 |
| CLI 能力缺口 | `docs/plans/进行中/CLI与创作端能力对齐长期跟踪.md` |
| 测试工具治理缺口 | `OPS/automations/state/test-tools-current.md` 或专项计划 |
| 自动任务当前结论 | `OPS/automations/state/issue-triage-current.md` |
| 可复用业务规则 | `docs/项目文档/` 对应模块 |

## 清理规则

每周检查：

- 已验证通过的问题是否仍留在进行中。
- 同一问题是否有多个平行记录。
- 过期脚本或历史测试记录是否仍被当前任务引用。
- `state/` 是否包含不再成立的结论。
- 已完成记录是否缺少验证结果。

清理不是删除证据。有效根因和验证证据应归档，过期待办和重复入口可以移除。

## 停机条件

必须停止并升级：

- 需要选择产品行为或验收标准。
- 涉及权限、鉴权、管理后台、发布、回滚、删除或真实数据清理。
- 同一失败连续多次自动修复仍失败。
- 需要密钥、真实外部服务或生产权限。
- 无法判断用户未提交改动是否与目标冲突。

## 输出要求

每次排查输出：

- 本次结论。
- 失败命令或检查项。
- 分类依据。
- 已排除项。
- 影响范围。
- 下一步动作。
- 需要人工确认的问题。
