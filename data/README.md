# data 目录诊断说明

> 状态：运行时数据与诊断现场
> 适用场景：判断问题来自系统逻辑、发布产物、会话状态、截图缓存还是本地数据。

## 定位

`data/` 不是源码目录，也不是固定测试 fixtures。它保存本地运行时产生的项目、会话、工作区、发布产物、截图、日志和数据库文件。Codex 排查问题时可以按需读取这里的数据，但不应在日常结构分析中把它当成代码入口。

## 使用原则

- 只读优先：排查时先读取和比对，不直接删除或覆盖。
- 按问题进入子目录：不要为了理解代码结构而全量扫描 `data/`。
- 注意隐私：`users.db`、会话、工作区和日志可能包含本地账号、配置、对话或业务内容。
- 不当 fixtures：当前运行数据会随操作变化，不应直接作为稳定测试样例。
- 需要复现样例时，应复制脱敏后的最小数据到专门 fixtures，而不是依赖当前 `data/` 状态。

## 子目录用途

| 路径 | 用途 | 常见排查问题 |
|---|---|---|
| `projects/` | 项目源数据、workspace、页面文件、元信息 | 项目保存失败、页面丢失、配置不一致 |
| `sessions/` | 创作会话元数据 | 会话过期、保存状态异常、编辑状态判断 |
| `workspaces/` | Agent 或编辑过程的工作区 | AI 修改未同步、工作区泄漏、文件变更丢失 |
| `snapshots/` | 项目或会话快照 | 版本回退、快照缺失、恢复异常 |
| `screenshots/` | 截图服务缓存和输出 | 预览截图过期、截图尺寸或缓存命中问题 |
| `published/` | 发布后的 viewer 数据 | 使用端看不到内容、发布产物不完整 |
| `templates/` | 项目模板数据 | 模板创建、复制、发布入口异常 |
| `knowledge/` | 知识库运行数据 | 知识读取、索引、报告生成问题 |
| `agent-run-logs/` | Agent 运行日志 | 工具调用、模型响应、工作流中断排查 |
| `logs/` | 服务运行日志 | 服务错误、请求失败、运行时异常 |
| `audit/` | 项目管理或 CLI 审计记录 | CLI 操作追踪、批量变更确认 |
| `.project-admin/` | 项目管理内部状态 | Project Admin CLI 状态排查 |
| `users.db*` | 本地用户数据库及 SQLite WAL/SHM | 登录、注册、权限、用户状态问题 |

## Codex 排查建议

1. 先从用户描述和代码入口判断涉及模块。
2. 只读取相关子目录的少量文件，例如某个 `projectId`、`sessionId` 或 `workspaceId`。
3. 对比代码期望的数据结构与实际 JSON/文件内容。
4. 如果确认是数据问题，说明原因和建议操作；不要默认修改或删除数据。
5. 如果确认是系统问题，再进入源码和测试修复。
